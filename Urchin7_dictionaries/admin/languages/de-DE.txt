184	<nobr><span class=title1><b>Globale Einstellungen: Automatische Aktualisierung (Ref. #A-2704)</b></span></nobr><p><b>Versionen<br></b>Über die Funktion für die automatische Aktualisierung können Sie die folgenden Urchin-Komponenten unabhängig voneinander aktualisieren:<ul><li>Urchin - Enthält alle Komponenten mit Ausnahme der Geodatenbanken (ca. 50 MB).</li><li>Hilfeartikel - Enthält die eingebettete Hilfe und die Hilfeartikel (ca. 5 MB pro Sprache).</li><li>Geodatenbanken - Enthält alle Geodatenbanken (ca. 200 MB).</li></ul>Verfügbare Aktionen:<ul><li>Jetzt überprüfen - Überprüft, ob Updates verfügbar sind.</li><li>Liste der Änderungen anzeigen - Zeigt die Liste der Änderungen für eine neue Version an.</li><li>Download jetzt planen - Startet den Download einer neuen Version.</li><li>Abbrechen - Bricht den Download ab.</li><li>Jetzt installieren - Startet den Installationsvorgang.</li><li>Fehlerprotokoll anzeigen - Zeigt die Fehler an.</li></ul></p><p><b>Aktualisierung planen<br></b>Mit diesen Steuerelementen können Sie festlegen, ob und wann die Komponenten aktualisiert werden. Die Komponenten werden standardmäßig täglich aktualisiert. Ist diese Option aktiviert, lädt Urchin jeden Tag zur festgelegten Uhrzeit automatisch Aktualisierungen herunter. Beachten Sie, dass die Aktualisierungen für die Geodatenbank relativ groß sind (ca. 200 MB), wodurch der Download die Netzwerkbandbreite beeinträchtigen kann.</p><p><b>HTTP-Proxy-Einstellungen (optional)<br></b>Geben Sie in diesen Feldern den Namen und den Port des Proxy-Servers sowie Nutzername und Passwort ein, falls der ausgehende Zugriff auf den Downloadserver von Google, der die Aktualisierungen der Geodaten hostet, einen Proxy erfordert.</p><p><b>Aktualisierungseinstellungen<br></b>Falls diese Option aktiviert ist, werden Hilfeartikel oder Geodatenbanken zum konfigurierten Zeitpunkt im stillen Modus aktualisiert, ohne dass Nutzer eingreifen müssen.</p>
185	<nobr><span class=title1><b>Profile: Speicherkapazität/Datenbank (Ref. #A-2108)</b></span></nobr><br>In diesem Fenster können Sie Datenverwaltungsoptionen festlegen. Für fehlertolerante Vorgänge stellt Urchin automatische Sicherungs- und Wiederherstellungsoptionen bereit. Durch diese Optionen wird ein sicherer Betrieb gewährleistet. Urchin bietet außerdem die Option zur automatischen Archivierung, mit der alte Daten automatisch komprimiert werden. <p><b>Datenbankspeicherauslastung</b><br>Bei normalem Betrieb verwendet Urchin ein Cachesystem, um den vom Verarbeitungsmodul belegten RAM zu begrenzen. Eventuell lässt sich die Leistung verbessern, wenn der gesamte Prozess im Arbeitsspeicher ausgeführt wird. Diese Option sollte jedoch nur auf eigenständigen Urchin-Systemen verwendet werden. Beachten Sie dabei auch, dass der Prozess unter Umständen den gesamten verfügbaren RAM im System belegt. Im Allgemeinen können Sie die Einstellung "Mit Cachesystem begrenzen" beibehalten.<p><b>Datenbanktabellenlimit</b><br>Um die Größe der Daten und die entsprechende Leistung von Urchin bei großen Websites im Griff zu behalten, wird durch diese Einstellung jede Tabelle in der Urchin-Datenbank auf die Anzahl der aufgeführten Datensätze beschränkt. Die Standardoption ist für die meisten Installationen geeignet. Falls das Limit innerhalb eines bestimmten Monats erreicht wird, werden zusätzliche Einträge unter "Sonstige" zusammengefasst. Legen Sie höchstens einen Wert von 100.000 fest, um eine optimale Leistung zu erzielen. <p><b>Unformatierte Tracking-Daten beibehalten</b><br>Falls diese Option deaktiviert ist, werden unformatierte Daten über Besucher und Sitzungen nach der Verarbeitung gelöscht. Dadurch kann die Protokollverarbeitungsleistung deutlich verbessert und die gespeicherte Datenmenge für große Websites reduziert werden. Sobald unformatierte Sitzungsdaten gelöscht wurden, werden zusätzliche Treffer für diese Sitzung als neue Sitzung behandelt, was zu einem geringfügigen Randeffekt im Bericht führen kann <p><b>Protokoll-Tracking</b><br>Urchin ist mit einer "intelligenten" Protokoll-Tracking-Funktion ausgestattet, mit der ermittelt wird, ob eine vorhandene Protokolldatei bereits verarbeitet wurde oder nicht. So können Sie Protokolle wechseln, wenn es sich anbietet, anstatt nach einem festen Zeitplan vorzugehen. Außerdem haben Sie die Möglichkeit, Urchin lediglich auf ein Verzeichnis mit Protokolldateien auszurichten, wobei es unerheblich ist, ob einige Dateien bereits verarbeitet wurden. WICHTIGER HINWEIS: Falls Urchin sich nicht auf dem System befindet, auf dem die Protokolldateien gespeichert sind, werden alle Protokolle vollständig auf das lokale Urchin-System kopiert, bevor ermittelt wird, ob die Protokolle bereits verarbeitet wurden. Hierdurch wird die Netzauslastung deutlich erhöht, wenn große Protokolldateien analysiert werden müssen.<p><b>Autom. Rollback für Datenbank</b><br>Bei aktivierter Option werden Daten automatisch repariert, wenn der Verarbeitungsvorgang angehalten oder unterbrochen wurde. Urchin erkennt diese Situation automatisch und führt einen Rollback der Daten bis zur letzten Sicherung (sofern vorhanden) durch, bevor der Vorgang fortgesetzt wird.<p><b>Sicherungen erstellen</b><br>Diese Option ermöglicht automatische Sicherungen nach der Verarbeitung. Sicherungen werden komprimiert, um Speicherplatz zu sparen.<p><b>Sicherungen bereinigen</b><br> Diese Option ermöglicht die automatische Bereinigung alter Sicherungen gemäß der Anzahl der beizubehaltenden Sicherungen, die über die folgende Option festgelegt wurde. Falls diese Option aktiviert ist, löscht Urchin automatisch alte Sicherungen, um Speicher freizugeben. <p><b>Anzahl der beizubehaltenden Sicherungen</b><br> Mit dieser Option geben Sie an, wie viele Sicherungen pro Monat im Rahmen der oben genannten Bereinigungsfunktion gespeichert bleiben.<p><b>Datenbank archivieren</b><br> Diese Option ermöglicht die automatische Archivierung alter Monatsdaten. Alte Daten werden automatisch in ein einzelnes Archiv komprimiert, Sicherungen werden entfernt und die Daten gesperrt. Bei der Berichterstellung können archivierte Daten zwar nach wie vor angezeigt werden, für die archivierten Monate werden jedoch keine Treffer mehr verarbeitet.<p><b>Datenbank archivieren nach</b><br>Mit dieser Option wird angegeben, ab welchem Alter (in Monaten) Daten automatisch archiviert werden.<p><b>ALLE Daten für Profil löschen</b><br>Mit dieser Option werden alle Daten für dieses Profil, einschließlich aller Sicherungen und Archive, gelöscht. Für das entsprechende Profil müssen dann alle Protokolldateien neu verarbeitet werden. <b>Warnung</b>: Vergewissern Sie sich, dass alle Protokolldateien vorliegen, bevor Sie diese Daten löschen.

192	<nobr><span class=title1><b>Planer: Aufgabenstatus (Ref. #A-2664)</b></span></nobr><br>In diesem Fenster wird der Status der derzeit ausgeführten Aufgabe angezeigt. Es bietet zudem Steuerelemente zum Abbrechen der ausgeführten Aufgabe. Oben im Fenster wird eine Statusanzeige angezeigt, die Aufschluss darüber gibt, wie viel Prozent der Verarbeitung in etwa abgeschlossen sind. Mithilfe der Schaltfläche “Aufgabe abbrechen” halten Sie die derzeit ausgeführte Aufgabe an und führen einen Rollback des Profils durch. Oberhalb der Statusanzeige befindet sich ein anklickbarer Link zum Berichterstellungsfenster für das Profil. Die Ausführungsausgabe des Protokollverarbeitungsmoduls wird im Fenster “Aufgabenverarbeitungsdetails” angezeigt. <p>Hinweis: Bei Verarbeitung einer großen Remote-Protokolldatei dauert das Kopieren unter Umständen sehr lange. In dieser Zeit ändert sich die Statusanzeige nicht. Sobald die Datei auf den Computer kopiert wurde, auf dem Urchin ausgeführt wird, wird die Statusanzeige relativ kontinuierlich aktualisiert.
193	<nobr><span class=title1><b>Planer: Protokolldetails (Ref. #A-2563)</b></span></nobr><br>Dieses Fenster dient dazu, die Details einer Aufgabenausführung anzuzeigen, um ein Debugging oder eine Leistungsbewertung vorzunehmen. Falls ein Fehler aufgetreten ist, erhalten Sie in diesem Fenster Hinweise zur Behebung des Problems. <p><b>Häufig auftretende Probleme</b><br>Zu den häufig auftretenden Problemen gehören Fehler wie "Protokolldatei nicht gefunden", die einfach durch Korrigieren des Pfad- oder Dateinamens behoben werden können. Wenn Sie Protokolldateien über FTP im Netzwerk verschieben, stellen Sie sicher, dass Nutzername und Passwort korrekt sind, indem Sie sich mit einem entsprechenden FTP-Programm beim Server anmelden. Wenn die Datenverarbeitungszeit nur einen Bruchteil der gesamten Laufzeit ausmacht, ist dies vermutlich auf ein Problem mit einer Protokolldatei zurückzuführen, die innerhalb des Netzwerks kopiert wird. Erwägen Sie in diesem Fall, Urchin direkt auf dem Server zu installieren.

448	<nobr><span class=title1><b>Filtermanager: Filtereinstellungen (Ref. #A-1701)</b></span></nobr><br>In diesem Fenster können Sie die Einstellungen für einen Filter konfigurieren und anpassen. Weitere Informationen zum Filtern erhalten Sie im Handbuch oder in unserem Online-Dokumentationscenter.<p><b>Beispiele</b><br>Falls Sie einen neuen Filter hinzufügen, wird eine Liste mit Beispielfiltern angezeigt. Durch Auswahl eines Beispielfilters werden die Einstellungen für den Filter automatisch vorgenommen und können anschließend nach Bedarf bearbeitet werden. Dies zeigt anschaulich, wie die Einstellungen für verschiedene, häufig verwendete Filter vorgenommen werden.<p><b>Optionales Konto</b> (nur Datacenter-Modus)<br>Wählen Sie für neue Filter das Konto aus, mit dem der Filter verknüpft werden soll. Für vorhandene Filter wird der Name des Kontos angezeigt, mit dem der Filter verknüpft ist.<p><b>Filtername</b><br>Dieses erforderliche Feld enthält einen frei wählbaren Namen, mit dem auf diesen Filter verwiesen werden kann. Geben Sie einen für diesen Filter eindeutigen Namen ein.<p><b>Filterart</b><br>Wählen Sie je nach gewünschter Funktion des Filters eine der folgenden Filterarten aus: <li><b>Muster ausschließen</b>: Mit dieser Filterart werden <i>nur</i> Protokolldateizeilen (Treffer) ausgeschlossen, die mit dem Filtermuster übereinstimmen. Die übereinstimmenden Zeilen werden vollständig ignoriert. Beachten Sie daher, dass mit einem Filter, der beispielsweise Netscape ausschließt, alle anderen Informationen in der entsprechenden Protokollzeile, wie Besucher, Pfad, Verweis und Domain-Informationen, ebenfalls ausgeschlossen werden. <li><b>Muster einschließen</b>: Mit diesem Filter werden <i>nur</i> Protokolldateizeilen (Treffer) eingeschlossen, die mit dem Filtermuster übereinstimmen. Alle anderen Zeilen werden ignoriert. Wie im Beispiel oben bereits erwähnt, stehen alle Treffer, die ignoriert werden, dem Urchin-System nicht zur Berichterstellung zur Verfügung. <li><b>Suchen und Ersetzen</b>: Hierbei handelt es sich um einen einfachen Filter, der für die Suche nach einem Muster innerhalb eines Feldes und für den Austausch dieses Musters durch ein alternatives Format verwendet werden kann. Falls das Muster mehr als einmal in dem entsprechenden Feld vorkommt, wird nur das letzte Muster ersetzt. Für die Übereinstimmung kann am Anfang des Feldes das Zeichen "^" und am Ende des Feldes das Zeichen "$" verwendet werden. <li><b>Suchtabelle</b>: Dieser Filter ermöglicht Ihnen die Auswahl einer Suchtabelle, die für die Zuordnung von Codes zu verständlichem Text verwendet werden kann. Zunächst muss eine entsprechende Suchtabellendatei erstellt und im Verzeichnis "lib/custom/lookuptables" der Urchin-Distribution installiert werden (intern) oder sich auf dem lokalen Dateisystem bzw. auf einer NFS- oder UNC- (nur Windows) Freigabe befinden. FTP/HTTP-Quellen werden nicht unterstützt.<li><b>Erweitert</b>: Mit diesem Filter können Sie ein Feld aus einem oder zwei anderen Feldern erstellen. Das Filtermodul wendet die Ausdrücke in den beiden Extraktionsfeldern auf die angegebenen Felder an und konstruiert anschließend ein Feld mit dem Konstruktorausdruck. Das Verfahren entspricht der Verwendung regulärer POSIX-Ausdrücke. <p><b>Filterfeld</b><br>Wählen Sie über das Dropdown-Menü das Feld in der Protokollquelle aus, auf das der Filter angewendet werden soll. Felder mit der Bezeichnung "RAW" stellen den eigentlichen Inhalt der Protokolldatei ohne Verarbeitung dar. Mit "AUTO" beschriftete Felder sind automatisch berechnete Felder, die auf der Basis von RAW- und anderen AUTO-Feldern ermittelt werden. Eine vollständige Liste der Funktion aller Felder erhalten Sie im Abschnitt zum Filtern in unserem Online-Dokumentationscenter. <p><b>Filtermuster</b><br>Legen Sie diesen Wert auf das Muster fest, mit dem der Filter übereinstimmen soll. Für das Muster können Platzhalter verwendet werden, die vom regulären POSIX-Ausdrucksstandard definiert werden.<p><b>Suchzeichenfolge</b><br>Für Suchen- und Ersetzen-Filter wird die Suchzeichenfolge als Muster verwendet, mit dem das Feld übereinstimmen muss.<p><b>Ersetzungszeichenfolge</b><br>Für Suchen- und Ersetzen-Filter wird die Ersetzungszeichenfolge als Ersatz für die Suchzeichenfolge verwendet.<p><b>Tabellenname</b> (nur Suchtabelle)<br>Dieses Feld, das bei der Filterart "Suchtabelle" angezeigt wird, ermöglicht Ihnen die Auswahl einer internen Suchtabelle, die für die Zuordnung von Codes zu verständlichen Namen verwendet wird. Die Telefonmodelltabelle ordnet abgekürzte Telefonplattformbezeichner dem Modell- und Herstellernamen für telefonbasierte Webbrowser zu. Sie können eigene Suchtabellen erstellen, um Kampagnen für das Kampagnen-Tracking-Modul Codes zuzuordnen oder um Variablen in einer URL Seitennamen zuzuordnen. In dieser Dropdown-Liste werden alle Dateien angezeigt, die Sie mit der Dateierweiterung ".lt" im Verzeichnis "lib/custom/lookuptables" der Urchin-Installation erstellen.<p><b>Dateiname</b> (nur externe Suchtabelle)<br>In diesem Feld, das bei ausgewählter Filterart "Suchtabelle" angezeigt wird, können Sie eine externe Suchtabellendatei auswählen, die Codes verständlichem Text zuordnet. Sie können Ihre eigenen Suchtabellen erstellen, um Codes Kampagnen (für das Kampagnen-Tracking-Modul) zuzuordnen oder um Variablen in einer URL Seitennamen zuzuordnen.<p><b>Nutzername/Passwort</b> (Windows, nur externe Suchtabelle)<br>Über diese Felder können Sie die Nutzeranmeldeinformationen für die UNC-Freigabe, auf der sich die externe Suchtabelle befindet, angeben.<p><b>Steuerelemente</b><br>Klicken Sie für einen neuen Filter auf die Schaltfläche "Fertig stellen" bzw. für einen vorhandenen Filter auf die Schaltfläche "Aktualisieren", um Ihre Änderungen zu speichern. Für vorhandene Filter können Sie über die Tabs oben im Fenster auf alle bearbeitbaren Einstellungen für den entsprechenden Filter zugreifen.

Urchin wird mit mehreren Geodatenbanken ausgeliefert. In jeder Datenbank sind die Informationen zu Land/Region unterschiedlich detailliert aufgeführt. Wählen Sie aus dem Dropdown-Menü die Geodatenbank aus, die Ihren Anforderungen am besten entspricht.<p><b>Geodatenbank</b><br>Urchin wird mit mehreren Geodatenbanken ausgeliefert, die unterschiedlich detaillierte Informationen zu geografischen Standorten enthalten:<br>- World (detaillierte Informationen auf Länderebene sowie zu Regionen und Städten für die ganze Welt)<br>- World Lightweight (Informationen auf Länderebene für die ganze Welt)<br>- Regionale Datenbanken, etwa für die USA, die Europäische Union oder Japan (detaillierte Informationen zu entsprechenden Informationen und Länderinformationen für die ganze Welt) <br><br>Die Geodatenbank-Einstellungen können global oder auf Profilebene definiert werden.<p><b>Steuerelemente</b><br>Klicken Sie auf die Schaltfläche "Aktualisieren", um Ihre Änderungen zu speichern. Geben Sie den vollständigen Domain-Namen einschließlich "http://" ein. <p><b>Website-Domains</b> (nur Superadministrator)<br>Diese Einstellung führt alle möglichen Domains für Ihre Website mit und ohne "www" auf. Anhand dieses Feldes wird ermittelt, ob ein Verweis extern ist. Verwenden Sie Kommas zum Trennen von Domain-Einträgen. <p><b>Standardseite</b><br> Hier sollte die Seite angegeben werden, die der Standardseite oder der Indexseite der Website-Verzeichnisse entspricht. Diese ist normalerweise "index.html", auf Windows IIS-Servern jedoch häufig auch "Default.htm" oder "index.htm". Anhand dieser Information kann Urchin Protokolleinträge wie "http://www.urchin.com/" und "http://www.urchin.com/index.html" abgleichen, bei denen es sich eigentlich um dieselbe Seite handelt. Wird die Standardseite nicht korrekt eingegeben, werden diese beiden Seiten als zwei verschiedene Seiten behandelt. Es sollte nur eine Standardseite angegeben werden. <p><b>Proxy-Einstellungen</b> (nur Superadministrator)<br>Bei einigen Installationen sind eventuell zum Anzeigen der Overlay-Berichte für die Startseite Proxy-Einstellungen erforderlich. Da die Overlay-Abfragen vom Urchin-System ausgehen, kann es vorkommen, dass die Firewall-Einstellungen keine externe Kommunikation zulassen. In diesem Fall werden diese Abfragen über einen Proxy ausgeführt. Der Proxy-Server sollte der Hostname des Servers sein. Der Proxy-Port sollte die Portnummer für die Kommunikation mit dem Proxy sein. Die Felder für den Proxy-Nutzernamen und das Passwort dienen der zusätzlichen Sicherheit, falls für den Zugriff auf den Proxy ein Nutzername und ein Passwort erforderlich sind. Lassen Sie alle Felder leer, es sei denn, Sie erhalten eine Fehlermeldung, wenn Sie versuchen, die Overlay-Funktion zu verwenden. Bitten Sie in diesem Fall Ihren Netzwerkadministrator um die Einstellungen.<p><b>Parallele Protokollverarbeitung</b> (nur Superadministrator)<br>Falls Sie mit mehreren großen Protokollen mit Lastausgleich arbeiten, können Sie die Leistung eventuell durch eine parallele Verarbeitung verbessern. Die Leistungsverbesserung ist jedoch von einer Reihe Faktoren abhängig, unter anderem der Größe der Daten und der E/A-Leistung des Computers. Aktivieren Sie diese Einstellung zur Leistungsoptimierung vor allem, wenn Treffer aus einer Nutzersitzung in mehreren Protokollen vorhanden sein können. <p><b>Zeitverschiebung</b> (nur Superadministrator)<br>Diese Einstellung bestimmt, in welcher Zeitzone Urchin die Daten in die Berichte aufnimmt. Da alle Treffer Verweise auf GMT (Greenwich Mean Time) enthalten, kann Urchin Berichte in jeder Zeitzone erstellen. Bei der Option "Ortszeit" werden Daten der Zeitzone entsprechend aufgezeichnet, die auf dem Server, auf dem Urchin ausgeführt wird, konfiguriert ist. Falls Sie eine andere Einstellung verwenden möchten, wählen Sie den Zeitversatz gegenüber GMT. <p><b>Organische Begriffsübereinstimmung</b> (nur Superadministrator)<br>Diese Einstellung steuert, wie Urchin Suchmaschinen und Suchbegriffe für die Berichte erkennt. Anhand dieser durch Kommas getrennten Liste sucht Urchin nach passenden Parametern in Suchanfragen auf verweisenden Websites. Urchin erfasst dann den Wert des Parameters als Keywords oder Suchbegriffe, die in dem Verweis verwendet werden. Fügen Sie anhand dieser Liste Einträge hinzu, um Suchbegriffe zu erfassen, die möglicherweise nicht automatisch eingeschlossen werden. Die Standardliste ist für die meisten Suchmaschinen geeignet.<p><b>Seitenaufruf-/MIME-Übereinstimmung</b> (nur Superadministrator)<br> Diese durch Kommas getrennte Liste steuert, welche Dateierweiterungen als Seitenaufrufe behandelt werden. Die Option kann so eingestellt werden, dass bestimmte Erweiterungen ein- oder ausgeschlossen werden. Die Standardliste eignet sich für die meisten Websites.<p><b>Übereinstimmung der Seitenparameter</b><br>Diese Einstellung steuert, welche Parameter Urchin im Parameterbericht erfasst bzw. auslässt. Die Parameter stammen vom Suchanfrageteil der angeforderten URL und können interne Suchbegriffe oder anderen dynamischen Content enthalten. Urchin erfasst diese Namen-Wert-Paare automatisch und nimmt sie in den Parameterbericht auf. In dieser durch Kommas getrennten Liste können bestimmte Parameter ein- oder ausgeschlossen werden. Für eine Website mit Millionen von willkürlichen Sitzungs-IDs wäre es beispielsweise sinnvoll, die SID-Variable aus dem Bericht auszuschließen, um Speicherressourcen zu sparen.<p><b>Downloads-Übereinstimmung</b> (nur Superadministrator)<br> Diese durch Kommas getrennte Liste steuert, welche Dateierweiterungen im Download-Bericht als heruntergeladene Dateien angesehen werden. Bearbeiten Sie die Liste nach Bedarf, um für Downloads verwendete Dateierweiterungen einzuschließen.<p><b>Geodetails</b><br><p><b>Geoinformationen</b><br>Mit dieser Einstellung können Sie steuern, welche Geoinformationen die Profilberichte enthalten sollen. Durch die Auswahl zusätzlicher Optionen wird der Speicher-Footprint für das Protokollverarbeitungsmodul folgendermaßen erweitert: je nach ausgewählter Geodatenbank etwa 350 MB für Netzwerke, 100 MB für Domains und 20 bis 150 MB für geografische Standorte. Wählen Sie aus dem Dropdown-Menü die Geodatenbank aus, die Ihren Anforderungen am besten entspricht.<p><b>Geodatenbank</b><br>Urchin wird mit mehreren Geodatenbanken ausgeliefert, die unterschiedlich detaillierte Informationen zu geografischen Standorten enthalten:<br>- World (detaillierte Informationen auf Länderebene sowie zu Regionen und Städten für die ganze Welt)<br>- World Lightweight (Informationen auf Länderebene für die ganze Welt)<br>- Regionale Datenbanken, etwa für die USA, die Europäische Union oder Japan (detaillierte Informationen zu entsprechenden Informationen und Länderinformationen für die ganze Welt)<br> <p><b>Zeichencodierung</b><br>Falls die zu verarbeitenden Protokolle Daten enthalten, die nicht in UTF-8 codiert sind, wählen Sie das korrekte Codierungsformat für die Daten aus. Systemeigene japanische und Latin-1-Formate werden unterstützt. <p><b>Währung und Genauigkeit</b><br>Legen Sie bei E-Commerce-Websites über die Einstellungen für die Währung und die Genauigkeit die Standardwährungssymbole, die Standardformatierung der Anzeige und die numerische Genauigkeit fest. <p><b>Steuerelemente:</b><br>Klicken Sie zum Speichern der Änderungen auf "Aktualisieren". 575	<nobr><span class=title1><b>Protokollmanager: Remote-Einstellungen (Ref. #A-1602)</b></span></nobr><br>In diesem Fenster können Sie festlegen, wie der Aufruf und der Zugriff auf die Remote-Protokolldatei für diese Protokollquelle erfolgen soll.<p><b>Serverprotokoll</b><br>Urchin kann vier verschiedene Arten von Remote-Protokolldateien verarbeiten: HTTP (www), HTTPS, FTP und UNC (nur Windows). Wenn der Zugriff auf Ihre Protokolldatei über einen Webbrowser erfolgen kann, verwenden Sie HTTP. Wenn kein Passwort erforderlich ist, lassen Sie die Felder "Nutzername" und "Passwort" leer. FTP kann mit einem Nutzernamen und einem Passwort verwendet werden, wobei die Daten in der Regel dem Nutzernamen zum Hochladen von Content auf Ihre Website entsprechen. Bei UNC, das nur unter Windows verfügbar ist, sind ein Nutzername und ein Passwort für den Zugriff auf die UNC-Freigabe erforderlich.<p><b>Servername/IP</b><br>Geben Sie den Servernamen (z. B. www.urchin.com) oder die IP-Adresse (z. B. 63.212.171.4) für den Speicherort der Protokolldatei ein. Verwenden Sie für UNC das Format "\Server\Freigabe".<p><b>Anmeldung lokal und remote ist identisch</b><br>Diese Option wird angezeigt, wenn UNC als Serverprotokoll ausgewählt wurde (nur unter Windows). Wählen Sie "Ja", wenn für den lokalen und den Remote-Server derselbe Anmeldename und dasselbe Passwort verwendet werden kann. Wählen Sie "Nein", wenn für den Remote-Server ein anderer Anmeldename oder ein anderes Passwort als für den lokalen Server erforderlich ist. Wenn Sie "Nein" wählen, werden zusätzliche Felder für die Anmeldung beim lokalen Server angezeigt.<p><b>Serverport</b><br>Der Serverport wird automatisch eingetragen. Falls Sie jedoch nicht den Standardport verwenden, können Sie diesen Port manuell bearbeiten.<p><b>Nutzername/Passwort</b><br>Diese Felder bleiben bei nicht authentifizierten HTTP- und anonymen FTP-Verbindungen leer. Zum Verwenden einer authentifizierten HTTP-/FTP-Verbindung geben Sie Ihren Nutzernamen und Ihr Passwort hier ein. Bei UNC sind für den Zugriff auf die Freigabe ein Nutzername und ein Passwort erforderlich. <p><b>Domain (nur UNC)</b><br>Wenn die UNC-Freigabe zu einer Domain gehört, geben Sie hier die Domain-Informationen ein. <p><b>FTP-Modus</b><br>Diese Option wird angezeigt, wenn FTP als Serverprotokoll ausgewählt wurde. Wählen Sie "Passiv" für die passive (PASV) Übertragungsmethode. Wählen Sie "Aktiv" für die direkte (PORT) Übertragungsmethode. <p><b>Remote-Protokollquellendatei</b><br>Die Option "Remote-Protokollquellendatei" gibt den Pfad zur Protokolldatei auf dem Remote-System an. Wenn Sie HTTP verwenden, ist es häufig einfacher, die Protokolldatei zunächst mit einem Webbrowser zu suchen. Geben Sie dazu im Browser die URL der Website gefolgt von einem Schrägstrich, allen verwendeten Verzeichnisnamen und dem Namen der Protokolldatei ein, z. B. http://www.urchin.com/logs/access_log. Wenn im Browserfenster daraufhin Text angezeigt wird, war der Pfad richtig. Geben Sie alle Informationen rechts von der Domain an, z. B. /logs/access_log. Beachten Sie bitte, dass Platzhalter bei HTTP-basierten Remote-Protokollquellenpfadnamen nicht unterstützt werden. Falls Sie FTP verwenden, können Sie auf dem Remote-Server nach der Protokolldatei suchen. Vor der Suche müssen Sie die Informationen für den Server und Nutzernamen konfigurieren.<p><b>Protokollziel</b> (nur remote)<br>Vor der Verarbeitung werden Remote-Protokolldateien auf das lokale System übertragen. Wählen Sie ein lokales Verzeichnis zum Speichern der Protokolldatei aus. Zur Erleichterung der Suche wird die Schaltfläche "Durchsuchen" zur Verfügung gestellt.<p><b>Proxy-Server</b> (nur HTTP und HTTPS)<br>Wenn das Protokoll über einen Proxy-Computer verfügbar ist, geben Sie die IP-Adresse oder den Hostnamen an. <p><b>Proxy-Port</b> (nur HTTP und HTTPS)<br>Wenn das Protokoll über einen Proxy-Computer verfügbar ist, geben Sie den Port an.<p><b>Proxy-Nutzername</b> und <b>Proxy-Passwort</b><br>Wenn das Protokoll über einen Proxy-Computer verfügbar ist, für den Nutzername und Passwort erforderlich sind, geben Sie die entsprechenden Daten in diese Felder ein.

Search Marketing-Konto, auf das zugegriffen wird (erforderlich).<li><b>Master-Kontonummer</b>: Entspricht der Master-Kontonummer des Anzeigenkunden (erforderlich).<li><b>Kontonummer</b>: Entspricht der Nummer des Yahoo! Search Marketing-Kontos (erforderlich).<li><b>Lizenzschlüssel</b>: Hierbei handelt es sich um den Lizenzschlüssel für den Nutzer, der sich gerade anmeldet, den Kunden oder die Agentur (erforderlich). <li><b>“Im Auftrag von”-Nutzername</b>: Entspricht dem “Im Auftrag von”-Nutzernamen des Kunden. Er ist nur erforderlich, wenn der Agenturnutzername anstatt des Kundennutzernamens angegeben wurde (optional).<li><b>“Im Auftrag von”-Passwort</b>: Hierbei handelt es sich um das “Im Auftrag von”-Passwort des Kunden (optional). <p><b>Download des CPC-Berichts aktiviert</b><br> Diese Option ist nur während der Bearbeitung einer CPC-Quelle sichtbar. Unter Umständen entstehen dem Kunden für jeden API-Abruf zur Prüfung der Verfügbarkeit Kosten (eine Einheit im Fall von AdWords). Daher überprüft Urchin die Berichtverfügbarkeit anhand eines konfigurierbaren Intervalls zwischen zwei Phasen. <br><br>Mit dieser Option kann der Nutzer das Intervall zwischen der Berichtplanung und der tatsächlichen Überprüfungs- bzw.
