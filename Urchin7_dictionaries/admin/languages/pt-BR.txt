184	<nobr><span class=title1><b>Configurações globais: atualização automática (Ref. #A-2704)</b></span></nobr><p><b>Versões<br></b>A funcionalidade Atualização automática permite atualizar de forma independente os seguintes componentes do Urchin:<ul><li>Urchin - contém todos os componentes, menos os bancos de dados geográficos (aproximadamente 50 MB)</li><li>Artigos de ajuda - contém os a ajuda incorporada e os artigos da Central de Ajuda (aproximadamente 5 MB por idioma)</li><li>Bancos de dados geográficos - contém todos os bancos de dados geográficos (aproximadamente 200 MB)</li></ul>Ações disponíveis:<ul><li>verificar agora - verificar se há atualizações</li><li>visualizar lista de alterações - visualizar a lista de alterações de uma nova versão</li><li>programar download agora - iniciar o download de uma nova versão</li><li>cancelar - cancelar o download</li><li>instalar agora - iniciar o processo de instalação</li><li>visualizar registro de erros - visualizar erros</li></ul></p><p><b>Programar atualização<br></b>Use esses controles para determinar se, e quando, os componentes serão atualizados. Por padrão, os componentes são atualizados diariamente. Se esse componente estiver ativado, o Urchin fará download automaticamente das atualizações todos os dias no horário configurado. As atualizações do banco de dados geográficos são relativamente grandes (aproximadamente 200 MB). Desse modo, o download pode causar um impacto na largura de banda da rede.</p><p><b>Configurações de proxy HTTP (opcional)<br></b>Se o acesso externo ao servidor de downloads do Google que hospeda as atualizações dos dados geográficos requer o uso de um proxy, use esses campos para especificar o nome e a porta, além do nome de usuário e da senha, do servidor proxy.</p><p><b>Configurações de atualização<br></b>Se estiverem ativados, os artigos de ajuda ou os bancos de dados geográficos serão atualizados de acordo com o horário configurado em segundo plano, sem a interferência de usuários.</p>
185	<nobr><span class=title1><b>Perfis: armazenamento/banco de dados (Ref. #A-2108)</b></span></nobr><br>Essa tela permite definir opções de gerenciamento de dados. Para as operações tolerantes a falhas, o Urchin fornece as opções automáticas de backup e recuperação. Use essas opções para garantir uma operação segura. O Urchin também fornece uma opção de arquivamento automático, que pode compactar automaticamente os dados antigos. <p><b>Uso da memória do banco de dados</b><br>Em operação normal, o Urchin usa um sistema de cache para limitar a quantidade de memória RAM que será usada pelo mecanismo de processamento. Pode haver um aumento no desempenho para a execução do processo inteiro na memória, mas isso deve ser usado somente em sistemas Urchin dedicados. Além disso, o processo pode consumir toda a memória RAM disponível no sistema. Para a maioria das operações, deixe esse recurso definido como “Limitar com sistema de cache”.<p><b>Limite por tabela do banco de dados</b><br>Para manter o volume de dados e o desempenho relacionado do Urchin controlado em sites grandes, essa configuração limitará toda tabela do banco de dados do Urchin conforme o número de registros listados. A opção padrão é apropriada para a maior parte das instalações. Em determinado mês, se esse limite for atingido, as entradas adicionais serão agrupadas em uma entrada denominada “Outros”. Para obter um desempenho ideal, use valores menores ou iguais a 100.000. <p><b>Manter dados de acompanhamento brutos</b><br>Essa opção, quando desativada, exclui os visitantes brutos e informações da sessão após o processamento. Isso pode melhorar significativamente o desempenho do processamento de registros e reduzir o volume de dados armazenados em sites grandes. Depois que os dados da sessão bruta forem excluídos, todos os acessos adicionais àquela sessão serão tratados como uma nova sessão, o que pode causar um efeito pequeno de margem no relatório. <p><b>Acompanhamento do registro</b><br>O Urchin está equipado com um recurso de controle do registro “inteligente” que permite determinar se um determinado arquivo de registro já foi processado ou não. Isso permite que você faça uma rotação dos registros sempre que for conveniente, e não de acordo com uma programação rígida. Também permite que você aponte o Urchin para um diretório de arquivos de registro sem se preocupar se alguns arquivos já foram processados. OBSERVAÇÃO IMPORTANTE: Se o Urchin não estiver instalado no sistema que armazena os arquivos de registro, ele copiará todos os registros para o sistema local do Urchin antes de determinar se os registros já foram processados. Isso causará uma sobrecarga significativa da rede, se houver arquivos de registro grandes sendo analisados.<p><b>Banco de dados de repetição automática</b><br>Quando está ativada, essa opção pode reparar os dados automaticamente, se o processamento for interrompido ou encerrado. O Urchin detectará automaticamente essa condição e repetirá os dados para o backup mais recente (presumindo que há um) antes de continuar.<p><b>Criar backups</b><br>Essa opção permite a criação automática de backups após o processamento. Os backups são compactados para o armazenamento.<p><b>Limpar backups</b><br> Essa opção permite a limpeza automática de backups antigos de acordo com o número desejado de backups que deve ser mantido na próxima opção. Se ela estiver ativada, o Urchin excluirá automaticamente os backups antigos, mantendo o armazenamento controlado. <p><b>Número de backups</b><br> Essa opção especifica quantos backups por mês devem ser mantidos para a função de limpeza mencionada acima.<p><b>Arquivar banco de dados</b><br> Essa opção permite o arquivamento automático dos dados mensais antigos. Os dados antigos são compactados automaticamente em um único arquivo, os backups são removidos e os dados são bloqueados. Os relatórios ainda podem exibir os dados arquivados, mas nenhum acesso daqueles meses que foram arquivados pode ser processado.<p><b>Arquivar banco de dados mais tarde</b><br>Essa opção especifica há quantos meses os dados antigos foram criados antes do arquivamento automático.<p><b>Excluir TODOS os dados do perfil</b><br>Use essa ação para excluir permanentemente TODOS os dados desse perfil, incluindo TODOS os backups e os arquivos. Será necessário reprocessar todos os arquivos de registro desse perfil. <b>Aviso</b>: Verifique se você tem todos os arquivos de registro antes de excluir esses dados.

192	<nobr><span class=title1><b>Programador: progresso da tarefa (Ref. #A-2664)</b></span></nobr><br>Essa tela mostra o progresso da tarefa que está em execução no momento. Ela também fornece controles para cancelar a tarefa em execução. Uma barra de progresso é exibida na parte superior da janela e mostra a porcentagem aproximada de processamento concluído. Use o botão “Cancelar tarefa” para interromper a tarefa em execução e restabelecer o perfil. Há um link clicável para o relatório do perfil acima da barra de progresso. A saída do tempo de execução do mecanismo de processamento do registro é exibida na janela “Detalhe do processo da tarefa”. <p>Observação: Se um arquivo de registro remoto grande estiver sendo processado, poderá levar muito tempo para copiar os dados. Durante esse tempo, a barra de progresso não se moverá. Depois que o arquivo for copiado para o computador que estiver executando o Urchin, a barra de progresso deverá ser atualizada de forma mais ou menos contínua.
193	<nobr><span class=title1><b>Programador: detalhe do histórico (Ref. #A-2563)</b></span></nobr><br>O objetivo dessa tela é mostrar o detalhe da execução de uma tarefa, geralmente para fins de avaliação de depuração e desempenho. Se houver um erro, essa tela vai fornecer a indicação de uma solução. <p><b>Problemas comuns</b><br>Os problemas comuns incluem erros de “arquivo de registro não encontrado”, que geralmente podem ser solucionados facilmente através da correção de um caminho ou nome de arquivo. Se o FTP estiver sendo utilizado para mover os arquivos de registro pela rede, verifique se o nome de usuário e senha estão corretos, fazendo login no servidor com um programa de FTP dedicado. Se o tempo de “data proc” (processamento de dados) for apenas uma pequena fração do tempo de execução total, provavelmente há um problema com a cópia do arquivo de registro pela rede. Convém instalar o Urchin diretamente no servidor.

448	<nobr><span class=title1><b>Gerenciador de filtro: configurações de filtro (Ref. #A-1701)</b></span></nobr><br>Essa tela permite definir e ajustar as configurações de um filtro. Para obter mais informações sobre filtragem, consulte o manual ou a Central de documentação on-line.<p><b>Exemplos</b><br>Ao adicionar um novo filtro, uma lista de filtros de exemplo será fornecida para sua conveniência. Se você selecionar um filtro de exemplo, as configurações do filtro serão preenchidas automaticamente que, em seguida, devem ser editadas para fins específicos. Essa pode ser uma maneira fácil de ver como as configurações são criadas para vários filtros comuns.<p><b>Conta opcional</b> (somente modo Datacenter)<br>Para novos filtros, permite selecionar a conta à qual o filtro deve ser associado. Para os filtros existentes, o nome da conta à qual o filtro pertence é exibido.<p><b>Nome do filtro</b><br>Esse campo obrigatório é um nome arbitrário que é utilizado por conveniência ao fazer referência a esse filtro. Insira algum dado que seja exclusivo do filtro.<p><b>Tipo de filtro</b><br>Selecione um dos tipos de Filtro, de acordo com a função que você quer que o filtro faça: <li><b>Excluir padrão</b>: Esse tipo de filtro excluirá <i>somente</i> as linhas do arquivo de registro (acessos) que correspondem ao Termo de filtragem. As linhas que não correspondem ao padrão serão completamente ignoradas. Desse modo, um filtro que exclui o Netscape, por exemplo, fará com que o Urchin exclua todas as outras informações daquela linha do registro, como visitante, caminho, referência e informações sobre o domínio. <li><b>Incluir padrão</b>: Esse tipo de filtro incluirá <i>somente</i> as linhas do arquivo do registro (acessos) que correspondem ao Termo de filtragem -- todas as outras serão ignoradas. Conforme o exemplo acima, todos os acessos que são ignorados tornam-se completamente indisponíveis para o sistema do Urchin do ponto de vista do relatório. <li><b>Pesquisar e substituir</b>: Esse é um filtro simples que pode ser utilizado para pesquisar um padrão dentro de um campo e substituir o padrão encontrado por uma forma alternativa. Se o padrão ocorrer mais de uma vez no campo, somente a última ocorrência será substituída. O caractere “^” pode ser utilizado para corresponder o início do campo e o caractere $ pode ser utilizado para vincular a correspondência ao final do campo. <li><b>Tabela de consulta</b>: A escolha desse filtro permite que você selecione um nome da tabela de consulta que pode ser usada para mapear os códigos para marcadores compreensíveis. Um arquivo de tabela de consulta correspondente deve ser criado e instalado no diretório lib/custom/lookuptables da distribuição do Urchin. As tabelas de consulta existentes podem ser selecionadas no campo "Nome da tabela".<li><b>Avançado</b>: Esse tipo de filtro permite que você crie um campo usando um ou dois outros campos. O mecanismo de filtragem aplicará as expressões dos dois campos "Extrair" nos campos especificados e, em seguida, construirá um campo utilizando a expressão “Construtor”. A técnica segue o uso da expressão regular POSIX. <p><b>Filtrar campo</b><br>Utilize o menu suspenso para selecionar o campo "Origem do registro" ao qual o filtro será aplicado. Os campos marcados como "BRUTO" representam o conteúdo real do arquivo de registro sem nenhum processamento. Os campos marcados como "AUTOMÁTICO" são campos calculados automaticamente que são determinados com base nos campos "BRUTO" e "AUTOMÁTICO". Para obter uma lista de como todos os campos são utilizados, consulte a seção da "Central de documentação on-line. <p><b>Padrão de filtro</b><br>Defina essa opção para o padrão ao qual o filtro deve corresponder. O padrão pode utilizar curingas que são definidos pela expressão regular padrão POSIX.<p><b>String de pesquisa</b><br>Para os filtros Pesquisar e substituir, a String de pesquisa é utilizada como o padrão para corresponder ao campo.<p><b>String de substituição</b><br>Para os filtros Pesquisar e substituir, a “String de substituição” é utilizada como a substituição da String de pesquisa.<p><b>Nome da tabela</b> (somente Tabela de consulta)<br>Esse campo, que fica visível quando você seleciona a Tabela de consulta como o tipo de filtro, permite a seleção da tabela que mapeia os códigos para nomes que podem ser compreendidos pelo ser humano. A tabela de modelos de telefone mapeia os identificadores da plataforma do telefone abreviada para o modelo e os nomes do fabricante dos navegadores on-line de telefone. Você pode criar suas próprias tabelas de consulta para mapear os códigos para campanhas (para o Módulo de acompanhamento de campanhas) ou para mapear variáveis em um URL para os nomes de página. Qualquer arquivo que você criar com uma extensão de arquivo .lt no diretório lib/custom/lookuptables da distribuição do Urchin aparecerá nesse menu suspenso.<p><b>Nome do arquivo</b> (Somente Tabela de consulta externa)<br>Esse campo, visível quando você seleciona a Tabela de consulta como o tipo de filtro, permite a seleção de um arquivo de Tabela de consulta externa, que mapeia os códigos para nomes que podem ser compreendidos pelo ser humano. Você pode criar suas próprias tabelas de consulta para mapear os códigos para campanhas (para o Módulo de acompanhamento de campanhas) ou para mapear variáveis em um URL para os nomes de página.<p><b>Nome de usuário/senha</b> (Windows, somente Tabela de consulta externa)<br>Esses campos são usados para especificar as credenciais do usuário para o compartilhamento UNC no qual a Tabela de consulta externa está localizada.<p><b>Controles</b><br>Clique no botão "Concluir" (para um novo filtro) ou no botão "Atualizar" (para um filtro existente) para salvar as alterações. Para filtros existentes, as guias na parte superior da tela permitem acessar todas as configurações editáveis desse filtro.

470	<nobr><span class=title1><b>Assistente de origem do registro: Opções (Ref. #A-470)</b></span></nobr><br>Essa tela permite que você especifique uma origem do registro local (por exemplo, no computador em que o Urchin está instalado) ou uma origem do registro remota (por exemplo, em outro computador). 471	<nobr><span class=title1><b>Assistente da origem do registro: configurações remotas (Ref. #A-471)</b></span></nobr><br>Essa tela permite especificar a forma de recuperar ou acessar o arquivo de registro remoto dessa origem de registro.<p><b>Protocolo de servidor</b><br>O Urchin pode gerenciar quatro tipos de arquivos de registro remoto: HTTP (www), HTTPS, FTP e UNC (somente Windows). Se seu arquivo de registro puder ser acessado através de um navegador da web, use HTTP. Se não for necessário utilizar senha, deixe os campos “nome de usuário” e “senha” em branco. O FTP pode ser usado com um nome de usuário e uma senha (geralmente os mesmos usados para enviar conteúdo para seu site). O UNC, que está disponível apenas nos sistemas Windows, exige um nome de usuário e uma senha para acessar o compartilhamento UNC.<p><b>Nome do servidor/IP</b><br>Insira o nome do servidor (por exemplo, www.urchin.com) ou o endereço IP (por exemplo, 63.212.171.4) em que se encontra o arquivo de registro. Para o UNC, use a notação \\server\share.<p><b>Porta do servidor</b><br>A porta do servidor será preenchida automaticamente. No entanto, se você estiver usando uma porta que não seja a padrão, poderá editar esse campo manualmente.<p><b>Mesmo login local e remoto</b> (somente UNC)<br>Essa opção ficará visível se o UNC tiver sido selecionado como Protocolo do servidor (somente Windows). Selecione “Sim” se os mesmos login e senha puderem ser usados para os servidores remoto e local. Selecione “Não” se o servidor remoto exigir login ou senha diferentes daqueles usados no servidor local. Se você selecionar “Não”, outros campos do login do servidor local ficarão visíveis.<p><b>Nome de usuário/senha</b><br>Esses campos podem ser deixados em branco para HTTP não autenticado e conexões de FTP anônimas. Para usar uma conexão HTTP/FTP autenticada, insira o nome de usuário e senha aqui. O UNC exige um nome de usuário e uma senha para acessar o compartilhamento. <p><b>Domínio</b> (somente UNC)<br>Se o compartilhamento UNC fizer parte de um domínio, insira as informações de rede do domínio aqui. <p><b>Modo FTP</b> (somente FTP)<br>Essa opção ficará visível se o FTP tiver sido selecionado como Protocolo de servidor. Selecione “Passivo” para o método de transferência passiva (PASV). Selecione “Ativo” para o método de transferência direta (PORT).

473	<nobr><span class=title1><b>Assistente de origem do registro: configurações (Ref. #A-473)</b></span></nobr><br>Essa tela permite especificar o formato ao arquivo de registro que está sendo processado. O tipo de formato de registro <i>automático</i> deve ser escolhido para a maioria dos servidores da web. Outros tipos de formato de registro de servidor da web são fornecidos para compatibilidade com versões anteriores do Urchin, mas normalmente não são necessários. Formatos de registro personalizados podem ser criados com os modelos do diretório lib/custom/logformats da distribuição do Urchin.<p><b>Formato do registro</b><br>Use a opção de formato do registro para especificar o formato dessa origem do registro. <ul><li><b>Automático:</b> Esse formato detectará automaticamente a maioria dos formatos de servidor da web, incluindo Apache, W3C (Microsoft IIS), NCSA, Netscape e iPlanet. <li><b>Campanha:</b> Use esse formato para dados de custo da campanha especificamente formatados para uso do Urchin. O formato do registro do Overture não pode ser detectado automaticamente.<li><b>W3C:</b> Esse formato pode ser detectado automaticamente, e é usado para servidores da web que fazem login no formato W3C padrão, como servidores Microsoft IIS.</ul>

A porta do proxy deve ser o número da porta para comunicação com o proxy. 876	<b>Perfis: metas</b><p>Edite uma meta clicando na guia correspondente na parte superior da tela. Em seguida, clique em <img alt='Analysis Options Button' title='magic wand' src ='/uicons/default/wizard.gif' border=''> na parte superior direita para ativar o Assistente de configuração de metas. Use o navegador da web incorporado ao assistente para navegar até a página que deseja definir como etapa de funil ou meta. Para selecionar a página como a etapa 1, clique em <b>Definir página como etapa: 1</b>. Repita o processo para todas as etapas do funil. Clique em <b>Definir página como meta</b> para definir a página como a meta e fechar o assistente. As páginas selecionadas serão exibidas nos campos "Endereço da web de etapas e meta". Não é necessário incluir a parte <i>http://www.domain.com</i> do URL no campo "Endereço da web da meta" ou "Endereço da web" em nenhuma etapa. O Urchin irá ignorar essa parte do URL.<p>IMPORTANTE: O Urchin permite usar expressões regulares em todos os campos de endereço da web para etapas e metas. Coloque todas as expressões regulares entre metacaracteres após a última barra ("/"), isto é, caracteres como "?" ou "." no endereço da web devem ser colocados entre barras invertidas ("\") se desejar que o Urchin os interprete literalmente. <p><b>Configurações de meta</b><br>O Urchin contará cada visita ao URL especificado em <b>Endereço da web da meta</b> como uma conversão de meta sempre que a opção <b>Ativar meta</b> estiver selecionada. O nome fornecido em <b>Nome da meta</b> será exibido nos relatórios do Urchin. O valor fornecido em <b>Valor da meta (receita)</b> será usado para calcular o retorno de investimento e outras métricas de avaliação.<p><b>Configurações de funil de meta</b><br>Especifique as etapas que levam à meta nesta seção. Cada etapa é uma página no caminho para a conversão da meta. Juntas, as etapas representam um funil de conversão. O Urchin mostra quantos visitantes concluem cada etapa e quantos deixam o funil em cada etapa no relatório Otimização do conteúdo-->Processo de funil e metas-->Abandono definido do funil. Para cada etapa, forneça um URL e um nome para a página. <p><b>Configurações avançadas de meta</b><br>Ao relatar conversões de meta e conclusões de etapa, o Urchin compara os URLs fornecidos com os URLs visitados. Quando <b>Opção de endereço da web</b> está definido como "URL sem consulta", o Urchin ignora a parte de consulta dos URLs nessas comparações. Se você definir <b>Diferenciar maiúsculas de minúsculas</b> como "Sim", essas comparações irão diferenciar maiúsculas de minúsculas. Definir <b>Etapa obrigatória 1</b> como "Sim" indica que somente os caminhos de conversão que incluem a primeira etapa devem ser incluídos nos relatórios de funil.
