184	<nobr><span class=title1><b>Configuración global: actualización automática (Ref. #A-2704)</b></span></nobr><p><b>Versiones<br></b>La funcionalidad de actualización automática permite actualizar de forma independiente estos componentes de Urchin:<ul><li>Urchin: contiene todos los componentes, menos las bases de datos geográficas (aproximadamente 50 MB).</li><li>Artículos de ayuda: contiene la ayuda insertada y los artículos del Centro de asistencia (aproximadamente 5 MB por cada idioma).</li><li>Bases de datos geográficas: contiene todas las bases de datos geográficas (aproximadamente 200 MB).</li></ul>Acciones disponibles:<ul><li>comprobar ahora: buscar actualizaciones</li><li>ver lista de cambios: ver la lista de cambios de una nueva versión</li><li>programar descarga ahora: iniciar la descarga de una nueva versión</li><li>cancelar: cancelar la descarga</li><li>instalar ahora: iniciar el proceso de instalación</li><li>ver registro de errores: ver los errores</li></ul></p><p><b>Programar actualización<br></b>Utilice estos controles para modificar si se modifican los componentes y cuándo se actualizan. De forma predeterminada, los componentes se actualizan diariamente. Si se habilita, cada día a la hora configurada, Urchin descargará las actualizaciones automáticamente. Tenga en cuenta que las actualizaciones de la base de datos geográfica son relativamente grande (aproximadamente 200 MB), por lo que la descarga puede afectar al ancho de banda de la red.</p><p><b>Configuración proxy HTTP (opcional)<br></b>Si el acceso de salida al servidor de descargas de Google que aloja las actualizaciones de datos geográficos requiere el uso de un proxy, utilice estos campos para especificar el nombre y el puerto del servidor proxy, así como el nombre de usuario y la contraseña.</p><p><b>Actualizar configuración<br></b>Si se habilita esta opción, los artículos de ayuda o las bases de datos geográficas se actualizarán según la hora configurada en un modo silencioso sin intervención del usuario.</p>
185	<nobr><span class=title1><b>Perfiles: almacenamiento de datos/base de datos (Ref. #A-2108)</b></span></nobr><br>Esta pantalla le permite configurar opciones de administración de datos. Para las operaciones tolerantes a errores, Urchin proporciona opciones de copia de seguridad y recuperación automáticas. Utilícelas para garantizar un funcionamiento seguro. Urchin también proporciona una opción de archivado automático que comprime datos antiguos. <p><b>Utilización de la memoria de la base de datos</b><br>En condiciones normales, Urchin utiliza un sistema de caché para determinar cuánta memoria RAM emplea el motor de procesamiento. Es posible que aumente el rendimiento para poder ejecutar todo el proceso en la memoria, pero tenga en cuenta que esto solo debe utilizarse en sistemas Urchin dedicados y que el proceso puede requerir toda la memoria RAM disponible en el sistema. Para la mayoría de las operaciones, seleccione la opción "Limitar con el sistema de la caché".<p><b>Límite de la tabla de la base de datos</b><br>Para controlar el tamaño de los datos y el rendimiento relativo de Urchin en sitios de gran tamaño, esta configuración restringirá las tablas de las bases de datos de Urchin al número de registros indicado. La opción predeterminada es apropiada para la mayoría de las instalaciones. Si se alcanza este límite en un mes determinado, las entradas adicionales se acumularán en una entrada llamada "Otras". Para obtener un rendimiento óptimo, utilice valores inferiores o iguales a 100.000. <p><b>Conservar datos de seguimiento sin procesar</b><br>Cuando se desactiva, esta opción permite suprimir la información de usuario y sesión sin procesar tras haberla procesado. Esto puede mejorar considerablemente el rendimiento del procesamiento de registros, así como reducir la cantidad de datos almacenados en sitios de gran tamaño. Una vez que se haya suprimido la información de la sesión sin procesar, los resultados adicionales de esta se tratarán como una nueva sesión, lo que podrá repercutir ligeramente en los informes.  <p><b>Seguimiento de registros</b><br>Urchin incorpora una utilidad de seguimiento de registros "inteligente" que permite determinar si ya se ha procesado o no un archivo de registro concreto. Ello hace posible alternar los registros siempre que sea preciso, en lugar de ceñirse a una programación estricta. También permite apuntar a un directorio de archivos de registro independientemente de si ya se han procesado algunos. IMPORTANTE: si Urchin no se encuentra en el sistema en el que se almacenan los archivos de registro, copiará todos los registros en el sistema local de Urchin antes de determinar si estos se han procesado ya. Si hay muchos archivos analizándose, se producirá una sobrecarga considerable de la red.<p><b>Restauración automática de la base de datos</b><br>Cuando esta opción se encuentra habilitada, permite reparar automáticamente los datos si el procesamiento se detuvo o si se eliminó una parte de estos. Urchin detectará de forma automática tal situación y recuperará los datos de la copia de seguridad más reciente, en caso de que la haya.<p><b>Crear copias de seguridad</b><br>Esta opción permite la creación automática de copias de seguridad tras el procesamiento. Las copias de seguridad se comprimen para su almacenamiento compacto.<p><b>Limpiar copias de seguridad</b><br> Esta opción habilita la limpieza automática de copias de seguridad antiguas en función del número de copias que deben guardarse según se haya especificado en la opción siguiente. Si se habilita la configuración, Urchin suprimirá de manera automática las copias de seguridad antiguas.  <p><b>Copias de seguridad que deben conservarse</b><br> Esta opción determina cuántas copias de seguridad se conservarán al mes para la función de limpieza mencionada anteriormente.<p><b>Archivar base de datos</b><br> Esta opción habilita el archivado automático de los datos mensuales antiguos. La información antigua se comprime automáticamente en un único archivo, las copias de seguridad se eliminan y la información se protege. Los informes permiten ver la información archivada, pero no se procesarán más resultados de los meses archivados.<p><b>Archivar bases de datos después de</b><br>Esta opción determina la antigüedad de la información antes de iniciar el archivo de manera automática.<p><b>Suprimir TODA la información del perfil</b><br>Utilice esta acción para eliminar de manera definitiva TODA la información del perfil en cuestión, incluidas TODAS las copias de seguridad y los archivos. Deberá volver a procesar todos los archivos de registro del perfil. <b>Advertencia</b>: verifique que tiene todos los archivos de registro antes de suprimir estos datos.

192	<nobr><span class=title1><b>Programador: progreso de la tarea (Ref. #A-2664)</b></span></nobr><br>Esta pantalla muestra el progreso de la tarea en ejecución y ofrece controles para cancelar la tarea en curso. Se muestra una barra de progreso en la parte superior de la ventana en la que se indica el porcentaje aproximado del proceso completado. Utilice el botón “Cancelar tarea” para detener la tarea y recuperar el perfil. Encima de esta, se incluye un enlace que remite a la función de informes del perfil. La información sobre el tiempo de ejecución del motor de procesamiento de registros se muestra en la ventana “Detalles del procesamiento de tareas”. <p>Nota: si se procesa un archivo de registro remoto de gran tamaño, el tiempo de copiado será considerable. Durante la copia, la barra de progreso no se moverá. Una vez que el archivo se haya copiado en el equipo que ejecuta Urchin, la barra se actualizará de manera más o menos continuada.
193	<nobr><span class=title1><b>Programador: detalles del historial (Ref. #A-2563)</b></span></nobr><br>El objetivo de esta pantalla es mostrar la información detallada de la ejecución de una tarea, habitualmente con el fin de evaluar el rendimiento o de depurar errores. Si se ha producido un error, la pantalla proporcionará algunas indicaciones acerca de cómo solucionarlo. <p><b>Problemas habituales</b><br>Entre los problemas más frecuentes se incluyen los errores del tipo "el archivo de registro no se encuentra", que habitualmente se solucionan con facilidad corrigiendo el nombre de ruta o archivo. Si se utiliza FTP para transferir archivos de registro por la red, compruebe que el nombre de usuario y la contraseña sean correctos. Para ello, acceda al servidor con un programa especial para FTP. Si el tiempo de "proc. datos" (procesamiento de datos) es únicamente una pequeña fracción del valor de "Tiempo de ejecución", probablemente se haya producido un error al copiar el archivo del registro en la red. Le recomendamos que instale Urchin directamente en el servidor.
194	<nobr><span class=title1><b>Examinar archivos (Ref. #A-5000)</b></span></nobr><br>En esta pantalla podrá examinar el sistema de archivos local para encontrar el origen del archivo de registro que busca. Es especialmente útil cuando no se recuerda con exactitud el nombre del archivo de registro. El cuadro “Ir al directorio”, situado a la izquierda, se utiliza para seleccionar un directorio de navegación. Los archivos del directorio aparecerán en el cuadro “Seleccionar archivo”, a la derecha. Una vez que haya encontrado el archivo de registro deseado, haga clic en él en el cuadro “Seleccionar archivo” y, a continuación, haga clic en “Aceptar”.<p><b>Directorio actual</b><br>El campo “Directorio actual” de la parte superior de la ventana se puede modificar directamente. Pulse Intro para ver el contenido del directorio tras la modificación.<p><b>Subir un directorio</b><br>Al hacer clic en el icono correspondiente (icono de carpeta con una flecha) subirá del directorio actual al directorio principal. Aparecerá el contenido del nuevo directorio.<p><b>Ir al directorio</b><br>La ventana de la izquierda permite seleccionar un directorio en el directorio actual. Haga clic una vez en uno de los directorios que aparecen en la ventana “Ir al directorio” para cambiar de directorio.<p><b>Seleccionar archivo</b><br>La ventana “Seleccionar archivo”, situada a la derecha, le permitirá seleccionar un archivo. Si hace clic una vez en un determinado archivo, aparecerá su nombre en el campo “Nombre de archivo”. Cuando haya seleccionado el archivo, haga clic en el botón “Aceptar”.<p><b>Aceptar/Verificar/Cancelar</b><br>El botón “Aceptar” cerrará la ventana y rellenará el campo de ruta de la ventana anterior con lo que haya seleccionado. El botón “Cancelar” permite cerrar la ventana. Con el botón “Verificar” podrá abrir una ventana nueva para comprobar los permisos y el tamaño de los archivos en cuestión si utiliza un comodín en “Nombre de archivo”.

448	<nobr><span class=title1><b>Administrador de filtros: configuración del filtro (Ref. #A-1701)</b></span></nobr><br>En esta pantalla podrá realizar las configuraciones y ajustes de un filtro determinado. Si desea obtener más información acerca de los filtros, consulte el manual o nuestro Centro de documentación online.<p><b>Ejemplos</b><br>Cuando se añade un filtro nuevo, se facilita una lista de filtros de ejemplo. Si selecciona un filtro de ejemplo, se rellenarán todas las configuraciones que, posteriormente, podrá editar según le convenga. Esta puede ser una forma sencilla de ver cómo se crean las configuraciones para varios filtros comunes.<p><b>Cuenta opcional</b> (modo de centro de datos únicamente)<br>Para nuevos filtros, le permite seleccionar la cuenta a la que debe asociarse el filtro. Para los filtros existentes, se muestra el nombre de la cuenta a la que pertenece el filtro.<p><b>Nombre del filtro</b><br>Este campo obligatorio es un nombre arbitrario que se utiliza para hacer referencia al filtro. Introduzca un nombre exclusivo.<p><b>Tipo de filtro</b><br>En función de lo que desee que realice el filtro, seleccione uno de los tipos de filtro: <li><b>Excluir patrón</b>: este tipo de filtro excluirá <i>únicamente</i> líneas del registro (resultados) que concuerden con el patrón del filtro. Las líneas que coincidan se omitirán. Por lo tanto, no olvide que un filtro que excluye, por ejemplo, Netscape, hará que Urchin excluya el resto de la información presente en esa línea del registro como, por ejemplo, el usuario, la ruta, la URL de referencia y la información relativa al dominio. <li><b>Incluir patrón</b>: este tipo de filtro incluirá <i>únicamente</i> las líneas del registro (resultados) que concuerden con el patrón del filtro; se omitirán todas las demás. Al igual que en el ejemplo anterior, los resultados que se omiten no podrán utilizarse para la creación de informes en el sistema de Urchin. <li><b>Buscar y reemplazar</b>: este es un filtro sencillo que se puede utilizar para buscar un patrón dentro de un campo y reemplazarlo por una forma alternativa. Si el patrón aparece más de una vez en el campo, únicamente se modifica la última vez que aparece. El carácter "^" puede emplearse para la concordancia al principio del campo, y el carácter "$" puede utilizarse para marcar la concordancia hasta el final del campo. <li><b>Tabla de consulta</b>: si selecciona este filtro, podrá elegir el nombre de una tabla de consulta, que se puede utilizar para asignar códigos a etiquetas inteligibles para el usuario. Se debe crear primero el archivo para la tabla e instalarlo en el directorio "lib/custom/lookuptables" de la distribución de Urchin (interno) o en el sistema de archivos local, NFS o un recurso compartido UNC (solo Windows). No se admiten las fuentes FTP/HTTP.<li><b>Avanzado</b>: este tipo de filtro permite crear un campo a partir de uno o dos campos distintos. El motor del filtro aplicará las expresiones de los dos campos "Extraer" a los campos especificados y creará un campo utilizando la expresión "Constructor". La técnica sigue el uso de expresiones regulares POSIX. <p><b>Campo del filtro</b><br>Utilice el menú desplegable para seleccionar a qué campo del origen del archivo de registro debe aplicarse el filtro. Los campos etiquetados "RAW" representan el contenido real del archivo de registro sin procesamiento. Los campos etiquetados "AUTO" son campos calculados de manera automática que están determinados en función de "RAW" y otros campos "AUTO". Para obtener un listado completo de cuáles son los usos de todos los campos, consulte la sección de filtros del Centro de documentación online. <p><b>Patrón del filtro</b><br>Configúrelo según el patrón que deberá concordar con el filtro. El patrón puede utilizar comodines definidos por el estándar de expresiones regulares POSIX.<p><b>Cadena de búsqueda</b><br>Para los filtros de buscar y reemplazar, la cadena de búsqueda se emplea como el patrón con el que debe concordar el campo.<p><b>Reemplazar cadena</b><br>Para los filtros de buscar y reemplazar, la cadena de reemplazo se utiliza como sustituta de la cadena de búsqueda.<p><b>Nombre de tabla</b> (tabla de consulta solamente)<br>Este campo, visible cuando se selecciona la tabla de consulta como tipo de filtro, permite seleccionar una tabla que convierte códigos en nombres inteligibles. La tabla de modelos de telefonía asigna los identificadores de plataforma telefónica abreviados a los nombres de fabricantes y modelos de los navegadores web basados en telefonía. Puede crear sus propias tablas de consulta para asignar códigos a campañas (para el módulo de seguimiento de campañas), así como asignar variables de una URL a nombres de páginas. Cualquier archivo que cree con extensión de archivo .lt en el directorio "lib/custom/lookuptables" de la distribución de Urchin, aparecerá en la lista desplegable de tablas internas.<p><b>Nombre de archivo</b> (solo tabla de consulta externa)<br>Este campo, visible cuando se selecciona la tabla de consulta como el tipo de filtro, permite seleccionar la tabla de consulta como el tipo de filtro y también permite seleccionar un archivo de tabla de consulta externa, que asigna códigos a nombres inteligibles por el usuario. Puede crear su propia tabla de consulta para asignar códigos a campañas (para el módulo de seguimiento de campañas) o para asignar variables de una URL a nombres de página.<p><b>Nombre de usuario/contraseña</b> (Windows, solo tabla de consulta externa)<br>Estos campos se utilizan para especificar las credenciales de usuario para el recurso compartido UNC en el que se encuentra la tabla de consulta externa.<p><b>Controles</b><br>Haga clic en el botón "Finalizar" (para filtros nuevos) o en el botón "Actualizar" (para filtros existentes) para guardar los cambios. En el caso de los filtros existentes, las pestañas de la parte superior de la pantalla ofrecen acceso a todas las configuraciones del filtro en cuestión que se pueden editar.

Consulte la documentación online para obtener información más detallada. Tenga en cuenta que el formato de los registros de campaña no puede detectarse automáticamente.<li><b>Elf2:</b> use este formato para registros de comercio electrónico en formato ELF2. Tenga en cuenta que el formato de los registros de comercio electrónico no se puede detectar de manera automática.<li><b>Google:</b> utilice este formato para registros que contengan información sobre la inversión de coste por clic de Google. Tenga en cuenta que el formato de los registros de Google no se puede detectar de manera automática.<li><b>NCSA:</b> normalmente, este formato puede detectarse automáticamente y suelen usarlo los servidores web Apache. Tenga en cuenta que para la generación completa de informes debe utilizarse el registro combinado extendido, que en la mayoría de los casos no es el comportamiento predeterminado de Apache. <li><b>Netscape:</b> los servidores Netscape (distintos de iPlanet) heredados utilizan este formato de manera predeterminada.<li><b>Overture:</b> utilice este formato para registros que contengan información sobre la inversión de coste por clic de Overture. Tenga en cuenta que el formato de los registros de Overture no se puede detectar de manera automática.<li><b>W3C:</b> este formato puede detectarse automáticamente y se usa para los servidores web que acceden en formato W3C estándar, como los servidores Microsoft IIS.</ul>

569	<nobr><span class=title1><b>Perfiles: informes (Ref. #A-2106)</b></span></nobr><br>La pantalla de configuración de informes permite controlar diversas funciones de procesamiento de Urchin importantes, entre estas, la identificación de determinados filtros y configuraciones de proxy.<p><b>URL del sitio web</b><br>Esta configuración se utiliza para crear enlaces de referencia al sitio web real. La URL debe ser el nombre de dominio completo del sitio, incluido http://. <p><b>Dominios del sitio</b> (solo para superadministradores)<br>Esta configuración debe enumerar todos los dominios posibles del sitio web con y sin “www”. Este campo se utiliza para determinar si una referencia es externa al sitio. Utilice comas para separar cada una de las entradas de dominio.  <p><b>Página predeterminada</b><br> Deberá ajustarse según la página o el índice predeterminado de los directorios de este sitio. Normalmente, será “index.html”, pero en los servidores Windows IIS, suele ser “Default.htm” o “index.htm”. Esta información permite a Urchin conciliar las entradas de registro como “http://www.urchin.com/“ y “http://www.urchin.com/index.html”, que, de hecho, son la misma página. Si la información de la página predeterminada no se introduce correctamente, estas páginas se registrarán como dos distintas. Solo debe especificarse una página predeterminada. <p><b>Configuración del servidor proxy</b> (solo para superadministradores)<br>Algunas instalaciones pueden necesitar configuración de proxy para mostrar los informes de superposición de la página principal. Dado que las consultas de superposición se generan en el sistema de Urchin, es posible que la configuración del cortafuegos impida la comunicación externa. En ese caso, puede utilizar un proxy para poder realizar este tipo de consultas. El servidor proxy debe ser el nombre de host del servidor. El puerto de proxy debe ser el número de puerto que comunique con el proxy. Asimismo, los campos de nombre de usuario y de contraseña de proxy pueden garantizar una mayor seguridad en caso de que se requiera un nombre de usuario y una contraseña para acceder al proxy. Deje todos los campos en blanco, salvo si obtiene un error al utilizar la función de superposición. En ese caso, póngase en contacto con su administrador de red para que le indique la configuración adecuada.<p><b>Procesamiento de registros paralelo</b> (solo para superadministradores)<br>Si tiene una gran cantidad de registros de carga distribuida, generarlos en paralelo podría aumentar el rendimiento. Esto depende de muchos factores, tales como la cantidad de información y la capacidad de E/S del equipo. Se recomienda habilitar esta opción para mejorar el rendimiento, sobre todo si los resultados de una sesión de un solo usuario aparecen en varios registros.  <p><b>Compensación horaria</b> (solo para superadministradores)<br>Esta configuración determina en qué zona horaria registra Urchin los datos. Puesto que todos los resultados contienen referencias a GMT, Urchin puede registrar datos en cualquier zona horaria. La opción “Hora local” registrará datos en función de la zona horaria configurada en el servidor en el que se ejecute Urchin. Para utilizar una configuración distinta, seleccione la compensación desde GMT.  <p><b>Concordancia de palabras clave orgánicas</b> (solo para superadministradores)<br>Esta configuración controla cómo detecta Urchin los motores de búsqueda y las palabras clave para esos informes. A través de esta lista separada por comas, Urchin busca los parámetros en las consultas de los sitios de referencia que concuerdan. A continuación, Urchin recopila el valor del parámetro, como las palabras clave o términos de búsqueda utilizados en la URL indicada. Utilice esta lista para añadir entradas y así poder recopilar los términos de búsqueda que no se incluyan de manera automática. La lista predeterminada funciona para la mayoría de los motores de búsqueda.<p><b>Concordancia de tipo MIME de páginas vistas</b> (solo para superadministradores)<br> Esta lista separada por comas controla qué extensiones de archivo se tratan como páginas vistas. Puede configurar la opción para incluir o excluir determinadas extensiones. La lista predeterminada funciona para la mayoría de los sitios.<p><b>Concordancia de parámetros de página</b><br>Esta configuración controla los parámetros que Urchin recogerá en el informe de parámetros. Los parámetros provienen de la parte de la consulta de la URL solicitada y pueden contener términos de búsqueda interna u otro contenido dinámico. Urchin recopilará de manera automática estos pares de nombre-valor y los reflejará en el informe de parámetros. Esta lista separada por comas le permite excluir o incluir determinados parámetros. Por ejemplo, para un sitio con millones de ID de sesión aleatorios se recomienda excluir la variable SID del informe para ahorrar recursos de almacenamiento.<p><b>Concordancia de descargas</b> (solo para superadministradores)<br> Esta lista separada por comas controla qué extensiones de archivo se tratan como archivos descargados en el informe de descargas. Edite la lista según proceda para incluir extensiones de archivo utilizadas para las descargas.<p><b>Información geográfica detallada</b><br><p><b>Información geográfica</b><br>Esta configuración permite controlar qué datos geográficos deben incluirse en los informes de perfiles. Si se seleccionan otras opciones, aumenta el uso de memoria del motor de procesamiento de registros (aprox. 350 MB para redes, 100 MB para dominios, y entre 20 y 150 MB para ubicaciones geográficas, según la base de datos con información de este tipo que elija). Urchin incluye varias bases de datos con información geográfica, que varían en función de la cantidad de datos sobre la región o el país que se facilite. Elija la que mejor se adapte a sus necesidades en el menú desplegable.<p><b>Base de datos con información geográfica</b><br>Urchin incluye varias bases de datos con información geográfica que proporcionan diferentes niveles de información de ubicaciones geográficas:<br>- mundo (contiene información detallada de países, regiones y ciudades de todo el mundo),<br>- versión reducida de mundo (solo contiene información de países de todo el mundo),<br>- bases de datos regionales, como Estados Unidos y Japón, que contienen información detallada de las regiones correspondientes e información de país para el resto del mundo.<br> <p><b>Codificación de caracteres</b><br>Si los registros que va a procesar incluyen datos no codificados en UTF-8, seleccione el formato de codificación correcto para los datos. Se admiten formatos de japonés nativo y latín 1.  <p><b>Moneda y precisión</b><br>Para los sitios de comercio electrónico, utilice la configuración de moneda y precisión para cambiar los símbolos de moneda predeterminados, el formato de visualización y la precisión numérica.  <p><b>Controles:</b><br>Haga clic en el botón "Actualizar" para guardar los cambios.
