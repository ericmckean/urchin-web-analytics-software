184	<nobr><span class=title1><b>전체 설정: 자동 업데이트 (Ref. #A-2704)</b></span></nobr><p><b>버전<br></b>자동 업데이트 기능을 사용하면 다음과 같은 Urchin 구성요소를 개별적으로 업데이트할 수 있습니다. <ul><li>Urchin - 지리 데이터베이스를 제외한 모든 구성요소 포함(약 50MB) </li><li>도움말 기사 - Urchin과 함께 제공되는 도움말과 도움말 센터 기사 포함(언어당 약 5MB) </li><li>지리 데이터베이스 - 모든 지리 데이터베이스 포함(약 200MB) </li></ul>가능한 액션: <ul><li>지금 확인 - 업데이트가 있는지 확인 </li><li>변경 목록 보기 - 새 버전의 변경 목록 보기 </li><li>다운로드 지금 예약 - 새 버전 다운로드 시작 </li><li>취소 - 다운로드 취소 </li><li>지금 설치 - 설치 과정 시작 </li><li>오류 로그 보기 - 오류 보기 </li></ul></p><p><b>업데이트 예약<br></b>이 옵션을 통해 구성요소의 업데이트 여부와 시간을 변경할 수 있습니다. 기본적으로 구성요소는 매일 업데이트됩니다. 이 옵션이 활성화되면 매일 지정된 시간에 Urchin이 업데이트를 자동으로 다운로드합니다. 지리 데이터베이스 업데이트 크기가 비교적 크기 때문에(약 200MB) 다운로드 시 네트워크 대역폭에 영향을 줄 수 있습니다.</p><p><b>HTTP 프록시 설정(선택사항)<br></b>외부에서 지리 데이터 업데이트를 호스팅하는 Google 다운로드 서버에 액세스할 때 프록시를 사용할 경우 이 입력란을 사용해 프록시 서버 이름과 포트, 사용자 이름 및 비밀번호를 지정하십시오.</p><p><b>업데이트 설정<br></b>이 옵션이 활성화되면 도움말 기사나 지리 데이터베이스가 사용자 개입 없이 지정된 시간에 자동 모드로 업데이트됩니다.</p>
185	<nobr><span class=title1><b>프로필: 저장/DB (Ref. #A-2108)</b></span></nobr><br>이 화면에서 데이터 관리 옵션을 설정할 수 있습니다. 무장애 실행을 위해 Urchin은 자동 백업 및 복구 옵션을 제공합니다. 이러한 옵션들을 사용하면 시스템을 안전하게 실행할 수 있습니다. Urchin은 오래된 데이터를 자동으로 압축할 수 있는 자동 보관 옵션도 제공합니다. <p><b>DB 메모리 사용률</b><br>정상적인 상태에서 Urchin은 캐시 시스템을 사용하여 처리 엔진에서 사용되는 RAM 용량을 제한합니다. 이렇게 하면 성능이 증가되어 메모리에서 전체 프로세스를 실행할 수 있지만 이러한 방식은 Urchin 전용 시스템에서만 사용해야 하며 프로세스가 시스템에서 사용 가능한 RAM을 모두 소진할 수 있다는 점에 유의해야 합니다. 대부분의 작업에서 이 설정을 "캐시 시스템에서 제한"으로 남겨두십시오.<p><b>DB 테이블 제한</b><br>데이터의 크기와 대형 사이트에서 Urchin이 보여 주는 관련 성과를 지속적으로 점검하기 위해 이 설정은 표시된 레코드 수에 맞춰 Urchin 데이터베이스의 테이블을 하나로 제한합니다. 기본 옵션은 대부분의 설치에 적합합니다. 특정 달에 이러한 한계에 도달하면 추가 항목들이 '기타' 항목으로 일괄 처리됩니다. 성능을 최적화하려면 100,000 이하의 값을 사용하십시오. <p><b>원시 추적 데이터 보관</b><br>이 옵션을 끄면 처리 이후 원시 방문자 및 세션 정보가 삭제되므로 로그 처리 성능이 크게 개선되어 대규모 사이트를 위한 저장된 데이터의 양을 줄일 수 있습니다. 원시 세션 데이터가 삭제되면 보고에서 해당 세션의 추가 조회수가 새로운 세션으로 처리되어 에지 효과가 줄어들게 됩니다. <p><b>로그 추적</b><br>Urchin에는 특정 로그 파일의 처리 여부를 판별할 수 있는 '지능형' 로그 추적 기능이 탑재되어 있습니다. 이 기능을 사용하면 비정기적으로 편리한 때에 로그를 순환시킬 수 있습니다. 또한 일부 로그 파일의 처리 여부에 상관 없이 로그 파일 디렉토리에서 Urchin을 실행할 수도 있습니다. 중요사항: Urchin이 실제로 로그 파일을 저장하는 시스템에 존재하지 않을 경우 Urchin이 있는 로컬 시스템으로 로그를 모두 복사한 다음 로그가 이미 처리되었는지 여부를 판단합니다. 이렇게 되면 대용량 로그 파일에 대한 분석 작업이 진행 중인 경우 상당한 네트워크 부하를 유발시키게 됩니다.<p><b>자동 롤백 DB</b><br>이 옵션이 활성화되면 처리가 중단되거나 부분적으로 정지된 경우 데이터를 자동으로 복구합니다. Urchin은 이러한 상태를 자동으로 감지하여 데이터를 가장 최근의 백업(데이터가 있을 경우) 상태로 롤백시킨 다음 작업을 계속 진행합니다.<p><b>백업 생성</b><br>이 옵션을 사용하면 데이터 처리 이후 자동으로 백업이 생성됩니다. 백업은 저장 공간을 절약하기 위해 압축됩니다.<p><b>백업 데이터 삭제</b><br> 이 옵션을 사용하면 다음 옵션에서 보관하기 원하는 백업 데이터 수에 맞춰 이전의 백업 데이터가 자동으로 삭제됩니다. 이 옵션이 활성화되면 Urchin은 이전의 백업 데이터를 자동으로 삭제하고 저장 공간을 점검합니다. <p><b>보관할 백업 데이터 수</b><br> 이 옵션은 앞에서 언급한 삭제 기능을 위해 매달 보관해야 하는 백업 데이터의 양을 지정합니다.<p><b>DB 보관</b><br> 이 옵션을 통해 이전의 월별 데이터가 자동으로 보관됩니다. 이전의 데이터는 하나의 아카이브로 자동으로 압축되며 백업 데이터는 제거됩니다. 또한 보관된 데이터는 잠금 상태가 됩니다. 보고에 보관된 데이터가 계속 나타나지만 데이터가 보관된 달의 조회수는 더 이상 처리되지 않을 수 있습니다.<p><b>이후 DB 보관</b><br>이 옵션은 몇 달 전의 데이터부터 자동 보관할 것인지 그 시점을 지정합니다.<p><b>프로필의 모든 데이터 삭제</b><br>이 작업을 통해 모든 백업 데이터와 아카이브를 포함한 이 프로필의 모든 데이터가 영구히 삭제되므로 이 프로필의 모든 로그 파일을 다시 처리해야 합니다. <b>경고</b>: 오래된 데이터를 삭제할 때는 반드시 로그 파일이 모두 있는지 확인한 다음 삭제하시기 바랍니다.

192	<nobr><span class=title1><b>스케줄러: 작업 진행 상태 (Ref. #A-2664)</b></span></nobr><br>현재 실행 중인 작업의 진행 상태를 보여주는 화면으로, 실행 중인 작업을 취소하는 기능도 제공합니다. 대략적인 진행률을 나타내는 진행 표시줄이 창 상단에 표시됩니다. '작업 취소' 버튼을 사용하여 실행 중인 작업을 중지하고 프로필을 취소할 수 있습니다. 진행 표시줄 상단에는 해당 프로필 보고서에 대한 링크가 제공됩니다. 로그 처리 엔진의 실행 결과는 작업 처리 내역 창에 표시됩니다. <p>참고: 대용량의 원격 로그 파일을 처리하는 경우 복사 시간이 매우 길어 복사 중에 진행 표시줄이 멈추어 있을 수도 있습니다. Urchin이 실행 중인 컴퓨터로 파일이 모두 복사된 후에는 진행 표시줄이 계속 업데이트될 것입니다.

FTP를 사용하여 네트워크를 통해 로그 파일을 이동하는 경우 FTP 전용 프로그램으로 서버에 로그인하여 사용자 이름과 비밀번호가 올바른지 확인하십시오. '데이터 처리'(data proc) 시간이 전체 실행 시간의 일부에 불과한 경우 네트워크를 통해 로그 파일을 복사하는 데 문제가 있을 수 있으며 사용자가 직접 서버에 Urchin을 설치하는 것을 고려해 볼 수 있습니다.
194	<nobr><span class=title1><b>파일 탐색기 (Ref. #A-5000)</b></span></nobr><br>이 화면에서 로컬 파일 시스템을 검색하여 원하는 로그 소스의 위치를 찾을 수 있습니다. 로그 파일의 이름을 잘 모를 경우 이 탐색기가 특히 유용합니다. 디렉토리로 이동 상자(왼쪽)를 사용하면 탐색할 디렉토리를 선택할 수 있습니다. 디렉토리의 파일들이 파일 선택 상자(오른쪽)에 표시됩니다. 필요한 로그 파일을 찾으면 파일 선택 상자에서 해당 파일을 선택한 다음 확인을 클릭하십시오.<p><b>현재 디렉토리</b><br>창 상단의 현재 디렉토리 입력란을 직접 편집할 수 있습니다. 편집 후 해당 디렉토리의 내용을 표시하려면 돌아가기를 누르십시오.<p><b>한 수준 위로</b><br>오른쪽 상단의 한 수준 위로 아이콘(화살표가 있는 폴더 아이콘)을 클릭하면 현재 디렉토리가 한 수준 위로 올라가 부모 디렉토리로 이동합니다. 새 디렉토리의 내용이 표시됩니다.<p><b>디렉토리로 이동</b><br>왼쪽 창에서 현재 디렉토리 내의 디렉토리를 선택할 수 있습니다. 디렉토리로 이동 창에 표시된 디렉토리를 선택하면 디렉토리들이 해당 디렉토리로 변경됩니다.<p><b>파일 선택</b><br>오른쪽의 파일 선택 창에서 파일을 선택할 수 있습니다. 원하는 파일을 선택하면 파일 이름이 파일 이름 입력란에 표시됩니다. 파일 선택이 완료되면 확인을 클릭하십시오.<p><b>확인/검사/취소</b><br>확인 버튼을 클릭하면 창이 닫히고 선택된 파일이 이전 화면의 경로 입력란에 표시됩니다. 취소 버튼을 클릭하면 창이 닫힙니다. 검사 버튼을 클릭하면 권한 상태를 점검할 수 있으며, 파일 이름에서 와일드카드가 사용되는 경우 파일 한 개 또는 여러 개의 크기를 확인할 수 있습니다.

349	<nobr><span class=title1><b>프로필 마법사: 복사 (Ref. #A-349)</b></span></nobr><br>이 화면에서 기존 프로필을 복사할 수 있습니다. 생성된 프로필은 이 사이트를 지칭하는 이름, 기본 URL(도메인 이름), 추가 URL을 지정해야 하는 것을 제외하고 동일합니다.<p><b>선택 계정</b>(데이터센터 모드)<br>데이터센터 모드에서 계정 설정을 사용하면 계정 수준의 액세스와 관리를 제어하는 데 사용될 수 있는 계정에 이 프로필을 연결할 수 있습니다.<p><b>프로필 이름</b><br>이 필수 입력란에서 해당 프로필과 연관된 이름을 설정할 수 있습니다. 이 고유 이름은 임의의 이름이나 웹사이트 이름 중 하나가 될 수 있습니다. www.urchin.com 같은 웹사이트 이름을 입력하면 사용자의 편의를 위해 나머지 두 개의 입력란이 자동으로 입력됩니다.<p><b>웹사이트 URL</b><br>이 설정은 사용자의 웹사이트로 복귀하는 추천 링크를 생성하는 데 사용됩니다. 이 URL은 http://를 포함하는 사이트의 전체 도메인 이름이어야 합니다. <p><b>사이트 도메인</b><br>이 설정을 사용하면 사용자 웹사이트에 사용할 수 있는 모든 도메인(www가 포함되거나 포함되지 않은)을 표시할 수 있습니다. 이 입력란은 외부에서 사용자 사이트를 추천하는지 판단하는 데 사용됩니다. 쉼표를 사용하여 각 도메인 항목을 구분하십시오.<p><b>UTM 도메인</b>(UTM에서 추적하는 프로필용)<br>정상 상태에서는 이 입력란을 비워 두십시오. UTM은 사이트의 도메인을 자동으로 감지합니다. UTM 도메인 기능은 별도로 추적해야 하는 중첩 도메인이 있는 경우에만 사용됩니다.

375	<nobr><span class=title1><b>설치 마법사: 관리자 설정 (Ref. #A-375)</b></span></nobr><br>이 설정을 사용하면 Urchin 시스템 액세스를 통제할 수 있습니다.<p><b>웹 기반 보고서 액세스</b><br>이 설정을 통해 Urchin이 실행되지 않는 컴퓨터에서 보고서를 볼 수 있는지 여부를 지정할 수 있습니다. '켜기'를 선택하면 보고서 원격 액세스가 허용됩니다. <p><b>웹 기반 관리</b><br>이 설정은 Urchin 구성에 대한 웹 기반 관리 액세스를 활성화 또는 비활성화합니다. '켜기'를 선택하면 Urchin이 설치되지 않은 컴퓨터에서 Urchin의 동작을 제어할 수 있습니다. 중요사항: 실제로 Urchin이 실행되지 않는 컴퓨터에서 이 설정을 적용할 경우 웹 기반 관리를 활성화하지 않으면 Urchin이 설치된 컴퓨터에서만 관리 시스템에 액세스할 수 있습니다. <p><b>데이터센터 모드</b><br>이 옵션은 구성에서 계정과 하위 관리자를 활성화합니다. 이 추가 제어 옵션들을 사용하면 데이터센터에서 복잡한 고객을 관리하고, 해당 고객에게 제한적 관리 기능을 제공할 수 있습니다. 데이터센터는 프로필, 로그 파일, 사용자를 특정 계정하에 그룹화함으로써 해당 고객의 데이터 위치와 관리 권한을 조정할 수 있습니다.
376	<nobr><span class=title1><b>설치 마법사: 관리 사용자 (Ref. #A-376)</b></span></nobr><br>이 화면에서 Urchin 시스템의 관리자 비밀번호를 설정할 수 있습니다. 모든 Urchin 설치에는 기본적으로 동일한 비밀번호가 지정되어 있기 때문에 설치 중에 관리자 비밀번호를 변경하는 것이 중요합니다.<p><b>관리자 비밀번호</b><br>이 화면에서 Urchin 설치에 필요한 관리자 비밀번호를 변경할 수 있습니다. 시스템 보안을 유지할 수 있는 고유 비밀번호를 선택하는 것이 좋습니다. Urchin 구성 변경 시 필요하므로 비밀번호를 반드시 기억하십시오.<p>이전 단계에서 원격 관리자 액세스가 비활성화된 경우 Urchin이 실행되는 시스템에서 직접 Urchin에 액세스해야 합니다. 이 경우 Urchin은 비밀번호를 요청하지 않습니다.<p>Urchin 설치 구성 시 SSL을 추가 보안 조치로 사용하려는 경우 문서 센터를 추천하면 최신 정보를 얻을 수 있습니다.
377	<nobr><span class=title1><b>로그 검증 (Ref. #A-5003)</b></span></nobr><br>이 화면을 사용하면 선택한 로그 파일을 검토하여 올바른 로그 소스인지 확인할 수 있습니다. 로그 정보를 얻으려면 왼쪽 상자에서 로그 이름을 클릭하여 아래 상자에 표시되는 텍스트를 검토하십시오. 여기에서 해당 로그와 관련된 위치(경로), 크기, 변경일, 권한 등을 확인할 수 있습니다. 맨 위의 입력란에 표시되는 텍스트를 변경하면 로그 선택사항에 대한 매개변수를 변경할 수 있습니다. 원할 경우 별 모양의 정규식(예: access_log*는 'access_log' 접두사가 붙은 모든 로그를 선택함) 또는 날짜식(예: access_logDDMMYYYY)을 사용할 수 있습니다. Enter 키를 클릭하면 패턴 일치 결과를 확인할 수 있습니다. 확인을 클릭하여 계속 진행하십시오.<p><b>로그 파일</b><br>창 상단의 로그 파일 입력란은 직접 수정할 수 있습니다. 해당 항목의 결과를 표시하려면 돌아가기를 누르십시오.<p><b>선택된 파일</b><br>선택된 파일 창에서 파일을 클릭하면 해당 파일의 파일 정보를 볼 수 있습니다. 파일 정보는 창의 하단에 표시됩니다.<p><b>디렉토리의 기타 파일</b><br>이 창에서 파일을 클릭하면 해당 파일의 파일 정보를 볼 수 있습니다. 파일 정보는 창의 하단에 표시됩니다.<p><b>확인/취소</b><br>확인 버튼을 클릭하면 창이 닫히고 선택된 파일이 이전 창의 경로 입력란에 표시됩니다. 취소 버튼을 클릭하면 창이 닫힙니다.
378	<nobr><span class=title1><b>설치 마법사: DNS 조회 (Ref. #A-378)</b></span></nobr><br>이 화면을 사용하면 이 Urchin 설치를 위한 DNS 조회 매개변수를 설정할 수 있습니다. 즉, DNS 조회는 63.212.171.4와 같은 숫자 IP 주소를 www.urchin.com과 같은 도메인 이름으로 해석합니다. 일반적으로 DNS 조회를 활성화하면 중요한 도메인 정보를 보고서에 포함할 수 있습니다. <p><b>DNS 조회</b><br>'켜기' 버튼을 선택하면 이 기능이 활성화됩니다.<p><b>해결자 IP</b><br>이 입력란을 사용하면 사용자의 주소나 서비스 제공업체의 주소에 상관없이 해결자 IP 입력란에 유효 DNS 서버의 IP 주소를 입력할 수 있습니다. IP를 모를 경우 시스템 관리자 또는 해당 ISP에게 문의하시기 바랍니다. DNS 서버는 네트워크의 로컬 서버인 경우에만 올바르게 작동합니다.

UTM을 설치한 경우 해당 방식을 옵션으로 선택하십시오. UTM을 설치하지 않은 경우에는 IP+User-Agent 방식을 선택하십시오. UTM을 사용하지 않는 경우 일부 보고서가 나타나지 않습니다.<p><b>UTM 도메인</b><br>정상 상태에서는 이 입력란을 비워 두십시오. UTM은 사이트의 도메인을 자동으로 감지합니다. UTM 도메인 기능은 별도로 추적해야 하는 중첩 도메인이 있는 경우에만 사용됩니다.
392	<nobr><span class=title1><b>프로필 마법사: 로그 설정 (Ref. #A-391)</b></span></nobr><br>이 화면은 해당 프로필에 대해 처리될 모든 로그 소스를 표시합니다. 오른쪽 상단의 추가 버튼을 클릭하면 로그 소스 마법사가 실행되어 로그 소스를 추가할 수 있습니다. 여러 로그 소스가 있는 분산 처리 사이트의 경우 추가 라이센스가 필요합니다. 라이센스 추가 과정이 완료되면 이 화면으로 복귀합니다. 마법사를 완료하려면 마침을 클릭하십시오.<p><b>분산 처리 옵션</b><br>이 옵션은 분산 처리 라이센스를 구입한 경우에 이용할 수 있습니다. 병렬 로그 처리가 실행되면 Urchin은 모든 로그 파일을 한 번에 열어 한 번에 한 섹션씩 순서대로 읽어들입니다. 여기서 각 섹션은 15분 간의 로그 활동에 해당됩니다. 이 옵션을 사용하면 분산 처리 사이트의 성능이 크게 향상됩니다.<br><p><b>추가</b><br>오른쪽 상단의 추가 버튼을 클릭하면 로그 소스가 이 프로필에 추가됩니다. 라이센스 유효 기간이 만료되면 추가 버튼을 사용할 수 없습니다.

"Native"는 Urchin의 기본 인증 방식이며 다른 인증 유형값은 관리자가 Urchin 인스턴스를 구성할 때 사용할 수 있는 사용자설정 인증 방식입니다. '인증 유형' 목록에서 "Native" 이외의 값을 선택하면 '비밀번호' 및 '비밀번호 확인' 입력란이 비활성화됩니다. 관리자가 Urchin 인스턴스에 사용자설정 인증 공급업체를 하나 이상 구성한 경우에만 '인증 유형' 목록이 표시됩니다.<p><b>성명</b><br>사용자의 이름을 지정할 수 있는 입력란입니다.

448	<nobr><span class=title1><b>필터 관리자: 필터 설정 (Ref. #A-1701)</b></span></nobr><br>이 화면에서 필터 설정을 구성하고 조정할 수 있습니다. 필터링에 대한 자세한 내용은 사용 설명서 또는 Google의 온라인 문서 센터를 추천하십시오.<p><b>예제</b><br>새 필터를 추가할 때 사용자의 편의를 위해 예제 필터 목록이 제공됩니다. 예제 필터를 선택하면 필터 설정이 자동으로 입력되며 수정할 특정 사항이 있을 경우 해당 사항을 수정하면 됩니다. 예제 필터를 사용하면 여러 공통 필터의 설정값이 생성되는 방식을 쉽게 파악할 수 있습니다.<p><b>선택 계정</b>(데이터센터 모드만 해당)<br>새로운 필터에서는 필터에 연관되는 계정을 선택할 수 있습니다. 기존 필터에서는 필터가 속한 계정 이름이 표시됩니다.<p><b>필터 이름</b><br>이 필수 입력란은 이 필터를 추천할 경우 편리하게 사용되는 임의의 이름을 지정합니다. 이 필터에만 해당되는 고유한 이름을 입력하십시오.<p><b>필터 유형</b><br>원하는 필터 기능에 따라 필터 유형 중 하나를 선택하십시오. <li><b>제외 패턴</b>: 이 유형의 필터는 필터 패턴과 일치하는 로그 파일 행(조회수)<i>만</i> 제외합니다. 일치하는 행들은 완전히 무시되기 때문에 Netscape 등을 제외하는 필터의 경우 Urchin이 해당 로그 행에서 방문자, 경로, 추천 및 도메인 정보와 같은 다른 모든 정보를 제외하게 됩니다. <li><b>포함 패턴</b>: 이 유형의 필터는 필터 패턴과 일치하는 로그 파일 행(조회수)<i>만</i> 포함하며, 다른 모든 행들은 무시됩니다. 위의 예와 같이 무시된 조회수는 Urchin 시스템으로 전혀 보고되지 않습니다. <li><b>찾기와 바꾸기</b>: 입력란 내의 패턴을 검색하여 발견된 패턴을 대체 유형으로 바꾸는 데 사용되는 간단한 필터입니다. 입력란에서 패턴이 두 번 이상 발생하면 마지막으로 발생한 패턴만 교체됩니다. '^' 문자는 입력란의 시작 부분에서 일치 항목을 찾는 데 사용되며, '$'는 입력란의 끝 부분과 해당 일치 항목을 연결하는 데 사용됩니다. <li><b>조회 테이블</b>: 이 필터를 선택하면 코드를 인식 가능한 레이블로 매핑하는 조회 테이블을 선택할 수 있습니다. 해당 조회 테이블이 먼저 생성되어 Urchin 설치의 'lib/custom/lookuptables' 디렉토리에 설치(내부)되거나 로컬 파일 시스템, NFS 또는 UNC(Windows만 해당) 공유에 있어야 합니다. FTP/HTTP 소스는 지원되지 않습니다. <li><b>고급</b>: 이 유형의 필터를 사용하면 한두 개의 다른 입력란에서 입력란을 구성할 수 있습니다. 필터링 엔진에서 2개의 추출 입력란에 있는 식들을 지정된 입력란에 적용한 다음 생성자 식을 이용하여 입력란을 구성합니다. 방법은 POSIX 정규식 용례에 따릅니다. <p><b>필터 입력란 </b><br>풀다운 메뉴를 사용하면 필터를 적용할 로그 소스 입력란을 선택할 수 있습니다. '원시'라고 표시된 입력란은 처리되지 않은 로그 파일의 실제 내용을 나타냅니다. '자동'으로 표시된 입력란은 원시 입력란 또는 기타 자동 입력란에 기초하여 결정되는 자동 계산 입력란입니다. 모든 입력란의 용도를 보여 주는 전체 목록은 Google 온라인 설명서 센터의 필터링 섹션을 추천하십시오. <p><b>필터 패턴</b><br>이 입력란을 사용하여 필터 일치 패턴을 설정하십시오. 패턴에서는 POSIX 정규식 표준에 정의된 와일드카드를 사용할 수 있습니다.<p><b>검색 문자열</b><br>찾기와 바꾸기 필터의 경우 검색 문자열이 입력란의 일치 패턴으로 사용됩니다.<p><b>대체 문자열</b><br>찾기와 바꾸기 필터의 경우 검색 문자열을 대신하여 대체 문자열이 사용됩니다.<p><b>테이블 이름</b>(조회 테이블만 해당)<br>조회 테이블을 필터 유형으로 선택하면 보이는 이 입력란을 사용하면 내부 조회 테이블을 선택하여 코드를 이해하기 쉬운 이름에 매핑할 수 있습니다. 전화 모델 테이블은 약식 전화 플랫폼 식별자를 전화 기반 웹브라우저용 모델명과 제조업체명에 매핑합니다. 고유의 조회 테이블을 생성하여 코드를 캠페인에 매핑하거나(캠페인 추적 모듈의 경우) URL 변수들을 페이지 이름에 매핑할 수 있습니다. Urchin 설치의 'lib/custom/lookuptables' 디렉토리에서 확장자가 .lt인 모든 파일이 내부 테이블 드롭다운 목록에 표시됩니다.<p><b>파일 이름</b>(외부 조회 테이블만 해당)<br>조회 테이블을 필터 유형으로 선택하면 보이는 이 입력란을 사용하면 외부 조회 테이블 파일을 선택하여 코드를 이해하기 쉬운 이름에 매핑할 수 있습니다. 고유의 조회 테이블을 생성하여 코드를 캠페인에 매핑하거나(캠페인 추적 모듈의 경우) URL 변수들을 페이지 이름에 매핑할 수 있습니다.<p><b>사용자 이름 및 비밀번호</b>(Windows, 외부 조회 테이블만 해당)<br>이러한 입력란은 외부 조회 테이블이 있는 UNC 공유에 대한 사용자 인증 정보를 지정하는 데 사용됩니다.<p><b>제어</b><br>변경사항을 저장하려면 새 필터의 경우 마침 버튼을, 기존 필터의 경우 업데이트 버튼을 클릭하십시오. 기존 필터의 경우 화면 상단의 탭을 사용하면 해당 필터에 대해 수정 가능한 모든 설정에 액세스할 수 있습니다.
449	<nobr><span class=title1><b>필터: 프로필 (Ref. #A-1702)</b></span></nobr><br>이 화면에서 하나 이상의 기존 프로필에 이 필터를 적용할 수 있습니다. 필터가 프로필에 적용되면 해당 프로필과 연관된 모든 로그 소스가 해당 프로필 처리 중에 필터링됩니다.<p><b>프로필에 적용</b><br>이 필터를 프로필에 적용하려면 가용 프로필 상자에서 프로필 이름을 선택한 다음 오른쪽 방향 화살표를 클릭하십시오. 이 필터를 프로필에서 제거하려면 적용 대상 상자에서 프로필을 선택한 다음 왼쪽 방향 화살표를 클릭하십시오. 변경사항을 적용하려면 업데이트를 클릭하십시오.<p><b>제어</b><br>변경사항을 저장하려면 업데이트 버튼을 클릭하십시오. 화면 상단의 탭을 사용하면 해당 필터에 대해 수정 가능한 모든 설정에 액세스할 수 있습니다.
450	<nobr><span class=title1><b>필터: 로그 소스 (Ref. #A-1703)</b></span></nobr><br>이 화면에서 하나 이상의 기존 로그 소스에 이 필터를 적용할 수 있습니다. 필터가 로그 소스에 적용되면 해당 로그 소스를 처리할 때 실행 중인 프로필과 무관하게 해당 필터가 사용됩니다.<p><b>로그 소스에 적용</b><br>이 필터를 로그 소스에 적용하려면 가용 로그 소스 상자에서 로그 소스 이름을 선택한 다음 오른쪽 방향 화살표를 클릭하십시오. 이 필터를 로그 소스에서 제거하려면 적용 대상 상자에서 로그 소스를 선택한 다음 왼쪽 방향 화살표를 클릭하십시오. 변경사항을 적용하려면 업데이트를 클릭하십시오.<p><b>제어</b><br>변경사항을 저장하려면 업데이트 버튼을 클릭하십시오. 화면 상단의 탭을 사용하면 해당 필터에 대해 수정 가능한 모든 설정에 액세스할 수 있습니다.
451	<nobr><span class=title1><b>필터 관리자: 요약 (Ref. #A-1704)</b></span></nobr><br>이 화면은 해당 필터에 사용되는 전체 매개변수 집합을 표시합니다. 이러한 설정들을 신중하게 검토한 다음 프로필을 처리해야만 모든 작업이 올바르게 진행됩니다. 조정이 필요한 경우 해당 탭을 클릭하고, 필요에 따라 설정을 업데이트하십시오. <p><b>제어 </b><br>화면 상단의 탭을 사용하면 해당 필터에 대해 수정 가능한 모든 설정에 액세스할 수 있습니다.
452	<nobr><span class=title1> <b>사용자: 사용자 정보 (Ref. #A-1301)</b></span></nobr><br>해당 사용자의 여러 설정을 수정할 수 있는 화면입니다. <p><b>선택 계정</b>(데이터센터 모드만 해당)<br>사용자와 연관된 계정을 표시합니다.<p><b>사용자 이름</b><br>해당 사용자의 고유 이름을 설정할 수 있는 입력란입니다. 사용자 이름에는 공백이 없어야 합니다.<p><b>인증 유형</b><br>'인증 유형' 목록에 있는 Native, LDAP, MSAD 등의 유형 중에서 사용자를 인증하는 방식을 선택할 수 있습니다. 'Native'는 Urchin의 기본 인증 방식이며 다른 인증 유형값은 관리자가 Urchin 인스턴스를 구성할 때 사용할 수 있는 사용자설정 인증 방식입니다. '인증 유형' 목록에서 'Native' 이외의 값을 선택하면 '비밀번호' 및 '비밀번호 확인' 입력란이 비활성화됩니다. 관리자가 Urchin 인스턴스에 사용자설정 인증 공급업체를 하나 이상 구성한 경우에만 '인증 유형' 목록이 표시됩니다.<p><b>비밀번호</b><br>비밀번호는 8자 이상이어야 하며 문자 사이에 공백이 없어야 합니다. 또한 비밀번호는 대소문자를 구분하므로 주의하시기 바랍니다. 비밀번호를 변경하려면 비밀번호 입력란과 비밀번호 확인 입력란 모두에 비밀번호를 입력합니다. 변경 후에는 반드시 업데이트 버튼을 클릭하십시오. <p><b>이름</b><br>로그인하는 사용자의 이름을 지정할 수 있는 입력란입니다.<p><b>제어</b><br>변경사항을 저장하려면 업데이트 버튼을 클릭하십시오. 화면 상단의 탭을 사용하면 해당 사용자에 대해 수정 가능한 모든 설정에 액세스할 수 있습니다.
453	<nobr><span class=title1><b>사용자: 환경설정 (Ref. #A-1302)</b></span></nobr><br>이 화면에서 관리 수준과 기본 언어를 포함하여 사용자의 여러 설정을 수정할 수 있습니다. <p><b>관리자 수준</b><br>사용자의 관리자 권한 유무를 조정하는 옵션입니다. '사용자' 설정을 통해 보고서를 조회하고 환경설정을 변경할 수 있지만 구성은 관리할 수 없습니다. '사용자: 업데이트 없음'에는 수정 기능이 없으며 보고서 보기만 가능합니다. '계정 관리자'는 계정 그룹 내에서 제한된 관리 기능을 갖습니다(데이터센터 모드에서 사용됨). '수퍼 관리자'는 시스템 전체에 대한 통제 권한을 갖습니다. 기존의 수퍼 관리자만 관리 수준을 수퍼 관리자로 설정할 수 있습니다.<p><b>보고서 언어</b><br>이 옵션을 통해 해당 사용자의 언어로 보고서를 표시할 수 있습니다. 사용자는 '환경설정' 탭을 이용하여 언어를 변경할 수 있습니다.<p><b>지역</b><br>지역 설정을 사용하면 해당 사용자에 맞게 현지화 유형을 조정하여 보고서를 표시할 수 있습니다. 이 설정은 여러 언어설정에서 다르게 표시될 수 있는 날짜, 시간 및 숫자 형식을 조정합니다. 목록에 사용자 국가가 없을 경우 같은 날짜/번호 형식을 사용하는 지역을 선택하십시오. <p><b>애드워즈 도구 액세스</b><br>보고 인터페이스에서 '애드워즈 도구'에 대한 사용자의 액세스를 관리할 수 있는 옵션입니다. 이 옵션을 사용하여 사용자가 액세스할 수 있는 CPC 데이터가 포함된 모든 프로필에 대해 애드워즈 도구를 사용할 수 있도록 설정할 수 있습니다. 또는 이 옵션을 사용중지하여 CPC 관련 보고서를 읽기 전용으로 설정할 수도 있습니다. 그러면 모든 애드워즈 도구가 숨김 설정됩니다.</p><p><b>제어</b><br>구성을 변경한 후에는 반드시 업데이트 버튼을 클릭하여 변경사항을 저장하십시오. 화면 상단의 탭을 사용하면 해당 사용자에 대해 수정 가능한 모든 설정에 액세스할 수 있습니다.
454	<nobr><span class=title1><b>사용자: 그룹 (Ref. #A-1303)</b></span></nobr><br>이 화면에서 해당 사용자가 어떤 그룹(있을 경우)에 속하는지 선택할 수 있습니다. 그룹 수준에서 보고서 보기 권한을 부여할 수 있습니다. 사용자는 자신의 그룹이 액세스하는 모든 보고서를 볼 수 있습니다.<p><b>그룹에 추가</b><br>이 사용자를 그룹 구성원으로 추가하려면 가용 그룹 상자에서 그룹 이름을 선택한 다음 오른쪽 방향 화살표를 클릭하십시오. 이 사용자를 그룹에서 제거하려면 해당 그룹 회원 상자에서 그룹을 선택한 다음 왼쪽 방향 화살표를 클릭하십시오. 변경사항을 적용하려면 업데이트를 클릭하십시오.<p><b>제어 </b><br>반드시 업데이트 버튼을 클릭하여 변경사항을 저장하십시오. 화면 상단의 탭을 사용하면 해당 사용자에 대해 수정 가능한 모든 설정에 액세스할 수 있습니다.
455	<nobr><span class=title1><b>사용자: 보고서 액세스 (Ref. #A-1304)</b></span></nobr><br>이 화면을 사용하면 이 사용자에게 하나 이상의 기존 프로필에 대해 보고서 보기 권한을 부여할 수 있습니다. 그룹 권한 외에도 이 사용자에게는 이 화면 내에서 개별적으로 설정된 보고서 액세스 권한이 부여됩니다.<p><b>액세스 권한 부여</b><br>사용자에게 프로필 보고서에 액세스할 수 있는 권한을 부여하려면 가용 프로필 상자에서 프로필 이름을 선택한 다음 오른쪽 방향 화살표를 클릭하십시오. 액세스를 거부하려면 액세스 권한 부여 상자에서 프로필을 선택한 다음 왼쪽 방향 화살표를 클릭하십시오. 변경사항을 적용하려면 업데이트를 클릭하십시오.<p><b>제어</b><br>변경사항을 저장하려면 업데이트 버튼을 클릭하십시오. 화면 상단의 탭을 사용하면 해당 사용자에 대해 수정 가능한 모든 설정에 액세스할 수 있습니다.

470	<nobr><span class=title1><b>로그 소스 마법사: 옵션 (Ref. #A-470)</b></span></nobr><br>이 화면에서 로컬 로그 소스(Urchin이 설치된 컴퓨터에 있는) 또는 원격 로그 소스(다른 컴퓨터에 있는)를 지정할 수 있습니다. 계속 진행하려면 다음을 클릭하십시오. <p><b>로컬 로그 소스</b><br>Urchin이 실행되고 있는 컴퓨터의 로컬 파일 시스템(NFS 방식 파일시스템 포함)에 있는 로그 파일은 모두 로컬 파일로 간주됩니다. 로그 파일이 Urchin 시스템의 로컬 파일인 경우 이 옵션을 선택하십시오. <p><b>원격 로그 소스</b><br>Urchin은 FTP 프로토콜, HTTP 프로토콜 또는 HTTPS 프로토콜을 이용하여 처리하기 전에 Urchin 시스템으로 원격 로그를 전송할 수 있습니다. Windows 시스템에서 Urchin은 UNC 경로 이름에도 액세스할 수 있습니다. 사용자 로그 파일이 원격 파일이고 UNC를 통해 전송 또는 액세스해야 하는 경우 이 옵션을 선택하십시오.
471	<nobr><span class=title1><b>로그 소스 마법사: 원격 설정 (Ref. #A-471)</b></span></nobr><br>이 화면에서 이 로그 소스의 원격 로그 파일을 검색하거나 액세스하는 방법을 지정할 수 있습니다.<p><b>서버 프로토콜</b><br>Urchin은 네 종류의 원격 로그 파일, 즉 HTTP(www), HTTPS, FTP, UNC(Windows 전용)를 처리할 수 있습니다. 웹브라우저를 통해 사용자 로그 파일에 액세스할 수 있을 경우 HTTP를 사용하십시오. 비밀번호가 요구되지 않을 경우 사용자 이름 입력란과 비밀번호 입력란에 아무 것도 입력하지 마십시오. FTP의 경우 사용자 이름(대체로 사용자 사이트에 파일을 업로드하기 위한 사용자 이름과 동일)과 비밀번호를 입력하여 사용할 수 있습니다. Windows 시스템에서만 사용할 수 있는 UNC는 UNC 공유에 액세스하기 위한 사용자 이름과 비밀번호를 요구합니다.<p><b>서버 이름/IP</b><br>로그 파일이 위치한 서버 이름(예: www.urchin.com) 또는 IP 주소(예: 63.212.171.4)를 입력하십시오. UNC의 경우 \\server\share 기호를 사용하십시오.<p><b>서버 포트</b><br>서버 포트는 자동으로 입력됩니다. 그러나 비표준 포트를 사용하는 경우 서버 포트를 직접 수정할 수 있습니다.<p><b>동일한 로컬 및 원격 로그인</b>(UNC만 해당)<br>이 옵션은 UNC가 서버 프로토콜로서 선택된 경우에만 표시됩니다(Windows만 해당). 로컬 서버와 원격 서버 모두에서 동일한 로그인 및 비밀번호를 사용할 수 있을 경우 예를 선택하십시오. 원격 서버에서 로컬 서버와 다른 로그인 또는 비밀번호를 요구하는 경우 아니요를 선택하십시오. 아니요를 선택할 경우 로컬 서버 로그인에 사용되는 추가 입력란이 표시됩니다.<p><b>사용자 이름/비밀번호</b><br>이 입력란은 비인증 HTTP 연결과 익명 FTP 연결 시 공란으로 표시될 수 있습니다. 인증 HTTP/FTP 연결을 사용하려면 이 입력란에 사용자 이름과 비밀번호를 입력하십시오. UNC는 공유에 액세스하기 위한 사용자 이름과 비밀번호를 요구합니다. <p><b>도메인</b>(UNC만 해당)<br>UNC 공유가 도메인의 일부인 경우 이 입력란에 도메인 네트워크 정보를 입력하십시오. <p><b>FTP 모드</b>(FTP만 해당) <br>이 옵션은 FTP가 서버 프로토콜로서 선택되면 표시됩니다. 수동(PASV) 전송 방법에 대해서는 수동을 선택하십시오. 직접(PORT) 전송 방법에 대해서는 운영을 선택하십시오.
472	<nobr><span class=title1><b>로그 소스 마법사: 로컬/원격 (Ref. #A-472)</b></span></nobr><br>이 화면을 사용해서 로그 소스의 이름과 위치/대상을 지정하십시오.<p><b>로그 파일 경로</b>(로컬 로그만 해당)<br>이 입력란을 사용해 로그 파일의 전체 경로와 파일 이름을 입력하십시오(예: /var/logs/access-log). 사용자의 편의를 위해 찾기 버튼이 제공됩니다. 파일 이름에 별표와 같은 와일드카드를 사용할 수 있습니다. <p><b>원격 로그 소스 파일</b>(원격 로그에만 해당)<br>이 입력란을 사용해서 로그 파일에 대한 원격 경로를 지정하십시오. 이 경로는 도메인 정보가 없는 전체 경로입니다. 원격 HTTP 기반 로그 소스 경로 이름의 경우 와일드카드가 지원되지 않습니다. FTP 연결과 UNC 연결의 경우 사용자의 편의를 위해 찾기 버튼이 제공됩니다.<p><b>로그 대상</b>(원격만 해당)<br>처리 이전에 원격 로그 파일이 로컬 시스템으로 전송됩니다. 로그 파일을 저장하기 위한 로컬 디렉토리를 선택하십시오. 찾기 버튼이 제공됩니다.<p><b>선택 계정</b>(데이터센터 모드만 해당)<br>데이터센터 모드에서 이 선택 입력란을 사용하여 해당 계정에서만 사용하도록 하는 계정에 이 로그 소스를 연결할 수 있습니다.<p><b>로그 소스 이름</b><br>이 필수 입력란은 이 로그 소스를 추천할 경우 편리하게 사용되는 임의의 이름을 지정합니다. 이 로그 소스에만 해당되는 고유한 이름을 입력하십시오.
473	<nobr><span class=title1><b>로그 소스 마법사: 설정 (Ref. #A-473)</b></span></nobr><br>이 화면에서 처리할 로그 파일의 형식을 지정할 수 있습니다. 대부분의 웹서버에서는 <i>자동</i> 로그 형식을 선택합니다. 이전 Urchin 버전과의 호환성을 위해 다른 웹서버 로그 형식도 제공되지만 이러한 형식은 일반적으로 필요하지 않습니다. 사용자설정 로그 형식은 Urchin 폴더의 lib/custom/logformats 디렉토리에 있는 템플릿을 사용하여 작성할 수 있습니다.<p><b>로그 형식</b><br>로그 형식 옵션을 사용하여 이 로그 소스의 형식을 지정하십시오. <ul><li><b>자동:</b> 이 형식에서는 Apache, W3C(Microsoft IIS), NCSA, Netscape 및 iPlanet을 포함하여 대부분의 웹서버 로그 형식을 자동으로 감지합니다. <li><b>캠페인:</b> 이 형식은 특히 Urchin에 맞게 형식이 지정된 캠페인 비용 데이터에 사용하십시오. 자세한 내용은 온라인 설명서를 추천하시기 바랍니다. 캠페인 로그 형식은 자동 감지될 수 없습니다.<li><b>Elf2:</b> 이 형식은 ELF2 형식의 전자상거래 로그에 사용하십시오. 전자상거래 로그 형식은 자동 감지될 수 없습니다.<li><b>Google:</b> 이 형식은 Google CPC 지출 데이터를 포함하는 로그에 사용하십시오. Google 로그 형식은 자동 감지될 수 없습니다.<li><b>NCSA:</b> 이 형식은 일반적으로 자동 감지되며 Apache 웹서버에서 사용됩니다. 보고를 완전하게 수행하려면 대부분의 경우 기본 Apache 동작이 아닌 확장 결합 로깅을 사용해야 합니다. <li><b>Netscape:</b> 이전의 Netscape 서버(비iPlanet)에서는 기본적으로 이 형식을 사용합니다. <li><b>Overture:</b> 이 형식은 Overture CPC 지출 데이터를 포함하는 로그에 사용하십시오. Overture 로그 형식은 자동 감지될 수 없습니다. <li><b>W3C:</b> 이 형식은 자동 감지될 수 있으며 Microsoft IIS 서버와 같이 로그가 표준 W3C 형식인 웹서버에 사용됩니다.</ul>

539	<nobr><span class=title1><b>설정: 전체 설정 (Ref. #A-2701)</b></span></nobr><br>이 화면에서 해당 Urchin 설치를 위한 액세스 권한을 설정할 수 있습니다. 이러한 설정은 Urchin을 서버와 데스크톱 컴퓨터 중 어디에 설치하는지 여부에 따라 달라집니다.<p><b>웹 기반 보고서 액세스</b><br>'켜기'를 선택하면 사용자가 웹브라우저를 통해 원격으로 로그인하여 보고서를 볼 수 있습니다. '끄기'를 선택하면 Urchin이 설치된 시스템의 콘솔에서만 보고서를 볼 수 있습니다.<p><b>웹 기반 관리</b><br>'켜기'를 선택하면 '관리' 사용자가 웹브라우저를 통해 원격으로 로그인하여 구성을 변경할 수 있습니다. Urchin이 설치된 시스템의 콘솔에서만 구성을 변경하려면 '끄기'를 선택하십시오.<p>일반적으로 서버 설치의 경우 두 설정 모두 '켜기'로 설정되며, 데스크톱 설치의 경우 최소 웹 기반 관리가 '끄기'로 설정됩니다.<p>이 설정은 etc/session.conf 파일의 ALLOW_AUTO_LOGIN 옵션에 의해 무효화될 수 있음에 유의하십시오.<p><b>직접 보고서 연결</b><br>보안관련 중요사항: 이 기능을 활성화하면 세션 기반 보안 제어가 해제되어 보고서로 직접 연결되는 링크를 작성할 수 있습니다. 이미 비밀번호나 네트워크를 통해 보호되고 있는 시스템의 경우 이 기능이 유용할 수 있습니다. 이 기능이 활성화되면 다음 형식을 통해 보고서에 직접 링크될 수 있습니다.<p>&nbsp;&nbsp;(baseurl)/report.cgi?profile=(profilename)&user=(username) <p>여기서 (baseurl)은 현재 URL의 session.cgi 앞에 있는 모든 정보이고, (profilename)은 프로필의 URI-제거 이름이며, (username)은 보고서에 액세스하는 사용자입니다. user= 설정은 선택사항이며 언어 및 현지화 설정을 조정합니다. 별도로 지정되지 않을 경우 관리 사용자인 것으로 가정합니다. 구성 및 액세스 제어 옵션에 대한 자세한 내용은 Urchin 폴더 htdocs 디렉토리의 .report.conf 파일을 추천하십시오.<p><b>서버 포트 번호</b><br>이 옵션은 Urchin 웹서비스를 실행하는 포트 번호를 지정합니다. 이 옵션을 변경한 후에는 제품의 bin 디렉토리에 있는 "urchinctl" 응용프로그램을 사용하여 Urchin 서비스를 다시 시작해야 합니다. 자세한 내용은 설치 설명서를 추천하십시오. 기본적으로 Urchin은 포트 9999에서 실행됩니다. 물론 설치 중에 이 설정을 변경할 수 있습니다.<p><b>제어</b><br>변경사항을 저장하려면 반드시 업데이트 버튼을 클릭하십시오. 화면 상단의 탭을 사용하여 다른 설정에 액세스할 수 있습니다.
540	<nobr><span class=title1><b>설정: 서버 설정 (Ref. #A-2702)</b></span></nobr><br>이 화면에서 Urchin 관리 시스템과 보고서 작성 시스템이 사용하는 서버 포트를 변경할 수 있습니다. <p><b>서버 포트 번호</b><br>웹 기반 관리 및 보고서 보기가 Urchin이 설치된 서버의 지정 포트에서 실행됩니다. 기본적으로 서버 포트 번호는 포트 9999로 설정되어 있습니다. 중요사항: 표준 HTTP(포트 80) 또는 메일전송(포트 25)과 같은 또 다른 서비스에서 사용 중인 포트로 이 번호를 변경하지 마십시오. 포트를 9999에서 다른 포트로 변경하려면 시스템 관리자와 이 문제를 협의하는 것이 좋습니다. 이 값을 변경할 경우 Urchin 서비스를 다시 시작해야만 변경사항이 적용됩니다. 두 가지 Urchin 서비스에는 스케줄러('urchind')와 서버('urchinwebd')가 있습니다. Windows 플랫폼에서 이러한 서비스들을 재시작하려면 시작->프로그램->Urchin 메뉴에서 비활성화 옵션과 활성화 옵션을 사용하십시오. UNIX형 플랫폼에서는 Urchin 디렉토리의 'bin/urchinctl restart'를 실행하십시오.

542	<nobr><span class=title1><b>환경설정 (Ref. #A-1001)</b></span></nobr><br>이 화면에서 비밀번호를 설정/변경할 수 있으며 Urchin 보고서의 기본 언어와 현지화 설정을 조정할 수 있습니다. 변경사항을 적용하려면 업데이트를 클릭하십시오. 관리자를 위해 권한이 예약된 경우 일부 설정을 수정할 수 없습니다.<p><b>비밀번호</b><br>비밀번호를 변경하려면 비밀번호 입력란과 비밀번호 확인 입력란에 새 비밀번호를 입력한 다음 업데이트 버튼을 클릭하십시오. 다음 번 로그인 시 새 비밀번호가 적용됩니다. 비밀번호에는 공백 또는 기타 예약 문자가 없어야 합니다. <b>관리자 주의사항:</b> 반드시 관리자 비밀번호를 기억하십시오. <p><b>보고서 언어</b><br>이 메뉴를 사용하여 보고서를 볼 때 사용할 언어를 선택하십시오. 보고 인터페이스에 속한 모든 탐색 정보와 간단한 도움말 정보가 선택한 언어로 표시됩니다. 언어를 변경한 후 반드시 업데이트를 클릭하십시오.<p><b>지역</b><br>지역 설정을 사용하면 보고서 보기를 위한 현지화 유형을 조정할 수 있습니다. 이 설정은 여러 언어설정에서 다르게 표시될 수 있는 날짜, 시간, 숫자 형식을 조정합니다. 목록에 사용자 국가가 없을 경우 같은 날짜/번호 형식을 사용하는 지역을 사용자 국가로 선택하십시오. 변경사항을 저장하려면 업데이트 버튼을 클릭하십시오.
543	<nobr><span class=title1><b>프로세스 설정: DNS 설정 (Ref. #A-2301)</b></span></nobr><br>이 화면에서 IP 주소를 도메인 이름으로 변환(예: 63.212.171.4 -> urchin.com)하는 Urchin의 DNS 조회 기능을 구성할 수 있습니다. 이를 통해 Urchin은 도메인 이름, 네트워크, 국가에 대해 보고할 수 있습니다. DNS를 구성하려면 로컬 이름 서버의 IP 주소가 필요합니다. 자세한 내용은 관리자에게 문의하십시오.<p><b>DNS 조회</b><br>'on' 버튼을 선택하면 DNS 조회가 활성화됩니다.<p><b>해결 IP</b><br>Urchin이 IP를 도메인으로 전환하는 과정에서 검색하기에 적합한 로컬 DNS 이름 서버의 IP 주소를 입력하십시오.<p><b>DNS 루프</b><br>이 설정을 통해 Urchin이 포기하기 전에 시도하는 해결 횟수를 조정할 수 있습니다. 대부분의 설치에서 5를 기본값으로 설정하는 것이 좋습니다.<p><b>해결 대상</b><br>Urchin은 해결 대상에 도달할 경우 DNS 처리를 중지합니다. 처리 속도와 해결 사이의 적절한 균형을 유지하려면 기본값을 80%로 설정하는 것이 좋습니다. 이 숫자를 높이면 실행 속도가 낮아질 수도 있습니다.<p><b>제어</b><br>변경사항을 저장하려면 업데이트 버튼을 클릭하십시오. 화면 상단의 탭을 사용하여 다른 설정에 액세스할 수 있습니다.
544	<nobr><span class=title1><b>전체 설정: DB 설정 (Ref. #A-2302)</b></span></nobr><br>이 화면에서 성능과 리소스 사용률을 제어하기 위한 일부 기본 매개변수들을 설정할 수 있습니다. 기본값은 일반적으로 속도, 프로세서, 디스크 사용률을 최상으로 결합한 값입니다. 그러나 Urchin의 실행 속도를 높이거나 Urchin이 실행되는 컴퓨터에 Urchin이 미치는 영향을 줄일 필요가 있을 경우 그에 따라 기본값을 조정하십시오. <p><b>DB 테이블 제한 </b><br>대형 사이트에서 Urchin의 데이터베이스 크기를 유지하려면 이 설정을 사용하여 나열된 레코드 수에 맞춰 테이블을 하나로 제한하십시오. 기본 옵션은 대부분의 설치에 적합합니다. 특정 달에 이러한 한계에 도달하면 추가 항목들이 '기타' 항목으로 일괄 처리됩니다. 따라서 사용자 보고서에서 기타 항목의 비율이 높게 나타나는 경우 이 값을 높게 설정하십시오. 성능을 최적화하려면 100,000 이하의 값을 사용하십시오. <p><b>메모리 사용률 목표 </b><br>이 설정은 실행 시간 동안 사용할 메모리 용량과 관련하여 Urchin 프로세스에 대한 가이드 역할을 수행합니다. 대형 사이트에서 메모리 공간을 더 늘리면 시스템에서 디스크 캐싱이 감소하여 성능 향상될 수 있습니다. 기본 설정은 최소값으로 되어 있으므로 시스템에 실제 메모리가 충분할 경우 512MB 이상으로 늘릴 것을 권해드립니다. <p><b>지리 세부정보 </b><br><p><b>지리 정보 </b><br>이 설정에 따라 프로필 보고서에 포함될 지리 정보가 결정됩니다. 추가 옵션을 선택할 경우 로그 처리 엔진의 메모리 사용량이 증가할 수 있습니다(네트워크와 도메인에 대해 각각 약 350MB 및 100MB의 메모리가 소요되며 지리적 위치에 대해서는 선택한 지리 데이터베이스에 따라 20MB에서 최대 150MB 필요). Urchin에는 제공하는 국가 또는 지역 세부정보의 수준에 따라 크기가 다른 여러 지리 데이터베이스가 포함되므로, 드롭다운 메뉴에서 필요한 지리 데이터베이스를 적절히 선택하십시오. <p><b>지리 데이터베이스 </b><br>Urchin에는 다양한 수준의 지리적 위치 정보를 제공하는 다양한 지리 데이터베이스가 포함되어 있습니다. <br>- 전 세계(자세한 전 세계 국가/지역/도시 정보 포함) <br>- 전 세계 경량급(전 세계 국가 수준의 정보만 포함) <br>- 미국, 유럽연합 및 일본과 같은 지역용 데이터베이스(나머지 세계에 대한 해당 지역 및 국가 수준의 자세한 정보 포함) <br><br>지리 데이터베이스 설정은 전체적으로 정의하거나 프로필 수준에서 정의할 수 있습니다. <p><b>제어 </b><br>변경사항을 저장하려면 업데이트 버튼을 클릭하십시오. 화면 상단의 탭을 사용하여 다른 설정에 액세스할 수 있습니다.
545	<nobr><span class=title1><b>프로세스 설정: 메모리 사용률 (Ref. #A-2303)</b></span></nobr><br>이 화면에서 Urchin 로그 처리 엔진의 대략적인 메모리 사용률을 조정할 수 있습니다. 대용량 로그 파일이 있지 않는 한 기본 설정을 사용하는 것이 좋습니다. 대용량 로그의 경우 메모리 사용률을 높여 가용 자원을 활용함으로써 Urchin의 속도를 높일 수 있습니다. 메모리 사용률을 너무 높게 설정하면 스와핑 때문에 메모리 할당 오류 또는 속도 저하가 발생할 수 있습니다.<p><b>메모리 사용률 목표</b><br>풀다운 메뉴를 사용하여 Urchin이 처리 중 사용할 메모리 용량을 선택하십시오. Urchin은 데이터에 따라 선택된 사용량보다 약간 더 많거나 적은 메모리를 사용할 수도 있습니다. 사용자설정 옵션은 경험 많은 Urchin 관리자들에 의해 특수한 경우에만 사용되어야 합니다.<p><b>사용자설정 메모리 사용률 설정</b><br>사용자설정 메모리 설정이 선택되면 기본 Urchin 버퍼 각각을 수동으로 설정할 수 있는 추가 옵션들이 표시됩니다. 이러한 값들을 메가바이트 단위로 설정하면 버퍼링 용량과 전체 성능이 변경됩니다. 이 값들은 조정하지 않는 것이 좋습니다.<p><b>제어</b><br>변경사항을 저장하려면 업데이트 버튼을 클릭하십시오. 화면 상단의 탭을 사용하여 다른 설정에 액세스할 수 있습니다.

547	<nobr><span class=title1><b>프로필 (Ref. #A-2100)</b></span></nobr><br>이 화면을 사용하면 이 Urchin 설치 구성에 사용된 프로필을 표시하고, 각 프로필의 매개변수를 조정하고, 관리자가 프로필을 추가할 수 있습니다. 보고서를 작성할 웹사이트마다 구성 프로필이 필요합니다.<p><b>추가</b>(수퍼 관리자만 해당)<br>프로필을 추가하려면 화면 오른쪽 상단의 추가 버튼을 클릭하십시오. 프로필 추가 마법사가 시작되고 간단한 단계들을 거쳐 프로필을 추가하게 됩니다.<p><b>수정</b><br>목록에 표시된 각 프로필 옆에는 일련의 버튼들이 있으며, 이러한 버튼들을 이용하여 각 프로필을 조정할 수 있습니다. 수정 버튼을 클릭하면 탭 형식으로 구성된 이 프로필의 모든 설정에 액세스할 수 있습니다.<p><b>삭제</b>(수퍼 관리자만 해당)<br>프로필 옆의 삭제 버튼을 클릭하면 구성에서 해당 프로필을 제거할 수 있습니다. 그러면 해당 프로필의 모든 데이터와 구성이 영구히 제거되며 처리 기록도 삭제됩니다.<p><b>복사</b>(수퍼 관리자만 해당)<br>복사 버튼을 클릭하면 기존 프로필과 설정이 비슷한 새 프로필을 쉽게 작성할 수 있습니다. 복사 마법사를 시작하여 새 프로필 이름을 입력하려면 이 버튼을 클릭합니다. <p><b>실행</b>(수퍼 관리자만 해당)<br>실행 버튼을 클릭하면 프로필을 즉시 실행하거나 프로필 실행 주기를 예약할 수 있는 실행/예약 화면으로 바로 이동하게 됩니다. 프로필이 실행되기 전까지는 어떤 보고서 데이터도 이용할 수 없습니다.<p><b>보기</b><br>보기 버튼을 클릭하면 각 프로필의 보고 창이 바로 시작됩니다.<p><b>제어</b><br>#표시 풀다운 메뉴를 사용하면 한 번에 나열되는 항목 수를 변경할 수 있습니다. 이전 버튼 또는 다음 버튼을 사용하면 한 번에 한 페이지씩 항목을 스크롤할 수 있습니다. <p><b>계정</b>(데이터센터 모드만 해당)<br>데이터센터 모드 실행 중에 계정 메뉴를 사용하면 나열되는 항목들을 필터링할 수 있습니다.

561	<nobr><span class=title1><b>프로필: 로그 소스 (Ref. #A-2102)</b></span></nobr><br>이 화면을 사용하면 <b>가용 로그 소스</b> 목록에서 <b>처리할 로그 소스</b>를 가져와 추가할 수 있습니다. <b>처리할 로그 소스</b> 열에는 이 프로필의 일부로 처리된 로그 소스가 표시됩니다. 일반적으로 각 프로필에는 연관된 로그 소스가 하나씩만 있게 되지만 분산 처리 구성이나 전자상거래 로그(ELF 형식) 또는 CPC 로그가 있는 사이트의 경우 복수의 로그 소스가 있을 수 있습니다. <p><b>제어: </b><br><ul><li><b>추가 </b><br>가용 로그 소스에 로그 소스를 추가하려면 추가 아이콘을 클릭하십시오. 로그 소스 추가 마법사가 시작되고 로컬 또는 원격 로그 소스를 단계별로 추가하게 됩니다. <li><b>수정</b><br>수정하고 싶은 가용 로그 소스를 선택한 다음 수정 아이콘을 클릭하십시오. <li>변경사항을 저장하려면 <b>업데이트</b> 버튼을 클릭하십시오. <li>화면 상단의 탭을 사용하면 해당 프로필에 대해 수정 가능한 모든 설정에 액세스할 수 있습니다.

563	<nobr><span class=title1><b>프로필: 사용자 (Ref. #A-2104)</b></span></nobr><br>이 화면을 사용하면 Urchin 시스템에서 구성된 사용자에게 해당 프로필 보고서에 대한 보기 권한을 부여할 수 있습니다. 비관리 사용자가 Urchin 시스템에 로그인하는 경우 해당 사용자는 관리자가 명시적으로 추가한 보고서에만 액세스할 수 있습니다. <p><b>사용자 할당</b><br>특정 사용자의 프로필에 대한 액세스 권한을 부여하려면 왼쪽 상자의 '가용 사용자'에서 사용자를 선택한 다음 오른쪽 방향 화살표를 클릭하십시오. 특정 사용자의 프로필에 대해 액세스를 거부하려면 오른쪽 상자에서 해당 사용자를 선택한 다음 왼쪽 방향 화살표를 클릭하십시오. 변경사항을 저장하려면 업데이트를 클릭하십시오.<p><b>대시보드 무시</b><br>이 옵션을 사용하면 사용 가능한 보고서와 사용자별로 사용 가능한 대시보드를 변경할 수 있습니다. 사용자를 강조표시하려면 "액세스 허가됨" 창에서 사용자를 한 번 클릭하십시오. 그런 다음 "대시보드 무시" 버튼을 클릭하여 선택한 사용자에 대한 전체 보고 섹션을 활성화하거나 비활성화할 수 있는 창을 연 후 해당 기본 대시보드를 선택하십시오. 사용자를 가용 사용자에서 액세스 허가됨으로 이동하려는 경우 해당 사용자에 대해 해당 옵션을 사용하려면 먼저 "업데이트"해야 합니다.<p><b>제어 </b><br><ul><li><b>추가</b><br>사용자 마법사를 시작한 후 가용 사용자 목록에 추가하려면 추가 아이콘을 클릭하십시오. <li><b>수정</b><br>강조표시한 사용자를 수정하려면 수정 아이콘을 클릭하십시오. <li>변경사항을 저장하려면 업데이트 버튼을 클릭하십시오. <li>화면 상단의 탭을 사용하면 해당 프로필에 대해 수정 가능한 모든 설정에 액세스할 수 있습니다.

565	<nobr><span class=title1><b>로그 관리자: 로그 설정 (Ref. #A-1601)</b></span></nobr><br>이 화면에서 로그 소스의 위치와 형식을 구성할 수 있습니다. 보고를 올바르게 수행하려면 이 설정들이 반드시 올바르게 구성되어야 합니다. <p><b>선택 계정</b>(데이터센터 모드만 해당)<br>해당 로그 소스와 연관된 계정을 표시합니다. <p><b>로그 파일 위치 </b><br>로컬 파일 시스템이나 이 컴퓨터에 탑재된 네트워크 파일 시스템에서 로그 파일에 액세스할 수 있는 경우 로컬을 선택하십시오. FTP 또는 HTTP를 이용하여 원격 시스템에서 로그 파일을 전송하거나 UNC 공유를 통해 로그 파일에 액세스해야 하는 경우(Windows만 해당) 원격을 선택하십시오. 원격 로그의 경우 업데이트 버튼을 클릭하면 '원격 설정' 탭이 상단에 표시됩니다. 탭을 클릭하여 원격 로그 액세스 세부 정보를 구성하십시오.<p><b>로그 소스 이름</b><br>이 로그 소스를 위해 선택하는 이름입니다. <p><b>로그 형식</b><br>이 옵션을 사용하여 처리할 로그 파일의 형식을 지정하십시오. 대부분의 웹서버에서는 <i>자동</i> 로그 형식을 선택합니다. 이전 Urchin 버전과의 호환성을 위해 다른 웹서버 로그 형식도 제공되지만 이러한 형식은 일반적으로 필요하지 않습니다. 사용자설정 로그 형식은 Urchin 폴더의 lib/custom/logformats 디렉토리에 있는 템플릿을 사용하여 작성할 수 있습니다. <ul><li><b>자동:</b> 이 형식에서는 Apache, W3C(Microsoft IIS), NCSA, Netscape 및 iPlanet을 포함하여 대부분의 웹서버 로그 형식을 자동으로 감지합니다. <li><b>캠페인:</b> 이 형식은 특히 Urchin에 맞게 형식이 지정된 캠페인 비용 데이터에 사용하십시오. 자세한 내용은 온라인 설명서를 추천하시기 바랍니다. 캠페인 로그 형식은 자동 감지될 수 없습니다. <li><b>Elf2:</b> 이 형식은 ELF2 형식의 전자상거래 로그에 사용하십시오. 전자상거래 로그 형식은 자동 감지될 수 없습니다. <li><b>Google:</b> 이 형식은 Google CPC 지출 데이터를 포함하는 로그에 사용하십시오. Google 로그 형식은 자동 감지될 수 없습니다. <li><b>NCSA:</b> 이 형식은 일반적으로 자동 감지되며 Apache 웹서버에서 사용됩니다. 보고를 완전하게 수행하려면 대부분의 경우 기본 Apache 동작이 아닌 확장 결합 로깅을 사용해야 합니다. <li><b>Netscape:</b> 이전의 Netscape 서버(비iPlanet)에서는 기본적으로 이 형식을 사용합니다. <li><b>Overture:</b> 이 형식은 Overture CPC 지출 데이터를 포함하는 로그에 사용하십시오. Overture 로그 형식은 자동 감지될 수 없습니다. <li><b>W3C:</b> 이 형식은 자동 감지될 수 있으며 Microsoft IIS 서버와 같이 로그가 표준 W3C 형식인 웹서버에 사용됩니다.</ul></ul></ul> <p><b>로그 파일 경로</b><br>로컬 로그 소스의 경우 로그 파일 경로는 로컬 파일 시스템에서 로그 파일이 저장될 위치를 지정합니다. 사용자의 편의를 위해 찾기 버튼을 제공합니다. <p><b>제어</b><br>변경사항을 저장하려면 업데이트 버튼을 클릭하십시오. 화면 상단의 탭을 사용하면 해당 로그 소스에 대해 수정 가능한 모든 설정에 액세스할 수 있습니다.

567	<nobr><span class=title1><b>계정: 정보 (Ref. #A-1201)</b></span></nobr><br>이 화면에서 이 계정의 여러 설정을 수정할 수 있습니다.<p><b>계정 이름 </b><br>계정마다 고유의 이름을 지정해야 합니다. 계정 이름은 임의로 지정할 수도 있고 고객 이름이나 웹사이트에 기초할 수도 있습니다.<p><b>연락처와 이메일</b><br>연락처 정보는 선택사항이며 고객을 편리하게 추적하기 위한 수단으로 제공됩니다.<p><b>보고서 데이터 위치</b><br>선택사항인 이 설정은 해당 계정에 속하는 모든 프로필의 보고서 데이터를 저장할 디렉토리 경로를 지정합니다. 기본 위치는 Urchinn 폴더 내의 data/reports/<i>account_name</i> 디렉토리입니다. etc 디렉토리의 urchin.conf 파일을 수정하면 전체 시스템에서 데이터 디렉토리 전체를 이동할 수 있습니다. 사용자의 편의를 위해 찾기 버튼이 제공됩니다.<p><b>실행 기록 위치</b><br>선택사항인 이 설정은 해당 계정에 속하는 모든 프로필에 대한 실행 시간 기록 데이터를 저장할 디렉토리 경로를 지정합니다. 기본 위치는 Urchinn 폴더 내의 data/history/<i>account_name</i> 디렉토리입니다. 사용자의 편의를 위해 찾기 버튼이 제공됩니다. <p><b>디렉토리 찾기 위치</b><br>선택사항인 이 설정은 프로필 생성을 위해 계정 관리자의 액세스를 허용하는 경우 보안 조치로서 제공됩니다. 이 입력란에 입력된 디렉토리 경로는 해당 디렉토리에서만 로그 파일을 찾도록 계정 관리자의 권한을 제한합니다. 기본적으로는 탐색에 제한이 없습니다. etc 디렉토리에서 urchin.conf 파일을 수정하면 전체 시스템에서 이 설정을 변경할 수 있습니다. 사용자의 편의를 위해 찾기 버튼이 제공됩니다.<p><b>제어</b><br>변경사항을 저장하려면 업데이트 버튼을 클릭하십시오. 화면 상단의 탭을 사용하면 해당 계정에 대해 수정 가능한 모든 설정에 액세스할 수 있습니다.
568	<nobr><span class=title1><b>프로필: 실행/예약 (Ref. #A-2105)</b></span></nobr><br>이 화면을 사용하면 작업 실행(프로필과 연관된 로그 소스의 처리)을 예약하거나 오른쪽의 지금 실행 버튼을 클릭하여 작업을 즉시 실행할 수 있습니다. 예약 버튼 중 하나를 클릭하면 선택한 빈도에 따라 원하는 시간 및 분과 같은 추가 옵션들이 제공됩니다. 변경사항을 적용하려면 업데이트 버튼을 클릭하십시오. 지금 실행 버튼을 클릭하면 프로필이 즉시 실행되도록 예약됩니다. 실행 세부정보를 보려면 스케줄러의 작업 상태를 확인하십시오.<p><b>지금 실행</b><br>이 프로필을 즉시 처리하려면 오른쪽에 녹색으로 크게 표시된 지금 실행 버튼을 클릭하십시오. 그러면 작업이 Urchin 스케줄러 실행 대기열에 입력됩니다.<p><b>예약</b><br>이 프로필 작업이 나중에 또는 정기적으로 실행되도록 예약하려면 시간별, 일별, 주별 또는 월별의 빈도 옵션 중 하나를 선택하십시오. 그러면 해당 빈도에 따라 이 프로필 작업이 실행됩니다. 정확한 시작 시간을 지정하려면 빈도가 선택된 다음 표시되는 날짜 조정 항목들을 사용하십시오. 모든 시간은 Urchin이 실행되는 시스템의 현지 시간으로 지정됩니다.<p><b>제어</b><br>변경사항을 저장하려면 업데이트 버튼을 클릭하십시오. 화면 상단의 탭을 사용하면 해당 프로필에 대해 수정 가능한 모든 설정에 액세스할 수 있습니다.
569	<nobr><span class=title1><b>프로필: 보고 (Ref. #A-2106)</b></span></nobr><br>특정 필터 및 프록시 설정의 식별을 포함하여 Urchin의 여러 가지 중요한 처리 기능을 관리할 수 있는 보고 설정 화면입니다.<p><b>웹사이트 URL</b><br>사용자의 실제 웹사이트로 연결되는 추천 링크를 생성하는 데 사용되는 설정입니다. 이 URL은 http://를 포함하는 사이트의 전체 도메인 이름이어야 합니다. <p><b>사이트 도메인</b>(수퍼 관리자만 해당)<br>이 설정을 사용하면 사용자 웹사이트에 사용할 수 있는 모든 도메인("www"가 포함되거나 포함되지 않은)을 표시할 수 있습니다. 이 입력란은 외부에서 사이트를 추천하는지 판단하는 데 사용됩니다. 쉼표를 사용하여 각 도메인 항목을 구분하십시오. <p><b>기본 페이지</b><br> 이 사이트의 디렉토리에 있는 기본(또는 색인) 페이지를 가리키는 설정입니다. 대체로 기본 페이지는 "index.html"이지만 Windows IIS 서버의 경우 "Default.htm" 또는 "index.htm"인 경우도 종종 있습니다. 이 정보를 통해 Urchin이 실제로 동일한 페이지인 "http://www.urchin.com/" 및 "http://www.urchin.com/index.html"과 같은 로그 항목을 조정합니다. 기본 페이지 정보가 올바르게 입력되지 않으면 이러한 페이지들은 별개의 두 페이지로 보고됩니다. 하나의 페이지만이 기본 페이지로 지정되어야 합니다. <p><b>프록시 설정</b>(수퍼 관리자만 해당)<br>일부 설치에서는 홈페이지 오버레이 보고서를 표시하도록 프록시를 설정해야 할 수 있습니다. 오버레이 검색어가 Urchin 시스템에서 시작되므로 방화벽 설정이 외부 통신을 허용하지 않을 수 있습니다. 실제로 이러한 경우가 발생하면 프록시를 사용해 이러한 유형의 검색어를 허용할 수 있습니다. 프록시 서버는 서버의 호스트 이름이어야 합니다. 프록시 포트는 프록시와 통신할 포트 번호여야 합니다. 프록시에 액세스하기 위해 사용자 이름과 비밀번호가 필요한 경우 프록시 사용자 이름과 비밀번호 입력란에서 추가적인 보안 조치가 제공될 수도 있습니다. 오버레이 기능을 사용하는 중에 오류가 발생하지 않으면 이 입력란을 모두 비워 두십시오. 오류가 발생한 경우에는 네트워크 설정 관리자에게 문의하십시오.<p><b>병렬 로그 처리</b>(수퍼 관리자만 해당)<br>대규모 분산 처리 로그의 경우 해당 로그를 병렬로 실행하면 실적을 높일 수 있습니다. 이 처리 작업은 데이터의 크기와 시스템 IO 실적을 포함해 다양한 요인에 의해 달라집니다. 특히 단일 사용자 세션의 조회수가 여러 로그에 있을 수 있는 경우 이 설정을 활성화하여 실적을 높이는 데 사용해 보십시오. <p><b>시간 오프셋</b>(수퍼 관리자만 해당)<br>이 설정을 사용하면 Urchin이 데이터를 보고하는 시간대를 지정할 수 있습니다. 모든 조회수에는 GMT 기준 시간이 포함되기 때문에 Urchin은 모든 시간대에서 보고할 수 있습니다. '현지 시간' 옵션은 Urchin이 실행되는 서버에서 구성된 시간대에 따라 데이터를 기록합니다. 다른 설정을 사용하려면 GMT에서 오프셋을 선택하십시오. <p><b>자연어 키워드 검색</b>(수퍼 관리자만 해당)<br>이 설정을 사용하면 Urchin에서 보고서를 위한 검색엔진 및 키워드 감지 방식을 제어할 수 있습니다. 쉼표로 구분되는 이 목록을 사용하여 Urchin에서는 일치하는 추천 사이트 검색어에서 매개변수를 찾습니다. 그런 다음 Urchin은 추천에서 사용된 키워드 또는 검색어로서 매개변수 값을 캡처합니다. 이 목록을 사용하면 항목을 추가하여 자동으로 포함될 수 없는 검색어를 캡처할 수 있습니다. 기본 목록은 대부분의 검색엔진에 적용됩니다.<p><b>페이지뷰 마임즈 일치</b>(수퍼 관리자만 해당)<br> 쉼표로 구분하여 페이지뷰로 처리될 파일 확장자를 지정하는 목록입니다. 논리를 설정하여 특정 확장자를 포함하거나 제외할 수 있습니다. 대부분의 사이트에서는 기본 목록만으로 충분합니다. <p><b>페이지 매개변수 일치</b><br>이 설정은 Urchin이 매개변수 보고서에서 캡처하거나 캡처하지 않을 매개변수를 지정합니다. 요청한 URL의 검색어 부분이 매개변수가 되며 내부 검색어 또는 기타 동적 콘텐츠가 포함될 수 있습니다. Urchin은 이러한 이름값 쌍들을 자동으로 캡처하여 매개변수 보고서에서 보고합니다. 쉼표로 구분되는 이 목록을 이용하여 특정 매개변수를 제외하거나 포함할 수 있습니다. 예를 들어, 수많은 무작위 세션 ID가 포함된 사이트의 경우 이 보고서에서 SID 변수를 제외하여 스토리지 자원을 절약할 수 있습니다.<p><b>다운로드 검색</b>(수퍼 관리자만 해당)<br> 다운로드 보고서에서 다운로드된 파일로 처리될 파일 확장자를 쉼표로 구분하여 지정하는 목록입니다. 필요한 경우 목록을 수정하여 다운로드에 사용된 파일 확장자를 포함할 수 있습니다.<p><b>지리 세부정보</b><br><p><b>지리 정보</b><br>이 설정에 따라 프로필 보고서에 포함될 지리 정보가 결정됩니다. 추가 옵션을 선택할 경우 로그 처리 엔진의 메모리 사용량이 증가할 수 있습니다(네트워크와 도메인에 대해 각각 약 350MB 및 100MB의 메모리가 소요되며 지리적 위치에 대해서는 선택한 지리 데이터베이스에 따라 20MB에서 최대 150MB 필요). Urchin에는 제공하는 국가 또는 지역 세부정보의 수준에 따라 크기가 다른 여러 지리 데이터베이스가 포함되므로, 드롭다운 메뉴에서 필요한 지리 데이터베이스를 적절히 선택하십시오.<p><b>지리 데이터베이스</b><br>Urchin에는 다양한 수준의 지리적 위치 정보를 제공하는 다양한 지리 데이터베이스가 포함되어 있습니다.<br>- 전 세계(자세한 전 세계 국가/지역/도시 정보 포함)<br>- 전 세계 경량급(전 세계 국가 수준의 정보만 포함)<br>- 미국, 유럽연합 및 일본과 같은 지역용 데이터베이스(나머지 세계에 대한 해당 지역 및 국가 수준의 자세한 정보 포함)<br> <p><b>문자 인코딩</b><br>처리할 로그에 UTF-8로 인코딩되지 않은 문자가 있으면 올바른 데이터 인코딩 형식을 선택하십시오. 일본어 및 라틴어 1 형식이 지원됩니다. <p><b>통화 및 자릿수</b><br>통화 및 자릿수 설정을 사용해 전자상거래 사이트에서 표시할 기본 통화 기호와 표시 형식, 숫자 자릿수를 변경하십시오. <p><b>제어:</b><br>변경사항을 저장하려면 업데이트 버튼을 클릭하십시오. 화면 상단의 탭을 사용하면 해당 프로필에 대해 수정 가능한 모든 설정에 액세스할 수 있습니다.

575	<nobr><span class=title1><b>로그 관리자: 원격 설정 (Ref. #A-1602)</b></span></nobr><br>이 화면에서 이 로그 소스의 원격 로그 파일을 검색하거나 액세스하는 방법을 지정할 수 있습니다.<p><b>서버 프로토콜</b><br>Urchin은 네 종류의 원격 로그 파일, 즉 HTTP(www), HTTPS, FTP, UNC(Windows 전용)를 처리할 수 있습니다. 웹 브라우저를 통해 사용자 로그 파일에 액세스할 수 있을 경우 HTTP를 사용하십시오. 비밀번호가 요구되지 않을 경우 사용자 이름 입력란과 비밀번호 입력란에 아무 것도 입력하지 마십시오. FTP의 경우 사용자 이름(대체로 사용자 사이트에 파일을 업로드하기 위한 사용자 이름과 동일)과 비밀번호를 입력하여 사용할 수 있습니다. Windows 시스템에서만 사용할 수 있는 UNC는 UNC 공유에 액세스하기 위한 사용자 이름과 비밀번호를 요구합니다.<p><b>서버 이름/IP</b><br>로그 파일이 위치한 서버 이름(예: www.urchin.com) 또는 IP 주소(예: 63.212.171.4)를 입력하십시오. UNC의 경우 \\server\share 기호를 사용하십시오.<p><b>동일한 로컬 및 원격 로그인</b><br>이 옵션은 UNC가 서버 프로토콜로서 선택된 경우에만 표시됩니다(Windows만 해당). 로컬 서버와 원격 서버 모두에서 동일한 로그인 및 비밀번호를 사용할 수 있을 경우 예를 선택하십시오. 원격 서버에서 로컬 서버와 다른 로그인 또는 비밀번호를 요구하는 경우 아니요를 선택하십시오. 아니요를 선택할 경우 로컬 서버 로그인에 사용되는 추가 입력란이 표시됩니다.<p><b>서버 포트</b><br>서버 포트는 자동으로 입력됩니다. 그러나 비표준 포트를 사용하는 경우 서버 포트를 직접 수정할 수 있습니다.<p><b>사용자 이름/비밀번호</b><br>이 입력란은 비인증 HTTP 연결과 익명 FTP 연결 시 공란으로 표시됩니다. 인증 HTTP/FTP 연결을 사용하려면 이 입력란에 사용자 이름과 비밀번호를 입력하십시오. UNC는 공유에 액세스하기 위한 사용자 이름과 비밀번호를 요구합니다. <p><b>도메인(UNC만 해당)</b><br>UNC 공유가 도메인의 일부인 경우 이 입력란에 도메인 네트워크 정보를 입력하십시오. <p><b>FTP 모드</b><br>이 옵션은 FTP가 서버 프로토콜로서 선택되면 표시됩니다. 수동(PASV) 전송 방법에 대해서는 수동을 선택하십시오. 직접(PORT) 전송 방법에 대해서는 운영을 선택하십시오. <p><b>원격 로그 소스 파일</b><br>원격 로그 소스 파일은 원격 시스템의 로그 파일로 접근할 수 있는 경로를 지정합니다. HTTP를 사용하는 경우 먼저 웹 브라우저를 통해 로그 파일을 더 쉽게 찾는 경우도 종종 있습니다. 이렇게 하려면 해당 사이트의 URL을 브라우저에 입력한 다음 슬래시, 사용된 디렉토리 이름, 로그 파일 경로를 입력하면 됩니다(예: http://www.urchin.com/logs/access_log). 브라우저 화면이 텍스트로 가득 채워지면 경로를 올바르게 입력한 것입니다. 모든 사항은 도메인 오른쪽에 입력하십시오(예: /logs/access_log). 원격 HTTP 기반 로그 소스 경로 이름의 경우 와일드카드가 지원되지 않습니다. FTP를 사용하는 경우 원격 서버를 탐색하여 로그 파일을 찾을 수 있습니다. 탐색 전에 서버/사용자 이름 정보를 구성해야 합니다.<p><b>로그 대상</b>(원격 로그만 해당)<br>처리 전에 원격 로그 파일이 로컬 시스템으로 전송됩니다. 로그 파일을 저장하기 위한 로컬 디렉토리를 선택하십시오. 찾기 버튼이 제공됩니다.<p><b>프록시 서버</b>(HTTP와 HTTPS만 해당)<br>프록시 시스템을 통해 로그를 이용할 수 있는 경우 IP 주소 또는 호스트 이름을 지정하십시오. <p><b>프록시 포트</b>(HTTP와 HTTPS만 해당)<br>프록시 시스템을 통해 로그를 이용할 수 있는 경우 포트를 지정하십시오.<p><b>프록시 사용자 이름</b>과 <b>프록시 비밀번호 </b><br>사용자 이름과 비밀번호를 요구하는 프록시 시스템을 통해 로그를 이용할 수 있는 경우 해당 입력란에 사용자 이름과 비밀번호를 입력하십시오.
576	<nobr><span class=title1><b>로그 관리자: 로그 필터 (Ref. #A-1603)</b></span></nobr><br>이 화면을 사용하여 로그 소스에 필터를 적용하십시오. 필터를 적용하려면 왼쪽 상자의 '가용 필터' 상자에서 필터를 선택한 후 오른쪽 방향 화살표를 클릭하십시오. 필터를 제거하려면 오른쪽 상자에서 필터를 선택한 후 왼쪽 방향 화살표를 클릭하십시오. 변경사항을 저장하려면 업데이트를 클릭하십시오.<p><b>제어</b><br><ul><li><b>추가</b><br>필터 마법사를 실행하여 가용 필터 목록에 추가하려면 추가 아이콘을 클릭하십시오. <li><b>수정</b><br>선택된 필터를 수정하려면 수정 아이콘을 클릭하십시오. <li>필터가 적용되는 순서를 변경하려면 위쪽 및 아래쪽 화살표를 클릭하십시오.<br><li>변경사항을 저장하려면 업데이트 버튼을 클릭하십시오.
577	<nobr><span class=title1><b>로그 관리자: 고급 설정 (Ref. #A-1604)</b></span></nobr><br>이 화면에서 처리 완료 이후 로그 파일의 취급 방법과 사용할 인코딩 유형을 포함하여 이 로그 소스의 고급 설정을 구성할 수 있습니다.<p><b>로그 저장 위치</b><br>Urchin은 로그 소스가 적용된 모든 프로필의 로그 소스를 처리하고 나면 해당 로그 파일을 보관, 삭제 또는 방치할 수 있습니다. <ul><li>처리 이후 로그를 방치 상태에 두려면 방치를 선택하십시오. <li>로그 파일을 gzip으로 압축하여 날짜를 표시하려면 보관 – 압축을 선택하십시오. <li>로그를 압축하지 않고 보관하려면 보관 – 비압축을 선택하십시오. 이 선택 항목은 이미 압축된 로그에 유용합니다. <li>처리 이후 로그 파일을 삭제하려면 삭제를 선택하십시오.</ul> Urchin에서 보관 및 삭제 선택 항목을 처리하려면 적절한 권한이 필요합니다. 여러 프로필에 의해 사용되는 로그 소스의 경우 방치 옵션만 선택해야 합니다. <b>경고:</b> 실행 중인 로그 파일은 사용하지 마십시오. <p><b>로그 파일 인코딩</b><br>2바이트 일본어 로그를 처리하는 경우 로그를 Shift-JIS로 변환하십시오. 그렇지 않으면 2바이트 문자가 아닌 모든 로그에 적합한 기본 영어(ISO) 설정을 유지하십시오.<p><b>URL 디코딩</b><br>응용프로그램 서버 또는 기타 동적 페이지 생성 시스템을 사용하고 있고 페이지 URL을 보다 의미있는(보다 쉽게 읽고 이해할 수 있는) 보고서 항목으로 변환하려는 경우 '켜기' 버튼을 클릭하십시오.<p><b>검색어 토큰</b><br>동적으로 작성되는 페이지 URL은 일반적으로 검색어 토큰을 사용하여 URL의 기본 정보와 특정 검색어 정보를 구분합니다. Urchin은 해당 내용이 무엇인지 알아야만 보고서에 대한 URL을 제대로 구문분석할 수 있습니다. 일반적으로 검색어 토큰은 물음표입니다.<p><b>제어</b><br>변경사항을 저장하려면 업데이트 버튼을 클릭하십시오. 화면 상단의 탭을 사용하면 해당 로그 소스에 대해 수정 가능한 모든 설정에 액세스할 수 있습니다.

582	<nobr><span class=title1><b>로그 관리자: 고급 설정 (Ref. #A-1605)</b></span></nobr><br>이 화면에서는 처리 완료 시 로그 파일을 통해 수행할 수 있는 작업을 포함하여 이 로그 소스의 고급 설정을 구성할 수 있습니다. 로그 파일 인코딩 및 디코딩이 필터로 이동되었습니다.<p><b>로그 저장 위치</b><br>로그 저장 위치를 사용하면 로그 파일이 적용되는 모든 프로필에 대해 로그 파일을 처리하고 나서 어떻게 취급해야 하는지 나타낼 수 있습니다. <ul><li>처리가 완료된 후 추가 조치가 필요하지 않다는 것을 표시하려면 방치를 선택하십시오. <li>로그 파일을 압축하여 날짜를 표시하려면 보관 – 압축을 선택하십시오. <li>로그를 압축하지 않고 보관하여 날짜를 표시하려면 보관 – 비압축을 선택하십시오. 이 선택 항목은 이미 압축된 로그에 유용합니다. <li>처리 이후 로그 파일을 삭제하려면 삭제를 선택하십시오. </ul> Urchin에서 보관 및 삭제 선택 항목을 처리하려면 적절한 권한이 필요합니다. 여러 프로필에 의해 사용되는 로그 소스의 경우 방치 옵션만 선택해야 합니다. <b>경고:</b> 실행 중인 로그 파일에 보관 또는 삭제 옵션을 사용하지 마십시오. <p><b>URI 스템을 소문자로 변환</b><br> 이 옵션을 선택하면 동일한 파일이 대문자와 소문자 모두가 될 수 있는 Windows 플랫폼에서 생성되는 로그 URL을 강제로 소문자로 변경하여 콘텐츠 보고서에 해당 파일에 서로 다른 두 항목이 표시되는 것을 방지합니다.<p><b>검색어 토큰</b><br>동적으로 작성된 페이지 URL은 일반적으로 검색어 토큰을 사용하여 URL의 기본 정보와 특정 검색어 정보를 구분합니다. Urchin은 해당 내용이 무엇인지 알아야만 보고서에 대한 URL을 제대로 구문분석할 수 있습니다. 일반적으로 검색어 토큰은 물음표입니다.<p><b>로그 경로 이름의 날짜/시간 와일드카드 대체</b><br>로그 파일 경로 이름에 현재 시간에 기초한 날짜 표시 와일드카드를 포함하면 Urchin을 자동화하여 올바른 로그 파일을 열 수 있습니다. 그러나 일반적으로 Urchin은 로그 파일이 순환되어 날짜가 표시된 다음에 실행됩니다. Urchin이 올바른 로그 파일을 찾으려면 날짜 와일드카드 교체 시에 시간을 조정하는 방법을 알아야 할 수도 있습니다. 기본적으로 Urchin은 어제의 로그 파일(현지 – 24시간)을 찾습니다. 와일드카드는 온라인 설명서 센터에 설명된 <i>strftime</i> 규칙을 따릅니다. 기간은 로그 파일 이름의 srtftime 호환 문자열을 로그 파일 이름에 일치시킬 때 Urchin이 제 때 복귀할 수 있는 일수를 나타냅니다. 이 설정을 사용하면 현재 로그 파일이 최신이 아니고 기간 범위 내 다음 날에 대한 로그 파일이 제공될 경우 Urchin에서 최신 로그 파일로 처리할 수 있어 매우 유용합니다.<p><b>제어</b><br>변경사항을 저장하려면 업데이트 버튼을 클릭하십시오. 화면 상단의 탭을 사용하면 해당 로그 소스에 대해 수정 가능한 모든 설정에 액세스할 수 있습니다.

876	<b>프로필: 목표</b><p>화면 상단에서 해당 탭을 클릭하여 목표를 수정합니다. 그런 다음 오른쪽 상단의 <img alt='Analysis Options Button' title='magic wand' src ='/uicons/default/wizard.gif' border=''> 이미지를 클릭하여 목표 설정 마법사를 활성화합니다. 마법사에 포함된 웹브라우저를 사용하여 유입경로 단계 또는 목표로 설정할 페이지로 이동합니다. 해당 페이지를 1단계로 선택하려면 <b>페이지를 1단계로 설정</b>을 클릭합니다. 유입경로 단계별로 이와 동일한 작업을 합니다. <b>페이지를 목표로 설정</b>을 클릭하여 해당 페이지를 목표로 설정하고 마법사를 닫습니다. 선택된 페이지들이 단계 입력란과 목표 웹 주소 입력란에 표시됩니다. 각 단계에 대한 목표 웹 주소 입력란 및 웹 주소 입력란에 URL을 입력할 때는 <i>http://www.domain.com</i> 부분을 포함하지 않아도 됩니다. Urchin에서는 URL의 이 부분이 무시됩니다.<p>중요: Urchin에서는 단계 및 목표에 대한 모든 웹 주소 입력란에 정규식을 사용할 수 있습니다. 웹 주소의 마지막 슬래시('/') 뒤에 오는 정규식 메타 문자, 즉 '?' 또는 '.'와 같은 문자는 역슬래시 문자('\')를 사용하여 이스케이프 처리해야 Urchin에서 문자로 인식할 수 있습니다. <p><b>목표 설정</b><br>Urchin은 <b>목표 활성화</b>가 '사용'으로 설정될 때마다 <b>목표 웹 주소</b>에서 목표 전환값으로 지정한 URL의 각 방문수를 계산합니다. <b>목표 이름</b>에 입력하는 이름은 Urchin 보고서에 표시됩니다. <b>목표값(수익)</b>은 투자수익(ROI)과 다른 가치평가 통계를 계산하는 데 사용됩니다.<p><b>목표 유입경로 설정</b><br>해당 섹션의 목표로 이어지는 단계들을 지정합니다. 각 단계는 목표로 전환되기 위한 경로의 페이지입니다. 단계들을 합치면 하나의 전환 유입경로가 됩니다. 1040	<nobr><span class=title1><b>CPC 소스 관리자 (Ref. #A-1500)</b></span></nobr><br>이 화면을 사용하면 이 Urchin 설치 구성에 사용된 CPC 소스를 표시하고, 각 CPC 소스의 매개변수를 조정하고, 관리자가 새로운 CPC 소스를 추가할 수 있습니다. CPC 소스를 개별적으로 프로필에 적용할 수 있습니다.<p><b>추가</b><br>CPC 소스를 추가하려면 화면 오른쪽 상단의 추가 버튼을 클릭하십시오. CPC 소스 추가 마법사가 시작되고 간단한 단계들을 거쳐 CPC 소스를 추가하게 됩니다.<p><b>수정</b><br>목록에 표시된 각 CPC 소스 옆에는 일련의 버튼들이 있으며, 이러한 버튼들을 이용하여 각 CPC 소스를 조정할 수 있습니다. 수정 버튼을 클릭하면 탭 형식으로 구성된 CPC 소스의 모든 설정에 액세스할 수 있습니다.<p><b>삭제</b><br>CPC 소스 옆의 삭제 버튼을 클릭하면 구성 및 이를 추천하는 모든 프로필에서 해당 CPC 소스를 제거할 수 있습니다. <p><b>다운로드</b><br>다운로드 버튼을 클릭하면 CPC 소스 보고서를 즉시 다운로드하거나 보고서 실행 주기를 예약할 수 있는 다운로드 화면으로 바로 이동하게 됩니다.<p><b>제어</b><br>#표시 풀다운 메뉴를 사용하면 한 번에 나열되는 항목 수를 변경할 수 있습니다. 이전 또는 다음 버튼을 사용하면 한 번에 한 페이지씩 항목을 스크롤할 수 있습니다. <p><b>계정</b>(데이터센터 모드만 해당)<br>데이터센터 모드 실행 중에 계정 메뉴를 사용하면 나열되는 항목들을 필터링할 수 있습니다.

1042	<nobr><span class=title1><b>CPC 소스 관리자: CPC 설정 (Ref. #A-1501)</b></span></nobr><br>이 화면에서 CPC 소스 설정을 구성하고 조정할 수 있습니다. CPC 소스에 대한 자세한 내용은 사용 설명서 또는 Google의 온라인 도움말 센터를 추천하십시오.<p><b>선택 계정</b>(데이터센터 모드만 해당)<br>새로운 CPC 소스에서 이 옵션을 사용하면 해당 CPC 소스에 연관되는 계정을 선택할 수 있습니다. 기존 CPC 소스에서는 CPC 소스가 속한 계정 이름이 표시됩니다.<p><b>CPC 소스 이름</b><br>CPC 소스를 추천할 때 편리하게 사용할 수 있도록 임의의 이름을 지정하는 필수 입력란입니다. CPC 소스에 대한 고유한 이름을 입력하십시오.<p><b>CPC 소스 유형</b><br>Urchin에서는 다음과 같은 유형의 CPC 소스를 만들 수 있습니다.<p><b>Google 애드워즈</b><br>Google 애드워즈 CPC 소스에 입력해야 하는 데이터는 다음과 같습니다. <li><b>이메일</b>: 광고주의 이메일 주소 또는 내 고객센터(MCC) 로그인 ID입니다(필수사항). <li><b>비밀번호</b>: 애드워즈 계정에 액세스할 수 있는 비밀번호입니다(필수사항). <li><b>개발자 토큰</b>: 애드워즈 API 공인 개발자를 식별할 수 있는 고유 문자열입니다(선택사항). Urchin에는 월 단위로 변경되는 개발자 토큰이 기본으로 제공됩니다. 사용자가 개발자 토큰을 제공하지 않으면 Urchin은 기본 개발자 토큰을 사용합니다. 애드워즈 API는 해당 CPC 소스에서 이루어지는 모든 API 호출에 대해 개발자 토큰으로 비용을 청구합니다. 따라서 사용자가 개발자 토큰을 사용할 경우 API 사용에 따른 비용이 해당 사용자의 로그인 또는 내 고객센터 계정에 청구됩니다. 기본 용도로 사용할 경우에는 항상 이 입력란을 비워두십시오. <li><b>응용프로그램 토큰</b>: 애드워즈 API에 액세스할 때 사용하는 응용프로그램 토큰입니다. 개발자 토큰을 제공한 경우 응용프로그램 토큰도 제공해야 합니다. 그렇지 않을 경우 이 입력란을 비워 두십시오. <li><b>클라이언트 이메일 및 고객 ID</b>: 광고주의 이메일 대신 내 고객센터 로그인 이메일을 사용하는 경우 둘 중 하나의 입력란을 작성해야 합니다.<p><b>Yahoo! Search Marketing</b><br>Yahoo! CPC 소스에 입력해야 하는 데이터는 다음과 같습니다. <li><b>사용자 이름</b>: 액세스하고 있는 Yahoo Search Marketing 계정의 사용자 이름입니다(필수사항). 광고주의 이메일 주소 또는 광고주를 대신해 보고서에 액세스하는 대행사의 이메일 주소입니다. <li><b>비밀번호</b>: Yahoo Search Marketing 계정에 액세스할 수 있는 비밀번호입니다(필수사항). <li><b>마스터 계정 ID</b>: 광고주의 마스터 계정 ID입니다(필수사항). <li><b>계정 ID</b>: 액세스하고 있는 Yahoo! Search Marketing의 계정 ID입니다(필수사항). <li><b>라이센스 키</b>: 로그인한 사용자, 광고주 또는 대행사의 라이센스 키입니다(필수사항). <li><b>대리 사용자 이름</b>: 광고주의 대리 사용자 이름입니다. 광고주가 아닌 대행사로 로그인한 경우에만 필수 입력사항입니다(선택사항). <li><b>대리 비밀번호</b>: 광고주의 대리 비밀번호입니다(선택사항). <p><b>CPC 보고서 다운로드 사용</b><br>(CPC 소스 수정 과정에서만 표시됨) 해당 CPC 소스에 대한 보고서 다운로드 여부를 지정하는 옵션입니다. <b>참고</b>: 앞으로 보고서를 다운로드할 수 없도록 선택해도 다운로드된 기존의 보고서는 해당 CPC 소스가 연관된 프로필에서 계속 사용할 수 있습니다.
