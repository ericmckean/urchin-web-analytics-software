184	<nobr><span class=title1><b>Paramètres globaux : Mise à jour automatique (Ref. #A-2704)</b></span></nobr><p><b>Versions<br></b>La fonction de mise à jour automatique permet de mettre à jour les composants Urchin suivants de manière indépendante :<ul><li>Urchin : contient tous les composants sauf les bases de données géographiques (environ 50 Mo)</li><li>Articles d'aide : contient l'aide intégrée et les articles du centre d'aide (environ 5 Mo par langue)</li><li>Bases de données géographiques : contient toutes les bases de données géographiques (environ 200 Mo)</li></ul>Actions disponibles :<ul><li>vérifier maintenant : rechercher les mises à jour</li><li>afficher la liste des modifications : afficher la liste des modifications d'une nouvelle version</li><li>planifier un téléchargement maintenant : lancer le téléchargement d'une nouvelle version</li><li>annuler : annuler le téléchargement</li><li>installer maintenant : lancer le processus d'installation</li><li>afficher le journal d'erreurs : consulter les erreurs</li></ul></p><p><b>Planifier la mise à jour<br></b>Utilisez ces commandes pour spécifier si les composants doivent être mis à jour et, si oui, quand cette opération doit avoir lieu. Par défaut, ils sont mis à jour quotidiennement. Si cette option est activée, Urchin télécharge automatiquement les mises à jour, une fois par jour à l'heure spécifiée. Les mises à jour de la base de données géographique étant relativement volumineuses (environ 200 Mo), leur téléchargement peut avoir un impact sur la bande passante du réseau.</p><p><b>Paramètres proxy HTTP (facultatif)<br></b>Si un proxy est nécessaire pour l'accès sortant vers le serveur de téléchargement Google qui héberge les mises à jour des données géographiques, utilisez ces champs pour spécifier le nom et le port du serveur, ainsi qu'un nom d'utilisateur et un mot de passe.</p><p><b>Mettre à jour les paramètres<br></b>Si cette option est activée, les articles d'aide ou les bases de données géographiques seront mis à jour en fonction de l'heure configurée, de façon automatisée et sans intervention de l'utilisateur.</p>
185	<nobr><span class=title1><b>Profils : Stockage/Base de données (Ref. #A-2108)</b></span></nobr><br>Cet écran permet de définir des options de gestion des données. Urchin propose des options de sauvegarde automatique et de récupération des données pour la tolérance de pannes. Utilisez ces options pour travailler sans risque. Urchin propose également une option d'archivage automatique qui peut compresser automatiquement les données obsolètes. <p><b>Utilisation de la mémoire de la base de données</b><br>En fonctionnement normal, Urchin utilise un système cache pour limiter la quantité de mémoire RAM utilisée par le moteur de traitement. Les performances peuvent être augmentées pour exécuter l'intégralité du processus en mémoire, mais cette méthode doit uniquement être utilisée sur des systèmes Urchin dédiés. De plus, sachez que ce processus risque de consommer toute la mémoire RAM disponible sur le système. Pour la plupart des opérations, choisissez "Limiter avec un système cache" pour cette option.<p><b>Nombre maximal d'enregistrements par table</b><br>Lors de la vérification de sites volumineux, ce paramètre vous permet d'indiquer le nombre maximal d'enregistrements dans les tables des bases de données Urchin et donc de maîtriser la taille des données et les performances d'Urchin. L'option par défaut convient pour la plupart des installations. Si cette limite est atteinte pour un mois donné, les entrées supplémentaires sont regroupées sous une entrée intitulée "Autre". Pour bénéficier de performances optimales, utilisez une valeur inférieure ou égale à 100 000. <p><b>Conserver les informations de suivi brutes</b><br>Lorsque cette option est désactivée, les informations brutes relatives aux sessions et aux visiteurs sont supprimées une fois traitées. Cela permet d'améliorer nettement les performances de traitement des journaux et de réduire la quantité de données stockées pour les sites volumineux. Une fois les données de session brutes supprimées, tous les appels de fichier supplémentaires pour cette session sont considérés comme de nouvelles sessions, ce qui peut provoquer un léger effet secondaire dans la génération des rapports. <p><b>Suivi du journal</b><br>Urchin est équipé d'un outil de suivi de journaux "intelligent" qui lui permet de déterminer si tel ou tel fichier journal a déjà été traité. Une rotation des journaux est ainsi possible chaque fois que cela est utile, ce qui évite une planification rigide. Cette option permet également de renvoyer Urchin à un répertoire de fichiers journaux sans avoir besoin de vérifier si certains de ces fichiers ont déjà été traités. REMARQUE IMPORTANTE : si Urchin ne se trouve pas physiquement sur le système où sont stockés les fichiers journaux, il copie tous les journaux sur le système Urchin local avant de déterminer s'ils ont déjà été traités. Par conséquent, le réseau devient vite encombré en cas d'analyse de fichiers journaux volumineux.<p><b>Restauration de BD automatique</b><br>Lorsque cette option est sélectionnée, les données sont restaurées automatiquement si le traitement a été interrompu ou supprimé. Urchin détecte automatiquement l'interruption ou la suppression de l'opération et restaure les données de la sauvegarde la plus récente (s'il en existe une) avant de continuer.<p><b>Créer des sauvegardes</b><br>Cette option permet la création automatique de sauvegardes après le traitement. Les sauvegardes sont compressées afin de réduire l'espace de stockage requis.<p><b>Nettoyer les sauvegardes</b><br> Cette option permet le nettoyage automatique des sauvegardes antérieures en fonction du nombre de sauvegardes à conserver (spécifié à l'aide de l'option suivante). Si cette option est sélectionnée, Urchin supprime automatiquement les sauvegardes antérieures, permettant ainsi de maîtriser l'espace de stockage. <p><b>Nombre de sauvegardes à conserver</b><br> Cette option spécifie le nombre de sauvegardes à conserver chaque mois lorsque l'option de nettoyage ci-dessus est sélectionnée.<p><b>Archiver la base de données</b><br> Cette option permet d'archiver automatiquement, chaque mois, les données obsolètes. Ces dernières sont automatiquement compressées en un seul dossier d'archives, les sauvegardes sont supprimées et les données verrouillées. Il est toujours possible de consulter les données archivées, mais les appels de fichier ne peuvent plus être traités pour les mois archivés.<p><b>Archiver la base de données après</b><br>Cette option spécifie l'ancienneté (en mois) des données à archiver automatiquement.<p><b>Supprimer TOUTES les données du profil</b><br>Utilisez cette option pour supprimer définitivement TOUTES les données de ce profil, y compris TOUTES les archives et sauvegardes. Il faut ensuite traiter de nouveau tous les fichiers journaux de ce profil. <b>Avertissement</b> : vérifiez que vous disposez de tous les fichiers journaux avant de supprimer ces données.

192	<nobr><span class=title1><b>Planificateur : Progression de la tâche (Ref. #A-2664)</b></span></nobr><br>Cet écran affiche la progression de la tâche en cours d'exécution. Il comporte également des commandes permettant d'annuler cette dernière. Une barre de progression s'affiche en haut de la fenêtre. Elle indique l'état d'avancement approximatif du traitement sous forme de pourcentage. Le bouton “Annuler la tâche” permet de stopper la tâche en cours et de restaurer le profil. Un lien permettant d'accéder aux rapports concernant le profil est disponible au-dessus de la barre de progression. Le résultat de l'exécution fourni par le moteur de traitement du journal s'affiche dans la fenêtre “Détails du traitement de la tâche”. <p>Remarque : Si un fichier journal distant assez volumineux est en cours de traitement, la copie peut prendre du temps. Dans ce cas, la barre de progression reste immobile. Une fois le fichier copié sur l'ordinateur qui exécute Urchin, la mise à jour de la barre de progression devrait se faire de façon continue.
193	<nobr><span class=title1><b>Planificateur : Historique détaillé (Ref. #A-2563)</b></span></nobr><br>Cet écran affiche les détails de l'exécution d'une tâche, qui servent en général au débogage ou à l'évaluation des performances. Si une erreur s'est produite, cet écran indique comment la corriger. <p><b>Problèmes courants</b><br>Parmi les problèmes courants, on trouve les erreurs de type "fichier journal introuvable", qui en général se règlent en corrigeant le chemin d'accès ou le nom du fichier. Si les fichiers journaux sont transmis sur le réseau via un serveur FTP, vérifiez l'exactitude du nom d'utilisateur et du mot de passe en vous connectant au serveur à l'aide d'un programme FTP dédié. Si le temps de traitement ne représente qu'une petite fraction du temps d'exécution total, il existe probablement un problème de copie du fichier journal sur le réseau. Dans ce cas, il est recommandé d'installer Urchin directement sur le serveur.
194	<nobr><span class=title1><b>Navigateur de fichiers (Ref. #A-5000)</b></span></nobr><br>Cet écran permet de naviguer dans le système de fichiers local afin de trouver la source de journal souhaitée. Cette option est particulièrement utile lorsque vous n'êtes pas certain du nom du fichier journal. La zone “Accéder au répertoire” (à gauche) permet de sélectionner un répertoire de navigation. Les fichiers se trouvant dans ce répertoire s'affichent dans la zone “Sélectionner un fichier” (à droite). Une fois que vous avez trouvé le fichier journal recherché, cliquez sur celui-ci dans la zone “Sélectionner un fichier”, puis sur “OK”.<p><b>Répertoire actuel</b><br>Vous pouvez modifier directement le champ “Répertoire actuel” situé en haut de la fenêtre. Appuyez sur la touche “Entrée” pour afficher le contenu de ce répertoire après modification.<p><b>Répertoire supérieur</b><br>Si vous cliquez sur le bouton “Répertoire supérieur”, en haut à droite (icône de dossier avec une flèche), le répertoire actuel avance d'un niveau vers le répertoire parent. Le contenu du nouveau répertoire s'affiche.<p><b>Accéder au répertoire</b><br>La fenêtre de gauche permet de sélectionner un répertoire à l'intérieur du répertoire actuel. Cliquez sur un répertoire apparaissant dans la fenêtre “Accéder au répertoire” pour le sélectionner.<p><b>Sélectionner un fichier</b><br>Pour sélectionner un fichier, utilisez la fenêtre “Sélectionner un fichier” située à droite. Cliquez une fois sur le fichier souhaité pour que le nom du fichier apparaisse dans le champ “Nom de fichier”. Cliquez sur “OK” lorsque vous avez sélectionné le fichier.<p><b>OK/Vérifier/Annuler</b><br>Lorsque vous cliquez sur “OK”, la fenêtre se ferme et votre sélection apparaît dans le champ du chemin d'accès de la fenêtre précédente. Si vous cliquez sur le bouton “Annuler”, la fenêtre se ferme. Si vous cliquez sur le bouton “Vérifier”, une nouvelle fenêtre s'ouvre, vous permettant de vérifier les autorisations et la taille d'un ou de plusieurs fichiers. Il faut pour cela que le nom du ou des fichiers contienne un caractère générique.

249	<nobr><span class=title1><b>Assistant profil : Paramètres généraux (Ref. #A-249)</b></span></nobr><br>Utilisez cet écran pour définir plusieurs paramètres permettant de déterminer la façon dont les données du profil sont traitées.<p><b>URL du site Web</b><br>Ce paramètre permet de créer des liens renvoyant à votre site Web réel. Dans ce champ doit apparaître le nom de domaine complet du site, précédé de http://. <p><b>Domaines du site</b><br>Ce paramètre doit répertorier tous les domaines possibles pour votre site Web, avec ou sans www. Ce champ permet de déterminer si une visite a été générée par un site externe (les visites générées par des liens internes ne sont pas utilisées pour la création de rapports sur les sites référents). Séparez les domaines par des virgules.<p><b>Page par défaut</b><br>Utilisez cette option pour saisir la page par défaut de votre site. Dans la plupart des cas, il est possible d'accéder à la même page via www.site.com/ et www.site.com/index.html ou default.html. Urchin a besoin de connaître le nom de cette page pour pouvoir combiner les entrées en interne lorsque les deux styles sont utilisés.<p><b>Traitement du journal parallèle</b><p>Activez cette option pour une meilleure efficacité du traitement du journal avec des journaux à équilibrage de charge. Pour ceci, notez qu'il est nécessaire de configurer une source de journal distincte pour le fichier journal de chaque serveur utilisé pour équilibrer la charge.

448	<nobr><span class=title1><b>Gestionnaire de filtres : Paramètres de filtrage (Ref. #A-1701)</b></span></nobr><br>Cet écran permet de configurer et d'ajuster les paramètres d'un filtre. Pour plus d'informations sur le filtrage, consultez notre manuel ou notre centre de documentation en ligne.<p><b>Exemples</b><br>Lorsque vous ajoutez un filtre, une liste de modèles de filtres est mise à votre disposition. Si vous sélectionnez l'un de ces modèles, les paramètres du filtre sont automatiquement renseignés et il vous suffit de les modifier pour les personnaliser. Cela permet de voir facilement comment les paramètres sont créés pour divers filtres standard.<p><b>Compte facultatif</b> (mode Datacenter uniquement)<br>Pour les nouveaux filtres, ce paramètre vous permet de sélectionner le compte auquel associer le filtre. Pour les filtres existants, le nom du compte auquel le filtre appartient s'affiche.<p><b>Nom du filtre</b><br>Dans ce champ obligatoire, saisissez le nom de votre choix qui sera utilisé pour désigner ce filtre. Il doit s'agir d'un nom unique.<p><b>Type de filtre</b><br>Sélectionnez un type de filtre en fonction de vos besoins : <li><b>Exclure la règle</b> : ce type de filtre exclut <i>uniquement</i> les lignes du fichier journal (appels de fichier) concordant avec la règle de filtrage. Les lignes qui ne concordent pas sont ignorées : gardez par conséquent à l'esprit que si un filtre exclut Netscape (par exemple), Urchin exclut toutes les informations de cette ligne de journal, notamment le visiteur, le chemin, le site référent et le domaine. <li><b>Inclure la règle</b> : ce type de filtre inclut <i>uniquement</i> les lignes du fichier journal (appels de fichier) concordant avec la règle de filtrage. Toutes les autres sont ignorées. Comme dans l'exemple ci-dessus, le système Urchin ne peut accéder à aucun appel de fichier ayant été ignoré lors de la création de rapports. <li><b>Rechercher et remplacer</b> : ce filtre permet de rechercher une chaîne dans un champ et de la remplacer par une autre. Si la règle apparaît plusieurs fois dans le champ, seule la dernière occurrence est remplacée. Il est possible d'utiliser le caractère "^" pour que la concordance soit appliquée au début du champ, et "$" pour qu'elle soit appliquée à la fin du champ. <li><b>Tableau de conversion</b> : ce filtre permet de choisir un nom de tableau de conversion pouvant servir à faire correspondre les codes avec des libellés intelligibles. Il est nécessaire de créer au préalable un tableau de conversion et de l'installer dans le répertoire lib/custom/lookuptables de la distribution Urchin. Vous pouvez choisir les tableaux de conversion existants dans la zone "Nom du tableau".<li><b>Avancé</b> : ce type de filtre permet de générer un champ à partir d'un ou deux autres champs. Le moteur de filtrage applique alors les expressions des deux champs "Extraire" aux champs spécifiés, puis génère un champ à l'aide de l'expression "Constructeur". La technique est la même que celle de l'utilisation de l'expression régulière POSIX. <p><b>Champ de filtrage</b><br>Utilisez le menu déroulant pour sélectionner le champ de la source de journal auquel le filtre doit être appliqué. Les champs intitulés "RAW" représentent le contenu réel du fichier journal avant traitement. Les champs intitulés "AUTO" sont des champs calculés automatiquement qui sont déterminés en fonction des champs "RAW" et d'autres champs "AUTO". Pour connaître la fonction de chaque champ, consultez la section sur le filtrage dans le centre de documentation en ligne. <p><b>Règle de filtrage</b><br>Choisissez la règle que le filtre doit respecter. Elle peut utiliser des caractères génériques définis par la norme d'expression régulière POSIX.<p><b>Chaîne de recherche</b><br>Pour les filtres "Rechercher et remplacer", la chaîne de recherche est utilisée comme règle pour la concordance avec le champ.<p><b>Chaîne de remplacement</b><br>Pour les filtres "Rechercher et remplacer", la chaîne de remplacement remplace la chaîne de recherche.<p><b>Nom du tableau</b> (tableau de conversion uniquement)<br>Ce champ, qui apparaît lorsque vous sélectionnez "Tableau de conversion" comme type de filtre, permet de choisir un tableau faisant correspondre les codes avec des noms intelligibles par l'homme. Le tableau de modèles de téléphone fait correspondre les identifiants de plateforme téléphonique abrégés avec les noms de modèle et de fabricant pour les navigateurs Web utilisés à partir d'un téléphone. Il est possible de créer ses propres tableaux de conversion afin de faire correspondre des codes aux campagnes (pour le module de suivi de campagne) ou de faire correspondre les variables d'une URL à des noms de pages. Tout fichier associé à l'extension .lt créé dans le répertoire lib/custom/lookuptables de la distribution Urchin apparaît dans ce menu déroulant.<p><b>Nom de fichier</b> (tableau de conversion externe uniquement)<br> Ce champ, qui apparaît lorsque vous sélectionnez "Tableau de conversion" comme type de filtre, permet de choisir un tableau de conversion externe, faisant correspondre les codes avec des noms intelligibles par l'homme. Vous pouvez créer vos propres tableaux de conversion afin de faire correspondre des codes aux campagnes (pour le module de suivi de campagne) ou de faire correspondre les variables d'une URL à des noms de pages.<p><b>Nom d'utilisateur/Mot de passe</b> (Windows, tableau de conversion externe uniquement)<br>Ces champs permettent de spécifier les informations d'identification au partage UNC sur lequel le tableau de conversion externe est situé.<p><b>Commandes</b><br>Pour enregistrer les changements, cliquez sur le bouton "Terminer" (pour un nouveau filtre) ou "Mettre à jour" (pour un filtre existant). Pour les filtres existants, les onglets du haut de l'écran permettent d'accéder à tous les paramètres modifiables pour ce filtre.

544	<nobr><span class=title1><b>Paramètres globaux : Paramètres de la base de données (Ref. #A-2302)</b></span></nobr><br>Cet écran permet de spécifier des paramètres généraux visant à contrôler les performances en fonction de l'utilisation des ressources. En général, les valeurs par défaut offrent le meilleur compromis en termes de vitesse/processeur et utilisation du disque. Toutefois, vous pouvez modifier ces paramètres si vous souhaitez augmenter la vitesse d'Urchin ou réduire son impact sur l'ordinateur sur lequel il est installé. <p><b>Nombre maximal d'enregistrements par table</b><br>Dans le cas de sites volumineux, ce paramètre permet d'indiquer le nombre maximal d'enregistrements dans les tables et donc de maîtriser la taille de la base de données Urchin. Notez que si vos rapports contiennent de nombreuses entrées "Autre", il faudra augmenter la valeur de ce paramètre. Pour bénéficier de performances optimales, utilisez une valeur inférieure ou égale à 100 000<p><b>Cible d'utilisation de la mémoire</b><br>Ce paramètre fournit des indications au processus Urchin quant à la quantité de mémoire à utiliser lors de l'exécution. Vous pouvez augmenter l'empreinte mémoire du moteur de traitement des journaux (environ 350 Mo pour les réseaux, 100 Mo pour les domaines, et entre 20 et 150 Mo pour les origines géographiques, en fonction de la base de données géographique choisie) en sélectionnant des options supplémentaires. Urchin comprend un certain nombre de bases de données géographiques, qui ne sont pas toutes aussi détaillées en termes d'informations sur le pays ou la région. À partir du menu déroulant, vous pouvez sélectionner la base de données géographique qui correspond le mieux à vos besoins.<p><b>Base de données géographique</b><br>Urchin comprend des bases de données géographiques, qui fournissent différents niveaux d'informations concernant l'origine géographique :<br>- Monde (informations détaillées sur le pays, la région et la ville, pour le monde entier)<br>- Monde version allégée (informations au niveau du pays uniquement, pour le monde entier)<br>- Bases de données régionales (États-Unis, Union européenne et Japon, par exemple), contenant des informations détaillées au niveau des régions et pays correspondants, pour le reste du monde). <br><br>Les paramètres des bases de données géographiques peuvent être définis de façon globale ou au niveau du profil.<p><b>Commandes</b><br>Cliquez sur le bouton "Mettre à jour" pour enregistrer les modifications. Notez que le format de journaux Google ne peut pas être détecté automatiquement.<li><b>NCSA :</b> ce format peut en général être détecté automatiquement. Il est généralement utilisé par les serveurs Web Apache. Notez que, pour une création complète de rapports, il faut utiliser la connexion combinée étendue, qui ne correspond en général pas au comportement d'Apache par défaut. <li><b>Netscape :</b> les serveurs Netscape existants (mais pas les serveurs iPlanet) utilisent ce format par défaut.<li><b>Overture :</b> utilisez ce format pour les journaux contenant des données de coût par clic Overture. 569	<nobr><span class=title1><b>Profils : Création de rapports (Ref. #A-2106)</b></span></nobr><br>L'écran "Paramètres de rapport" permet de contrôler plusieurs fonctionnalités importantes de traitement dans Urchin, notamment l'identification de certains paramètres de filtres et de proxy.<p><b>URL du site Web</b><br>Ce paramètre permet de créer des liens renvoyant à votre site Web réel. Dans ce champ doit apparaître le nom de domaine complet du site, précédé de http://. <p><b>Domaines du site</b> (super administrateur uniquement)<br>Ce paramètre doit répertorier tous les domaines possibles pour votre site Web, avec ou sans "www". Ce champ permet de déterminer si un site référent est externe à votre site. Séparez les domaines par des virgules.<p><b>Page par défaut</b><br> Elle doit être définie sur la page (ou l'index) par défaut des répertoires de ce site. Il s'agit en général de "index.html", mais sur les serveurs Windows IIS, il s'agit souvent de "Default.htm" ou "index.htm". Ces informations permettent à Urchin de corriger les entrées du journal telles que "http://www.urchin.com/" et "http://www.urchin.com/index.html", qui correspondent en réalité à la même page. Si les informations sur la page par défaut ne sont pas entrées correctement, ces deux pages seront considérées comme distinctes. Vous ne pouvez définir qu'une seule page par défaut. <p><b>Paramètres de serveur proxy</b> (super administrateur uniquement)<br>Certaines installations peuvent nécessiter des paramètres de proxy afin d'afficher les rapports "Synthèse données/site" de la page d'accueil. Étant donné que les requêtes de synthèse données/site sont lancées à partir du système Urchin, il est possible que les paramètres du pare-feu n'autorisent pas de communication externe. Dans ce cas, vous pouvez utiliser un proxy pour autoriser ce type de requête. Le serveur proxy doit correspondre au nom d'hôte du serveur. Le port proxy doit correspondre au numéro de port utilisé pour communiquer avec le proxy. Les champs "nom d'utilisateur" et "mot de passe" du proxy peuvent également fournir une sécurité accrue si un nom d'utilisateur et un mot de passe sont exigés pour accéder au proxy. Laissez tous ces champs vides, sauf si vous obtenez un message d'erreur lorsque vous essayez d'utiliser la fonctionnalité "Synthèse données/site". Dans ce cas, contactez votre administrateur réseau pour obtenir les paramètres.<p><b>Traitement du journal parallèle</b> (super administrateur uniquement)<br>Si vous avez de nombreux journaux à équilibrage de charge, le fait de les exécuter en parallèle vous permet d'améliorer les performances du système. Cela dépend de nombreux facteurs, notamment la quantité de données et les performances E/S de la machine. Nous vous conseillons d'activer ce paramètre pour améliorer les performances, en particulier lorsque des appels de fichier provenant d'une seule session utilisateur sont susceptibles d'être présents dans plusieurs journaux. <p><b>Décalage horaire</b> (super administrateur uniquement)<br>Ce paramètre détermine le fuseau horaire dans lequel Urchin établit les rapports de données. Tous les appels de fichier contenant une référence GMT, Urchin peut établir des rapports dans l'ensemble des fuseaux horaires. L'option "Heure locale" permet d'enregistrer les données en fonction du fuseau horaire configuré sur le serveur exécutant Urchin. Pour utiliser un paramètre différent, sélectionnez le décalage par rapport à GMT. <p><b>Correspondance de mots clés naturels</b> (super administrateur uniquement)<br>Ce paramètre contrôle la manière dont Urchin détecte les moteurs de recherche et les mots clés pour les rapports. À l'aide de cette liste (séparée par des virgules), Urchin recherche les paramètres des requêtes sur les sites référents présentant une correspondance, puis capture la valeur du paramètre, c'est-à-dire le mot clé ou le terme de recherche sur le site référent. Vous pouvez ajouter des entrées à cette liste pour les termes de recherche qui n'ont pas été inclus automatiquement. La liste par défaut peut être utilisée par la plupart des moteurs de recherche.<p><b>Correspondance Mime des pages vues</b> (super administrateur uniquement)<br>Cette liste séparée par des virgules contrôle les extensions de fichiers traitées comme pages vues. Elle permet d'inclure ou d'exclure certaines extensions. La liste par défaut peut être utilisée par la plupart des sites.<p><b>Correspondance des paramètres de page</b><br>Cette variable contrôle les paramètres qui devront être capturés par Urchin dans le rapport sur les paramètres. Les paramètres proviennent de la partie requêtes de l'URL demandée et peuvent contenir des termes de recherche internes ou tout autre contenu dynamique. Urchin capture automatiquement ces paires nom-valeur et les inclut dans le rapport sur les paramètres. Cette liste séparée par des virgules permet d'exclure ou d'inclure certains paramètres. Par exemple, pour un site contenant des millions d'identifiants de session aléatoires, exclure la variable SID de ce rapport permettra d'économiser des ressources de stockage.<p><b>Correspondance des téléchargements</b> (super administrateur uniquement)<br> Cette liste séparée par des virgules contrôle les extensions de fichiers traitées comme des fichiers téléchargés dans le rapport sur les téléchargements. Modifiez la liste pour inclure les extensions de fichiers utilisées pour les téléchargements. <p><b>Détail géographique</b><br><p><b>Informations géographiques</b><br>Ce paramètre permet de définir quelles informations géographiques seront incluses dans les rapports sur les profils. Vous pouvez augmenter l'empreinte mémoire du moteur de traitement des journaux (environ 350 Mo pour les réseaux, 100 Mo pour les domaines, et entre 20 et 150 Mo pour les origines géographiques, en fonction de la base de données géographique choisie) en sélectionnant des options supplémentaires. Urchin comprend un certain nombre de bases de données géographiques, qui ne sont pas toutes aussi détaillées en termes d'informations sur le pays ou la région. À partir du menu déroulant, vous pouvez sélectionner la base de données géographique qui correspond le mieux à vos besoins.<p><b>Base de données géographique</b><br>Urchin comprend des bases de données géographiques, qui fournissent différents niveaux d'informations concernant l'origine géographique :<br>- Monde (informations détaillées sur le pays, la région et la ville, pour le monde entier)<br>- Monde version allégée (informations au niveau du pays uniquement, pour le monde entier)<br>- Bases de données régionales (États-Unis, Union européenne et Japon, par exemple), contenant des informations détaillées au niveau des régions et pays correspondants, pour le reste du monde) <br> <p><b>Codage de caractères</b><br>Si les journaux que vous allez traiter contiennent des données non codées selon la norme UTF-8, choisissez un codage approprié. 575	<nobr><span class=title1><b>Gestionnaire de journaux : Paramètres distants (Ref. #A-1602)</b></span></nobr><br>Cet écran permet de spécifier comment extraire ou accéder au fichier journal distant de cette source de journal.<p><b>Protocole de serveur</b><br>Urchin peut gérer quatre types de fichiers journaux distants : HTTP (www), HTTPS, FTP et UNC (Windows uniquement). Si votre fichier journal est accessible via un navigateur Web, utilisez HTTP. Si aucun mot de passe n'est requis, laissez les champs de nom d'utilisateur et mot de passe vides. FTP peut être utilisé avec un nom d'utilisateur et un mot de passe (en général, les mêmes que le nom d'utilisateur permettant d'envoyer un contenu sur votre site). UNC, qui n'est disponible que sur les systèmes Windows, nécessite un nom d'utilisateur et un mot de passe pour accéder au partage UNC.<p><b>Nom/Adresse IP du serveur</b><br>Entrez le nom du serveur (par exemple, www.urchin.com) ou l'adresse IP (par exemple, 63.212.171.4) correspondant à l'emplacement du fichier journal. Pour UNC, utilisez la notation \\server\share.<p><b>Connexions locale et distante identiques</b><br>Cette option est visible si UNC a été sélectionné comme protocole de serveur (Windows uniquement). Sélectionnez "Oui" si le même nom de connexion et mot de passe peuvent être utilisés pour les serveurs local et distant. Sélectionnez "Non" si le serveur distant requiert un nom de connexion ou mot de passe différent du serveur local. Les champs supplémentaires de connexion au serveur local deviennent visibles si vous sélectionnez "Non".<p><b>Port du serveur</b><br>Le port du serveur est spécifié automatiquement. Cependant, si vous n'utilisez pas un port standard, vous pouvez le modifier manuellement.<p><b>Nom d'utilisateur/Mot de passe</b><br>Ces champs sont vides pour les connexions FTP anonymes et HTTP non sécurisées. Pour utiliser une connexion HTTP/FTP sécurisée, entrez votre nom d'utilisateur et votre mot de passe ici. UNC nécessite un nom d'utilisateur et un mot de passe pour accéder au partage. <p><b>Domaine (UNC uniquement)</b><br>Si le partage UNC appartient à un domaine, entrez ici les informations sur le réseau du domaine. <p><b>Mode FTP</b><br>Cette option est visible si FTP a été sélectionné comme protocole de serveur. Sélectionnez "Passive" pour une méthode de transfert passive (PASV). Sélectionnez "Active" pour une méthode de transfert directe (PORT). <p><b>Fichier source de journal distant</b><br>Le fichier source de journal distant spécifie le chemin d'accès au fichier journal sur le système distant. Si vous utilisez HTTP, il est souvent plus facile de localiser le fichier journal avec un navigateur Web pour commencer. Pour cela, entrez l'URL du site dans un navigateur, suivi d'une barre oblique, de tous les noms de répertoire utilisés et du nom du fichier journal (par exemple, http://www.urchin.com/logs/access_log). Si l'écran du navigateur se remplit de texte, le chemin d'accès est correct. Il vous suffit de tout entrer à droite du domaine (par exemple, /logs/access_log). Notez que les caractères génériques ne sont pas pris en charge pour les noms de chemin d'accès distants aux journaux basés sur HTTP. Si vous utilisez FTP, vous pouvez parcourir le serveur distant pour trouver le fichier journal. Avant cela, il vous faut configurer les informations sur le nom d'utilisateur/serveur.<p><b>Destination du journal</b> (à distance uniquement)<br>Pour être traités, les fichiers journaux distants sont transférés sur le système local. Sélectionnez le répertoire local où sera stocké le fichier. Un bouton de navigation est disponible.<p><b>Serveur proxy</b> (HTTP et HTTPS uniquement)<br>Si le journal est disponible via un ordinateur proxy, spécifiez l'adresse IP ou le nom d'hôte. <p><b>Port proxy</b> (HTTP et HTTPS uniquement)<br>Si le journal est disponible via un ordinateur proxy, spécifiez le port.<p><b>Nom d'utilisateur du proxy</b> et <b>Mot de passe du proxy</b><br>Si le journal est disponible via un ordinateur proxy qui requiert un nom d'utilisateur et un mot de passe, spécifiez-les dans ces champs.

Le codage et le décodage du fichier journal ont été déplacés dans les filtres.<p><b>Avenir du journal</b><br>Permet d'indiquer comment les fichiers journaux seront gérés après leur traitement pour tous les profils auxquels ils s'appliquent. <ul><li>Sélectionnez "Ne pas toucher" pour indiquer qu'aucune autre action ne doit être entreprise après le traitement. <li>Sélectionner "Archiver - Compresser" pour enregistrer le fichier journal au format zip et lui attribuer une date. <li>Sélectionner "Archiver - Aucune compression" pour archiver et dater un journal sans le compresser. Pour les sources de journal utilisées par plusieurs profils, seule l'option "Ne pas toucher" doit être sélectionnée. <b>Avertissement :</b> n'utilisez pas les options "Archiver" ou "Supprimer" avec des fichiers journaux en ligne. <p><b>Racine de l'URL en minuscule</b><br> Lorsqu'elle est activée, cette option impose automatiquement les minuscules dans l'URL pour les journaux créés sur des plateformes Windows où l'URL d'un même fichier peut être en majuscules ou en minuscules, ce qui peut entraîner la présence de deux entrées dans les rapports de contenu.<p><b>Jeton</b><br>Les URL de pages générées dynamiquement utilisent en général un jeton pour séparer la base de l'URL des informations de requêtes spécifiques. Urchin a besoin de le connaître afin de pouvoir analyser correctement l'URL pour ses rapports. Le jeton est généralement un point d'interrogation.<p><b>Substitution de la date et de l'heure par des caractères génériques dans le nom du chemin d'accès au journal</b><br>Pour automatiser Urchin et faire en sorte qu'il ouvre les fichiers journaux appropriés, le nom du chemin d'accès au fichier journal peut contenir des caractères génériques horodatés pouvant être basés sur l'heure actuelle. Il est néanmoins commun pour Urchin de s'exécuter après le roulement et l'horodatage des fichiers journaux. Pour qu'Urchin trouve le bon fichier journal, il peut avoir besoin de savoir comment ajuster l'heure lors du remplacement des caractères génériques de la date. Par défaut, Urchin recherchera les fichiers journaux d'hier (local - 24 heures). L'utilisation de caractères génériques suit la convention <i>strftime</i> décrite dans le centre de documentation en ligne. "Nombre de jours" indique le nombre de jours de retour dans le temps pour Urchin lors du ciblage d'une chaîne compatible strftime dans le nom du fichier journal. Ce paramètre est très utile dans les environnements où le fichier journal du jour n'est peut-être pas présent. Il permet à Urchin de rattraper son retard dans le traitement si le fichier journal devient disponible à une date ultérieure comprise dans le nombre de jours indiqué dans la fenêtre.<p><b>Commandes</b><br>Cliquez sur le bouton "Mettre à jour" pour enregistrer les modifications. Les onglets du haut de l'écran permettent d'accéder à tous les paramètres modifiables pour cette source de journal.

L'association de la valeur "Oui" au paramètre <b>Étape 1 obligatoire</b> indique que seuls les chemins vers une conversion contenant la première étape seront inclus dans les rapports sur l'entonnoir de conversion.
