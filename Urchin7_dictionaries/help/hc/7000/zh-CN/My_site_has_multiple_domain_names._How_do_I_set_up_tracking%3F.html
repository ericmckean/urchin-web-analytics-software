我的网站有多个域名，如何设置跟踪？

如果您的网站使用位于不同根域上的安全购物车（例如 www.mystore.com 和 www.securecart.com），或使用多个指向同一内容的域别名，则您需要进行某些更改，以确保网站和 UTM 兼容。

  • 请注意多个子域和多个根域间的区别。 www.example.com 和 securecart.example.com 是多个子域的示例，而 www.example.com 和 www.sample.com 代表多个根域。 如果网站有多个子域，请参阅： 如何在一个配置文件中为我的网站跟踪所有子域？ 如何在单独的配置文件中为我的网站跟踪所有子域？ 用于配置信息。

多个域名：域别名

如果您的网站使用多个域别名，建议您将别名的访问量重定向到主网站。这将确保 UTM 的访问者跟踪在主域下开始，并且所有访问者都被连续跟踪。

下面将演示如何将使用别名的域重定向到 Apache 或 IIS 服务器上的主域。

在 Apache 中重定向别名

  • 如果您正在使用 Apache 网络服务器，可方便地更改配置，将某一别名下发生的访问量重定向到主站点。实现此目的的一种方法是创建两个 VirtualHost 实体。第一个 VirtualHost 为包含正常配置的主域，第二个 VirtualHost 用于所有别名并重定向到主域： 

        #---primary virtualhost
        Servername www.mysite.com
        Serveralias mysite.com
        ...
        #---second virtualhost
        Servername mysite.org
        Serveralias www.mysite.org mysite.net www.mysite.net
        RewriteEngine on
        RewriteRule ^(.*) http://www.mysite.com$1 [R=301]

  • 第二个 VirtualHost 使用重写入规则和 301（永久移动）重定向代码将所有访问量重定向到原始站点。此时，对 UTM 而言，该网站是一个单域网站，可以正常进行 UTM 安装。

在 IIS 中重定向别名

  • 如果您正在使用 Microsoft IIS 网络服务器，可方便地更改配置，将某一别名下发生的访问量重定向到主站点。实现此目的的一种方法是在 IIS 配置中创建两个网站。第一个网站为包含正常配置的主域 (www.mysite.com)，第二个网站用于所有其他别名（mysite.net、mysite.org 等）并重定向到主域。 在 IIS Manager 中，右键点击其中一个网站，打开属性对话框。在"网站"标签上，点击"高级..."按钮。这将打开一个窗口，您可在其中使用"主机标题"字段为网站指定其他域名。在主网站中设置主域，使用第二个网站容纳所有别名。 在配置并启用容纳所有别名的第二个网站后，使用下列重定向代码创建空白主页： 

        <META HTTP-EQUIV=Refresh CONTENT="0; URL=http://www.mysite.com/">
  • 这将指示访问者的浏览器立即重定向到主网址。此时，主网站成为一个简单的单域配置，可使用默认设置正常进行 UTM 安装。
