如何将内部访问者从网站报告中删除？

概述

要仅包含来自特定 IP 地址的项目、排除内部监控系统的 IP 地址等，最常见的做法是将 IP 地址过滤器与配置文件配合使用。Urchin 可让您轻松完成这项工作。但是，请务必使用正确的 '过滤区域' 规范设置 IP 地址过滤器。

步骤

要配置适当的内部访问量过滤器，最简单的方法是按照下列步骤操作：

  • 在 Urchin 管理界面中，进入"配置"->"Urchin 配置文件"->"过滤器管理器"

  • 点击"添加"按钮，创建新的过滤器

  • 从"=> 选择示范过滤器"下拉列表中选择"滤除内部访问量"

  • 以想要滤除的 IP 地址替换"过滤模式"框中的示范 IP 地址。请不要忘记在 IP 地址中的每一点之前加上反斜线（"\"）字符！
  • 点击"完成"，然后在下一个窗口中点击"更多选项"
  • 在"可用配置文件"框中，选择所有希望应用过滤器的配置文件，然后点击向右箭头，将它们发送到"应用于"框中。
  • 点击"更新"，将过滤器应用于选定的配置文件。

其他注意事项

请注意，Urchin 过滤模式采用 POSIX 正则表达式。因此，请务必对 IP 地址规范中的 '.' 字符进行转义，以避免 Urchin 将它们解译为正则表达式的元字符。例如，要过滤 IP 地址 '172.16.1.1'，可将过滤模式指定为 '172\.16\.1\.1'。再举一个例子，要过滤 IP 地址范围 '192.168.xxx.xxx，可将过滤字段指定为 '192\.168\.'。以下是其他一些示例：

以管道符号来分隔多个 IP 地址，例如 192\.168\.12\.32$|10\.10\.1\.23$|63\.212\.171\.5$；请注意，使用 $ 表示在最后显示的数字之后没有任何其他数字。因此，要同时包含 63.212.171.5 和 63.212.171.53，可将过滤器写为 63\.212\.171\.5；但是，要只匹配 63\.212\.171\.5，则应该将过滤模式写为 63\.212\.171\.5$

可使用单个字符串来表示一个 IP 地址范围，例如，可将从 192.168.10.10 到 192.168.10.50 这一范围写为 192.168.10.[1-5][0-9]$

匹配行首：过滤模式 63\.212\.171\.5 也会匹配 163\.212\.171\.5。如果只希望包含以特定数字开头的 IP，请使用 '匹配字符串开头的表达式''，例如 63\.212\.171\.5$

完整正则表达式列表 

 *'''通配符'''     '''含义'''
 *'''.'''       匹配任何单个字符
 *'''*'''       匹配先前的 0 个或多个项目
 *'''+'''       匹配先前的 1 个或多个项目
 *'''?'''       匹配先前的 0 个或 1 个项目
 *'''()'''      将括号中的内容记忆为项目
 *'''[]'''      匹配此列表中的 1 个项目
 *'''-'''       在列表中创建范围
 *'''|'''       或
 *'''^'''       匹配到字段开头处
 *'''$'''       匹配到字段结尾处
 *'''\'''       作为以上所有文字字符的转义字符
