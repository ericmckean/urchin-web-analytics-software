Urchin： Urchin 日志处理

概述

Urchin Log Processing Engine (urchin) 是 Urchin 的核心日志处理组件。 一般情况下，当任务运行时，日志处理引擎会通过 Urchin Scheduler (urchind) 来调用。 但是，也可直接从 command shell 执行 urchin 以运行特定配置文件。 这种做法在高度脚本化的环境中特别有用。这些环境中运行的 Urchin 任务来自“Windows 任务调度程序”或 UNIX 类型系统上的 cron 等外部来源。 另外，这种做法还特别适用于在一些特殊情况（例如，仅处理特定日期的点击量或进行某些类型的调试）下运行配置文件。

Urchin 并不是真正的实用程序 – 之所以在此处被称作实用程序，是因为它提供了一些有限的命令行功能，这些功能在某些环境中非常有用。

用法

urchin 位于 Urchin 发行套装的 bin 目录下。 其用法如下： 

urchin [options...] (-p <profile name> | -P <profile id>)

选项： 

-a <account name>       指定配置文件的帐户名。 默认的帐户名是“(NONE)”

-D                      启用调试模式

-d <YYYYMMDD>           指定重新处理的日期

-H                      指定将运行输出记录到历史记录文件。 如果指定了 -H，而没有指定 -T，则将创建新的任务历史记录

-h                      显示帮助信息并退出

-P <profile id>         指定要处理的配置文件 ID

-p <profile name>       指定要处理的配置文件名称

-T <task id>            指定要更新的任务历史记录。 指定 -T 强制使用 -H

-t                      显示配置数据并退出

-v                      显示版本号并退出

注意事项

  • 对于 UNIX 类型的系统，urchin 应该作为已安装 Urchin 的用户/UID 运行，以确保此 UID 拥有配置文件的数据库。这是因为，如果数据库不存在，urchin 将会创建数据库。
