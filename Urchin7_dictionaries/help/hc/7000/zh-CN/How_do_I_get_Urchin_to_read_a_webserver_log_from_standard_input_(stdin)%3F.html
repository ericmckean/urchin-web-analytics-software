怎样让 Urchin 从标准输入 (stdin) 读取网络服务器日志？

某些环境下可能会要求 Urchin 日志处理引擎直接从标准输入 (stdin) 而不是从实际文件读取网络服务器日志数据。Urchin 支持从标准输入而不是从实际文件读取日志源的配置。

要将 Urchin 配置为执行此操作，请首先通过以下参数在日志源管理器中配置名为"stdin"的新日志源：

日志设置

  • Log File Location:local

  • Log File Path:-

  • Log Format:auto

高级设置

  • Log Destiny:Don't Touch

  • Query Token:?

  • URI Stem to Lower Case:off

请注意： 必须将"Log File Path"设置为短划线"-"字符（不包括引号），这一点非常重要。

现在，对于希望其从"stdin"读取日志的配置文件，只需在"配置文件 -> 日志源"配置屏幕中选择新的"stdin"日志源即可。

重要注意事项： 您无法运行日志源为"stdin"且被配置为通过 Urchin 调度程序运行的 Urchin 配置文件，因为该调度程序无法与日志输入流正确关联。对于日志源为"stdin"的配置文件，处理日志的唯一方法是手动运行 Urchin。下面举例说明如何调用 Urchin：

  1. 在 Urchin 系统中打开一个命令行界面窗口
  2. 将目录更改为 Urchin 发行套装的"bin"目录

  3. [UNIX 类型的系统] 使用"./urchin -p myprofile < /path/to/webserver.log"运行 urchin

    • [Windows 系统] 使用"urchin -p myprofile < C:\path\to\webserver.log"运行 urchin

将"stdin"用作日志源可给予您极大的灵活性，因为这样您就可以在 Urchin 读取日志之前轻松地进行预处理。

其他示例

从包含多台不同虚拟主机的点击信息的日志中提取某台虚拟主机的点击信息，并将结果提供给 Urchin： 

grep " www.mydomain.com "/path/to/alldomains.log | urchin -p mydomain.com

通过某些嵌入的 SQL 脚本提取关系数据库的源网络服务器点击信息，并将结果提供给 Urchin 进行处理 

sql-extract --date 20040302 --domain mydomain.com | urchin -p mydomain.com
