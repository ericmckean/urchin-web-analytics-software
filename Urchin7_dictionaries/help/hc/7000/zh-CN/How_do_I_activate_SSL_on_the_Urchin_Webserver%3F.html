如何在 Urchin 网络服务器中启用 SSL？

随 Urchin 提供的 Urchin 网络服务器能够通过 SSL 对通信进行加密。要启用 SSL，您需要拥有一个由认证机构签署的有效证书或一个自签名证书。

要在 Urchin 网络服务器中启用 SSL，请按以下步骤操作：

  1. 将您的 SSL 证书文件复制到 Urchin var 目录中，并将其命名为 server.crt
  2. 将您的 SSL 密钥文件复制到 Urchin var 目录中，并将其命名为 server.key
  3. 修改位于 Urchin var 目录下的 urchinwebd.conf.template 文件
    1. （如果您使用的是 Urchin 6.6+，请跳过此步骤。）注释掉 Port 行并取消对 Listen 指令的注释。 
                #XXXPortXXX
          Listen 443

    2. 将 ServerName 指令从 localhost 更改为您的网络服务器的名称。例如： 

                ServerName: www.urchin.com

    3. 请注意：urchinwebd.conf.template 文件中的 ServerName 必须与证书文件中的服务器名称一致。

  4. 使用 urchinctl 和 -e 选项启动或重新启动网络服务器。Urchinctl 位于 Urchin bin 目录下。-e 选项指示 urchinctl 在网络服务器中启用 SSL。例如，要重新启动网络服务器并在 443 端口上启用 SSL，请执行以下操作： 对于 Urchin 6.6 之前的版本： 
       urchinctl -e -w restart
    对于 Urchin 6.6+： 
       urchinctl -e -p 443 -w restart

    请注意：要启动服务器而不启用 SSL，只需去掉 -e 选项即可。

现在，您可以通过以下网址来访问启用了 SSL 的服务器：https://servername.domain.com:port/

请注意：对 urchinwebd.conf.template 中的 SSL 设置进行自定义可能会带来一些问题，并可能导致网络服务器无法启动。

Windows 说明

要在 Urchin 网络服务器中启用 SSL，请按以下步骤操作：

  1. 将您的 SSL 证书文件复制到 Urchin var 目录中，并将其命名为 server.crt
  2. 将您的 SSL 密钥文件复制到 Urchin var 目录中，并将其命名为 server.key
  3. 修改位于 Urchin var 目录下的 urchinwebd.conf.template 文件
    1. （如果您使用的是 Urchin 6.6+，请跳过此步骤。）注释掉 Port 行并取消对 Listen 指令的注释。 
                #XXXPortXXX
          Listen 443
    2. 取消对以下行的注释，以便在网络服务器启动时载入 mod_ssl.so： 
                LoadModule ssl_module etc/mod_ssl.so

    3. 将 ServerName 指令从 localhost 更改为您的网络服务器的名称。例如： 

                ServerName: www.urchin.com

    4. 请注意：urchinwebd.conf.template 文件中的 ServerName 必须与证书文件中的服务器名称一致。

  4. 使用 urchinctl 和 -e 选项删除并安装网络服务器服务。Urchinctl 位于 Urchin bin 目录下。-e 选项指示 urchinctl 在网络服务器中启用 SSL。例如，要删除和安装网络服务器并在 443 端口上启用 SSL，请按以下步骤操作： 对于 Urchin 6.6 之前的版本： 
       urchinctl.exe -w remove
   urchinctl.exe -w -e install
    对于 Urchin 6.6+： 
       urchinctl.exe -w remove
   urchinctl.exe -e -p 443 -w install

    请注意：要启动服务器而不启用 SSL，只需去掉 -e 选项即可。

现在，您可以通过以下网址来访问启用了 SSL 的服务器：https://servername.domain.com:port/

请注意：对 urchinwebd.conf.template 中的 SSL 设置进行自定义可能会带来一些问题，并可能导致网络服务器无法启动。
