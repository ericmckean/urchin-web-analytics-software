管理实用程序概述

概述

Urchin 提供了大量用于诊断和配置操作的实用程序。 这些实用程序位于 Urchin 发行套装的 util 目录下。 本文档只是对这些实用程序进行简要介绍， 并未对其用法提供全面的说明。

所有实用程序都支持“-h”和“-v”命令行参数。 调用实用程序时加上“-h”参数可获得有关相应工具的用法和可用选项的摘要说明。 调用时加上“-v”参数可显示相应实用程序的 Urchin 版本。

geo-update

此实用程序用于检查对于 Urchin 内部 DNS 数据库的更新并下载可用的更新。 通过它您可以从 Google 下载服务器以及某些本地目录（urchin 6 的新功能）下载更新。

inspector

此实用程序可对已安装的 Urchin 发行套装进行基本的检查，确保发行套装的整体结构完好、随相应产品提供的所有二进制程序版本无误并且基本许可正确（对于 UNIX 类型的平台）。 该实用程序还可报告 Urchin 配置文件调度程序 (urchind)、CPC 调度程序（urchincpcd、urchincpc）以及所捆绑的 Apache 网络服务器 (urchinwebd) 的操作状态。 另外，它还能够验证许可信息。

uconf-manager (Urchin 6.402+)

通过 Urchin 配置管理器实用程序 (uconf-manager)，您可以添加、修改和删除数据库记录。

  • Urchin 6.402此实用程序最初在 Urchin 6.402 中引入，仅用于处理配置数据库中的 'uusers' 表。 它还可以用于重设 Urchin 管理员密码、自动化用户创建过程以及将用户转移到其他帐户。
  • Urchin 6.500+： 在 urchin 6.5 及以后的版本中，此实用程序可支持对更多的表（包括 uaccounts、ucosts、ufilters、ugoals、ugroups、ugroups_uusers、ulogs、ulogs_ufilters、uprofiles、uprofiles_ucosts、uprofiles_ufilters、uprofiles_ugroups、uprofiles_ulogs、uprofiles_uusers 和 uusers）进行数据库操作。 此实用程序可用来执行通过 Urchin 管理界面无法完成的操作。

uconf2sql

Urchin 5 以及更早的版本将配置信息存储在专有文件中，而 Urchin 6 则将这些信息存储在关系数据库（MySql 或 PostgreSql）中。 为了顺利地转移配置信息，Urchin 5 和 Urchin 6 分别提供了用于导出和导入配置信息的实用程序。 在 Urchin 5 中，实用程序 uconf-export 可将配置数据库导出到文件中（XML 格式），而在 Urchin 6 中，实用程序 uconf2sql 可将配置数据从上述 XML 文件导入到 Urchin 6 数据库中。

convert-u5data

与配置数据库类似，Urchin 6 中的报告数据库（用于存储报告信息）的结构也发生了变化。 通过实用程序 convert-u5data，您可以将报告数据库直接从已安装的 Urchin 5 导入到已安装的 urchin 6 中。

udb-sanitizer

udb-sanitizer 实用程序用于对报告数据库执行管理方面的操作， 包括将其恢复为存档数据库、删除涉及特定日期或日期范围的信息（在需要重新处理的情况下）。

urchinctl

（请注意： 此实用程序位于 bin 目录下，而不是 util 目录下）。 此实用程序可用于启动和关闭 Urchin 调度程序及 Urchin 网络服务器服务。 对于 UNIX 类型的系统，urchinctl 通常会由系统的某个引导时 (boot-time) 脚本调用，以自动启动或关闭 Urchin 服务。 对于 windows 平台，它还可用于安装和卸载 urchin 服务。

urchincpc (Urchin 6.5+)

（请注意： 此实用程序位于 bin 目录下，而不是 util 目录下）。 Urchin 每次点击费用数据导入工具通过 Google AdWords API 来提取每次点击费用数据，以便进一步处理并生成报告。 有关详情，请参阅每次点击费用数据导入实用程序。

urchincpcd (Urchin 6.5+)

（请注意： 此实用程序位于 bin 目录下，而不是 util 目录下）。 Urchin 每次点击费用来源调度程序（简称为 CPC 调度程序）用于根据用户定义的时间来安排每次点击费用下载任务（类似于配置文件调度程序）。 有关详情，请参阅每次点击费用数据导入实用程序。

urchin_daemons

对于 UNIX 类型的计算机，Urchin 运行后台调度程序 (urchind) 和 Apache 网络服务器 (urchinwebd)。 这些程序会在系统引导时启动。 urchin_daemons 脚本可以置于相应 UNIX 类型 OS 适当位置的系统初始化脚本中，这样即可正确启动 Urchin 服务的后台程序。 对于 windows 平台，这些二进制程序会作为系统服务运行，并且可通过 windows 服务管理器启动和关闭。

ugetlicense

通过 Urchin 许可操作实用程序 ugetlicense，您可以使用序列号来执行操作。 如果发生当前安装的 Urchin 无法继续使用，需要重新进行安装或迁移到其他服务器等情况，您就会需要以这种方式来进行操作。
