Urchin 6 与 Urchin 5 之间有什么不同？

本文对 Urchin 5 与 Urchin 6 之间的改动进行了说明。

主要功能

  • 基本许可包含数量不限的配置文件、日志源和广告系列跟踪；无需另购附加模块
  • 拥有全面的 SEO/SEM 广告系列跟踪功能，每个配置文件可包含 4 个目标
  • 大部分报告提供丰富的跨群体信息
  • 全套访问者地理位置报告（不只是访问者域名）
  • 处理速度与 Urchin 5 相当，但提供更丰富的数据

平台支持

  • 以 2 个版本（Linux 2.4 和 2.6 kernels），即可支持大量的 Linux 平台
  • 增加了对 FreeBSD 5 和 FreeBSD 6 的支持
  • 增加了对 MacOS X 和 Solaris 的支持

安装程序

  • Windows 安装程序现在以 MSI 软件包的形式进行分发，对无人参与安装提供了更好的支持，并与 SMS 相集成

配置

  • 关系数据库（MySQL 或 PostgreSQL）后端
  • 支持由远程配置服务器托管的配置数据库

网络服务器

  • 已升级到最新的 Apache 1.3.X 版本
  • OpenSSL 和 mod-ssl 已升级到最新的版本
  • 删除了 Urchin 中用不到的默认模块
  • 增加了 mod_expires 以正确标示缓存控制标头

任务调度程序

  • 调度程序现在以两个进程运行：主调度程序和从调度程序
  • 通过后端配置数据库的脚本界面轻松实现对任务的管理

访问者跟踪

  • 以与 Google Analytics（分析）相兼容的 urchin.js 跟踪 javascript 替换了 Urchin 5 __utm.js 跟踪 javascript。如果您使用 __utm.js 跟踪网站，则需要更改您的网站标记，以使用 urchin.js。

日志处理

  • 地理数据存储在内存中（运行时内存需求量较大）
  • 日志源中的"天数范围"功能允许对与特定日期模式相匹配的日志文件进行多日搜索
  • 能够让配置文件完全在内存中运行

数据与存储

  • 配置文件数据库现在默认为每个月 100,000 个记录（以前为 10,000 个），并可以选择提高到每月最多 500,000 个记录

  • 经过扩展的 Geodata：来自 Quova 的全套地理位置数据，替换了仅限域的 MaxMind 数据

  • 每月的表格记录上限从默认的 10,000 个记录提高为 100,000 个记录
  • 每个配置文件每个月可有 50 个文件，现在按子目录加以组织

报告 UI

  • 以 Flash 替换 Adobe SVG 渲染图形和图表
  • 仅以 CSV 和 XML 格式导出报告（删除了不可靠的 MS Word/Excel 导出）
  • 废除了"所有配置文件"报告

电子商务

  • 电子商务交易可通过跟踪 javascript 中的特殊函数直接写入网络服务器日志中（与 GA 一样）
  • 同时支持采用 ELF2 格式的外部购物车日志

安全

  • 对跨网站的脚本 (XSS) 和 XSRF 漏洞进行了全面测试
