如何限定 Urchin 只访问某一个 IP 地址？

概述

对于有多个网络接口或有多个 IP 地址绑定到单个网络接口的系统，Urchin 网络接口的默认行为是绑定到所有已配置的 IP 地址。不过，您可能希望将 Urchin 配置为只使用某一 IP 地址进行管理和报告。

步骤

要限制 Urchin 的管理/报告权限，需要对 Urchin Apache 网络服务器所用的配置文件稍加更改。请按以下步骤操作：

对于 UNIX 类型的系统：

  • 以 Urchin 运行时所用的用户身份打开一个命令行界面
  • 通过"cd"命令进入安装 Urchin 的目录
  • 通过以下命令终止 Urchin 网络服务器："bin/urchinctl -w stop"
  • 用文本编辑器打开 Urchin 发行套装"var"目录中的"urchinwebd.conf.template"文件

  • 在 <Directory /> 行的上方，插入以下行 

       Listen XXX.XXX.XXX.XXX:port
  • 其中，"XXX.XXX.XXX.XXX"和"port"是只允许 Urchin 使用的 IP 地址和端口号。
  • 通过以下命令重新启动 Urchin 网络服务器： 

bin/urchinctl -w start

对于 Windows 系统：

  • 停止 Urchin 服务："开始"->"程序"->"Urchin 6"->"禁用 Urchin 服务"

  • 打开 Windows Explorer，导航到"C:\Program Files\Urchin6\var"
  • 使用文本编辑器打开"urchinwebd.conf.template"文件

  • 在 <Directory /> 行的上方，插入以下行 

       Listen XXX.XXX.XXX.XXX:port
  • 其中，"XXX.XXX.XXX.XXX"和"port"是只允许 Urchin 使用的 IP 地址和端口号。

  • 重新启动 Urchin 服务："开始"->"程序"->"Urchin 6"->"启用 Urchin 服务"

重要注意事项：由于 IIS 5 及更高版本中采用了套接字池，因此可能需要禁用 IIS 对绑定到特定端口的所有网站共享套接字的默认行为，而不管具体配置的 IP 地址是什么。这种情况在使用虚拟主机的环境中很常见，您需要为特定的虚拟主机将 Urchin 报告设置为端口 80。

注意事项

  • 此技术仅适用于限定 Urchin 只能使用特定的 IP 地址进行管理/报告。您无法将访问权限限制为基于域名的虚拟主机。
  • 对 Urchin 的升级将保存一份"urchinwebd.conf.template"文件的备份副本并覆盖该文件。您将需要在升级之后重新应用对该文件所做过的任何修改。
