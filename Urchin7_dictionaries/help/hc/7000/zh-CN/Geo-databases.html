Urchin 6.602 Geo-databases

Urchin 6.602 提供了两个地理数据库：

  • geo.db - 包含详细的都市圈 (country/region/city)、网络（组织）和域信息。 虽然有所更新，但其信息的详细程度与 urchin 6 的之前版本 (6.400 - 6.601) 是一样的。

  • geo.light.db - 只包含国家/地区级别的都市圈信息，不含区域、城市、网络和组织信息。 此数据库对内存的需求量要少得多，载入时间也短得多，因此特别适用于日志处理引擎性能至关重要的情况。 使用这一轻型版地理数据库会对下列报告产生影响。

    • o 下列“地图”报告只能显示国家/地区级别的都市圈数据（不含区域和城市信息）：

      • 管理人员视图 -> 内容提要（控制台）

      • 管理人员视图 -> 地理分布图

      • 营销商视图 -> 营销商概述（控制台）

      • 网站管理员视图 -> 网站管理员概述（控制台）

      • 营销优化 -> 访问者群体细分效果 -> 地理分布图

      o“网络位置”报告（营销优化 -> 访问者群体细分效果 -> 网络位置）不会显示任何数据

      o“域”报告（营销优化 -> 访问者群体细分效果 -> 域）不会显示任何数据

默认情况下使用的是 geo.db。 要切换为 geo.light.db 数据库，请在 urchin.conf 文件中取消相应行的注释标记： 


# 地理数据库设置

# 取消下列行的注释标记，启用轻型版地理数据库
# GeodbLight: ON
