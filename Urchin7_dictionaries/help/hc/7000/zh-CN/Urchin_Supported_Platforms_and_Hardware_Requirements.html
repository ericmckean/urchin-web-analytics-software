Urchin 支持的平台和硬件要求

Urchin 可在多种架构和操作系统上运行。 只有在要处理日志的系统上才需要安装 Urchin， 如果是要查看报告，只需使用网络浏览器即可。

支持的平台

Windows

  • Windows 2003 Server
  • Windows XP
  • Windows 2000
  • Windows Vista
  • Windows 2008 Server

UNIX-type Systems

  • Linux (kernel 2.6) - 大多数现有版本均可使用
  • Linux (kernel 2.4) - 大多数现有版本均可使用
  • FreeBSD 4（不支持 6.5 及以上版本）
  • FreeBSD 5.3+（不支持 Urchin 6.602 及以上版本）
  • FreeBSD 6.2+

Supported databases

  • MySQL 4.1+（不支持 Urchin 6.6 及以上版本）
  • MySQL 5.03+
  • PostgreSQL 8.2.5

支持的浏览器

  • Internet Explorer 6.x, 7.x
  • Firefox 2.x, 3.x
  • Chrome 1.x, 2.x

关于非明确支持平台的说明

如果没有适合您的平台的 Urchin 发行版，您可以使用与您的操作系统兼容的 Urchin 发行版。

  • Windows 98, Windows 3.x:

Urchin 不能安装在 Windows 98 或 3.x 上，但您可以在这些平台上使用 Internet Explorer 5.x 及更高版本的浏览器来查看报告。

  • Linux:

Linux 的版本很多，并非每个版本都有对应的 Urchin 发行版。 不过，各个 Linux 版本一般都具有非常高的兼容性，因此很可能会有某一 Urchin 发行版可以在您的计算机上使用。 以下是一部分已知的可兼容发行版：

  • Ubuntu (kernel 2.6)

  • RedHat Enterprise Linux 2 & higher

  • SuSE Linux 8 & higher

  • Trustix 2 & higher

  • Solaris:

我们目前不支持 Solaris。

Urchin 6 系统要求

凭借 Urchin 的优异性能，您只需很少的硬件投资即可获得很高的回报。 例如，即使是速度较慢、已不适合桌面应用的早期 Pentium II 机器，也可以充当 Urchin 服务器。 为获得最佳性能，建议使用下列硬件配置：

小型到中型单一网站分析

  • 2Ghz 或更高频率的处理器
  • 2GB 内存
  • 250GB 以上的 IDE 硬盘
  • 100Mbs 以太网接口

服务提供商/企业安装

  • 2.5Ghz Pentium dual-core / 2Ghz UltraSPARC / CPU 速度大致相当的 PPC/MIPS/ 等
  • 3GB RAM
  • Ultra2/Wide SCSI 硬盘（如 Seagate Cheetah）
  • 100base-T 以太网
  • 备份系统

内存/系统/磁盘使用情况

  • Urchin 内存 (RAM) 大小可配置为 20-500Mb（Urchin 6.5 为 20 - 1024Mb）
  • Urchin 可配置为以低、正常或高这三种优先级运行
  • Urchin 的数据存储将占用约 10% 的原始日志空间

FreeBSD 用户的重要注意事项

(1) FreeBSD 的硬编码处理数据量最大为 500MB。 由于 Urchin 的地理数据必须在日志处理过程中驻留于内存，因此无法满足此限制， 导致显示下列运行时错误： 

ERROR: (8011-323-1057) Failed to allocate memory

要解决此问题，您需要提高 FreeBSD 系统的默认处理数据量，方法是修改 /boot/loader.conf 文件并加入以下几行代码： 

# Increase max process data segment size to 1GB
kern.maxdsiz="1073741824"

然后重新启动系统。

警告！！进行此类系统限额更改操作时请务必小心谨慎。 如果将 kern.maxdsiz 设置得过大，系统将可能无法启动。 强烈建议您在进行这项修改前阅读 FreeBSD 文档，并评估相应修改对您网站的配置所带来的潜在风险。

(2) FreeBSD 有一个名为 accf_http 的特殊驱动程序。 该驱动程序会在某一完整的 HTTP 请求到达之前缓冲传入的连接，从而加快速度和优化性能。 默认情况下，该驱动程序不会载入内核，您将看到以下警告： 

[warn] (2)No such file or directory: Failed to enable the
‘httpready’ Accept Filter

为避免发生这种情况，您需要启用 HTTP 接受过滤器 FreeBSD 内核模块 (accf_http)，方法是修改 /boot/loader.conf 文件并加入以下几行代码： 

# Wait for full HTTP request accept filter
accf_http_load="YES"

然后重新启动系统。

Linux glibc 库版本

要在 Linux（kernel 2.4 或 2.6）上安装 Urchin 6.602 附带的 PostgreSQL 8.4.01，必须使用 glibc 2.3.3+。
