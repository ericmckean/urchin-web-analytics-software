Urchin 集成功能概述

简介

Urchin 前端建立在模块化的组件和业界标准技术的基础上，可让管理员将 Urchin 报告访问权限集成到现有的或规划中的基础结构中。为了实现我们对托管提供商和数据中心的承诺，我们测试并支持三个集成点，以确保能将 Urchin 集成到大部分体系结构中。

以下图表说明了 Urchin 前端的主要组件以及三个集成点。此系统提供基于网络的管理和报告功能。

如图表左侧所示，Urchin 系统的所有内容都是由作为"urchinwebd"安装的 Apache 服务器所提供。Apache 服务器会处理请求，并使用 CGI 界面将其发送到"会话控制器"应用程序 (session.cgi)。"会话控制器"会执行身份验证，并根据情况将请求发送给"管理员引擎"（管理员）或"Urchin 报告引擎"(urchin.cgi)。嵌入会话标识的内容则会传回给用户。

标有 "A"、"B" 和 "C" 的三个点就是先前提及的三个集成点。点 "A" 允许管理员替换或绕过 Urchin 网络服务器。点 "B" 允许进行外部验证或不验证。点 "C" 则允许通过包装程序或门户网站直接访问报告。

点 A：网络服务器集成

许多托管公司可能已经有投入运行的网络服务器，或在其系统中运行经过特殊编译的网络服务器。只要网络服务器提供基本的 CGI 操作，就可替换 Apache 二进制文件，或使用现有的网络服务器来提供对 Urchin 的访问。

这个集成点可以维持 Urchin 系统的完整性。Urchin 界面将用于管理、身份验证和报告传递。需要在 Urchin 系统中配置用户和配置文件，以便 Urchin 在用户进入系统时知道他们可以访问哪些报告。

点 B：外部身份验证或绕过身份验证程序

LDAP 等复杂的托管环境通常采用现有的集中式控制手段对用户进行身份验证。只需更改一项配置，会话控制器就可以调用您选择的身份验证例程。系统会提供简单的界面，以返回登录成功或失败的信息。可轻松对现有身份验证例程进行包装，以便为这两个系统提供正确的框架。在点 "B" 使用集成功能时，也可以使用虚拟例程绕过身份验证，然后直接链接到目标网页。

使用点 "B" 的集成功能时，除身份验证是由外部应用程序所执行之外，整体 Urchin 系统会维持不变。需要在 Urchin 系统中配置用户和配置文件，以便 Urchin 在用户进入系统时知道他们可以访问哪些报告。有关使用外部身份验证的界面的完整信息以及绕过身份验证的方法的示范，请参阅 使用外部身份验证或绕过身份验证程序。

点 C：从包装程序或门户网站直接链接到报告

我们有许多客户在客户与网站之间是一对一的关系，希望从特定网站的管理区域直接链接到该网站的报告。此区域通常都受身份验证保护。点 C 集成可让用户轻松通过包装程序脚本或现有的门户网站直接链接到报告。在此情况下，用户会绕过 Urchin 身份验证和初始报告选择屏幕，直接进入 Urchin 报告。

使用点 "C" 的集成功能时，我们假设服务提供商可通过一对一关系或现有的配置数据库来控制任一用户有权查看的内容。仍需配置 Urchin 才能生成配置文件报告，但这一操作在管理界面内外都可实现自动化。如果已在管理界面中启用"直接报告链接"，Urchin 也提供直接链接到特定 URL 的 Urchin 报告的功能。通过嵌入到此 URL 所指定目录中的".report.conf"文件控制对 Urchin 报告的访问权限。

对于复杂的门户网站集成，Urchin 报告引擎会传播会话和其他门户网站变量，以便让会话继续进行。
