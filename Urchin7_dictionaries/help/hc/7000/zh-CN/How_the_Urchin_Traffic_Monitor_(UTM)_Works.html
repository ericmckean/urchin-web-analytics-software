Urchin 访问量监控器 (UTM) 的工作原理

概述

Urchin 访问量监控器 (UTM) 最初是在 Urchin 4 中提供，目前正在申请专利。它经过特别设计，能够最准确地计量贵网站的唯一身份访问者。对于需要深入了解在线访问者行为的企业来说，UTM 是非常有价值的技术：它不但将最佳的客户和服务器端信息相结合，另外还能让您实现对这些数据的控制。这项技术安装起来非常容易，它可供商务用户准确标识唯一身份访问者和点击路径，返回将首次访问者和回访者都包括在内的忠诚度指标以及他们的使用频率。最新版本的 UTM 提供了增强的广告系列跟踪功能。

Urchin 访问量监控系统中有两个组件：一是 UTM 传感器，也就是安装在网站内容中的代码段；另一个是 UTM 引擎，是 Urchin 日志处理引擎的一部分。UTM 传感器支持客户端数据收集功能，为每次综合浏览收集到的数据会通过渠道传回 Urchin。UTM 引擎会将客户端信息与现有的服务器端数据相结合，以提供更准确而完整的网站活动全景。

UTM 传感器是一小段 JavaScript 代码，可完成两项重要功能。第一，此传感器在每次综合浏览中会强制至少将一次点击送入 Google 网络服务器，以消除缓存的影响。第二，UTM 传感器可使用客户端的"第一方"Cookie 唯一确定每一位访问者，以跟踪首次访问者及回访者。这个 Cookie 标识符是一种仅供网络服务器查看的通信标记，其查看方式与会话 ID 相同。它不是向系统外部发送信息的第三方 Cookie，因此不会违反隐私政策。

UTM 系统的工作方式如下。网络服务器提供两种基本功能：内容传递和日志记录。网站内容中包含传递到用户浏览器的 UTM 传感器。此 UTM 传感器会设置唯一标识符，并将额外的请求发送回同一网络服务器。UTM 引擎则是 Urchin 日志处理引擎的一部分，它会使用此请求中的数据提供准确而更为完整的访问者行为说明。

UTM 传感器

UTM 传感器可消除缓存和代理功能的影响，从而提高日志文件数据的准确性和完整性。例如，当用户回到前面查看过的网页时，会从浏览器的缓存内存传递此网页的内容。如果缓存由服务提供商提供，这一模式同样适用。网络服务器并不知道这次综合浏览。但是，UTM 传感器会启动额外的独立点击，以强制至少将一个小型记录发送回网络服务器中。此信息会记录在日志文件中，现在日志文件中就会包含有关用户所发起的这次综合浏览的信息。

UTM 传感器的第二项重要功能是它能够实现对会话和唯一身份访问者的唯一标识。通过目前正在申请专利的与浏览器 Cookie 相结合的功能，传感器可以检测并初始化唯一身份访问者和会话标识符，做到无论服务提供商有何种代理行为，都可以准确监控新访者和回访者。大多数服务提供商对代理功能的利用方式都是循环使用 IP 地址和对防火墙后的用户进行聚类分析。因为日志文件跟踪一般都会使用 IP 地址作为用户标识符，这样做可能会为正常跟踪带来问题。

UTM 传感器使用了 Cookie 标识符（而不是 IP 地址），能够拨开代理这层云雾。例如，当某个首次来访的唯一身份访问者采用某个 IP 地址通过防火墙访问网站时，传递的综合浏览信息中会包含 UTM 传感器，此传感器会在访问者的浏览器上设置标识符。

当同一访问者再次来访时，这一唯一身份 ID 就会与各个请求一起传递到网络服务器中。因此，就算用户现在被分配了另一个 IP 地址，UTM 技术还是能够采用原始 ID 正确地识别此访问者。除了可消除复杂代理技术的影响外，它还能跟踪使用笔记本电脑在不同地点通过不同服务提供商上网的差旅访问者。
