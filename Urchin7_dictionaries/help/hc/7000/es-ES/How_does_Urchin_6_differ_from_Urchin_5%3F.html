¿En qué se diferencia Urchin 6 de Urchin 5?

En este artículo se describen los cambios que hay entre Urchin 5 y Urchin 6.

Funciones principales

  • Perfiles ilimitados, fuentes de registro y seguimiento de campañas, todo incluido con la licencia básica, sin módulos complementarios.
  • Completas funciones de seguimiento de campañas SEO/SEM, 4 objetivos por perfil.
  • Segmentación cruzada exhaustiva, disponible desde la mayoría de los informes.
  • Paquete completo de informes de ubicación geográfica de usuarios (no solo dominio de usuarios).
  • Velocidad de procesamiento similar a la de Urchin 5 pero con muchos más datos.

Compatibilidad de plataformas

  • Amplia variedad de plataformas Linux compatibles con solo dos 2 versiones (Linux 2.4 y 2.6 kernels).
  • Compatibilidad añadida para FreeBSD 5 y FreeBSD 6.
  • Eliminación de compatibilidad para MacOS X y Solaris.

Programas de instalación

  • El programa de instalación de Windows se distribuye ahora como un paquete MSI; mejor compatibilidad de instalación sin asistencia e integración con SMS.

Configuración

  • Servidor de la base de datos relacional (MySQL o PostgreSQL).
  • Admite base de datos de configuración alojada en servidor de configuración remoto.

Servidor web

  • Actualizado a la versión más reciente de Apache 1.3.X.
  • OpenSSL y mod-ssl actualizados a las versiones más recientes.
  • Eliminados los módulos predeterminados que Urchin no utiliza.
  • mod_expires añadidos para cabeceras de control de caché adecuadas.

Programador de tareas

  • El programador se ejecuta ahora como dos procesos: programador principal y programador secundario.
  • Tareas administradas fácilmente a través de la interfaz de encriptación a la base de datos de configuración de servidor.

Seguimiento de los usuarios del sitio

  • La función de seguimiento __utm.js de JavaScript de Urchin 5 se ha sustituido por una función de seguimiento urchin.js de javascript compatible con Google Analytics. Si realiza un seguimiento de su sitio mediante __utm.js, deberá cambiar las etiquetas del sitio para que utilicen urchin.js.

Procesamiento de registros

  • Datos geográficos guardados en memoria (mayor uso de memoria de tiempo de ejecución).
  • La función de intervalo de días en fuentes de registros le permite realizar búsquedas de varios días para archivos de registro que se correspondan con un patrón de fecha concreto.
  • Posibilidad de ejecutar perfiles por completo en memoria.

Datos y almacenamiento

  • Las bases de datos de perfiles incluyen ahora de forma predeterminada hasta 100.000 registros/mes (en lugar de 10.000) con opción a aumentar hasta 500.000 registros por mes.

  • Geodata extendido: conjunto completo de datos de ubicación geográfica de Quova que sustituye a los datos exclusivos de dominio de MaxMind.

  • Límite de registros de tablas mensual aumentado de 10.000 a 100.000 registros de forma predeterminada.
  • 50 archivos mensuales por perfil ahora organizados por subdirectorio.

UI de informes

  • Flash sustituye a Adobe SVG para elaborar gráficos y cuadros.
  • Exportación de informes solo en CSV y XML (eliminada la exportación poco fiable en MS Word/Excel).
  • El informe Todos los perfiles está inactivo.

Comercio electrónico

  • Las transacciones de comercio electrónico pueden escribirse directamente en registros de servidor web a través de funciones especiales de JavaScript de seguimiento (idénticas a GA).

  • Se admiten también registros de carritos de la compra externos en formato ELF2.

Seguridad

  • En todas las pruebas para encriptación de sitios cruzada (XSS) y vulnerabilidades XSRF.
