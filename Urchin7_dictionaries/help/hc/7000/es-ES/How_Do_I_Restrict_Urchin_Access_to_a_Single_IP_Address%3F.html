¿Qué debo hacer para restringir el acceso de Urchin a una sola dirección IP?

Visión general

En un sistema que disponga de varias interfaces de red o tenga varias direcciones IP asociadas a una única interfaz de red, el comportamiento predeterminado de la interfaz web de Urchin consiste en asociarse a todas las direcciones IP configuradas. Sin embargo, se recomienda configurar Urchin de manera que restrinja la administración y la generación de informes a una única dirección IP.

Procedimiento

Para restringir la administración/generación de informes de Urchin es necesario realizar un sencillo cambio en el archivo de configuración que utiliza el servidor web Apache de Urchin. Realice los pasos siguientes:

Para sistemas UNIX:

  • Abra un shell de comandos como el usuario que ejecute Urchin.
  • Ejecute el comando "cd" hasta llegar al directorio donde está instalado Urchin.
  • Detenga el servidor web de Urchin con el comando: "bin/urchinctl -w stop".
  • Con un editor de textos, abra el archivo "urchinwebd.conf.template" del directorio "var" del software de Urchin.

  • Encima de la línea <Directory />, introduzca la línea 

       Listen XXX.XXX.XXX.XXX:port
  • Donde "XXX.XXX.XXX.XXX" y "port" son la dirección IP y el número de puerto a los que desea restringir Urchin.
  • Reinicie el servidor web de Urchin con el comando: 

bin/urchinctl -w start

Para sistemas Windows:

  • Detenga los servicios de Urchin: "Inicio -> Programas -> Urchin 6 -> Inhabilitar servicios Urchin".

  • Abra Windows Explorer y acceda a "C:\Program Files\Urchin6\var".
  • Abra el archivo "urchinwebd.conf.template" en un editor de textos.

  • Encima de la línea <Directory />, introduzca la línea 

       Listen XXX.XXX.XXX.XXX:port
  • Donde "XXX.XXX.XXX.XXX" y "port" son la dirección IP y el número de puerto a los que desea restringir Urchin.

  • Reinicie los servicios de Urchin: "Inicio -> Programas-> Urchin 6 -> Habilitar servicios Urchin".

Nota importante: debido al uso de zócalo de IIS 5 y versiones superiores, tal vez sea necesario inhabilitar el comportamiento predeterminado de IIS para que comparta zócalos en todos los sitios web asociados a un puerto concreto, independientemente de la dirección IP que esté configurada. Esto suele producirse en entornos donde se utilizan hosts virtuales y es preferible configurar la generación de informes de Urchin en el puerto 80 para un host virtual concreto.

Nota importante

  • Esta técnica solo sirve para restringir la administración/generación de informes de Urchin a una dirección IP concreta. No puede restringir el acceso a un host virtual basado en nombre.
  • Las actualizaciones a versiones superiores de Urchin reemplazarán el archivo "urchinwebd.conf.template" después de guardar una copia de seguridad. Tras una actualización, deberá volver a aplicar las modificaciones que haya realizado en el archivo en cuestión.
