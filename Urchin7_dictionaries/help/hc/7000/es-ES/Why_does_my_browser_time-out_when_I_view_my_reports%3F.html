¿Cómo puedo configurar Urchin para que determinadas palabras clave o URL se consideren referencias directas en lugar de palabras clave?

Si desea que determinadas palabras clave o URL se consideren referencias directas en lugar de palabras clave, puede configurar el archivo urchin.js de manera que Urchin las registre como referencias directas en sus informes.

  1. Utilice cualquier editor de textos para abrir el archivo urchin.js.
  2. Desplácese hacia abajo hasta encontrar la siguiente cabecera: 

          //-- Auto/Organic Keywords to Ignore
  1. Añada la palabra clave orgánica que debe omitirse al final de la lista, en el formulario: 

          _uOno[#]="keyword";
  • ..como se indica en el siguiente ejemplo: 

        //-- Auto/Organic Keywords to Ignore
        var _uOno=new Array();
        _uOno[0]="urchin";
        _uOno[1]="urchin.com";
        _uOno[2]="www.urchin.com";

¿Por qué el navegador caduca cuando veo mis informes?

La aplicación que consulta la base de datos de Urchin y genera los informes caducará si los datos no muestran antes de los 45 segundos. Si el tamaño de la base de datos es suficientemente grande, puede que 45 segundos no sean suficientes para obtener todos los datos solicitados. Cuando eso sucede, el navegador caduca y suele mostrar un mensaje de error.

El valor de espera se puede extender si tiene privilegios de acceso suficientes al servidor que ejecuta Urchin.

Ampliación del tiempo de espera

  • Conéctese al servidor Urchin mediante un shell de comando.
  • Acceda al directorio .../[urchin-install-dir]/etc/.
  • Abra el archivo "sesión.conf" en un editor de textos.
  • Aumente el valor de 'APPLICATION_TIMEOUT:' (en segundos).
  • Guarde el archivo, y los cambios se aplicarán inmediatamente
