Visión general

En este artículo se ofrece una vista general acerca de cómo configurar el registro para servidores distintos de Apache e IIS para facilitar el procesamiento y la creación de informes para Urchin. Urchin procesa cualquier registro de servidor web mientras pueda entender el modo en el que los datos se organizan en cada entrada de archivo de registro. La información de este artículo se aplica solo a los registros de acceso. Si está interesado en obtener detalles acerca del registro de e-commerce, consulte la sección Módulo de E-commerce del Centro de documentación.

Independientemente del tipo de servidor web o formato de registro existe un conjunto de campos obligatorios mínimos necesarios para que Urchin produzca informes. Estos son:

  • fecha
  • hora
  • nombre de host o dirección IP del sistema que lo solicita
  • solicitud (por ejemplo, los documentos que solicitó el sistema de su servidor web)
  • código de estado generado por solicitud (numérico)
  • bytes (bytes transferidos del servidor al cliente)

Además, para realizar los informes más completos necesita los campos siguientes:

  • referencia
  • user-agent
  • cookies (si el Supervisor de tráfico de Urchin está instalado en el sitio)

Configuración

El listado de todos los datos específicos acerca de cómo realizar cambios en las características de registro de cada servidor web sería demasiado tedioso. Por lo general, el enfoque más sencillo es configurar el registro para que adopte la forma de NCSA o de W3C de Urchin y, a continuación, seleccionar el formato predeterminado adecuado en el menú desplegable "Formato del registro" en "Fuente del registro". Si el servidor web admite este enfoque, consulte el documento Registro: Apache e IIS. La información que se incluye allí acerca de los campos de datos necesarios puede serle de utilidad, aunque los detalles de los métodos para realizar los cambios no se apliquen necesariamente a su servidor web.
