Mi sitio dispone de varios nombres de dominio. ¿Qué debo hacer para configurar el seguimiento?

Si el sitio utiliza un carrito de la compra ubicado en un dominio raíz distinto (p. ej., www.mitienda.es y www.securecart.com) o emplea varios alias de dominio orientados al mismo contenido, deberá aplicar determinados cambios para garantizar que sus sitios y el Supervisor de tráfico de Urchin (UTM) sean compatibles.

  • Tenga en cuenta que existe una diferencia entre varios subdominios y varios dominios raíz. www.ejemplo.es y carritoseguro.ejemplo.es son ejemplos de varios subdominios, mientras que www.ejemplo.es y www.muestra.es representan varios dominios raíz. Si el sitio tiene varios subdominios, consulte: ¿Cómo puedo realizar el seguimiento de todos los subdominios del sitio en un perfil? ¿Cómo puedo realizar el seguimiento de todos los subdominios del sitio en perfiles separados? para obtener información sobre la configuración.

Varios nombres de dominio: alias de dominio

Si el sitio emplea varios alias de dominio, le recomendamos redirigir el tráfico de los alias al sitio principal. De este modo, se asegura de que el seguimiento de usuarios de UTM se establece bajo el dominio principal y de que se realiza un seguimiento de todos los usuarios de manera coherente.

Las siguientes instrucciones proporcionan un ejemplo de cómo redirigir el tráfico de los dominios con alias al dominio principal en los servidores IIS y Apache.

Redirección de alias en Apache

  • Si utiliza un servidor web Apache, la configuración se puede modificar fácilmente para redirigir todo el tráfico que se origina en uno de los alias al sitio principal. Una forma de hacerlo es crear dos entradas de host virtual. El primero sería el dominio principal, que incluye la configuración normal; el segundo host virtual incluye todos los alias y redirige el tráfico al sitio principal: 

        #---primary virtualhost
        Nombredeservidor www.misitio.es
        Aliasdeservidor misitio.es
        ...
        #---second virtualhost
        Nombredeservidor misitio.org
        Aliasdeservidor www.misitio.org misitio.net www.misitio.net
        RewriteEngine on
        RewriteRule ^(.*) http://www.misitio.es$1 [R=301]

  • El segundo VirtualHost utiliza una regla de reescritura con un código de redirección 301 (trasladado permanentemente) para reenviar todo el tráfico al sitio original. Llegados a este punto, con respecto a UTM, el sitio parece tener un único dominio y está listo para una instalación normal de UTM.

Redirección de alias en servidores IIS

  • Si utiliza un servidor web Microsoft IIS, la configuración se puede modificar fácilmente para redirigir todo el tráfico que se origina en uno de los alias al sitio principal. Una forma de hacerlo es crear dos sitios web en la configuración del servidor IIS. El primero será el dominio principal (www.misitio.es), que incluirá su configuración normal; el segundo incluirá todos los otros alias (misitio.net, misitio.org, etc.) y redirigirá el tráfico al sitio principal. En el administrador del servidor IIS, haga clic con el botón derecho del ratón en uno de los sitios web para que aparezca el cuadro de diálogo de propiedades. En la pestaña "Sito web", haga clic en el botón "Avanzado...". Se abre una ventana en la que puede asignar dominios adicionales al sitio web mediante el campo "Cabecera del host". Defina el dominio principal en el sitio web principal y utilice el sitio web secundario para albergar todos los alias. Una vez que haya configurado y habilitado el alojamiento de todos los alias en el sitio web secundario, cree una página web vacía con el siguiente código de redirección: 

        <META HTTP-EQUIV=Refresh CONTENT="0; URL=http://www.misitio.es/">
  • Esto redirige el navegador del usuario directamente a la URL principal. En este momento, el sitio web principal aparece como una configuración de un único dominio y se puede proceder a la instalación de UTM con los ajustes predeterminados.
