Problemas conocidos de Urchin 6.6, 6.601 y 6.602

Revisión de Urchin 6.602: 23 de octubre de 2009

Administración

  • El Asistente para la configuración de objetivos emplea la tecnología de superposición de sitios, que no permite procesar páginas protegidas con contraseña y que se confunde fácilmente con las páginas complejas (por ejemplo, las que incluyen marcos, javascript, css, redireccionamientos automáticos, etc.).

Datos geográficos

  • Debido al formato de la nueva base de datos geográfica, ya no es posible importar información local de asignación de dirección IP a nombre de dominio.

Procesamiento de registros

  • Si se procesan registros sin mantener el orden cronológico, es probable que el procesamiento de los sitios con gran volumen de tráfico sea muy lento.
  • El hecho de incrementar el límite de la base de datos a más de los 100.000 registros predeterminados puede reducir el rendimiento a la hora de procesar los registros, especialmente en los sitios con gran volumen de tráfico.
  • El motor de procesamiento de registros asigna memoria a cada uno de los filtros aplicados a un determinado perfil de los archivos de registro físico que se abran durante el procesamiento. Esto puede comportar solicitudes para asignar grandes cantidades de memoria y que no pueda asignarse toda la memoria necesaria cuando un perfil se configure con un gran número de filtros y cuando muchos archivos de registro físicos coincidan con las fuentes del archivo de registro, debido, por ejemplo, a una coincidencia de comodín demasiado amplia.
  • La existencia de tablas de consulta de gran tamaño puede provocar la asignación de grandes cantidades de memoria y que dicha asignación no se lleve a cabo, sobre todo si se abren muchos archivos de registro físicos durante el procesamiento de registros.

  • Se conocen ciertos problemas con el uso de las capas de UNC. Si desea obtener más información al respecto, consulte Uso de fuentes del archivo de registro.

Programadores

  • Los programadores (perfiles y valores de CPC) de Urchin utilizan la base de datos para almacenar y para obtener las marcas de fecha y de hora a fin de programar varias tareas. Esto significa que, una vez configurada la base de datos, no debe reajustar las zonas horarias del equipo que ejecute la base de datos urchin db. Asimismo, le recomendamos encarecidamente que no reajuste las zonas horarias del equipo que ejecute los programadores de Urchin.

Comercio electrónico

  • Urchin puede leer datos de comercio electrónico incluidos directamente en los registros del servidor web cuando las transacciones se registran mediante invocaciones a javascript de UTM o a través de un registro ELF2 de comercio electrónico explícito creado por el carrito de la compra. Urchin crea transacciones duplicadas si se muestran los mismos datos de comercio electrónico tanto en los registros del servidor web como en los registros ELF2. Así pues, es importante que los registros ELF2 no se procesen si en los registros del servidor web aparecen los mismos datos.

Utilidades

  • La función "día cero" de udb-sanitizer puede resultar muy lenta en perfiles que incluyen grandes cantidades de datos.

Informes

  • Es posible que los informes de segmentación cruzada se generen muy lentamente (o que se agote el tiempo de espera) para sitios con grandes cantidades de tráfico o si el periodo especificado es muy largo.
  • Los resultados de "Páginas/visitas" difieren entre el informe "Conversión de medios" y los subinformes asociados a "Opciones de análisis".

Integración de AdWords

  • El acceso directo a AdWords desde Urchin no funciona si Urchin 6.6+ está instalado en la versión de 64 bits de Windows 2008.

Instalación de PostgreSQL

  • Durante la instalación del servidor PostgreSQL en Linux Kernel 2.6 y en Windows, el programa de instalación requiere privilegios root/admin para crear el usuario del sistema "postgres".
  • En Linux, el programa de instalación de PostgreSQL requiere glibc 2.3.3 o posterior.
  • Si durante la instalación de Urchin en Linux Kernel 2.4 se selecciona el servidor PostgreSQL integrado, es posible que la importación de datos de CPC no funcione. En ese caso, se recomienda utilizar un servidor PostgreSQL externo.

Revisión de Urchin 6.601: 23 de julio de 2009

Administración

  • En el modo Centro de datos, los administradores de cuentas solo disponen de un nivel de privilegio, que es más restrictivo que los niveles de privilegio de los administradores afiliados que existían en Urchin 5.
  • El Asistente para la configuración de objetivos emplea la tecnología de superposición de sitios, que no permite procesar páginas protegidas con contraseña y que se confunde fácilmente con las páginas complejas (por ejemplo, las que incluyen marcos, javascript, css, redireccionamientos automáticos, etc.).

Datos geográficos

  • Debido al formato de la nueva base de datos geográfica, ya no es posible importar información local de asignación de dirección IP a nombre de dominio.

Procesamiento de registros

  • Por cuestiones de rendimiento, el motor de procesamiento de registros carga la base de datos con información geográfica completa en la memoria durante el proceso, con lo que el uso de la memoria es bastante mayor (~800 MB mín.), incluso cuando se procesan perfiles de sitios muy pequeños. Con ello, también se añade latencia al tiempo de inicialización del procesamiento de registros de cada perfil, que suele ser de dos a cinco segundos.
  • Si se procesan registros sin mantener el orden cronológico, es probable que el procesamiento de los sitios con gran volumen de tráfico sea muy lento.
  • El hecho de incrementar el límite de la base de datos a más de los 100.000 registros predeterminados puede reducir el rendimiento a la hora de procesar los registros, especialmente en los sitios con gran volumen de tráfico.
  • El motor de procesamiento de registros asigna memoria a cada uno de los filtros aplicados a un determinado perfil de los archivos de registro físico que se abran durante el procesamiento. Esto puede comportar solicitudes para asignar grandes cantidades de memoria y que no pueda asignarse toda la memoria necesaria cuando un perfil se configure con un gran número de filtros y cuando muchos archivos de registro físicos coincidan con las fuentes del archivo de registro, debido, por ejemplo, a una coincidencia de comodín demasiado amplia.
  • La existencia de tablas de consulta de gran tamaño puede provocar la asignación de grandes cantidades de memoria y que dicha asignación no se lleve a cabo, sobre todo si se abren muchos archivos de registro físicos durante el procesamiento de registros.

Programadores

  • Los programadores (perfiles y valores de CPC) de Urchin utilizan la base de datos para almacenar y para obtener las marcas de fecha y de hora a fin de programar varias tareas. Esto significa que, una vez configurada la base de datos, no debe reajustar las zonas horarias del equipo que ejecute la base de datos urchin db. Asimismo, le recomendamos encarecidamente que no reajuste las zonas horarias del equipo que ejecute los programadores de urchin.

Comercio electrónico

  • Urchin puede leer datos de comercio electrónico incluidos directamente en los registros del servidor web cuando las transacciones se registran mediante invocaciones a javascript de UTM o a través de un registro ELF2 de comercio electrónico explícito creado por el carrito de la compra. Urchin crea transacciones duplicadas si se muestran los mismos datos de comercio electrónico tanto en los registros del servidor web como en los registros ELF2. Así pues, es importante que los registros ELF2 no se procesen si en los registros del servidor web aparecen los mismos datos.

Funciones

  • Las utilidades de compresión y descompresión que incorpora Urchin tienen un límite de tamaño de archivo de 2 GB. Esto puede afectar a la capacidad de Urchin de crear archivos o copias de seguridad de datos de informes de sitios grandes, o bien de utilizar la opción "comprimir" en el destino de registro de las fuentes del archivo de registro.
  • La función "día cero" de udb-sanitizer puede resultar muy lenta en perfiles que incluyen grandes cantidades de datos.

Informes

  • Es posible que los informes de segmentación cruzada se generen muy lentamente (o que se agote el tiempo de espera) para sitios con grandes cantidades de tráfico o si el periodo especificado es muy largo.

Autenticación externa

  • Urchin 6.6+ incluye módulos de autenticación externa, entre ellos, el módulo LDAP. En la actualidad, el módulo LDAP no funciona con proveedores de LDAP que se ejecuten mediante "SSL/TLS".

Instalación

  • No se recomienda actualizar las copias de Urchin con licencia para la versión de prueba, ya que puede generar problemas de acceso o de licencia.

Cookies

  • Con el fin de facilitar una mejor interacción en la interfaz de informes, Urchin 6.6+ emplea cookies de cliente. Para ver correctamente los informes, las cookies de cliente deben estar habilitadas en el navegador del cliente.
