¿Qué debo hacer para activar SSL en el servidor web de Urchin?

El servidor web de Urchin que se proporciona con Urchin puede encriptar la comunicación vía SSL. Para habilitar SSL, debe disponer de un certificado válido firmado por una autoridad de certificación o un certificado con autofirma.

Para habilitar SSL en el servidor web de Urchin:

  1. Copie su archivo de certificado de SSL en el directorio var de Urchin y guárdelo con el nombre server.crt.
  2. Copie su archivo de clave de SSL en el directorio var de Urchin y guárdelo con el nombre server.key.
  3. Edite el archivo urchinwebd.conf.template que se encuentra en el directorio var de Urchin.
    1. (Omita este paso si utiliza Urchin 6.6 y versiones superiores.) Convierta en comentario la línea Port y elimine el comentario de la directiva Listen. 
                #XXXPortXXX
          Listen 443

    2. Cambie la directiva ServerName de localhost por el nombre de su servidor web. Por ejemplo: 

                ServerName: www.urchin.com

    3. NOTA: el valor de ServerName del archivo urchinwebd.conf.template debe coincidir con el nombre del servidor que se incluye en el archivo del certificado.

  4. Inicie o reinicie el servidor web utilizando urchinctl con la opción -e. Urchinctl se encuentra en el directorio bin de Urchin. La opción -e insta a urchinctl a que habilite SSL en el servidor web. Por ejemplo, para reiniciar el servidor web con SSL habilitado en el puerto 443: Para versiones anteriores a Urchin 6.6: 
       urchinctl -e -w restart
    Para Urchin 6.6 y versiones superiores: 
       urchinctl -e -p 443 -w restart

    Nota: para iniciar el servidor sin SSL habilitado, quite la opción -e.

Ahora puede acceder al servidor con SSL habilitado a través de https://servername.domain.com:port/

Nota: personalizar la configuración de SSL en el archivo urchinwebd.conf.template puede conllevar problemas que podrían impedir que se iniciase el servidor web.

Instrucciones para Windows

Para habilitar SSL en el servidor web de Urchin:

  1. Copie su archivo de certificado de SSL en el directorio var de Urchin y guárdelo con el nombre server.crt.
  2. Copie su archivo de clave de SSL en el directorio var de Urchin y guárdelo con el nombre server.key.
  3. Edite el archivo urchinwebd.conf.template que se encuentra en el directorio var de Urchin.
    1. (Omita este paso si utiliza Urchin 6.6 y versiones superiores.) Convierta en comentario la línea Port y elimine el comentario de la directiva Listen. 
                #XXXPortXXX
          Listen 443
    2. Elimine el comentario de la línea para cargar mod_ssl.so cuando se inicie el servidor web: 
                LoadModule ssl_module etc/mod_ssl.so

    3. Cambie la directiva ServerName de localhost por el nombre de su servidor web. Por ejemplo: 

                ServerName: www.urchin.com

    4. NOTA: el valor de ServerName del archivo urchinwebd.conf.template debe coincidir con el nombre del servidor que se incluye en el archivo de certificado.

  4. Suprima e instale el servicio de servidor web utilizando urchinctl con la opción -e. Urchinctl se encuentra en el directorio bin de Urchin. La opción -e insta a urchinctl a que habilite SSL en el servidor web. Por ejemplo, para suprimir e instalar el servidor web con SSL habilitado en el puerto 443: Para versiones anteriores a Urchin 6.6: 
       urchinctl.exe -w remove
   urchinctl.exe -w -e install
    Para Urchin 6.6 y versiones superiores: 
       urchinctl.exe -w remove
   urchinctl.exe -e -p 443 -w install

    Nota: para iniciar el servidor sin SSL habilitado, quite la opción -e.

Ahora puede acceder al servidor con SSL habilitado a través de https://servername.domain.com:port/

Nota: personalizar la configuración de SSL en el archivo urchinwebd.conf.template puede conllevar problemas que podrían impedir que se iniciase el servidor web.
