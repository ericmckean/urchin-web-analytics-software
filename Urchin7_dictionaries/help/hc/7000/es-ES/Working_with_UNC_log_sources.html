Uso de fuentes del archivo de registro UNC

Cuando se crea una fuente del archivo de registro UNC en Urchin, se especifican los parámetros siguientes: Server Name/IP, Remote Server Login, Local Server Login, Remote Log Source File.

A continuación, se explica el proceso mediante el cual Urchin obtiene el registro de una fuente UNC: - Urchin intenta buscar los archivos que se especifican en "Remote Log Source File". - Si no logra encontrarlos, Urchin intenta establecer conexión con la unidad compartida de UNC, que se especifica en el campo "Server Name/IP". - Si la conexión falla, Urchin asume que se está ejecutando en una cuenta de usuario sin acceso a la red, por ejemplo, una cuenta del sistema local. En este caso, crea una sesión de acceso para la cuenta de usuario, según se especifica en "Local Server Login", y suplanta la cuenta, es decir, Urchin empieza a procesarse en esta cuenta. Si las opciones "Same Local y Remote Login" se establecen en "Sí", las credenciales para la cuenta de suplantación se obtienen de "Remote Server Login". - Una vez que la cuenta de suplantación funciona correctamente, Urchin vuelve a intentar establecer conexión con la unidad compartida de red de UNC para posteriormente intentar buscar los archivos de nuevo. - Cuando Urchin termina el proceso con la fuente del archivo de registro, finaliza la conexión con UNC y también la suplantación, es decir, Urchin empieza a procesarse en la cuenta en la que se ejecutaba antes de la suplantación.

La suplantación presenta los siguientes inconvenientes: - Mientras la suplantación está habilitada, Urchin se ejecuta en una cuenta distinta, la cual quizás no tenga acceso al directorio de datos de Urchin, por lo que no podrán crearse bases de datos de informes. - Es posible que las bases de datos de informes creadas durante la suplantación no estén disponibles para el proceso de Urchin una vez finalizada la suplantación. - El rendimiento se ve afectado, sobre todo si hay varios archivos de registro UNC asociados al perfil, ya que Urchin debe iniciar o finalizar la suplantación para cada uno de esos archivos.

Así pues, esta es la mejor forma de proceder con unidades compartidas de UNC en Urchin: - Ejecute Urchin en la cuenta que disponga de acceso a la red. En este caso, no se produce la suplantación. - A la cuenta de usuario en la que se esté ejecutando Urchin, conceda acceso de lectura a las unidades compartidas de registro UNC. En este caso, quizás no sea necesario crear una conexión con UNC ni una cuenta de suplantación.
