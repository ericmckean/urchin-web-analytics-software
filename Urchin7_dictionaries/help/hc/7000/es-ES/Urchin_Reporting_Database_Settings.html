Configuración de la base de datos de informes de Urchin

Revisión: 16 de diciembre de 2008

Visión general

Urchin incorpora dos opciones para administrar la escalabilidad y la precisión de la base de datos de informes:

Límite de la tabla de la base de datos

Esta opción permite mantener los datos de sitios de gran tamaño dentro de unos límites razonables. Utilícela para restringir el número de registros de una única tabla de la base de datos de informes de Urchin. La opción predeterminada es apropiada para la mayoría de las instalaciones. Si, durante un mes, el número de registros de una tabla supera este límite, los registros sobrantes se agrupan en el último registro (denominado "Otros"). Por lo tanto, cuanto mayor sea el tamaño de la tabla, más precisos serán los datos de los informes.

Si un informe de Urchin indica que hay un alto porcentaje de entradas denominadas Otros, debería incrementar el límite de la tabla de la base de datos para aumentar la precisión.

Porcentaje de utilización de la memoria

Urchin proporciona esta opción para determinar el límite máximo de tamaño de la base de datos de informes de la memoria y así poder limitar la cantidad de memoria que puede asignarse durante el procesamiento de registros. Para sitios de tamaño superior, asignar más memoria puede aumentar el rendimiento, ya que se producen menos intercambios de páginas. Aunque la configuración predeterminada es conservadora, si el sistema dispone de suficiente memoria física se recomienda establecer un valor más alto.

Configuración experimental

Actualmente, Urchin admite los siguientes valores máximos para la configuración de la base de datos de informes:

Límite de la base de datos : <= 500.000 registros

Porcentaje de utilización de la memoria : <= 1024 MB

Empezando por la versión 6.5, Urchin puede admitir valores ampliados para los siguientes ajustes, que solo deben utilizar Usuarios de Urchin con experiencia. Dichos valores ampliados no se muestran en la interfaz de administración y deben configurarse a través de la base de datos de la configuración. A continuación, presentamos los valores máximos para la configuración ampliada de la base de datos de informes de Urchin 6.5:

Límite de la base de datos : <= 2.000.000 de registros

Porcentaje de utilización de la memoria : <= 2 GB

Directrices para la configuración experimental

  • Dado que esta configuración ampliada es experimental y puede producir errores de memoria insuficiente en algunos sistemas, debe tener en cuenta la siguiente información antes de utilizarla.

  • Por lo general, las aplicaciones de 32 bits disponen de cuatro gigabytes (GB) de espacio de dirección virtual, que se reparte entre el espacio de dirección del usuario y el espacio de dirección kernel. El proceso del usuario únicamente puede asignar memoria en el espacio de dirección del usuario.

  • Plataforma Windows: la aplicación del usuario dispone de 2 GB, mientras que los 2 GB restantes están a disposición del kernel. Aunque windows OS permite ampliar el espacio del usuario hasta 3 GB a través de mejora de 4 gigabytes, los procesos de Urchin aún no admiten este caso tan poco probable y siempre se recopilan o asignan con límites de 2 GB. Por lo tanto, en windows OS, el usuario no debe establecer un límite de la tabla de la base de datos superior a 500.000 registros ni un porcentaje de utilización de la memoria superior a 1 GB.

    • Para obtener el máximo rendimiento, establezca los siguientes valores:

      • Porcentaje de utilización de la memoria : 1.024 MB

      • Límites de la tabla de la base de datos : 500.000 filas en la tabla de la base de datos

    • Para obtener la máxima precisión, establezca los siguientes valores:

      • Porcentaje de utilización de la memoria : 256 MB

      • Límites de la tabla de la base de datos : 1.000.000 de filas en la tabla de la base de datos

  • Plataformas Unix: la aplicación del usuario dispone de 3 GB y el kernel dispone de 1 GB. Por lo tanto, en la plataforma UNIX, el usuario no debe establecer un límite de la tabla de la base de datos superior a 500.000 registros ni un porcentaje de utilización de la memoria superior a 2 GB.

    • Para obtener el máximo rendimiento, establezca los siguientes valores:

      • Porcentaje de utilización de la memoria : 2.048 MB

      • Límites de la tabla de la base de datos : 500.000 filas en la tabla de la base de datos

    • Para obtener la máxima precisión, establezca los siguientes valores:

      • Porcentaje de utilización de la memoria : 1.024 MB

      • Límites de la tabla de la base de datos : 2.000.000 de filas en la tabla de la base de datos

Configuración del 'límite de la tabla de la base de datos' ampliado

Como ocurre con el valor habitual del 'Límite de la tabla de la base de datos', estos valores ampliados pueden configurarse en los perfiles tanto de forma individual como global. La consulta siguiente establece el límite de la tabla de la base de datos en 1.000.000 de registros para un perfil con id '1'. 

UPDATE uprofiles SET uipr_db_table_limit=1000000 WHERE uspr_id=1;

Esta otra consulta establece el límite global de la tabla de la base de datos en 1.000.000 de registros. Se aplicará a todos los perfiles cuyo valor para el campo uipr_db_table_limit sea 0. 

UPDATE uglobals SET uigl_db_table_limit=1000000 WHERE ucgl_name='Global Settings';

Configuración del porcentaje de utilización de la memoria' ampliado

El porcentaje de utilización de la memoria solo puede establecerse de forma global.

Se aplica automáticamente a los perfiles cuando la utilización de la memoria de la base de datos se establece en 'límite del sistema de la caché' (o si el campo 'ubpr_limit_db_cache_size' de la base de datos se ajusta en 1).

La consulta que se indica a continuación establece el porcentaje global de utilización de la memoria en 2 GB. Recuerde que el campo de la base de datos interpreta los números como 'megabytes' (512 serían 512 MB). 

UPDATE uglobals SET uigl_db_cache_size=2048 WHERE ucgl_name='Global Settings';
