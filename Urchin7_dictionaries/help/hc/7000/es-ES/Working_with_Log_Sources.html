Uso de Fuentes del registro

Visión general

Al crear un perfil, se suele añadir una fuente del registro. Urchin identifica las características de un registro de acceso (a veces, denominado registro de transferencia) de uno de sus sitios web mediante una fuente del registro. Los registros de acceso contienen todos los resultados, o solicitudes de documentos web, de su sitio web. Algunas de las características del archivo de registro que se asocian con una fuente del registro son la ruta en la que se encuentra el primero, su formato (por ejemplo, W3C o NCSA), el hecho de que el registro sea local o se encuentre en un sistema remoto y el que deba aplicarse un filtro a este durante el procesamiento.

Un concepto importante a tener en cuenta es que las fuentes del registro existen independientemente de los perfiles. Cada perfil debe tener al menos una fuente del registro asociada a él para obtener informes; sin embargo, es posible que varios utilicen la misma fuente. Por ejemplo, puede crear varios perfiles utilizando la misma fuente del registro, pero aplicar a cada uno de ellos un filtro distinto para producir resultados de informe variados. Por lo tanto, no es necesaria una proporción 1:1 entre fuentes del registro y perfiles.

Configuración de fuentes del registro

Para comenzar a añadir una fuente del registro al sistema, acceda al sistema administrativo de Urchin como administrador y haga clic en el botón Configuración de la izquierda. A continuación, haga clic en el botón Administrador de registros. Para crear una nueva fuente del registro, haga clic en el botón Añadir, situado en la parte superior derecha de la pantalla. A continuación, accede al Asistente para añadir una fuente del registro, que le indica una serie de pasos sencillos para ayudarle a configurar la fuente del registro de forma rápida y sencilla. En todas sus pantallas se incluye ayuda detallada para explicar la información de configuración que se indica.

En la pantalla Configuración de registro visualiza que tiene que elegir un formato de registro. Esta configuración indica a Urchin la disposición de los datos en el archivo de registro. Es importante que seleccione el formato adecuado para su registro. En caso contrario, Urchin no podrá producir datos de informe relevantes. Urchin comprende un conjunto predeterminado de formatos de registro que puede elegir por medio de un menú desplegable. Estos son:

  • Automático: Urchin utiliza este formato para detectar de manera automática formatos de registro NCSA, W3C, Netscape y ELF2. En vez de seleccionar explícitamente uno de ellos, puede seleccionar Automático. De esta forma, Urchin deduce correctamente cómo leer los datos si su formato del registro está en esta lista.
  • NCSA: formato extendido/combinado modificado de Apache (consulte Registro: Apache e IIS para obtener una descripción de este formato).
  • W3C: los servidores Microsoft IIS utilizan normalmente este formato, aunque también se pueden configurar otros servidores web para producir registros W3C.
  • Netscape: los servidores Netscape e iPlanet utilizan este formato de manera predeterminada.
  • ELF2: formato del registro de E-Commerce.
  • Google: utilice este formato para registros que contengan datos de la inversión en coste por clic de Google. Tenga en cuenta que el formato de registros de Google no se puede detectar de manera automática.
  • Overture: utilice este formato para registros que contengan información sobre la inversión de coste por clic de Overture. Tenga en cuenta que el formato de registros de Overture no se puede detectar de manera automática.
  • Personalizado: aunque inicialmente no se muestre en el menú desplegable, puede crear sus propios formatos de registro, que aparecerán en el menú desplegable cuando estén debidamente configurados.

Si cree que su servidor web no está produciendo registros en alguno de los formatos predeterminados reconocidos puede, o bien volver a configurarlo para que lo haga en uno de estos formatos, o bien crear un formato de registro personalizado que se ajuste al modo en el que su servidor web realiza los registros en este momento. Si desea volver a configurar el registro del servidor web, debe seleccionar el registro de estilo W3C o NCSA.

Distribución de la carga y procesamiento de registros paralelos

En sitios web con tráfico intenso, el uso de un servidor web puede no ser suficiente. Mediante la distribución de la carga, el tráfico puede dispersarse entre varios servidores web que funcionen en paralelo. El conjunto de servidores web paralelos se denomina "clúster". De esta forma, el registro de toda la actividad del sitio web se realiza en cualquiera de los registros paralelos disponibles. El Asistente para configurar perfiles ofrece una opción de procesamiento de registros paralelos. Al activar el procesamiento de registros paralelos, Urchin abre todos los archivos de registro a la vez y los lee de manera rotatoria, una sección cada vez. Cada una de ellas corresponde a 15 minutos de actividad del registro. La habilitación de esta función aumenta de manera notable el rendimiento de los sitios de carga distribuida.
