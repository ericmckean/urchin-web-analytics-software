Las URL de cada paso de mi ruta de redireccionamiento definido son idénticas. ¿Qué debo hacer para realizar el seguimiento de cada uno de los pasos como si se tratase de una única URL?

Es posible que desee realizar el seguimiento del progreso de los usuarios por un redireccionamiento que utilice la misma URL para cada paso. Por ejemplo, su redireccionamiento de registro puede ser como este:

  • Paso 1 (Regístrese)
    • www.misitio.es/sign_up.cgi
  • Paso 2 (Aceptación del acuerdo)
    • www.misitio.es/sign_up.cgi
  • Paso 3 (Finalizar)
    • www.misitio.es/sign_up.cgi

Para solucionarlo, utilice en cada paso la función urchinTracker de Javascript (posiblemente, en un proceso de carga onload), tal y como se muestra a continuación:

  • urchinTracker("/funnel_G1/step1.html")
  • urchinTracker("/funnel_G1/step2.html")
  • urchinTracker("/funnel_G1/step3.html")

A continuación, configure los pasos de su redireccionamiento siguiendo estos parámetros:

  • http://www.misitio.es/funnel_G1/step1.html

  • http://www.misitio.es/funnel_G1/step2.html

  • http://www.misitio.es/funnel_G1/step3.html

Tenga en cuenta que ni el argumento de la ruta ni del nombre de archivo de urchinTracker() deben representar una ruta o nombre de archivo existentes. El argumento de urchinTracker sólo ofrece un nombre de página inventado al que Urchin puede adjuntar vistas de página.
