¿Qué debo hacer para actualizar a Urchin 6?

  • Contribuciones efectuadas a este artículo por Analytics Pros, asesor autorizado del software Urchin

No existe ruta de actualización de Urchin 5 a Urchin 6; sin embargo, podrá migrar sus datos de una versión del producto a la otra. Siga las instrucciones incluidas en este artículo para migrar su configuración (cuentas, usuarios, etc.) y datos de informes.

  • Notas importantes:

    • No instale Urchin 6 en la misma carpeta que Urchin 5. Si lo hace, habrá interferencias entre las versiones 6 y 5 de Urchin. Además, el instalador de Urchin no emitirá advertencias si intenta instalar Urchin 6 en la misma carpeta que Urchin 5.

    • No se recomienda migrar datos con meses de solapamiento entre los datos de informes de las versiones 5 y 6. Si la base de datos de destino ya contiene datos para un mes que intenta importar, dicha operación será mucho más lenta que si no los contuviera.

Migración de los datos de configuración

1. En la instalación de Urchin 5, ejecute la utilidad uconf-export. Esta recopila el contenido de la base de datos de configuración, que en Urchin 5 se trata de un archivo sin formato. De forma predeterminada (sin introducir argumentos), la utilidad exporta el texto a la salida estándar, en este caso, el monitor. Sólo tiene que introducir el argumento -f para indicar el nombre del archivo al cual desea exportar los datos. Nota: el archivo de salida debe ser accesible desde la instalación de Urchin 6. 

uconf-export -f PATH+FILE_NAME

2. Desde la instalación de Urchin 6, ejecute 

uconf2sql -f PATH+FILE_NAME

donde PATH+FILE_NAME representa el archivo al que se han exportado los datos.

Con esta operación concluye el proceso de migración de los datos de configuración de Urchin 5 a Urchin 6.

Migración de los datos de informe

1. Desde Urchin 5, guarde el directorio 'Urchin5 BASE' en una ubicación a la que se pueda acceder desde Urchin 6. Por ejemplo, si ejecuta Urchin 5 y 6 en Windows, puede compartir el directorio de Urchin 5 y, a continuación, montarlo como una unidad en el sistema de archivos de Urchin 6.

2. En Urchin 6, ejecute la utilidad convert-u5data mediante este procedimiento: 

convert-u5data  [-a cuenta] [-p NOMBRE_PERFIL] -c RUTA_2_U5_Directorio_de_instalación

Con ello, se recopilarán todos los datos de informes de los archivos de Urchin 5 y se migrarán a Urchin 6.

Notas de uso de convert-u5data

  • -a es opcional; si no se introduce, se buscarán perfiles en todas las cuentas para realizar la migración.
  • -p es opcional; si no se introduce, se migrarán todos los perfiles incluidos en las cuentas localizadas.
  • -d es opcional; se utiliza para ejecutar la utilidad en modo de depuración (Urchin 6.6 y versiones superiores).
  • -z es opcional; se utiliza para convertir bases de datos archivadas (Urchin 6.6 y versiones superiores).

Nota: si ha personalizado la ubicación del directorio de datos de Urchin mediante la modificación de ./Urchin5_Install_Base/etc/urchin.conf para usar una ubicación distinta de "./data" deberá asegurarse de que la ubicación y el usuario que ejecuta convert-u5data tienen acceso a la misma ruta que la definida en urchin.conf. Por ejemplo, si ha actualizado urchin.conf para usar "D:\urchin5_data\", el sistema que ejecuta convert-u5data buscará los datos de Urchin 5 en "D:\urchin5_data\". Si se trata de una ruta que no es válida en el sistema donde se ejecuta el comando, aparecerá el mensaje de error Error al cambiar de directorio. Saliendo. En este caso, el procedimiento recomendado consiste en copiar la carpeta de datos de Urchin 5 en la carpeta "data" de la ubicación "Urchin5_Install_Base" y actualizar urchin.conf para utilizar "./data" como la ubicación de la carpeta de datos.
