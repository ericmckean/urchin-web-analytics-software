La utilización de NFS precisa bloqueo

La configuración de Urchin, que utiliza archivos de datos situados en la carpeta data/conf de la instalación, establece bloqueos de lectura y escritura durante la configuración y administración. Si la carpeta data/conf se sitúa sobre NFS, será necesario que el servidor NFS ejecute los demonios de bloqueo adecuados para gestionar el bloqueo remoto.

Si el bloqueo no funciona correctamente, la instalación de Urchin podría detenerse por tiempo indefinido.

Notas técnicas

Urchin utiliza la función fcntl() en UNIX para realizar bloqueos exclusivos y de aviso (lectura/escritura). Es posible que algunos sistemas anteriores no admitan todas las solicitudes de bloqueo de rpc de distintas plataformas. Asegúrese de utilizar versiones recientes de su OS con las últimas revisiones. Cerciórese de que el servidor de NFS ejecuta 'statd' y 'lockd' o equivalente.
