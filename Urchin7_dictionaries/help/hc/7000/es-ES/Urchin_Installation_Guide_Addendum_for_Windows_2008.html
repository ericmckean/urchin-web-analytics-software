Las notas de este artículo corresponden a la instalación de Urchin en Windows Server 2008 y en Windows Vista. Los pasos descritos en la Guía de instalación de Urchin(Windows) funcionan para Windows Server 2008 y Windows Vista, sin embargo, se debe tener en cuenta lo mencionado a continuación.

Windows 2008 incluye una función de seguridad nueva denominada "User Account Control" (UAC). Si desea obtener más información, visite http://technet.microsoft.com/en-us/library/cc709691.aspx.

La activación de la función UAC comporta estos problemas conocidos:

  1. Si se activa UAC, la aplicación no puede escribir correctamente en la carpeta "Archivos de programa". Más información en la página: http://msdn.microsoft.com/en-us/library/aa905330.aspx

  2. Los archivos ejecutables de Urchin no pueden ejecutarse desde un símbolo del sistema predeterminado. Debe utilizar el símbolo del sistema como administrador o bien establecer las propiedades de dicho archivo como se indica a continuación: haga clic con el botón derecho y seleccione "Propiedades"-> "Compatibilidad"-> "Ejecutar este programa como administrador" -> "Aceptar". Tenga en cuenta que los servicios de Urchin Windows operan tal y como se espera sin ningún esfuerzo adicional necesario.

Solución:

  1. Desactive la función UAC e instale Urchin donde desee. Ejecute Urchin con los servicios de Windows o con los archivos binarios.
  2. Si la función UAC está activa, no instale Urchin en la carpeta "Archivos de programa". Instálela, por ejemplo, en el directorio "Usuarios". Cambie las propiedades para ejecutar archivos binarios en modo de administrador (tal y como hemos descrito anteriormente en el apartado 2) o utilice los servicios de Windows sin ningún cambio.
