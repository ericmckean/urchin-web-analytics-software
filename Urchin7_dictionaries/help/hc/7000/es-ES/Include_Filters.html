Filtros "Incluir"/"Excluir"

Introducción

Los filtros "Incluir" y "Excluir", configurados en la interfaz del administrador y aplicados a un origen de registro o perfil se emplean para eliminar resultados no deseados al procesar un archivo de registros. Utilizan expresiones regulares POSIX para las concordancias de datos en los campos de un resultado. Si no está familiarizado con las expresiones regulares, consulte el documento Visión general de expresiones regulares que se incluye en esta sección antes de continuar.

Uso de los filtros "Incluir" y "Excluir" de Urchin

Estos filtros se aplican después de los filtros "Buscar y reemplazar", "Tabla de consulta" y "Avanzado". Urchin aplica los filtros "Incluir" y "Excluir" de forma consecutiva. Si se aplica el filtro "Excluir" y el patrón coincide con los datos, Urchin descarta el resultado y pasa al siguiente. En caso de que el patrón no coincida, Urchin aplica a dicho resultado el filtro siguiente. Esto significa que se puede crear un único filtro "Excluir" con varios patrones separados por '|' o bien varios filtros "Excluir" con un solo patrón para cada uno.

Los filtros "Incluir" se aplican con la lógica inversa. Cuando se aplica un filtro "Incluir", el resultado se descarta si el patrón no coincide con los datos. Si se aplican varios filtros Incluir, el resultado debe coincidir con todos ellos a fin de que se guarde el resultado. Si desea englobar varios patrones de un determinado campo, genere un único filtro "Incluir" con expresiones individuales separadas por '|'.

Uso de los filtros "Incluir" y "Excluir"

En la ilustración anterior, el filtro "Excluir" requiere una expresión y un campo de filtro. Durante el procesamiento, la expresión del filtro se compara con el campo de filtro y el resultado se descarta si el filtro coincide con dicha información. Consulte la lista completa de campos disponibles en el artículo "Campos de filtro". El ejemplo anterior ilustra cómo excluir resultados de imagen filtrando todos los tipos de MIME que coinciden con gif, jpg, png, jpeg e ico. La lista puede personalizarse para hacer coincidir los datos con cualquier tipo de MIME.

En la ilustración anterior, el filtro Incluir requiere una expresión y un campo de filtro. Durante el procesamiento, la expresión del filtro se compara con el campo de filtro y el resultado se descarta si el filtro no coincide con dicha información. Consulte la lista completa de campos disponibles en el artículo '¿Qué información proporcionan los campos de filtro?'. El ejemplo anterior muestra cómo filtrar sólo resultados, para ello, debe especificar la subcarpeta mydir de la solicitud.

Controles

El control 'Distinción entre mayúsculas y minúsculas' le permite especificar si el filtro debe aplicarse siguiendo dicho parámetro.
