Uso de UTM con alias de dominio

Información preliminar

Puesto que las cookies son objetos basados en un dominio, existen algunas consideraciones importantes para tener en cuenta cuando un sitio tiene varios dominios. Una cookie que se establece bajo un dominio "misitio.es" se transfiere a todos los subdominios tales como "www.misitio.es". Por el contrario, esta cookie no se transfiere a "misitio.net" u otros dominios raíz diferentes.

Si el sitio web solo tiene un dominio que responde a "misitio.es" y "www.misitio.es", proceda con la instalación estándar de UTM. Sin embargo, si el sitio web tiene uno o varios alias, le recomendamos redirigir el tráfico de los alias al sitio principal. De este modo, se asegura de que el seguimiento de usuarios de UTM se establece bajo el dominio principal y de que se realiza el seguimiento de todos los usuarios de manera coherente.

Si no procedemos de esta manera, es probable que un usuario aparezca como dos usuarios en el caso de que accedan al mismo sitio a través de dos dominios distintos. Las siguientes instrucciones proporcionan un ejemplo de cómo redirigir el tráfico de los dominios con alias al dominio principal en los servidores IIS y Apache.

Redirección de alias en Apache

Si utiliza un servidor web Apache, la configuración se puede modificar fácilmente para redirigir todo el tráfico originado en uno de los alias al sitio principal. Una forma de hacerlo es crear dos entradas de host virtual. El primero sería el dominio principal, que incluye su configuración normal, y el segundo host virtual incluye todos los alias y redirige el tráfico al sitio principal. Ejemplo: 

#---primary virtualhost
<VirtualHost 1.2.3.4>
Nombredeservidor www.misitio.es
Aliasdeservidor misitio.es
...
</VirtualHost>
#---second virtualhost
<VirtualHost 1.2.3.4>
Nombredeservidor misitio.org
Aliasdeservidor www.misitio.org misitio.net www.misitio.net
RewriteEngine on
RewriteRule ^(.*)       http://www.misitio.es$1     [R=301]
</VirtualHost>

El host virtual secundario utiliza una regla de reescritura con un código de redirección 301 (trasladado permanentemente) para reenviar todo el tráfico al sitio original. Los resultados del código 301 se registran igualmente en el archivo de registro; resulta útil para saber los dominios a los que acceden los usuarios, pero todo el tráfico restante es forzado al único dominio. Llegados a este punto, con respecto a UTM, el sitio parece tener un único dominio y está listo para una instalación normal de UTM.

Nota: tenga en cuenta que debe trabajar con su administrador y consultar el sitio apache.org para conocer los parámetros de configuración.

Redirección de alias en servidores IIS

Si utiliza un servidor web Microsoft IIS, la configuración se puede modificar fácilmente para redirigir todo el tráfico originado en uno de los alias al sitio principal. Una forma de hacerlo es crear dos sitios web en la configuración del servidor IIS. El primero sería el dominio principal (www.misitio.es), que incluye su configuración normal, y el segundo incluye todos los alias (misitio.net, misitio.org, etc.) y redirige el tráfico al sitio principal.

En el administrador del servidor IIS, haga clic con el botón derecho del ratón en uno de los sitios web para que aparezca el cuadro de diálogo Propiedades. En la pestaña "Sito web", haga clic en el botón "Avanzado...". Se abre una ventana en la que puede asignar dominios adicionales al sitio web mediante el campo "Cabecera del host". Defina el dominio principal en el sitio web principal y utilice el sitio web secundario para albergar todos los alias.

Una vez que haya configurado y habilitado el alojamiento de todos los alias en el sitio web secundario, cree una página web vacía con el siguiente código de redirección: 

<head>
<META HTTP-EQUIV=Refresh CONTENT="0; URL=http://www.misitio.es/">
</head>

Esto redirige el navegador del usuario inmediatamente a la URL principal. En este momento, el sitio web principal aparece como una configuración de un único dominio, y se puede proceder a la instalación habitual de UTM con los ajustes predeterminados.
