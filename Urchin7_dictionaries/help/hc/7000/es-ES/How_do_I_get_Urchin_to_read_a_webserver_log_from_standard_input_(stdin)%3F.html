¿Qué debo hacer para que Urchin lea un registro de servidor web desde una entrada estándar (stdin)?

En determinados casos es recomendable que el motor de procesamiento de registros de Urchin pueda leer los datos de registro del servidor web directamente desde una entrada estándar (stdin), en lugar de realizar la lectura desde un archivo determinado. Urchin permite configurar una fuente de registro que realice la lectura desde entradas estándar en lugar de un archivo.

Para configurar Urchin para que realice este tipo de lectura, en primer lugar configure una nueva fuente de registro denominada "stdin" en el administrador de fuentes de registros. Para ello, utilice los siguientes parámetros:

Configuración de registro

  • Ubicación del archivo de registro: local
  • Ruta del archivo de registro: -
  • Formato del registro: automático

Configuración avanzada

  • Destino del registro: No tocar
  • Token de consulta: ?
  • Raíz de URI en minúscula: desactivar

NOTA: es muy importante que la "ruta del archivo de registro" se establezca en un solo guión ("-") (sin comillas).

Ahora, si quiere que un perfil lea registros desde "stdin", sólo tiene que seleccionar la nueva fuente de registro "stdin" en la pantalla de configuración Perfil -> Fuente de registro.

Nota importante: no puede ejecutar perfiles de Urchin que tengan configurado una fuente de registro "stdin" a través del Programador de Urchin ya que este no puede asociar correctamente el flujo de entrada de registros. El único modo de procesar registros para un perfil con una fuente de registro "stdin" es ejecutar Urchin de forma manual. A continuación, se incluye un sencillo ejemplo de cómo invocar a Urchin:

  1. Abra una ventana de shell de comandos en el sistema Urchin.
  2. Cambie el directorio a "bin" de la copia de Urchin.

  3. [sistemas de tipo UNIX] Ejecute urchin con "./urchin -p myprofile < /path/to/webserver.log".

    • [sistemas Windows] Ejecute urchin con "urchin -p myprofile < C:\path\to\webserver.log".

Utilizar "stdin" como fuente de registro le otorga mucha flexibilidad, ya que ahora puede preprocesar fácilmente un registro antes de que Urchin lo lea.

Otros ejemplos

Extraiga resultados para un único host virtual desde un registro formado por resultados de diversos hosts virtuales diferentes y envíe los resultados a Urchin: 

grep " www.midominio.es " /path/to/alldomains.log | urchin -p mydomain.com

Extraiga resultados de servidor web sin procesar desde una base de datos relacional utilizando algún tipo de secuencia de comandos SQL incrustada y envíelos a Urchin para que los procese. 

sql-extract --date 20040302 --domain mydomain.com | urchin -p mydomain.com
