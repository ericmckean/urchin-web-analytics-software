urchin: Procesamiento de registros de Urchin

Visión general

Urchin Log Processing Engine, urchin, es el componente principal de procesamiento de registros de Urchin. Normalmente, este motor se ejecuta desde el Urchin Scheduler (urchind) cuando se ejecuta una tarea. Sin embargo, es posible ejecutar urchin directamente desde un shell de comandos para ejecutar un perfil concreto. Esto resulta muy útil en entornos con muchas secuencias de comandos donde se ejecutan las tareas Urchin desde una fuente externa, como el Programador de tareas de Windows o el cron en los sistemas de tipo UNIX. También es muy útil para ejecutar un perfil en determinadas circunstancias como, por ejemplo, para procesar resultados de un día en particular solamente o para realizar algún tipo de depuración.

Urchin no es una utilidad realmente: se encuentra documentado aquí porque ofrece ciertas funciones limitadas en la línea de comandos que pueden ser útiles en algunos entornos.

Utilización

urchin se encuentra ubicado en el directorio bin de la distribución de Urchin. Sus funciones son: 

urchin [options...] (-p <profile name> | -P <profile id>)

Opciones: 

-a <account name>       especifica el nombre de la cuenta del perfil; el nombre predeterminado es '(NINGUNO)'.

-D                      habilita el modo de depuración.

-d <YYYYMMDD>           especifica la fecha del nuevo procesamiento.

-H                      especifica la ejecución de registros de un archivo de salida a un archivo de historial; se crea un nuevo registro del historial de tareas si se especifica -H y no se especifica -T

-h                      imprime la información de ayuda y sale del programa,

-P <profile id>         especifica el ID del perfil a procesar,

-p <profile name>       especifica el nombre del perfil que debe procesarse.

-T <task id>            especifica el registro del historial de tareas que debe actualizarse; si se especifica -T, se debe especificar también –H.

-t                      imprime los datos de configuración y sale del programa.

-v                      imprime el número de versión y sale del programa.

Nota importante

  • En los sistemas de tipo UNIX, urchin debe ejecutarse como el usuario/UID en el que Urchin se encuentra instalado para asegurarse de que las bases de datos del perfil pertenecen a ese UID, ya que urchin las creará si no existen.
