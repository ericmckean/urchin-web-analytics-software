¿Existe alguna limitación en el tamaño del archivo de registro?

El motor de procesamiento de registros de Urchin puede procesar registros del servidor web superiores a 2GB, independientemente de si se han comprimido con bzip (.bz2), gzip (.gz) o zip (.zip), ya que Urchin descomprime los archivos utilizando la biblioteca zlib.

Sin embargo, el motor de procesamiento de registros de Urchin utiliza los binarios descomprimidos/comprimidos del directorio util para crear copias de seguridad y archivos de los datos de informes de cada perfil. Si los informes de un mes superan los 2GB una vez comprimidos, Urchin no podrá realizar copias de seguridad ni archivos, por lo que se producirá un error en el procesamiento del registro, a menos que estén inhabilitadas las copias de seguridad y los archivos automáticos. En ese caso, debe realizar manualmente copias de seguridad de los archivos por medio de una utilidad externa.
