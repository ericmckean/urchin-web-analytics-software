Administración de registros

Visión general

La administración de registros constituye un aspecto importante cuando se ejecuta software como Urchin. Dado que los sitios muy concurridos generan grandes archivos de registro muy rápidamente (en algunos casos, de hasta varios gigabytes en un mes), debería considerar la administración de registros detenidamente. Se recomienda el establecimiento de una práctica de rotación de registros estándar, como la compresión u otros métodos de archivo convencionales. Consulte Prácticas recomendadas para la rotación de registros para obtener más información acerca del establecimiento de un procedimiento de este tipo. La administración de registros es necesaria solo por motivos de uso de recursos, no para evitar que se vuelvan a procesar datos. Urchin no necesita ningún tipo de rotación de registros para evitar la duplicación de datos, ya que está equipado con una función de seguimiento del registro que asegura que no se vuelvan a procesar datos del registro ya leídos. Para Urchin no es necesario volver a leer un archivo de registro una vez procesado, si lo desea, puede eliminar los registros tras cada ejecución del procesamiento. Sin embargo, es habitual mantener registros antiguos durante un periodo de tiempo determinado por motivos de auditorías o para guardar un historial.

Administración de registros por medio de Urchin

Cada fuente del registro tiene una configuración de destino del registro con las opciones No tocar, Archivar/Comprimir y Suprimir (Configuración -> Perfiles de Urchin -> Administrador de registros -> Editar (casi cualquier perfil) -> pestaña "Configuración avanzada"). Una vez que haya finalizado el procesamiento de todos los perfiles que utilizan una fuente del registro, Urchin utiliza la configuración de destino del registro para determinar la disposición de la fuente del registro. Es posible acceder a la configuración de destino de registro de una fuente del registro determinada en la pestaña "Configuración avanzada". Se recomienda que configure el destino del registro en "Archivar/Comprimir" para poder ahorrar espacio en el disco si desea guardar sus registros durante un período determinado. Si está de acuerdo con el hecho de que una vez que haya procesado un registro, éste se elimine, puede determinar que el destino del registro sea "Suprimir". Sin embargo, tenga en cuenta que esto significa que en el futuro no tendrá la opción de volver a ejecutar ese registro con Urchin, a menos que disponga de una copia de seguridad en algún otro sitio.

Nota importante

Deberán tenerse en consideración las siguientes situaciones especiales:

  • No utilice las opciones Archivar ni Suprimir con una fuente del registro si está procesando registros activos. Un registro activo es aquel en el que está escribiendo un servidor web. La utilización de esta configuración con un registro activo produce una pérdida de datos.
  • Si el destino del registro de una fuente del registro recuperado de forma remota está configurado en No tocar, dicho registro crecerá de forma continua a menos que exista algún proceso externo a Urchin que esté gestionando la administración de registros en el equipo en el que se creó el registro. Dado que Urchin debe transferir una copia del archivo de registro remoto al sistema local antes del procesamiento, a medida que el archivo aumente el tamaño, Urchin necesita más tiempo para transferirlo. Como efecto secundario, se prolonga el tiempo de ejecución global de Urchin.
