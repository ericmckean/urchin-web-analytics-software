geo-update: Utilidad Actualización de la base de datos DNS

Visión general

La utilidad geo-update se utiliza para comprobar las actualizaciones de las bases de datos DNS internas de Urchin y para descargar las actualizaciones en caso de que estén disponibles. Permite descargar las actualizaciones de los servidores de descarga de Google, así como de algún directorio local (novedad en Urchin 6).

Utilización

geo-update puede ejecutarse mediante el Programador de Urchin (urchind) cada mes o de forma manual a través de una línea de comandos. Cuando se hace mediante el Programador, siempre se actualiza desde el servidor de descarga de Google; y cuando se hace a través de una línea de comandos, ofrece la opción adicional de actualizar la base de datos DNS desde una ruta local. Esta tarea se establece para comprobar si hay nuevas actualizaciones una vez al mes y el usuario puede configurarla para que tenga lugar en cualquier momento a través de la interfaz de administración.

Si se crea una tarea para geo-update, esta aparecerá en la pantalla "Historial de tareas" con el nombre de "geo-update".

La función predeterminada es comprobar la existencia de actualizaciones, descargar nuevas bases de datos y sobrescribir las existentes. Las funciones predeterminadas pueden sustituirse ejecutando geo-update desde la línea de comandos utilizando las opciones adecuadas. El funcionamiento es el siguiente: 

geo-update [-Hhv] [-D | -F] [-i archivo]

donde 

  -F            Obliga a descargar las bases de datos geográficas.
  -f            Nombre de la ruta del archivo "geo.zip.file" en el sistema de archivos local (opcional).
  -H            Especifica la ejecución de registros de un archivo de salida a un archivo de historial.
  -h            Imprime la información de ayuda y sale del programa.
  -T <task id>  Especifica el registro del historial de tareas para actualizar.
                Nota: "-T" requiere el uso de "-H".
  -V            Imprime el número de versión de la base de datos geográficos y sale del programa.
  -v            Imprime el número de versión y sale del programa.

Mediante la opción "-F" se producirá la descarga de nuevas bases de datos, incluso si estas ya están actualizadas. Esta función es de utilidad si importa información de dominio personalizada que no es correcta o si su base de datos del dominio ha quedado de algún modo perjudicada y desea empezar todo desde cero.

El hecho de registrar resultados en un archivo del historial mediante la opción "-H" hará que la información esté disponible al consultar la pantalla "Historial de tareas" en el área "Configuración->"Programador de la interfaz de administración de Urchin".

El uso de la opción -T obliga al programa a actualizar una tarea particular desde la interfaz de la línea de comandos especificada por la tarea <Task_ID>. El valor "Task_ID" (identificador de la tarea) puede hallarse en la base de datos de Urchin 6 realizando la siguiente consulta: 

mysql> select * from ugeo_tasks\G

Ejemplos

Para obligar al programa a descargar la base de datos de dominios más reciente: 

geo-update -F

Para actualizar la tarea de forma manual desde la interfaz de la línea de comandos (cuando ya se obtiene el identificador de tarea de la base de datos): 

geo-update -T <Task_ID> -H

geo-update requiere conexión a Internet para poder buscar y descargar las nuevas actualizaciones. La utilidad emplea el puerto 80 para comunicarse con el servidor web que ofrece las últimas actualizaciones. Es posible que haya tenido problemas al pasar por cortafuegos y servidores proxy cuando se llevan a cabo las actualizaciones. En ese caso, consulte a su administrador de red.
