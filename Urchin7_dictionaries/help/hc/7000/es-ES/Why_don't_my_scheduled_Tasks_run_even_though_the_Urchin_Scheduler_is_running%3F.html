¿Por qué no se ejecutan mis tareas programadas aunque sí se ejecute el Programador Urchin?

Descripción del problema

En ciertas circunstancias, el Programador Urchin (urchind) puede dejar de ejecutar las tareas programadas incluso cuando se ejecuta el Programador. Algunos síntomas de esto son los siguientes:

  1. Las tareas se muestran como programadas en la pantalla Configuración -> Programador -> Programador de tareas de la herramienta de administración Urchin y se reprograman como deben, pero no se llegan a ejecutar.

  2. Al hacer clic en el botón "Ejecutar ahora" de la pantalla Ejecutar/Programar perfil, aparece la ventana emergente de progreso, pero en la barra de progreso solo se muestra "Error (0%)" y la tarea no se completa nunca.

Causa

Normalmente, esto se debe a que el Programador Urchin se inició desde un directorio que se ha eliminado o para el cual se modificaron los permisos, de modo que Urchin ya no tiene acceso a él.

Por ejemplo, es muy común descomprimir el instalador de Urchin en un directorio temporal para llevar a cabo una instalación o actualización. Durante el proceso de instalación/actualización, el Programador Urchin se inicia con el directorio activo en ese momento como directorio temporal. Si ese directorio se elimina posteriormente, el Programador Urchin dejará de funcionar correctamente.

Solución

Para garantizar que las aplicaciones Urchin pueden iniciarse y ejecutarse correctamente, asegúrese de que inicia el Programador Urchin desde una ubicación permanente a la que Urchin tenga siempre los permisos de acceso adecuados. Si tiene problemas con el Programador, lo más conveniente es reiniciar los servicios de Urchin mediante el siguiente procedimiento:

Sistemas Windows

  1. Inicio -> Programas -> Urchin -> Inhabilitar servicios Urchin

  2. Inicio -> Programas -> Urchin -> Habilitar servicios Urchin

Sistemas de tipo UNIX

  1. Inicie un shell de comandos como "root" o como el usuario que ejecuta Urchin.
  2. Cambie el directorio a la ubicación donde está instalado Urchin, por ejemplo "cd /usr/local/urchin"
  3. Reinicie los servicios de Urchin con "bin/urchinctl restart"
