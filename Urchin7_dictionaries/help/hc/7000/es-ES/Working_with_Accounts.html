Uso de las cuentas

Visión general

Una cuenta es una asociación de primer nivel que se utiliza para agrupar perfiles relacionados, fuentes de registro, usuarios y grupos bajo una única etiqueta de identificación, que normalmente es el nombre de una empresa o un cliente. Para las instalaciones de Urchin donde existan numerosas y complejas organizaciones de clientes, el hecho de crear una cuenta le permite al administrador de Urchin realizar fácilmente un seguimiento de todos los componentes de informes de Urchin para un cliente particular o una entidad corporativa. Los derechos de acceso a los informes de Urchin pueden controlarse a través de una asociación de cuenta, y dentro de una cuenta se pueden asignar aún más detalles de los derechos de acceso a los grupos o usuarios. De esta manera se protege su información en numerosos niveles y de la manera que lo desee. Asimismo, los derechos de administración del nivel de la cuenta pueden asignarse a un usuario que puede de esta manera actuar como un administrador local de Urchin para la cuenta. Esto permite repartir la responsabilidad de administración, configuración y mantenimiento de los informes de Urchin en una cuenta.

Es importante saber que, ya que las cuentas son el "elemento" organizativo de máximo nivel en la interfaz de administración de Urchin, debe crear las cuentas antes de crear perfiles, fuentes de registro, usuarios o grupos que posteriormente se asociarán a la cuenta. La elección de una cuenta debe hacerse cuando se crea un elemento de Urchin y no puede cambiarse luego. Al momento de crear una cuenta, si no elije una cuenta específica se configurará una cuenta predeterminada de (NINGUNA). Esto le indica a Urchin que el elemento en cuestión no tiene cuenta.

Creación de cuentas

Para crear una cuenta, vaya a la pantalla Configuración->Usuarios y grupos->Cuentas y haga clic en el botón Añadir. Solo se requiere el Nombre de cuenta. Urchin no utiliza el nombre de contacto ni el correo electrónico de contacto ya que son campos estrictamente informativos para beneficio del administrador. La ubicación de los datos de informes (opcional) especifica el lugar del almacenamiento de los datos del informe de todos los perfiles que pertenecen a la cuenta en cuestión. La ubicación predeterminada es el directorio data/reports en la distribución de Urchin. Cambiar la ubicación de los datos de informes le permite separar físicamente dentro de su sistema de archivos los datos de los informes para diferentes organizaciones. La ubicación de navegación del directorio (opcional) está pensada como medida de seguridad al otorgar acceso al administrador de una cuenta para crear perfiles. El directorio que se introduzca en este campo limitará la capacidad del administrador de la cuenta para navegar por los archivos de registro de ese directorio solamente. De manera predeterminada, el administrador de cuenta puede buscar archivos de registro en cualquier ubicación.

Uso de cuentas

Para asignar un perfil, fuente de registro, usuario o grupo a una cuenta, utilice el menú desplegable que tiene la etiqueta Cuenta opcional en la pantalla inicial del asistente de configuración al crear el elemento en cuestión. Una vez que una cuenta se asigna a un perfil, fuente de registro, uUsuario, etc., la interfaz del administrador de Urchin restringe las opciones de modificación para aquellos elementos que se asocian con la cuenta. De este modo, la cuenta actúa para controlar los derechos de acceso a un alto nivel para que pueda aislar una organización de otra sin necesidad de establecer permisos de acceso específicos en cada informe.

Las cuentas, además, ayudan repartir las responsabilidades de administración para la configuración de Urchin. Dentro de una cuenta, el administrador principal de Urchin puede asignar privilegios locales de administración a los usuarios de cuentas. Se pueden asignar cuatro niveles administrativos de control a un usuario. Consulte Uso de la función Usuarios y grupos.

Cuando vea las pantallas de administración de los parámetros de configuración de Urchin, podrá filtrar las entradas para mostrar de forma selectiva sólo aquellos con un nombre de cuenta particular mediante el menú desplegable Cuenta ubicado en la barra superior de la tabla. Si bien (NINGUNA) significa que no hay ninguna cuenta asociada, por razones de filtrado (NINGUNA) se muestra como una opción en el menú desplegable Cuenta. De este modo, podrá ver sólo las entradas que no están asociadas con ninguna cuenta.
