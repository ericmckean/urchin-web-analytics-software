Visión general de las utilidades de administración

Visión general

Urchin incorpora una serie de utilidades de diagnóstico y configuración que se encuentran en el directorio útil de la distribución de Urchin. Este documento ofrece una vista general a modo de introducción de estas utilidades. Tenga en cuenta que no se trata de una guía de uso exhaustiva.

Todas las utilidades admiten los argumentos de línea de comandos "-h" y "-v". Cuando se ejecuta una utilidad mediante el argumento "-h", se obtiene un resumen sobre su utilización y las opciones disponibles para dicha herramienta. Cuando se hace mediante el argumento "-v", se imprime la versión Urchin de la utilidad.

geo-update

Esta utilidad se emplea para comprobar las actualizaciones de la base de datos DNS interna de Urchin y para descargar las actualizaciones en caso de que estén disponibles. Permite descargar las actualizaciones de los servidores de descarga de Google, así como de algún directorio local (novedad en urchin 6).

inspector

Esta utilidad realiza comprobaciones básicas en la distribución de Urchin instalada para garantizar que la estructura general de la distribución permanezca intacta, que todos los archivos binarios incorporados estén actualizados a la versión adecuada y que los permisos subyacentes sean correctos (en plataformas de tipo UNIX). La utilidad también informa de los estados de funcionamiento de los Programadores de perfiles Urchin (urchind), de los Programadores de CPC (urchincpcd, urchincpc) y del servidor web Apache integrado (urchinwebd). Además, comprueba la información de la licencia.

uconf-manager (Urchin 6.402+)

La utilidad Administrador de configuración de Urchin (uconf-manager) permite añadir, editar y eliminar registros de la base de datos.

  • Urchin 6.402: Esta utilidad se incorporó en primer lugar a Urchin 6.402 y solo funcionaba en la tabla 'uusers' de la base de datos de la configuración. Puede utilizarse para restablecer la contraseña de administrador de Urchin, para automatizar la creación de usuarios y para moverlos a una cuenta distinta.
  • Urchin 6.500+: En urchin 6.5 y en las versiones posteriores, esta utilidad es compatible con operaciones de la base de datos de más tablas (uaccounts, ucosts, ufilters, ugoals, ugroups, ugroups_uusers, ulogs, ulogs_ufilters, uprofiles, uprofiles_ucosts, uprofiles_ufilters, uprofiles_ugroups, uprofiles_ulogs, uprofiles_uusers, y uusers). Asimismo, puede utilizarse para realizar acciones que no puedan llevarse a cabo a través de la interfaz de administración de Urchin.

uconf2sql

Urchin 5 y las versiones anteriores guardan los datos de configuración en archivos de propiedades; sin embargo, Urchin 6 los guarda en bases de datos relacionales (MySql o PostgreSql). Para una transición correcta de los datos de configuración, tanto Urchin 5 como Urchin 6 ofrecen utilidades para exportarlos e importarlos. Urchin 5 cuenta con una utilidad, denominada uconf-export, que permite exportar la base de datos de configuración a un archivo (en formato XML), mientras que Urchin 6 cuenta con otra, denominada uconf2sql, que importa los datos de configuración del archivo XML a la base de datos de Urchin 6.

convert-u5data

Al igual que sucede con la base de datos de configuración, la estructura de la base de datos de informes (que guarda información sobre los informes) también se ha modificado en Urchin 6. Esta utilidad, convert-u5data, ofrece la posibilidad de importar las bases de datos de informes directamente de la instalación de Urchin 5 a la instalación de urchin 6.

udb-sanitizer

La utilidad udb-sanitizer permite realizar operaciones relacionadas con la administración de las bases de datos de informes, lo que incluye volver a la versión anterior de una base de datos guardada y eliminar información sobre una fecha determinada o sobre un intervalo de fechas (en caso de que sea necesario un nuevo procesamiento).

urchinctl

Nota: la utilidad reside en el directorio bin, no en el directorio útil. Esta utilidad permite iniciar y detener el Programador de Urchin y los servicios del servidor web de Urchin. En sistemas de tipo UNIX, urchinctl normalmente se invoca mediante una secuencia de comandos de arranque para iniciar o cerrar de forma automática los servicios de Urchin. En plataformas windows, también se pueden instalar y desinstalar estos servicios de urchin.

urchincpc (Urchin 6.5+)

Nota: la utilidad reside en el directorio bin, no en el directorio útil. La herramienta Importación de datos de CPC de Urchin extrae los datos de CPC mediante el API de Google AdWords para procesarlos y para crear informes. Si desea obtener más información, consulte Utilidades de importación de datos de CPC.

urchincpcd (Urchin 6.5+)

Nota: la utilidad reside en el directorio bin, no en el directorio útil. El Programador de origen de CPC de Urchin (o, simplemente, Programador de CPC) se utiliza para descargar tareas en función de la programación que defina el usuario (similar al Programador de perfiles). Si desea obtener más información, consulte Utilidades de importación de datos de CPC.

urchin_daemons

En los equipos de tipo UNIX, Urchin ejecuta un daemon de programación de tareas (urchind) y un servidor web Apache (urchinwebd). Estos programas deberían iniciarse en el arranque. La secuencia de comandos de urchin_daemons puede añadirse a las secuencias de comandos de inicialización del sistema en la ubicación adecuada para su sistema operativo (OS) UNIX y hace que los daemon de los servicios de Urchin se ejecuten correctamente. En plataformas windows, estos archivos binarios se ejecutan como servicios del sistema y pueden iniciarse y detenerse mediante el administrador de servicios de windows.

ugetlicense

La utilidad de funcionamiento de licencias de Urchin, ugetlicense, permite realizar operaciones relativas al número de serie, que serán necesarias en el caso de que una instalación de Urchin vigente deje de estar disponible y deba volver a instalarse o migrarse a un servidor distinto, etc.
