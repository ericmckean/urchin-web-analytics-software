Lista de cambios de Urchin 6.6, 6.601 y 6.602

Funciones incorporadas

A continuación, se incluye un resumen de las principales características incorporadas en las versiones Urchin 6.6, Urchin 6.601 y Urchin 6.602.

  • Integración avanzada de Google AdWords

    • Alertas de presupuesto: esta función advierte si hay que reajustar el presupuesto de una campaña de AdWords. Si alguna de las fuentes de CPC (asociadas al perfil que se está visualizando) incluye una alerta de presupuesto en el sistema de AdWords, se muestra de forma automática en la interfaz de informes.

    • Herramienta de generación de palabras clave: esta herramienta permite generar palabras clave relacionadas y conocer su rendimiento. Asimismo, permite añadirlas a las campañas de AdWords y eliminar otras que ya existan.

    • Enlaces directos a AdWords: esta función permite navegar directamente de Urchin a la pantalla correspondiente de su cuenta de AdWords.

    • Administrador de etiquetas de Urchin: esta función incluye una etiqueta de inserción de palabras clave dinámicas, {keyword}, en las URL de destino de los anuncios. Simplifica el proceso de etiquetado de URL y permite importar fácilmente a Urchin datos de costes de AdWords.

      • (consulte https://adwords.google.com/support/bin/answer.py?hl=es&answer=74996)

    • Optimizador de AdWords: esta herramienta permite optimizar las campañas directamente en AdWords.

  • Vista y sección para anunciantes

    • Plantilla del panel para anunciantes (vista) y sección “Optimización de la publicidad”: Estas mejoras, relacionadas con el marketing en búsquedas y dirigidas a anunciantes, ofrecen formas nuevas para ver los datos en un entorno centralizado para publicidad de pago y para proporcionar las funciones relacionadas con AdWords mencionadas anteriormente.

  • Informes nuevos

    • Informe "Tiempo en el sitio web": este informe se encuentra en “Optimización del contenido” -> “Rendimiento del contenido” -> “Métricas de implicación”.

    • Informe "Comparativa del rendimiento": este informe permite comparar el rendimiento de campañas, palabras clave y contenido de las distintas fuentes y medios (p. ej., Google|CPC y Yahoo|orgánico). Lo encontrará en “Optimización de la publicidad” -> “Resultados de la campaña de marketing”.

    • "Estructura de CPC": este conjunto de informes se encuentra en “Optimización de la publicidad”.
      • En “Vista de campañas” se muestra información de las campañas de pago asociadas al Supervisor de tráfico de Urchin (UTM). Se incluye información para campañas de Google y de Yahoo.
      • En “Vista de palabras clave” se ofrece información sobre las palabras clave de pago.

  • API de datos

    • API de exportación de datos: se trata de una función nueva e importante que permite obtener datos de informes de Urchin a través de los protocolos SOAP 1.x y REST.

  • Autenticación externa (LDAP)

    • Además de la autenticación específica de Urchin (ahora denominada "autenticación nativa"), Urchin 6.6 admite autenticación externa que puede configurarse según cada usuario.
    • Módulos de autenticación externa: Urchin 6.6 ofrece compatibilidad con los sistemas de autenticación LDAP y MSAD. Además, se pueden integrar módulos personalizados modificando los archivos de configuración.
    • Compatibilidad con SSL/TLS: ahora, el programa también admite estos protocolos.

  • Nueva "página" principal de Urchin

    • Se ha modificado la "página principal" de Urchin (también denominada Informe de lista completa) de forma que ahora incorpora métricas de los perfiles a los que tiene acceso el usuario en cuestión. El usuario puede escoger una de las tres vistas siguientes: “Básico”, “Empresa” y “Administrador”.
    • Las métricas de los perfiles corresponden a los valores de un día (el anterior) y a la última semana, mes y año.

  • Importación automatizada de datos de CPC procedentes de Yahoo! Search Marketing (YSM)

    • Urchin no solo permite importar datos de CPC procedentes de Google AdWords, sino que ahora estos datos también pueden provenir de Yahoo! Search Marketing.

  • Programa de instalación de un solo clic

    • Ahora, todos los programas de instalación de Urchin, excepto FreeBSD 6, incluyen el servidor de bases de datos Postgresql, el cual puede instalarse durante el proceso de instalación. Con ello, evita tener que instalar y configurar un servidor de bases de datos antes de instalar Urchin, tal y como obligan los requisitos. Si lo desea, el usuario puede utilizar servidores que ya tenga. Para ello, deberá inhabilitar la opción durante el proceso de instalación. NOTA: es posible que se experimenten problemas con Linux Kernel 2.4. Para obtener más información al respecto, consulte el documento “Problemas conocidos”.

  • Carga optimizada de GeoDB

    • Ahora puede escoger entre la versión completa o la versión reducida de GeoDB para minimizar el consumo de memoria durante el procesamiento de los registros. La versión reducida de GeoDB contiene información de área metropolitana relativa al país únicamente.

  • Utilidad Zip\UnZip actualizada

    • La utilidad Zip actualizada permite trabajar con archivos de datos superiores a 2 Gb.

  • Clasificación de datos en el panel del informe Resumen de marketing

    • Las fuentes, palabras clave y campañas del informe Resumen de marketing pueden ordenarse por métricas y sus posibles desviaciones.

  • Modificaciones varias

    • Control de procesos: optimizador de AdWords.

    • Acceso de usuario a las herramientas de AdWords: se ha incorporado una opción nueva, "Acceso a las fuentes de CPC" según cada usuario. De forma predeterminada, solo el superadministrador tiene habilitadas las herramientas de AdWords. Para el resto de los usuarios, el acceso debe concederse de forma explícita a través de esta nueva opción.

    • Bing.com: se ha añadido compatibilidad para el nuevo motor de búsqueda bing.com de Microsoft.
    • Filtros de perfil: ahora, estos filtros se aplican a las fuentes de CPC.
    • Pestaña “Perfiles”: en “Administración de fuentes del archivo de registro” se ha añadido una pestaña nueva, "“Perfiles”", que enlaza la fuente del archivo de registro con los distintos perfiles.
    • Descarga remota de archivos de registro: en el archivo urchin.conf se ha añadido un parámetro configurable para la caducidad del tiempo de espera en las descargas remotas de registros.

Correcciones

A continuación, se incluye un resumen de las principales correcciones incorporadas en las versiones Urchin 6.6, Urchin 6.601 y Urchin 6.602.

  • Administración

    • En Urchin 6.5 y versiones anteriores, al modificar el nombre de un perfil en la interfaz de administrador, las carpetas asociadas no se modificaban. Esto ya se ha corregido.

  • Procesamiento de registros

    • Había casos en los que Urchin generaba un error de segmentación al procesar registros ELF2. Esto daba lugar a procesamientos incompletos de los registros y a incoherencias en los datos de los informes. Esto ya se ha corregido.
    • La aplicación de la configuración de hora local del perfil durante el procesamiento de los registros era incorrecta. Esto daba lugar a cambios de hora en los datos de los informes en los distintos meses. El problema ocurría en los meses en los que se iniciaba o finalizaba el cambio de hora en la zona horaria del perfil en cuestión. Esto ya se ha corregido.

    • Se ha solucionado el problema con el procesamiento de varias fuentes del archivo de registro de UNC. Si desea obtener información sobre el resto de los problemas asociados a las fuentes del archivo de registro de UNC y recomendaciones al respecto, consulte el siguiente artículo de ayuda online: https://secure.urchin.com/helpwiki/en/Working_with_UNC_log_sources

  • Compatibilidad con SSL

    • Se ha corregido el problema de conexión a Urchin a través de https en Windows 2003.

  • Caracteres especiales en contraseñas de fuentes de CPC

    • Ciertos caracteres especiales no estaban permitidos en las contraseñas de fuentes de CPC. Esto se ha corregido y ahora los caracteres !@#$%^~`&* _+»<>{}[]()?-=:;\|/,. también están admitidos.

  • Licencias

    • En Linux y en FreeBSD se requería volver a configurar y a activar la licencia después de actualizar el programa a la versión 6.600. Esto se ha solucionado en Urchin 6.601 y ya no es necesario volver a activar la licencia.
    • Antes del lanzamiento de Urchin 6.602, actualizar el programa con una licencia de prueba podía dar lugar a problemas de acceso y de licencia. Esto ya se ha corregido. NOTA: si se actualiza una versión de Urchin asociada a una licencia de prueba, el plazo de esta última no se ampliará.
    • Para licencias normales, el número máximo de fuentes del archivo de registro ha pasado de 1.000 a 3.000. Este cambio requiere la generación de una nueva clave de licencia desde el servidor de licencias de urchin. Las nuevas instalaciones que se realicen de urchin (independientemente de la versión de urchin) incorporarán este cambio de forma automática. En el caso de las instalaciones existentes o de las actualizaciones, será necesaria la ejecución manual de la utilidad ugetlicense por parte de los administradores de urchin.

  • Migración

    • Se ha mejorado considerablemente la utilidad de migración de datos de informes (convert-u5data); su rendimiento también se ha optimizado. En general, ahora se tardará mucho menos en importar grandes volúmenes de datos de informes.
    • En algunos casos, cuando fallaba o se interrumpía la migración de las bases de datos de informes de Urchin 5 a Urchin 6, los intentos posteriores para reanudar la importación no daban resultado y se mostraba el mensaje de advertencia siguiente: "ADVERTENCIA: (8010-577-1192) el tamaño del archivo de la base de datos es incorrecto. Ejecute sanitizer". Esto ya se ha corregido.
    • Algunos datos de informes no se importaban correctamente a urchin 6 durante la migración de u5 a u6. Esto ya se ha corregido.

  • Modificaciones varias

    • Se ha actualizado la ayuda integrada en Urchin y varios artículos del Centro de asistencia online de Urchin.

Notas importantes sobre el uso

  • MySQL

    • Urchin 6.6xx no es compatible con MySQL 4.x.x. Para que Urchin 6.6 funcione correctamente, debe actualizar MySQL a MySQL 5.03+.

  • FreeBSD 5.x

    • Se ha excluido la plataforma FreeBSD 5.x.

  • Cookies de cliente

    • Con el fin de facilitar una mejor interacción en la interfaz de informes, Urchin 6.6 emplea cookies de cliente. Para ver correctamente los informes, las cookies de cliente deben estar habilitadas en el navegador del cliente.

  • Uso restringido de la licencia de prueba

    • En Urchin 6.6, la licencia de prueba presenta ciertas restricciones. Se admiten hasta cinco perfiles y cinco fuentes del archivo de registro, pero no hay restricciones en cuanto a filtros, fuentes de CPC, usuarios, cuentas o grupos.

  • Actualización

    • Urchin 6.6 incorpora una sección de informes nueva, además de vistas nuevas. Si se actualiza el programa a partir de una versión anterior, es posible que estos informes y estas vistas no muestren datos. Se recomienda volver a procesar los perfiles existentes para incluir información en los informes nuevos.
