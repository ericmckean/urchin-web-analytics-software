Funcionamiento del Supervisor de tráfico de Urchin (UTM)

Visión general

El Supervisor de tráfico de Urchin (UTM), pendiente de patente y disponible por vez primera en Urchin 4, se diseñó especialmente para medir de forma precisa los usuarios únicos de un sitio web. Para empresas que buscan entender mejor el comportamiento de sus usuarios en línea, el UTM ofrece una tecnología extremadamente valiosa que combina lo mejor de la información del servidor y el cliente, al tiempo que permite controlar los datos. Esta tecnología de fácil instalación permite a los propietarios de las empresas identificar con precisión usuarios únicos y rutas de clics, así como realizar mediciones de fidelidad en las que se incluyen usuarios que acceden por primera vez, usuarios que vuelven y la frecuencia de uso. Las versiones más recientes de UTM ofrecen una capacidad de seguimiento de campañas mejorada.

Existen dos componentes en el sistema del Supervisor de tráfico de Urchin: el Sensor UTM, que es un fragmento de código que se instala en el contenido del sitio web; y el Motor UTM, que forma parte del Motor Urchin de procesamiento de registros. El Sensor UTM permite la recopilación de datos del cliente, que se redireccionan a Urchin para cada visita de página. El Motor UTM combina la información del cliente con los datos existentes del servidor para ofrecer una descripción más completa y ajustada de la actividad del sitio web.

El Sensor UTM es un pequeño código JavaScript que tiene dos importantes funciones: en primer lugar, evita los efectos de la memoria caché al forzar la progresión de al menos un resultado a los servidores web de Google para cada visita de página. En segundo lugar, el Sensor UTM identifica de forma exclusiva a cada usuario utilizando cookies de origen del cliente para mantener un seguimiento de los usuarios que acceden por primera vez y los usuarios que vuelven. Este identificador de cookies es una etiqueta de comunicación visible solamente para el servidor web, al igual que los numeros de identificación de sesión. No se trata de una cookie de terceros que envía información fuera del sistema infringiendo muchas políticas de privacidad.

El sistema UTM funciona del modo siguiente. El servidor web ofrece dos funciones básicas: la entrega de contenido y el registro. El Sensor UTM, que se entrega al navegador del usuario, se incluye en el contenido del sitio web. El Sensor UTM establece identificadores únicos y devuelve una solicitud adicional al mismo servidor. El Motor UTM, que forma parte del motor de procesamiento de registros de Urchin, utiliza los datos de esta solicitud para ofrecer una descripción más completa y ajustada del comportamiento del usuario.

Sensor UTM

El Sensor UTM aumenta la precisión y exhaustividad de los datos del archivo de registro impidiendo el efecto del almacenamiento en la memoria caché y el proxy. Por ejemplo, si un usuario vuelve a una página ya visitada anteriormente, su contenido se le ofrece a partir de la memoria caché del navegador. Este mismo modelo se aplica en el caso de que quien ofrezca la información almacenada en la memoria caché sea un proveedor de servicios. El servidor web desconoce esta visita de página; sin embargo, el Sensor UTM activa un resultado único adicional que fuerza al menos la vuelta de un pequeño registro al servidor. Esta información se registra en los archivos de registro, que ahora conocen la visita de página original.

La segunda función importante del Sensor UTM es la identificación exclusiva de sesiones o de usuarios. Por medio de una combinación de cookies del navegador pendiente de patente, el Sensor detecta e inicializa identificadores de sesión y usuario únicos, lo que permite la exacta supervisión de los usuarios que vuelven, independientemente del comportamiento del proxy proveedor de servicios. La mayor parte de los proveedores de servicios se aprovechan del uso de proxy reciclando direcciones IP y concentrando usuarios tras cortafuegos. Esto puede producir problemas con el seguimiento de archivos de registro habituales, que normalmente utiliza la dirección IP como un identificador para el usuario.

El Sensor UTM puede traspasar el proxy utilizando los identificadores de las cookies en vez de direcciones IP. Por ejemplo, la primera vez que un usuario único accede a un sitio web a través de un cortafuegos con una dirección IP, la visita de página entregada contiene el Sensor UTM, que establece el identificador en el navegador del usuario.

Cuando vuelva el mismo usuario, el ID único se pasa al servidor web junto con cada solicitud, por lo que incluso aunque se le haya asignado una segunda dirección IP al usuario, la tecnología UTM identifica correctamente al usuario con el ID original. Además de evitar los efectos de complejas técnicas de proxy, esto permite realizar un seguimiento de usuarios que viajan y puedan utilizar sus equipos portátiles desde distintas ubicaciones y conectarse a través de distintos proveedores.
