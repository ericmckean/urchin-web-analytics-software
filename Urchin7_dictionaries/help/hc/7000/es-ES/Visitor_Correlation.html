Correlación del usuario

Visión general

La correlación del usuario es el proceso de identificación del comportamiento del usuario aun si las sesiones proceden de diferentes archivos de registro, de una web independiente o de los servidores E-commerce. Urchin utiliza datos de las diferentes fuentes de registro para analizar las relaciones entre las sesiones y las transacciones y luego coteja esta información para proporcionar una idea clara de la asociación entre la actividad del usuario y las compras. De ese modo, ofrece un valioso informe del retorno de la inversión. Por ejemplo, las referencias de los motores de búsqueda y las palabras clave específicas pueden correlacionarse con el valor de la compra del sitio E-commerce para informarle qué referencias y términos de búsqueda son más rentables. Por lo general, tiene un registro para su sitio web público y otro registro de su sitio web de transacción segura. Para correlacionar fuentes de datos dispares, Urchin debe utilizar el mismo método de identificación de usuario para cada sitio.

Tipos de correlación del usuario

Urchin puede correlacionar usuarios en función de varios métodos. Su elección del método de seguimiento de usuarios afecta directamente la precisión de la información que Urchin dispone para correlacionar las transacciones de E-commerce con otra actividad del sitio web. Elija el método adecuado según el nivel de detalle que desea obtener.

Para cada uno de los métodos del usuario se requieren los siguientes datos en el archivo de registro de E-commerce:

  • UTM: cookies
  • IP+User-Agent: host remoto o dirección IP y user-agent (es decir, tipo de navegador.

Estos elementos deben tenerse en cuenta cuando examina su formato de registros de E-commerce. Revise Formato de registro de ELF2 y Registros de E-commerce personalizados para asegurarse de que los registros contengan la información adecuada.

Configuración

En esta sección se presenta una visión general de los problemas de configuración de la correlación del usuario de e-commerce.

Sitios sin UTM

Si no tiene instalado el sensor de UTM en sus sitios, la configuración de la correlación del usuario dependerá del formato e-commerce que utilice.

  • ELF2: utilice IP+User-Agent como método de seguimiento de usuarios. En tal caso, Urchin automáticamente correlaciona las sesiones siempre que todos los registros contengan los campos IP y User-Agent. Simplemente elija IP+User-Agent en Método de seguimiento de usuarios en la pantalla Informes del perfil.

Sitios con UTM

Para los sitios con UTM, debe instalarse la misma versión del sensor de UTM en todas las páginas de las que desee hacer un seguimiento. En la pantalla Informes del perfil, configure Método de seguimiento de usuarios en Supervisor de tráfico de Urchin. En condiciones normales, deje la opción Dominio del UTM en blanco. El UTM detecta automáticamente el dominio del sitio. La función Dominio del UTM solo se utiliza en el caso de dominios que se superpongan y que requieran un seguimiento por separado.
