Requisitos de configuración para Urchin 6

  • Autor: EpikOne, asesor autorizado del software Urchin

En este artículo se enumeran los problemas de funcionamiento que se deben tener en cuenta antes de instalar y ejecutar Urchin. Se incluye información necesaria para el funcionamiento correcto del programa, así como otros datos importantes para maximizar el uso de Urchin una vez instalado.

Información básica para la instalación de Urchin

  • En Windows, deberá instalar el programa cuando haya accedido como administrador.
  • En sistemas del tipo UNIX, lo puede instalar como cualquier usuario, pero si no lo instala como superusuario, solo podrá instalarlo en determinadas partes del sistema de archivos.
  • Urchin se suministra con un archivo binario del servidor web Apache para la configuración y creación de informes. Los administradores del sistema deberían saber que este nuevo servicio web se ejecutará después de instalar Urchin.
  • Debe instalar Urchin 6 en una parte del disco que tenga bastante espacio (por ejemplo, cientos de megabytes disponibles) para que el aumento de datos en las bases de datos no suponga un problema a la larga. Consulte la sección sobre los problemas con la administración y el rendimiento para obtener más información al respecto.

  • Si va a actualizar de Urchin 5, deberá importar las bases de datos a Urchin 6 mediante las utilidades uconf2sql y convert-u5data. No es posible realizar una actualización directa de Urchin 5 a Urchin 6 con solo ejecutar el archivo de instalación de Urchin 6. Consulte el apartado sobre actualizaciones en la sección “Cómo empezar” del centro de documentación.

  • La actualización de Urchin 5 a Urchin 6 requiere registrar de nuevo la licencia del producto.

Información básica sobre el procesamiento de Urchin

  • Acceso a los registros del servidor web: debe conocer la ruta de los archivos de registro del sitio en cuestión y debe tener permiso para acceder a ellos. Si los registros se encuentran en un sistema remoto, también necesitará un nombre de cuenta y una contraseña para obtener los registros (FTP/HTTP/HTTPS/UNC).
  • Formato del registro configurado correctamente: aunque Urchin puede procesar formatos de registro personalizados, los requisitos de administración se simplifican si configura el servidor web para poder registrar un formato estándar. Se recomienda utilizar Extended Combined Log Format o formato de registro combinado extendido (por ejemplo, NCSA o registros Apache) o W3C Extended Log Format, un formato de registro extendido W3C (por ejemplo, registros IIS). Para los sitios IIS, se debe desactivar el registro de procesos. Consulte la sección sobre configuraciones avanzadas para obtener más información al respecto.
  • Cuenta de usuario único para procesos de Urchin: en sistemas de tipo UNIX, es aconsejable extremar la seguridad ejecutando los programas Urchin como un ID de usuario especial que se utilice exclusivamente para Urchin y solo tenga privilegios limitados. Para configurar dicha cuenta es necesario tener los privilegios correspondientes o de superusuario en el sistema en cuestión.
  • Programación: deberá definir una programación para el procesamiento de Urchin para enviar los informes de forma oportuna, así como el tiempo necesario para procesar grandes cantidades de datos.

Información sobre el procesamiento avanzado de Urchin

  • Si desea supervisar los usuarios únicos, realice los siguientes pasos:
    • Instale el código de sensor del Supervisor de tráfico de Urchin (UTM) en las páginas web del sitio en cuestión.
    • Active el registro de cookies en el formato de registro para el servidor web.
    • Establezca el método de seguimiento a Supervisor de tráfico de Urchin (UTM) en el perfil de Urchin del sitio web.
  • Si decide no supervisar los usuarios únicos, debería definir el grado de información que desea reflejar en los informes de usuario y de sesión y, a continuación, seleccionar la opción adecuada para el método de seguimiento de usuario en cada sitio. El otro único método de seguimiento disponible en Urchin 6 es IP/User-Agent.

Problemas con el rendimiento y la administración

  • Rotación del registro: si no cuenta con un mecanismo externo para archivar o eliminar registros del servidor web una vez procesados por Urchin, puede configurar dicho programa para que realice esta tarea. Vaya a “Configuraciones avanzadas” para cada “Origen del registro”.
  • Conservación de bases de datos anteriores de Urchin para informes del historial: cuando se crea una base de datos de un mes determinado, la información está disponible para realizar los análisis del historial que considere oportunos. Los usuarios deben decidir los datos de historial que deben conservar y eliminar el resto con el fin de ahorrar espacio en el disco. Se puede configurar Urchin para que comprima las bases de datos anteriores a una determinada fecha.
  • Requisitos de memoria: Urchin cuenta con controles de configuración para limitar la cantidad de RAM que utiliza cuando procesa registros. La configuración predeterminada es 128 MB, lo cual puede ser insuficiente para sitios con registros mayores de 10 MB. Procure disponer de RAM suficiente en caso de que Urchin requiera un uso mayor de la memoria. Ajuste la configuración de la memoria del software para optimizar el rendimiento.
  • Ubicación del almacenamiento de datos de Urchin: mediante el archivo etc/urchin.conf, Urchin se puede configurar para almacenar las bases de datos de informes en una área del sistema de archivos fuera de la distribución Urchin. Esto permite asignar una cantidad suficiente de espacio al sistema de archivos en caso de que se amplíen las bases de datos.

Problemas de acceso remoto e integración

  • Uso de SSL (Secure Socket Layer, nivel de zócalo de seguridad) para la administración e informes de Urchin: el servidor web Apache que viene integrado con Urchin cuenta con asistencia para SSL. La configuración predeterminada no tiene activado el SSL; sin embargo, se puede activar según desee el usuario.
  • Configuración del cortafuegos: si la topología de la red incluye cortafuegos, servidores proxy y otros dispositivos entre el servidor de procesamiento de Urchin y los usuarios que desean ver informes o los sistemas cuyos registros deben obtenerse, habrá que configurar dichos dispositivos para que no interfieran con el acceso remoto de Urchin. Por lo general, esto se puede realizar sin menoscabar la seguridad que ofrece dicha topología.
