¿Cómo puedo realizar el seguimiento de todos los subdominios de mi sitio web en un perfil?

  • Autor: ivantage Limited, asesor autorizado del software Urchin

Es posible realizar el seguimiento de subdominios dentro del mismo perfil que el dominio. Para ello, debe añadir una línea al código de seguimiento en cada una de las páginas del subdominio. En las instrucciones a continuación se presupone que este perfil se configuró con el dominio principal como el sitio web indicado (p. ej., ejemplo.co.uk)

  1. Añada la siguiente línea (en negrita) al código de seguimiento de todas las páginas tanto de su subdominio como de su dominio principal: 

         <script src="/urchin.js" type="text/javascript">
         </script>
         <script type="text/javascript">
           _udn="example.co.uk";
          </script>
  • Esta línea debe hacer referencia al dominio principal (ejemplo.co.uk).

Tenga en cuenta que si bien este cambio le permite realizar el seguimiento de subdominios dentro de un perfil, es posible que sus informes no establezcan la diferencia entre páginas de www.ejemplo.co.uk y de ayuda.ejemplo.co.uk. Por ejemplo, en el informe "Contenido principal" puede obtener resultados de www.ejemplo.co.uk/indice.html y de ayuda.ejemplo.co.uk/masinfo.html; sin embargo, el informe muestra lo siguiente: 

   /index.html
   /more.html

Si desea establecer diferencias entre los subdominios, puede crear un filtro "Avanzado" para su perfil con la siguiente configuración: 

   Filter Type: Custom filter > Advanced
   Field A: cs_host (Raw)
   Extract A: (.*)
   Field B: Request_Stem (Auto)
   Extract B: (.*)
   Output To: Request_Stem (Auto)
   Constructor: /$A1$B1

Con este filtro configurado, los ejemplos anteriores aparecen con el subdominio adjunto: 

   www.ejemplo.co.uk/indice.html
   ayuda.ejemplo.co.uk/masinfo.html

NOTA: este filtro depende del nombre de host de su sitio web, incluido el subdominio proporcionado a su archivo de registro de acceso web. Si su archivo de registro de acceso web no contiene el nombre de host, los informes de Urchin 6 no podrán registrar la diferencia, por ejemplo, entre indice.html en los dos dominios: 

 www.ejemplo.co.uk/indice.html
 ayuda.ejemplo.co.uk/masinfo.html
