Registro: iPlanet

Visión general

En este artículo se ofrece una vista general acerca de cómo configurar el registro para un servidor web iPlanet para facilitar el procesamiento y la creación de informes para Urchin.

Utilice el tipo "Netscape" para la configuración de la fuente del registro.

Existe un conjunto de campos obligatorios mínimos necesarios para que Urchin produzca informes. Estos son:

  • fecha
  • hora
  • nombre de host o dirección IP del sistema que lo solicita
  • solicitud (por ejemplo, los documentos que solicitó el sistema de su servidor web)
  • código de estado generado por solicitud (numérico)
  • bytes (bytes transferidos del servidor al cliente)

Además, para realizar informes más completos necesita los campos siguientes:

  • referencia
  • user-agent
  • cookies (si el Supervisor de tráfico de Urchin está instalado en su sitio)

Configuración 

Init fn=flex-init access="$accesslog"
format.access="%Ses->client.ip% - %Req->vars.auth-user%
[%SYSDATE%] \"%Req->reqpb.clf-request%\"
%Req->srvhdrs.clf-status% %Req->srvhdrs.content-length%
\"%Req->headers.user-agent%\"
\"%Req->headers.referer%\"
\"%Req->headers.cookie%\""
