Informes de listas completas (6.602)

Los informes de listas completas permiten revisar el rendimiento de varios perfiles en “Página principal” de Urchin. Esta página ofrece tres tipos de vista, "Básico", “Administrador“ y “Empresa”, que puede seleccionar en la lista desplegable “Vista”.

Si hace clic en el botón “Editar”, accederá a la pestaña “Configuración del perfil”/“Informes”. En la columna “Estado” se muestra el estado del perfil, que es independiente del periodo seleccionado. El icono “Filtrado” () aparece junto a los perfiles que han sido filtrados. Junto a los perfiles que no tienen fuentes de CPC asociadas se muestra el icono “Cuentas de anuncios” (). Haga clic en este icono para acceder a AdWords. Si se han asociado fuentes de CPC al perfil, la columna “Cuentas de anuncios” muestra el número de fuentes de CPC. Acceda a los informes haciendo clic en un nombre perfil. Para acceder al sitio en cuestión, haga clic en el enlace situado junto al nombre de perfil.

A continuación, se muestran las vistas “Básico”, “Administrador” y “Empresa”.

Con el selector de periodo podrá seleccionar los datos del día, la semana, el mes y el año más recientes.

Los datos del periodo seleccionado se muestran en las columnas "Resultados", "Duración media de la visita", "Bytes", "Visitas", "Porcentaje de rebotes", "Objetivos cumplidos" y "Número de páginas vistas". En el resto de las columnas ("Nombre del perfil", "Estado", "Fuentes del archivo de registro", "Filtrado" y "Cuentas de anuncios") se incluye información relacionada con el perfil.

Cada usuario puede ver los datos según sus permisos. En las tablas siguientes se muestra una comparativa de la configuración de los permisos.

En Urchin 6.602 y en versiones posteriores, la vista predeterminada de “Página principal” (Informes de lista RollUp) de cada usuario puede definirse de forma manual en la base de datos de configuración de Urchin a través de los campos uglobals.uigl_rollup_view_type y uusers.uius_rollup_view_type:

  • Valores de uglobals.uigl_rollup_view_type:
    • 0: auto
      • Este es el valor predeterminado en la base de datos. En este caso, el valor general del tipo de vista se deduce del tipo de usuario:

        • “Superadministrador”, “Administrador de la cuenta” -> vista “Administrador”.

          “Usuario”, “Usuario: sin actualizar” -> vista “Empresa”

    • 1: Vista “Básico”
    • 2: Vista “Administrador”
    • 3: Vista “Empresa”
  • uusers.view_type:
    • -1: utilizar valor general
      • Este es el valor predeterminado cuando se crea un usuario. En este caso, se utiliza el tipo de vista de lista completa general (uglobals.uigl_rollup_view_type) como el tipo de vista para el usuario en cuestión.
    • 0: auto
      • En este caso, el tipo de vista de usuario se deduce del tipo de usuario:

        • “Superadministrador”, “Administrador de la cuenta” -> vista “Administrador”.

          “Usuario”, “Usuario: sin actualizar” -> vista “Empresa”

    • 1: Vista “Básico”
    • 2: Vista “Administrador”
    • 3: Vista “Empresa”

NOTAS DE USO IMPORTANTES

  • Si se intenta acceder a una cuenta que contiene una gran cantidad de datos, puede aparecer el error siguiente: 

    Session Error - unable to call default routine. (2002) Please login again.
  • Esto se debe a que la extracción de datos para los informes de listas completas ha superado el tiempo de espera predeterminado de la aplicación (APPLICATION_TIMEOUT) de 45 segundos. Posibles soluciones:
  • Aumente el valor de APPLICATION_TIMEOUT en el archivo XYZ para resolver el problema. Tenga en cuenta las repercusiones de este cambio.
  • Cambie el tipo de vista a "Básico". Para ello, actualice el campo uglobals.uigl_rollup_view_type y todos los campos uusers.uius_rollup_view_type de la base de datos de configuración de Urchin con el valor '1'.
