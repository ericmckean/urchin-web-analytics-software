Visión general de las funciones de integración de Urchin

Introducción

La interfaz de Urchin está formada por componentes modulares y técnicas estándar del sector para permitir que los administradores integren el acceso a los informes de Urchin en sus infraestructuras presentes o futuras. Como parte de nuestro compromiso por alojar proveedores y centros de datos, probamos y admitimos tres puntos de integración que deberían permitir a Urchin integrarse en la mayoría de las arquitecturas.

El siguiente diagrama muestra los componentes principales de la interfaz de Urchin y los tres puntos de integración. Las funciones de administración y generación de informes se basan en Internet y se ofrecen desde este sistema.

Todo el contenido del sistema Urchin se ofrece a través de un servidor Apache que está instalado como el ejemplo "urchinwebd" que se muestra en la parte izquierda del diagrama. El servidor Apache gestiona las solicitudes y las pasa mediante la interfaz de CGI a una aplicación de controlador de sesión (session.cgi). Este realiza un proceso de autenticación y, según la acción, pasa la solicitud al motor de administración (admin) o al motor de informes de Urchin (urchin.cgi). El contenido con identificadores de sesión incrustados vuelve al usuario.

Los tres puntos marcados como "A", "B" y "C" son los tres puntos de integración mencionados anteriormente. El punto "A" permite al administrador sustituir u omitir el servidor web de Urchin. El punto "B" permite realizar una autenticación externa u omitir el proceso de autenticación. Y el punto "C" permite el acceso directo a la generación de informes a través de un contenedor o portal.

Punto A: integración de servidor web

Muchas empresas de alojamiento tienen ya un servidor web activo o disponen de un servidor web especialmente compilado que se ejecuta en sus sistemas. Mientras el servidor web ofrezca operaciones de CGI básicas, puede reemplazar el archivo binario de Apache o utilizar un servidor web existente para ofrecer acceso a Urchin.

Este punto de integración mantiene el resto del sistema Urchin intacto. La interfaz de Urchin se utiliza para procesos de administración, autenticación y publicación de informes. Los usuarios y los perfiles deben configurarse dentro del sistema Urchin de manera que cuando un usuario acceda al sistema, Urchin sepa a qué informes puede permitirle el acceso.

Punto B: autenticación externa u omisión de autenticación

Los entornos de alojamiento complejos suelen tener controles centralizados para ofrecer procesos de autenticación de usuarios como LDAP. Con un sencillo cambio de configuración, el controlador de sesión puede invocar a la rutina de autenticación que usted elija. Se incluye una interfaz sencilla para devolver accesos correctos o fallidos. Las rutinas de autenticación existentes pueden ajustarse fácilmente para ofrecer el marco correcto para ambos sistemas. Si se utiliza la integración en el punto "B", también es posible omitir la autenticación con una rutina de prueba y enlazar directamente con la página de destino.

Al utilizar la integración en el punto "B", el sistema Urchin permanece intacto, con la excepción de que la autenticación se realiza mediante una aplicación externa. Los usuarios y los perfiles deben configurarse dentro del sistema Urchin de manera que cuando un usuario acceda al sistema, Urchin sabrá a qué informes puede permitirles acceder. Para obtener los datos completos sobre la interfaz para utilizar la autenticación externa y un ejemplo de cómo omitir el proceso de autenticación, consulte Utilización de la autenticación externa u omisión de la autenticación.

Punto C: enlace directo a informes desde un contenedor o portal

Muchos de nuestros clientes mantienen una relación directa entre cliente y sitio web y desean enlazar a la generación de informes para un sitio web concreto directamente desde la zona de administración del sitio web. Esta zona suele estar ya autenticada. La integración del punto C facilita el enlace directo a la generación de informes a través de una secuencia de comandos de contenedor o desde un portal existente. En este caso, la autenticación de Urchin y la pantalla de selección de informes inicial se omiten, ya que los usuarios acceden directamente a los informes de Urchin.

Al utilizar la integración en el punto "C", se da por hecho que el proveedor de servicios tiene control sobre las personas que ven los distintos elementos, ya sea en una relación directa o a través de una base de datos de configuración existente. Es necesario configurar Urchin para generar informes Perfil, pero es posible automatizar el proceso dentro o fuera de la interfaz de administración. Urchin ofrece la posibilidad de enlazar directamente con un informe de Urchin con una URL concreta cuando la opción "Enlaces directos a informes" está habilitada en la interfaz de administración. El acceso al informe de Urchin se controla a través de un archivo ".report.conf" incrustado en el directorio que especifica esta URL.

Para realizar una integración de portal compleja, el motor de generación de informes de Urchin propagará la sesión y otras variables del portal para mantener la sesión operativa.
