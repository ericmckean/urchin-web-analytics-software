Filtrado global de resultados desde el software de supervisión

Visión general

La mayor parte de los entornos de alojamiento ofrecen algún tipo de supervisión de servidores web para mantener Acuerdos de nivel de servicio (SLA). Sin embargo, como efecto secundario, los resultados de esta supervisión pueden distorsionar los informes de Urchin para los sitios web supervisados, inflando de forma artificial el recuento de sesiones, visitas a páginas, resultados y bytes.

Recomendación

En entornos de alojamiento que utilizan esta supervisión, se recomienda la aplicación de un filtro de Urchin estándar/global a todos los perfiles de Urchin configurados para cada perfil para quitar los resultados generados por el software de supervisión. Esto se puede realizar fácilmente en un entorno en el que una instalación de Urchin centralizada (administrada por la compañía de alojamiento) ofrece informes para todos los sitios web de los clientes. En entornos dedicados/de colocación en los que los propios clientes mantienen una instancia de Urchin en sus servidores, la compañía de alojamiento debe ofrecer un filtro de muestra adecuado para la supervisión que se está utilizando.

Para ayudar a la implementación del filtrado de Urchin, el host y el cliente deben colaborar para crear una página específica en el sitio web de este último que solo utilice el software de supervisión, por ejemplo: 

     http://www.customerdomain.com/healthpage.html

Ejemplos

Ejemplo 1: Filtrado de la dirección IP para el sistema de supervisión 

      Filter Type: Exclude
      Filter Field: IP
      Filter Spec: 172\.16\.1\.1
  • Esto elimina cualquier resultado con la dirección IP 172.16.1.1 del registro del servidor web a medida que Urchin lo procesa.

Ejemplo 2: Filtrado de una página específica que coincida con el sistema de supervisión 

      Filter Type: Exclude
      Filter Field: REQUEST
      Filter Spec: ^/healthpage.html
  • Esto elimina cualquier resultado con una petición de /healthpage.html del registro del servidor web a medida que Urchin lo procesa.

Nota importante

  1. También puede considerar crear perfiles adicionales no filtrados para clientes, de forma que puedan ver la carga de tráfico real (incluido el filtrado) de los servidores web.
  2. La empresa de alojamiento puede ofrecerle un perfil que proporcione informes exclusivamente para los resultados de supervisión, por ejemplo, filtra solo en los resultados del software de supervisión. Este perfil puede utilizarse para mostrar que se está realizando la supervisión adecuada y que se están siguiendo los SLA.
