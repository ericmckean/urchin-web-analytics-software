Plataformas compatibles con Urchin y requisitos de hardware

Urchin funciona en varias arquitecturas y sistemas operativos. La instalación de Urchin solo es necesaria en los sistemas que procesen registros. Para ver informes, únicamente se necesita un navegador web.

Plataformas compatibles

Windows

  • Windows 2003 Server
  • Windows XP
  • Windows 2000
  • Windows Vista
  • Windows 2008 Server

UNIX-type Systems

  • Linux (kernel 2.6): es genérico para la mayor parte de las variantes utilizadas
  • Linux (kernel 2.4): es genérico para la mayor parte de las variantes utilizadas
  • FreeBSD 4 (no es compatible con 6.5+)
  • FreeBSD 5.3+ (no es compatible con Urchin 6.602+)
  • FreeBSD 6.2+

Supported databases

  • MySQL 4.1+ (no es compatible con Urchin 6.6+)
  • MySQL 5.03+
  • PostgreSQL 8.2.5

Navegadores admitidos

  • Internet Explorer 6.x, 7.x
  • Firefox 2.x, 3.x
  • Chrome 1.x, 2.x

Nota sobre plataformas que no se enumeran explícitamente como compatibles

Si no existe una versión de Urchin para la plataforma que utiliza, puede usar una versión de Urchin que sea compatible con su sistema operativo.

  • Windows 98, Windows 3.x:

Urchin no puede instalarse en Windows 98 ni en 3.x, pero estas plataformas pueden utilizarse para ver informes con Internet Explorer (5.x o posterior).

  • Linux:

Hay muchas variantes de Linux y no todas ellas son explícitamente compatibles con una determinada versión de Urchin. Sin embargo, existe un alto grado de compatibilidad entre las variantes de Linux, por lo que es probable que alguna versión de Urchin funcione en su equipo. Estas son algunas de las versiones compatibles que se conocen:

  • Ubuntu (kernel 2.6)

  • RedHat Enterprise Linux 2 & higher

  • SuSE Linux 8 & higher

  • Trustix 2 & higher

  • Solaris:

Actualmente, Solaris no es compatible.

Requisitos del sistema para Urchin 6

El rendimiento de Urchin permite obtener mejores resultados con una menor inversión en hardware. Por ejemplo, aunque un Pentium II antiguo pueda ser demasiado lento para realizar tareas de maquetación, puede funcionar bien como servidor de Urchin. Para obtener el máximo rendimiento, se recomienda utilizar el siguiente hardware:

Análisis de sitios web pequeños o medianos

  • procesador a 2 Ghz o superior,
  • RAM de 2 GB,
  • disco duro IDE de 250 GB o más,
  • interfaz Ethernet de 100 MB.

Proveedor de servicios/instalaciones empresariales

  • Pentium dual-core a 2,5 Ghz/UltraSPARC a 2 Ghz/PPC/MIPS/etc. con un intervalo de velocidad similar al de la CPU,
  • RAM de 3 GB,
  • disco duro Ultra2/Wide SCSI (p. ej., Seagate Cheetah),
  • Ethernet 100base-T,
  • sistema de copia de seguridad.

Utilización de la memoria/sistema/disco

  • La memoria (RAM) de Urchin puede configurarse para que se utilice entre los 20 y los 500 MB (20 y 1.024 MB para Urchin 6.5).
  • Urchin puede configurarse para que funcione con una prioridad baja, normal o alta.
  • El almacenamiento de datos de Urchin utiliza aproximadamente el 10% del tamaño de los registros sin procesar.

Nota importante para usuarios de FreeBSD

(1) FreeBSD cuenta con un proceso de codificación de forma rígida cuyo límite de tamaño de datos es de 500 MB. Este límite se supera debido a los datos geográficos de Urchin, que deben permanecer en la memoria durante el procesamiento de registros. Como consecuencia, recibirá el siguiente error de tiempo de ejecución: 

ERROR: (8011-323-1057) Failed to allocate memory

Para solucionarlo, aumente el tamaño de los datos del proceso predeterminado del sistema FreeBSD. Para ello, añada las líneas siguientes al archivo /boot/loader.conf: 

# Increase max process data segment size to 1GB
kern.maxdsiz="1073741824"

A continuación, reinicie el sistema.

ADVERTENCIA Tenga mucha precaución cuando cambie los límites del sistema en este sentido. Si aumenta demasiado el límite de tamaño de kern.maxdsiz, es posible que el sistema no pueda reiniciarse. Le recomendamos encarecidamente que consulte la documentación de FreeBSD antes de realizar tal modificación. Evalúe también los posibles riesgos ya que afectan a la configuración específica del sitio.

(2) FreeBSD incluye un controlador especial denominado accf_http. Se ocupa de almacenar las conexiones entrantes en el búfer hasta llegar a una cantidad de solicitudes de HTTP determinada para agilizar y optimizar el rendimiento. Este controlador no se carga en el kernel de forma predeterminada y aparece la siguiente advertencia: 

[warn] (2)No such file or directory: Failed to enable the
‘httpready’ Accept Filter

Para evitar este problema, debe habilitar “Aceptar filtro HTTP” del módulo kernel de FreeBSD (accf_http). Para ello, añada las líneas siguientes al archivo /boot/loader.conf: 

# Wait for full HTTP request accept filter
accf_http_load="YES"

A continuación, reinicie el sistema.

Versión de biblioteca de glibc de Linux

Para instalar el servidor PostgreSQL 8.4.01 que se suministra con Urchin 6.602 en Linux(kernel 2.4 o 2.6), se requiere glibc 2.3.3+.
