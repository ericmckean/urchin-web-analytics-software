Reprocesamiento de registros

Visión general

Algunas circunstancias pueden garantizarle que los datos de registro vuelvan a procesar como, entre otros, el hecho de que un servidor DNS no estuviera activo durante el procesamiento o filtros aplicados de forma incorrecta. En el documento siguiente se describe el procedimiento necesario para reprocesar los datos de registro del servidor web.

Tenga en cuenta que el reprocesamiento de los registros requiere el uso de utilidades de Urchin que solo están disponibles desde un entorno de shell de línea de comandos. No es posible realizar el procedimiento completo exclusivamente desde la GUI administrativa basada en la Web de Urchin.

Reprocesamiento de un solo día

  • En la GUI del administrador de Urchin, edite el perfil y desactive el seguimiento de registros de la pestaña Almacenamiento/base de datos. Asegúrese de hacer clic en Actualizar para guardar los cambios.
  • En la pestaña Fuentes del registro, asegúrese de que se especifican los archivos de registro que se van a volver a procesar. Los datos del registro solo deben contener resultados para las fechas para las que se ponen a cero las estadísticas.
  • Invoque al shell de comandos en el sistema Urchin y 

cd /<URCHIN_DIR>/util
  • Ejecute la utilidad udb-sanitizer del directory/folder 'util' de la distribución Urchin con el comando 

udb-sanitizer -p profile-name -d YYYYMM
  • donde YYYYMM** representan el año y el mes del día que desea volver a procesar

  • Seleccione la opción 5, Estadísticas a cero durante uno o más días. La utilidad le solicita que introduzca el día correcto y pone a cero las estadísticas de ese día en concreto. Para obtener más ejemplos acerca de cómo utilizar udb-sanitizer, consulte udb-sanitizer: Mantenimiento de bases de datos.

  • Haga clic en el botón Ejecutar ahora en la pestaña Ejecutar/Programar para que el perfil vuelva a procesar los datos del registro.
  • Restablezca la fuente del registro cambiando el valor de "Ruta del archivo de registro" a la configuración original.
  • En la pestaña Almacenamiento/Base de datos del área de edición de perfiles, vuelva a activar Seguimiento de registros.

Reprocesamiento de un mes completo:

El procedimiento para volver a procesar el valor de los datos de un mes entero es similar al de un solo día descrito anteriormente; simplemente tiene que utilizar opciones de udb-sanitizer diferentes. Consulte udb-sanitizer: Mantenimiento de bases de datos para obtener más ejemplos.
