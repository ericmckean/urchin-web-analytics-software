Guía de instalación de Urchin 6 (Windows)

Autor: EpikOne, asesor autorizado del software Urchin

Antes de empezar: nota importante para usuarios de Urchin 6.500

En Urchin 6.501 se corrigieron varios errores del programador de CPC. Si se dispone a actualizar el programa de Urchin 6.500 a Urchin 6.602, los datos de CPC procesados pueden ser incorrectos o estar incompletos, lo que daría lugar a resultados erróneos en los informes de Urchin. Para obtener datos correctos de Google Adwords, le recomendamos que (a) instale Urchin 6.602 en otra ubicación o que (b) elimine los datos de CPC antiguos en caso de que quiera instalar el programa en la ubicación existente de 6.500. Si este último es su caso, elimine el contenido de los directorios siguientes: 

<path_to_urchin_6500_installation>\data\cpc\...
<path_to_urchin_6500_installation>\data\history\cpc\...

No es necesario que realice ninguna acción si actualiza desde las versiones Urchin 6.600 y 6.501 o desde Urchin 6.402 y versiones anteriores.

Visión general de los pasos de la instalación

  • Instale el servidor de bases de datos en el sistema Urchin o añada la base de datos de Urchin a un servidor existente. Estos son los motores de bases de datos compatibles actualmente:
    • MySQL 5.0.3 o versiones posteriores (Nota: a partir de Urchin 6.6, ya no se ofrece compatibilidad con MySQL 4.x.x),
    • PostgreSQL 8.2.5 o versiones posteriores.
  • Cree la base de datos y el usuario de Urchin en el servidor de base de datos y conceda permisos de acceso.
  • Descomprima Urchin en una carpeta temporal.
  • Ejecute el archivo MSI de instalación.
  • Registre la licencia para la instalación de Urchin.

Opción 1: utilización de la base de datos MySQL existente con Urchin

  • Si aún no dispone de un servidor MySQL en el sistema, puede obtener una copia en la dirección http://dev.mysql.com/downloads/. El paquete Windows Essentials contiene los componentes necesarios para usar Urchin.

  • Instale el servidor MySQL siguiendo las instrucciones del sitio web de MySQL.
  • Asegúrese de que sabe el nombre de usuario del administrador de la base de datos MySQL (por ejemplo, raíz) y la contraseña.
  • Elija el nombre de la base de datos, el nombre de usuario de MySQL local y la contraseña que utilizará con Urchin. Guarde esta información, ya que la necesitará durante la instalación de Urchin. Mediante el ejemplo siguiente se crea una base de datos denominada urchin, cuyo nombre de usuario de MySQL es urchin y la contraseña, urchinpassword; aunque, obviamente, sería recomendable que utilizara su propia contraseña.

  • Inicie el cliente de la línea de comandos MySQL a través de “Inicio” -> “Programas” -> “MySQL”-> “Servidor MySQL”-> “Cliente de la línea de comandos MySQL”.

  • En la ventana “Cliente de la línea de comandos MySQL”, cree la base de datos, el usuario y los permisos de acceso de Urchin mediante los siguientes comandos: 

mysql> create database urchin character set utf8;
mysql> GRANT ALL ON urchin.* to 'urchin'@'localhost' IDENTIFIED BY
'urchinpassword';

Opción 2: utilización de la base de datos PostgreSQL existente con Urchin

  • Si aún no dispone de un servidor PostgreSQL en el sistema, puede obtener una copia en la dirección http://www.postgresql.org.

  • Instale el servidor PostgreSQL siguiendo las instrucciones del sitio web de PostgreSQL.
  • Asegúrese de que sabe el nombre de usuario del administrador de la base de datos PostgreSQL (por ejemplo, postgres) y la contraseña.
  • Elija el nombre de la base de datos, el nombre de usuario de PostgreSQL local y la contraseña que utilizará con Urchin. Guarde esta información, ya que la necesitará durante la instalación de Urchin. Mediante el ejemplo siguiente se crea una base de datos denominada urchin, cuyo nombre de usuario de PostgreSQL es urchin y la contraseña, urchinpass; aunque, obviamente, sería recomendable que utilizara su propia contraseña.

  • Inicie el cliente de la línea de comandos PostgreSQL a través de “Inicio”-> “Programas” -> “PostgreSQL 8.2”-> “psql a 'postgres'”.

  • En la ventana “Cliente de la línea de comandos PostgreSQL”, cree la base de datos, el usuario y los permisos de acceso de Urchin mediante los siguientes comandos: 

postgres=# CREATE DATABASE urchin WITH ENCODING 'UTF8';
postgres=# CREATE USER urchin WITH PASSWORD 'urchinpass';
postgres=# GRANT ALL ON DATABASE urchin TO urchin;
postgres=# \connect urchin;
postgres=# CREATE LANGUAGE 'PLPGSQL';

Opción 3: utilización del servidor PostgreSQL integrado con Urchin

  • El programa de instalación de Urchin 6.602 se suministra con el servidor PostgreSQL integrado como opción durante la instalación de Urchin.

  • Para poder usar esta opción, el programa de instalación de Urchin debería ser invocado por un usuario con privilegios de administrador. Dicho programa de instalación le pedirá que indique el número de puerto y la contraseña del usuario "postgres". Nota: si el usuario "postgres" ya existe en el sistema, la contraseña que se indique debe ser la que ya tiene asociada. Si el usuario "postgres" no existe en el sistema, se creará con la contraseña que se facilite.

  • Las siguientes limitaciones afectan al servidor PostgreSQL integrado:
    • No debe instalarse en modo de actualización.
    • Se necesitan privilegios de administrador para la instalación.

Instalación de Urchin

  • Descomprima el archivo ZIP de instalación de Urchin en una carpeta temporal.

  • Si desea instalar el servidor PostgreSQL integrado, asegúrese de que no exista ya un servidor PostgreSQL en el sistema. Para llevar a cabo la instalación, seleccione la opción "Instalar PostgreSQL" y facilite la información que se le solicite sobre el servidor PostgreSQL, por ejemplo, el número de puerto o la contraseña del usuario "postgres". Nota: si tiene algún problema para instalar el servidor PostgreSQL integrado, consulte http://www.postgresql.org.

  • En caso de que quiera usar una base de datos existente, asegúrese de tener a mano el nombre de la base de datos de Urchin, el usuario y la contraseña de MySQL o de PostgreSQL que haya elegido anteriormente.
  • Urchin 6 solo admite una instalación de Urchin 6 por equipo (solo Windows). Si el equipo ya tiene instalada una versión de Urchin, solo se puede instalar una actualización. Tenga en cuenta que no es posible instalar una versión anterior del programa.
  • Si va a llevar a cabo una instalación 'nueva', primero debe eliminar del equipo todas las versiones de Urchin 6. Para eliminar Urchin, siga los pasos que se indican a continuación:

    • “Inicio” -> “Programas” -> “Urchin 6”-> “Inhabilitar servicios Urchin”.

    • “Inicio” > “Configuración” -> “Panel de control” -> “Agregar o quitar programas”.

    • Haga clic en "Software Urchin de Google" y elija “Quitar”.
    • Una vez que haya eliminado Urchin, cambie el nombre de la carpeta de instalación de urchin (p. ej., de C:\Program Files\Urchin6 a Urchin6-OLD).
  • Urchin 6.602 solo puede utilizarse como actualización de determinadas versiones anteriores de Urchin 6. Estas son las versiones que pueden actualizarse:
    • Urchin 6.6, Service Pack 1 : versión 6.601
    • Urchin 6.6, versión en inglés : versión 6.600
    • Urchin 6.5, Service Pack 2 : versión 6.501
    • Urchin 6.5 : versión 6.500 (véase la nota importante anterior)
    • Urchin 6, Service Pack 1 : versión 6.402
    • Urchin 6, versión internacional: versión 6.401
    • Urchin 6, versión en inglés : versión 6.400
  • Para llevar a cabo la actualización, no es necesario detener o desinstalar Urchin 6. El programa de instalación de Urchin 6.602 primero comprueba si hay alguna instalación de Urchin 6, en cuyo caso la desinstala. A continuación, instala los archivos y la configuración de Urchin 6.602.
  • Para iniciar la instalación de Urchin 6.602, haga doble clic en el archivo urchin6602_windows_installer.msi y siga las instrucciones interactivas que aparecen en pantalla.

  • Nota: si tiene instalada una versión anterior a Urchin 6 y desea conservar la configuración, asegúrese de que la opción "Volver a crear las tablas de bases de datos EXISTENTES específicas de Urchin (si existen)" esté desmarcada. En ese caso, Urchin mantiene intacta la configuración existente, pero puede crear 'tablas adicionales' o 'columnas en tablas que ya existen'.

  • Urchin debería funcionar correctamente. Para concluir el proceso, introduzca la siguiente URL en el navegador: 

http://yourserver:<port>

donde <port> es el puerto de la red que eligió durante la instalación (el puerto predeterminado es 9999).

  • Si desea restaurar los datos de informe de perfil de una versión anterior de Urchin 6, copie las carpetas history y reports de C:\Program Files\Urchin6-OLD\data a C:\Program Files\Urchin6.
  • Haga clic en el enlace “Obtener licencia para la versión de prueba” y siga los pasos para habilitar Urchin y ajustar la configuración.

Configuración de la función de seguimiento

Una vez que haya instalado Urchin, según el uso que vaya a darle a la aplicación, deberá llevar a cabo ciertos pasos para configurar la función de seguimiento en su sitio web.

Caso 1: utilización de Urchin con seguimiento de IP+User-Agent

No es necesario ajustar ninguna otra configuración. Cuando añada perfiles en Urchin, asegúrese de especificar IP+User-Agent como el método de seguimiento de usuarios.

Caso 2: utilización de Urchin con seguimiento de UTM

Para utilizar Urchin con el seguimiento de UTM, es necesario instalar un pequeño fragmento de código de seguimiento javascript en cada una de las páginas del sitio web. Para ello, siga los pasos que se indican a continuación:

  • Del directorio util\utm de la instalación de Urchin, copie los archivos urchin.js. y __utm.gif en la raíz de documentos de su sitio web (por ejemplo, en el directorio de nivel superior del contenido).

  • Inserte el siguiente código de seguimiento en todas las páginas del sitio web, concretamente en la sección HEAD y justo después de cualquier etiqueta META: 

<script src="/urchin.js" type="text/javascript">
</script>
<script type="text/javascript">
  _userv=0;
  urchinTracker();
</script>
  • Cuando añada perfiles en Urchin, asegúrese de especificar el Supervisor de tráfico de Urchin (UTM) como el método de seguimiento de usuarios.

Caso 3: utilización de Urchin junto con Google Analytics

Recuerde que las nuevas funciones de seguimiento que incorpora el código de seguimiento "ga.js" de Google Analytics no son compatibles con Urchin 6. Si desea supervisar su sitio web tanto con Google Analytics como con el software Urchin, le recomendamos que utilice el código de seguimiento basado en urchin.js para obtener unos resultados óptimos. Si opta por utilizar ga.js, no podrá beneficiarse de las nuevas funciones de seguimiento de ga.js.

Para utilizar Urchin en un sitio web que ya se supervisa con Google Analytics, solo es necesario realizar algunas modificaciones en el código de seguimiento que emplea Google Analytics. Para ello, siga los pasos que se indican a continuación:

  • Del directorio util\utm de la instalación de Urchin, copie el archivo __utm.gif en la raíz de documentos de su sitio web (por ejemplo, en el directorio de nivel superior del contenido).

  • Añada la línea _userv=2; al código de seguimiento existente en las páginas de su sitio web, tal y como se indica a continuación. No modifique ningún otro fragmento del código existente. 

<script src="http://www.google-analytics.com/urchin.js"
type="text/javascript">
<script type="text/javascript">
  _userv=2;
  _uacct="UA-XXXXXXX-Y";
  urchinTracker();
</script>
  • Cuando añada perfiles en Urchin, asegúrese de especificar el Supervisor de tráfico de Urchin (UTM) como el método de seguimiento de usuarios.

Pasos adicionales de instalación para Windows Vista y para Server 2008

A parte de las versiones admitidas de Windows (2000, XP y Server 2003), Urchin 6.6+ también es compatible con Windows Vista y con Server 2008. Estas recientes versiones de Windows incorporan ciertas funciones de seguridad que pueden afectar a la ejecución de Urchin. Lea a continuación las explicaciones al respecto.

Problema con el UAC o Control de cuentas de usuario

Windows 2008 incluye una función de seguridad nueva denominada User Access Control (UAC). Si desea obtener más información al respecto, visite http://technet.microsoft.com/en-us/library/cc709691.aspx.

La activación de la función UAC comporta estos problemas conocidos:

  1. . Si se habilita UAC, la aplicación no puede escribir correctamente en la carpeta Program files. Más información en la página: http://msdn.microsoft.com/en-us/library/aa905330.aspx.

  2. . Los archivos ejecutables de Urchin no pueden ejecutarse desde un símbolo del sistema predeterminado. Debe utilizar el símbolo del sistema como administrador o bien establecer las propiedades de dicho archivo tal y como se indica a continuación: haga clic con el botón derecho del ratón y seleccione “Propiedades”-> “Compatibilidad”-> “Ejecutar este programa como administrador” -> “Aceptar”. Tenga en cuenta que los servicios de Urchin para Windows operan tal y como se espera sin ningún esfuerzo adicional necesario.

UAC: solución

  1. Desactive la función UAC e instale Urchin en la ubicación que desee. Ejecute Urchin con los servicios de Windows o con los archivos binarios.
  2. Si la función UAC está habilitada, NO instale Urchin en la carpeta Program files. Instálelo, por ejemplo, en el directorio Users. Cambie las propiedades para ejecutar archivos binarios en modo de administrador (tal y como hemos descrito anteriormente en el apartado 2) o utilice los servicios de Windows sin ningún cambio.
