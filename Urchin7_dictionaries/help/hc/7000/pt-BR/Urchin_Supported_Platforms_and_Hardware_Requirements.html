Plataformas suportadas e requisitos de hardware do Urchin

O Urchin é executado em diversas arquiteturas e sistemas operacionais. A instalação do Urchin necessita apenas de um sistema que processará os registros. Para visualizar relatórios, apenas um navegador da web é necessário.

Plataformas suportadas

Windows

  • Windows 2003 Server
  • Windows XP
  • Windows 2000
  • Windows Vista
  • Windows 2008 Server

UNIX-type Systems

  • Linux (kernel 2.6) ― genérico para a maioria dos tipos usados
  • Linux (kernel 2.4) ― genérico para a maioria dos tipos usados
  • FreeBSD 4 (sem suporte para 6.5+)
  • FreeBSD 5.3+ (sem suporte para Urchin 6.602+)
  • FreeBSD 6.2+

Supported databases

  • MySQL 4.1+ (sem suporte para Urchin 6.6+)
  • MySQL 5.03+
  • PostgreSQL 8.2.5

Navegadores suportados

  • Internet Explorer 6.x, 7.x
  • Firefox 2.x, 3.x
  • Chrome 1.x, 2.x

Observação sobre plataforma suportadas não explícitas

Se não houver distribuição do Urchin para sua plataforma, será possível usar uma distribuição do Urchin que seja compatível com seu OS.

  • Windows 98, Windows 3.x:

O Urchin não pode ser instalado em Windows 98 or 3.x, mas essas plataforma podem ser usadas para visualizar relatórios com o Internet Explorer (a partir da versão 5.x).

  • Linux:

Há muitos tipos diferentes de Linux, nem todos são explicitamente suportados por uma distribuição do Urchin. No entanto, normalmente há um alto grau de compatibilidade entre o tipos de Linux de modo que, provavelmente, uma das distribuições do Urchin funcionará em sua máquina. Veja algumas das distribuições compatíveis conhecidas:

  • Ubuntu (kernel 2.6)

  • RedHat Enterprise Linux 2 & higher

  • SuSE Linux 8 & higher

  • Trustix 2 & higher

  • Solaris:

Atualmente o Solaris não é suportado.

Requisitos do sistema do Urchin 6

O desempenho do Urchin's permite a você obter mais de menos investimento em hardware. Mesmo um Pentium II mais antigo, por exemplo, pode ser muito lento para usar na área de trabalho, mas poderá usar um servidor Urchin perfeitamente. Para um melhor desempenho, o seguinte hardware é recomendado:

Análise de website simples, de pequeno a médio

  • Processador 2 Ghz ou melhor
  • 2 GB de RAM
  • 250 GB+ disco rígido IDE
  • Interface ethernet de 100 MB

Provedor de serviços/Instalações empresariais

  • Pentium dual-core 2,5 Ghz/ UltraSPARC de 2 Ghz/CPU de velocidade similar PPC/MIPS/etc.
  • 3GB RAM
  • Disco rígido Ultra2/Wide SCSI (por exemplo, um Seagate Cheetah)
  • 100base-T ethernet
  • Sistema de backup

Memória/Sistema/Uso de disco

  • O uso de memória (RAM) do Urchin pode ser configurado para usar entre 20-500 MB (20-1024 MB para Urchin 6.5)
  • O Urchin pode ser configurado para ser executado como prioridade alta, normal ou baixa
  • O armazenamento de dados do Urchin usará aproximadamente 10% do tamanho dos registros brutos

Observação importante para Usuários FreeBSD

(1) O FreeBSD tem tamanho de dados de processo codificado de 500 MB. Este limite é excedido por causa dos dados geográficos do Urchin's que precisam estar na residência residente durante o processamento de registro. Por isso, você verá o seguinte erro do tempo de execução: 

ERROR: (8011-323-1057) Failed to allocate memory

Para identificar isso, será necessário aumentar o tamanho de dados processo padrão do sistema FreeBSD, o que você fazer editando o arquivo /boot/loader.conf e adicionando estas linhas: 

# Increase max process data segment size to 1GB
kern.maxdsiz="1073741824"

Depois disso, será necessário reiniciar a máquina.

CUIDADO. Seja muito cuidadoso ao alterar limites de sistema como esse. Se você definir kern.maxdsiz como muito grande, seu sistema pode não se tornar passível de carregamento. Leia a documentação do FreeBSD antes de fazer esta modificação e acesse os riscos potenciais à medida que você aplica à configuração específica de site.

(2) O FreeBSD tem um acionador chamado accf_http. Ele guardará as conexões recebidas até que algumas solicitações HTTP cheguem para aumentar a velocidade e otimize o desempenho. Por padrão, esse acionador não é carregado no kernel, e você verá o seguinte aviso: 

[warn] (2)No such file or directory: Failed to enable the
‘httpready’ Accept Filter

Para evitar isso, será necessário ativar o módulo do kernel do FreeBSD Aceitar filtro de HTTP (accf_http), o que você fazer editando o arquivo /boot/loader.conf e adicionando estas linhas: 

# Wait for full HTTP request accept filter
accf_http_load="YES"

Depois disso, será necessário reiniciar a máquina.

Versão da biblioteca do Linux glibc

Para instalar o PostgreSQL 8.4.01 acompanhado com Urchin 6.602 em Linux(kernel 2.4 ou 2.6), glibc 2.3.3+ é obrigatório.
