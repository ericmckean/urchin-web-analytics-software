Como usar um servidor da web IIS existente (plataformas Windows)

Por padrão, a administração e os relatórios do Urchin são realizados com um servidor Apache autônomo que é fornecido com o Urchin. Na grande maioria das instalações do Urchin, esse é o método preferido para exibir as interfaces de administração e relatórios do Urchin. No entanto, em situações raras, talvez seja necessário utilizar um servidor da web IIS existente. Isso pode ocorrer devido aos requisitos do site que não permitem o uso de um servidor da web de terceiros no servidor ou devido à necessidade de configurar os relatórios do Urchin como um host virtual em um servidor IIS existente.

EXONERAÇÃO DE RESPONSABILIDADE: essas modificações na instalação do Urchin não são suportadas.

Procedimento

Observação: este procedimento supõe que o Urchin foi instalado no local padrão C:\Arquivos de programas\Urchin. Se você tiver instalado o Urchin em algum outro lugar, substitua pelo local adequado no exemplo abaixo.

Etapa 1: crie um novo usuário para a interface da web do Urchin

  • Vá até "Ferramentas administrativas" -> "Gerenciamento do computador"

  • Na lateral esquerda da tela “Gerenciamento do computador”, clique em "Usuários e grupos locais"
  • Clique com o botão direito do mouse na pasta "Usuários" e selecione "Novo usuário..."
  • Insira "IUSR_URCHIN" no campo "Nome de usuário:"
  • Desmarque a caixa "O usuário deve alterar a senha no próximo login"
  • Marque a caixa "O usuário não pode alterar a senha"
  • Clique em "Criar" e em "Fechar"
  • Clique duas vezes na pasta "Usuários" à esquerda
  • Clique com o botão direito do mouse em IUSR_URCHIN, à direita, e selecione "Propriedades..."
  • Na guia "Membro de", remova todas as entradas existentes, clique no botão "Adicionar..", escolha "Convidados" na janela pop-up, clique em "Adicionar" novamente e, em seguida, clique em "OK"
  • Clique em "Aplicar" e "Fechar" para salvar suas alterações

Etapa 2: instale o Urchin (se ainda não tiver feito isso)

Etapa 3: desative o servidor da web Apache do Urchin

  1. Vá até "Ferramentas administrativas" -> "Serviços"

  2. Em "Serviços", localize o registro "Servidor da web do Urchin"
  3. Clique com o botão direito do mouse em "Servidor da web do Urchin" e selecione "Parar"
  4. Clique com o botão direito do mouse em "Servidor da web do Urchin" e selecione "Propriedades"; em seguida, mude “Tipo de inicialização" para "Desativado"
  5. Clique em "OK"

Etapa 4: adicione um novo website ao IIS. Vá até "Ferramentas administrativas" -> "IIS Manager"

  1. Clique com o botão direito do mouse em "Sites"
  2. No campo "Descrição:", digite "Urchin" e clique em "Avançar"
  3. Selecione o endereço IP e o número da porta (normalmente 9999) e clique em “Avançar”
  4. No campo "Caminho:", navegue até o local onde o Urchin está instalado (normalmente C:\Arquivos de programas\Urchin\htdocs) e clique em “Avançar”
  5. Adicione uma marca de verificação em "Executar:" e clique em "Avançar" e em "Concluir"
  6. Clique com o botão direito do mouse no novo website do Urchin e vá até "Propriedades"
    1. Na guia "Site", desmarque "Ativar registro"
    2. Clique na guia "Diretório inicial" e marque "Acesso à origem do script"
    3. Clique na guia "Documentos" e remova as duas entradas padrão; em seguida, clique em “Adicionar” e insira "session.cgi" na janela pop-up e clique em “OK”.
    4. Clique na guia "Segurança do diretório" e clique em “Editar” na área "Controle de autenticação e acesso". Verifique se a caixa "Acesso anônimo" está selecionada e clique em “Editar...” para alterar a "Conta usada para acesso anônimo". Na janela pop-up, selecione "IUSR_URCHIN" para o “Nome de usuário”. Clique em “OK” e em “OK” novamente para voltar à janela “Propriedades”.
    5. Clique em “OK” para salvar as alterações e sair da janela “Propriedades”

Etapa 1: configure as permissões do diretório 1. Clique com o botão direito do mouse em "Iniciar" e selecione "Explorar"

  1. Navegue até o local onde o Urchin está instalado (normalmente C:\Arquivos de programas\Urchin)
  2. Clique com o botão direito do mouse na pasta "Urchin" e selecione "Propriedades"
  3. Clique na guia "Segurança"
  4. Clique no botão "Avançado". Desmarque "Permitir que permissões herdadas do pai se propaguem para este objeto" e clique em "Remover" na janela pop-up.
  5. Clique em "Adicionar", selecione o usuário “Administrador” e clique em "Adicionar"
  6. Clique em "Adicionar", selecione "IUSR_URCHIN" e clique em "Adicionar"
  7. No campo "Nome:", verifique se aparecem apenas as entradas “Administrador” e “IUSR_URCHIN”

  8. Conceda permissões de gravação para a pasta <urchin_home>\var

  9. Volte a “Preferências” => “Segurança” e verifique se as seguintes permissões estão autorizadas para o usuário IUSR_URCHIN:

    • Ler e executar
    • Conteúdo da pasta List
    • Lidas
    • Observação: deixe as permissões completas padrão para SYSTEM
  10. Clique em “OK” para salvar as permissões.
  11. Clique na pasta "Urchin" na janela “Windows Explorer”.
  12. Clique com o botão direito do mouse na pasta "data" e selecione "Propriedades"
  13. Clique na guia "Segurança".
  14. Clique em “Avançado”. Desmarque "Permitir que permissões herdadas do pai se propaguem para este objeto" e clique em "Remover" na janela pop-up.
  15. Clique em "Adicionar", selecione o usuário “Administrador” e clique em "Adicionar"
  16. Clique em "Adicionar", selecione "IUSR_URCHIN" e clique em "Adicionar"
  17. No campo "Nome:", verifique se aparecem apenas as entradas “Administrador” e “IUSR_URCHIN”

  18. Volte a “Preferências” => “Segurança” e verifique se as seguintes permissões são autorizadas para os usuários “Administrador” e IUSR_URCHIN:

    • Controle total
    • Modificar
    • Ler e executar
    • Conteúdo da pasta List
    • Lidas
    • Gravar
  19. Clique em "OK"

Etapa 6: somente IIS 6. Vá até o "Gerenciador de extensões de serviço da web"

  1. Clique em “Adicionar nova extensão de serviço da web…”
  2. Insira "Urchin CGI" em "Nome da extensão:"
  3. Clique em “Adicionar”
  4. Navegue e realce report.cgi e session.cgi (por padrão, somente os arquivos *.dll são mostrados de modo que não é necessário alterar o filtro para exibir "Todos os arquivos")
    • Local padrão: C:\Arquivos de programa\Urchin\htdocs
  5. Marque "Definir o status da extensão a ser permitido"
  6. Clique em "OK"
  7. Vá até a entrada IIS principal de seu servidor: "nome do host"
  8. Clique com o botão direito do mouse e selecione "Propriedades"
  9. Clique em "Tipos Mime"
  10. Clique em "Novo"
  11. Insira ".cgi" no campo "Extensão:".
  12. Insira "application/octet-stream" no campo "Tipo Mime".
  13. Clique em "OK"
  14. Clique em "OK"

Seu servidor da web IIS agora deve estar configurado para chamar a interface da web do Urchin se você se conectar usando o URL http://my.server.com:port, onde “port” é a porta especificada para ser usada pelo Urchin durante sua instalação (o padrão é 9999).

Como usar um servidor da web IIS existente para os serviços da web da API de dados do Urchin (Urchin 6.6+)

Os serviços da web da API de dados do Urchin (Urchin 6.6+) são fornecidos com um módulo IIS que pode ser implantado e configurado para encaminhar solicitações para a API de dados do Urchin. Para configurar a API de dados no servidor da web IIS, consulte Posso integrar a API de dados do Urchin no IIS?.
