geo-update: utilitário de atualização do banco de dados DNS

Visão geral

O utilitário geo-update é usado para verificar as atualizações do banco de dados DNS interno do Urchin e para fazer download das atualizações se estiverem disponíveis. Ele permite fazer download das atualizações dos servidores de download do Google, bem como de um diretório local (novo para o Urchin 6).

Utilização

O utilitário geo-update pode ser invocado pelo agendador do Urchin (urchind) mensalmente ou manualmente através da linha de comando. Quando invocado por meio do agendador, ele sempre é atualizado a partir do servidor de download do Google. Quando invocado na linha de comando, ele oferece uma opção adicional de atualização do banco de dados DNS a partir de algum caminho local. Essa tarefa é configurada para verificar novas atualizações uma vez por mês e pode ser configurada pelo usuário para ocorrer em um determinado período usando a interface admin.

Se uma tarefa tiver sido criada para atualização geográfica, ela aparecerá na tela "Histórico da tarefa" com o nome "atualização geográfica".

O comportamento padrão é verificar as atualizações, fazer download dos novos bancos de dados e substituir os existentes. Os comportamentos padrão podem ser modificados pela execução da atualização geográfica na linha de comando com as opções apropriadas. O uso funciona da seguinte maneira: 

geo-update [-Hhv] [-D | -F] [-i arquivo]

em que 

  -F            Força o download dos bancos de dados geográficos
  -f            Nome do caminho do arquivo geo.zip no sistema de arquivos local [opcional]
  -H            Especifica para registrar a saída da execução no arquivo histórico
  -h            Imprime informações de ajuda e é encerrado
  -T <task id>  Especifica o registro do histórico da tarefa a ser atualizado
                O -T da especificação força o uso do -H
  -V            Imprime o número da versão do banco de dados geográfico e é encerrado
  -v            Imprime o número da versão e é encerrado

O uso da opção -F forçará o download dos novos bancos de dados, mesmo que eles já estejam atualizados. Esse recurso será útil se você tiver importado as informações de domínio personalizadas incorretas ou se tiver danificado de outra forma o seu banco de dados de domínio e desejar iniciar com uma nova cópia.

O registro da saída para um arquivo histórico com a opção -H tornará as informações da execução disponíveis ao exibir a tela “Histórico de tarefas”, na área “Configuração->Área do agendador da interface de administração do Urchin”.

O uso da opção -T permite que você reforce a atualização de uma determinada tarefa no CLI especificado pelo <Task_ID>. O valor Task_ID pode ser encontrado no banco de dados do Urchin 6 usando a seguinte consulta: 

mysql> select * from ugeo_tasks\G

Exemplos

Para forçar o download do banco de dados de domínios mais recente: 

geo-update -F

Para atualizar a tarefa manualmente no CLI (quando o ID da tarefa já tiver sido recuperado do banco de dados): 

geo-update -T <Task_ID> -H

O utilitário geo-update precisa de uma conexão com a internet para poder verificar e fazer download de novas atualizações. O utilitário usa a porta 80 para se comunicar com o servidor da web que fornece as atualizações. É possível que você tenha problemas ao utilizar firewalls e servidores proxy ao fazer as atualizações. Se esse for o caso, consulte o seu administrador de rede.
