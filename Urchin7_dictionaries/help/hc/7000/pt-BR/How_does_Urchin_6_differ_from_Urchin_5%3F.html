Como o Urchin 6 difere do Urchin 5?

Este artigo descreve as alterações entre o Urchin 5 e o Urchin 6.

Principais recursos

  • Perfis ilimitados, origens de registro e monitoramento de campanhas, tudo isso incluso com a licença base, sem módulos complementares.
  • Recursos abrangentes de monitoramento de campanhas SEO/SEM, 4 metas por perfil.
  • Elaborada segmentação cruzada disponível a partir da maioria dos relatórios.
  • Conjunto completo de relatórios de localização geográfica dos visitantes (e não apenas de domínios dos visitantes).
  • Velocidade de processamento a par com o Urchin 5, mas com dados muito mais sofisticados.

Suporte de plataformas

  • Ampla variedade de plataformas Linux suportadas com apenas 2 compilações (Linux 2.4 e 2.6 kernels).
  • Suporte adicionado para FreeBSD 5 e FreeBSD 6.
  • Suporte removido para MacOS X e Solaris.

Instaladores

  • O instalador do Windows é agora distribuído como um pacote MSI, resultando em melhor suporte para instalação autônoma e integração com o SMS.

Configuração

  • Back-end de banco de dados relacional (MySQL ou PostgreSQL).
  • Suporte para banco de dados de configuração hospedado no servidor de configuração remoto.

Servidor da web

  • Atualizado para a versão mais recente do Apache 1.3.X.
  • OpenSSL e mod-ssl atualizados para as versões mais recentes.
  • Módulos padrão removidos não usados pelo Urchin.
  • mod_expires adicionados para os cabeçalhos corretos de controle de cache.

Agendador de tarefas

  • O agendador é agora executado como dois processos - agendador principal e agendador secundário.
  • Tarefas facilmente gerenciadas por meio da interface de scripts com o banco de dados de configuração back-end.

Monitoramento de visitantes

  • O javascript de monitoramento __utm.js do Urchin 5 foi substituído por um javascript de monitoramento urchin.js compatível com o Google Analytics.Se você estiver monitorando o seu site com o uso de __utm.js, precisará alterar as tags do site de forma a usar urchin.js.

Processamento de registros

  • Dados geográficos armazenados na memória (maior área de ocupação da memória em tempo de execução).
  • O recurso de Intervalo de dias em origens de registro permite a pesquisa em vários dias de arquivos de registro que correspondam a um padrão de data particular.
  • Capacidade de executar perfis totalmente na memória.

Dados e armazenamento

  • Agora, os bancos de dados de perfil assumem como padrão 100 mil registros/mês (em vez de 10 mil), com a opção de aumento para 500 mil registros por mês.

  • Dados geográficos expandidos:conjunto completo de dados de localização geográfica do Quova, substituindo os dados somente domínio do MaxMind.

  • Limite mensal de registros na tabela aumentado do padrão de 10 mil para 100 mil registros.
  • 50 arquivos mensais por perfil, agora organizados por subdiretório.

UI de geração de relatórios

  • O Flash substitui o Adobe SVG para o processamento de gráficos e tabelas.
  • Exportação de relatórios somente em CSV e XML (a exportação não segura em MS Word/Excel foi removida).
  • Relatório Todos os perfis extinto.

Comércio eletrônico

  • Transações de comércio eletrônico podem ser gravadas diretamente em registros do servidor da web por meio de funções especiais no javascript de monitoramento (idêntico ao GA).
  • Também há suporte para registros externos de carrinho de compras no formato ELF2.

Segurança

  • Por meio de testes para detectar vulnerabilidades de script entre sites (XSS) e XSRF.
