Por que estou recebendo "httpd:não foi possível definir a opção de soquete TCP_NODELAY" no meu registro de erros?

Visão geral

Essa mensagem quase sempre indica que o cliente se desconectou antes que o servidor da web Apache do Urchin (urchinwebd) atingisse o ponto no qual ele define certas opções na conexão de rede (soquete) entre o navegador e o servidor Apache.

Solução

Essa mensagem é meramente informativa.Entretanto, convém entrar em contato com o administrador da rede para determinar o que pode estar causando o encerramento prematuro das conexões com a rede.Algumas causas possíveis são:

  • um robô/spider da web incorretamente configurado ou um agente de monitoramento que não encerra apropriadamente a conexão da rede com o servidor Apache do Urchin.
  • uma rede ou firewall incorretamente configurado que está resultando no descarte indevido de uma parte do tráfego de pacotes.
  • o resultado de uma verificação nas portas da rede para detectar vulnerabilidades.
  • um ataque de negação de serviço contra o servidor da web Apache do Urchin.
