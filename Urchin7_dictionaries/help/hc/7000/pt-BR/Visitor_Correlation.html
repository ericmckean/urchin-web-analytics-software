Correlação de visitantes

Visão geral

A correlação de visitantes é o processo de se identificar o comportamento dos visitantes, mesmo que as sessões sejam provenientes de diferentes arquivos de registro, de servidores da web ou de E-commerce independentes. O Urchin usa os dados das várias origens de registro para analisar as relações entre sessões e transações e, então, correlaciona essas informações com o objetivo de proporcionar uma imagem clara de como a atividade dos visitantes está relacionada a atividades de compras, possibilitando, assim, valiosos relatórios de retorno do investimento.Por exemplo, as referências de mecanismos de pesquisa e palavras-chave específicas podem ser correlacionadas com a quantidade adquirida no seu site de E-commerce, para informar a você quais referências e termos de pesquisa estão gerando as maiores receitas.Em geral, haverá um registro para o seu site público e outro registro a partir do seu site de transações seguras.Para correlacionar origens de dados desiguais, o Urchin deve usar o mesmo método de identificação de visitantes para cada site.

Tipos de correlação de visitantes

O Urchin é capaz de correlacionar visitantes com base em vários métodos.A sua opção de método de acompanhamento de visitantes afetará diretamente a precisão das informações que o Urchin tem à disposição, para correlacionar as transações de E-commerce com outras atividades no site da web.Escolha o método mais adequado para o nível de detalhes que você deseja obter.

Os dados a seguir são necessários no arquivo de registro de E-commerce para cada um dos métodos de visitantes:

  • UTM: cookies
  • IP+User-Agent: host remoto ou endereço IP e user-agent (por exemplo, tipo de navegador)

Esses itens precisam ser considerados no momento de examinar o seu formato de registro de E-commerce.Revise Formato de registro ELF2 e Registros de E-commerce personalizados para obter detalhes sobre como garantir que os dados apropriados estejam nos seus registros.

Configuração

Esta seção apresenta uma visão geral dos problemas de configuração com a correlação de visitantes de e-commerce.

Sites não-UTM

Se você não tiver o sensor UTM instalado nos seus sites, a sua configuração de correlação de visitantes dependerá do formato de e-commerce que está sendo usado.

  • ELF2: use IP+User-Agent como o método de monitoramento de visitantes.Nesse caso, o Urchin irá correlacionar automaticamente as várias sessões, desde que todos os registros contenham os campos de IP e User-Agent.Basta escolher "IP+User-Agent" para o "Método de monitoramento de visitantes" na tela "Relatórios" do Perfil.

Sites ativados para o UTM

Para sites ativados para o UTM, a mesma versão do sensor UTM deve ser instalada em todas as páginas que você deseja monitorar.Na tela "Configurações do perfil", defina "Método de monitoramento de visitantes" como "Monitor de tráfego do Urchin".Em condições normais, deixe "Domínio do UTM" em branco.O UTM irá detectar automaticamente o domínio do site.O recurso "Domínio do UTM" é usado somente quando você possui domínios sobrepostos que precisam ser monitorados separadamente.
