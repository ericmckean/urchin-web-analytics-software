Os links para o meu site já contêm minhas próprias variáveis de monitoramento de campanhas.Posso utilizar essas variáveis em vez das do Urchin?

Se você já codificou seus links on-line com as suas próprias variáveis de monitoramento, será possível mapear essas variáveis existentes para as variáveis de monitoramento de campanhas do Urchin.Seus links codificados devem conter pelo menos três variáveis.Essas variáveis vão ser mapeadas para:

  1. utm_source: a origem da sua campanha (por exemplo, Google, Yahoo)
  2. utm_medium: a mídia da sua campanha (cpc, banner, e-mail)
  3. utm_campaign: o nome da sua campanha (promoção_de_primavera)

Seguindo essas instruções, vamos presumir que o seu site contenha o seguinte URL: 

            www.example.com.br/index.html?camp=GADS&kwd=memory&version=47&type=ppc&adver=google

Etapa 1. Faça uma correspondência entre as suas variáveis e as variáveis do Urchin.Crie uma tabela com o formato mostrado no exemplo abaixo: 

               Campanha     Ad Source   Tipo de anúncio     Palavra-chave   Conteúdo A/B  Master
Códigos do usuário=    camp         adver       type        kwd       version      
Códigos do Urchin =  utm_campaign utm_source    utm_medium  utm_term  utm_content  utm_id

Etapa 2. Modifique o arquivo do UTM.

  • Abra o arquivo urchin.js em um editor de texto.No início do arquivo, há uma seção definida pelo usuário que contém as variáveis utm: 

                //-- UTM Campaign Tracking Settings
                var _uctm=1;
                var _ucto="15768000";
                var _uccn="utm_campaign";
                var _ucmd="utm_medium";
                var _ucsr="utm_source";
                var _uctr="utm_term";
                var _ucct="utm_content";
                var _ucid="utm_id";
                var _ucno="utm_nooverride";
  • Utilizando o exemplo do link codificado citado anteriormente, esta seção seria editada assim: 

                //-- UTM Campaign Tracking Settings
                var _uctm=1;
                var _ucto="15768000";
                var _uccn="camp";
                var _ucmd="type";
                var _ucsr="adver";
                var _uctr="kwd";
                var _ucct="version";
                var _ucid="utm_id";
                var _ucno="utm_nooverride";

Etapa 3. Crie filtros para essas novas variáveis de monitoramento.

  • Esta é a configuração básica que deve ser utilizada para cada filtro: 

                Tipo de filtro:Avançado
                Campo A :utm_campaign_uri (raw)
                Trecho A:new_variable =([^&]*)
                Campo B:(Nenhum)
                Trecho B:(vazio)
                Destino dos dados de saída:utm _( old_variable )
                Construtor:$A1
                Campo A obrigatório:sim
                Campo B obrigatório:não
                Substituir campo de saída:sim
                Diferenciar maiúsculas de minúsculas:não
  • old_variable refere-se à variável de monitoramento padrão do Urchin, enquanto new_variable refere-se à sua própria variável de monitoramento.Lembre-se de que você precisará de um filtro para cada variável modificada.
