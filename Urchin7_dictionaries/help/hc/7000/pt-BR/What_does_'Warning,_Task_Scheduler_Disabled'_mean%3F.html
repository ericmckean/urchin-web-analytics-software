O que significa “Aviso, agendador de tarefas desativado”?

O agendador de tarefas do Urchin é um serviço responsável por executar os arquivos executáveis de processamento do Urchin. O Urchin não pode processar tarefas se o agendador está desativado. Os motivos comuns para a exibição do erro “Aviso, agendador de tarefas desativado" são os seguintes:

  • O serviço urchind não está em execução. Isso pode acontecer quando o servidor é reinicializado e o Urchin não está incluído nos scripts de inicialização do sistema.
  • (Somente Unix) O serviço urchind está em execução, mas é de propriedade de um usuário diferente do que o usuário que possui a distribuição do Urchin.

Para determinar se o agendador está em execução, realize estas tarefas:

  • Abra um shell de comando e navegue até o diretório /path/to/urchin/bin/
  • type: urchinctl status (no Unix, type ./urchinctl status)

Se o serviço do agendador não estiver em execução, você poderá iniciá-lo a partir da linha de comando realizando estas tarefas:

  • Abra um shell de comando e navegue até o diretório /path/to/urchin/bin/
  • type: urchinctl -s start (no Unix, type ./urchinctl -s start)
  • se os dois serviços não estiverem em execução, digite: urchinctl start

Se o serviço estiver em execução, verifique os processos em execução no servidor para determinar quem possui o processo do Urchin da seguinte maneira: ps -ef | grep urchind

Observe o "Nome de usuário" do proprietário do processo na saída retornada: 

Nome de usuário 33597 0.0 0.2 352 204 ?? Ss 27Feb08 17:42.18 /usr/local/urchin6/bin/urchind

Se o usuário não for igual ao usuário que possui a distribuição do Urchin, interrompa o serviço e reinicie-o com o usuário adequado. Para tal, realize uma "alternância" para o mesmo usuário que possui atualmente o processo urchind e interrompa o urchinctl. Use a chave de alternância "-s" conforme descrito acima. Em seguida, volte ao usuário do Urchin e inicie o urchinctl, usando mais uma vez a chave de alternância "-s".
