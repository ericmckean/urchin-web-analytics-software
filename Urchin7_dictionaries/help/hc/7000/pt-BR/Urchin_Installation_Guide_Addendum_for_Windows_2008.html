As observações deste artigo se aplicam à instalação do Urchin no Windows Server 2008 e no Windows Vista. Embora as etapas no Guia de instalação do Urchin (Windows) funcionem para o Windows Server 2008 e o Windows Vista, as considerações a seguir são aplicáveis.

O Windows 2008 contém um novo recurso de segurança chamado UAC (User Account Control ou Controle de Conta de Usuário). Para obter detalhes, consulte http://technet.microsoft.com/pt-br/library/cc709691.aspx.

Os problemas conhecidos que ocorrem com o UAC ativado são:

  1. Se o UAC estiver ativado, o aplicativo não poderá ser gravado corretamente na pasta “Arquivos de programas”. Encontre mais detalhes aqui: http://msdn.microsoft.com/en-us/library/aa905330.aspx

  2. Os executáveis do Urchin não podem ser executados a partir de um prompt de comandos por padrão. Você deverá usar o prompt de comandos como administrador ou definir as propriedades do executável da seguinte maneira: Clique com o botão direito e selecione Propriedades-> Compatibilidade -> “Executar este programa como administrador” -> OK. Os serviços do Urchin Windows operam conforme o esperado, sem precisar de nenhum esforço adicional.

Solução:

  1. Desative o UAC e configure o Urchin onde quiser. Execute o Urchin usando os binários ou os serviços do Windows.
  2. SE O UAC estiver ativado, NÃO instale o Urchin na pasta “Arquivos de programas”. Instale-o no diretório “Usuários”, por exemplo. Modifique as propriedades para executar binários no modo de administrador (conforme descrito no item 2 acima) ou use os serviços do Windows sem nenhuma alteração.
