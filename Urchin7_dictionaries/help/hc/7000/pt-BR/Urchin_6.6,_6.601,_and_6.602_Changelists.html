Lista de alterações do Urchin 6.6, 6.601 e 6.602

Recursos adicionados

Aqui está um breve resumo dos principais recursos apresentados nas versões Urchin 6.6, Urchin 6.601 e Urchin 6.602.

  • Plena integração com o Google AdWords

    • Alertas de orçamento: esse recurso avisa você caso o orçamento de uma campanha do AdWords precise ser ajustado. Ela ficará visível automaticamente na interface de relatórios se alguma das origens de CPC (associadas ao perfil exibido) tiver um alerta de orçamento no sistema do AdWords.

    • Ferramenta de geração de palavras-chave: essa ferramenta permite a você gerar palavras-chave relacionadas e ver as informações de desempenho. Ela também permite adicionar palavras-chave recém-geradas e excluir palavras-chave existentes de campanhas do AdWords.

    • Links diretos para o AdWords: esse recurso permite a você navegar diretamente do Urchin para a tela apropriada na conta do AdWords.

    • Gerenciador da tag do Urchin: esse recurso insere uma tag de inserção de palavra-chave dinâmica {keyword} em URLs de destino do anúncio. Ele simplifica a experiência de codificação de URLs e permite a você importar perfeitamente dados de custo do AdWords para o Urchin.

      • (consulte https://adwords.google.com/support/bin/answer.py?hl=pt_BR&answer=74996)

    • Otimizador do AdWords: essa ferramenta permite a você otimizar a campanha diretamente no AdWords.

  • Visualização e seção do anunciante

    • Modelo do painel de controle do anunciante (visualização) e seção “Otimização de anúncio”: essas melhorias orientadas ao marketing de pesquisa/anunciante fornecem novas formas de visualizar dados de maneira voltada ao anunciante pago e de suportar os recursos relacionados do Google AdWords mencionados acima.

  • Novos relatórios

    • Relatório "Tempo no site": esse relatório pode ser encontrado em "Otimização de conteúdo" -> "Desempenho do conteúdo" -> "Métricas de engajamento".

    • Relatório "Comparação de desempenho": esse relatório permite a você comparar desempenho de Campanha, Palavra-chave e Conteúdo em várias origens/meios (por exemplo, Google|CPC x Yahoo|orgânico). Localizado em “Otimização de anúncio” -> “Resultados da campanha de marketing”.

    • "Estrutura do CPC": esse grupo de relatórios pode ser encontrado em Otimização de anúncio
      • Visualização de campanha exibe informações de campanha UTM paga (as campanhas do Google e do Yahoo são compatíveis).
      • Visualização de palavras-chave exibe informações de palavras-chave pagas.

  • API de dados

    • API de exportação de dados: esse novo recurso importante permite a você recuperar dados de relatórios do Urchin por meio de protocolos SOAP 1.x e REST.

  • Autenticação externa (LDAP)

    • Além da autenticação específica do Urchin (agora chamada de "Nativa"), o Urchin 6.6 é compatível com autenticação externa que pode ser configurada por usuário.
    • Módulos de autenticação externa: o suporte a LDAP e MSAD é fornecido com o Urchin 6.6, embora módulos personalizados possam ser integrados modificando-se os arquivos de configuração.
    • Suporte a SSL/TLS: esses protocolos também são compatíveis.

  • Nova página "inicial" do Urchin

    • A "página inicial" do Urchin (ou o Relatório de visualização completa) foi modificada para fornecer métricas para todos os perfis visíveis ao usuário conectado. O usuário pode escolher entre três visualizações: Básico, Negócios e Administrador.
    • As métricas de perfil são fornecidas para um dia (ontem) e para a semana, o mês e o ano mais recentes.

  • Importação automática de dados do CPC do YSM (Yahoo! Search Marketing)

    • Além do Google AdWords, o Urchin agora permite a você importar dados do CPC do Yahoo! Search Marketing.

  • Instalador com um único clique

    • Todos os instaladores do Urchin, exceto o FreeBSD 6, agora incluem o servidor de banco de dados Postgresql, que pode ser instalado durante o processo de instalação. Isso elimina o requisito de instalação e de configuração de um servidor de banco de dados antes da instalação do Urchin. O usuário pode continuar usando os servidores de banco de dados existentes, desativando essa opção durante a instalação. OBSERVAÇÃO: você pode enfrentar problemas com o Linux Kernel 2.4. Para obter detalhes, consulte o documento Problemas conhecidos.

  • Carregamento otimizado do GeoDB

    • Agora você pode escolher entre a versão completa e a versão leve do GeoDB para minimizar o consumo de memória durante o processamento do registro. O GeoDB leve só fornece informações da área metropolitana até o nível nacional.

  • Utilitário Zip\UnZip atualizado

    • O utilitário Zip atualizado permite a você trabalhar com arquivos de dados menores que 2 Gb.

  • Classificação no Painel de controle do resumo de marketing

    • No relatório Resumo de marketing, você pode classificar origens, palavras-chave e campanhas por métricas e seus desvios.

  • Diversos

    • Controle de processo: o Urchin 6.6 agora permite a você concluir trabalhos com tranquilidade (ou seja, sem corromper o banco de dados), o que é útil quando um trabalho de processamento do registro fica "preso" por algum motivo.

    • Acesso do usuário às ferramentas do AdWords: uma nova configuração "Acesso a origens de CPC" foi apresentada por usuário. Por padrão, apenas Super Admin tem as ferramentas do AdWords ativadas. Para todos os outros usuários, esse acesso deve ser concedido explicitamente por meio dessa nova configuração.

    • Bing.com: Foi adicionado suporte ao novo mecanismo de pesquisa bing.com da Microsoft.
    • Filtros de perfil: esses filtros agora são aplicados a origens de CPC.
    • Guia “Perfis”: uma nova guia chamada "Perfis" foi adicionada a Gerenciamento de origem do registro para vincular a origem do registro a vários perfis.
    • Download remoto do arquivo de registro: um parâmetro configurável de tempo limite do download remoto do registro foi adicionado ao arquivo urchin.conf.

Correções de bug

Aqui está um breve resumo das principais correções de bug apresentadas nas versões Urchin 6.6, Urchin 6.601 e Urchin 6.602.

  • Administração

    • No Urchin 6.5 e em versões anteriores, as pastas relacionadas ao perfil não eram renomeadas quando um perfil era renomeado na interface administrativa. Isso foi corrigido.

  • Processamento de registro

    • Em certos casos, o Urchin encontrava uma falha na segmentação ao processar registros ELF2. Isso resultava no trabalho incompleto do processamento de registro e em dados de relatórios inconsistentes. Isso foi corrigido.
    • O tratamento incorreto da configuração de hora local do perfil durante o processamento de registro estava resultando na mudança de horas dos meses nos dados de relatórios. Isso estava acontecendo nos meses em que começava ou terminava o horário de verão no fuso horário do perfil selecionado. Isso foi corrigido.

    • Um problema no processamento de várias origens de registro UNC foi corrigido. Para conhecer os demais problemas nas origens de registro UNC e as práticas recomendadas, consulte o artigo da Ajuda on-line: https://secure.urchin.com/helpwiki/en/Working_with_UNC_log_sources

  • Suporte a SSL

    • Um problema na conexão com o Urchin via https no Windows 2003 foi corrigido.

  • Caracteres especiais em senhas de origem do CPC

    • Determinados caracteres especiais não eram permitidos em senhas de origem do CPC. Isso foi corrigido e os caracteres !@#$%^~`&* _+»<>{}[]()№?-=:;\|/,. agora são permitidos.

  • Licenciamento

    • A redefinição de licença regular e a reativação eram obrigatórias após a atualização para o 6.600 em Linux e FreeBSD. Isso foi corrigido no Urchin 6.601, e a reativação não é mais obrigatória.
    • Antes do Urchin 6.602, a atualização durante o uso de uma licença demo podia resultar em problemas no login/licenciamento. Esse problema foi corrigido. OBSERVAÇÃO: A atualização de uma instância do Urchin que esteja usando uma licença Demo não prorroga o período de demonstração.
    • Para uma licença regular, o número máximo de origens de registro aumentou de 1.000 para 3.000. Essa alteração exige a geração de uma nova chave de licença no servidor de licenciamento do urchin. As instalações “novas” do urchin (independentemente da versão do urchin) receberão essas alterações automaticamente. As instalações existentes ou as atualizações exigirão a execução manual do utilitário ugetlicense pelos administradores do urchin.

  • Migração

    • O utilitário de migração de dados de relatórios (convert-u5data) foi aprimorado significativamente e o desempenho foi otimizado ainda mais. Na maior parte dos casos, agora demorará muito menos tempo para importar dados de relatórios “grandes”.
    • Em certos casos, quando a migração dos bancos de dados de relatórios do Urchin 5 para o Urchin 6 falhava ou era interrompida, havia falha nas tentativas subsequentes de reiniciar a importação com a mensagem de aviso: "AVISO: (8010-577-1192) O tamanho do arquivo de banco de dados está errado - execute a limpeza". Esse problema foi corrigido.
    • Durante a migração de dados de relatórios do u5 para o u6, alguns dados não eram importados corretamente para o urchin 6. Isso foi corrigido.

  • Diversos

    • A Ajuda incorporada do Urchin e vários artigos da Central de Ajuda do Urchin on-line foram atualizados.

Observações de uso importantes

  • MySQL

    • O Urchin 6.6xx não é compatível com MySQL 4.x.x. Para que o Urchin 6.6 funcione corretamente, a instância existente do MySQL deve ser atualizada para o MySQL 5.03+.

  • FreeBSD 5.x

    • A plataforma FreeBSD 5.x foi deixada de lado.

  • Cookies no lado do cliente

    • Para facilitar uma melhor interação na interface de relatórios, o Urchin 6.6 usa cookies no lado do cliente. A visualização apropriada dos relatórios exige que os cookies do lado do cliente sejam ativados no navegador do cliente.

  • Uso restrito sob a licença “demo”

    • O Urchin 6.6 apresenta determinadas restrições de licença de demonstração. Os perfis e as origens de registro estão limitados a cinco cada. Não há nenhum limite para filtros, origens de CPC, usuários, contas ou grupos.

  • Upgrade

    • O Urchin 6.6 apresenta uma nova seção de relatório e visualizações. Após o upgrade das versões anteriores, esses relatórios e visualizações podem deixar de mostrar dados. Recomendamos o reprocessamento de perfis existentes para preencher os novos relatórios.
