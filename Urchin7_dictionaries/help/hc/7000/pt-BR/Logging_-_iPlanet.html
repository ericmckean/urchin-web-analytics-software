Como efetuar registros - iPlanet

Visão geral

Este artigo fornece uma breve visão geral de como configurar registros para um servidor da web iPlanet, de forma a facilitar as operações apropriadas de processamento e geração de relatórios para o Urchin.

Use o tipo "Netscape" para a configuração "Origem do registro".

Existe um conjunto de campos mínimos necessários para que o Urchin possa gerar relatórios.Eles são:

  • data
  • hora
  • nome do host ou endereço ip do sistema solicitante
  • solicitação (ou seja, que documento foi solicitado pelo sistema ao servidor da web)
  • código de status gerado pela solicitação (numérico)
  • bytes (bytes transferidos do servidor para o cliente)

Além disso, para a geração de relatórios mais completa, os seguintes campos são necessários:

  • referência
  • user-agent
  • cookies (se o Monitor de tráfego do Urchin estiver instalado no seu site)

Configuração 

Init fn=flex-init access="$accesslog"
format.access="%Ses->client.ip% - %Req->vars.auth-user%
[%SYSDATE%] \"%Req->reqpb.clf-request%\"
%Req->srvhdrs.clf-status% %Req->srvhdrs.content-length%
\"%Req->headers.user-agent%\"
\"%Req->headers.referer%\"
\"%Req->headers.cookie%\""
