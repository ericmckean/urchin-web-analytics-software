Como posso fazer a atualização para o Urchin 6?

  • As contribuições com este artigo foram feitas por Analytics Pros, consultor autorizado do software Urchin

Não há caminho de atualização do Urchin 5 para o Urchin 6. No entanto, você pode migrar os seus dados do Urchin 5 para o Urchin 6. Siga as instruções deste artigo para migrar seus dados de configuração (contas, usuários etc.) e de relatório.

  • Observações importantes:

    • Não instale o Urchin 6 na mesma pasta que o Urchin 5. Isso corromperá o Urchin 6 e o Urchin 5. Além disso, o instalador do Urchin não fornecerá um aviso se você tentar instalar o Urchin 6 na mesma pasta em que o Urchin 5 se encontra.

    • Não é recomendável migrar dados com meses de sobreposição entre os dados de relatório do Urchin 5 e do Urchin 6. Se o banco de dados de destino já contiver dados de um mês que você está tentando importar, sua importação será muito mais lenta.

Como migrar dados de configuração

1. Na instalação do Urchin 5, execute o utilitário uconf-export. Esse utilitário recupera o conteúdo do banco de dados de configuração (que, no Urchin 5, é um arquivo simples). Por padrão, (sem argumentos) esse utilitário envia texto para std out (monitorar). Basta usar o argumento -f para indicar o nome do arquivo para o qual você deseja enviar os dados. Observação: o arquivo de saída deve ficar acessível na instalação do Urchin 6. 

uconf-export -f PATH+FILE_NAME

2. Na instalação do Urchin 6, execute 

uconf2sql -f PATH+FILE_NAME

em que PATH+FILE_NAME é o arquivo para o qual você exportou os dados.

Isso concluirá o processo de migração dos seus dados de configuração do Urchin 5 para o Urchin 6.

Como migrar dados de relatório

1. Na instalação do Urchin 5, salve o diretório "Urchin5 BASE" em um local que seja acessível na instalação do Urchin 6. Por exemplo, se estiver executando o Urchin 5 e 6 no Windows, você poderá compartilhar o diretório do Urchin 5 e montá-lo como uma unidade no sistema de arquivos do Urchin 6.

2. Na instalação do Urchin 6, execute o utilitário convert-u5data da seguinte maneira: 

convert-u5data  [-a account] [-p PROFILE_NAME] -c PATH_2_U5_InstallDir

Isso recuperará todos os dados de relatórios dos arquivos do Urchin 5 e os migrará para o Urchin 6.

Observações de uso do convert-u5data

  • -a é opcional, se não for usado - todas as contas serão pesquisadas para verificar a existência de perfis a serem migrados
  • -p é opcional, se não for usado - todos os perfis das contas pesquisadas serão migrados
  • -d é opcional, usado para executar o utilitário no modo de depuração (Urchin 6.6+)
  • -z é opcional, usado para converter bancos de dados arquivados (Urchin 6.6+)

Observação: se você tiver personalizado o local do diretório de dados do Urchin modificando ./Urchin5_Install_Base/etc/urchin.conf para usar um local diferente de "./data", precisará garantir que o local e o usuário que executam convert-u5data tenham acesso ao "mesmo caminho definido em urchin.conf. Por exemplo, se você tiver atualizado urchin.conf para usar "D:\urchin5_data\", o sistema que executa convert-u5data procurará pelos dados do Urchin 5 em "D:\urchin5_data\". Se esse caminho não for válido no sistema em que o comando está em execução, você receberá a mensagem de erro Erro ao alterar os Diretórios. Fechando. Nesse caso, o procedimento recomendado é copiar a pasta de dados do Urchin 5 de volta para a pasta "dados", no local Urchin5_Install_Base", e atualizar o urchin.conf para usar "./data" para a configuração de local da pasta de dados.
