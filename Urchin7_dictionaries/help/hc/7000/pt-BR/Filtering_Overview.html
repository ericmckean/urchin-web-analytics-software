Visão geral da filtragem

Este artigo foi escrito por Panalysis, um Consultor autorizado do software Urchin

A filtragem é um conceito importante no Urchin 6.Filtros permitem controlar como os dados são exibidos em relatórios do Urchin.Esse controle pode ser simplesmente a inclusão ou a exclusão de dados ou modificações complexas.

O uso típico de filtros inclui:

  • Exclusão das suas próprias atividades do site da web.
  • Criação de relatórios que estão restritos a uma área ou diretório específico no site da web.
  • Adição do nome de domínio ao relatório "Conteúdo principal".
  • Adição dos parâmetros de consulta de URL ao relatório "Conteúdo principal".

Os filtros operam por meio da aplicação de uma das cinco ações possíveis a um campo específico no arquivo de registro ou nos relatórios do Urchin.As ações possíveis incluem:

  • Excluir - remoção de dados dos relatórios.

  • Incluir - limitar os dados exibidos em relatórios a itens específicos.

  • Pesquisar e substituir - modificar os dados substituindo-os por outros dados.

  • Tabela de consulta - adicionar conteúdo aos relatórios com base em uma tabela externa de dados.Por exemplo, converter ?id=1 em ?id=Minha página inicial.

  • Avançado - modificar os dados por meio da extração e da combinação de dados.

Para acessar filtros, é necessário fazer login como Administrador.Após o login, você pode criar filtros clicando na guia "Configuração", no canto superior direito da tela, e acessando "Perfis do Urchin".A partir desse local, é possível adicionar o filtro por meio do "Gerenciador de filtro" ou editando ou criando um Perfil, adicionando um Filtro da guia "Filtros" e clicando no ícone de sinal de adição.

Seqüência de filtragem

No Urchin 6, você pode atribuir um filtro a um Perfil ou a uma Origem do registro.

O Urchin processa filtros logo antes da criação dos relatórios.A seqüência de processamento usada pelo Urchin é a seguinte:

Ler o arquivo de registro bruto -> Criar os campos "Brutos" do arquivo de registro -> Processar os campos nos campos "Automáticos" finais -> Processar filtros no perfil -> Processar filtros na origem do registro -> Mostrar o relatório final.

Isso significa que você pode apenas usar filtros para modificar os dados que são exibidos no relatório.Por isso, a maioria dos filtros usará os campos marcados como "(Automáticos)" para modificar os dados exibidos nos relatórios finais.Obviamente, existem casos nos quais você precisará usar os campos marcados como "(Brutos)", mas esses casos são relativamente raros.

Importante:se os filtros de tipos diferentes forem usados, eles serão aplicados na seguinte ordem:

  • avançado, pesquisar e substituir
  • tabela de consulta
  • excluir e incluir

Se vários filtros do mesmo tipo forem especificados, eles serão processados na mesma ordem na qual estão vinculados às origens de registro, por meio da interface de administração (ou diretamente dentro do banco de dados).Cada filtro apenas receberá os dados disponíveis após a aplicação do filtro anterior.

Um erro comum está no uso de filtros "Incluir", nos casos em que dois filtros desse tipo são aplicados em seqüência.Por exemplo:

Filtro 1:incluir um diretório /somedirectory/ Filtro 2:incluir outro diretório /someotherdirectory/

Como uma página não pode residir em ambos os diretórios, o relatório resultante mostrará 0 páginas.O motivo é que o segundo diretório apenas receberá os dados que estão localizados no diretório /somediretory.Como o segundo filtro irá apenas incluir o padrão /someotherdirectory, o resultado será 0 páginas, já que nenhuma página restante depois de Filtro 1 corresponderá a esse padrão.

A solução correta para o problema é usar uma Expressão regular que corresponda a ambos os diretórios no primeiro filtro.Por exemplo, ^/(somedirectory|somotherdirectory)/.*

Dica: teste seus padrões de filtro no relatório relevante antes de adicioná-los ao Perfil do Urchin usando o Campo de filtro na parte superior do relatório.Por exemplo, teste um URL no relatório "Otimização do conteúdo -> Desempenho do conteúdo -> Conteúdo principal".
