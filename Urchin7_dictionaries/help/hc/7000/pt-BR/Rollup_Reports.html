Relatórios de totalização (6.602)

Relatórios de totalização permitem que você revise o desempenho de vários perfis na Página inicial do Urchin. A Página inicial oferece três visualizações: "Básica", “Administrador“ e “Empresa”, as quais você pode selecionar a partir da lista suspensa “Visualização”.

Clicar no botão “Editar”, leva você para a guia "Configurações de perfil/Relatórios". A coluna “Status” exibe o status atual do perfil. Esse status é independente do período selecionado. O ícone “Filtrado” () é exibido próximo aos perfis que foram filtrados. Perfis sem origens de CPC (cost per click ou custo por clique) exibem o ícone “Contas de anúncios” (). Clique no ícone “Contas de anúncios” para navegar até AdWords. Se as origens de CPC foram associadas ao perfil, a coluna “Contas de anúncios” exibem o número de origens de CPC. Para ver os relatórios, clique no nome de um perfil. Para navegar até o site real, clique no link próximo ao nome do perfil.

As visualizações Básica, Administrador e Empresa são mostradas a seguir.

O seletor de período permite a você selecionar dados do dia, da semana e do ano mais recentes.

Dados do período escolhido são exibidos nas colunas "Acessos", "Duração média da visita", "Bytes", "Visitas", "Taxa de rejeições", "Metas concluídas" e "Exibições de página". O restante das colunas ("Nome do perfil", "Status", "Origens do registro", "Filtrado" e "Contas de anúncios") exibe as informações atuais do perfil.

A partir do Urchin 6.602, a visualização da Página inicial padrão (Relatórios de RollUp) de cada usuário pode ser definida manualmente no banco de dados de configuração do Urchin por meio dos campos uglobals.uigl_rollup_view_type e uusers.uius_rollup_view_type:

  • Valores uglobals.uigl_rollup_view_type:
    • 0: auto
      • Esta é a visualização padrão no banco de dados. Neste caso, o valor global do tipo de visualização é inferido a partir do tipo de usuário:

        • Visualização Superadministrador, Administrador de conta -> Administrador;

          Visualização Usuário ¯ Sem atualização -> Negócios

    • 1: Visualização Básica
    • 2: Visualização Administrador
    • 3: Visualização Negócios
  • uusers.view_type:
    • -1: Usar valor global
      • Este será o valor padrão sempre que um novo usuário for criado. Isso corresponde a usar o tipo de visualização de totalização global (uglobals.uigl_rollup_view_type) como o tipo de visualização para este usuário.
    • 0: auto
      • Neste caso, o tipo de visualização do usuário é inferido a partir do tipo de usuário:

        • Visualização Superadministrador, Administrador de conta -> Administrador;

          Visualização Usuário ― Sem atualização -> Negócios

    • 1: Visualização Básica
    • 2: Visualização Administrador
    • 3: Visualização Negócios

IMPORTANTES OBSERVAÇÕES DE USO

  • Tentar fazer login em uma conta que contenha uma grande quantidade de dados pode resultar no seguinte erro: 

    Session Error - unable to call default routine. (2002) Please login again.
  • Isso foi causado quando a extração de dados para relatórios de totalização excedeu o padrão APPLICATION_TIMEOUT de 45 segundos. As seguintes soluções são sugeridas:
  • Aumente o valor de APPLICATION_TIMEOUT no arquivo XYZ para um valor maior para que este problema desapareça. Veja as repercussões desta alteração.
  • Altere o tipo de visualização para "Visualização básica" atualizando uglobals.uigl_rollup_view_type e todos os campos uusers.uius_rollup_view_type no banco de dados de configuração do Urchin de valor'1'.
