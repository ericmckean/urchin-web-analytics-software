Existem limitações para o tamanho dos arquivos de registro?

O mecanismo de processamento de registros do Urchin pode processar registros do servidor da web com mais

  • de 2GB, independentemente de eles estarem compactados com os programas bzip (.bz2), gzip

(.gz) ou zip (.zip), já que o Urchin descompacta os arquivos usando a biblioteca zlib.

Entretanto, o mecanismo de processamento de registros do Urchin utiliza os binários unzip/zip no diretório util para criar backups e arquivamentos dos dados de relatórios para cada perfil.Se os dados de relatórios referentes a um mês forem suficientemente grandes para fazer com que o arquivo zip exceda 2GB, o Urchin não conseguirá criar backups ou arquivamentos e, a menos que os backups e arquivamentos automáticos estejam desabilitados, o processamento de registros irá falhar.Nesse caso, será necessário fazer o backup manual dos arquivos com outro utilitário externo.
