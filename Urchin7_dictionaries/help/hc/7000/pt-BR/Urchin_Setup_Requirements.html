Requisitos de configuração do Urchin 6

  • Este artigo foi escrito por EpikOne, consultor autorizado do software Urchin

Este artigo trata de questões operacionais que devem ser levadas em consideração antes de instalar e executar o Urchin. Algumas informações são obrigatórias para operar o Urchin com êxito. Outros itens são importantes para usar o Urchin com mais eficiência depois que o software está instalado.

Considerações básicas sobre a instalação do Urchin

  • No Windows, você deve instalar enquanto estiver conectado como administrador.
  • Em sistemas tipo UNIX, você pode instalar como qualquer usuário, mas, se não instalar como o superusuário, haverá restrições quanto às áreas do sistema de arquivos que podem ser instaladas.
  • O Urchin vem acompanhado do binário do servidor da web Apache para entrega de relatório e configuração. Os administradores de sistema devem estar cientes de que esse novo serviço da web estará em execução depois que o Urchin for instalado.
  • Instale o Urchin 6 em um local do disco que tenha bastante espaço (por exemplo, um local com pelo menos várias centenas de megabytes) para permitir o crescimento dos bancos de dados do Urchin com o passar do tempo. Consulte a seção Problemas de desempenho e gerenciamento para obter mais informações.

  • Se estiver fazendo upgrade a partir do Urchin 5, você precisará importar seus bancos de dados no Urchin 6 usando os utilitários uconf2sql e convert-u5data. Não existe nenhum upgrade direto do Urchin 5 para o Urchin 6, simplesmente executando o instalador do Urchin 6. Consulte Upgrades na seção Primeiros passos do Centro de documentação.

  • O upgrade do Urchin 5 para o Urchin 6 requer o novo licenciamento do produto.

Considerações básicas sobre o processamento do Urchin

  • Acesso aos registros do servidor da web - Você deve conhecer o caminho até os arquivos de registro de um determinado site e ter permissão para acessar esses arquivos. Se os registros estiverem em um sistema remoto, você também precisará de um nome de conta e de uma senha para recuperá-los (FTP/HTTP/HTTPS/UNC).
  • Formato do registro configurado adequadamente - Embora o Urchin possa processar formatos de registro personalizados, você simplificará os requisitos de gerenciamento se configurar seu servidor da web adequadamente para fazer login em um formato padrão. Recomenda-se o uso do formato de registro “Combinado estendido” (por exemplo, registros do NCSA ou do Apache) ou o formato “W3C estendido” (por exemplo, registros do IIS). Para os sites IIS, o registro de “Processar contabilidade” deve ser desativado. Consulte a seção Configuração avançada para obter mais informações.
  • Conta de usuário exclusiva para processos do Urchin - Em sistemas tipo UNIX, é recomendável aumentar a segurança, executando os programas do Urchin com um ID de usuário especial usado exclusivamente para o Urchin e que tenha somente alguns privilégios. A configuração dessa conta requer que você tenha muitos privilégios ou privilégios de superusuário no sistema em questão.
  • Agendamento - Você precisa escolher uma agenda de execução para o processamento do Urchin fornecer relatórios no momento adequado e também uma conta para o tempo necessário de processamento, caso tenha grandes conjuntos de dados.

Considerações avançadas sobre o processamento do Urchin

  • Se desejar acompanhar os visitantes únicos, você terá que realizar estas etapas básicas:
    • Instale o código do sensor UTM nas páginas da web do seu site
    • Ative o registro de cookies no formato de registro do seu servidor da web
    • Defina o método de rastreamento no perfil do Urchin para o site como UTM
  • Se não desejar acompanhar os visitantes únicos, você deverá verificar qual nível de granularidade deseja para o relatório de visitantes ou sessões e selecionar o método alternativo de rastreamento de visitantes adequado para cada site. O único método de rastreamento alternativo disponível no Urchin 6 é IP/User-Agent.

Problemas de desempenho e gerenciamento

  • Rotação de registros - Se você não tiver algum mecanismo externo para arquivar ou remover registros do servidor da web depois de serem processados pelo Urchin, configure o Urchin para realizar essa tarefa em “Configurações avançadas” para cada origem do registro.
  • Retenção dos bancos de dados anteriores do Urchin para relatórios históricos - Assim que os bancos de dados de um determinado mês são criados, eles são disponibilizados a partir desse mês para análises do histórico. Os usuários devem determinar o tempo que precisam manter dados históricos para eliminar dados desnecessários e economizar espaço em disco. O Urchin pode ser configurado para compactar bancos de dados que são mais antigos do que uma determinada data.
  • Requisitos de memória - O Urchin tem controles de configuração para limitar a quantidade de RAM utilizada durante o processamento de registros. O padrão é 128 MB, que pode ser muito conservador para sites com registros com mais de 10 MB. O sistema deve ter RAM suficiente para que você possa aumentar o uso da memória do Urchin conforme necessário e ajustar as configurações de memória do software para obter o máximo desempenho de processamento.
  • Local do armazenamento de dados do Urchin - Utilizando o arquivo etc/urchin.conf, o Urchin pode ser configurado para que os bancos de dados de relatório sejam armazenados em uma área do sistema de arquivos fora da distribuição do Urchin. Isso permite alocar um espaço dedicado do sistema de arquivos suficiente para o crescimento do banco de dados quando for mais adequado.

Acesso remoto e problemas de integração

  • Uso de SSL para administração e relatório do Urchin - O servidor da web Apache fornecido com o Urchin oferece suporte para SSL. A configuração não tem o SSL ativado por padrão, no entanto, essa opção pode ser ativada pelo usuário (se desejar).
  • Configuração do firewall - Se a topologia da rede incluir firewalls, servidores proxy e outros elementos que ficarão entre o servidor de processamento do Urchin e os usuários que tentam visualizar relatórios ou sistemas que hospedam os registros que precisam recuperar, esses dispositivos deverão ser configurados de modo que não interfiram no acesso remoto do Urchin. Isso normalmente pode ser feito sem afetar a segurança que esse tipo de topologia tenta fornecer.
