Como efetuar registros - Outros servidores da web

Visão geral

Este artigo fornece uma breve visão geral de como configurar registros para servidores da web além do Apache e do IIS, de forma a facilitar as operações apropriadas de processamento e geração de relatórios para o Urchin. O Urchin irá processar qualquer registro de servidor da web, desde que ele consiga compreender como os dados são organizados em cada entrada de arquivo de registro.As informações neste artigo aplicam-se somente a registros de acesso.Se você estiver interessado em detalhes de registros de e-commerce, consulte a seção Módulo de E-commerce do Centro de documentação.

Independentemente do tipo de servidor da web ou do formato de registro, existe um conjunto de campos mínimos necessários para que o Urchin possa gerar relatórios.Eles são:

  • data
  • hora
  • nome do host ou endereço ip do sistema solicitante
  • solicitação (ou seja, que documento foi solicitado pelo sistema ao servidor da web)
  • código de status gerado pela solicitação (numérico)
  • bytes (bytes transferidos do servidor para o cliente)

Além disso, para a geração de relatórios mais completa, os seguintes campos são necessários:

  • referência
  • user-agent
  • cookies (se o Monitor de tráfego do Urchin estiver instalado no seu site)

Configuração

Seria muito trabalhoso e inconveniente relacionar os pormenores de como fazer alterações nas características de registro para cada servidor da web.Em geral, a abordagem mais fácil é configurar os registros de forma que eles fiquem em conformidade com o formato NCSA ou W3C do Urchin e, em seguida, escolher o formato padrão apropriado no menu suspenso "Formato do registro" da Origem do registro.Se o seu servidor da web puder oferecer suporte a essa abordagem, consulte o documento Como efetuar registros - Apache e IIS.As informações nesse documento sobre como os campos de dados necessários são configurados podem ser úteis, mesmo que os detalhes sobre os métodos para a realização de alterações não se apliquem necessariamente ao seu servidor da web.
