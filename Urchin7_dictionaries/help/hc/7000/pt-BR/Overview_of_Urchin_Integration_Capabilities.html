Visão geral dos recursos de integração do Urchin

Introdução

O front-end do Urchin se baseia em componentes modulares e em técnicas de padrão industrial, para permitir que os administradores integrem o acesso a relatórios do Urchin em suas infra-estruturas existentes ou planejadas.Como parte do nosso comprometimento em hospedar provedores e centros de dados, testamos e aceitamos três pontos de integração, que possibilitarão a integração do Urchin à maioria das arquiteturas.

O diagrama a seguir ilustra os principais componentes do front-end do Urchin e os três pontos de integração.Ambas as funções de administração e geração de relatórios se baseiam na web e são fornecidas a partir desse sistema.

Todo o conteúdo do sistema Urchin é fornecido por meio de um servidor Apache instalado como "urchinwebd", mostrado no lado esquerdo do diagrama.As solicitações são tratadas pelo servidor Apache e transferidas com o uso da interface CGI a um aplicativo Controlador de sessão (session.cgi).O Controlador de sessão realiza a Autenticação e, dependendo da ação, transfere a solicitação para o Mecanismo de administração (admin) ou o Mecanismo de relatório do Urchin (urchin.cgi).O conteúdo com identificadores de sessão incorporados é transferido de volta ao usuário.

Os três pontos denominados "A", "B" e "C" são os três pontos de integração mencionados previamente.O ponto "A" permite ao administrador substituir ou ignorar o servidor da web do Urchin.O ponto "B" possibilita a autenticação externa ou a ausência de autenticação.Por sua vez, o ponto "C" permite o acesso direto à geração de relatórios via wrapper ou portal.

Ponto A:integração com o servidor da web

Muitas empresas de hospedagem já terão um servidor da web em execução ou terão um servidor da web especialmente compilado que é executado em seus sistemas.Desde que um servidor da web esteja fornecendo operações CGI básicas, você pode substituir o binário do Apache ou usar um servidor da web existente para fornecer acesso ao Urchin.

Esse ponto de integração mantém o restante do sistema Urchin intacto.A interface do Urchin será usada para administração, autenticação e fornecimento de relatórios.Usuários e Perfis precisarão ser configurados no sistema Urchin, de forma que, quando um usuário entrar no sistema, o Urchin saiba quais relatórios esse usuário poderá acessar.

Ponto B:autenticação externa ou desvio da autenticação

Ambientes de hospedagem complexos terão, muitas vezes, controles centralizados para proporcionar a autenticação dos usuários, como o LDAP.Com uma simples alteração de configuração, o Controlador de sessões pode chamar uma rotina de autenticação que você escolher.Uma interface simples é fornecida para retornar logins com êxito ou falha.Rotinas de autenticação existentes podem ser facilmente incluídas, de modo a possibilitar a estrutura correta para ambos os sistemas.Usando a integração no ponto "B", também é possível ignorar a autenticação com uma rotina fictícia e estabelecer um vínculo direto com a página de destino.

Com o uso da integração no ponto "B", o sistema Urchin em geral permanece intacto, exceto pela autenticação, que é efetuada por um aplicativo externo.Usuários e Perfis precisarão ser configurados no sistema Urchin, de forma que, quando um usuário entrar no sistema, o Urchin saiba quais relatórios esse usuário poderá acessar.Para obter detalhes completos sobre a interface para o uso de aplicativos externos e para ver um exemplo de como ignorar a autenticação, consulte Como usar a autenticação externa ou o desvio da autenticação.

Ponto C:vinculação direta com relatórios a partir de um wrapper ou portal

Muitos dos nossos clientes têm uma relação de um para um entre um cliente e um site e desejam vincular-se à geração de relatórios para um site específico diretamente a partir da área de administração desse site.Em geral, essa área já está autenticada.A integração do ponto C facilita a vinculação direta com a geração de relatórios, seja via um script de wrapper ou de um portal existente.Nesse cenário, a tela de autenticação e relatório inicial do Urchin é ignorada, e os usuários são levados diretamente aos relatórios do Urchin.

Ao usar a integração no ponto "C", pressupõe-se que o provedor de serviços tenha controle sobre quem visualiza o quê, seja por uma relação de um para um ou por um banco de dados de configuração existente. O Urchin ainda precisará ser configurado para a geração de relatórios de Perfil, mas isso pode ser automatizado dentro ou fora da interface de administração. O Urchin oferece a capacidade de estabelecer um vínculo direto com um relatório do Urchin usando um URL específico quando a opção "Vinculação direta ao relatório" está ativada na interface de administração.O acesso ao relatório do Urchin é controlado por meio de um arquivo ".report.conf" embutido no diretório especificado por esse URL.

No caso de uma integração de portal complexa, o mecanismo de geração de relatórios do Urchin irá propagar as variáveis de sessão e outras variáveis de portal, de forma a manter a sessão operacional.
