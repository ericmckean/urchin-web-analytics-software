Filtragem global de hits a partir do software de monitoramento

Visão geral

A maioria dos ambientes de hospedagem fornece um certo tipo de monitoramento de servidores da web de clientes para manter SLAs (Acordos de nível de serviço ou Service Level Agreements).Entretanto, como efeito colateral, os hits desse monitoramento podem distorcer de fato a geração de relatórios do Urchin para os sites da web monitorados - aumentando artificialmente as contagens de sessões, exibições de página e bytes.

Recomendação

Em ambientes de hospedagem que utilizam esse tipo de monitoramento, é altamente recomendável que um filtro padrão/global do Urchin seja aplicado aos perfis configurados do Urchin de cada cliente para eliminar os hits gerados pelo software de monitoramento.Isso é feito facilmente em um ambiente no qual uma instalação centralizada do Urchin (gerenciada pela empresa de hospedagem) fornece relatórios para os sites da web de cada cliente.Em ambientes dedicados e com compartilhamento de localização nos quais os próprios clientes mantêm uma instância do Urchin em seus servidores, a empresa de hospedagem deve fornecer um filtro de amostra que seja apropriado para o monitoramento que está sendo usado.

Para auxiliar na implementação da filtragem do Urchin, o Host e o cliente precisam trabalhar em conjunto para criar uma página específica no site do cliente que seja utilizada somente pelo software de monitoramento; por exemplo, algo como: 

     http://www.customerdomain.com/healthpage.html

Exemplos

Exemplo 1:Descartar o endereço IP para o sistema de monitoramento 

      Filter Type: Exclude
      Filter Field: IP
      Filter Spec: 172\.16\.1\.1
  • Isso irá remover do registro do servidor da web todos os hits com o endereço 172.16.1.1 à medida que o Urchin processar esse registro.

Exemplo 2:Descartar a página específica acessada pelo sistema de monitoramento 

      Filter Type: Exclude
      Filter Field: REQUEST
      Filter Spec: ^/healthpage.html
  • Isso irá remover do registro do servidor da web todos os hits com uma solicitação para /healthpage.html à medida que o Urchin processar esse registro.

Considerações

  1. Talvez seja preferível criar Perfis adicionais não filtrados para os clientes, para que eles possam ver a carga de tráfego real (incluindo a filtragem) nos servidores da web.
  2. A empresa de hospedagem pode optar por fornecer um Perfil que ofereça a geração de relatórios exclusivamente para os hits de monitoramento; por exemplo, filtrando somente os hits provenientes do software de monitoramento.Esse perfil poderia ser usado para mostrar que o devido monitoramento está sendo feito e que os SLAs estão sendo cumpridos.
