Como faço para restringir o acesso do Urchin a um único endereço IP?

Visão geral

Em um sistema que possui várias interfaces de rede ou vários endereços IP vinculados a uma única interface de rede, o comportamento padrão é que a interface da web do Urchin vincule todos os endereços IP configurados.Entretanto, talvez seja preferível configurar o Urchin de forma a restringir o procedimento de administração e geração de relatórios a um único endereço IP.

Procedimento

Restringir o procedimento de administração e geração de relatórios do Urchin requer uma simples alteração no arquivo de configuração utilizado pelo servidor da web Apache do Urchin.Siga estas etapas:

Para sistemas UNIX:

  • Abra um shell de comando como o usuário usado que o Urchin executa como
  • "cd" para o diretório no qual o Urchin está instalado.
  • Pare o servidor da web do Urchin com o comando:"bin/urchinctl -w stop"
  • Usando um editor de texto, abra o arquivo "urchinwebd.conf.template" no diretório "var" da distribuição do Urchin.

  • Acima da linha <Directory />, insira a linha 

       Listen XXX.XXX.XXX.XXX:port
  • em que "XXX.XXX.XXX.XXX" e "port" são o endereço IP e o número da porta aos quais você deseja restringir o Urchin.
  • Reinicie o servidor da web do Urchin com o comando: 

bin/urchinctl -w start

Para sistemas Windows:

  • Pare os Serviços do Urchin:"Iniciar -> Programas -> Urchin 6 -> Desativar serviços do Urchin".

  • Abra o Windows Explorer e navegue até "C:\Program Files\Urchin6\var".
  • Usando um editor de texto, abra o arquivo "urchinwebd.conf.template".

  • Acima da linha <Directory />, insira a linha 

       Listen XXX.XXX.XXX.XXX:port
  • em que "XXX.XXX.XXX.XXX" e "port" são o endereço IP e o número da porta aos quais você deseja restringir o Urchin.

  • Reinicie os Serviços do Urchin:"Iniciar -> Programas -> Urchin 6 -> Ativar serviços do Urchin".

Observação importante:devido ao uso do pooling de soquetes no IIS 5 e versões posteriores, talvez seja necessário desativar o comportamento padrão do IIS de modo a compartilhar soquetes para todos os sites vinculados a uma porta específica, independentemente de qual endereço IP real tenha sido configurado.Esse pode ser o caso típico em ambientes nos quais são utilizados hosts virtuais e quando você deseja configurar a geração de relatórios do Urchin na porta 80 para um host virtual específico.

Considerações

  • Essa técnica somente funciona para restringir o procedimento de administração/geração de relatórios do Urchin a um endereço IP específico.Não é possível restringir o acesso a hosts virtuais com base em nomes.
  • Os upgrades para o Urchin irão sobrescrever o arquivo "urchinwebd.conf.template" após o salvamento de uma cópia de backup.Você precisará reaplicar todas as edições que tenham sido efetuadas nesse arquivo após um upgrade.
