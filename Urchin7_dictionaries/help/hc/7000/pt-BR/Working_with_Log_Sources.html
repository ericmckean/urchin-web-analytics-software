Como trabalhar com origens de registro

Visão geral

Em geral, você irá adicionar uma Origem de registro durante a criação de um Perfil.Uma Origem de registro é uma forma de o Urchin identificar as características de um registro de acesso (às vezes chamado de registro de transferência) para um dos seus sites da web.Registros de acesso contêm todos os hits, ou solicitações de documentos da web, que são feitos para o seu site.Algumas das características associadas a uma Origem de registro são o caminho para o arquivo de registro, o formato do arquivo de registro (por exemplo, W3C ou NCSA), se o registro é local ou está em um sistema remoto e se um filtro deve ser aplicado ao arquivo de registro durante o processamento.

Um conceito importante que deve ser compreendido é que Origens de registro existem independentemente de Perfis.Cada Perfil deve ter pelo menos uma Origem de registro associada a ele para obter relatórios.Entretanto, vários Perfis poderiam usar de modo concebível a mesma Origem do registro.Por exemplo, é possível criar vários Perfis usando a mesma Origem do registro, mas fornecer a cada Perfil um filtro diferente para gerar resultados de relatório variados.Dessa forma, não existe necessariamente uma proporção de um 1 para 1 entre Origens de registro e Perfis.

Como configurar origens de registro

Para começar a adicionar uma Origem de registro ao sistema, faça login como administrador no sistema administrativo do Urchin e clique no botão "Configuração", à esquerda.Em seguida, clique no botão "Gerenciador de registro".Para criar uma nova Origem do registro, clique no botão "Adicionar", na parte superior direita da tela.Você será levado ao "Assistente para adicionar origem do registro".Esse assistente contém uma série de etapas simples projetadas para ajudar você a configurar a Origem do registro de maneira rápida e fácil.Cada tela do assistente tem informações de ajuda explícitas para explicar as informações de configuração exibidas na tela em questão.

Na tela "Configurações do registro", você perceberá que precisa escolher um Formato do registro.Essa configuração informa ao Urchin como os dados no seu arquivo de registro estão dispostos.É importante selecionar o formato correto para o registro, ou o Urchin não conseguirá gerar dados de relatório significativos. O Urchin compreende um conjunto padrão de formatos de registro que você pode escolher em um menu suspenso.Eles são:

  • Automático:o Urchin usa esse formato para detectar automaticamente os formatos de registro NCSA, W3C, Netscape e ELF2.Em vez de selecionar explicitamente um deles, você pode escolher "Automático", e o Urchin irá deduzir corretamente como fazer a leitura dos dados se o seu formato de registro estiver nessa lista.
  • NCSA: Formato estendido/combinado modificado do Apache (consulte Como efetuar registros - Apache e IIS, para obter uma descrição desse formato)
  • W3C: servidores Microsoft IIS usam geralmente esse formato, embora outros servidores da web também possam ser configurados para gerar registros W3C.
  • Netscape: Servidores Netscape e iPlanet usam esse formato por padrão.
  • ELF2: Formato do registro de E-Commerce

  • use esse formato para registros que contêm dados de gastos de custo por clique do Google.Observe que o formato de registro do Google não pode ser detectado automaticamente.

  • Overture:use esse formato para registros que contêm dados de gastos do custo por clique do Overture.Observe que o formato de registro do Overture não pode ser detectado automaticamente.

  • Personalizado:embora essa opção não esteja inicialmente listada no menu suspenso, você pode criar os seus próprios formatos de log personalizados, que aparecerão automaticamente no menu suspenso se estiverem corretamente configurados.

Se você não acha que o seu servidor da web esteja gerando registros em um dos formatos padrão reconhecidos, será possível reconfigurar esse servidor para efetuar registros em um desses formatos ou criar um formato de registro personalizado que esteja em conformidade com o seu método de registro atual.Se quiser reconfigurar os registros do seu servidor da web, escolha registros ao estilo W3C ou NCSA.

Equilíbrio de carga e processamento paralelo de registros

Em sites da web com muito tráfego, o uso de um servidor da web pode não ser suficiente.Com o uso do equilíbrio de carga, o tráfego pode ser espalhado entre vários servidores da web que operam em paralelo.O conjunto de servidores da web paralelos usado é chamado de cluster.Dessa maneira, os registros de atividade para cada site são efetuados em qualquer um dos registros paralelos disponíveis. O "Assistente de configuração de perfil" fornece a opção "Processamento paralelo de registros".Quando o "Processamento paralelo de registros" está ativado, o Urchin abre todos os arquivos de registro de uma vez e faz a leitura dos arquivos de forma alternada, uma seção de cada vez, sendo que cada uma corresponde a 15 minutos de atividade de registro.Ativar a opção "Processamento paralelo de registros" aumenta significantemente o desempenho em sites com equilíbrio de carga.
