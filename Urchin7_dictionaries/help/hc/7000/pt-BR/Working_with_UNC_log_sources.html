Como trabalhar com origens de registro UNC

Quando uma origem de registro UNC é criada no Urchin, estes parâmetros são especificados: Server Name/IP, Remote Server Login, Local Server Login, Remote Log Source File.

O Urchin de fluxo os usa para obter o registro de uma origem UNC como se segue: - Urchin tenta procurar o(s) arquivo(s) especificado(s) no "Remote Log Source File"; - Se a navegação falhar, o Urchin tenta criar uma conexão com o compartilhamento UNC, especificado no campo "Server Name/IP"; - Se a conexão falhou, o Urchin supõe que ela esteja sendo executada em uma conta de usuário que não tenha acesso à rede, por exemplo, uma conta de Sistema local. Nesse caso, ele cria uma sessão de login para a conta de usuário, especificado em "Local Server Login", e falsifica a si mesmo nessa conta, ou seja, o processo do Urchin passa a ser executado nessa conta. Se a opção "Same Local e Remote Login" estiver definida como "Sim", as credenciais de falsificação de identidade são retiradas de "Remote Server Login"; - Quando a falsificação de identidade é feita com êxito, o Urchin tenta mais uma vez criar uma conexão UNC com o compartilhamento de rede e tenta navegar de novo. - Quando o Urchin estiver concluído a origem de registro, ele fechará a conexão UNC e finalizará a falsificação de identidade, ou seja, o processo do Urchin começará a ser executado na conta que era executada antes da falsificação de identidade.

A falsificação de identidade tem estes empecilhos: Enquanto está ativado, o processo do Urchin está sendo executado em uma conta diferente, que não tem acesso ao diretório de dados do Urchin, falhando, assim, ao criar bancos de dados de relatórios. - Bancos de dados de relatórios, criados enquanto a falsificação de identidade estava ativada, poderão não ser acessíveis para o processo do Urchin quando a falsificação de identidade estiver concluída. Ela diminui o desempenho, especialmente quando vários arquivos de registro UNC são anexados ao perfil, uma vez que o Urchin precisa iniciar/finalizar a falsificação de cada arquivo.

Desse modo, as melhores práticas para usar os compartilhamentos UNC no Urchin são: - Executar o Urchin na conta que tem acesso à rede. Neste caso, a falsificação de identidade não ocorre. - Conceder acesso de leitura aos compartilhamentos de registro UNC para a conta de usuário do Urchin em execução. Nesse caso, criar a falsificação de identidade e a conexão UNC pode não necessário.
