Utilitários de importação de dados de CPC (Urchin 6.5+)

O Urchin 6.5 inclui novos utilitários para trabalhar com dados de CPC.

urchincpc - A ferramenta de importação de dados de CPC do Urchin extrai dados de CPC por meio do Google AdWords e do Yahoo! API do Search Marketing para fins de processamento e geração de relatórios.

  • Observação: os dados de CPC arquivados baixados do AdWords não podem exceder 250 MB (Urchin 6.501+).

urchincpcd - O Programador de origem de CPC do Urchin (ou simplesmente Agendador de CPC) é usado para programar tarefas de download de CPC com base em uma programação definida pelo usuário (semelhante ao Programador de perfil).

O Agendador de CPC (urchincpcd) é executado em um host local junto com o Programador principal do Urchin (urchind). O Programador principal deve estar ativado para que as origens de CPC sejam programadas. O Agendador de CPC verifica periodicamente a fila de tarefas de download de dados de CPC e controla a sua execução.

A ferramenta de importação de CPC (urchincpc) será iniciada automaticamente se houver alguma tarefa programada. O Agendador de CPC gera processos de trabalho internamente, porém é possível chamá-los diretamente pela linha de comando para iniciar a execução das tarefas disponíveis na fila.

Os dados de CPC baixados são armazenados localmente em /path/to/urchin/data/cpc/ e usados automaticamente pelo mecanismo do Urchin durante o processamento dos perfis vinculados.

O Agendador de CPC do Urchin é gerenciado pelo utilitário urchinctl. Para usar exclusivamente o Programador de origem de CPC, use a opção "-c". Por exemplo, 

urchinctl -c iniciar/parar/reiniciar

irá iniciar/parar/reiniciar somente o Agendador de CPC.

Para programar um download imediato dos dados de CPC, vá até "Configurações" -> "Gerenciador de origens de CPC" -> "Fazer download" ou "Editar" -> guia "Programar" de determinada origem de CPC e clique no botão "Fazer download agora". O Agendador de CPC deve estar ativado para processar a solicitação.

Uso: Agendador de CPC - urchincpcd

O Agendador de CPC está disponível na pasta bin da instalação do Urchin 6.5+. As seguintes opções estão disponíveis pela linha de comando:

  • -d ativa o modo de depuração. Mais informações de depuração são registradas na saída. No Windows, funciona somente com -console. Por exemplo: 

    •            urchincpcd.exe -console -d

  • -h imprime informações de ajuda e é encerrado

  • -v imprime o número da versão e é encerrado

Para o Windows, forneça um dos seguintes comandos como primeiro argumento:

  • -install para instalar o serviço Agendador de CPC do Windows

  • -remove para remover o serviço Agendador de CPC do Windows

  • -start para iniciar o serviço Agendador de CPC do Windows

  • -stop para parar o serviço Agendador de CPC do Windows

  • -console para iniciar o Agendador de CPC no modo console

Uso: ferramenta de importação de CPC - urchincpc

As seguintes opções estão disponíveis pela linha de comando:

  • -c [id] faz download da origem de custo especificada; "id" é o ID da origem de CPC

  • -d ativa o modo de depuração

  • -h imprime informações de ajuda e é encerrado

  • -v imprime o número da versão e é encerrado

Se o Agendador de CPC for iniciado sem parâmetros ou somente com a opção -d, ele começará a processar as tarefas que estiverem disponíveis na fila e será encerrado quando não houver mais nenhuma tarefa para ser processada.

Como fazer download de dados de CPC pelo proxy (6.501+)

Para ativar o download de dados de CPC pelo proxy, edite o arquivo urchin.conf, exclua as barras de comentário das entradas WebServiceProxyHost, WebServiceProxyPort, WebServiceProxyUsername e WebServiceProxy e forneça os valores desejados. 

 ######################################################
# Configurações do gerenciador de origens de CPC
#...
# WebServiceProxyHost, WebServiceProxyHost: Host e porta do HTTP/HTTPS
# proxy que o Gerenciador de origens de CPC deve usar para acessar origens de CPC externas.
# WebServiceProxyUsername, WebServiceProxyPassword: nome de usuário e senha para
# proxy que impõe a autorização.
######################################################
#WebServiceProxyHost: <proxy host>
#WebServiceProxyPort: <proxy port>
#WebServiceProxyUsername: <username>
#WebServiceProxyPassword: <password>

Para o WebServiceProxyHost, você pode inserir um endereço IP ou um nome de domínio completo.
