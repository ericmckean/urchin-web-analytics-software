Meu site tem vários nomes de domínio.Como faço para configurar o monitoramento?

Se seu site utiliza um carrinho de compras seguro em um domínio raiz diferente (por exemplo, www.minhaloja.com.br e www.securecart.com) ou usa vários aliases de domínio para indicar o mesmo conteúdo, será necessário fazer algumas alterações para se certificar de que o site e o UTM sejam compatíveis.

  • Observe que há diferença entre vários subdomínios e vários domínios raiz. www.example.com.br e carrinhoseguro.example.com.br são exemplos de vários subdomínios, enquanto www.example.com.br e www.amostra.com.br representam vários domínios raiz. Se seu site tem vários subdomínios, veja: Como faço para monitorar todos os subdomínios do meu site em um único perfil? Como faço para monitorar todos os subdomínios do meu site em perfis separados? para obter informações sobre configuração.

Vários nomes de domínios:Aliases de domínio

Se seu site usa vários aliases de domínio, recomendamos que você redirecione o tráfego dos aliases para o site principal.Isso garante que o monitoramento de visitantes do UTM seja efetuado no domínio principal e que todos os visitantes sejam monitorados de forma consistente.

As instruções a seguir mostram um exemplo de como redirecionar os domínios de alias para um domínio principal nos servidores Apache e IIS.

Como redirecionar aliases no Apache

  • Se você estiver utilizando um servidor da web Apache, a configuração poderá ser facilmente modificada para redirecionar todo o tráfego de um dos aliases para o site principal.Uma das formas de fazer isso é criar duas entradas VirtualHost.A primeira será o domínio principal que incluirá as configurações normais; a segunda VirtualHost será para todos os aliases e redirecionará para o principal: 

        #---primary virtualhost
        Nome do servidor www.meusite.com.br
        Alias do servidor meusite.com.br
        ...
        #---second virtualhost
        Nome do servidor meusite.org
        Alias do servidor www.meusite.org meusite.net www.meusite.net
        RewriteEngine on
        RewriteRule ^(.*) http://www.meusite.com.br$1 [R=301]

  • A segunda VirtualHost utiliza uma regra de regravação com um código de redirecionamento 301 (Movido permanentemente) para direcionar todo o tráfego para o site original.Nesse ponto, no que se refere ao UTM, o site parece ser um site de um domínio e está pronto para a instalação normal do UTM.

Como redirecionar aliases no IIS

  • Se você estiver usando um servidor da web Microsoft IIS, a configuração poderá ser facilmente modificada para redirecionar todo o tráfego de um alias para o site principal.Uma das formas de fazer isso é criar dois sites na configuração do IIS.O primeiro será o domínio principal (www.meusite.com.br) que incluirá a configuração normal; o segundo será para todos os outros aliases (meusite.net, meusite.org, etc.) e redirecionará para o principal. No IIS Manager, clique com o botão direito do mouse em um dos sites e abra a caixa de diálogo de propriedades.Na guia "Site da web", clique no botão "Avançado....".Isso ativa uma janela em que é possível atribuir domínios adicionais ao site utilizando o campo "Cabeçalho de host".Defina o domínio principal no site principal e utilize o segundo site para hospedar todos os aliases. Quando o segundo site com todos os aliases estiver configurado e ativado, crie uma página inicial em branco com o seguinte código de redirecionamento: 

        <META HTTP-EQUIV=Refresh CONTENT="0; URL=http://www.meusite.com.br/">
  • Isso fará com que o navegador do visitante redirecione imediatamente para o URL principal.Nesse ponto, o site principal aparenta ser uma configuração simples de um domínio e a instalação normal do UTM pode continuar com as configurações padrão.
