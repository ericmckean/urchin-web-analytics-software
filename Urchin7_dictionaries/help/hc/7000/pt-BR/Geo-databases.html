Urchin 6.602 Geo-databases

O Urchin 6.602 acompanha dois bancos de dados geográficos:

  • geo.db - contém informações detalhadas da área metropolitana (country/region/city), de redes (organizações) e domínios. Ele contém o mesmo nível de informações (embora atualizadas) existentes nas versões anteriores do urchin 6 (6.400 - 6.601).

  • geo.light.db - contém apenas as informações da área metropolitana até o nível nacional; sem informações de regiões, cidades, redes e organizações. Como esse banco de dados exige muito menos memória e tempo de carregamento, use-o sempre que o desempenho do mecanismo de processamento do registro for importante. O uso desse banco de dados geográfico leve afeta os seguintes relatórios.

    • o Os seguintes relatórios de "Mapa geográfico" só mostram dados da área metropolitana até o nível nacional (sem informações da região e da cidade):

      • Visualização executiva -> Visão geral executiva (painel de controle)

      • Visualização executiva -> Sobreposição de mapa geográfico

      • Visualização do profissional de marketing -> Visão geral do profissional de marketing (painel de controle)

      • Visualização do webmaster -> Visão geral do webmaster (painel de controle)

      • Otimização de marketing -> Desempenho do segmento do visitante -> Sobreposição de mapa geográfico

      o Nenhum dado mostrado no relatório "Locais de rede" (Otimização de marketing -> Desempenho do segmento do visitante -> Local de rede)

      o Nenhum dado mostrado no relatório "Domínios" (Otimização de marketing -> Desempenho do segmento do visitante -> Domínios)

O geo.db é usado por padrão. Para alternar para o banco de dados geo.light.db, remova o comentário na linha correspondente do arquivo urchin.conf: 


# Banco de dados geográfico Configurações

# Remova o comentário da linha abaixo para ativar o banco de dados geográfico leve
# GeodbLight: Ativado
