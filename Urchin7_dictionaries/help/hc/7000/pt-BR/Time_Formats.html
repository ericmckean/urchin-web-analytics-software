Formatos personalizados de data/hora

Introdução

O Urchin pode processar praticamente qualquer formato de data ou hora contido em um arquivo de registro.A única exigência é fornecer ao Urchin o formato de data ou hora necessário que corresponda ao padrão da data contida no arquivo de registro.Este artigo descreve as variáveis usadas para especificar o formato de data e hora.Esses formatos são especificados dentro de um arquivo de formato de registro personalizado.Consulte o artigo "Formatos de registro personalizados" mais acima nesta seção.

Como funciona a análise de data/hora

O Urchin determina a data e a hora comparando um formato especificado com um ou mais campos de data/hora no arquivo de registro.

Por exemplo, um registro do IIS contém a data no seguinte formato: 2002-11-12 O Urchin é capaz de determinar o ano, o mês e o dia usando o seguinte formato: %Y-%m-%d

Como criar um formato de data/hora

Para criar um formato personalizado de data/hora, examine primeiro a ordem e o padrão dos dados de data/hora contidos no seu arquivo de registro.Em seguida, selecione uma das seguintes variáveis de Data/hora listadas abaixo para compor o formato.Os fusos horários são expressos como ajustes positivos e negativos do UTC (horário universal coordenado).

Por exemplo, se o seu arquivo de registro contiver o horário como "07:01:47", você precisará criar um padrão para corresponder isso.A primeira coisa a observar é que o padrão é horas:minutos:segundos.Observando a lista de variáveis a seguir, você perceberá que %H é a variável para horas, %M é a variável para minutos e %S é a variável para segundos.O agrupamento dessas variáveis gera um formato de "%H:%M:%S".Se você tiver um caractere '%' literal no campo de formato de data ou hora, poderá especificar o literal % como %%.

As variáveis mais comuns são:%Y, %m, %d, %H, %M e %S.

Definições de variáveis de data/hora

  • %A = representação nacional do nome completo do dia da semana.
  • %a = representação nacional do nome abreviado do dia da semana.
  • %B = representação nacional do nome completo do mês.
  • %b = representação nacional do nome abreviado do mês.
  • %d = o dia do mês como um número decimal (01-31).
  • %e = o dia do mês como um número decimal (1-31); dígitos únicos são precedidos por um espaço em branco.
  • %H = a hora (relógio de 24 horas) como um número decimal (00-23).
  • %I = a hora (relógio de 12 horas) como um número decimal (01-12).
  • %j = o dia do ano como um número decimal (001-366).
  • %k = a hora (relógio de 24 horas) como um número decimal (0-23); dígitos únicos são precedidos por um espaço em branco.
  • %l = a hora (relógio de 12 horas) como um número decimal (1-12); dígitos únicos são precedidos por um espaço em branco.
  • %M = o minuto como um número decimal (00-59).
  • %m = o mês como um número decimal (01-12).
  • %p = representação nacional de "AM" ou "PM", conforme apropriado.
  • %S = o segundo como um número decimal (00-60).
  • %s = o número de segundos desde a Época, UTC (consulte mktime(3)).
  • %w = o dia da semana (domingo como o primeiro dia da semana) como um número decimal (0-6).
  • %Y = o ano, com século, como um número decimal.
  • %y = o ano, sem século, como um número decimal (00-99).
  • %z = o ajuste de fuso horário a partir de UTC; um sinal de mais à esquerda representa o leste de UTC, enquanto um sinal de menos representa o oeste de UTC, seguido de horas e minutos com dois dígitos cada e sem delimitador entre eles (forma comum para cabeçalhos de data RFC 822).
  • %% = `%'.
