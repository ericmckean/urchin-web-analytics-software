urchin: processamento de registros do Urchin

Visão geral

O Urchin Log Processing Engine, urchin, é o principal componente de processamento de registros do Urchin. Geralmente, o mecanismo de processamento de registros é invocado a partir do Urchin Scheduler (urchind) quando uma tarefa é executada. No entanto, é possível executar o urchin diretamente de um shell de comando para executar um perfil específico. Isso é útil em ambientes que usam muito os scripts onde a execução das tarefas do Urchin é realizada a partir de uma fonte externa como o Agendador de tarefas do Windows ou cron em sistemas tipo UNIX. Também é útil para executar um perfil em circunstâncias especiais, como para processar apenas os cliques de um dia específico ou para realizar algum tipo de depuração.

O Urchin não é realmente um utilitário, mas foi documentado aqui porque fornece alguns recursos limitados de linha de comando que podem ser úteis em determinados ambientes.

Aplicação

O urchin está localizado no diretório bin da distribuição do Urchin. O uso funciona da seguinte maneira: 

urchin [options...] (-p <profile name> | -P <profile id>)

Opções: 

-a <account name>       Especifica o nome da conta do perfil. O nome da conta padrão é “(NONE)”

-D                      Ativa o modo de depuração

-d <YYYYMMDD>           Especifica a data do novo processamento

-H                      Especifica para registrar a saída da execução no arquivo histórico. Um novo registro do histórico da tarefa é criado se -H for especificado e -T não for especificado

-h                      Imprime informações de ajuda e é encerrado

-P <profile id>         Especifica o ID do perfil a ser processado

-p <profile name>       Especifica o nome do perfil a ser processado

-T <task id>            Especifica o registro do histórico da tarefa a ser atualizado. Especificar -T força o uso de -H

-t                      Imprime dados de configuração e é encerrado

-v                      Imprime o número da versão e é encerrado

Considerações

  • Em sistemas tipo UNIX, o urchin deve ser executado como o usuário/UID que instalou o Urchin para garantir que os bancos de dados do perfil são de propriedade desse UID, visto que o urchin irá criá-los se ainda não existirem.
