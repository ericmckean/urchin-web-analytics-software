Código de erro para transferências com falha de registros do servidor da web remoto FTP e HTTP

Visão geral

Uma Origem de registro do Urchin pode ser configurada de forma a coletar um registro de servidor da web a partir de um servidor remoto via FTP ou HTTP.Em circunstâncias normais, a transferência será bem sucedida e nenhum erro aparecerá no registro de tempo de execução.Entretanto, se um erro for encontrado durante a transferência (por exemplo, um nome de usuário ou uma senha inválido, um servidor remoto inacessível, um registro remoto ilegível, etc.), o Urchin registrará um código de erro na saída de tempo de execução, visualizável no Histórico de tarefas do Perfil.Esse código de erro aparece entre parênteses ao lado da mensagem "falha" após a tentativa de transferência do registro do servidor da web. Por exemplo, (-9).

Os códigos de erros são listados a seguir, junto com uma mensagem de texto explicando o problema que foi encontrado.

Lista de códigos de erro

CURLE_UNSUPPORTED_PROTOCOL (1) - O URL transmitido a libcurl usou um protocolo não suportado por esse libcurl.O suporte pode ser uma opção de tempo de compilação que não foi usada, pode ser uma string de protocolo incorretamente escrita ou apenas um protocolo para o qual libcurl não tem um código.

CURLE_FAILED_INIT (2) - Falha muito precoce do código de inicialização.Trata-se provavelmente de um erro ou problema interno.

CURLE_URL_MALFORMAT (3) - O URL não foi corretamente formatado.

CURLE_COULDNT_RESOLVE_PROXY (5) - Não foi possível resolver o proxy.O host do proxy especificado não pode ser resolvido.

CURLE_COULDNT_RESOLVE_HOST (6) - Não foi possível resolver o host.O host remoto especificado não foi resolvido.

CURLE_COULDNT_CONNECT (7) - Falha ao conectar ao host ou proxy.

CURLE_FTP_WEIRD_SERVER_REPLY (8) - Após a conexão com um servidor FTP, libcurl espera obter uma determinada resposta.Esse código de erro implica que ele recebeu uma resposta estranha ou incorreta.O servidor remoto especificado provavelmente não é um servidor FTP em bom estado.

CURLE_REMOTE_ACCESS_DENIED (9) - Acesso negado ao recurso especificado no URL.No caso do FTP, isso ocorre durante a tentativa de mudar para o diretório remoto.

CURLE_FTP_WEIRD_PASS_REPLY (11) - Após o envio da senha FTP ao servidor, libcurl espera obter uma resposta apropriada.Esse código de erro indica que um código inesperado foi retornado.

CURLE_FTP_WEIRD_PASV_REPLY (13) - libcurl não conseguiu obter um resultado sensível do servidor em resposta a um comando PASV ou EPSV.O servidor está com falhas.

CURLE_FTP_WEIRD_227_FORMAT (14) - Servidores FTP retornam uma linha 227 como resposta a um comando PASV.Se libcurl não conseguir analisar essa linha, esse código de retorno será transmitido.

CURLE_FTP_CANT_GET_HOST (15) - Falha interna ao pesquisar o host usado para a nova conexão.

CURLE_FTP_COULDNT_SET_TYPE (17) - Erro recebido ao tentar definir o modo de transferência como binário ou ascii.

CURLE_PARTIAL_FILE (18) - Uma transferência de arquivo foi mais curta ou mais longa do que o esperado.Isso acontece quando o servidor relata primeiramente um tamanho de transferência esperado e, em seguida, fornece dados que não correspondem ao tamanho previamente informado.

CURLE_FTP_COULDNT_RETR_FILE (19) - Esta foi uma resposta estranha a um comando 'RETR' ou uma transferência de zero byte concluída.

CURLE_QUOTE_ERROR (21) - Ao enviar comandos "QUOTE" personalizados ao servidor remoto, um dos comandos retornou um código de erro 400 ou maior (para FTP) ou indicou de outra forma a conclusão mal-sucedida do comando.

CURLE_HTTP_RETURNED_ERROR (22) - Esse erro será retornado se CURLOPT_FAILONERROR for definido como TRUE e o servidor HTTP retornar um código de erro >= 400.Anteriormente, esse código de erro era conhecido como CURLE_HTTP_NOT_FOUND.

CURLE_WRITE_ERROR (23) - Ocorreu um erro ao gravar os dados recebidos em um arquivo local ou um erro foi retornado para libcurl a partir de um retorno de chamada de gravação.

CURLE_UPLOAD_FAILED (25) - Falha ao iniciar o upload.No caso do FTP, o servidor negou previsivelmente o comando STOR.O buffer do erro geralmente contém a explicação do servidor sobre isso.Anteriormente, esse código de erro era conhecido como CURLE_FTP_COULDNT_STOR_FILE.

CURLE_READ_ERROR (26) - Houve um problema na leitura de um arquivo local ou um erro foi retornado pelo retorno de chamada de leitura.

CURLE_OUT_OF_MEMORY (27) - Falha em uma solicitação de alocação de memória.Se esse erro ocorrer, trata-se de uma deficiência séria e uma indicação de que as coisas realmente não estão indo bem.

CURLE_OPERATION_TIMEDOUT (28) - Tempo limite da operação.O período de tempo limite especificado foi atingido de acordo com as condições.

CURLE_FTP_PORT_FAILED (30) - O comando PORT FTP retornou um erro.Isso costuma ocorrer com mais freqüência quando você não especificou um endereço bom o suficiente para ser usado por libcurl.Consulte CURLOPT_FTPPORT.

CURLE_FTP_COULDNT_USE_REST (31) - O comando REST FTP retornou um erro.Isso nunca deverá acontecer se o servidor estiver em bom funcionamento.

CURLE_RANGE_ERROR (33) - O servidor não oferece suporte a solicitações de intervalo ou não as aceita.

CURLE_HTTP_POST_ERROR (34) - Trata-se de um erro estranho que ocorre principalmente devido a uma confusão interna.

CURLE_SSL_CONNECT_ERROR (35) - Ocorreu um problema no processo de handshake do SSL/TLS.Convém observar o buffer do erro e ler a mensagem indicada, uma vez que ela descreve melhor o problema.O problema pode estar relacionado a certificados (formatos de arquivo, caminhos, permissões) e senhas, entre outros.

CURLE_FTP_BAD_DOWNLOAD_RESUME (36) - Tentativa de retomar o FTP além do tamanho do arquivo.

CURLE_FILE_COULDNT_READ_FILE (37) - Não foi possível abrir um arquivo especificado com FILE://.A causa mais provável é que o caminho do arquivo não identifica um arquivo existente.Você verificou as permissões do arquivo?

CURLE_LDAP_CANNOT_BIND (38) - Não é possível vincular o LDAP.Falha na operação de vinculação do LDAP.

CURLE_LDAP_SEARCH_FAILED (39) - Falha na pesquisa LDAP.

CURLE_FUNCTION_NOT_FOUND (41) - Função não encontrada.Uma função zlib necessária não foi encontrada.

CURLE_ABORTED_BY_CALLBACK (42) - Anulado pelo retorno de chamada.Um retorno de chamada retornou "abort" para libcurl.

CURLE_BAD_FUNCTION_ARGUMENT (43) - Erro interno.Uma função foi chamada com um parâmetro inválido.

CURLE_INTERFACE_FAILED (45) - Erro de interface.Uma interface de saída especificada não pode ser usada.Defina que interface usar para o endereço IP de origem das conexões de saída com CURLOPT_INTERFACE.Anteriormente, esse código de erro era conhecido como CURLE_HTTP_PORT_FAILED.

CURLE_TOO_MANY_REDIRECTS (47) - Muitos redirecionamentos.Ao seguir redirecionamentos, libcurl atingiu o valor máximo.Defina seu limite com CURLOPT_MAXREDIRS.

CURLE_UNKNOWN_TELNET_OPTION (48) - Uma opção definida com CURLOPT_TELNETOPTIONS não foi reconhecida/detectada.Consulte a documentação apropriada.

CURLE_TELNET_OPTION_SYNTAX (49) - Uma string de opção telnet foi ilegalmente formatada.

CURLE_PEER_FAILED_VERIFICATION (51) - O certificado SSL ou impressão SSH md5 do servidor remoto não foi considerado OK.

CURLE_GOT_NOTHING (52) - Nada foi retornado do servidor e, nessas circunstâncias, a obtenção de nada é considerada um erro.

CURLE_SSL_ENGINE_NOTFOUND (53) - O mecanismo de criptografia especificado não foi encontrado.

CURLE_SSL_ENGINE_SETFAILED (54) - Falha ao definir o mecanismo de criptografia SSL selecionado como padrão!

CURLE_SEND_ERROR (55) - Falha ao enviar dados de rede.

CURLE_RECV_ERROR (56) - Falha ao receber dados de rede.

CURLE_SSL_CERTPROBLEM (58) - Problema com o certificado de cliente local.

CURLE_SSL_CIPHER (59) - Não foi possível usar a cifra especificada.

CURLE_SSL_CACERT (60) - O certificado do par não pode ser autenticado com certificados de CA conhecidos.

CURLE_BAD_CONTENT_ENCODING (61) - Codificação de transferência não reconhecida.

CURLE_LDAP_INVALID_URL (62) - URL LDAP inválido.

CURLE_FILESIZE_EXCEEDED (63) - Tamanho de arquivo máximo excedido.

CURLE_USE_SSL_FAILED (64) - Falha no nível de SSL FTP solicitado.

CURLE_SEND_FAIL_REWIND (65) - Ao efetuar uma operação de envio, curl precisou rebobinar os dados a serem retransmitidos, mas essa operação falhou.

CURLE_SSL_ENGINE_INITFAILED (66) - Falha ao inicializar o mecanismo SSL

CURLE_LOGIN_DENIED (67) - O servidor remoto negou o login de curl (adicionado em 7.13.1)

CURLE_TFTP_NOTFOUND (68) - Arquivo não encontrado no servidor TFTP.

CURLE_TFTP_PERM (69) - Problema de permissão no servidor TFTP

CURLE_REMOTE_DISK_FULL (70) - Sem espaço em disco no servidor.

CURLE_TFTP_ILLEGAL (71) - Operação TFTP ilegal.

CURLE_TFTP_UNKNOWNID (72) - ID de transferência TFTP desconhecido.

CURLE_REMOTE_FILE_EXISTS (73) - O arquivo já existe e não será sobregravado.

CURLE_TFTP_NOSUCHUSER (74) - Esse erro nunca deve ser retornado por um servidor TFTP em funcionamento correto.

CURLE_CONV_FAILED (75) - Falha na conversão de caracteres.

CURLE_CONV_REQD (76) - O chamador deve registrar retornos de chamada de conversão.

CURLE_SSL_CACERT_BADFILE (77) - Problema com a leitura do certificado de CA SSL (caminho?direitos de acesso?)

CURLE_REMOTE_FILE_NOT_FOUND (78) - O recurso referenciado no URL não existe.

CURLE_SSH (79) - Erro não especificado durante a sessão SSH.

CURLE_SSL_SHUTDOWN_FAILED (80) - Falha ao encerrar a conexão SSL.
