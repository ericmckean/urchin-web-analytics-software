O que significa "Erro de sessão - não foi possível chamar o aplicativo.(4002)..."?

Quando você visualiza relatórios do Urchin, o navegador consulta os arquivos de banco de dados do Urchin para localizar os dados que correspondem ao período solicitado e à visualização de relatório.Se a resposta para a consulta de dados não for recebida dentro do padrão de 45 segundos, o tempo limite será atingido, exibindo a mensagem ""Erro de sessão - não foi possível chamar o aplicativo.(4002) Faça login novamente".Quando isso acontecer, será necessário aumentar o valor do tempo limite de acordo.Para aumentar esse valor, você precisa ter acesso via linha de comandos aos arquivos de distribuição do Urchin.Se você não tiver o devido acesso, entre em contato com a pessoa que fornece os seus relatórios e indique a leitura deste artigo.

Para aumentar o tempo limite do aplicativo:

  • Abra um shell de comando para o servidor do Urchin.
  • Abra o arquivo:/path/to/urchin/etc/session.conf

  • Localize a diretiva denominada:APPLICATION_TIMEOUT:45 em que o valor é representado em segundos.

  • Aumente o valor proporcionalmente e salve o arquivo.As alterações entrarão em vigor imediatamente.
