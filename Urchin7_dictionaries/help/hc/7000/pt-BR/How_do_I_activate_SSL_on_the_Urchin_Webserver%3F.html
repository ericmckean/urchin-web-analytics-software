Como faço para ativar o SSL no servidor da web do Urchin?

O servidor da web que acompanha o Urchin é capaz de criptografar comunicações via SSL. Para ativar o SSL, você precisará ter um certificado válido assinado por uma autoridade de certificação ou um certificado auto assinado.

Para ativar o SSL no servidor da web do Urchin:

  1. Copie o arquivo de certificado SSL para o diretório var do Urchin e nomeie-o como server.crt
  2. Copie o arquivo de chave SSL para o diretório var do Urchin e nomeie-o como server.key
  3. Edite o arquivo urchinwebd.conf.template, localizado no diretório var do Urchin.
    1. Pule essa etapa se estiver usando o Urchin 6.6+. Comente a linha da Porta e exclua as barras de comentário da diretiva Listen. 
                #XXXPortXXX
          Listen 443

    2. Altere a diretiva ServerName de host local para o nome do seu servidor da web. Por exemplo: 

                ServerName: www.urchin.com

    3. Observação: a diretiva ServerName no arquivo urchinwebd.conf.template precisa corresponder ao nome do servidor que consta no arquivo de certificado.

  4. Inicie ou reinicie o servidor da web usando urchinctl com a opção -e. Urchinctl está localizado no diretório bin do Urchin. A opção -e fornece instruções a urchinctl para que ele ative o SSL no servidor da web. Por exemplo, para reiniciar o servidor da web com o SSL ativado na porta 443: Para versões anteriores ao Urchin 6.6: 
       urchinctl -e -w reiniciar
    Para o Urchin 6.6+: 
       urchinctl -e -p 443 -w reiniciar

    Observação: para iniciar o servidor sem o SSL ativado, remova a opção -e.

Agora, você pode acessar o seu servidor ativado para SSL usando https://servername.domain.com:port/

Observação: a personalização das configurações de SSL em urchinwebd.conf.template pode resultar em problemas que talvez impeçam a inicialização do servidor da web.

Instruções para Windows

Para ativar o SSL no servidor da web do Urchin:

  1. Copie o arquivo de certificado SSL para o diretório var do Urchin e nomeie-o como server.crt
  2. Copie o arquivo de chave SSL para o diretório var do Urchin e nomeie-o como server.key
  3. Edite o arquivo urchinwebd.conf.template, localizado no diretório var do Urchin.
    1. Pule essa etapa se estiver usando o Urchin 6.6+. Comente a linha da Porta e exclua as barras de comentário da diretiva Listen. 
                #XXXPortXXX
          Listen 443
    2. Exclua as barras de comentário da linha a seguir para carregar mod_ssl.so quando o servidor da web for iniciado: 
                LoadModule ssl_module etc/mod_ssl.so

    3. Altere a diretiva ServerName de host local para o nome do seu servidor da web. Por exemplo: 

                ServerName: www.urchin.com

    4. Observação: a diretiva ServerName no arquivo urchinwebd.conf.template precisa corresponder ao nome do servidor que consta no arquivo de certificado.

  4. Remova e instale o serviço do servidor da web usando urchinctl com a opção -e. Urchinctl está localizado no diretório bin do Urchin. A opção -e fornece instruções a urchinctl para que ele ative o SSL no servidor da web. Por exemplo, para remover e instalar o servidor da web com o SSL ativado na porta 443: Para versões anteriores ao Urchin 6.6: 
       urchinctl.exe -w remover
   urchinctl.exe -w -e instalar
    Para o Urchin 6.6+: 
       urchinctl.exe -w remover
   urchinctl.exe -e -p 443 -w instalar

    Observação: para iniciar o servidor sem o SSL ativado, remova a opção -e.

Agora, você pode acessar o seu servidor ativado para SSL usando https://servername.domain.com:port/

Observação: a personalização das configurações de SSL em urchinwebd.conf.template pode resultar em problemas que talvez impeçam a inicialização do servidor da web.
