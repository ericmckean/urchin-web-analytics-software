Gerenciamento de registros

Visão geral

O gerenciamento de registros é uma questão importante para a execução de softwares como o Urchin.Como os sites de tráfego intenso geram arquivos de registro grandes com extrema rapidez (em alguns casos, chegando a vários gigabytes em questão de um mês), o gerenciamento de registros deve ser cautelosamente considerado.Convém estabelecer uma prática padrão de rodízio de registros.A compactação e outras formas de arquivamento de arquivos off-line são práticas padrão.Consulte Práticas recomendadas para rodízio de registros para obter informações adicionais sobre como estabelecer esse procedimento.O gerenciamento de registros somente é necessário para considerações de uso de recursos de disco e não para o propósito de evitar o reprocessamento de dados. O Urchin não precisa de nenhum tipo de rodízio de registros para evitar a duplicação de dados, pois ele conta com um recurso de controle de registro que garante que os dados de registro previamente lidos não sejam reprocessados.Como o Urchin nunca precisará repetir a leitura de um arquivo de registro após o processamento, ao seu próprio critério, você pode optar por excluir os registros após a execução de cada processamento.Entretanto, por motivos históricos ou para fins de auditoria, não é raro manter registros antigos durante um período de tempo especificado.

Como gerenciar registros através do Urchin

Cada Origem do registro possui uma configuração "Destino do registro" com as opções "Não toque", "Arquivar/compactar" e "Excluir" ("Configuração" -> "Perfis do Urchin" -> "Gerenciador de registro" -> "Editar" (ao lado de qualquer perfil) -> guia "Configurações avançadas").Quando todos os Perfis que estiverem usando uma Origem do registro tiverem finalizado o processamento, o Urchin irá usar a configuração "Destino do registro" para determinar a disposição da Origem do registro.A configuração "Destino do registro" pode ser acessada na guia "Configurações avançadas" de uma determinada Origem de registro.Convém definir o Destino do registro como "Arquivar/compactar", para que você possa poupar espaço em disco se quiser manter registros por um certo período de tempo.Se você não se importar com o fato de que um registro será removido depois de ser processado, poderá escolher "Excluir" como Destino do registro.Entretanto, lembre-se de que isso significa que, no futuro, não será possível executar novamente o Urchin com base nesse registro, a não ser que você possua um backup armazenado em outro local.

Considerações

Algumas situações especiais devem ser observadas:

  • Não use as opções "Arquivar" ou "Excluir" com uma Origem do registro se você estiver processando registros dinâmicos.Um registro dinâmico é um registro ativamente gravado por um servidor da web.O uso dessa configuração com um registro dinâmico resultará na perda dos dados.
  • Se o Destino do registro para uma Origem do registro recuperada remotamente estiver definido como "Não toque", esse registro irá crescer continuamente, a não ser que haja algum processo externo ao Urchin responsável pelo gerenciamento de registros na máquina em que o registro foi criado.Como o Urchin deve transferir uma cópia do arquivo de registro remoto para o sistema local antes do processamento, à medida que esse arquivo crescer, o Urchin irá demorar cada vez mais para concluir a sua transferência.Conseqüentemente, isso irá prolongar o tempo de execução geral do Urchin.
