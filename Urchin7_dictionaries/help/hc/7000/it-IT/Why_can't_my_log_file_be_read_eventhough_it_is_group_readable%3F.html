Perché il mio file di log non può essere letto anche se è leggibile dai gruppi?

Il numero di errore "7026-76-*" che genera un file di log che non può essere letto pur essendo leggibile dai gruppi è un problema noto relativo alla funzionalità setuid/setgid del file binario urchin. Per motivi di sicurezza, il motore di urchin viene eseguito come proprietario del file binario anche se viene avviato come root. In queste condizioni, l'applicazione vede solo l'esatto ID del gruppo e non l'elenco di gruppi.

Per risolvere il problema potrebbe essere necessario modificare le autorizzazioni del file di log impostandole su leggibile da tutti, oppure modificare il gruppo in modo che corrisponda esattamente al gruppo dell'utente urchin. Questo problema riguarda solo le architetture di tipo UNIX.
