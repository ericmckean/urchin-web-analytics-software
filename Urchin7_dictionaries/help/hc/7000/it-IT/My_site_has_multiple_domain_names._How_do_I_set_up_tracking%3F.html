Il mio sito contiene più nomi di dominio. Come faccio a impostare il monitoraggio?

Se il tuo sito utilizza un carrello degli acquisti sicuro ubicato su un dominio principale differente (ad esempio, www.mionegozio.it e www.securecart.com) o impiega più alias di dominio per indirizzare allo stesso contenuto, devi apportare alcune modifiche per assicurarti che i tuoi siti e UTM siano compatibili.

  • Tieni presente che c'è differenza tra più sottodomini e più domini principali. www.esempio.it e carrellosicuro.esempio.it sono esempi di più sottodomini, mentre www.esempio.it e www.campione.it rappresentano più domini principali. Se il tuo sito contiene più sottodomini, consulta le sezioni: Come faccio a monitorare tutti i sottodomini del mio sito in un unico profilo? Come faccio a monitorare tutti i sottodomini del mio sito in profili separati? per ulteriori informazioni sui dati per la configurazione.

Più nomi di dominio: alias di dominio

Se il tuo sito utilizza più alias di dominio, è consigliabile reindirizzare il traffico dagli alias al sito principale. Ciò garantisce che il monitoraggio dei visitatori UTM venga impostato sotto il dominio principale e che tutti i visitatori siano monitorati in modo coerente.

Le seguenti istruzioni forniscono un esempio di come reindirizzare i domini con alias al dominio principale nei server Apache e IIS.

Reindirizzamento degli alias in Apache

  • Se utilizzi un server web Apache, la configurazione può essere facilmente modificata in modo da reindirizzare al sito principale tutto il traffico originato sotto uno degli alias. Un modo per effettuare questa operazione è di creare due voci VirtualHost. Il primo sarà il dominio principale che includerà la configurazione normale; il secondo VirtualHost sarà per tutti gli alias e reindirizzerà al dominio principale: 

        #---primary virtualhost
        Nome server www.miosito.it
        Alias server miosito.it
        ...
        #---second virtualhost
        Nome server miosito.org
        Alias server ww.miosito.org miosito.net www.miosito.net
        RewriteEngine on
        RewriteRule ^(.*) http://www.miosito.it$1 [R=301]

  • La seconda voce VirtualHost utilizza una regola di riscrittura con un codice di reindirizzamento 301 (trasferito in modo permanente) per inoltrare tutto il traffico al sito originale. A questo punto, per quanto riguarda UTM, il sito appare come un sito con un dominio ed è pronto per l'installazione UTM standard.

Reindirizzamento degli alias in IIS

  • Se utilizzi un server web Microsoft IIS, la configurazione può essere facilmente modificata in modo da reindirizzare al sito principale tutto il traffico originato sotto uno degli alias. Un modo per effettuare questa operazione è di creare due siti web nella configurazione IIS. Il primo sarà il dominio principale (www.miosito.it), che includerà la configurazione normale; il secondo sarà per tutti gli altri alias (miosito.net, miosito.org ecc.) e reindirizzerà al dominio principale. In IIS Manager fai clic con il pulsante destro del mouse su uno dei siti web e visualizza la finestra di dialogo delle proprietà. Nella scheda Sito web fai clic sul pulsante "Avanzato...". Viene visualizzata la finestra in cui è possibile assegnare ulteriori domini al sito web utilizzando il campo "Intestazione host". Imposta il dominio principale nel sito web principale e utilizza il secondo sito per ospitare tutti gli alias. Una volta configurato e abilitato il secondo sito web che ospita tutti gli alias, crea una home page vuota con il seguente codice di reindirizzamento: 

        <META HTTP-EQUIV=Refresh CONTENT="0; URL=http://www.miosito.it/">
  • In questo modo il browser del visitatore effettuerà immediatamente il reindirizzamento all'URL principale. A questo punto, il sito web principale appare come una semplice configurazione a singolo dominio ed è possibile eseguire l'installazione UTM standard con le impostazioni predefinite.
