Utilità di controllo dei servizi di Urchin (urchinctl)

Panoramica

L'Urchin Services Control utility, urchinctl, verifica lo stato dei servizi Urchin Scheduler e Urchin Webserver e consente di avviare e arrestare questi servizi. Di solito, nei sistemi UNIX l'utilità urchinctl viene richiamata da uno degli script di avvio del sistema per attivare o disattivare automaticamente i servizi Urchin.

L'utilità urchinctl consente di effettuare le seguenti operazioni:

  • Verifica dello stato del programma di pianificazione master e slave di Urchin6.
  • Avvio, arresto o riavvio del programma di pianificazione o del server web (o di entrambi).
  • Avvio del server web su una porta alternativa.
  • Avvio del server web con crittografia SSL.

Utilizzo

L'utilità urchinctl è ubicata nella directory bin della distribuzione di Urchin.

L'utilità può essere utilizzata nei seguenti modi: 

  urchinctl -h (consente di stampare il messaggio sull'utilizzo e di uscire)

  urchinctl -v (consente di stampare la versione e di uscire)

  urchinctl [-e] [-p port] [[-s] | [-w]] azione

dove: 

  -e attiva la crittografia (SSL) nel server web

  -p specifica la porta di ascolto del server web

  -s esegue l'azione SOLO sul programma di pianificazione di Urchin

  -w esegue l'azione SOLO sul server web Urchin

e l'azione è una delle seguenti: 

  start (avvia il servizio o i servizi)

  stop (arresta il servizio o i servizi)

  restart (arresta e quindi riavvia il servizio o i servizi)

  status (visualizza lo stato di runtime del server web/programma di pianificazione)

Per impostazione predefinita, l'azione viene eseguita sia sul server web che nel programma di pianificazione, a meno che non siano specificate le opzioni "-s" (programma di pianificazione) oppure "-w" (server web). Tieni presente che queste opzioni si escludono reciprocamente.

Considerazioni

Nei sistemi UNIX, urchinctl deve essere eseguita come utente/UID di installazione di Urchin per assicurarsi che i processi urchinwebd e urchind siano avviati come questo UID.
