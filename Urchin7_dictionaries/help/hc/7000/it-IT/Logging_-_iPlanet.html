Registrazione: iPlanet

Panoramica

Questo articolo fornisce una breve panoramica su come configurare la registrazione per un server web iPlanet per eseguire elaborazioni e creare rapporti in modo corretto per Urchin.

Per l'impostazione della sorgente log utilizza il tipo "Netscape".

Per la produzione di rapporti in Urchin esiste un insieme minimo di campi obbligatori. Sono:

  • Data
  • Ora
  • Nome host o indirizzo IP del sistema richiedente
  • Richiesta (ad esempio quale documento ha richiesto il sistema al tuo server web)
  • Codice di stato generato dalla richiesta (valore numerico)
  • Byte (byte trasferiti dal server al client)

Per creare rapporti completi sono inoltre necessari i seguenti campi:

  • Referral
  • User Agent
  • Cookie (se Urchin Traffic Monitor Ã¨ installato nel tuo sito)

Configurazione 

Init fn=flex-init access="$accesslog"
format.access="%Ses->client.ip% - %Req->vars.auth-user%
[%SYSDATE%] \"%Req->reqpb.clf-request%\"
%Req->srvhdrs.clf-status% %Req->srvhdrs.content-length%
\"%Req->headers.user-agent%\"
\"%Req->headers.referer%\"
\"%Req->headers.cookie%\""
