Come faccio ad attivare SSL sul server web Urchin?

Il server web Urchin fornito con Urchin è in grado di crittografare la comunicazione via SSL. Per attivare SSL, devi avere un certificato valido firmato da un'autorità di certificazione oppure un'autocertificazione.

Per attivare SSL nel server web Urchin:

  1. Copia il file del certificato SSL in tuo possesso nella directory var di Urchin e assegnagli il nome server.crt
  2. Copia la chiave SSL in tuo possesso nella directory var di Urchin e assegnagli il nome server.key
  3. Modifica il file urchinwebd.conf.template che si trova nella directory var Urchin.
    1. (Salta questo passaggio se utilizzi Urchin 6.6 o versioni successive). Commenta la riga Port ed elimina il commento dalla direttiva Listen. 
                #XXXPortXXX
          Listen 443

    2. Modifica la direttiva ServerName da localhost al nome del tuo server web. Ad esempio: 

                ServerName: www.urchin.com

    3. NOTA. Il ServerName che compare nel file urchinwebd.conf.template deve essere identico al nome del server indicato nel file di certificato.

  4. Avvia o riavvia il server web utilizzando urchinctl con l'opzione -e. Urchinctl si trova nella directory bin di Urchin. L'opzione -e istruisce urchinctl affinché attivi SSL nel server web. Ad esempio, per riavviare il server web con SSL attivato nella porta 443: Per le versioni precedenti a Urchin 6.6: 
       urchinctl -e -w restart
    Per Urchin 6.6 e versioni successive: 
       urchinctl -e -p 443 -w restart

    Nota. Per avviare il server web senza SSL attivato, rimuovi l'opzione -e dal comando.

Ora puoi accedere al server con SSL attivato utilizzando https://servername.domain.com:port/

Nota. La personalizzazione delle impostazioni SSL nel file urchinwebd.conf.template può dare luogo a problemi in grado di impedire l'avvio del server web.

Istruzioni Windows

Per attivare SSL nel server web Urchin:

  1. Copia il file del certificato SSL in tuo possesso nella directory var di Urchin e assegnagli il nome server.crt
  2. Copia la chiave SSL in tuo possesso nella directory var di Urchin e assegnagli il nome server.key
  3. Modifica il file urchinwebd.conf.template che si trova nella directory var Urchin.
    1. (Salta questo passaggio se utilizzi Urchin 6.6 o versioni successive). Commenta la riga Port ed elimina il commento dalla direttiva Listen. 
                #XXXPortXXX
          Listen 443
    2. Rimuovi il commento dalla seguente riga per caricare mod_ssl.so all'avvio del server web: 
                LoadModule ssl_module etc/mod_ssl.so

    3. Modifica la direttiva ServerName da localhost al nome del tuo server web. Ad esempio: 

                ServerName: www.urchin.com

    4. NOTA. Il ServerName che compare nel file urchinwebd.conf.template deve essere identico al nome del server indicato nel file di certificato.

  4. Rimuovi e installa il servizio server web utilizzando urchinctl con l'opzione -e. Urchinctl si trova nella directory bin di Urchin. L'opzione -e istruisce urchinctl affinché attivi SSL nel server web. Ad esempio, per riavviare e installare il server web con SSL attivato nella porta 443: Per le versioni precedenti a Urchin 6.6: 
       urchinctl.exe -w remove
   urchinctl.exe -w -e install
    Per Urchin 6.6 e versioni successive: 
       urchinctl.exe -w remove
   urchinctl.exe -e -p 443 -w install

    Nota. Per avviare il server web senza SSL attivato, rimuovi l'opzione -e dal comando.

Ora puoi accedere al server con SSL attivato utilizzando https://servername.domain.com:port/

Nota. La personalizzazione delle impostazioni SSL nel file urchinwebd.conf.template può dare luogo a problemi in grado di impedire l'avvio del server web.
