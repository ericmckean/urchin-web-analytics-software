Guida all'installazione di Urchin 6 (Windows)

Questo articolo è stato scritto da EpikOne, consulente autorizzato del software Urchin

Prima di iniziare - Nota importante per gli utenti di Urchin 6.500

Urchin 6.501 ha corretto alcuni problemi nel programma di pianificazione CPC. Se stai eseguendo l'upgrade da Urchin 6.500 a Urchin 6.602, i dati CPC elaborati potrebbero essere errati o incompleti, generando di conseguenza risultati errati nei rapporti Urchin. Per correggere i dati di Google AdWords, ti consigliamo di (a) installare Urchin 6.602 in un nuovo percorso oppure, (b) se desideri installarlo su un'istanza 6.500 già esistente, di eliminare i dati CPC obsoleti. In quest'ultimo caso, elimina tutti i contenuti delle seguenti directory: 

<path_to_urchin_6500_installation>\data\cpc\...
<path_to_urchin_6500_installation>\data\history\cpc\...

Non devi invece eseguire alcuna azione se stai eseguendo l'upgrade da Urchin 6.600, 6.501 o da Urchin 6.402 e versioni precedenti.

Panoramica della procedura di installazione

  • Installazione del server del database desiderato nel sistema Urchin o aggiunta di un database Urchin a un server esistente. Motori database attualmente supportati:
    • MySQL 5.0.3 o successivi (Nota. A partire dalla versione Urchin 6.6, MySQL 4.x.x non è più supportato)
    • PostgreSQL 8.2.5 o versioni successive
  • Creazione di database e utente Urchin nel server del database e assegnazione delle autorizzazioni di accesso
  • Decompressione della distribuzione Urchin in una cartella temporanea
  • Esecuzione del programma di installazione MSI
  • Procedura di licenza per l'installazione di Urchin

Opzione 1. Uso del database MySQL esistente con Urchin

  • Se non hai ancora installato un server MySQL nel sistema, puoi ottenere una distribuzione alla pagina http://dev.mysql.com/downloads/. Il pacchetto Windows Essentials contiene tutti i componenti necessari per l'utilizzo di Urchin.

  • Installa il server MySQL seguendo le istruzioni fornite sul sito MySQL.
  • Assicurati di disporre del login (ad es. root) e della password amministratore del database MySQL.
  • Scegli il nome del database, la password e il nome utente MySQL locale da utilizzare con Urchin. Salva queste informazioni, che dovranno essere utilizzate successivamente durante l'installazione di Urchin. Nel seguente esempio vengono creati un database denominato urchin e un nome utente MySQL di urchin associato alla password urchinpassword (che ovviamente potrà essere sostituita da una password personalizzata a tua scelta).

  • Avvia il client della riga di comando di MySQL utilizzando Start -> Programmi -> MySQL -> MySQL Server -> MySQL Command Line Client.

  • Nella finestra del client a riga di comando MySQL crea il database Urchin, l'utente e le autorizzazioni con i seguenti comandi: 

mysql> create database urchin character set utf8;
mysql> GRANT ALL ON urchin.* to 'urchin'@'localhost' IDENTIFIED BY
'urchinpassword';

Opzione 2. Uso del database PostgreSQL esistente con Urchin

  • Se non hai ancora installato un server PostgreSQL nel sistema, puoi ottenere una distribuzione alla pagina http://www.postgresql.org.

  • Installa il server PostgreSQL seguendo le istruzioni fornite sul sito PostgreSQL.
  • Assicurati di disporre del login (ad es. postgres) e della password amministratore del database PostgreSQL.
  • Scegli il nome del database, la password e il nome utente PostgreSQL locale da utilizzare con Urchin. Salva queste informazioni, che dovranno essere utilizzate successivamente durante l'installazione di Urchin. Nel seguente esempio vengono creati un database chiamato urchin e un nome utente PostgreSQL di urchin associato alla password urchinpass (che ovviamente potrà essere sostituita da una password personalizzata a tua scelta).

  • Avvia il client a riga di comando PostgreSQL utilizzando la sequenza Start -> Programmi -> PostgreSQL 8.2 -> psql to 'postgres'.

  • Nella finestra del client a riga di comando PostgreSQL crea il database Urchin, l'utente e le autorizzazioni con i seguenti comandi: 

postgres=# CREATE DATABASE urchin WITH ENCODING 'UTF8';
postgres=# CREATE USER urchin WITH PASSWORD 'urchinpass';
postgres=# GRANT ALL ON DATABASE urchin TO urchin;
postgres=# \connect urchin;
postgres=# CREATE LANGUAGE 'PLPGSQL';

Opzione 3. Uso del server PostgreSQL incluso nel bundle insieme a Urchin

  • Il programma di installazione di Urchin 6.602 viene fornito in bundle insieme al server PostgreSQL come opzione durante l'installazione di Urchin.

  • Per poter utilizzare questa opzione, il programma di installazione di Urchin deve essere richiamato con privilegi di amministratore. Il programma di installazione ti chiederà di specificare il numero di porta e la password per l'utente "postgres". Nota. Se l'utente "postgres" è già presente nel sistema, dovrà essere inserita la password esistente. Se l'utente "postgres" non è già presente nel sistema, verrà creato insieme alla password specificata.

  • Il server PostgreSQL incluso nel bundle presenta le seguenti limitazioni:
    • Non deve essere installato in modalità upgrade.
    • Per l'installazione sono richiesti privilegi di amministratore.

Installazione di Urchin

  • Decomprimi il file ZIP della distribuzione Urchin in una cartella temporanea.

  • Se desideri installare il server PostgreSQL incluso nel bundle, assicurati che non sia già presente un server PostgreSQL sul sistema. Per eseguire l'installazione, seleziona l'opzione "Installa PostgreSQL" e specifica dettagli del server PostgreSQL quali numero di porta e password utente "postgres". Nota. Consulta http://www.postgresql.org in caso di eventuali problemi nell'installazione del server PostgreSQL incluso nel bundle.

  • Se desideri utilizzare un database esistente, assicurati di disporre del nome del database Urchin, nonché del nome utente e della password per il server MySQL o PostgreSQL selezionato in precedenza.
  • Urchin 6 consente una sola istanza di Urchin 6 sullo stesso computer (solo Windows). Se sul computer è già installata una versione di Urchin, è consentito eseguire esclusivamente l'upgrade (non è consentito il downgrade).
  • Per eseguire una 'nuova' installazione, rimuovi ogni eventuale versione di Urchin 6 installata sul computer. Per rimuovere Urchin, procedi nel seguente modo:

    • Start -> Programmi -> Urchin 6 -> Disabilita servizi Urchin

    • Start -> Impostazioni -> Pannello di controllo -> Installazione applicazioni

    • Fai clic su "Software Urchin di Google" e seleziona Rimuovi
    • Una volta che Urchin è stato rimosso, rinomina la cartella di installazione di urchin (ad es. da C:\Program Files\Urchin6 a Urchin6-OLD).
  • Urchin 6.602 può essere utilizzato esclusivamente per eseguire l'upgrade di alcune versioni precedenti di Urchin 6, tra cui, in particolare:
    • Urchin 6.6 - Service Pack 1 : Versione 6.601
    • Urchin 6.6 - Versione in inglese : Versione 6.600
    • Urchin 6.5 - Service Pack 2 : Versione 6.501
    • Urchin 6.5 : Versione 6.500 (vedi nota importante qui sopra)
    • Urchin 6 - Service Pack 1 : Versione 6.402
    • Urchin 6 - Versione internazionale: Versione 6.401
    • Urchin 6 - Versione in inglese : Versione 6.400
  • Per eseguire operazioni di upgrade e di interruzione/disinstallazione, non è richiesta l'istanza Urchin 6 esistente. Il programma di installazione di Urchin 6.602 verifica innanzitutto la presenza di eventuali installazioni di Urchin 6 e, all'occorrenza, le rimuove. Successivamente, installa i file e le configurazioni di Urchin 6.602.
  • Per avviare l'installazione di Urchin 6.602, fai doppio clic sul file urchin6602_windows_installer.msi e segui le istruzioni interattive.

  • Nota. Se hai installato una versione precedente di Urchin 6 e desideri conservarne la configurazione, assicurati che l'opzione "Ricrea le tabelle database specifiche di Urchin ESISTENTI (se presenti)" sia deselezionata. In questo caso, Urchin mantiene inalterata la configurazione esistente, tuttavia potrebbe essere necessario creare 'tabelle aggiuntive' o 'colonne nelle tabelle esistenti'.

  • Adesso la versione di Urchin dovrebbe essere in esecuzione sul sistema. Per completare l'installazione, utilizza il tuo browser web per visualizzare il seguente URL: 

http://yourserver:<port>

dove <port> rappresenta la porta di rete che hai scelto durante l'installazione (il valore predefinito è 9999).

  • Per ripristinare i dati contenuti nei rapporti del profilo da una precedente versione di Urchin 6, copia la cronologia e le cartelle dei rapporti da C:\Program Files\Urchin6-OLD\data in C:\Program Files\Urchin6
  • Fai clic sul link Richiedi licenza demo e segui la procedura per attivare Urchin ed eseguire i passaggi iniziali dell'installazione.

Configurazione del monitoraggio

Una volta installato Urchin, potrebbero essere necessari ulteriori passaggi per impostare il monitoraggio sul tuo sito web, a seconda del tuo specifico caso.

Caso 1. Uso di Urchin con il monitoraggio IP+User-Agent

Non è necessaria una configurazione aggiuntiva. Quando aggiungi profili a Urchin, assicurati di specificare IP+User-Agent come metodo di monitoraggio dei visitatori.

Caso 2. Utilizzo di Urchin con il monitoraggio UTM

Per utilizzare Urchin con il monitoraggio UTM, è necessario installare una porzione di codice di monitoraggio javascript in ogni pagina del sito web. La procedura è la seguente:

  • Copia i file urchin.js e __utm.gif dalla directory util\utm della distribuzione Urchin nella root dei documenti del tuo sito web (ad esempio, la directory di livello superiore dei tuoi contenuti)

  • In ciascuna pagina del sito web inserisci, subito dopo i META tag nella sezione HEAD, il seguente codice di monitoraggio: 

<script src="/urchin.js" type="text/javascript">
</script>
<script type="text/javascript">
  _userv=0;
  urchinTracker();
</script>
  • Quando aggiungi profili a Urchin, assicurati di specificare UTM (Urchin Traffic Monitor) come metodo di monitoraggio dei visitatori

Caso 3. Utilizzo di Urchin in contemporanea con Google Analytics

Tieni presente che le nuove funzioni di monitoraggio nel codice di monitoraggio "ga.js" di Google Analytics non sono compatibili con Urchin 6. Se desideri monitorare il sito web utilizzando sia Google Analytics che il software Urchin, ti consigliamo di utilizzare il codice di monitoraggio urchin.js-based per ottimizzare le prestazioni. Se scegli di utilizzare ga.js, non sarai in grado di utilizzare le nuove funzioni di monitoraggio di ga.js.

Per utilizzare Urchin con un sito web che viene già monitorato da Google Analytics, basta apportare una leggera modifica al codice di monitoraggio esistente di Google Analytics. La procedura è la seguente:

  • Copia solo il file __utm.gif dalla directory util\utm della distribuzione Urchin nella root dei documenti del sito web (ad esempio, la directory di livello superiore dei contenuti).

  • Aggiungi la riga _userv=2; (come indicato di seguito) al codice di monitoraggio esistente nelle pagine del sito web. Assicurati di non modificare altre parti del codice di monitoraggio esistente. 

<script src="http://www.google-analytics.com/urchin.js"
type="text/javascript">
<script type="text/javascript">
  _userv=2;
  _uacct="UA-XXXXXXX-Y";
  urchinTracker();
</script>
  • Quando aggiungi profili a Urchin, assicurati di specificare UTM (Urchin Traffic Monitor) come metodo di monitoraggio dei visitatori

Passaggi di installazione supplementari per Windows Vista e Server 2008

Oltre che con le versioni di Windows già supportate (2000, XP e Server 2003), Urchin 6.6+ è compatibile anche con Windows Vista e Server 2008. Alcune funzioni di sicurezza di queste nuove versioni di Windows possono influire sul funzionamento di Urchin. Di seguito viene spiegato in che modo.

UAC - Problema

Windows 2008 offre una nuova funzione di sicurezza, chiamata User Access Control (UAC). Per maggiori dettagli, consulta la pagina http://technet.microsoft.com/en-us/library/cc709691.aspx.

I problemi noti che si presentano quando il modulo UAC è attivato sono:

  1. . Se UAC è attivato, l'applicazione non è in grado di eseguire correttamente le operazioni di scrittura nella cartella "Programmi". Per maggiori dettagli, consulta la pagina: http://msdn.microsoft.com/en-us/library/aa905330.aspx

  2. . Per impostazione predefinita, gli eseguibili di Urchin non possono essere eseguiti da un prompt dei comandi. Devi utilizzare il prompt dei comandi come amministratore oppure impostare le proprietà dell'eseguibile nel seguente modo: Fai clic con il pulsante destro del mouse e seleziona Proprietà-> Compatibilità -> Esegui questo programma come amministratore -> OK. Tieni presente che i servizi Windows di Urchin funzionano come previsto senza dover eseguire ulteriori azioni.

UAC - Soluzione

  1. Disattiva UAC e installa Urchin in un percorso a tua scelta. Esegui Urchin utilizzando i file binari o i servizi Windows.
  2. Se UAC è attivato, NON installare Urchin nella cartella "Programmi". Installalo, ad esempio, nella directory Utenti. Modifica le proprietà per eseguire i file binari in modalità amministratore (come descritto al punto 2 sopra) oppure utilizza i servizi Windows senza apportare modifiche.
