Come faccio a configurare Urchin in modo che certe parole chiave o URL vengano considerati come referral diretti anziché parole chiave?

Puoi fare in modo che certe parole chiave o URL vengano considerati come referral diretti anziché parole chiave. È possibile, infatti, configurare il file urchin.js in modo che Urchin registri questi dati nei rapporti come referral diretti.

  1. Utilizza un editor di testo per aprire il file urchin.js.
  2. Scorri fino alla seguente intestazione: 

          //-- Auto/Organic Keywords to Ignore
  1. Aggiungi una nuova parola chiave organica da ignorare in fondo all'elenco nel seguente formato: 

          _uOno[#]="keyword";
  • ..come indicato nell'esempio seguente: 

        //-- Auto/Organic Keywords to Ignore
        var _uOno=new Array();
        _uOno[0]="urchin";
        _uOno[1]="urchin.com";
        _uOno[2]="www.urchin.com";

Perché si verifica il timeout del browser quando visualizzo i rapporti?

Il timeout dell'applicazione che invia le query al database Urchin e produce i rapporti si verifica dopo 45 secondi dalla mancata restituzione dei dati. A seconda delle dimensioni del database, è possibile che 45 secondi non siano sufficienti per recuperare tutti i dati del rapporto richiesti. In questi casi si verifica il timeout e viene visualizzato un errore.

Se hai accesso al server in cui risiede Urchin con privilegi appropriati, puoi aumentare il valore di timeout.

Aumento del valore di timeout dell'applicazione

  • Accedi al server Urchin utilizzando un pannello di comandi.
  • Naviga alla directory .../[urchin-install-dir]/etc/.
  • Utilizzando un editor di testo, apri "session.conf".
  • Aumenta il valore di 'APPLICATION_TIMEOUT:' (in secondi).
  • Salva il file. Le modifiche verranno applicate immediatamente.
