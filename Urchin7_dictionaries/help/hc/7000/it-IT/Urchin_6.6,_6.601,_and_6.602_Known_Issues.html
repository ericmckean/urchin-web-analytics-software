Problemi noti di Urchin 6.6, 6.601 e 6.602

Urchin 6.602 - Revisione: 23 ottobre 2009

Amministrazione

  • La procedura guidata Configurazione dell'obiettivo utilizza la tecnologia di overlay del sito che non è in grado di gestire le pagine protette da password e viene facilmente confusa dalle pagine più complesse, ad esempio le pagine con frame, javascript, css, reindirizzamenti automatici ecc.

Dati geografici

  • A causa del nuovo formato del database dei dati geografici, non è più possibile importare dati per la mappatura da IP locale a nome di dominio.

Elaborazione log

  • L'elaborazione dei log non in ordine cronologico molto probabilmente produrrà un'elaborazione molto lenta per i siti con elevati volumi di traffico.
  • Aumentare il limite del DB oltre il valore predefinito di 100.000 record può ridurre le prestazioni di elaborazione dei log, in particolare per i siti con grandi volumi di traffico.
  • Il motore di elaborazione dei log assegna una quantità di memoria a ciascun filtro applicato a un profilo per ogni file di log fisico aperto durante l'elaborazione. Ciò può provocare richieste di assegnazione di grandi quantità di memoria ed eventualmente errori nell'assegnazione di tutta la memoria richiesta quando un profilo è configurato con molti filtri e una gran numero di file di log fisici sono abbinati alle sorgenti di log, ad esempio a seguito di una corrispondenza troppo estesa generata da un carattere jolly.
  • Tabelle di ricerca molto grandi possono richiedere l'assegnazione di grandi quantità di memoria ed eventualmente provocare errori di assegnazione della memoria, specialmente se durante l'elaborazione dei log viene aperto un gran numero di file di log fisici.

  • Vi sono alcuni problemi noti per l'utilizzo dei socket UNC. Per maggiori dettagli, consulta Uso delle sorgenti log.

Programmi di pianificazione

  • I programmi di pianificazione di Urchin (profilo e CPC) utilizzano il database per memorizzare e recuperare gli indicatori di ora e data al fine di pianificare le varie pianificazioni. Ciò significa che il database, una volta configurato, non deve essere spostato da un fuso orario all'altro, ovvero non si deve cambiare il fuso orario del computer su cui è in esecuzione urchin db. Inoltre, si consiglia vivamente di non cambiare il fuso orario del computer su cui sono in esecuzione di programmi di pianificazione di Urchin.

E-commerce

  • Urchin può leggere i dati e-commerce che sono incorporati direttamente nei log del server web se le transazioni sono registrate nei log tramite chiamate UTM javascript oppure tramite un log ELF2 e-commerce esplicito creato dal carrello degli acquisti. Urchin creerà transazioni duplicate se gli stessi dati e-commerce sono presenti sia nel log del server web sia nel log ELF2, pertanto è importante che i log ELF2 non vengano elaborati se gli stessi dati sono incorporati anche nei log del server web.

Utilità

  • La funzionalità "zero day" in udb-sanitizer può rivelarsi molto lenta per i profili con grandi quantità di dati.

Rapporti

  • I rapporti cross-segment possono essere molto lenti (oppure andare completamente in timeout) per i siti con grandi quantità di traffico e/o intervalli di date molto lunghi.
  • I risultati di "P/Visite" differiscono tra il rapporto "Conversione mezzi" e ciascun sottorapporto in "Opzioni di analisi".

Integrazione con AdWords

  • L'accesso diretto ad AdWords da Urchin non funziona se Urchin 6.6+ è installato su una versione a 64-bit di Windows 2008.

Installazione di PostgreSQL

  • Durante l'installazione del server PostgreSQL su Linux Kernel 2.6 e Windows, il programma di installazione richiede i privilegi root/admin per creare l'utente di sistema "postgres".
  • Su Linux, Il programma di installazione di PostgreSQL richiede glibc 2.3.3 o versione successiva.
  • L'importazione dei dati CPC potrebbe non funzionare se viene selezionato il server PostgreSQL in bundle durante l'installazione di Urchin su Linux Kernel 2.4. In questo caso, consigliamo l'utilizzo di un server PostgreSQL esterno.

Urchin 6.601 - Revisione: 23 luglio 2009

Amministrazione

  • In modalità Data Center, gli amministratori di account dispongono di un solo livello di privilegi, che è più restrittivo dei livelli di privilegio Amministratore affiliato disponibili in Urchin 5.
  • La procedura guidata Configurazione dell'obiettivo utilizza la tecnologia di overlay del sito che non è in grado di gestire le pagine protette da password e viene facilmente confusa dalle pagine più complesse, ad esempio le pagine con frame, javascript, css, reindirizzamenti automatici ecc.

Dati geografici

  • A causa del nuovo formato del database dei dati geografici, non è più possibile importare dati per la mappatura da IP locale a nome di dominio.

Elaborazione log

  • Per mantenere prestazioni elevate, il motore di elaborazione dei log carica in memoria l'intero database di dati geografici durante l'elaborazione, ciò porta all'utilizzo di grandi quantità di memoria (minimo circa 800 MB) anche per l'elaborazione di profili per i siti molto piccoli. Ciò inoltre aggiunge una piccola latenza al tempo di inizializzazione dell'elaborazione dei log per ciascun profilo, in genere 2-5 secondi.
  • L'elaborazione dei log non in ordine cronologico molto probabilmente produrrà un'elaborazione molto lenta per i siti con elevati volumi di traffico.
  • Aumentare il limite del DB oltre il valore predefinito di 100.000 record può ridurre le prestazioni di elaborazione dei log, in particolare per i siti con grandi volumi di traffico.
  • Il motore di elaborazione dei log assegna una quantità di memoria a ciascun filtro applicato a un profilo per ogni file di log fisico aperto durante l'elaborazione. Ciò può provocare richieste di assegnazione di grandi quantità di memoria ed eventualmente errori nell'assegnazione di tutta la memoria richiesta quando un profilo è configurato con molti filtri e una gran numero di file di log fisici sono abbinati alle sorgenti di log, ad esempio a seguito di una corrispondenza troppo estesa generata da un carattere jolly.
  • Tabelle di ricerca molto grandi possono richiedere l'assegnazione di grandi quantità di memoria ed eventualmente provocare errori di assegnazione della memoria, specialmente se durante l'elaborazione dei log viene aperto un gran numero di file di log fisici.

Programmi di pianificazione

  • I programmi di pianificazione di Urchin (profilo e CPC) utilizzano il database per memorizzare e recuperare gli indicatori di ora e data al fine di pianificare le varie pianificazioni. Ciò significa che il database, una volta configurato, non deve essere spostato da un fuso orario all'altro, ovvero non si deve cambiare il fuso orario del computer su cui è in esecuzione urchin db. Inoltre, si consiglia vivamente di non cambiare il fuso orario del computer su cui sono in esecuzione di programmi di pianificazione di urchin.

E-commerce

  • Urchin può leggere i dati e-commerce che sono incorporati direttamente nei log del server web se le transazioni sono registrate nei log tramite chiamate UTM javascript oppure tramite un log ELF2 e-commerce esplicito creato dal carrello degli acquisti. Urchin creerà transazioni duplicate se gli stessi dati e-commerce sono presenti sia nel log del server web sia nel log ELF2, pertanto è importante che i log ELF2 non vengano elaborati se gli stessi dati sono incorporati anche nei log del server web.

Utilità

  • Le utilità zip/unzip fornite con Urchin hanno un limite delle dimensioni dei file di 2 Gb. Ciò può avere delle conseguenze sulla capacità di Urchin di creare backup/archivi dei dati per i rapporti per siti molto grandi oppure di utilizzare l'opzione di "compressione" nel Destino log per le sorgenti log.
  • La funzionalità "zero day" in udb-sanitizer può rivelarsi molto lenta per i profili con grandi quantità di dati.

Rapporti

  • I rapporti cross-segment possono essere molto lenti (oppure andare completamente in timeout) per i siti con grandi quantità di traffico e/o intervalli di date molto lunghi.

Autenticazione esterna

  • Urchin 6.6+ viene fornito con moduli di autenticazione esterna che includono LDAP. Al momento, il modulo LDAP non funziona con i provider LDAP che utilizzano "SSL/TLS".

Installazione

  • L'upgrade di un'istanza di Urchin che utilizza una licenza Demo non è consigliato e può portare a problemi di login/licenza.

Cookie

  • Per offrire una migliore interazione nell'interfaccia dei rapporti, Urchin 6.6+ utilizza dei cookie lato client. La visualizzazione corretta dei rapporti richiede che i cookie lato client siano attivati nel browser del client.

  • Il tentativo di effettuare l'accesso a un account che contiene una grande quantità di dati può generare il seguente errore: Errore sessione. Impossibile richiamare la routine predefinita. (2002) Effettuare nuovamente l'accesso. Ciò si verifica quando l'estrazione dei dati per i rapporti elenco completo supera il valore definito APPLICATION_TIMEOUT di 45 secondi.
