Impostazioni del database dei rapporti di Urchin

(Revisione: 16 dicembre 2008)

Panoramica

Urchin offre due opzioni per la gestione della scalabilità e della precisione del database dei rapporti:

Limite tabella DB

Questa opzione consente di mantenere il volume di dati di siti di grandi dimensioni entro limiti ragionevoli. Utilizza questa opzione per limitare il numero di record in un'unica tabella del database dei rapporti di Urchin. L'opzione predefinita è adatta alla maggior parte delle installazioni. Se, nel corso di un mese, il numero di record in una tabella supera questo limite, i record che eccedono tale limite verranno raggruppati nell'ultimo record (denominato "Altro"). Pertanto, quanto maggiori sono le dimensioni della tabella, tanto più precisi sono i dati dei rapporti.

Se un rapporto Urchin indica una percentuale elevata di voci classificate come Altro, devi aumentare il limite della tabella del database per aumentare la precisione.

Target di utilizzo della memoria

Al fine di limitare la quantità di memoria che può essere allocata durante l'elaborazione dei log, Urchin offre questa opzione per specificare il limite superiore per le dimensioni del database dei rapporti in memoria. Nel caso di siti di grandi dimensioni, una maggiore allocazione di memoria può aumentare le prestazioni, in quanto verrà ridotto il swap delle pagine. L'impostazione predefinita tiene conto della configurazione media dei sistemi. Tuttavia, ti consigliamo di impostare un valore più alto se il sistema dispone di memoria fisica sufficiente.

Impostazioni sperimentali

Attualmente, Urchin supporta i seguenti valori massimi per le impostazioni del database dei rapporti:

Limite tabella DB : <= 500.000 record

Target di utilizzo della memoria : <= 1024 MB

A partire dalla versione 6.5, Urchin è in grado di supportare un valore esteso per queste impostazioni, che dovrebbe essere utilizzato solo da utenti Urchin esperti. Questi valori estesi non saranno visibili nell'interfaccia di amministrazione e dovranno essere configurati tramite il database di configurazione. In Urchin 6.5 i valori massimi per le impostazioni estese del database dei rapporti sono i seguenti.

Limite tabella DB : <= 2.000.000 record

Target di utilizzo della memoria : <= 2GB

Linee guida alle impostazioni sperimentali

  • Poiché queste impostazioni estese sono sperimentali e possono causare errori del tipo Memoria insufficiente su alcuni sistemi, si consiglia di leggere le seguenti informazioni e linee guida prima di utilizzarle.

  • In genere, ogni applicazione a 32 bit dispone di 4 gigabyte (GB) di spazio di indirizzi virtuali, suddiviso tra spazio di indirizzo utente e spazio di indirizzo kernel. I processi utente possono allocare memoria solo nello spazio di indirizzo utente.

  • Piattaforma Windows: sono disponibili 2 GB per le applicazioni utente, mentre i 2 GB rimanenti sono a disposizione del kernel. Sebbene windows OS consenta di estendere lo spazio utente fino a 3 GB tramite la funzione "4 GB Tuning" http://msdn.microsoft.com/en-us/library/bb613473(VS.85).aspx, nei processi Urchin non si prende in considerazione questo scenario altamente improbabile. La compilazione e le associazioni vengono sempre eseguite assumendo come limite 2 GB. Pertanto, in windows OS l'utente non dovrebbe impostare Limite tabella DB su un valore superiore a 500.000 record e Target di utilizzo della memoria su un valore superiore a 1 GB.

    • Per ottenere le massime prestazioni, imposta:

      • Target di utilizzo della memoria : 1024 MB

      • Limite tabella DB : 500.000 righe di tabella database

    • Per ottenere la massima precisione, imposta:

      • Target di utilizzo della memoria : 256 MB

      • Limite tabella DB : 1.000.000 righe di tabella database

  • Piattaforme Unix: sono disponibili 3 GB per le applicazioni utente, mentre il rimanente 1 GB è a disposizione del kernel. Pertanto, nelle piattaforme UNIX l'utente non dovrebbe impostare Limite tabella DB su un valore superiore a 500.000 record e Target di utilizzo della memoria su un valore superiore a 2 GB.

    • Per ottenere le massime prestazioni, imposta:

      • Target di utilizzo della memoria : 2048 MB

      • Limite tabella DB : 500.000 righe di tabella database

    • Per ottenere la massima precisione, imposta:

      • Target di utilizzo della memoria : 1024 MB

      • Limite tabella DB : 2.000.000 righe di tabella database

Configurazione del 'Limite tabella DB' esteso

Analogamente al valore standard 'Limite tabella DB', questi valori estesi possono essere configurati sia per singoli profili che a livello globale. La seguente query imposta il Limite tabella DB a 1.000.000 di record per un profilo con ID profilo = '1'. 

UPDATE uprofiles SET uipr_db_table_limit=1000000 WHERE uspr_id=1;

La seguente query imposta il Limite tabella DB globale a 1.000.000 di record. Verrà applicata a tutti i profili il cui valore per il campo uipr_db_table_limit è 0. 

UPDATE uglobals SET uigl_db_table_limit=1000000 WHERE ucgl_name='Global Settings';

Configurazione del Target di utilizzo della memoria' esteso

Il Target di utilizzo della memoria può essere impostato solo a livello globale.

Viene applicato automaticamente ai profili quando Utilizzo memoria DB è impostato su 'Limita a sistema cache' (o il campo 'ubpr_limit_db_cache_size' è impostato su 1).

La seguente query imposta il Target di utilizzo della memoria globale su 2 GB. Tieni presente che il campo db considera il numero in 'megabyte' (512 significa effettivamente 512 MB). 

UPDATE uglobals SET uigl_db_cache_size=2048 WHERE ucgl_name='Global Settings';
