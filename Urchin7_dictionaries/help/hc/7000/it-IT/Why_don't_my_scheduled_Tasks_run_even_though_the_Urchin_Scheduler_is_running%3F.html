Perché le mie attività pianificate non vengono eseguite nonostante il programma di pianificazione di Urchin sia in esecuzione?

Descrizione del problema

In alcune circostanze il programma di pianificazione di Urchin (urchind) potrebbe non eseguire le attività pianificate pur essendo in esecuzione. È indice di questa situazione quanto descritto di seguito:

  1. Le attività vengono visualizzate come pianificate nella schermata Configurazione -> Programma di pianificazione -> Programma di pianificazione attività dello strumento di amministrazione di Urchin e vengono ripianificate come previsto, ma in realtà non vengono mai eseguite.

  2. Facendo clic sul pulsante "Esegui ora" nella schermata Esegui/Pianifica relativa al profilo viene visualizzata la finestra popup di avanzamento prevista, ma nella barra di avanzamento viene visualizzato solo "Errore (0%)" e l'attività non viene mai completata.

Causa

In genere, ciò accade perché il programma di pianificazione di Urchin è stato avviato da una directory che è stata rimossa, oppure le autorizzazioni sono state modificate e ora Urchin non può più accedervi.

Ad esempio, è una procedura standard spacchettare il programma di installazione di Urchin in una directory temporanea per eseguire un'installazione o un aggiornamento. Durante il processo di installazione/aggiornamento, il programma di pianificazione di Urchin viene avviato con la directory di lavoro corrente impostata come directory temporanea. Se tale directory viene successivamente rimossa, il programma di pianificazione di Urchin non funzionerà più correttamente.

Soluzione

Per garantire che le applicazioni Urchin possano essere avviate ed eseguite correttamente, assicurati di avviare il programma di pianificazione di Urchin da una posizione permanente per la quale Urchin avrà sempre le autorizzazioni di accesso corrette. Se riscontri problemi con il programma di pianificazione, è preferibile riavviare i servizi Urchin procedendo nel modo descritto di seguito:

Sistemi Windows

  1. Start -> Programmi -> Urchin -> Disabilita servizi Urchin

  2. Start -> Programmi -> Urchin -> Attiva servizi Urchin

Sistemi di tipo UNIX

  1. Avvia un'interfaccia di riga di comando come "root" o come utente cha ha eseguito l'accesso a Urchin
  2. Passa alla directory in cui è installato Urchin, ad esempio "cd /usr/local/urchin"
  3. Riavvia i servizi Urchin con "bin/urchinctl restart"
