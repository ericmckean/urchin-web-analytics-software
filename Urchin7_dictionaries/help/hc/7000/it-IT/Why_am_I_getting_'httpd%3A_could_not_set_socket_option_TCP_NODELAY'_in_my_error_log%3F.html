Perché compare "httpd: could not set socket option TCP_NODELAY" nel log degli errori?

Panoramica

Questo messaggio indica quasi sempre che il client è stato disconnesso prima che il server web Apache di Urchin (urchinwebd) raggiungesse il punto in cui vengono impostate determinate opzioni nella connessione di rete (socket) tra il browser e il server Apache.

Soluzione

Questo messaggio è semplicemente informativo. Tuttavia, potrebbe essere utile chiedere all'amministratore di rete di determinare la possibile causa della disconnessione prematura delle connessioni di rete. Di seguito vengono indicate alcune cause possibili:

  • Un robot/spider web o un agente di monitoraggio configurato in modo errato che non chiude correttamente la connessione di rete al server Apache di Urchin.
  • Un firewall o una rete configurata in modo errato che causa l'eliminazione impropria di traffico di pacchetti.
  • Il risultato di un'analisi delle porte di rete alla ricerca di vulnerabilità.
  • Un attacco di tipo denial-of-service contro il server web Apache di Urchin.
