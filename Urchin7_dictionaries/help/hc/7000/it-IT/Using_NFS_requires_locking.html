L'utilizzo di NFS richiede il blocco

La configurazione di Urchin, che utilizza file di dati ubicati nella cartella data/conf dell'installazione, determina dei blocchi di lettura e di scrittura durante l'impostazione e l'amministrazione. Se la cartella data/conf si trova su NFS, è necessario che il server NFS esegua i daemon di blocco appropriati per gestire i blocchi remoti.

Se il blocco non funziona correttamente, l'installazione di Urchin può rimanere in sospeso per un tempo indefinito.

Note tecniche

Urchin utilizza la funzione fcntl() su UNIX per effettuare il blocco consultivo ed esclusivo (di lettura/scrittura). Alcuni sistemi meno recenti potrebbero non supportare tutte le richieste di blocco rpc di piattaforme diverse. Assicurati di utilizzare una versione recente del tuo SO con le ultime patch. Verifica inoltre che il server NFS sia in funzione con le opzioni 'statd' e 'lockd' o equivalenti.
