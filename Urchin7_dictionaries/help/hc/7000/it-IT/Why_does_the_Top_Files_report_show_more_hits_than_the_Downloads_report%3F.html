Perché il rapporto File principali mostra più hit del rapporto Download?

Accade spesso che il rapporto File principali mostri un numero di hit superiore a quello del rapporto Download per lo stesso file. I motivi sono i seguenti. Nel caso di un file PDF, molti browser includono un plug-in che visualizza i file PDF una pagina alla volta. La richiesta della pagina iniziale genera un codice di stato 200 nel server web. Tutte le pagine successive generano un codice di stato 206 (download parziale), poiché il plug-in del browser scarica solo la pagina visualizzata e non il documento completo. Di conseguenza, se visualizzi 5 pagine PDF dello stesso documento, il server web registra un hit con il codice di stato 200 e 4 hit con il codice di stato 206 considerandoli tutti come lo stesso documento PDF.

Ecco in che modo i rapporti Urchin mostrano questi dati:

Download: un hit di download richiede un codice di stato 200, 302 o 304, pertanto il rapporto mostra solo un download per il file PDF (ad esempio, il singolo hit con il codice di stato 200).

Il rapporto File principali indica tutte le richieste di file completate se il codice di stato corrisponde a 2XX, 302 o 304 e il tipo MIME del file è appropriato per essere considerato una pagina visualizzata. Per impostazione predefinita, Urchin tratta tutti i tipi MIME come pagine visualizzate ad eccezione dei file con le seguenti estensioni: .gif, .jpg, .jpeg, .png, .js, .css, .cur, .ico, .ida.

Di conseguenza, le 5 richieste PDF nell'esempio sopra riportato verrebbero tutte conteggiate come hit di questo documento nel rapporto File principali, poiché ogni hit contiene un codice di stato 200 o 206.

Ogni hit di download parziale (stato 206) viene conteggiato come una pagina visualizzata distinta, poiché ciascun hit corrisponde più o meno a una singola pagina di un documento PDF a più pagine.
