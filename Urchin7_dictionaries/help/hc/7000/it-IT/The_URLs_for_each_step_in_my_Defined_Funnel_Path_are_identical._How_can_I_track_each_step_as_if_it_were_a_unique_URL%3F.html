Gli URL di ciascun passaggio del mio percorso obiettivo definito sono identici. Come faccio a monitorare ciascun passaggio come se fosse un unico URL?

Puoi monitorare lo stato di avanzamento dei visitatori all'interno di un percorso obiettivo che abbia lo stesso URL per ciascun passaggio. Ad esempio, il percorso obiettivo della registrazione può apparire come segue:

  • Passaggio 1 (registrazione)
    • www.miosito.it/sign_up.cgi
  • Passaggio 2 (accettazione del contratto)
    • www.miosito.it/sign_up.cgi
  • Passaggio 3 (termine)
    • www.miosito.it/sign_up.cgi

Per risolvere il problema, puoi richiamare la funzione Javascript urchinTracker durante ciascun passaggio (probabilmente durante un evento onload) nel seguente modo:

  • urchinTracker("/funnel_G1/step1.html")
  • urchinTracker("/funnel_G1/step2.html")
  • urchinTracker("/funnel_G1/step3.html")

Quindi, imposta ciascun passaggio del percorso obiettivo nel seguente modo:

  • http://www.miosito.it/funnel_G1/step1.html

  • http://www.miosito.it/funnel_G1/step2.html

  • http://www.miosito.it/funnel_G1/step3.html

Tieni presente che l'argomento del percorso/nome di file che richiama urchinTracker() non deve rappresentare necessariamente un percorso o nome di file esistente. L'argomento che richiama urchinTracker fornisce semplicemente un nome di pagina fittizio a cui Urchin può allegare pagine visualizzate.
