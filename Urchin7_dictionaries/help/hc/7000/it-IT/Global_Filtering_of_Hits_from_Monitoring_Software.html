Filtro globale degli hit dal software di monitoraggio

Panoramica

La maggior parte degli ambienti di hosting fornisce un sistema di monitoraggio dei server web dei clienti in modo da rispettare i contratti sul livello del servizio (SLA, Service Level Agreement). Come effetto secondario, tuttavia, gli hit registrati dal monitoraggio possono alterare notevolmente i rapporti di Urchin relativi ai siti web monitorati, aumentando artificialmente il numero di sessioni, pagine visualizzate, hit e byte.

Suggerimento

Negli ambienti di hosting che utilizzano tale monitoraggio è fortemente consigliabile applicare un filtro Urchin standard/globale a ciascun profilo Urchin configurato del cliente, in modo da rimuovere gli hit generati dal software di monitoraggio. Si tratta di un'operazione molto semplice in un ambiente dove un'installazione centralizzata di Urchin (gestita dalla società di hosting) fornisce i rapporti per i siti web di ciascun cliente. Negli ambienti dedicati/di colocation dove i clienti stessi mantengono un'istanza di Urchin sui propri server, la società di hosting dovrebbe fornire un filtro di esempio appropriato per il monitoraggio utilizzato.

Per agevolare l'implementazione del filtro Urchin, l'host e il cliente devono lavorare insieme per creare una pagina specifica nel sito web del cliente che viene utilizzata solo dal software di monitoraggio, ad esempio: 

     http://www.customerdomain.com/healthpage.html

Esempi

Esempio 1: filtra ed escludi l'indirizzo IP per il sistema di monitoraggio 

      Filter Type: Exclude
      Filter Field: IP
      Filter Spec: 172\.16\.1\.1
  • In tal modo vengono esclusi dal log del server web eventuali hit con indirizzo IP 172.16.1.1 perché in fase di elaborazione da parte di Urchin.

Esempio 2: filtra ed escludi una pagina specifica interessata dal sistema di monitoraggio 

      Filter Type: Exclude
      Filter Field: REQUEST
      Filter Spec: ^/healthpage.html
  • In tal modo vengono esclusi dal log del server web eventuali hit con una richiesta di /healthpage.htm perché in fase di elaborazione da parte di Urchin.

Considerazioni

  1. È opportuno creare profili aggiuntivi non filtrati per i clienti in modo che possano vedere il carico di traffico reale (incluso il filtraggio) sul server web.
  2. È consigliabile che la società di hosting fornisca un profilo che generi esclusivamente rapporti relativi agli hit di monitoraggio, ad esempio che includa solo gli hit provenienti dal software di monitoraggio. Questo profilo potrebbe essere utilizzato per mostrare che è stato effettuato il monitoraggio corretto e che gli SLA sono stati rispettati.
