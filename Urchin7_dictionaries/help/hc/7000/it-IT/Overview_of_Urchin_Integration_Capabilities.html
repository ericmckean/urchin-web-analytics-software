Panoramica delle capacità di integrazione di Urchin

Introduzione

Il front-end Urchin è basato su componenti modulari e tecniche standard del settore per consentire agli amministratori di integrare l'accesso ai rapporti di Urchin nelle infrastrutture in uso o pianificate. Nell'ambito del nostro impegno nei confronti dei fornitori di servizi di hosting e dei data center, abbiamo testato e supportiamo tre punti di integrazione che consentono di utilizzare Urchin con la maggior parte delle architetture.

I componenti principali del front-end Urchin e i tre punti di integrazione vengono illustrati nel seguente schema. Sia la funzione amministrativa, sia la funzione di rendicontazione sono basate sul Web e fornite dal sistema.

Tutti i contenuti del sistema Urchin vengono distribuiti tramite un server Apache installato come "urchinwebd", indicato nella parte sinistra dello schema. Le richieste vengono gestite dal server Apache e trasmesse tramite l'interfaccia CGI a un controller di sessione (session.cgi). Il controller di sessione esegue l'autenticazione e, a seconda dell'azione, trasferisce la richiesta al motore amministrazione (admin) oppure al motore rapporti (urchin.cgi) di Urchin. I contenuti con identificatori di sessione incorporati vengo trasmessi nuovamente all'utente.

"A", "B" e "C" sono i tre punti di integrazione precedentemente citati. Il punto "A" consente all'amministratore di sostituire o escludere il server web Urchin. Il punto "B" consente l'autenticazione esterna o esclude l'autenticazione. Il punto "C" consente l'accesso diretto ai rapporti tramite wrapper o portale.

Punto A: integrazione con il server web

Molte aziende di hosting dispongono già di un server web attivo o di un server web compilato in modo specifico che viene eseguito nei loro sistemi. Se un server web fornisce le operazioni CGI di base, è possibile sostituire il binario Apache o utilizzare il server web in uso per fornire l'accesso a Urchin.

Questo punto di integrazione consente di mantenere invariato il resto del sistema Urchin. L'interfaccia Urchin verrà utilizzata per l'amministrazione, l'autenticazione e la generazione dei rapporti. Sarà necessario configurare utenti e profili nel sistema Urchin in modo che quando un utente esegue l'accesso, Urchin sappia a quali rapporti autorizzare l'accesso.

Punto B: autenticazione esterna o esclusione dell'autenticazione

Complessi ambienti di hosting dispongono spesso di controlli centralizzati per fornire il servizio di autenticazione degli utenti, quale LDAP. Con una semplice variazione della configurazione, il controller di sessione può richiamare una routine di autenticazione di tua scelta. Viene fornita una semplice interfaccia per restituire gli accessi eseguiti e quelli non riusciti. Le routine di autenticazione in uso possono essere facilmente elaborate con wrapper per fornire la struttura corretta per entrambi i sistemi. Utilizzando il punto di integrazione "B", è possibile anche escludere l'autenticazione con una routine fittizia e un collegamento diretto alla pagina di destinazione.

Utilizzando il punto di integrazione "B", il sistema Urchin rimane complessivamente invariato, ma l'autenticazione viene eseguita da un'applicazione esterna. Sarà necessario configurare utenti e profili nel sistema Urchin in modo che quando un utente esegue l'accesso, Urchin sappia quali rapporti rendere disponibili. Per tutte le informazioni sull'interfaccia che consente di utilizzare l'autenticazione esterna e per un esempio di come escludere l'autenticazione, consulta Uso dell'autenticazione esterna o esclusione dell'autenticazione.

Punto C: collegamento diretto ai rapporti tramite wrapper o portale

Molti dei nostri clienti hanno una relazione uno a uno tra cliente e sito web e desiderano collegarsi ai rapporti di un determinato sito web direttamente dall'area di amministrazione ad esso relativa. Di norma quest'area è già autenticata. Il punto di integrazione C consente di collegarsi con facilità direttamente ai rapporti tramite uno script wrapper o da un portale esistente. In questo modo, l'autenticazione Urchin e la schermata iniziale di selezione dei rapporti vengono escluse e gli utenti indirizzati direttamente ai rapporti Urchin.

Utilizzando il punto di integrazione "C", si presuppone che il fornitore di servizi controlli gli utenti e le informazioni consultate con una relazione uno a uno o con un database di configurazione esistente. Sarà comunque necessario configurare Urchin per la generazione dei rapporti sui profili, ma questa operazione può essere automatizzata internamente o esternamente all'interfaccia di amministrazione. Quando è attivata l'opzione "Collegamento diretto al rapporto" nell'interfaccia di amministrazione, Urchin consente di collegarsi direttamente a un rapporto Urchin con un URL specifico. L'accesso al rapporto Urchin viene controllato tramite un file ".report.conf" integrato nella directory specificata da questo URL.

Per l'integrazione di un portale complesso, il motore rapporti di Urchin propagherà la sessione e altre variabili del portale per mantenere attiva la sessione.
