Le note contenute in questo articolo riguardano l'installazione di Urchin su Windows Server 2008 e Windows Vista. Sebbene la procedura descritta in Urchin Guida all'installazione di Urchin (Windows) funzioni sia per Windows Server 2008 che per Windows Vista, occorre tenere presenti le seguenti considerazioni.

Windows 2008 contiene una nuova funzionalità chiamata Controllo Account Utente (UAC), per maggiori dettagli consulta http://technet.microsoft.com/en-us/library/cc709691.aspx.

I problemi noti che si presentano quando UAC è attivato sono:

  1. Se UAC è attivato, l'applicazione non è in grado di eseguire correttamente le operazioni di scrittura nella cartella “Programmi”. Per maggiori dettagli: http://msdn.microsoft.com/en-us/library/aa905330.aspx

  2. Per impostazione predefinita, gli eseguibili Urchin non possono essere eseguiti da un prompt dei comandi. Devi utilizzare il prompt dei comandi come amministratore oppure impostare le proprietà dell'eseguibile come segue: fai clic con il pulsante destro del mouse e seleziona Proprietà-> Compatibilità -> “Esegui questo programma come amministratore” -> OK. Tieni presente che i servizi Urchin Windows funzionano come previsto senza dover compiere ulteriori azioni.

Soluzione:

  1. Disattiva UAC e installa Urchin dove preferisci. Esegui Urchin utilizzando i file binari o i servizi Windows.
  2. Se UAC è attivato, NON installare Urchin nella cartella Programmi. Installalo, ad esempio, nella directory Utenti. Modifica le proprietà per eseguire i file binari in modalità amministratore (come descritto nel punto 2 sopra) oppure utilizza i servizi Windows senza apportare modifiche.
