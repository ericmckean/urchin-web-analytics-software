Che cosa significa "Errore sessione. Impossibile richiamare l'applicazione. (4002)..."?

Quando leggi i rapporti Urchin, il browser esegue una query nei file di database di Urchin alla ricerca di dati che corrispondono all'intervallo di date e alla visualizzazione del rapporto richiesti. Se la risposta alla query non viene ricevuta entro il periodo predefinito di 45 secondi, si verifica un errore di timeout e viene visualizzato il messaggio di errore "Errore sessione. Impossibile richiamare l'applicazione. (4002) Effettuare nuovamente l'accesso.". Quando si verifica questa situazione, è necessario aumentare il valore di timeout di conseguenza. Per eseguire questa operazione, devi disporre dell'accesso dalla riga di comando ai file della distribuzione Urchin. In caso contrario, contatta la persona che fornisce i rapporti facendo riferimento a questo articolo.

Per aumentare il valore di timeout dell'applicazione, procedi nel seguente modo:

  • Apri un'interfaccia di riga di comando nel server Urchin.
  • Apri il file /path/to/urchin/etc/session.conf.

  • Individua la direttiva denominata APPLICATION_TIMEOUT:45 dove il valore è rappresentato in secondi.

  • Aumenta il valore di conseguenza e salva il file. Le modifiche avranno effetto immediatamente.
