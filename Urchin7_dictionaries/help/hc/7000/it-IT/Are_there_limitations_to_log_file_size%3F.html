Esistono limitazioni per le dimensioni del file di log?

Il motore di elaborazione log di Urchin è in grado di elaborare log di server web

  • più grandi di 2GB, siano essi compressi con bzip (.bz2), gzip (.gz)

o zip (.zip), poiché Urchin decomprime i file utilizzando la libreria zlib.

Tuttavia, il motore di elaborazione log di Urchin utilizza i file binari unzip/zip della directory util per creare copie di backup e archivi dei dati dei rapporti per ciascun profilo. Se per l'elevato volume di dati dei rapporti di un mese le dimensioni del file zip superano la soglia dei 2GB, Urchin non sarà in grado di eseguire copie di backup o archivi e l'elaborazione log non riuscirà se non viene disabilitata la funzione di creazione automatica delle copie di backup e degli archivi. In questo caso, dovrai creare manualmente le copie di backup dei file con un'altra utilità esterna.
