Come faccio a riprendere la regolare attività quando viene visualizzato il messaggio di errore: "Unable to open database for writing since it has been archived"?

Quando tenti di eseguire un profilo in Urchin 6, ricevi questo messaggio di errore: 

WARNING: (7063-54-63) Unable to open database for writing since it has been archived

Questo errore si verifica perché con Urchin i database di monitoraggio log vengono archiviati mensilmente. Durante l'elaborazione, Urchin legge le prime due righe del file di log, determina il mese di appartenenza del log e tenta di aprire i database relativi a quel mese. Poiché i database per quel mese sono stati archiviati, viene restituito un errore perché Urchin non consente la lettura/scrittura dei database archiviati. Esistono due espedienti:

  1. Disattiva la funzione di archiviazione e decomprimi i file *-archive.zip relativi ai profili interessati, quindi rimuovi i file *-archive.zip.
  2. Modifica il carattere jolly del log in modo da includere una stringa per la corrispondenza della data (ad esempio, è possibile utilizzare AA* o AAAA* per identificare soltanto i file relativi all'anno in corso).
