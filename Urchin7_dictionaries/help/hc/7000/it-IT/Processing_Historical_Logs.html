Elaborazione dei log storici

Panoramica

Dopo aver installato Urchin puoi elaborare i log storici. Questa operazione può essere eseguita facilmente. Specifica semplicemente una directory e un nome di file parziale e/o caratteri jolly (ad esempio le espressioni regolari) nelle schermate Impostazioni log di Gestione log. NOTA. Non puoi utilizzare caratteri jolly in sorgenti log HTTP e HTTPS remote.

Come elaborare i log storici

In primo luogo, aggiungi una sorgente log al sistema. Fai clic sul pulsante Configurazione nel menu in alto, quindi sul link Profili Urchin -> Gestione log. Nella schermata principale fai clic sul pulsante Aggiungi in alto a destra. Nella prima schermata seleziona Aggiungi sorgente log locale e continua. Nella schermata successiva fai clic su Sfoglia; viene visualizzato il Browser di file. Individua la directory corretta nella parte sinistra della finestra. Nella parte destra della finestra vengono visualizzati i file della directory, mentre nella parte sinistra vengono visualizzate tutte le altre sottodirectory della directory attualmente aperta. Quando viene visualizzata la directory corretta, immetti un nome di file parziale e un asterisco, o un'altra espressione regolare, e fai clic sul pulsante Verifica. Viene aperta una finestra che mostra tutte le corrispondenze al pattern. Fai clic su un nome di file qualsiasi per ottenere le relative informazioni: percorso, dimensioni, data dell'ultima modifica e autorizzazioni all'uso del file. Se la corrispondenza al pattern è corretta, fai clic su OK, quindi di nuovo su OK nella finestra Browser di file. Specifica quindi il nome della sorgente log e fai clic su Avanti. Specifica il formato log dall'elenco a discesa e fai clic sul pulsante Fine. Se il Nome sorgente log (che deve essere univoco) è valido, fai clic sul pulsante Completato per trovare l'elenco Gestione log della sorgente log.

Quindi, se non è stato ancora fatto, associa il file di log con un profilo facendo clic sul pulsante Configurazione nel menu in alto, quindi sul pulsante Profili Urchin -> Profili. Una volta completata l'associazione, fai clic sul pulsante Esegui/Pianifica accanto al profilo nell'elenco Profili principale. Puoi pianificare l'esecuzione del profilo oppure fare clic sul pulsante Esegui ora per avviare l'elaborazione immediatamente.

Urchin non ha bisogno di alcun tipo di rotazione log per evitare la duplicazione dei dati poiché è dotato di una funzionalità di monitoraggio log che assicura che vengano elaborati solo i nuovi hit. Tuttavia, come menzionato in precedenza, i log possono esaurire velocemente grandi quantità di spazio su disco, quindi è consigliabile comprimere e archiviare periodicamente i file di log. Poiché per Urchin non è mai necessario rileggere i file di log dopo che sono stati elaborati, è perfettamente accettabile eliminare i log dopo ogni elaborazione. Tuttavia, molte persone conservano i log per un determinato periodo di tempo nel caso dovessero essere necessari per varie ragioni, ad esempio quando viene creato un nuovo profilo per il sito e si desidera disporre dell'analisi storica.

Suggerimenti

La gestione log non è essenziale sin dall'inizio, ma diventa importante all'aumentare dei log. È consigliabile decidere un piano di gestione log durante la distribuzione iniziale di Urchin.
