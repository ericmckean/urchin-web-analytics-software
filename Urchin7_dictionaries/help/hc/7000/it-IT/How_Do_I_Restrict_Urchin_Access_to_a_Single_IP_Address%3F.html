Come faccio a limitare l'accesso di Urchin a un solo indirizzo IP?

Introduzione

Su un sistema che dispone di molteplici interfacce di rete o di più indirizzi IP collegati a una sola interfaccia di rete, per impostazione predefinita l'interfaccia web Urchin è collegata a tutti gli indirizzi IP configurati. Può essere tuttavia opportuno configurare Urchin in modo che limiti l'amministrazione e la creazione di rapporti a un solo indirizzo IP.

Procedura

La limitazione dell'amministrazione e della creazione di rapporti di Urchin richiede una semplice modifica del file di configurazione utilizzato dal server web Urchin Apache. A tale scopo, procedi nel modo seguente.

Per sistemi di tipo UNIX:

  • Apri un pannello di comandi come utente eseguito da Urchin
  • Specifica il comando "cd" per la directory in cui è installato Urchin
  • Interrompi il server web Urchin con il comando: "bin/urchinctl -w stop"
  • Utilizzando un editor di testo, apri il file "urchinwebd.conf.template" nella directory "var" della distribuzione Urchin.

  • Sopra la riga <Directory /> inserisci la riga 

       Listen XXX.XXX.XXX.XXX:port
  • dove "XXX.XXX.XXX.XXX" e "port" sono l'indirizzo IP e il numero della porta a cui desideri limitare Urchin.
  • Riavvia il server web Urchin con il comando: 

bin/urchinctl -w start

Per i sistemi Windows:

  • Interrompi i servizi Urchin: "Start -> Programmi -> Urchin 6 -> Disattiva servizi Urchin"

  • Apri Windows Explorer e vai a "C:\Program Files\Urchin6\var"
  • Utilizzando un editor di testo, apri il file "urchinwebd.conf.template"

  • Sopra la riga <Directory /> inserisci la riga 

       Listen XXX.XXX.XXX.XXX:port
  • dove "XXX.XXX.XXX.XXX" e "port" sono l'indirizzo IP e il numero della porta a cui desideri limitare Urchin.

  • Riavvia i servizi Urchin: "Start -> Programmi -> Urchin 6 -> Attiva servizi Urchin.

Nota importante. A causa dell'uso del pooling del socket in IIS 5 e versioni successive, può risultare necessario disattivare il comportamento predefinito di IIS di condivisione dei socket per tutti i siti web collegati a una particolare porta, a prescindere dall'indirizzo IP effettivamente configurato. Questa situazione è tipica di ambienti in cui sono utilizzati host virtuali e si desidera impostare la creazione di rapporti Urchin sulla porta 80 per uno specifico host virtuale.

Considerazioni

  • Questo metodo consente solo di limitare l'amministrazione e la creazione di rapporti Urchin a uno specifico indirizzo IP. Non puoi limitare l'accesso agli host virtuali basati sul nome.
  • Gli aggiornamenti a Urchin sovrascrivono il file "urchinwebd.conf.template" dopo il salvataggio di una copia di backup. Dopo un aggiornamento dovrai quindi apportare nuovamente le modifiche effettuate al file.
