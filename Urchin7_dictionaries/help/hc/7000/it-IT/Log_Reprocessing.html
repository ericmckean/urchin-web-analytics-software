Rielaborazione log

Introduzione

In alcune circostanze potrebbe rendersi necessaria una rielaborazione dei dati di log, ad esempio quando un server DNS non è attivo al momento dell'elaborazione, in caso di filtri applicati in modo non corretto e così via. Il seguente documento descrive la procedura corretta per revocare e rielaborare i dati di log del server web.

Tieni presente che la rielaborazione log richiede l'utilizzo di utilità di Urchin disponibili soltanto da un'interfaccia di riga di comando. Non è possibile eseguire la procedura completa esclusivamente dalla GUI amministrativa basata sul Web di Urchin.

Rielaborazione di un singolo giorno

  • Nella GUI amministrativa di Urchin, modica il profilo e disattiva Monitoraggio log nella scheda Memorizzazione/DB. Fai clic su Aggiorna per salvare la modifica.
  • Nella scheda Sorgenti log, verifica che siano specificati i corretti file di log da rielaborare. I dati di log devono contenere solo gli hit relativi alle date per le quali stai azzerando le statistiche.
  • Richiama un'interfaccia di riga di comando sul sistema Urchin e digita 

cd /<URCHIN_DIR>/util
  • Esegui l'utilità udb-sanitizer nella directory/folder 'util' della distribuzione Urchin con il comando 

udb-sanitizer -p profile-name -d YYYYMM
  • dove YYYYMM rappresenta l'anno e il mese in cui è contenuto il giorno da rielaborare.

  • Seleziona l'opzione 5, che corrisponde all'azzeramento di uno o più giorni. L'utilità chiederà il giorno corretto e azzererà le statistiche relative a quel determinato giorno. Per ulteriori esempi sull'utilizzo di udb-sanitizer, consulta udb-sanitizer: Database Maintenance.

  • Per rielaborare i dati di log, fai clic sul pulsante Esegui ora nella scheda Esegui/Pianifica relativa al profilo.
  • Reimposta la sorgente log impostando il percorso del file di log sul valore originale.
  • Nella scheda Memorizzazione/DB nell'area di modifica del profilo, riattiva Monitoraggio log.

Rielaborazione di un intero mese

La procedura di rielaborazione dei dati di un intero mese è simile a quella relativa a un singolo giorno descritta sopra; è necessario semplicemente utilizzare opzioni di udb-sanitizer diverse. Per gli esempi, consulta udb-sanitizer: Database Maintenance.
