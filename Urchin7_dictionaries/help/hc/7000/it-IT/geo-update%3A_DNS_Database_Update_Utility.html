geo-update: Utilità di aggiornamento del database DNS

Panoramica

L'utilità geo-update consente di verificare gli aggiornamenti del database DNS interno di Urchin e di scaricare gli aggiornamenti disponibili sia dai server di download di Google, sia da una directory locale (novità di Urchin 6).

Utilizzo

L'utilità geo-update può essere attivata mensilmente dal programma di pianificazione di Urchin (urchind) oppure manualmente dalla riga di comando. Quando viene attivata tramite il programma di pianificazione, esegue sempre l'aggiornamento dal server di download di Google. Quando viene attivata dalla riga di comando, fornisce un'opzione aggiuntiva che consente di aggiornare il database DNS da un percorso locale. L'attività è impostata per rilevare gli aggiornamenti una volta al mese e l'utente può configurare l'orario della ricerca tramite l'interfaccia di amministrazione.

Se è stata creata un'attività per gli aggiornamenti geografici, essa viene visualizzata nella schermata Cronologia attività con il nome "geo-update"

Per impostazione predefinita, l'attività rileva gli aggiornamenti, scarica i nuovi database e sovrascrive quelli in uso. Le operazioni predefinite possono essere sovrascritte eseguendo l'utilità geo-update dalla riga di comando con le apposite opzioni. Utilizzo delle opzioni: 

geo-update [-Hhv] [-D | -F] [-i file]

dove 

  -F            Consente di eseguire il download dei database geografici
  -f            Consente di assegnare il percorso del file geo.zip nel file system locale [facoltativo]
  -H            Consente di registrare l'output dell'esecuzione nel file cronologico
  -h            Consente di stampare informazioni della guida e di uscire
  -T <task id>  Consente di indicare il record della cronologia attività da aggiornare;
                se si specifica -T occorre utilizzare -H
  -V            Consente di stampare il numero della versione del database geografico e di uscire
  -v            Consente di stampare il numero di versione e di uscire

L'uso dell'opzione -F comporta il download di nuovi database anche se quelli in uso sono già aggiornati. Questa funzione è utile se sono state importate informazioni sul dominio personalizzato non corrette oppure se il database è danneggiato del dominio e si desidera disporre di una copia nuova.

Se si registra l'output in un file cronologico con l'opzione -H è possibile rendere disponibili le informazioni sull'esecuzione quando viene visualizzata la schermata Cronologia attività nell'area Configurazione->Programma di pianificazione dell'interfaccia di amministrazione di Urchin.

L'opzione -T consente di eseguire dalla riga di comando l'aggiornamento di una determinata attività, specificata con <Task_ID>. Per trovare il valore Task_ID nel database di Urchin 6 utilizza la seguente query: 

mysql> select * from ugeo_tasks\G

Esempi

Per eseguire il download del database dei domini aggiornato: 

geo-update -F

Per aggiornare manualmente l'attività dalla riga di comando (dopo aver recuperato l'ID attività dal DB): 

geo-update -T <Task_ID> -H

Per poter rilevare e scaricare gli aggiornamenti l'utilità geo-update richiede una connessione Internet. L'utilità si serve della porta 80 per comunicare con il server web che fornisce gli aggiornamenti. Quando vengono eseguiti gli aggiornamenti è possibile che si verifichino dei problemi per superare i firewall e accedere ai server proxy. In questi casi, contatta l'amministratore di rete.
