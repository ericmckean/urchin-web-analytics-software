Sorgenti log

Panoramica

In genere, una sorgente log viene aggiunta durante la creazione di un profilo. La sorgente log consente a Urchin di identificare le caratteristiche di un log di accesso (chiamato a volte log di trasferimento) per uno dei tuoi siti web. I log di accesso contengono tutti gli hit, ovvero richieste di documenti web, ricevuti dal tuo sito. Alcune caratteristiche del file di log associate a una sorgente log sono il percorso del file di log, il formato del file di log (ad esempio W3C o NCSA), se il log è locale o si trova su un sistema remoto e se deve essere applicato un filtro al file di log durante l'elaborazione.

È importante capire che le sorgenti log esistono indipendentemente dai profili. Per poter generare rapporti, a ogni profilo deve essere associata almeno una sorgente log. Tuttavia, diversi profili possono utilizzare la stessa sorgente log. Ad esempio, puoi creare più profili utilizzando la stessa sorgente log, ma assegnare a ogni profilo un filtro diverso per produrre diversi risultati nei rapporti. Quindi, non esiste necessariamente un rapporto 1:1 tra le sorgenti log e i profili.

Configurazione delle sorgenti log

Per aggiungere una sorgente log al sistema, esegui l'accesso al sistema amministrativo di Urchin come amministratore e fai clic sul pulsante Configurazione a sinistra. Quindi, fai clic sul pulsante Gestione log. Per creare una nuova sorgente log, fai clic sul pulsante Aggiungi nella parte superiore destra della schermata. Viene visualizzata la procedura guidata che consente di aggiungere una sorgente log. Si tratta di una semplice serie di passaggi che consentono di configurare la sorgente log in poco tempo e con facilità. Ogni schermata della procedura guidata contiene informazioni esplicite che spiegano i dati di configurazione visualizzati nella schermata stessa.

Nella schermata Impostazioni log noterai che dovrai scegliere un formato log. Questa impostazione comunica a Urchin come sono disposti i dati nel tuo file di log. È importante selezionare il formato corretto per il log, altrimenti Urchin non sarà in grado di produrre rapporti con dati significativi. Urchin capisce un insieme predefinito di formati log tra cui scegliere tramite un menu a discesa. Sono:

  • Automatico: Urchin utilizza questo formato per rilevare automaticamente i formati log NCSA, W3C, Netscape e ELF2. Anziché selezionare esplicitamente uno di questi formati, puoi scegliere Automatico per consentire a Urchin di dedurre correttamente la modalità di lettura dei dati se il tuo formato log è nell'elenco.
  • NCSA: formato esteso/combinato modificato di Apache (per la descrizione di questo formato, consulta Registrazione: Apache e IIS).
  • W3C: in genere questo formato viene utilizzato dai server Microsoft IIS, sebbene anche altri server web possano essere configurati per produrre log W3C.
  • Netscape: i server Netscape e iPlanet utilizzano questo formato per impostazione predefinita.
  • ELF2: formato log E-Commerce.
  • Google: utilizza questo formato per i log che contengono i dati sul costo per clic di Google. Il formato log Google non può essere rilevato automaticamente.
  • Overture: utilizza questo formato per i log che contengono i dati sul costo per clic di Overture. Il formato log Overture non può essere rilevato automaticamente.
  • Personalizzato: sebbene non sia elencato inizialmente nel menu a discesa, puoi creare formati log personalizzati, che verranno visualizzati automaticamente nel menu a discesa se vengono configurati correttamente.

Se ritieni che il tuo server web attualmente non produca log in uno dei formati predefiniti riconosciuti, puoi riconfigurare il server web in modo che effettui la registrazione in uno di questi formati, oppure puoi creare un formato log personalizzato che sia conforme al metodo di registrazione corrente del server web. Se desideri riconfigurare la registrazione del server web, scegli la registrazione W3C o NCSA.

Bilanciamento del carico ed elaborazione log in parallelo

Nei siti web caratterizzati da un elevato traffico l'utilizzo di un solo server web potrebbe non essere sufficiente. Utilizzando il bilanciamento del carico, il traffico può essere distribuito tra più server web che lavorano in parallelo. L'insieme di server web utilizzati in parallelo è denominato cluster. In questo modo la registrazione delle attività di ciascun sito web avviene in uno qualsiasi dei log in parallelo disponibili. La procedura guidata per l'installazione del profilo fornisce un'opzione di elaborazione log in parallelo. Se Elaborazione log in parallelo è attivato, Urchin apre tutti i file di log contemporaneamente e li legge a rotazione, una sezione alla volta; ciascuna sezione corrisponde a 15 minuti di attività di log. L'attivazione dell'elaborazione log in parallelo aumenta significativamente le prestazioni sui siti con bilanciamento del carico.
