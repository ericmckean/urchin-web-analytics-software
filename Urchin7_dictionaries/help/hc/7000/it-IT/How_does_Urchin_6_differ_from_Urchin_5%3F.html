In cosa si differenzia Urchin 6 da Urchin 5?

Questo articolo descrive le differenze tra Urchin 5 e Urchin 6.

Funzioni principali

  • Profili illimitati, sorgenti log e monitoraggio della campagna compresi nella licenza di base; nessun modulo aggiuntivo
  • Funzioni complete di monitoraggio delle campagne SEO/SEM, 4 obiettivi per profilo
  • Disponibilità della segmentazione incrociata potenziata nella maggior parte dei rapporti
  • Suite completa di rapporti sull'area geografica dei visitatori (non solamente sul dominio dei visitatori)
  • Velocità di elaborazione pari a quella di Urchin 5, ma con volumi di dati più elevati

Piattaforme supportate

  • Supporto di un'ampia gamma di piattaforme Linux con due sole build (Linux 2.4 e 2.6 kernels)
  • Supporto di FreeBSD 5 e FreeBSD 6
  • Cessazione del supporto di MacOS X e Solaris

Programmi di installazione

  • Il programma di installazione Windows viene ora distribuito come pacchetto MSI, inoltre viene fornita una migliore assistenza per l'installazione non presidiata e l'integrazione con SMS

Configurazione

  • Database relazionale back end (MySQL o PostgreSQL)
  • Supporto di database di configurazione ospitati su server di configurazione remoti

Server web

  • Upgrade all'ultima versione di Apache 1.3.X
  • Upgrade alle ultime versioni di OpenSSL e mod-ssl
  • Rimozione dei moduli predefiniti non utilizzati da Urchin
  • Aggiunta di mod_expires per i cache control header corretti

Programma di pianificazione attività

  • Il programma di pianificazione viene ora suddiviso in due processi: programma di pianificazione master e programma di pianificazione slave
  • Semplice gestione delle attività tramite un'interfaccia script con DB di configurazione back end

Monitoraggio dei visitatori

  • Il codice di monitoraggio javascript __utm.js di Urchin 5 è stato sostituito dal codice di monitoraggio javascript urchin.js, compatibile con Google Analytics. Se monitorizzi il sito con __utm.js, dovrai ricodificare il sito con urchin.js.

Elaborazione log

  • Archiviazione dei dati geografici nella memoria (maggiore footprint di momoria durante l'esecuzione)
  • La funzione Intervallo di giorni nelle sorgenti log consente la ricerca in più giorni di file di log corrispondenti a un determinato pattern di date
  • Esecuzione completa di profili nella memoria

Dati e memorizzazione

  • Per impostazione predefinita, i database dei profili memorizzano 100.000 record al mese (invece di 10.000) con la possibilità di aumentare il volume fino a 500.000 record al mese

  • Geodata espansi: un insieme completo di dati sulle località geografiche di Quova, sostituisce i dati MaxMind solo per il dominio

  • Limite di registrazione tabelle mensile aumentato dal valore predefinito di 10.000 a 100.000 record
  • I 50 file mensili per profilo sono ora organizzati per sottodirectory

Interfaccia utente dei rapporti

  • Flash sostituisce Adobe SVG per la rappresentazione di grafici e prospetti
  • Esportazione dei rapporti solo in CSV e XML (l'esportazione in MS Word/Excel non viene più supportata perché non affidabile)
  • Il rapporto Tutti i profili non è più disponibile

E-commerce

  • Le transazioni e-commerce possono essere scritte direttamente sui log del server web tramite funzioni speciali nel codice javascript di monitoraggio (identico a GA)
  • Supporto anche dei log dei carrelli degli acquisti esterni in formato ELF2

Sicurezza

  • Test accurati dell'esecuzione di script su più siti (XSS) e dei punti vulnerabili XSRF
