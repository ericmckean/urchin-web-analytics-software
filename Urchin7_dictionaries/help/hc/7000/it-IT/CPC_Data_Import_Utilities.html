Utilità per l'importazione dei dati CPC (Urchin 6.5 o versioni successive)

Urchin 6.5 include nuove utilità per l'elaborazione dei dati CPC.

urchincpc - Lo strumento Urchin CPC Data Importing estrae dati CPC tramite l'API di Google AdWords e di Yahoo! Search Marketing per consentire un'ulteriore elaborazione dei dati e la generazione di rapporti.

  • Nota. I dati CPC archiviati, scaricati da AdWords, non possono eccedere i 250 MB (Urchin 6.501+).

urchincpcd - Il programma di pianificazione Urchin CPC Source Scheduler (o semplicemente CPC Scheduler) viene utilizzato per programmare attività di download di dati CPC sulla base di una pianificazione definita dall'utente (analogamente al programma di pianificazione Profile Scheduler).

Il programma di pianificazione CPC Scheduler (urchincpcd) viene eseguito su un host locale insieme al programma Urchin Master Scheduler (urchind). Master Scheduler deve essere installato e attivato affinché sia possibile pianificare sorgenti CPC. CPC Scheduler verifica periodicamente la coda delle attività di download dei dati CPC e ne controlla l'esecuzione.

Lo strumento CPC Importing (urchincpc) viene avviato automaticamente se sono presenti attività pianificate. CPC Scheduler genera i processi di lavoro internamente, tuttavia è possibile richiamare direttamente tali processi tramite la riga di comando per avviare l'esecuzione delle attività disponibili in coda.

I dati CPC scaricati vengono archiviati a livello locale nel percorso /path/to/urchin/data/cpc/ e automaticamente prelevati dal motore di Urchin durante l'elaborazione dei profili collegati.

Urchin CPC Scheduler viene gestito dall'utilità urchinctl. Per gestire esclusivamente CPC Source Scheduler, utilizza l'opzione "-c". Ad esempio, 

urchinctl -c start/stop/restart

avvia/arresta/riavvia solo il programma di pianificazione CPC Scheduler.

Per pianificare un download immediato di dati CPC, apri la scheda Configurazione -> Gestione sorgenti CPC -> Scarica o Modifica -> Pianificazione per una particolare sorgente CPC, poi fai clic sul pulsante Scarica adesso. Tieni presente che CPC Scheduler deve essere installato e attivato affinché sia in grado di elaborare la richiesta.

Uso: CPC Scheduler - urchincpcd

Il programma di pianificazione CPC Scheduler è disponibile nella cartella di installazione bin di Urchin 6.5 o versioni successive. Tramite la riga di comando sono disponibili le seguenti opzioni:

  • -d attiva la modalità di debug. Ulteriori informazioni sul debug vengono fornite nei risultati. In Windows, ad esempio, funziona solo con 

    •            urchincpcd.exe -console -d

  • -h stampa informazioni della guida e chiude il processo

  • -v stampa il numero di versione e chiude il processo

In Windows è necessario specificare uno dei seguenti comandi come primo argomento:

  • -install per installare il servizio Windows CPC scheduler

  • -remove per rimuovere il servizio Windows CPC Scheduler

  • -start per avviare il servizio Windows CPC Scheduler

  • -stop per interrompere il servizio Windows CPC Scheduler

  • -console per avviare il programma di pianificazione CPC Scheduler in modalità console

Uso: strumento CPC Importing - urchincpc

Tramite la riga di comando sono disponibili le seguenti opzioni:

  • -c [id] scarica la sorgente di costo specificata, dove "id" rappresenta l'ID della sorgente CPC

  • -d attiva la modalità di debug

  • -h stampa informazioni della guida e chiude il processo

  • -v stampa il numero di versione e chiude il processo

Tieni presente che, se CPC Scheduler viene avviato senza parametri o con la sola opzione -d, inizierà a elaborare tutte le attività disponibili in coda e chiuderà il processo una volta che non saranno più presenti attività da elaborare.

Download di dati CPC tramite proxy (6.501 e versioni successive)

Per attivare il download dei dati CPC tramite proxy, modifica il file urchin.conf, rimuovi i commenti dalle voci WebServiceProxyHost, WebServiceProxyPort, WebServiceProxyUsername, WebServiceProxyPassword e fornisci i valori desiderati. 

 ######################################################
# Impostazioni Gestione sorgenti CPC
#...
# WebServiceProxyHost, WebServiceProxyHost: Host e porta di HTTP/HTTPS
# Gestione sorgenti CPC deve utilizzare un proxy per accedere alle sorgenti CPC esterne.
# WebServiceProxyUsername, WebServiceProxyPassword: nome utente e password
# proxy che richiede l'autorizzazione.
######################################################
#WebServiceProxyHost: <proxy host>
#WebServiceProxyPort: <proxy port>
#WebServiceProxyUsername: <username>
#WebServiceProxyPassword: <password>

Per WebServiceProxyHost, puoi inserire un indirizzo IP o il nome del dominio completo.
