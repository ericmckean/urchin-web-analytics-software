Utilizzo di UTM con gli alias di dominio

Premessa

I cookie sono oggetti basati sui domini, pertanto occorre tenere in considerazione alcuni aspetti importanti se un sito ha più domini. Un cookie impostato sotto un dominio, ad esempio "miosito.it", viene trasmesso a tutti i sottodomini, come "www.miosito.it". Tuttavia, questo cookie non viene trasmesso a "miosito.net" o a qualsiasi altro dominio principale differente.

Se il tuo sito web ha solo un dominio che corrisponde a "miosito.it" e "www.miosito.it", puoi eseguire l'installazione UTM standard. Tuttavia, se il tuo sito web utilizza uno o più alias, è consigliabile reindirizzare il traffico dagli alias al sito principale. Ciò garantisce che il monitoraggio dei visitatori UTM venga impostato sotto il dominio principale e che tutti i visitatori siano monitorati in modo coerente.

In caso contrario, un visitatore può essere considerato come due visitatori se accede allo stesso sito tramite due domini sperati. Le seguenti istruzioni forniscono un esempio di come reindirizzare i domini con alias al dominio principale nei server Apache e IIS.

Reindirizzamento degli alias in Apache

Se utilizzi un server web Apache, la configurazione può essere facilmente modificata in modo da reindirizzare al sito principale tutto il traffico originato sotto uno degli alias. Un modo per effettuare questa operazione è di creare due voci VirtualHost. Il primo sarà il dominio principale che includerà la configurazione normale; il secondo VirtualHost sarà per tutti gli alias e reindirizzerà al dominio principale. Esempio: 

#---primary virtualhost
<VirtualHost 1.2.3.4>
Nome server www.miosito.it
Alias server miosito.it
...
</VirtualHost>
#---second virtualhost
<VirtualHost 1.2.3.4>
Nome server miosito.org
Alias server ww.miosito.org miosito.net www.miosito.net
RewriteEngine on
RewriteRule ^(.*)       http://www.miosito.it$1     [R=301]
</VirtualHost>

La seconda voce VirtualHost utilizza una regola di riscrittura con un codice di reindirizzamento 301 (trasferito in modo permanente) per inoltrare tutto il traffico al sito originale. Un singolo hit 301 viene ugualmente registrato nel file di log, ed è utile per monitorare i domini ai quali accedono i visitatori, ma tutto il traffico rimanente viene indirizzato sotto il singolo dominio. A questo punto, per quanto riguarda UTM, il sito appare come un sito con un dominio ed è pronto per l'installazione UTM standard.

Nota. Tieni presente che è necessario eseguire queste operazioni con il tuo amministratore e fare riferimento al sito apache.org per i parametri di configurazione.

Reindirizzamento degli alias in IIS

Se utilizzi un server web Microsoft IIS, la configurazione può essere facilmente modificata in modo da reindirizzare al sito principale tutto il traffico originato sotto uno degli alias. Un modo per effettuare questa operazione è di creare due siti web nella configurazione IIS. Il primo sarà il dominio principale (www.miosito.it), che includerà la configurazione normale; il secondo sarà per tutti gli altri alias (miosito.net, miosito.org ecc.) e reindirizzerà al dominio principale.

In IIS Manager fai clic con il pulsante destro del mouse su uno dei siti web e visualizza la finestra di dialogo delle proprietà. Nella scheda Sito web fai clic sul pulsante "Avanzato...". Viene visualizzata la finestra in cui è possibile assegnare ulteriori domini al sito web utilizzando il campo "Intestazione host". Imposta il dominio principale nel sito web principale e utilizza il secondo sito per ospitare tutti gli alias.

Una volta configurato e abilitato il secondo sito web che ospita tutti gli alias, crea una home page vuota con il seguente codice di reindirizzamento: 

<head>
<META HTTP-EQUIV=Refresh CONTENT="0; URL=http://www.miosito.it/">
</head>

In questo modo il browser del visitatore effettuerà immediatamente il reindirizzamento all'URL principale. A questo punto, il sito web principale appare come una semplice configurazione a singolo dominio ed è possibile eseguire l'installazione UTM standard con le impostazioni predefinite.
