Come faccio a monitorare tutti i sottodomini del mio sito in un unico profilo?

  • Articolo di ivantage Limited,consulente autorizzato del software Urchin

È possibile monitorare i sottodomini all'interno dello stesso profilo del dominio. A tal fine, devi aggiungere una riga al codice di monitoraggio su ciascuna pagina del sottodominio. Nelle istruzioni seguenti si presuppone che questo profilo sia stato impostato utilizzando il sito web indicato come dominio principale (es. esempio.it).

  1. Aggiungi la seguente riga (in grassetto) al codice di monitoraggio di tutte le pagine del tuo sottodominio e del tuo dominio principale: 

         <script src="/urchin.js" type="text/javascript">
         </script>
         <script type="text/javascript">
           _udn="example.co.uk";
          </script>
  • Questa riga dovrebbe fare riferimento al dominio principale (esempio.it).

Tieni presente che, mentre questa modifica ti consente di monitorare i sottodomini all'interno di un unico profilo, i rapporti potrebbero non fare distinzione tra le pagine provenienti da www.esempio.it e quelle provenienti da guida.esempio.it. Ad esempio, nel rapporto Contenuti principali, potrebbero essere presenti hit di www.esempio.it/indice.html e guida.esempio.it/altro.html, ma nel rapporto verrebbe visualizzato quanto segue: 

   /index.html
   /more.html

Se desideri distinguere tra i sottodomini, puoi creare un filtro Avanzato per il tuo profilo con le seguenti impostazioni: 

   Filter Type: Custom filter > Advanced
   Field A: cs_host (Raw)
   Extract A: (.*)
   Field B: Request_Stem (Auto)
   Extract B: (.*)
   Output To: Request_Stem (Auto)
   Constructor: /$A1$B1

Con questo filtro attivato, gli esempi precedenti verrebbero visualizzati con il sottodominio: 

   www.esempio.it/indice.html
   guida.esempio.it/altro.html

NOTA. Questo filtro si basa sul nome host del tuo sito web, incluso il sottodominio scritto nel tuo file di log per l'accesso web. Se il tuo file di log per l'accesso web non contiene il nome host, Urchin 6 non sarà in grado di registrare nei rapporti la differenza tra, ad esempio, index.html nei due domini: 

 www.esempio.it/indice.html
 guida.esempio.it/altro.html
