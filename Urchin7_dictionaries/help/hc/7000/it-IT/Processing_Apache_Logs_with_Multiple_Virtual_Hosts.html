Elaborazione di log Apache con più host virtuali

Introduzione

Le organizzazioni possono disporre di server web Apache con più host virtuali, nei quali la registrazione relativa a tutti i siti web viene effettuata su un solo file di log. Mentre per Urchin è più pratico elaborare un singolo log per ogni host virtuale, è possibile configurare dei profili per ogni host virtuale che ha letto il singolo log combinato.

Procedura

Passaggio 1. Assicurati che il tuo server web Apache sia configurato per la creazione di log nel formato idoneo per Urchin. Verifica in particolare che i campi seguenti siano presenti nel log nell'ordine indicato.

  1. Host remoto [%h]
  2. Host richiesto [%v]
  3. Nome utente [%u]
  4. Data e ora (NCSA) [%t]
  5. Richiedi (NCSA) [%r]

  6. Stato [%>s]

  7. Byte [%b]
  8. Referrer [%{Referer}i]
  9. User-Agent [%{User-Agent}i]
  10. Cookie (se registri i cookie, ad esempio per UTM) [%{Cookie}i]

Passaggio 2. All'interno dell'interfaccia di amministrazione di Urchin, vai a Configurazione -> Profili Urchin -> Gestione log e aggiungi una sorgente log che indichi l'unico file di log. Scegli "auto" per il formato del log.

Passaggio 3. In Configurazione -> Profili Urchin, crea un nuovo profilo per ogni host virtuale. Nella procedura guidata Aggiungi profilo, scegli "Aggiungi sorgente log esistente" quando aggiungi la sorgente log per ogni profilo, quindi seleziona la sorgente log configurata in precedenza.

Passaggio 4. A questo punto vai a Gestione filtri e fai clic sul pulsante Aggiungi per aggiungere un nuovo filtro. Nella schermata "Procedura guidata filtro: Impostazioni" seleziona "Includi SOLO pattern" per Tipo filtro, "cs_host(RAW)" per Campo filtro e immetti il nome dell'host virtuale nel campo Pattern filtro. Assegna quindi questo filtro al profilo corrispondente.

Passaggio 5. Ripeti i passaggi 3 - 4 per ogni host virtuale.

Considerazioni

Questa procedura è comoda per fornire i rapporti per molteplici host virtuali, tuttavia richiede che Urchin rielabori l'intero file di log per ogni host virtuale. In caso di file di log di grandi dimensioni, questo può comportare un sovraccarico considerevole e influire negativamente sulle prestazioni di Urchin. È fortemente consigliabile optare per la creazione di un file di log distinto per ogni host virtuale.

Sebbene questa procedura sia scritta in modo specifico per Apache, il processo dovrebbe essere simile per altri formati di log di server web che effettuano la registrazione nello stesso tipo di campi NCSA.

Al momento Urchin non è in grado di rilevare automaticamente i nuovi host virtuali e di configurare automaticamente i relativi nuovi profili. Tutti i profili devono essere impostati in modo esplicito.
