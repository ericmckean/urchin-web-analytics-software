Gestione log

Panoramica

La gestione log è un fattore importante quando si esegue un software come Urchin. Poiché i siti con un elevato traffico creano file di log di grandi dimensioni abbastanza velocemente (in alcuni casi fino a diversi gigabyte in un solo mese), è necessario considerare attentamente la gestione log. Si consiglia di praticare una rotazione log standard. La compressione e l'archiviazione dei file offline sono procedure standard. Per ulteriori informazioni su questa procedura, consulta la sezione Best practice per la rotazione log. La gestione log è necessaria solo per considerazioni relative all'utilizzo delle risorse del disco, non per evitare la rielaborazione dei dati. Urchin non necessita di alcun tipo di rotazione log per evitare la duplicazione dei dati, in quanto dispone di funzionalità di monitoraggio log che garantiscono che i dati di log letti in precedenza non vengano rielaborati. Poiché Urchin non dovrebbe mai avere la necessità di rileggere un file di log dopo essere stato elaborato, a tua discrezione puoi eliminare i log dopo ogni elaborazione. Tuttavia, non è raro mantenere i vecchi log per un periodo di tempo specificato per motivi storici o di verifica.

Gestione log tramite Urchin

Ogni sorgente log dispone di un'impostazione Destino log con le opzioni Non toccare, Archivio/Compressione ed Elimina(Configurazione -> Profili Urchin -> Gestione log -> Modifica (accanto a ciascun profilo) -> scheda Impostazioni avanzate). Dopo che tutti i profili che utilizzano una sorgente log hanno completato l'elaborazione, Urchin utilizza l'impostazione Destino log per determinare la posizione della sorgente log. L'impostazione Destino log è accessibile dalla scheda Impostazioni avanzate relativa a una data sorgente log. Si consiglia di impostare Destino log su Archivio/Compressione per risparmiare spazio su disco nel caso in cui desideri mantenere i log per un periodo di tempo. Se consideri accettabile il fatto che il log viene rimosso dopo essere stato elaborato, puoi scegliere il destino log Elimina. Tuttavia, tieni presente che in questo caso non avrai la possibilità di rieseguire Urchin su quel log in futuro, a meno che tu non disponga di una copia di backup in un'altra posizione.

Considerazioni

Tieni presente le seguenti situazioni particolari:

  • Non utilizzare le opzioni Archivio o Elimina con una sorgente log se stai elaborando log attivi. Un log attivo è un log nel quale è attivamente in corso la scrittura da parte di un server web. L'utilizzo di queste impostazioni con un log attivo causa una perdita di dati.
  • Se il destino per una sorgente log recuperata in modalità remota è impostato su Non toccare, quel log crescerà continuamente a meno che non esista un processo esterno a Urchin che gestisce la gestione log sul computer in cui viene creato il log. Poiché Urchin deve trasferire una copia del file di log remoto al sistema locale prima dell'elaborazione, con l'aumento delle dimensioni del file di log aumenterà anche il tempo impiegato da Urchin per trasferire il file. Ciò provocherà come effetto secondario l'allungamento del tempo di esecuzione generale di Urchin.
