Registrazione: altri server web

Panoramica

Questo articolo fornisce una breve panoramica su come configurare la registrazione per i server web diversi da Apache e IIS per eseguire elaborazioni e creare rapporti in modo corretto per Urchin. Urchin elabora il log di qualsiasi server web, a condizione che possa capire il modo in cui i dati sono organizzati in ciascuna voce del file di log. Le informazioni incluse in questo articolo si applicano solo ai log di accesso. Se sei interessato ai dettagli della registrazione e-commerce, consulta la sezione relativa al modulo E-commerce del Centro documentazione.

Per la produzione di rapporti in Urchin esiste un insieme minimo di campi obbligatori, a prescindere dal tipo di server web o dal formato di registrazione. Sono:

  • Data
  • Ora
  • Nome host o indirizzo IP del sistema richiedente
  • Richiesta (ad esempio quale documento ha richiesto il sistema al tuo server web)
  • Codice di stato generato dalla richiesta (valore numerico)
  • Byte (byte trasferiti dal server al client)

Per creare rapporti completi sono inoltre necessari i seguenti campi:

  • Referral
  • User Agent
  • Cookie (se Urchin Traffic Monitor è installato nel tuo sito)

Configurazione

Le specifiche su come apportare modifiche alle caratteristiche di registrazione di tutti i server web sarebbero troppo lunghe da elencare. In generale, l'approccio più facile consiste nel configurare la registrazione in modo da renderla conforme al modulo W3C o NCSA di Urchin, quindi scegliere il formato predefinito appropriato dal menu a discesa Formato log nella sorgente log. Se il server web può supportare questo approccio, consulta il documento Registrazione: Apache e IIS. Le informazioni contenute nel suddetto documento relative alla modalità di impostazione dei campi di dati potrebbero risultare utili, sebbene i dettagli sui metodi di modifica non si applichino necessariamente al tuo server web.
