Uso delle sorgenti log UNC

Quando viene creata una sorgente log UNC in Urchin, sono specificati i seguenti parametri: Server Name/IP, Remote Server Login, Local Server Login, Remote Log Source File.

Il flusso utilizzato da Urchin per ottenere il log da una sorgente UNC è il seguente: - Urchin prova a cercare i file specificati in "Remote Log Source File"; - Se la ricerca non riesce, Urchin prova a creare una connessione alla condivisione UNC, specificata nel campo "Server Name/IP"; - Se la connessione non riesce, Urchin presume che venga eseguita con un secondo account utente che non ha accesso di rete, ad esempio l'account del sistema locale. In questo caso, crea una sessione di logon per account utente, specificata in "Local Server Login" e impersona sé stesso come questo account, ad esempio il processo Urchin inizia ad essere seguito con questo account. Se le opzioni "Same Local e Remote Login" sono impostate su "Sì", le credenziali di impersonificazione sono prese da "Remote Server Login"; - Se l'impersonificazione riesce, Urchin prova nuovamente a creare una connessione UNC alla condivisione di rete e riprova a cercare il file. - Una volta che Urchin ha terminato con una sorgente log, chiude la connessione UNC e termina l'impersonificazione, ovvero il processo Urchin inizia ad essere eseguito con l'account nel quale era in esecuzione prima dell'impersonificazione.

L'impersonificazione presenta i seguenti inconvenienti: - Mentre è attiva, il processo Urchin è in esecuzione con un account differente, il quale potrebbe non aver accesso alla directory di dati di Urchin e ciò potrebbe portare alla mancata creazione di database dei rapporti. - I database dei rapporti, creati mentre l'impersonificazione era attiva potrebbero non essere accessibili per il processo Urchin una volta terminata l'impersonificazione. - Rallenta le prestazioni, in particolare quando più file di log UNC sono allegati al profilo, in quanto Urchin deve riavviare/terminare l'impersonificazione per ciascun file.

Pertanto, le best practice consigliate per l'utilizzo delle condivisioni UNC in Urchin sono: - Eseguire Urchin nell'account che ha accesso alla rete. In questo caso l'impersonificazione non si verifica - Concedere l'accesso alle condivisioni dei log UNC all'account utente nel quale è in esecuzione Urchin. In questo caso, la creazione della connessione UNC e dell'impersonificazione potrebbero non essere necessarie.
