Requisiti di configurazione per Urchin 6

  • Articolo di EpikOne, consulente autorizzato del software Urchin

Questo articolo elenca gli aspetti operativi da considerare prima di installare ed eseguire Urchin. Alcune delle informazioni indicate sono obbligatorie per il corretto funzionamento di Urchin. Altri elementi sono importanti per utilizzare Urchin nella maniera più efficace una volta installato il software.

Considerazioni di base sull'installazione di Urchin

  • In Windows devi eseguire l'installazione dopo avere eseguito l'accesso come Amministratore.
  • Sui sistemi UNIX puoi eseguire l'installazione come qualsiasi utente, ma se l'installazione non viene eseguita come utente avanzato, saranno presenti limitazioni in relazione alle aree del file system in cui eseguire l'installazione.
  • Urchin viene fornito con un server web Apache integrato per la configurazione e la consegna dei rapporti. Gli amministratori di sistema devono sapere che questo nuovo servizio web verrà eseguito dopo l'installazione di Urchin.
  • È necessario installare Urchin 6 in un disco con abbondante spazio disponibile (ovvero, diverse centinaia di megabyte come minimo) per consentire la crescita del database Urchin nel corso del tempo. Per ulteriori considerazioni, consulta la sezione Problematiche relative a gestione e prestazioni.

  • Se esegui l'upgrade da Urchin 5, sarà necessario importare il database in Urchin 6 tramite le utilità uconf2sql e convert-u5data. Non è possibile eseguire l'upgrade diretto da Urchin 5 a Urchin 6 semplicemente avviando il programma di installazione di Urchin 6. Consulta Upgrade nella sezione Guida introduttiva del Centro documentazione.

  • L'upgrade da Urchin 5 a Urchin 6 richiede una nuova registrazione della licenza del prodotto.

Considerazioni di base sull'elaborazione di Urchin

  • Accesso ai log dei server web: devi conoscere il percorso dei file di log di un determinato sito e devi essere autorizzato ad accedere a questi file. Se i log si trovano su un sistema remoto, occorreranno anche un nome account e una password da utilizzare per il recupero dei log (FTP/HTTP/HTTPS/UNC).
  • Formato log correttamente configurato: benché Urchin sia un grado di elaborare formati log personalizzati, semplificherai le operazioni di gestione configurando il tuo server web in modo che i log siano in formato standard. Si consiglia di utilizzare il formato Extended Combined Log Format (ad esempio log Apache o NCSA) oppure il formato W3C Extended Log Format (ad esempio, log IIS). Per i siti IIS, il log di Process Accounting deve essere disattivato. Per ulteriori considerazioni, consulta la sezione Configurazione avanzata.
  • Account utente univoco per i processi Urchin: sui sistemi UNIX, è opportuno potenziare la protezione configurando l'esecuzione dei programmi Urchin con un ID utente speciale utilizzato esclusivamente per Urchin e con privilegi limitati. La configurazione di tale account richiede privilegi elevati o di utente avanzato sul sistema in questione.
  • Pianificazione: dovrai pianificare l'esecuzione dell'elaborazione Urchin allo scopo di consegnare i rapporti in maniera tempestiva, oltre che per tenere traccia del tempo necessario per l'elaborazione nel caso di set di dati di grandi dimensioni.

Considerazioni sull'elaborazione avanzata di Urchin

  • Se desideri implementare il monitoraggio dei visitatori unici, procedi nel seguente modo:
    • Installa il codice sensore UTM nelle pagine web del tuo sito.
    • Attiva la registrazione dei cookie nel formato log del tuo server web.
    • Imposta su UTM la metodologia di monitoraggio nel profilo Urchin del sito web.
  • Se scegli di non utilizzare il monitoraggio dei visitatori unici, considera il livello di granularità desiderato per i rapporti relativi a utenti e sessioni e seleziona il metodo di monitoraggio dei visitatori alternativo per ogni singolo sito. Il solo altro metodo di monitoraggio possibile in Urchin 6 è IP/User-Agent.

Problematiche relative a gestione e prestazioni

  • Rotazione log: se non disponi di una funzionalità esterna per l'archiviazione o la rimozione dei log del server web una volta che essi sono stati elaborati da Urchin, puoi configurare Urchin perché esegua questa attività nelle impostazioni avanzate per ogni sorgente log.
  • Mantenimento di precedenti database Urchin per rapporti cronologici: una volta creati i database per un mese specifico, essi saranno da quel momento disponibili per l'analisi cronologica. Gli utenti devono considerare quanto a lungo conservare i dati cronologici, in modo da essere in grado di pianificare l'eliminazione dei dati non necessari per risparmiare spazio su disco. Urchin può essere configurato in modo da comprimere i database precedenti a una data specifica.
  • Requisiti di memoria: Urchin dispone di comandi di configurazione per limitare la quantità di RAM utilizzata per l'elaborazione dei log. L'impostazione predefinita è 128 MB, valore che può risultare troppo limitato per siti con log di dimensioni superiori a 10 MB. Prevedere di avere sufficiente memoria RAM del sistema in maniera tale da essere in grado di aumentare l'utilizzo di memoria di Urchin e regolare le impostazioni di memoria del software al fine di ottenere le massime prestazioni di elaborazione.
  • Ubicazione dell'archiviazione dei dati Urchin: utilizzando il file etc/urchin.conf, è possibile configurare Urchin in modo che i database rapporti siano archiviati in un'area del file system all'esterno della distribuzione Urchin. In questo modo potrai allocare spazio su disco sufficiente per gestire la crescita del database nell'ubicazione più comoda.

Problematiche di accesso remoto e integrazione

  • Utilizzo di SSL per l'amministrazione e i rapporti Urchin: il server web Apache incluso in bundle con Urchin è compilato per il supporto di SSL. La configurazione non attiva SSL per impostazione predefinita, tuttavia SSL può essere attivato quando l'utente lo desidera.
  • Configurazione firewall: se la topologia di rete include firewall, proxy server e altri elementi che si frappongono tra il server di elaborazione Urchin e gli utenti che desiderano visualizzare i rapporti o sistemi che archiviano log da recuperare, sarà necessario configurare questi dispositivi in maniera tale che non interferiscano con l'accesso remoto di Urchin. In genere, è possibile eseguire questa configurazione senza sovvertire le funzionalità di sicurezza assicurate da tale topologia.
