Elenco dei codici di errore per i trasferimenti FTP e HTTP non riusciti del log del server web remoto

Panoramica

È possibile configurare una sorgente log Urchin in modo che raccolga il log di un server web da un server remoto tramite FTP o HTTP. In circostanze normali, il trasferimento viene eseguito correttamente e non viene registrato alcun errore nel log di runtime. Tuttavia, se durante il trasferimento viene rilevato un errore (ad esempio nome utente/password non validi, server remoto non raggiungibile, log remoto non leggibile ecc.), Urchin registra un codice di errore nell'output di runtime, visualizzabile nella Cronologia attività relativa al profilo. Il codice di errore viene visualizzato tra parentesi accanto al messaggio di operazione non riuscita dopo che è stato effettuato il tentativo di trasferimento del log del server web, ad esempio (-9).

I codici sono elencati di seguito insieme a un messaggio di testo che spiega il problema riscontrato.

Elenco dei codice di errore

CURLE_UNSUPPORTED_PROTOCOL (1): l'URL passato a libcurl utilizza un protocollo non supportato da questo libcurl. Potrebbe trattarsi di un'opzione di compilazione non utilizzata, di una stringa di protocollo con ortografia errata o semplicemente di un protocollo di cui libcurl non ha il codice.

CURLE_FAILED_INIT (2): inizializzazione del codice non riuscita. Si tratta probabilmente di un errore o di un problema interno.

CURLE_URL_MALFORMAT (3): l'URL non è stato formattato correttamente.

CURLE_COULDNT_RESOLVE_PROXY (5): impossibile risolvere il proxy. Non è stato possibile risolvere l'host del proxy specificato.

CURLE_COULDNT_RESOLVE_HOST (6): impossibile risolvere l'host. Non è stato possibile risolvere l'host remoto specificato.

CURLE_COULDNT_CONNECT (7): connessione() all'host o al proxy non riuscita.

CURLE_FTP_WEIRD_SERVER_REPLY (8): dopo aver eseguito la connessione a un server FTP, libcurl si aspetta di ricevere una determinata risposta. Questo codice di errore implica che la risposta ricevuta è anomala o non valida. Probabilmente il server remoto specificato non è un server FTP valido.

CURLE_REMOTE_ACCESS_DENIED (9): l'accesso alla risorsa specificata nell'URL è stato negato. Per FTP, ciò avviene quando si tenta di passare alla directory remota.

CURLE_FTP_WEIRD_PASS_REPLY (11): dopo aver inviato la password FTP al server, libcurl si aspetta una risposta corretta. Questo codice di errore indica che è stato restituito un codice non previsto.

CURLE_FTP_WEIRD_PASV_REPLY (13): libcurl non è riuscito a ottenere un risultato valido dal server in risposta a un comando PASV o EPSV. Il server non è valido.

CURLE_FTP_WEIRD_227_FORMAT (14): i server FTP restituiscono una riga 227 in risposta a un comando PASV. Se libcurl non riesce ad analizzare la riga, viene restituito questo codice.

CURLE_FTP_CANT_GET_HOST (15): errore interno durante la ricerca dell'host utilizzato per la nuova connessione.

CURLE_FTP_COULDNT_SET_TYPE (17): è stato ricevuto un errore durante il tentativo di impostare la modalità di trasferimento su binary o ascii.

CURLE_PARTIAL_FILE (18): un trasferimento di file è stato più breve o più lungo del previsto. Ciò accade quando il server prima segnala le dimensioni previste per il trasferimento, poi invia dati che non corrispondono alle dimensioni specificate in precedenza.

CURLE_FTP_COULDNT_RETR_FILE (19): risposta anomala a un comando "RETR" o completamento di un trasferimento a zero byte.

CURLE_QUOTE_ERROR (21): durante l'invio di comandi "QUOTE" personalizzati al server remoto, uno dei comandi ha restituito un codice di errore 400 o superiore (per FTP) o ha indicato in altro modo il completamento non riuscito del comando.

CURLE_HTTP_RETURNED_ERROR (22): questo codice viene restituito se CURLOPT_FAILONERROR è impostato su TRUE e il server HTTP restituisce un codice di errore >= 400. Questo codice di errore era precedentemente conosciuto come CURLE_HTTP_NOT_FOUND.

CURLE_WRITE_ERROR (23): si è verificato un errore durante la scrittura dei dati ricevuti in un file locale oppure è stato restituito un errore a libcurl da un callback di scrittura.

CURLE_UPLOAD_FAILED (25): avvio del caricamento non riuscito. Per FTP, questo codice indica in genere che il server ha negato il comando STOR. Il buffer di errore contiene di norma la spiegazione del server relativa all'errore. Questo codice di errore era precedentemente conosciuto come CURLE_FTP_COULDNT_STOR_FILE.

CURLE_READ_ERROR (26): si è verificato un problema durante la lettura di un file locale o di un errore restituito dal callback di lettura.

CURLE_OUT_OF_MEMORY (27): richiesta di allocazione di memoria non riuscita. Se mai dovesse essere restituito questo errore, significa che si è verificato un problema grave.

CURLE_OPERATION_TIMEDOUT (28): timeout operazione. È stato raggiunto il periodo di timeout in base alle condizioni specificate.

CURLE_FTP_PORT_FAILED (30): il comando FTP PORT ha restituito un errore. Nella maggior parte dei casi ciò accade se non è stato specificato un indirizzo valido per l'utilizzo da parte di libcurl. Vedi CURLOPT_FTPPORT.

CURLE_FTP_COULDNT_USE_REST (31): il comando FTP REST ha restituito un errore. Ciò non dovrebbe mai accadere se il server è integro.

CURLE_RANGE_ERROR (33): il server non supporta o non accetta richieste di intervalli.

CURLE_HTTP_POST_ERROR (34): si tratta di un errore obsoleto che si verifica principalmente a causa di un problema interno.

CURLE_SSL_CONNECT_ERROR (35): si è verificato un problema in un punto dell'handshake SSL/TLS. È consigliabile leggere il messaggio riportato nel buffer di errore poiché fornisce ulteriori dettagli sul problema. Potrebbe essere un problema di certificato (formati di file, percorsi, autorizzazioni), password o altro.

CURLE_FTP_BAD_DOWNLOAD_RESUME (36): tentativo di riprendere il trasferimento FTP ignorando le dimensioni del file.

CURLE_FILE_COULDNT_READ_FILE (37): impossibile aprire un file specificato da FILE://. Molto probabilmente il percorso del file non identifica un file esistente. Controlla le autorizzazioni all'uso del file.

CURLE_LDAP_CANNOT_BIND (38): impossibile abbinare LDAP. Operazione di abbinamento LDAP non riuscita.

CURLE_LDAP_SEARCH_FAILED (39): ricerca LDAP non riuscita.

CURLE_FUNCTION_NOT_FOUND (41): funzione non trovata. Impossibile trovare una funzione zlib richiesta.

CURLE_ABORTED_BY_CALLBACK (42): operazione interrotta dal callback. Un callback ha restituito "abort" a libcurl.

CURLE_BAD_FUNCTION_ARGUMENT (43): errore interno. È stata chiamata una funzione con un parametro non valido.

CURLE_INTERFACE_FAILED (45): errore dell'interfaccia. Impossibile utilizzare un'interfaccia in uscita specificata. Imposta l'interfaccia da utilizzare per l'indirizzo IP sorgente delle connessioni in uscita con CURLOPT_INTERFACE. Questo codice di errore era precedentemente conosciuto come CURLE_HTTP_PORT_FAILED.

CURLE_TOO_MANY_REDIRECTS (47): troppi reindirizzamenti. Seguendo i reindirizzamenti libcurl ha raggiunto il numero massimo impostato. Imposta il limite desiderato con CURLOPT_MAXREDIRS.

CURLE_UNKNOWN_TELNET_OPTION (48): un'opzione impostata con CURLOPT_TELNETOPTIONS non è nota o non è stata riconosciuta. Consulta la documentazione appropriata.

CURLE_TELNET_OPTION_SYNTAX (49): una stringa di opzione telnet è stata formattata illegalmente.

CURLE_PEER_FAILED_VERIFICATION (51): il certificato SSL o l'impronta digitale SSH md5 del server remoto non è stata considerata valida.

CURLE_GOT_NOTHING (52): non è stato restituito niente dal server e in queste circostanze, ciò è considerato un errore.

CURLE_SSL_ENGINE_NOTFOUND (53): il crypto engine specificato non è stato trovato.

CURLE_SSL_ENGINE_SETFAILED (54): impossibile impostare il crypto engine SSL selezionato come valore predefinito.

CURLE_SEND_ERROR (55): impossibile inviare i dati di rete.

CURLE_RECV_ERROR (56): impossibile ricevere i dati di rete.

CURLE_SSL_CERTPROBLEM (58): si è verificato un problema con il certificato client locale.

CURLE_SSL_CIPHER (59): impossibile utilizzare la crittografia specificata.

CURLE_SSL_CACERT (60): impossibile autenticare il certificato peer con i certificati CA noti.

CURLE_BAD_CONTENT_ENCODING (61): codifica di trasferimento non riconosciuta.

CURLE_LDAP_INVALID_URL (62): URL LDAP non valido.

CURLE_FILESIZE_EXCEEDED (63): dimensioni massime del file superate.

CURLE_USE_SSL_FAILED (64): richiesta del livello FTP SSL non riuscita.

CURLE_SEND_FAIL_REWIND (65): durante un'operazione di invio curl ha dovuto riavvolgere i dati da ritrasmettere, ma l'operazione non è riuscita.

CURLE_SSL_ENGINE_INITFAILED (66): inizializzazione del motore SSL non riuscita.

CURLE_LOGIN_DENIED (67): il server remoto ha negato l'accesso a curl (aggiunto alla versione 7.13.1).

CURLE_TFTP_NOTFOUND (68): file non trovato sul server TFTP.

CURLE_TFTP_PERM (69): problema di autorizzazioni sul server TFTP.

CURLE_REMOTE_DISK_FULL (70): spazio su disco esaurito sul server.

CURLE_TFTP_ILLEGAL (71): operazione TFTP non valida.

CURLE_TFTP_UNKNOWNID (72): ID trasferimento TFTP sconosciuto.

CURLE_REMOTE_FILE_EXISTS (73): il file esiste già e non verrà sovrascritto.

CURLE_TFTP_NOSUCHUSER (74): questo errore non dovrebbe mai essere restituito da un server TFTP che funziona correttamente.

CURLE_CONV_FAILED (75): conversione carattere non riuscita.

CURLE_CONV_REQD (76): il chiamante deve registrare i callback di conversione.

CURLE_SSL_CACERT_BADFILE (77): problema di lettura del certificato CA SSL (percorso, diritti di accesso ecc.).

CURLE_REMOTE_FILE_NOT_FOUND (78): la risorsa a cui si fa riferimento nell'URL non esiste.

CURLE_SSH (79): si è verificato un errore durante la sessione SSH.

CURLE_SSL_SHUTDOWN_FAILED (80): impossibile chiudere la connessione SSL.
