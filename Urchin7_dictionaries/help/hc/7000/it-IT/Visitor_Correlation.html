Correlazione visitatori

Panoramica

La correlazione visitatori è il processo di identificazione del comportamento dei visitatori anche se le sessioni hanno origine da file di log differenti o da server web o di E-commerce indipendenti. Urchin utilizza i dati di varie sorgenti log per analizzare le relazioni tra le sessioni e le transazioni e mette a confronto queste informazioni per fornire una visione completa della correlazione tra l'attività dei visitatori e agli acquisti, fornendo quindi un rapporto dettagliato del ritorno sull'investimento. Ad esempio, i referral di motori di ricerca e parole chiave specifiche possono essere correlati con l'importo degli acquisti effettuati nel tuo sito di E-commerce per fornirti informazioni sui referral e i termini di ricerca che producono la maggior parte delle entrate. In genere hai a disposizione un log per il tuo sito web pubblico e un altro log dal tuo sito web protetto per le transazioni. Per correlare tra loro origini dati differenti, Urchin deve utilizzare lo stesso metodo di identificazione dei visitatori per ciascun sito.

Tipi di correlazione visitatori

  • Urchin è in grado di effettuare la correlazione visitatori basandosi su vari metodi. La scelta del metodo di monitoraggio dei visitatori influisce direttamente sull'accuratezza delle informazioni che Urchin ha a disposizione per correlare le transazioni di E-commerce con altre attività del sito web. Scegli il metodo più adatto al livello di dettaglio desiderato.

Per ciascun metodo di monitoraggio dei visitatori è necessario includere i seguenti dati nel file di log di E-commerce:

  • UTM: cookie
  • IP+User-Agent: host remoto o indirizzo IP e User Agent (ad esempio, tipo di browser)

È necessario prendere in considerazione questi elementi quando esamini il formato di registrazione E-commerce. Per ulteriori informazioni su come garantire la correttezza dei dati nei tuoi log, consulta le sezioni ELF2 Log Format e Log personalizzati di E-commerce.

Configurazione

Questa sezione fornisce una panoramica generale dei problemi di configurazione della correlazione visitatori per l'e-commerce.

Siti senza UTM

Se non hai installato un sensore UTM nei tuoi siti, la configurazione della correlazione visitatori dipende dal formato di e-commerce utilizzato.

  • ELF2: utilizza "IP+User Agent" come metodo di monitoraggio dei visitatori. In questo caso, Urchin correla automaticamente le varie sessioni se tutti i log contengono i campi IP e User Agent. Nella schermata "Rapporti del profilo", scegli "IP+User Agent" come metodo di monitoraggio dei visitatori.

Siti con UTM

Per i siti abilitati per UTM, è necessario installare la stessa versione del sensore UTM in tutte le pagine che desideri monitorare. Nella pagina Impostazioni profilo, imposta il "Metodo monitoraggio visitatore" su Urchin Traffic Monitor. Generalmente devi lasciare vuoto il dominio UTM, poiché UTM individua automaticamente il dominio per il sito. La funzione "Dominio UTM" viene utilizzata unicamente in caso di domini coincidenti che devono essere monitorati separatamente.
