Requisiti hardware e piattaforme supportate da Urchin

Urchin è compatibile con molte architetture e sistemi operativi. È necessario installare Urchin solo su un sistema che elaborerà i log. Per visualizzare i rapporti, è richiesto solo un browser web.

Piattaforme supportate

Windows

  • Windows 2003 Server
  • Windows XP
  • Windows 2000
  • Windows Vista
  • Windows 2008 Server

UNIX-type Systems

  • Linux (kernel 2.6), generico per la maggior parte delle versioni utilizzate
  • Linux (kernel 2.4), generico per la maggior parte delle versioni utilizzate
  • FreeBSD 4 (non supportato per 6.5+).
  • FreeBSD 5.3+ (non supportato per Urchin 6.602+).
  • FreeBSD 6.2+

Supported databases

  • MySQL 4.1+ (non supportato per Urchin 6.6+).
  • MySQL 5.03+
  • PostgreSQL 8.2.5

Browser supportati

  • Internet Explorer 6.x, 7.x
  • Firefox 2.x, 3.x
  • Chrome 1.x, 2.x

Nota sulle piattaforme non esplicitamente supportate

Se non è disponibile una distribuzione di Urchin per una determinata piattaforma, è possibile utilizzare una distribuzione di Urchin compatibile con il sistema operativo in uso.

  • Windows 98, Windows 3.x:

Urchin non può essere installato su Windows 98 o 3.x, ma queste piattaforme consentono di visualizzare i rapporti con Internet Explorer (a partire dalla versione 5.x).

  • Linux:

Esistono molte versioni diverse di Linux, non tutte esplicitamente supportate da una specifica distribuzione di Urchin. Tuttavia, le diverse versioni di Linux dimostrano solitamente un elevato livello di compatibilità, pertanto è probabile che lo siano anche con una delle distribuzioni di Urchin. Le seguenti distribuzioni sono sicuramente compatibili:

  • Ubuntu (kernel 2.6)

  • RedHat Enterprise Linux 2 & higher

  • SuSE Linux 8 & higher

  • Trustix 2 & higher

  • Solaris:

Solaris non è ancora supportato.

Requisiti di sistema per Urchin 6

Urchin fornisce prestazioni ottimali anche con dotazioni hardware ridotte. Ad esempio, persino l'ormai superato Pentium II che potrebbe risultare inadeguato per un computer desktop, può supportare discretamente un server Urchin. Per ottenere prestazioni ottimali, occorre disporre della seguente dotazione hardware:

Analisi di singoli siti web piccoli e medi

  • Processore da almeno 2 Ghz
  • 2 GB RAM
  • Hard disk IDE da almeno 250 GB
  • Interfaccia Ethernet da 100 Mbps

Fornitore di servizi/Installazioni aziendali

  • 2.5Ghz Pentium dual-core/2Ghz UltraSPARC/PPC/MIPS/ecc. con velocità CPU simili
  • 3GB RAM
  • Disco rigido Ultra2/Wide SCSI (ad esempio Seagate Cheetah)
  • Ethernet 100base-T
  • Sistema di backup

Memoria/Sistema/Utilizzo del disco

  • Urchin può essere configurato in modo da utilizzare 20-500 MB di memoria (RAM) (20 - 1024 MB per Urchin 6.5)
  • Urchin può essere configurato per essere eseguito con priorità bassa, normale o elevata
  • La memorizzazione dei dati Urchin richiede approssimativamente il 10% delle dimensioni dei log non elaborati

Nota importante per gli utenti di FreeBSD

(1) La codifica hardware di FreeBSD fornisce una capacità di elaborazione massima di 500 MB. Questo limite viene superato perché i dati geografici di Urchin devono risiedere nella memoria durante l'elaborazione dei log. Di conseguenza viene visualizzato il seguente errore di runtime: 

ERROR: (8011-323-1057) Failed to allocate memory

Per risolvere questo problema, è necessario incrementare la capacità di elaborazione dati predefinita del sistema FreeBSD, aggiungendo nel file /boot/loader.conf le seguenti righe: 

# Increase max process data segment size to 1GB
kern.maxdsiz="1073741824"

Al termine, riavvia il computer.

AVVISO!!! Fai molta attenzione quando modifichi i limiti del sistema. Se imposti dimensioni eccessive per kern.maxdsiz il sistema potrebbe non riavviarsi. Ti consigliamo di consultare la documentazione di FreeBSD prima di apportare questa modifica e valutare i potenziali rischi per la configurazione specifica del tuo sito.

(2) FreeBSD dispone di un driver speciale denominato accf_http. Tale driver esegue il buffering delle connessioni in ingresso fino all'arrivo di determinate richieste HTTP complete, per consentire una maggiore velocità e l'ottimizzazione delle prestazioni. Per impostazione predefinita, questo driver non è caricato nel kernel e viene visualizzato il seguente avviso: 

[warn] (2)No such file or directory: Failed to enable the
‘httpready’ Accept Filter

Per evitare questo problema, è necessario attivare il modulo (accf_http) del kernel di FreeBSD HTTP Accept Filter, aggiungendo al file /boot/loader.conf le seguenti righe: 

# Wait for full HTTP request accept filter
accf_http_load="YES"

Al termine, riavvia il computer.
