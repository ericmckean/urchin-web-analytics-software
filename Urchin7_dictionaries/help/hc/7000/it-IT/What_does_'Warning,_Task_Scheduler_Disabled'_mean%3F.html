Significato del messaggio "Avviso. Programma di pianificazione attività disabilitato".

Il programma di pianificazione delle attività di Urchin è responsabile dell'esecuzione del programma di elaborazione di Urchin. Se il programma di pianificazione delle attività è disabilitato, Urchin può eseguire l'elaborazione. I motivi comuni per cui viene visualizzato l'errore: Avviso. Programma di pianificazione attività disabilitato" sono i seguenti:

  • Il servizio urchind non è in esecuzione. Questo può accadere quando il server viene riavviato e Urchin non è incluso negli script di avvio del sistema.
  • (Solo Unix) Il servizio urchind è in esecuzione, ma è di proprietà di un utente diverso dall'utente proprietario della distribuzione Urchin.

Per determinare se il programma di pianificazione delle attività è in esecuzione, procedere nel seguente modo:

  • Apri un pannello di comandi e specifica il comando CD per la directory /path/to/urchin/bin/.
  • digita: urchinctl status (su Unix, digita ./urchinctl status)

Se il servizio del programma di pianificazione delle attività non è in esecuzione, puoi iniziare dalla riga di comando eseguendo le seguenti attività:

  • Apri un pannello di comandi e specifica il comando CD per la directory /path/to/urchin/bin/.
  • digita: urchinctl -s start (su Unix, digita ./urchinctl -s start)
  • se entrambi i servizi non sono in esecuzione, digita: urchinctl start

Se il servizio è in esecuzione, controlla i processi in esecuzione sul server per determinare chi è il proprietario del processo Urchin come indicato d seguito: ps -ef | grep urchind

Nota lo "User-Name" (nome utente) del proprietario del processo nell'output restituito: 

User-Name 33597 0.0 0.2 352 204 ?? Ss 27Feb08 17:42.18 /usr/local/urchin6/bin/urchind

Se l'utente non è lo stesso dell'utente proprietario della distribuzione Urchin, dovrai arrestare il servizio e riavviarlo sotto l'utente appropriato. Questa operazione può essere eseguita attraverso una "commutazione utente" sullo stesso utente che al momento è proprietario del processo urchind, e arrestando urchinctl. Utilizza la commutazione "-s" come indicato in precedenza. Quindi ritorna all'utente Urchin e avvia urchinctl, utilizzando di nuovo la commutazione "-s".
