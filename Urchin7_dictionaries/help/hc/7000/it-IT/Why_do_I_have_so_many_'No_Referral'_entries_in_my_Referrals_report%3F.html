Perché mi appaiono molte voci "Nessun referral" nel rapporto Referral?

La voce "Nessun referral" nel rapporto Referral di Urchin è dovuta generalmente a una o più delle seguenti condizioni:

  • I file di log del server web non sono formattati correttamente.
  • I visitatori accedono al sito direttamente (digitando l'URL o impostando il sito come segnalibro).
  • I visitatori accedono al sito direttamente facendo clic sul link contenuto in un'email.
  • Gli spider web dei motori di ricerca identificano le pagine web.
  • Referral interni derivano dalle richieste di pagine della cache (non applicabile se Urchin Traffic Monitor (UTM) è configurato correttamente).

Formato log: per consentire a Urchin di mostrare i referral nei rapporti, è necessario configurare i file di log in modo che includano un campo "Referral". Per ulteriori dettagli sulla configurazione dei file di log Apache o IIS, consulta la sezione Registrazione: Apache e IIS.

Indirizzo diretto: se un visitatore digita l'indirizzo del tuo sito web in una barra degli indirizzi, non verrà trasmessa nessuna informazione sui referral al server web. Ogni richiesta di file, o "hit", inviata a un server web viene registrata in un file e ciascuna voce presenta un campo "Referral" che può contenere dati o meno. Quando l'accesso a una pagina viene effettuato inserendo l'indirizzo manualmente come descritto sopra, non viene registrata alcuna informazione nel campo "Referral", perché il visitatore non è stato indirizzato da un link in un altro sito.

Link di email: se un visitatore fa clic sul link contenuto in un messaggio email (ad esempio, una campagna pubblicitaria email), i dati dei referral potrebbero non venire registrati. È possibile creare link di email che trasmettono le informazioni sui referral al server web in cui risiede il sito, ma solo chi crea il contenuto dell'email può impostarli di conseguenza.

Spider web: crawler, robot e spider dei motori di ricerca sono in genere la causa della maggior parte degli hit "Nessun referral". Si tratta di programmi che visitano i siti web e leggono le pagine e altre informazioni in essi contenute per creare le voci di un indice per i motori di ricerca.

Referral interni: Urchin non include i referral interni nei rapporti, se sono configurati correttamente. Un referral interno è un referral dello stesso sito web che contiene la prima pagina richiesta. Se un referral interno viene rilevato nel campo "Referral", verrà incluso nel rapporto come hit "Nessun referral". È raro che un referral interno venga visualizzato nella richiesta iniziale di una pagina web, tranne che in un caso specifico.

Molti provider Internet (ISP) di grandi dimensioni e siti di motori di ricerca utilizzano i server cache per acquisire i contenuti dei siti web. In questo modo possono ridurre il tempo e il traffico di rete necessari a mostrare le pagine ai visitatori più velocemente. Tuttavia, quando un visitatore richiede la pagina web e alla richiesta risponde un server cache, al log del tuo server web non vengono trasmesse le informazioni sulla richiesta. Di conseguenza, neanche il file di log del server web conterrà informazioni sulla richiesta. Il risultato finale di questa azione è che un nuovo visitatore spesso non verrà individuato dal tuo server web prima della seconda o terza richiesta della pagina. A quel punto, l'accesso alle pagine richieste verrà eseguito da link o altre pagine del tuo sito e nel campo "Referral" del file di log verranno visualizzate le tue pagine come referral. Da ciò derivano gli hit dei referral interni che urchin aggiunge al rapporto "Nessun referral". Se configuri Urchin Traffic Monitor (UTM), Urchin 6 può recuperare i dati del file di log mancanti. Per ulteriori informazioni, consulta la sezione Funzionamento di UTM (Urchin Traffic Monitor).
