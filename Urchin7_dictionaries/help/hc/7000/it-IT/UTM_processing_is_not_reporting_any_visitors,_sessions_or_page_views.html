L'elaborazione UTM non riporta visitatori, sessioni né pagine visualizzate

Questo articolo descrive problemi comuni di configurazione UTM. Per risolverli, è necessario:

  • accedere alla configurazione di Urchin
  • disporre di almeno un file di log attuale e completo, creato dopo aver configurato e installato UTM.

Innanzitutto, verifica di avere configurato Urchin e il sito come descritto nella documentazione Installazione rapida di UTM (Apache) o Installazione rapida di UTM (IIS).

Problemi di configurazione più comuni

  • Negli Strumenti di amministrazione, modifica il tuo profilo e fai clic sulla scheda Impostazioni. Verifica che il valore per "Dominio UTM" nella scheda dei rapporti corrisponda al valore della variabile "_udn" nel file __utm.js, nella root dei documenti del sito web. Se il valore di "Dominio UTM" non è impostato oppure non è impostato correttamente non verranno registrate sessioni, né pagine visualizzate. In condizioni normali lascia vuoto questo campo. UTM individuerà automaticamente il dominio per il sito.

  • Verifica che il file __utm.gif sia installato nella root dei documenti del sito web e di avere impostato i permessi in modo da consentirne l'accesso. Se il file __utm.gif è mancante o inaccessibile, i tentativi di accesso al file vengono registrati nel log del server web con codice di stato 404 (non trovato). Conseguentemente, non verranno registrate sessioni o pagine visualizzate.

  • Verifica che tutte le pagine del sito web contengano la riga SCRIPT che richiama il file __utm.js. Confronta il rapporto Ottimizzazione contenuto > Analisi navigazione > Frequenza rimbalzo entrate di Urchin 6 con il rapporto Rapporti IT > Pagine e file > File principali. Se ti rendi conto che gli utenti accedono alle tue pagine, ma non vengono registrate entrate è molto probabile che il file __utm.js non viene richiamato.

  • Utilizza la versione corretta del file __utm.js nella root dei documenti del sito web. Per tutte le versioni di Urchin 5 e Urchin 6, la versione corretta del file __utm.js è posizionata nella directory util/utm della distribuzione Urchin.

NOTA. In Urchin 6 è possibile utilizzare il filtro di esclusione dei file gif senza influenzare i rapporti sui visitatori in quanto esclude tutti i file gif a eccezione di __utm.gif
