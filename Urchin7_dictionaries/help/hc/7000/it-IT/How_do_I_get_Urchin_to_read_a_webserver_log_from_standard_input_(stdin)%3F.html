Come faccio a fare in modo che Urchin legga un log del server web dall'input standard (stdin)?

Può essere opportuno, in alcune circostanze, fare in modo che il motore di elaborazione dei log di Urchin sia in grado di leggere i dati del log del server web direttamente dall'input standard (stdin) invece di leggere un file effettivo. Urchin supporta la configurazione di una sorgente log che legge dall'input standard invece che da un file effettivo.

Per configurare Urchin a eseguire questa operazione, come prima cosa configura una nuova sorgente log con il nome "stdin" in Gestione sorgente log, utilizzando i parametri seguenti:

Impostazioni log

  • Ubicazione file di log: local
  • Percorso file di log: -
  • Formato log: auto

Impostazioni avanzate

  • Destino log: Non toccare
  • Token query: ?
  • Stem degli URI in minuscolo: disattivato

NOTA. È molto importante che "Percorso file di log" sia indicato solo con un carattere di trattino "-" (senza virgolette).

Ora, per ogni profilo per il quale desideri leggere i log da "stdin", scegli la nuova sorgente log "stdin" nella schermata di configurazione Profilo -> Sorgente log.

Nota importante. Non puoi eseguire i profili Urchin configurati con una sorgente log "stdin" mediante il programma di pianificazione Urchin in quanto tale programma non è in grado di associare correttamente lo stream dell'input log. Il solo modo per elaborare i log per un profilo con una sorgente log "stdin" è l'esecuzione manuale di Urchin. A titolo di esempio molto semplice, ecco come richiamare Urchin:

  1. Apri un pannello di comandi sul sistema Urchin
  2. Modifica la directory della distribuzione Urchin in "bin"

  3. [Sistemi di tipo UNIX] Esegui urchin con "./urchin -p myprofile < /path/to/webserver.log"

    • [Sistemi Windows] Esegui urchin con "urchin -p mioprofilo < C:\path\to\webserver.log"

L'utilizzo di "stdin" quale sorgente log ti consente una grande flessibilità, in quanto puoi facilmente eseguire una parte della pre-elaborazione prima che Urchin la legga.

Ulteriori esempi

Estrai gli hit per un solo host virtuale da un log costituito da hit di molti diversi host virtuali e fornisci i risultati a Urchin: 

grep "www.miodominio.com" /path/to/alldomains.log | urchin -p mydomain.com

Estrai gli hit non elaborati del server web da un database relazionale utilizzando uno script SQL incorporato e forniscili a Urchin per l'elaborazione 

sql-extract --date 20040302 --domain mydomain.com | urchin -p mydomain.com
