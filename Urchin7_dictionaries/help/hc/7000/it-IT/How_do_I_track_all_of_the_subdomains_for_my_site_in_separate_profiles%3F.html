Come faccio a monitorare tutti i sottodomini del mio sito in profili separati?

  • Articolo di ivantage Limited,consulente autorizzato del software Urchin

Con Urchin, il monitoraggio dei sottodomini in profili individuali dipende dalla configurazione del tuo server web. È possibile configurare il server web in modo da registrare file di log separati per ogni sottodominio. In questo modo, Urchin è in grado di elaborare il file di log di ciascun sottodominio separatamente. In alternativa, se viene generato un solo file di log e il log di accesso web è configurato per registrare il nome host che include il sottodominio, è possibile creare un filtro di inclusione per ogni profilo del sottodominio. Il filtro in un profilo

  • includerebbe solo il traffico verso, ad esempio, guida.esempio.it, mentre un altro filtro di inclusione potrebbe includere, ad esempio, solo il traffico verso www.esempio.it.

Pertanto, per monitorare i tuoi sottodomini in profili separati, applichi un filtro di inclusione a ogni profilo per includere solo un determinato nome host. Il primo passaggio consiste nella creazione di un profilo per ogni sottodominio. Una volta effettuata questa operazione, puoi creare i filtri riportati di seguito.

Ad esempio, se desideri monitorare sotto.esempio.it e www.esempio.it in due profili distinti, applica i seguenti filtri:

  • Per il profilo sotto.esempio.it 

    Tipo filtro: Includi SOLO pattern
    Campo filtro: utm_request_hostname (AUTO)
    Pattern filtro: ^sotto\.esempio\.it
  • Per il profilo www.esempio.it 

    Tipo filtro: Includi SOLO pattern
    Campo filtro: utm_request_hostname (AUTO)
    Pattern filtro: ^www\.esempio\.it
