Che cosa significa "Execution of urchin.cgi timed out"?

Questo errore viene visualizzato solo quando viene utilizzata la funzione Collegamento diretto rapporto di Urchin e il controller della sessione di Urchin viene ignorato. Puoi verificare osservando l'URL utilizzato quando leggi il rapporto Urchin. Se viene utilizzato il collegamento diretto rapporto, l'URL contiene "report.cgi"; se viene utilizzato il controller della sessione Urchin standard, l'URL contiene "session.cgi".

Quando leggi i rapporti Urchin tramite il collegamento diretto rapporto, il browser esegue una query nei file di database di Urchin tramite l'applicazione report.cgi alla ricerca di dati che corrispondono all'intervallo di date e alla visualizzazione del rapporto richiesti. Se la risposta alla query non viene ricevuta entro il periodo predefinito di 60 secondi, si verifica il timeout dell'applicazione CGI e nel browser viene visualizzato il messaggio di errore "Execution of urchin.cgi timed out". Se si verifica questa situazione, è necessario aumentare il valore di timeout dell'applicazione CGI di conseguenza. La modifica del timeout richiede l'accesso alla distribuzione Urchin dalla riga di comando.

Per aumentare il timeout dell'applicazione per il collegamento diretto rapporto, procedi nel seguente modo:

  • Apri un'interfaccia di riga di comando nel server Urchin.
  • Modifica il file /path/to/urchin/htdocs/.report.conf; se il file non esiste, copia il file .report.conf.sample in .report.conf per crearne uno.
  • Individua la riga "#Timeout = 60".
  • Rimuovi il commento dalla riga eliminando il carattere '#'.
  • Aumenta il valore predefinito di 60 secondi impostandolo su un numero più appropriato.
  • Salva il file. Le modifiche avranno effetto immediatamente.
