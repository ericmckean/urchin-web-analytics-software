Les URL de chaque étape de l'entonnoir de conversion défini sont identiques. Comment puis-je assurer le suivi de chaque étape comme s'il s'agissait d'une URL unique ?

Vous pouvez suivre la progression des utilisateurs dans un entonnoir de conversion avec une URL identique à chaque étape. Votre entonnoir de conversion pour une inscription peut, par exemple, se présenter comme suit :

  • Étape 1 (Inscription)
    • www.monsite.fr/sign_up.cgi
  • Étape 2 (Acceptation du contrat)
    • www.monsite.fr/sign_up.cgi
  • Étape 3 (Fin)
    • www.monsite.fr/sign_up.cgi

Pour contourner le problème, vous pouvez appeler la fonction Javascript urchinTracker à chaque étape (probablement au sein d'un événement onload), de la façon suivante :

  • urchinTracker("/funnel_G1/step1.html")
  • urchinTracker("/funnel_G1/step2.html")
  • urchinTracker("/funnel_G1/step3.html")

Définissez ensuite chaque étape de votre entonnoir de conversion comme suit :

  • http://www.monsite.fr/funnel_G1/step1.html

  • http://www.monsite.fr/funnel_G1/step2.html

  • http://www.monsite.fr/funnel_G1/step3.html

L'argument définissant le chemin d'accès/nom de fichier dans urchinTracker() ne doit pas nécessairement représenter un chemin d'accès ou un nom de fichier existant. L'argument fourni à urchinTracker indique simplement un nom de page auquel Urchin peut associer des consultations de page.
