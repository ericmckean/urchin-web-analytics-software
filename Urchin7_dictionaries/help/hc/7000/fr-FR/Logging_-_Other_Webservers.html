Journalisation - Autres serveurs Web

Présentation

Cet article présente brièvement la configuration requise sur des serveurs autres qu'Apache et IIS pour permettre à Urchin d'assurer convenablement le traitement des données et la création des rapports. Urchin peut analyser tous types de journal pourvu qu'on lui indique l'ordre des données incluses dans les entrées du fichier. Les informations présentées ici s'appliquent uniquement aux journaux d'accès. Si vous souhaitez obtenir des précisions sur la journalisation des transactions en ligne, consultez la section du Centre de documentation consacrée au module E-commerce.

Quel que soit le type de votre serveur Web ou le format de votre fichier journal, Urchin requiert un ensemble minimal de champs pour produire des rapports. En voici la liste :

  • date
  • heure
  • nom d'hôte ou adresse IP du système initiant la demande
  • requête (document auquel le système demandeur souhaite accéder)
  • code d'état généré par la requête (numérique)
  • octets (nombre d'octets transférés du serveur au client)

Pour exploiter au mieux la création de rapports, vous devez en outre disposer des champs suivants :

  • Site référent (referral)
  • User-Agent
  • Cookies, si l'UTM (Urchin Traffic Monitor) est installé sur votre site

Configuration

Il serait excessif de décrire par le menu les procédures de modification de la journalisation pour chaque serveur Web. De manière générale, la meilleure méthode consiste à configurer une journalisation au format Urchin NCSA ou W3C, puis de sélectionner le format par défaut approprié dans le menu déroulant Format du journal associé à la Source du journal. Si vous pouvez procédez de cette façon sur votre serveur Web, reportez-vous à la rubrique Journalisation - Apache et IIS. Cette dernière contient des informations qui pourraient vous être utiles sur la configuration des champs obligatoires, même si les détails des procédures de modification fournies ne s'appliquent pas nécessairement à votre serveur Web.
