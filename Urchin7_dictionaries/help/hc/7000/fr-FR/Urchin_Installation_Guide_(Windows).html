Urchin 6 - Guide d'installation (Windows)

Cet article a été rédigé par EpikOne, consultant Urchin agréé.

Avant de commencer : remarque importante à l'attention des utilisateurs d'Urchin 6.500

Plusieurs problèmes liés au planificateur de CPC sont désormais résolus dans Urchin 6.501. Si vous effectuez une mise à niveau à partir d'Urchin 6.500 vers Urchin 6.602, les données CPC traitées peuvent être incorrectes ou incomplètes, ce qui entraîne des erreurs dans les rapports Urchin. Pour faire en sorte que les données issues de Google AdWords soient correctes, nous vous conseillons soit (a) d'installer Urchin 6.602 à un nouvel emplacement, soit (b) de supprimer les anciennes données CPC si vous souhaitez effectuer l'installation sur la même instance que la version 6.500. Si vous optez pour la seconde solution, supprimez tous les éléments contenus dans les répertoires suivants : 

<path_to_urchin_6500_installation>\data\cpc\...
<path_to_urchin_6500_installation>\data\history\cpc\...

Aucune action n'est requise de votre part si vous effectuez une mise à niveau à partir d'Urchin 6.600, 6.501 ou Urchin 6.402, ou toute version antérieure.

Présentation de la procédure d'installation

  • Installez le serveur de base de données de votre choix sur le système Urchin ou ajoutez la base de données Urchin à un serveur existant. Moteurs de base de données actuellement pris en charge :
    • MySQL 5.0.3 ou version ultérieure (Notez qu'à partir de la version Urchin 6.6, MySQL 4.x.x n'est plus pris en charge.)
    • PostgreSQL 8.2.5 ou version ultérieure
  • Créez la base de données Urchin et l'utilisateur sur le serveur de la base de données, puis accordez les autorisations d'accès.
  • Extrayez la distribution Urchin dans un dossier temporaire.
  • Exécutez le programme d'installation MSI.
  • Récupérez la licence d'installation Urchin.

Option 1 : utilisation d'une base de données MySQL existante avec Urchin

  • Si aucun serveur MySQL n'est configuré sur votre système, vous pouvez obtenir une distribution à l'adresse http://dev.mysql.com/downloads/. Le progiciel Windows Essentials contient tous les composants nécessaires à l'utilisation d'Urchin.

  • Installez le serveur MySQL en respectant les instructions présentées sur le site MySQL.
  • Assurez-vous de connaître les informations de connexion (par exemple, root) et le mot de passe de l'administrateur de la base de données MySQL.
  • Sélectionnez le nom de la base de données, l'utilisateur MySQL local et le mot de passe que vous utiliserez avec Urchin. Notez ces informations, vous en aurez besoin lors de l'installation d'Urchin. L'exemple ci-dessous illustre la création d'une base de données appelée urchin, avec le nom d'utilisateur MySQL urchin et le mot de passe urchinpassword (bien évidemment, vous devez choisir votre propre mot de passe !).

  • Lancez le programme client en ligne de commande MySQL via "Démarrer" -> "Programmes" -> "MySQL" -> "MySQL Server" -> "MySQL Command Line Client".

  • Dans la fenêtre du client en ligne de commande MySQL, créez la base de données Urchin, l'utilisateur et les autorisations à l'aide des commandes suivantes : 

mysql> create database urchin character set utf8;
mysql> GRANT ALL ON urchin.* to 'urchin'@'localhost' IDENTIFIED BY
'urchinpassword';

Option 2 : utilisation d'une base de données PostgreSQL existante avec Urchin

  • Si aucun serveur PostgreSQL n'est configuré sur votre système, vous pouvez obtenir une distribution à l'adresse http://www.postgresql.org.

  • Installez le serveur PostgreSQL en respectant les instructions présentées sur le site PostgreSQL.
  • Assurez-vous de connaître les informations de connexion (par exemple, postgres) et le mot de passe de l'administrateur de la base de données PostgreSQL.
  • Sélectionnez le nom de la base de données, l'utilisateur PostgreSQL local et le mot de passe que vous utiliserez avec Urchin. Notez ces informations, vous en aurez besoin lors de l'installation d'Urchin. L'exemple ci-dessous illustre la création d'une base de données appelée urchin, avec le nom d'utilisateur PostgreSQL urchin et le mot de passe urchinpass (bien évidemment, vous devez choisir votre propre mot de passe !).

  • Lancez le programme client en ligne de commande PostgreSQL via "Démarrer" -> "Programmes" -> "PostgreSQL 8.2" -> "psql to 'postgres'".

  • Dans la fenêtre du client en ligne de commande PostgreSQL, créez la base de données Urchin, l'utilisateur et les autorisations à l'aide des commandes suivantes : 

postgres=# CREATE DATABASE urchin WITH ENCODING 'UTF8';
postgres=# CREATE USER urchin WITH PASSWORD 'urchinpass';
postgres=# GRANT ALL ON DATABASE urchin TO urchin;
postgres=# \connect urchin;
postgres=# CREATE LANGUAGE 'PLPGSQL';

Option 3 : utilisation d'un serveur PostgreSQL inclus avec Urchin

  • Le programme d'installation d'Urchin 6.602 est fourni avec le serveur PostgreSQL, qui est proposé sous forme d'option lors de l'installation d'Urchin.

  • Pour utiliser cette option, le programme d'installation d'Urchin doit être exécuté par un utilisateur disposant de droits d'administrateur. Le programme d'installation vous invitera à fournir le numéro de port et le mot de passe de l'utilisateur "postgres". Remarque : si l'utilisateur "postgres" existe déjà dans le système, le mot de passe défini pour celui-ci doit être fourni. Si l'utilisateur "postgres" n'existe pas déjà dans le système, il sera créé avec le mot de passe fourni.

  • Les limitations suivantes s'appliquent au serveur PostgreSQL inclus :
    • Il ne doit pas être installé en mode mise à niveau.
    • Des droits d'administrateur sont nécessaires pour procéder à l'installation.

Installation d'Urchin

  • Extrayez le contenu du fichier ZIP de la distribution Urchin dans un dossier temporaire.

  • Si vous souhaitez installer le serveur PostgreSQL inclus, assurez-vous qu'aucun serveur PostgreSQL ne se trouve déjà sur le système. Pour effectuer l'installation, sélectionnez l'option "Installer PostgreSQL", puis fournissez les informations relatives au serveur PostgreSQL, telles que le numéro de port et le mot de passe d'utilisateur "postgres". Remarque : si vous rencontrez des problèmes lors de l'installation du serveur PostgreSQL inclus, consultez la page http://www.postgresql.org.

  • Si vous souhaitez utiliser une base de données existante, assurez-vous de disposer du nom de la base de données, du nom d'utilisateur et du mot de passe Urchin (pour MySQL ou PostgreSQL) que vous avez sélectionnés ci-dessus.
  • Urchin 6 autorise une seule instance d'Urchin 6 sur une même machine (Windows uniquement). Si une version d'Urchin est déjà installée sur la machine, il sera seulement possible d'effectuer une mise à niveau (le passage à une version antérieure n'est pas autorisé).
  • Pour réaliser une 'nouvelle' installation, supprimez toute version d'Urchin 6 déjà installée sur la machine cible. Pour supprimer Urchin, procédez comme suit :

    • "Démarrer" -> "Programmes" -> "Urchin 6" -> "Désactiver les services Urchin".

    • Cliquez sur "Démarrer" -> "Paramètres" -> "Panneau de configuration" -> "Ajout/Suppression de programmes".

    • Cliquez sur "Logiciel Urchin de Google", puis sélectionnez "Supprimer".
    • Une fois Urchin supprimé, renommez le dossier d'installation urchin (en C:\Program Files\Urchin6-OLD, par exemple).
  • Urchin 6.602 peut uniquement être utilisée pour la mise à niveau de certaines versions précédentes d'Urchin 6. Les versions ci-dessous peuvent être mises à niveau :
    • Urchin 6.6 - Service Pack 1 : version 6.601
    • Urchin 6.6 - Version anglaise : version 6.600
    • Urchin 6.5 - Service Pack 2 : version 6.501
    • Urchin 6.5 : version 6.500 (voir la remarque importante ci-dessus)
    • Urchin 6 - Service Pack 1 : version 6.402
    • Urchin 6 - Version internationale : version 6.401
    • Urchin 6 - Version anglaise : version 6.400
  • L'arrêt ou la désinstallation de l'instance Urchin 6 existante n'est pas requis dans le cadre des mises à niveau. Le programme d'installation Urchin 6.602 vérifie d'abord la présence d'une installation d'Urchin 6 et la désinstalle. Ensuite, il désinstalle les fichiers et les configurations d'Urchin 6.602.
  • Pour démarrer l'installation d'Urchin 6.602, double-cliquez sur le fichier urchin6602_windows_installer.msi et suivez les invites interactives.

  • Remarque : Si vous disposez d'une version antérieure d'Urchin 6 et souhaitez conserver votre configuration, assurez-vous que l'option "Recréer les tables de base de données spécifiques d'Urchin si elles existent" n'est pas cochée. Dans ce cas, Urchin conservera la configuration précédente. Cependant, il est possible qu'il crée des 'tables supplémentaires' ou des 'colonnes dans les tables existantes'.

  • Vous disposez maintenant d'une version opérationnelle d'Urchin sur votre système. Pour terminer la configuration, indiquez l'URL suivante dans votre navigateur Web de prédilection : 

http://yourserver:<port>

où <port> correspond au port réseau que vous avez choisi au cours de l'installation (le paramètre par défaut est 9999).

  • Si vous souhaitez rétablir vos données de rapports sur les profils à partir d'une version antérieure d'Urchin 6, copiez les dossiers d'historique et de rapports de C:\Program Files\Urchin6-OLD\data vers C:\Program Files\Urchin6.
  • Cliquez sur le lien "Obtenir une licence de démo" et suivez la procédure d'obtention de licence permettant d'activer Urchin et de lancer la configuration.

Configuration du suivi

Une fois Urchin installé, la configuration du suivi sur votre site Web peut demander des étapes supplémentaires, en fonction de l'utilisation envisagée.

Cas 1 : Urchin et suivi IP+User-Agent

Aucune procédure de configuration supplémentaire n'est nécessaire. Lorsque vous ajoutez des profils dans Urchin, assurez-vous de spécifier IP+User-Agent comme méthode de suivi des visiteurs.

Cas 2 : Urchin et suivi UTM

Pour utiliser Urchin avec la méthode de suivi UTM, vous devez installer un court extrait de code de suivi JavaScript sur chacune des pages de votre site Web. Voici la procédure à suivre :

  • Copiez les fichiers urchin.js et __utm.gif du répertoire "util\utm" de la distribution Urchin et collez-les en racine de votre site Web (par exemple, dans le répertoire principal de votre contenu).

  • Sur chaque page de votre site Web, placez le code de suivi ci-après à la suite des balises META de la section HEAD : 

<script src="/urchin.js" type="text/javascript">
</script>
<script type="text/javascript">
  _userv=0;
  urchinTracker();
</script>
  • Lorsque vous ajoutez des profils dans Urchin, assurez-vous de spécifier Urchin Traffic Monitor (UTM) comme méthode de suivi des visiteurs.

Cas 3 : Urchin et Google Analytics

Notez que les nouvelles fonctionnalités de suivi du code Google Analytics "ga.js" ne sont pas compatibles avec les anciennes versions d'Urchin 6. Si vous souhaitez utiliser simultanément le logiciel Urchin et Google Analytics pour assurer le suivi de votre site Web, vous devrez continuer à utiliser le code de suivi urchin.js pour un fonctionnement optimal. Si vous choisissez d'utiliser ga.js, vous n'aurez pas accès aux nouvelles fonctionnalités de suivi de ga.js.

Pour utiliser Urchin avec un site Web dont le suivi est déjà effectué via Google Analytics, il vous suffit de modifier légèrement le code de suivi Google Analytics. Voici la procédure à suivre :

  • Copiez uniquement le fichier __utm.gif du répertoire "util\utm" de la distribution Urchin et collez-le en racine de votre site Web (par exemple, dans le répertoire principal de votre contenu).

  • Ajoutez la ligne _userv=2; (comme indiqué ci-dessous) au code de suivi des pages de votre site Web. Assurez-vous de ne modifier aucune autre partie du code de suivi existant. 

<script src="http://www.google-analytics.com/urchin.js"
type="text/javascript">
<script type="text/javascript">
  _userv=2;
  _uacct="UA-XXXXXXX-Y";
  urchinTracker();
</script>
  • Lorsque vous ajoutez des profils dans Urchin, assurez-vous de spécifier Urchin Traffic Monitor (UTM) comme méthode de suivi des visiteurs.

Étapes d'installation supplémentaires pour Windows Vista et Server 2008

Outre les versions de Windows déjà prises en charge (2000, XP et Server 2003), Urchin 6.6+ prend également en charge Windows Vista et Server 2008. Certaines fonctionnalités de sécurité présentes dans ces nouvelles versions de Windows peuvent avoir une incidence sur le fonctionnement d'Urchin. Vous trouverez davantage d'explications ci-après.

UAC - problème

Windows 2008 est doté d'une nouvelle fonctionnalité de sécurité appelée "User Access Control (UAC, contrôle de compte d'utilisateur)". Pour plus d'informations sur cette fonctionnalité, consultez la page http://technet.microsoft.com/en-us/library/cc709691.aspx.

L'activation de la fonctionnalité UAC entraîne deux problèmes, dont voici la description :

  1. . Si la fonctionnalité UAC est activée, l'application ne peut pas écrire correctement dans le dossier "Program files". Pour plus d'informations, consultez la page suivante : http://msdn.microsoft.com/en-us/library/aa905330.aspx

  2. . Les exécutables d'Urchin ne peuvent pas être lancés à partir d'une invite de commande par défaut. Vous devez utiliser l'invite de commande en tant qu'administrateur ou définir les propriétés de l'exécutable comme suit : cliquez avec le bouton droit de la souris, puis sélectionnez "Propriétés" -> "Compatibilité"-> "Exécuter ce programme en tant qu'administrateur" -> "OK". Notez que les services Windows d'Urchin fonctionnent comme prévu, sans action supplémentaire de votre part.

UAC - solution

  1. Désactivez la fonctionnalité UAC et configurez Urchin où vous le souhaitez. Exécutez Urchin à l'aide des fichiers binaires ou des services Windows.
  2. Si la fonctionnalité UAC est activée, N'INSTALLEZ PAS Urchin dans le dossier "Program files". Installez-le, par exemple, dans votre dossier d'utilisateur. Modifiez les propriétés afin d'exécuter les fichiers binaires en mode administrateur (comme indiqué au point 2 ci-dessus) ou utilisez les services Windows sans apporter aucune modification.
