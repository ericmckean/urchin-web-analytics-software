Pourquoi la mention "Accès direct" apparaît-elle aussi souvent dans mes rapports sur les sites référents ?

La mention "Accès direct" dans le rapport Urchin sur les sites référents est généralement due aux causes suivantes :

  • Les fichiers journaux de votre serveur Web ne sont pas formatés correctement.
  • Les visiteurs accèdent directement au site (en saisissant l'URL ou en ajoutant le site à leurs favoris).
  • Les visiteurs accèdent directement au site en cliquant sur un lien contenu dans un e-mail.
  • Les pages Web sont identifiées par des spiders de moteur de recherche.
  • Des visites générées par des liens internes à partir de requêtes de page mises en mémoire cache (ne s'applique pas lorsque l'UTM (Urchin Traffic Monitor) est correctement configuré).

Format du journal : pour qu'Urchin puisse créer des rapports sur les sites référents, les fichiers journaux doivent être configurés pour inclure un champ Site référent. Pour plus d'informations sur la configuration des fichiers journaux Apache ou IIS, consultez la section Journalisation - Apache et IIS.

Adresse directe : si un visiteur saisit l'adresse de votre site Web dans la barre d'adresse d'un navigateur, aucune information de site référent ne sera transmise au serveur Web. Chaque demande de fichier (ou "appel de fichier") à un serveur Web est consignée dans un fichier et chaque entrée a un champ de site référent qui peut contenir ou non des données. Lorsque l'accès à une adresse s'effectue manuellement comme décrit plus haut, aucune information n'est enregistrée dans le champ du site référent car le visiteur n'est pas arrivé sur le site en cliquant sur un lien situé sur un autre site.

Lien d'e-mail : lorsqu'un visiteur clique sur un lien contenu dans un e-mail (comme dans le cadre d'une campagne de publicité par e-mails), il se peut qu'aucune information de site référent ne soit transmise. Il est possible de créer des liens d'e-mail qui transmettent les informations de site référent au serveur Web sur lequel réside le site. Mais ceci doit être défini par la personne qui crée le contenu de l'e-mail.

Spiders : les spiders et robots d'indexation utilisés par les moteurs de recherche renvoient la majorité des appels de fichier "Accès direct". Il s'agit de programmes qui visitent les sites Web et lisent leurs pages et autres informations pour créer des entrées pour un index de moteur de recherche.

Visites générées par de liens internes : Urchin n'inclut pas les visites générées par des liens internes dans ses rapports si ceux-ci sont configurés correctement. Une visite générée par un lien interne est une visite générée à partir du site Web auquel appartient la première page demandée. Si une visite générée à partir d'un lien interne est détectée dans le champ de site référent, elle sera incluse dans le rapport comme un appel de fichier "Accès direct". Il peut arriver qu'une visite générée par un lien interne apparaisse dans la requête initiale de page Web. Mais il existe un cas particulier où ce type de visites générées par des sites référents est fréquent.

Un grand nombre de FAI et de sites de moteur de recherche importants utilise des serveurs caches pour capturer les contenus de site Web. Ainsi, ils peuvent réduire le temps et le trafic réseau nécessaire pour proposer plus rapidement ces pages aux visiteurs. Mais lorsqu'un visiteur demande votre page Web et qu'un serveur cache répond à cette requête, le journal de votre serveur Web ne reçoit aucune information sur cette requête. Le fichier journal de votre serveur Web ne contient donc aucune information sur cette requête. Il en résulte qu'un nouveau visiteur ne sera pas vu par votre serveur Web avant la deuxième ou la troisième requête de page. Entre-temps, la page demandée par ce visiteur aura été accédée à partir de liens contenus dans d'autres pages de votre site Web. Le champ du site référent du fichier journal affichera donc comme site référent vos propres pages. Ceci se traduit par les appels de fichiers de visites générées par des liens internes qu'urchin ajoute dans le rapport "Accès direct". Urchin 6 peut réclamer les données de fichier journal manquantes si vous configurez l'UTM (Urchin Traffic Monitor). Pour plus d'informations, consultez la section Fonctionnement de l'UTM (Urchin Traffic Monitor).
