Comment puis-je supprimer une transaction en ligne de mes rapports ?

Pour annuler une commande ou une transaction, localisez la transaction dans vos archives. Puis, créez et chargez une page d'achat dupliquée (__utmSetTrans) contenant le total négatif de la transaction permettant ainsi d'annuler la transaction d'origine. Par exemple, si le montant de la transaction d'origine s'élevait à 699 dollars, l'entrée dupliquée devrait indiquer -699 dollars comme montant total de la transaction.

Pour recréditer partiellement une transaction effectuée, vous devez :

  • créer une transaction utilisant le même identifiant et les mêmes données dans le formulaire de l'article que ceux utilisés pour l'achat ;
  • fournir une valeur négative dans le champ Total du formulaire de la transaction ;

  • indiquer une valeur positive comme prix unitaire et une valeur negative pour la quantité dans le formulaire de l'article ;

  • saisir des valeurs négatives dans les champs appropriés du formulaire de transaction si des frais d'expédition ou des taxes doivent également être recrédités.

Si le montant négatif correspond au total de la transaction, les entrées positive et négative associées apparaissent dans les rapports de transaction lorsque vous sélectionnez une période comprenant l'une ou l'autre de ces transactions. Toutefois, si la période sélectionnée inclut les deux transactions, celles-ci s'annulent et n'apparaissent plus dans le rapport, car le revenu total associé à cet identifiant de transaction est égal à zéro. La quantité négative associée à l'article produit le même effet dans le rapport sur les produits.

L'annulation de commandes qui n'ont pas abouti ou qui ont été refusées, quelle qu'en soit la raison, permet de s'assurer que les informations contenues dans vos rapports Urchin sont exactes.
