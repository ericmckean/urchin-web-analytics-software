Le traitement de l'UTM ne fait état d'aucun visiteur, session ou consultation de page.

L'article suivant décrit les problèmes courants liés à une configuration incorrecte. Pour résoudre ces problèmes, vous devez :

  • accéder à la configuration d'Urchin ;
  • disposer d'au moins un fichier journal existant et complet, créé après configuration et installation de l'UTM.

Dans un premier temps, vérifiez qu'Urchin et votre site sont configurés selon les instructions de la rubrique Installation rapide de l'UTM (Apache) ou Installation rapide de l'UTM (IIS).

Problèmes courants liés à une configuration incorrecte

  • Dans l'outil d'administration, modifiez votre profil et cliquez sur l'onglet Paramètres. Vérifiez que la valeur de l'option "Domaine UTM" sur l'onglet Création de rapports correspond à la valeur utilisée pour la variable "_udn" dans le fichier __utm.js du répertoire racine du site Web. Si la valeur du "Domaine UTM" n'est pas configurée ou l'est de manière incorrecte, aucune session ou consultation de page ne sera enregistrée. En fonctionnement normal, laissez ce champ vide. UTM détecte automatiquement le domaine du site.

  • Assurez-vous que le fichier __utm.gif est installé dans le répertoire racine du site Web et que l'accès y est autorisé. Un résultat ou une autorisation d'accès pour le fichier __utm.gif manque dans les appels du fichier __utm.gif enregistrés dans le journal du serveur Web, provoquant un code d'état 404 (introuvable). Par conséquent, aucune session ni consultation de page ne sera enregistrée.

  • Assurez-vous que toutes les pages du site Web contiennent la ligne de SCRIPT qui appelle __utm.js. Comparez le rapport Optimisation du contenu > Analyse de la navigation > Taux de rebond en entrée fourni par Urchin 6 avec le rapport Rapports IT > Pages et fichiers > Fichiers les plus demandés. Si vous constatez des appels de fichiers de vos pages mais aucun affichage de page, il est fort probable que le fichier __utm.js ne soit pas appelé dans votre page.

  • Utilisez la version adéquate du fichier __utm.js dans le répertoire racine du site Web. Quelle que soit votre version d'Urchin 5 ou d'Urchin 6, le fichier __utm.js approprié se trouve dans le répertoire util/utm de la distribution Urchin.

REMARQUE : Vous pouvez exclure les fichiers gif à l'aide d'un filtre Urchin 6 sans incidence sur les rapports relatifs aux visiteurs, car ce filtre écarte tous les fichiers gif, à l'exception du fichier __utm.gif.
