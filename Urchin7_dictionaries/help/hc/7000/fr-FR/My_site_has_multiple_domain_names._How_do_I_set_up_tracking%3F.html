Mon site comporte plusieurs noms de domaine. Comment puis-je paramétrer le suivi ?

Si votre site utilise un panier sécurisé situé sur un domaine racine différent (par exemple, www.maboutique.fr et www.securecart.com), ou emploie plusieurs alias de domaine pour pointer vers le même contenu, vous devrez apporter des modifications afin de vous assurer que vos sites et l'UTM sont compatibles.

  • Veuillez noter qu'un sous-domaine est différent d'un domaine racine. www.example.fr et securecart.example.com sont des exemples de plusieurs sous-domaines, tandis que www.example.fr et www.sample.com représentent deux domaines racine différents. Si votre site comporte plusieurs sous-domaines, veuillez lire les pages suivantes : Comment puis-je effectuer le suivi des sous-domaines de mon site dans un même profil ? Comment puis-je effectuer le suivi des sous-domaines de mon site dans des profils séparés ? pour obtenir des informations sur la configuration à adopter.

Plusieurs noms de domaines : Alias de domaine

Si votre site utilise plusieurs alias de domaine, il est conseillé de rediriger le trafic des alias vers votre site principal. Ainsi, le suivi des visiteurs par l'UTM sera configuré sur le domaine principal et tous les visiteurs seront suivis de manière cohérente.

Les instructions suivantes fournissent un exemple de redirection des alias de domaine vers le domaine principal sur des serveurs Apache et IIS.

Redirection d'alias sur Apache

  • Si vous utilisez un serveur Web Apache, il est très facile de modifier la configuration afin de rediriger tout le trafic provenant de l'un des alias vers le site principal. Pour cela, vous pouvez créer deux entrées VirtualHost. La première correspondra au domaine principal et inclura votre configuration normale, tandis que la seconde entrée VirtualHost, dédiée à tous les alias, sera redirigée vers le site principal : 

        #---primary virtualhost
        Servername www.monsite.com
        Serveralias monsite.com
        ...
        #---second virtualhost
        Servername monsite.org
        Serveralias www.monsite.org monsite.net www.monsite.net
        RewriteEngine on
        RewriteRule ^(.*) http://www.monsite.com/$1 [R=301]

  • La seconde entrée VirtualHost utilise une règle de réécriture avec un code de redirection 301 (Déplacement définitif) pour transférer tout le trafic vers le site d'origine. À ce stade, du point de vue de l'UTM, le site semble ne comporter qu'un seul domaine et est prêt pour l'installation de l'UTM.

Redirection d'alias sur IIS

  • Si vous utilisez un serveur Web Microsoft IIS, il est très facile de modifier la configuration afin de rediriger tout le trafic provenant de l'un des alias vers le site principal. Pour cela, vous pouvez créer deux sites Web dans la configuration de IIS. Le premier correspondra au domaine principal (www.monsite.com) et inclura votre configuration normale, tandis que le second sera dédié à tous les autres alias (monsite.net, monsite.org, etc.) et redirigera le trafic vers le site principal : Dans le gestionnaire IIS, cliquez avec le bouton droit sur les sites Web pour afficher la boîte de dialogue Propriétés. Sur l'onglet Site Web, cliquez sur le bouton Avancé. Une fenêtre s'ouvre, permettant d'affecter des domaines supplémentaires au site Web dans le champ "En-tête de l'hôte". Définissez le domaine principal dans le site Web principal et utilisez le second site Web pour héberger tous les alias. Une fois que le second site Web hébergeant tous les alias est configuré et activé, créez une page d'accueil vide contenant le code de redirection suivant : 

        <META HTTP-EQUIV=Refresh CONTENT="0; URL=http://www.monsite.com/">
  • Ce fragment de code indiquera au navigateur du visiteur de se rediriger automatiquement vers l'URL principale. À ce stade, le site principal semble ne comporter qu'un seul domaine et l'UTM peut être installé normalement, avec les paramètres par défaut.
