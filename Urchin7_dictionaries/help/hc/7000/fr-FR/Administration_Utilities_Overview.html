Présentation des utilitaires d'administration

Présentation

Urchin comprend un certain nombre d'utilitaires de diagnostic et de configuration situés dans le répertoire util de la distribution Urchin. Ce document a pour objectif de vous donner un aperçu global de ces utilitaires et ne constitue pas un guide d'utilisation exhaustif.

Tous les utilitaires prennent en charge les arguments de ligne de commande "-h" et "-v". Si vous lancez un utilitaire avec l'argument "-h", vous obtenez un résumé de son mode d'utilisation et la liste des options associées à cet outil. Une exécution avec l'argument "-v" permet d'imprimer la version Urchin de l'utilitaire concerné.

geo-update

Cet utilitaire vérifie l'existence de mises à jour pour la base de données DNS interne d'Urchin et les télécharge, le cas échéant. Il permet de télécharger les mises à jour depuis les serveurs de téléchargement Google, ainsi que depuis certains répertoires locaux (nouveauté d'urchin 6).

inspector

Cet utilitaire effectue des contrôles de base sur la distribution Urchin que vous avez installée. Il vérifie que la structure globale de la distribution est intacte, que tous les fichiers binaires livrés avec le produit correspondent à la version appropriée et que les autorisations associées sont correctes (sur les plates-formes de type UNIX). Cet utilitaire indique également l'état de fonctionnement des planificateurs de profils Urchin (urchind), des planificateurs CPC (urchincpcd, urchincpc) et du serveur Web Apache intégré (urchinwebd). Il vérifie en outre les données relatives à la licence.

uconf-manager (Urchin 6.402+)

L'utilitaire de gestion de la configuration Urchin (uconf-manager) vous permet d'ajouter, de modifier et de supprimer des enregistrements de base de données.

  • Urchin 6.402 : cet utilitaire est proposé depuis la version Urchin 6.402 et ne pouvait être utilisé qu'avec la table 'uusers' dans la base de données de configuration. Il permet de réinitialiser le mot de passe admin d'Urchin, d'automatiser la création d'utilisateurs et de déplacer des utilisateurs d'un compte à l'autre.
  • Urchin 6.500+ : dans les versions urchin 6.5 et ultérieures, cet utilitaire prend en charge les opérations de base de données sur un plus grand nombre de tables (uaccounts, ucosts, ufilters, ugoals, ugroups, ugroups_uusers, ulogs, ulogs_ufilters, uprofiles, uprofiles_ucosts, uprofiles_ufilters, uprofiles_ugroups, uprofiles_ulogs, uprofiles_uusers et uusers). Cet utilitaire est utile pour effectuer des opérations qui ne peuvent être exécutées via l'interface d'administration Urchin.

uconf2sql

Urchin 5 et les versions antérieures du programme enregistrent les informations de configuration dans des fichiers propriétaires. En revanche, Urchin 6 stocke ces informations dans une base de données relationnelle (MySql ou PostgreSql). Pour assurer une transition optimale des informations de configuration, Urchin 5 et Urchin 6 incluent des utilitaires permettant l'exportation et l'importation de ces informations. Urchin 5 fournit l'utilitaire uconf-export qui exporte la base de données de configuration dans un fichier (au format XML). Urchin 6 fournit l'utilitaire uconf2sql qui importe les données de configuration dans la base de données d'Urchin 6 à partir de ce fichier XML.

convert-u5data

Tout comme la base de données de configuration, la structure de la base de données de création de rapports (qui stocke les informations de rapports) a été modifiée dans Urchin 6. L'utilitaire convert-u5data propose une option permettant de transférer directement la base de données de création de rapports d'une installation Urchin 5 vers une installation Urchin 6.

udb-sanitizer

L'utilitaire udb-sanitizer permet de réaliser des opérations de gestion sur la base de données de création de rapports. Ces opérations incluent notamment la restauration d'une base de données archivée, la suppression d'informations associées à une période spécifique (lorsqu'un retraitement s'avère nécessaire).

urchinctl

Remarque : Cet utilitaire est situé dans le répertoire bin, non dans le répertoire util. Il permet de démarrer et d'arrêter le planificateur Urchin et les services du serveur Web Urchin. Dans un système de type UNIX, urchinctl est généralement appelé à partir de l'un des scripts de démarrage du système pour démarrer ou arrêter automatiquement les services Urchin. Sur les plates-formes windows, il fournit également des options d'installation et de désinstallation des services urchin.

urchincpc (Urchin 6.5+)

Remarque : Cet utilitaire est situé dans le répertoire bin, non dans le répertoire util. L'outil d'importation des données de CPC Urchin extrait les données de CPC via l'API Google AdWords pour les exploiter et créer des rapports. Pour plus d'informations, consultez la section Utilitaires d'importation des données de CPC.

urchincpcd (Urchin 6.5+)

Remarque : Cet utilitaire est situé dans le répertoire bin, non dans le répertoire util. Le planificateur de sources de CPC Urchin (communément appelé le planificateur CPC) permet de planifier les tâches de téléchargement CPC selon une fréquence définie par l'utilisateur (opération similaire à celle du planificateur de profils). Pour plus d'informations, consultez la section Utilitaires d'importation des données de CPC.

urchin_daemons

Sur les machines de type UNIX, Urchin exécute un démon de planificateur (urchind) et un serveur Web Apache (urchinwebd). Ces programmes doivent être lancés au démarrage du système. Le script urchin_daemons peut être ajouté aux scripts d'initialisation du système dans un emplacement approprié de votre système d'exploitation UNIX. Cela permet aux démons des services Urchin de se lancer correctement. Sur les plates-formes windows, ces fichiers binaires s'exécutent en tant que services du système et peuvent être démarrés et arrêtés à l'aide du gestionnaire de services windows.

ugetlicense

L'utilitaire de gestion des licences Urchin (ugetlicense) permet de réaliser des opérations liées au numéro de série. Ces opérations s'avèrent nécessaires lorsque votre installation Urchin n'est plus disponible et doit être réinstallée ou migrée vers un autre serveur, etc.
