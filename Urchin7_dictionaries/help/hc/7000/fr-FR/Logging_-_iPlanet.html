Journalisation - iPlanet

Présentation

Cet article explique brièvement comment configurer la journalisation sur un serveur Web iPlanet de manière à faciliter le traitement des données et la création des rapports Urchin.

Définissez la Source du journal sur le type "Netscape".

Les champs dont Urchin doit impérativement disposer pour produire des rapports sont les suivants :

  • date
  • heure
  • nom d'hôte ou adresse IP du système initiant la demande
  • requête (document auquel le système demandeur souhaite accéder)
  • code d'état généré par la requête (numérique)
  • octets (nombre d'octets transférés du serveur au client)

Pour exploiter au mieux la création de rapports, vous devez en outre disposer des champs suivants :

  • Site référent (referral)
  • User-Agent
  • Cookies, si l'UTM (Urchin Traffic Monitor) est installé sur votre site

Configuration 

Init fn=flex-init access="$accesslog"
format.access="%Ses->client.ip% - %Req->vars.auth-user%
[%SYSDATE%] \"%Req->reqpb.clf-request%\"
%Req->srvhdrs.clf-status% %Req->srvhdrs.content-length%
\"%Req->headers.user-agent%\"
\"%Req->headers.referer%\"
\"%Req->headers.cookie%\""
