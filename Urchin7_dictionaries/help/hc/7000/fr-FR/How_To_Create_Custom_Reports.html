Création de rapports personnalisés

Pour créer un rapport personnalisé, procédez comme suit :

  1. Choisissez un tableau de carte de données ou définissez-en un nouveau (reportez-vous à Cartes de données personnalisées).

  2. Créez un fichier de rapport (".rs2") en copiant <Urchin>/lib/custom/profiletypes/Default/default.rs2.sample.

  3. Définissez un nouveau rapport en ajoutant une ligne dans le fichier de rapport.

Le format du fichier de rapport est décrit ci-après :

  • ID (identifiant unique du rapport) :
    • C=Catégorie
    • D=Données
    • M=Menu
    • R=Rapport
    • S=Sous-rapport
    • P=Rapport de page
    • B=Tableau de bord
  • RS (tableau de bord/sections de rapports) :
    • 1=dirigeants
    • 2=responsable marketing
    • 4=webmaster
  • RR (exigences des rapports) :
    • 1=Logiciels
    • 2=Avancé
    • 4=Commerce électronique
    • 8=UTM
  • NTITLE : titre de navigation (n d'entrée de dictionnaire)
  • ONCOLOR : couleur de l'élément activé
  • OFFCOLOR : couleur de l'élément désactivé
  • HELP : aide du rapport (n d'entrée de dictionnaire)
  • NDATA : nombre d'éléments de données (les entrées doivent suivre immédiatement après)
    • Ce champ peut être suivi de sous-rapports supplémentaires avec type D (données)
  • CTITLES : titres des colonnes (n d'entrée de dictionnaire)
  • CTYPES : type de colonne utilisé pour la mise en forme
    • 10=chaîne
    • 11=date/mois de l'année
    • 12=date
    • 20=numéro
    • 21=chiffre d'affaires
    • 22=octets
    • 23=ratio de durée
    • 24=ratio
    • 25=pct
    • 26=durée
    • 30=pct total
    • 40=chiffre d'affaires
    • 41=chiffre d'affaires/num
    • 42=num/chiffre d'affaires
  • TABLE (numéro de tableau de données) :
    • T=total
    • TR=transaction
    • H=histogramme
    • V=visiteur
    • Tableau 1-n
  • DXFORMAT : format d'extraction de données
  • RF : champ obligatoire
  • RT : type de rapport
    • 1=totaux du graphique
    • 2=histogramme
    • 3=pile
    • 4=pages les plus consultées
    • 5=arborescence
    • 6=répertoire
    • 7=limites du graphique
    • 8=point
    • 9=clics
    • 10=entonnoir de conversion défini
    • 11=synthèse données/site de la page d'accueil
    • 12=synthèse géographique
    • 13=vérification de l'objectif
    • 14=chemin vers l'objectif
    • 20=rapport dates KPI
    • 21=top 10 KPI
    • 22=résumé des tendances
    • 23=résumé des conversions
    • 24=carte Google Mini
  • GTYPES (types de graphique) :
    • 1=bâtons
    • 2=linéaire
    • 3=circulaire
    • 4=barres horizontales
    • 5=entonnoir de conversion
    • 6=multiligne
  • FSEP : séparateur de champs
  • SP (options spéciales) (ajouter numéros bitmap) :
    • 1=totaux
    • 2=moyennes
    • 4=comparer des dates
    • 8=mettre en minuscules
    • 16=données par période
    • 32=synthèse
    • 64=choix de la langue
    • 128=segment
    • 256=durée de vie
    • 512=détails de la transaction
    • 1024=détail de l'historique du visiteur
    • 2048=recherche d'un terme
  • SR (indicateur de rapport spécial) :
    • 1=suggestions de mots clés
    • 2=clics
    • 3=répartition sur la journée
    • 4=liste des visiteurs
    • 5=synthèse géographique
    • 6=dictionnaire personnalisé
    • 7=comparaison de dates
  • FNT (type de filtre, numéro de filtre) :
    • 1: =

    • 2: >=

    • 3: <=

    • 4: >

    • 5: <

  • FNF : champ de numéro de filtre
  • FNV : valeur de filtre
  • FST (type de chaîne de filtre) :
    • 1=exclure
    • 2=inclure
  • FSF : champ de chaîne de filtre
  • FSX : chaîne d'expression rationnelle
