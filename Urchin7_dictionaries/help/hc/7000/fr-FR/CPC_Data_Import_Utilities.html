Utilitaires d'importation des données CPC (Urchin 6.5+)

Urchin 6.5 inclut de nouveaux utilitaires pour traiter les données CPC.

urchincpc - L'outil d'importation des données CPC Urchin extrait les données CPC via l'API Google AdWords et Yahoo! Search Marketing pour les exploiter et créer des rapports.

  • Remarque : Les données CPC archivées téléchargées depuis AdWords ne doivent pas dépasser 250 Mo (Urchin 6.501+).

urchincpcd - Le planificateur de sources de CPC Urchin (communément appelé le planificateur de CPC) permet de planifier les tâches de téléchargement CPC selon une fréquence définie par l'utilisateur (opération similaire à celle du planificateur de profils).

Le planificateur de CPC (urchincpcd) est exécuté sur un hôte local en même temps que le planificateur maître Urchin (urchind). Le planificateur maître doit obligatoirement être opérationnel pour permettre la planification des sources de CPC. Le planificateur de CPC vérifie régulièrement la file d'attente des tâches de téléchargement des données CPC et contrôle leur exécution.

L'outil d'importation des données CPC (urchincpc) est exécuté automatiquement s'il y a des tâches planifiées. Le planificateur de CPC lance les processus de travail de manière interne. Vous pouvez toutefois accéder à ces processus à partir d'une ligne de commande et exécuter les tâches disponibles dans la file d'attente.

Les données CPC téléchargées sont stockées localement dans le répertoire /path/to/urchin/data/cpc/ et sont automatiquement récupérées par le moteur Urchin lors du traitement des profils associés.

Le planificateur de CPC Urchin est géré par l'utilitaire urchinctl. Pour communiquer uniquement avec le planificateur de sources de CPC, utilisez l'option "-c". Par exemple, 

urchinctl -c start/stop/restart

va démarrer/arrêter/redémarrer uniquement le planificateur de CPC.

Pour planifier un téléchargement immédiat des données CPC, sélectionnez "Configuration" -> "Gestionnaire de sources de CPC" -> "Télécharger" ou "Modifier" -> onglet "Planifier" pour une source de CPC donnée, puis cliquez sur le bouton "Télécharger maintenant". Notez que pour traiter cette requête, le planificateur de CPC doit obligatoirement être opérationnel.

Utilisation : Planificateur de CPC - urchincpcd

Le planificateur de CPC est disponible dans le répertoire bin de votre installation Urchin 6.5+. Vous disposez des options suivantes à partir de la ligne de commande :

  • -d active le mode de débogage. Des informations supplémentaires concernant le débogage sont consignées dans les résultats. Sous Windows, ne fonctionne qu'avec -console, par exemple : 

    •            urchincpcd.exe -console -d

  • -h imprime les informations d'aide et quitte le processus

  • -v imprime le numéro de version et quitte le processus

Pour Windows, entrez l'une des commandes suivantes en tant que premier argument :

  • -install pour installer le service Windows de planificateur de CPC

.

  • -remove pour supprimer le service Windows de planificateur de CPC

.

  • -start pour démarrer le service Windows de planificateur de CPC

.

  • -stop pour arrêter le service Windows de planificateur de CPC

.

  • -console pour démarrer le planificateur de CPC en mode console

Utilisation : Outil d'importation des données CPC - urchincpc

Vous disposez des options suivantes à partir de la ligne de commande :

  • -c [id] télécharge la source de coûts spécifiée, "id" correspondant à l'identifiant de la source de CPC

.

  • -d active le mode de débogage

.

  • -h imprime les informations d'aide et quitte le processus

  • -v imprime le numéro de version et quitte le processus

Si le planificateur de CPC est démarré sans aucun paramètre ou uniquement avec l'option -d, il commence le traitement de toutes les tâches disponibles dans la file d'attente et quitte le processus une fois la dernière tâche traitée.

Téléchargement des données CPC via un serveur proxy (6.501+)

Pour activer le téléchargement des données CPC via un serveur proxy, modifiez le fichier urchin.conf, supprimez le caractère "#" (indiquant un commentaire) dans les entrées WebServiceProxyHost, WebServiceProxyPort, WebServiceProxyUsername et WebServiceProxyPassword, et spécifiez les valeurs de votre choix. 

 ######################################################
# Paramètres du Gestionnaire de sources de CPC
#...
# WebServiceProxyHost, WebServiceProxyHost : hôte et port du proxy HTTP/HTTPS
# que le Gestionnaire de sources de CPC doit utiliser pour accéder aux sources de CPC externes.
# WebServiceProxyUsername, WebServiceProxyPassword : nom d'utilisateur et mot de passe pour
# le proxy qui applique les autorisations.
######################################################
#WebServiceProxyHost: <proxy host>
#WebServiceProxyPort: <proxy port>
#WebServiceProxyUsername: <username>
#WebServiceProxyPassword: <password>

Pour WebServiceProxyHost, vous pouvez entrer une adresse IP ou un nom de domaine complet.
