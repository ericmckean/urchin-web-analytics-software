Urchin peut-il lire un journal de serveur Web à partir d'une entrée standard (stdin) ?

Dans certains cas, il peut être souhaitable d'utiliser le moteur Urchin de traitement des journaux pour lire le journal d'un serveur Web directement dans une entrée standard (stdin), et non à partir d'un fichier. Urchin prend en charge la configuration d'une Source du journal permettant d'analyser une entrée standard plutôt qu'un véritable fichier.

Pour configurer Urchin de cette manière, définissez tout d'abord une nouvelle Source du journal nommée "stdin" dans le Gestionnaire de journaux en utilisant les paramètres suivants :

Paramètres du journal

  • Emplacement du fichier journal : local
  • Chemin d'accès au fichier journal : -
  • Format du journal : auto

Paramètres avancés

  • Avenir du journal : Ne pas toucher
  • Jeton : ?
  • Racine de l'URI en minuscule : désactiver

REMARQUE : L'option "Chemin d'accès au fichier journal" ne doit comporter qu'un seul tiret "-" (sans guillemets). Ce point est très important.

Vous pouvez à présent configurer les profils voulus de sorte qu'ils analysent les journaux à partir de l'entrée standard "stdin". Il suffit pour cela de sélectionner votre nouvelle entrée standard "stdin" comme source du journal dans l'écran de configuration Profil -> Source du journal.

Remarque importante : Vous ne pouvez pas exécuter de profils Urchin utilisant une source de journal "stdin" via le Planificateur Urchin, car celui-ci n'est pas en mesure de traiter correctement le flux de journalisation entrant. La seule méthode permettant de traiter des journaux pour un profil utilisant une source de journal "stdin" consiste à exécuter Urchin manuellement. Voici un exemple très simple de la manière dont vous pouvez activer Urchin :

  1. Ouvrez une fenêtre d'interpréteur de commande sur le système Urchin.
  2. Accédez au répertoire "bin" de la distribution Urchin.

  3. [Systèmes de type UNIX] Lancez urchin via la commande "./urchin -p myprofile < /path/to/webserver.log".

    • [Systèmes Windows] Lancez urchin via la commande "urchin -p myprofile < C:\path\to\webserver.log".

L'utilisation d'une source de journal "stdin" offre une grande part de flexibilité, car vous pouvez facilement réaliser sur vos journaux des opérations de prétraitement avant leur analyse par Urchin.

Autres exemples

Extraire des appels de fichier pour un hôte virtuel unique à partir d'un journal composé d'appels de fichier provenant de nombreux hôtes virtuels distincts et transmettre les résultats à Urchin : 

grep www.mondomaine.fr /path/to/alldomains.log | urchin -p mydomain.com

Extraire des appels de fichier de serveur Web bruts à partir d'une base de données relationnelle utilisant un type de script SQL intégré et les transmettre à Urchin pour analyse : 

sql-extract --date 20040302 --domain mydomain.com | urchin -p mydomain.com
