Pourquoi le rapport Fichiers les plus demandés fait-il apparaître un plus grand nombre d'appels de fichiers que le rapport sur les téléchargements ?

Il est courant que le rapport Fichiers les plus demandés fasse apparaître un plus grand nombre d'appels de fichiers que le rapport sur les téléchargements pour le même fichier. La raison en est simple : dans le cas d'un fichier PDF, de nombreux navigateurs incluent un plug-in qui affiche les fichiers PDF page par page. La requête de page initiale génère un code d'état 200 du serveur Web. Toutes les pages suivantes génèrent un code d'état 206 (téléchargement partiel), car le plug-in du navigateur télécharge uniquement la page que vous consultez et non la totalité du document. Ainsi, si vous consultez cinq pages PDF du même document, le serveur Web enregistre un appel de fichier avec le code d'état 200 et quatre appels de fichier avec un code d'état 206, tous étant consignés comme étant le même document PDF. Ces informations sont affichées comme suit dans les rapports Urchin :

Téléchargements : un appel de fichier de téléchargement possédant le code d'état 200, 302 ou 304, le rapport affiche un seul téléchargement pour ce fichier PDF, à savoir un appel de fichier unique avec le code d'état 200.

Le rapport Fichiers les plus demandés affiche tous les appels de fichier aboutis dont le code d'état est 2XX, 302 ou 304 et où le fichier correspond au type MIME approprié pour être considéré comme une consultation de page. Par défaut, Urchin traite tous les types MIME comme des consultations de page à l'exception des fichiers possédant les suffixes suivants : gif,jpg,jpeg,png,js,css,cur,ico,ida.

Ainsi, les cinq requêtes de PDF mentionnées dans l'exemple ci-dessus seront toutes comptabilisées comme des appels de fichier à ce document dans le rapport Fichiers les plus demandés, car chaque appel de fichier contenait un code d'état 200 ou 206.

Chaque appel de fichier correspondant à un téléchargement partiel (état 206) est comptabilisé comme une consultation de page distincte, car chaque appel de fichier

correspond plus ou moins à une page unique d'un document PDF de plusieurs pages.
