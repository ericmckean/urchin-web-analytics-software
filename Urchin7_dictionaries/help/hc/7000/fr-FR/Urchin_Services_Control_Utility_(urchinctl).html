Utilitaire de contrôle des services Urchin (urchinctl)

Présentation

L'utilitaire Urchin Services Control utility, urchinctl, vérifie l'état du planificateur Urchin Scheduler et des services Urchin Webserver. Il vous permet également de les démarrer et de les arrêter. Dans un système de type UNIX, urchinctl est généralement appelé à partir de l'un des scripts de démarrage du système pour démarrer ou arrêter automatiquement les services Urchin.

Voici les types d'opérations pouvant être réalisées par urchinctl :

  • Vérification de l'état des planificateurs Urchin 6 maître et esclave
  • Démarrage, arrêt ou redémarrage du planificateur ou du serveur Web (ou des deux)
  • Démarrage du serveur Web sur un autre port
  • Démarrage du serveur Web avec cryptage SSL

Utilisation

urchinctl se trouve dans le répertoire bin de la distribution Urchin.

Son utilisation est la suivante : 

  urchinctl -h (imprime le message d'utilisation et quitte le processus)

  urchinctl -v (imprime le numéro de version et quitte le processus)

  urchinctl [-e] [-p port] [[-s] | [-w]] action

où : 

  -e active le cryptage (SSL) dans le serveur Web

  -p indique le port que le serveur Web doit écouter

  -s effectue l'action UNIQUEMENT sur le planificateur Urchin

  -w effectue l'action UNIQUEMENT sur le serveur Web Urchin

et l'action est l'une des suivantes : 

  start (démarre le(s) service(s))

  stop (arrête le(s) service(s))

  restart (arrête, puis démarre le(s) service(s))

  status (affiche l'état d'exécution du serveur Web/planificateur)

Par défaut, l'action est effectuée à la fois sur le serveur Web et sur le planificateur, sauf si les options "-s" (planificateur) ou "-w" (serveur Web) sont spécifiées. Notez que ces options s'excluent mutuellement.

Considérations

Sur les systèmes de type UNIX, vous devez exécuter urchinctl avec le nom d'utilisateur/l'ID utilisateur ayant servi à l'installation d'Urchin afin de vous assurer que les processus urchinwebd et urchind sont lancés avec cet ID utilisateur.
