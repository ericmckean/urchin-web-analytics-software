Utilisation d'un serveur Web IIS existant (plates-formes Windows)

Par défaut, les tâches d'administration et de création de rapports Urchin sont effectuées via un serveur Apache autonome livré avec le produit Urchin. Dans la grande majorité des installations Urchin, il s'agit de la méthode de prédilection pour la génération d'interfaces d'administration et de création de rapports. Toutefois, dans de rares cas, il peut s'avérer nécessaire d'utiliser un serveur Web IIS existant. Cela peut être dû aux contraintes du site si celui-ci n'autorise pas l'utilisation d'un serveur Web tiers sur le serveur, ou à la nécessité de configurer la fonctionnalité de rapports Urchin en tant qu'hôte virtuel sur un serveur IIS existant.

LIMITATION DE RESPONSABILITÉ : Ces modifications apportées à l'installation Urchin ne sont pas prises en charge.

Procédure

Remarque : Lors de cette procédure, on considère qu'Urchin a été installé à l'emplacement par défaut, C:\Program Files\Urchin. Si vous avez installé Urchin dans un autre répertoire, vous devez remplacer l'emplacement de l'exemple ci-dessous par l'emplacement réel.

Étape 1: Création d'un utilisateur pour l'interface Web Urchin.

  • Cliquez sur "Outils d'administration" -> "Gestion de l'ordinateur".

  • Dans la partie gauche de l'écran "Gestion de l'ordinateur", cliquez sur "Utilisateurs et groupes locaux".
  • Cliquez sur le dossier "Utilisateurs" avec le bouton droit de la souris et sélectionnez "Nouvel utilisateur...".
  • Entrez "IUSR_URCHIN" dans le champ "Nom d'utilisateur :".
  • Décochez la case "L'utilisateur doit changer de mot de passe à la prochaine ouverture de session".
  • Cochez la case "L'utilisateur ne peut pas changer de mot de passe".
  • Cliquez sur "Créer", puis sur "Fermer".
  • Double-cliquez sur le dossier "Utilisateurs" sur la gauche.
  • Cliquez sur "IUSR_URCHIN" avec le bouton droit de la souris et sélectionnez "Propriétés...".

* Sous l'onglet "Membre de", supprimez toutes les entrées existantes, puis cliquez sur le bouton "Ajouter..." et sélectionnez "Invités" dans la fenêtre contextuelle. Cliquez de nouveau sur "Ajouter", puis sur "OK".

  • Cliquez sur "Appliquer" et sur "Fermer" pour enregistrer vos modifications.

Étape 2 : Installation d'Urchin (si cette étape n'a pas déjà été effectuée).

Étape 3 : Désactivation du serveur Web Apache d'Urchin.

  1. Cliquez sur "Outils d'administration" -> "Services".

  2. Sous "Services", recherchez l'enregistrement "Serveur Web Urchin".
  3. Cliquez sur "Serveur Web Urchin" avec le bouton droit de la souris et sélectionnez "Arrêt".
  4. Cliquez sur "Serveur Web Urchin" avec le bouton droit de la souris et sélectionnez "Propriétés". Modifiez ensuite le Type de démarrage pour le définir sur "Désactivé".
  5. Cliquez sur "OK".

Étape 4 : Ajout d'un nouveau site Web à IIS. Cliquez sur "Outils d'administration" -> "Gestionnaire des services Internet (IIS)".

  1. Cliquez sur "Sites Web" avec le bouton droit de la souris.
  2. Dans le champ "Description :", tapez "Urchin" et cliquez sur "Suivant".
  3. Sélectionnez l'adresse IP et le numéro de port (généralement 9999), puis cliquez sur "Suivant".
  4. Dans le champ "Chemin d'accès :", naviguez jusqu'à l'emplacement d'installation d'Urchin (généralement C:\Program Files\Urchin\htdocs) et cliquez sur "Suivant".
  5. Cochez la case "Exécuter :" et cliquez sur "Suivant", puis sur "Terminer".
  6. Cliquez sur le nouveau site Web Urchin avec le bouton droit de la souris et sélectionnez "Propriétés".
  7. Sous l'onglet "Site Web", décochez la case "Activer l'enregistrement dans le journal".
  8. Cliquez sur l'onglet "Répertoire de base" et cochez la case "Accès à la source du script".
  9. Cliquez sur l'onglet "Documents" et supprimez les deux entrées par défaut. Cliquez ensuite sur "Ajouter" et entrez "session.cgi" dans la fenêtre qui s'affiche, puis cliquez sur "OK".
  10. Cliquez sur l'onglet "Sécurité de répertoire", puis cliquez sur "Modifier" dans la zone "Authentification et contrôle d'accès". Assurez-vous que la case "Accès anonyme" est cochée, puis cliquez sur "Modifier..." pour modifier le paramètre "Compte utilisé pour les connexions anonymes". Dans la fenêtre contextuelle, sélectionnez "IUSR_URCHIN" comme nom d'utilisateur. Cliquez sur "OK", puis de nouveau sur "OK" pour revenir à la fenêtre "Propriétés".
  11. Cliquez sur "OK" pour enregistrer vos modifications, puis quittez la fenêtre "Propriétés".

Étape 5 : Définition des autorisations relatives aux répertoires. Cliquez avec le bouton droit sur "Démarrer" sélectionnez "Explorer".

  1. Naviguez jusqu'à l'emplacement d'installation d'Urchin (généralement C:\Program Files\Urchin).
  2. Cliquez sur le dossier "Urchin" avec le bouton droit de la souris et sélectionnez "Propriétés".
  3. Cliquez sur l'onglet "Sécurité".
  4. Cliquez sur le bouton "Avancé". Décochez la case "Permettre aux autorisations pouvant être héritées du parent d'être propagées à cet objet", puis cliquez sur "Supprimer" dans la fenêtre contextuelle.
  5. Cliquez sur "Ajouter" et sélectionnez l'utilisateur Administrateur, puis cliquez de nouveau sur "Ajouter".
  6. Cliquez sur "Ajouter" et sélectionnez "IUSR_URCHIN", puis cliquez sur "Ajouter".
  7. Dans le champ "Nom :", assurez-vous que seules les entrées Administrateur et IUSR_URCHIN sont présentes.

  8. Accordez des autorisations d'accès en écriture dans le dossier <accueil_urchin>\var.

  9. Revenez à "Préférences" => "Sécurité" et assurez-vous que les autorisations suivantes sont accordées à l'utilisateur IUSR_URCHIN :

    • Lecture et exécution
    • Affichage du contenu du dossier
    • Lecture
    • Remarque : Laissez les autorisations complètes par défaut pour SYSTEM.
  10. Cliquez sur "OK" pour enregistrer les autorisations.
  11. Dans la fenêtre de l'Explorateur Windows, cliquez dans le répertoire "Urchin".
  12. Cliquez sur le dossier de données avec le bouton droit de la souris et sélectionnez "Propriétés".
  13. Cliquez sur l'onglet "Sécurité".
  14. Cliquez sur "Avancé". Décochez la case "Permettre aux autorisations pouvant être héritées du parent d'être propagées à cet objet", puis cliquez sur "Supprimer" dans la fenêtre contextuelle.
  15. Cliquez sur "Ajouter" et sélectionnez l'utilisateur Administrateur, puis cliquez de nouveau sur "Ajouter".
  16. Cliquez sur "Ajouter" et sélectionnez "IUSR_URCHIN", puis cliquez sur "Ajouter".
  17. Dans le champ "Nom :", assurez-vous que seules les entrées Administrateur et IUSR_URCHIN sont présentes.

  18. Revenez à "Préférences" => "Sécurité" et assurez-vous que les autorisations suivantes sont accordées aux utilisateurs Administrateur et IUSR_URCHIN :

    • Contrôle total
    • Modification
    • Lecture et exécution
    • Affichage du contenu du dossier
    • Lecture
    • Écriture
  19. Cliquez sur "OK".

Étape 6 : IIS 6 uniquement. Accédez au gestionnaire d'extensions des services Web.

  1. Cliquez sur "Ajouter une nouvelle extension de service Web".
  2. Entrez "Urchin CGI" dans le champ "Nom de l'extension :".
  3. Cliquez sur "Ajouter".
  4. Naviguez jusqu'aux fichiers report.cgi et session.cgi et mettez-les en surbrillance (par défaut, seuls les fichiers *.dll sont affichés, vous devez donc modifier le filtre afin d'afficher "Tous les fichiers").
    • Emplacement par défaut : C:\Program Files\Urchin\htdocs
  5. Cochez la case "Définir le statut de l'extension à Autorisée".
  6. Cliquez sur "OK".
  7. Accédez à l'entrée IIS principale pour votre serveur : "Nom de l'hôte".
  8. Cliquez avec le bouton droit et sélectionnez "Propriétés".
  9. Cliquez sur "Types MIME".
  10. Cliquez sur "Nouveau".
  11. Dans le champ "Extension :", entrez ".cgi".
  12. Dans le champ "Type MIME", entrez "application/octet-stream".
  13. Cliquez sur "OK".
  14. Cliquez sur "OK".

Votre serveur Web IIS est désormais configuré pour appeler l'interface Web Urchin si vous vous y connectez par le biais de l'URL http://mon.serveur.com:port dans laquelle "port" correspond au port que vous avez spécifié pour Urchin lors de l'installation (le paramètre par défaut est 9999).

Utilisation d'un serveur Web IIS existant pour les services Web d'Urchin Data API (Urchin 6.6+)

Les services Web d'Urchin Data API (Urchin 6.6+) sont livrés avec un module IIS qui peut être déployé et configuré pour gérer les demandes effectuées auprès d'Urchin Data API. Pour configurer Urchin Data API sur le serveur Web IIS, reportez-vous à la section Puis-je intégrer Urchin Data API à IIS ?.
