Par défaut, les sessions de visite expirent après 30 minutes d'inactivité. Comment puis-je modifier ce paramètre ?

Dans Urchin 6, le délai d'expiration par défaut des sessions UTM (Urchin Traffic Monitor) est fixé à 30 minutes d'inactivité dans les versions FreeBSD/Linux et Windows du package d'installation. Vous pouvez modifier ce paramètre dans le fichier "urchin.js" situé dans le dossier C:\Program Files\Urchin6\util\utm\urchin.js sous Windows et dans le dossier /util/utm/urchin.js sous FreeBSD/Linux.

Le délai d'expiration associé aux utilisateurs est défini dans les premières lignes de ce fichier de la manière suivante :

var _utimeout="1800"; // Définir le délai d'expiration de sessions inactives (en secondes)

En augmentant la valeur du délai d'expiration de 1800 à 3600 secondes, par exemple, vous prolongez de 30 minutes la période de suivi des visiteurs pour atteindre une heure.
