Présentation générale du filtrage

Cet article a été rédigé par Panalysis, un consultant agréé du logiciel Urchin.

Le filtrage est un concept essentiel dans Urchin 6. Les filtres vous permettent de déterminer la manière dont les données s'affichent dans vos rapports Urchin. Il peut s'agir simplement d'inclure ou d'exclure des données, ou d'apporter des modifications complexes.

Les filtres sont généralement utilisés pour :

  • Exclure vos propres activités du site Web.
  • Créer des rapports traitant une zone précise ou un répertoire spécifique du site Web.
  • Ajouter le nom de domaine au rapport Pages les plus consultées.
  • Ajouter les paramètres de requête des URL au rapport Pages les plus consultées.

Les filtres consistent à appliquer l'une des 5 actions disponibles à un champ donné dans le fichier journal ou dans les rapports Urchin. Ces actions sont les suivantes :

  • Exclure supprime des données dans les rapports.

  • Inclure restreint les données affichées dans les rapports à des éléments spécifiques.

  • Rechercher et remplacer modifie les données en les remplaçant par d'autres.

  • Tableau de conversion ajoute aux rapports des données issues d'un tableau de données externe. Par exemple, conversion de ?id=1 en ?id=Ma page d'accueil

  • Avancé modifie les données par extraction et association de données.

Pour avoir accès aux filtres, vous devez être connecté en tant qu'administrateur. Une fois connecté, vous pouvez créer des filtres en cliquant sur l'onglet Configuration situé dans l'angle supérieur droit de l'écran et en ouvrant les Profils Urchin. À partir de cette page, vous pouvez ajouter un filtre à l'aide du Gestionnaire de filtres, ou, modifier ou créer un profil afin d'y ajouter un filtre en cliquant sur l'icône plus de l'onglet Filtres.

Ordre de filtrage

Urchin 6 vous permet d'attribuer des filtres à des profils ou à des sources de journal.

Urchin traite ces filtres immédiatement avant la création des rapports. L'ordre de traitement utilisé par Urchin est le suivant :

Lecture du fichier journal brut -> Création des champs du fichier journal "Raw" -> Remplacement des champs par les champs "Auto" définitifs -> Application des filtres du profil -> Application des filtres de la source du journal -> Affichage du rapport final.

Cela signifie que les filtres servent uniquement à modifier les données qui apparaissent dans le rapport. Ainsi, la plupart des filtres utilisent des champs identifiés par la mention "(Auto)" pour modifier les données affichées dans le rapport final. Certains cas requièrent évidemment l'utilisation des champs marqués comme "(Raw)", mais ils sont relativement rares.

Important : Lorsque différents types de filtre sont utilisés, ceux-ci sont appliqués dans l'ordre suivant :

  • Avancé, Rechercher et remplacer
  • Tableau de conversion
  • Exclure et Inclure

Si vous définissez plusieurs filtres d'un même type, ils sont traités dans l'ordre dans lequel ils ont été associés aux sources de journal via l'interface admin (ou directement dans la base de données). Chaque filtre reçoit les données disponibles uniquement après l'application du filtre qui le précède.

L'une des erreurs les plus fréquentes consiste à utiliser successivement deux filtres Inclure. Exemple :

Filtre 1 : inclure le répertoire /somedirectory/ Filtre 2 : inclure un autre répertoire /someotherdirectory/

Le rapport ainsi créé ne comprend aucune page, car une même page ne peut pas figurer dans deux répertoires distincts. Cela s'explique par le fait que le second répertoire reçoit uniquement les données relatives au répertoire /somediretory. En effet, si le deuxième filtre inclut seulement la règle /someotherdirectory, on obtient un résultat équivalent à zéro page, puisque les pages conservées après l'application du filtre 1 ne correspondent pas à cette règle.

Pour résoudre ce problème, il suffit d'utiliser une expression rationnelle réunissant ces deux répertoires dans un seul filtre (par exemple, ^/(somedirectory|somotherdirectory)/.*).

Conseil : Pour tester vos règles de filtrage dans votre rapport avant de les ajouter au profil Urchin concerné, utilisez le Champ de filtrage situé en haut du rapport. Par exemple, testez une URL dans le rapport Optimisation du contenu -> Performances du contenu -> Pages les plus consultées.
