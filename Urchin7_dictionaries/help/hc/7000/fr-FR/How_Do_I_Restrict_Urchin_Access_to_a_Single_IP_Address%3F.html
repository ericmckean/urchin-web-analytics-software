Comment puis-je restreindre l'accès à Urchin à une seule adresse IP ?

Présentation

Dans les systèmes comportant plusieurs interfaces réseau ou plusieurs adresses IP reliées à une même interface réseau, l'interface Web d'Urchin est associée par défaut à l'ensemble des adresses IP configurées. Toutefois, il peut être souhaitable de restreindre l'administration d'Urchin et la création des rapports à une adresse IP unique.

Procédure

Pour restreindre l'interface admin ou la création de rapports Urchin, il suffit de modifier le fichier de configuration utilisé par le serveur Web Apache d'Urchin. Procédez comme suit :

Pour les systèmes de type UNIX

  • Ouvrez un interpréteur de commande avec l'utilisateur dont Urchin se sert pour s'exécuter.
  • Utilisez la commande "cd" pour ouvrir le répertoire dans lequel Urchin est installé.
  • Arrêtez le serveur Web Urchin avec la commande : "bin/urchinctl -w stop".
  • Avec un éditeur de texte, ouvrez le fichier "urchinwebd.conf.template" situé dans le répertoire "var" de la distribution Urchin.

  • Insérez la ligne suivante au-dessus de la ligne <Directory /> 

       Listen XXX.XXX.XXX.XXX:port
  • où "XXX.XXX.XXX.XXX" et "port" correspondent à l'adresse IP et au numéro de port auxquels vous souhaitez restreindre l'utilisation d'Urchin.
  • Redémarrez le serveur Web Urchin avec la commande : 

bin/urchinctl -w start

Pour les systèmes Windows

  • Arrêtez les services Urchin : "Démarrer->Programmes->Urchin 6->Désactiver les services Urchin".

  • Lancez l'Explorateur Windows et ouvrez le dossier "C:\Program Files\Urchin6\var".
  • Ouvrez le fichier "urchinwebd.conf.template" dans un éditeur de texte.

  • Insérez la ligne suivante au-dessus de la ligne <Directory /> 

       Listen XXX.XXX.XXX.XXX:port
  • où "XXX.XXX.XXX.XXX" et "port" correspondent à l'adresse IP et au numéro de port auxquels vous souhaitez restreindre l'utilisation d'Urchin.

  • Redémarrez les services Urchin : "Démarrer->Programmes->Urchin 6->Activer les services Urchin".

Important : En raison de la fonction de mise en pool des sockets utilisée par IIS versions 5 et ultérieures, il peut s'avérer nécessaire de désactiver le fonctionnement par défaut d'IIS afin de partager les sockets de tous les sites Web reliés à un port spécifique, quelle que soit l'adresse IP réelle configurée. Cette règle s'applique lorsque les environnements utilisent des hôtes virtuels et que vous souhaitez définir la création de rapports Urchin sur le port 80 pour un hôte virtuel précis.

Considérations

  • Cette technique fonctionne uniquement pour restreindre l'administration/la création de rapports Urchin à une adresse IP donnée. Vous ne pouvez pas restreindre l'accès à des hôtes virtuels fonctionnant avec des noms.
  • Toute mise à niveau d'Urchin remplace le fichier "urchinwebd.conf.template" après l'enregistrement d'une copie de sauvegarde. Par conséquent, vous devrez de nouveau effectuer vos modifications après chaque mise à niveau.
