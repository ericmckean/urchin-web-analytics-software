Installation rapide de l'UTM (Apache)

Cet article a été rédigé par Panalysis, un consultant agréé du logiciel Urchin.

La procédure présentée ci-après permet une installation rapide de l'UTM pour des sites Web fonctionnant sur un serveur Apache, quelle que soit la plate-forme associée.

Étape 1 : copiez les fichiers UTM à la racine du site Web. Ces fichiers (urchin.js et __utm.gif) sont situés dans le répertoire "util/utm" de la distribution Urchin. Copiez ces deux fichiers dans le répertoire principal du contenu de votre site Web.

Étape 2 : référencez l'UTM dans votre code HTML. Saisissez la ligne ci-après dans toutes vos pages HTML. Cette ligne peut être placée n'importe où dans la page, mais nous vous recommandons de l'insérer dans la section <head>. Si vous utilisez une inclusion ou un modèle, vous pouvez l'insérer à cet endroit. IMPORTANT : Le nom du fichier GIF commence par deux traits de soulignement. 

<script src="/urchin.js" type="text/javascript"></script>
<script type="text/javascript">
        urchinTracker();
</script>

Si vous utilisez un logiciel du type "HTML Tidy", vous devez inclure la ligne JavaScript dans la section HEAD de votre page, par exemple : 

<html>
<head>
  <meta http-equiv="Content-Type" content="text/html; charset=ISO-8859-1">
  <script src="/urchin.js" type="text/javascript"></script>
  <script type="text/javascript">
        urchinTracker();
  </script>
  ...
</head>

Étape 3 : activez les cookies dans votre journalisation Apache. Si cette fonctionnalité n'est pas encore activée, vous pouvez utiliser l'exemple httpd.conf suivant pour activer la journalisation des cookies : 

LogFormat "%h %v %u %t \"%r\" %>s %b \"%{Referer}i\"\"%{User-Agent}i\"\"%{Cookie}i\"" special
...
<VirtualHost 63.212.171.6>
ServerName www.mysite.com
DocumentRoot /www/www.mysite.com
CustomLog /var/logs/mysite.log special
</VirtualHost>

Étape 4 : définissez le profil Urchin de manière à utiliser l'UTM. Dans l'interface d'administration Urchin, modifiez le profil en question et cliquez sur l'onglet Création de rapports. Configurez la Méthode de suivi des visiteurs sur UTM. Attribuez au Domaine UTM l'adresse de votre site Web sans préfixe www. Lorsque vous avez terminé, cliquez sur le bouton Mettre à jour. Ouvrez ensuite l'onglet Paramètres des profils et choisissez UTM - Tout activer pour l'option Groupe de rapports par défaut, puis cliquez de nouveau sur Mettre à jour. L'opération est terminée. Votre site Web enregistrera désormais les données UTM dans votre fichier journal habituel, qui sera identifié la prochaine fois que vous lancerez Urchin.

Cela fonctionne-t-il ? Pour vérifier que l'UTM génère bien des entrées dans votre fichier journal, examinez votre journal après l'installation de l'UTM et cliquez sur quelques pages du site. Une entrée semblable à la suivante devrait apparaître à la fin du fichier journal : 

... "GET /__utm.gif?..." 200 ..."__utma=..."

Si les entrées __utma n'apparaissent pas, vérifiez que les cookies sont correctement activés dans la journalisation. Si un autre code d'état que 200 s'affiche, vérifiez que les fichiers ont bien été copiés à la racine du site.
