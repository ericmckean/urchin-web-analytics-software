Utilisation des sources de journal UNC

Lors de la création d'une source de journal UNC dans Urchin, les paramètres suivants sont spécifiés : Server Name/IP, Remote Server Login, Local Server Login, Remote Log Source File.

Urchin utilise le workflow suivant pour récupérer le fichier journal à partir d'une source UNC : - Urchin tente de rechercher les fichiers spécifiés dans "Remote Log Source File". - Si cette opération échoue, Urchin essaye alors de créer une connexion au partage UNC, spécifié dans le champ "Server Name/IP". - Si la connexion échoue, Urchin suppose que la tentative de connexion a été réalisée à partir d'un compte utilisateur n'ayant pas accès au réseau, tel que le compte "Système local". Dans ce cas, il crée une session de connexion pour le compte utilisateur, spécifié dans "Local Server Login" et se fait passer pour ce compte (le processus Urchin est alors exécuté dans ce compte). Si l'option "Connexions locale et distante identiques" est définie sur "Oui", les informations d'identification sont extraites à partir de "Remote Server Login". - Une fois l'opération d'emprunt d'identité réussie, Urchin tente une nouvelle fois de créer une connexion UNC au partage réseau et de rechercher les fichiers. - Lorsqu'Urchin en a terminé avec la source du journal, il ferme la connexion UNC et met fin à l'emprunt d'identité (le processus Urchin est alors de nouveau exécuté dans le compte qu'il utilisait avant l'emprunt d'identité).

L'emprunt d'identité présente les inconvénients suivants : - Le compte dans lequel le processus Urchin est exécuté peut ne pas avoir accès au répertoire de données Urchin, ce qui rend impossible la création de bases de données de rapports. - Il est possible que le processus Urchin ne puisse pas accéder aux bases de données créées lors de l'emprunt une fois que celui-ci sera terminé. - Il entraîne une diminution des performances, en particulier lorsque plusieurs fichiers journaux UNC sont associés au profil, car Urchin doit lancer/terminer l'emprunt d'identité pour chaque fichier.

Les meilleures pratiques dans le cadre de l'utilisation de partages UNC dans Urchin sont donc les suivantes : - Exécuter Urchin dans le compte qui a accès au réseau, car dans ce cas l'emprunt d'identité ne se produit pas. - Accorder un accès en lecture seule aux partages de fichiers journaux UNC pour le compte utilisateur au sein duquel Urchin est exécuté. La création d'une connexion UNC et l'emprunt d'identité peuvent être inutiles dans ce cas.
