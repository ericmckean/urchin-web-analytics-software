Comment puis-je effectuer la mise à niveau vers Urchin 6 ?

  • ' Analytics Pros, consultant agréé Urchin, a apporté sa contribution à cet article

Il n'existe aucun moyen de mettre à niveau Urchin 5 vers la version 6, mais vous pouvez procéder à la migration de vos données Urchin 5 vers Urchin 6. Pour migrer votre configuration (comptes, utilisateurs, etc.) et vos données de rapport, suivez les instructions proposées dans cet article.

  • Remarques importantes :

    • N'installez pas Urchin 6 dans le même dossier qu'Urchin 5. Vous risqueriez en effet de perturber le fonctionnement d'Urchin 6 et Urchin 5. Notez également que le programme d'installation Urchin n'affichera pas d'avertissement si vous tentez d'installer Urchin 6 dans le même dossier qu'Urchin 5.

    • Il est déconseillé de migrer les données de mois pour lesquels des données de rapport se chevauchent pour les versions 5 et 6 d'Urchin. Si la base de données cible comporte déjà des données pour un mois pour lequel vous tentez d'importer des données, la procédure sera beaucoup plus lente que d'habitude.

Migration des données de configuration

1. Lancez l'utilitaire uconf-export à partir de votre installation Urchin 5. Cet utilitaire extrait le contenu de votre base de données de configuration (un fichier plat dans Urchin 5). Par défaut (en l'absence d'arguments), l'utilitaire envoie le texte vers un fichier de sortie standard (moniteur). Il suffit d'utiliser l'argument -f pour indiquer le nom du fichier dans lequel vous souhaitez enregistrer les données. Remarque : Le fichier de sortie doit être accessible à partir de votre installation Urchin 6. 

uconf-export -f PATH+FILE_NAME

2. À partir de votre installation Urchin 6, lancez la commande 

uconf2sql -f CHEMIN+NOM_FICHIER

où CHEMIN+NOM_FICHIER correspond au fichier dans lequel vous avez exporté vos données.

Cela mettra fin à la procédure de migration de vos données de configuration Urchin 5 vers Urchin 6.

Migration des données de rapport

1. À partir de votre installation Urchin 5, copiez le répertoire "Urchin5 BASE" dans un emplacement accessible depuis votre installation Urchin 6. Par exemple, si vous exécutez Urchin 5 et 6 sous Windows, vous pouvez partager votre répertoire Urchin 5 afin de le monter en tant que lecteur dans votre système de fichiers Urchin 6.

2. Lancez l'utilitaire convert-u5data à partir de votre installation Urchin 6, comme suit : 

convert-u5data  [-a account] [-p PROFILE_NAME] -c PATH_2_U5_InstallDir

Cette opération permet d'extraire l'ensemble des données de rapport des fichiers Urchin 5 et de les migrer vers Urchin 6.

Notes d'utilisation pour convert-u5data

  • -a est facultatif. En l'absence de cet argument, tous les comptes sont explorés pour en extraire les éventuels profils à migrer.
  • -p est facultatif. En l'absence de cet argument, tous les profils des comptes explorés sont migrés.
  • -d est facultatif. Cette commande est utilisée pour exécuter l'utilitaire en mode de débogage (Urchin 6.6+).
  • -z est facultatif. Cette commande est utilisée pour convertir les bases de données archivées (Urchin 6.6+).

Remarque : Si vous avez personnalisé l'emplacement du répertoire de données Urchin en modifiant /Urchin5_Install_Base/etc/urchin.conf pour utiliser un emplacement différent de "./data", vous devez vous assurer que l'emplacement et l'utilisateur exécutant convert-u5data peuvent accéder au même chemin que celui défini dans urchin.conf. Par exemple, si vous avez mis à jour urchin.conf pour utiliser "D:\urchin5_data\", le système exécutant convert-u5data recherchera les données Urchin 5 dans "D:\urchin5_data\". S'il ne s'agit pas d'un chemin valide sur le système où la commande est exécutée, le message d'erreur suivant s'affichera : Erreur lors de la modification des répertoires. Annulation de la procédure. Le cas échéant, il est recommandé de copier le dossier de données Urchin 5 dans le dossier de données situé dans Urchin5_Install_Base, et de mettre à jour le fichier urchin.conf pour utiliser "./data" en tant que paramètre d'emplacement du dossier de données.
