Rapports d'ensemble (6.602)

Les rapports d'ensemble vous permettent d'examiner les performances de plusieurs profils sur la page d'accueil d'Urchin. La page d'accueil dispose de trois affichages : "Standard", "Administrateur" et "Entreprises". Vous pouvez les sélectionner dans la liste déroulante "Afficher".

Si vous cliquez sur le bouton "Modifier", vous accédez à l'onglet "Création de rapports" dans "Paramètres de profil". La colonne "État" indique l'état actuel du profil. Celui-ci est indépendant de la période sélectionnée. L'icône "Filtré" () apparaît en regard des profils filtrés. Les profils non liés à des sources de CPC s'affichent avec l'icône "Comptes d'annonces" (). Cliquez sur l'icône "Comptes d'annonces" pour être redirigé vers AdWords. Si des sources de CPC sont associées au profil, la colonne "Comptes d'annonces" indique leur nombre. Pour consulter les rapports, cliquez sur le nom d'un profil. Pour accéder au site, cliquez sur le lien situé en regard du nom du profil.

Le sélecteur de périodes vous permet de sélectionner des données par jour, semaine, mois et année (les plus récents).

Les données de la période sélectionnée sont affichées dans les colonnes "Appels de fichier", "Durée moyenne de la visite", "Octets", "Visites", "Taux de rebond", "Objectifs réalisés" et "Pages vues". Les autres colonnes ("Nom du profil", "État", "Sources de journal", "Filtré" et "Comptes d'annonces") affichent les informations du profil.

L'utilisateur accède aux données selon les autorisations dont il dispose. Les tableaux suivants comparent les différents paramètres d'autorisation.'

À partir d'Urchin 6.602, l'affichage par défaut de la page d'accueil (Rapports RollUp) de chaque utilisateur peut être défini manuellement dans la base de données de configuration Urchin via les champs uglobals.uigl_rollup_view_type et uusers.uius_rollup_view_type :

  • Valeurs du champ uglobals.uigl_rollup_view_type :
    • 0: auto
      • Il s'agit de la valeur par défaut dans la base de données. Dans ce cas, la valeur globale du type d'affichage est obtenue à partir du type d'utilisateur :

        • Super administrateur, Administrateur de compte -> Affichage "Administrateur"

          Utilisateur, Utilisateur – Pas de mise à jour -> Affichage "Entreprises"

    • 1: Affichage "Standard"
    • 2: Affichage "Administrateur"
    • 3: Affichage "Entreprises"
  • uusers.view_type:
    • -1: Utiliser la valeur globale
      • Il s'agit de la valeur par défaut à chaque création d'utilisateur. Cela revient à utiliser le type d'affichage Rapport d'ensemble (uglobals.uigl_rollup_view_type) pour cet utilisateur.
    • 0: auto
      • Dans ce cas, le type d'affichage de l'utilisateur est obtenu à partir du type d'utilisateur :

        • Super administrateur, Administrateur de compte -> Affichage "Administrateur"

          Utilisateur, Utilisateur – Pas de mise à jour -> Affichage "Entreprises"

    • 1: Affichage "Standard"
    • 2: Affichage "Administrateur"
    • 3: Affichage "Entreprises"

REMARQUES IMPORTANTES RELATIVES À L'UTILISATION

  • Si vous tentez de vous connecter à un compte qui comprend une quantité importante de données, l'erreur suivante peut s'afficher : 

    Session Error - unable to call default routine. (2002) Please login again.
  • Ce problème se produit lorsque l'extraction des données pour des rapports d'ensemble dépasse le délai d'expiration par défaut fixé à 45 secondes (APPLICATION_TIMEOUT). Vous pouvez utiliser les solutions suivantes :
  • Vous pouvez définir une valeur supérieure pour le délai d'expiration APPLICATION_TIMEOUT dans le fichier XYZ pour résoudre ce problème. Veuillez noter les effets de cette modification.
  • Vous pouvez définir le type d'affichage sur "Affichage "Standard"" en mettant à jour les champs uglobals.uigl_rollup_view_type et uusers.uius_rollup_view_type dans la base de données de configuration Urchin avec une valeur de '1'.
