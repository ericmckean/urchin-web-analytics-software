Comment effectuer le suivi des sous-domaines de mon site dans un même profil ?

  • Cet article a été rédigé par ivantage Limited, un consultant agréé du logiciel Urchin.

Vous pouvez effectuer le suivi des sous-domaines dans le même profil que le domaine. Pour cela, vous devez ajouter une ligne au code de suivi sur chaque page du sous-domaine. Les instructions ci-dessous supposent que ce profil a été configuré sur le domaine principal de la même manière que le site Web répertorié (par exemple, example.co.uk).

  1. Ajoutez la ligne suivante (en gras) au code de suivi sur toutes les pages de votre sous-domaine et de votre domaine principal : 

         <script src="/urchin.js" type="text/javascript">
         </script>
         <script type="text/javascript">
           _udn="example.co.uk";
          </script>
  • Cette ligne doit faire référence au domaine principal (example.co.uk).

Notez que même si cette modification vous permet d'assurer le suivi de vos sous-domaines dans un seul profil, vos rapports risquent de ne pas distinguer les pages provenant de www.example.co.uk de celles provenant de help.example.co.uk. Par exemple, votre rapport Pages les plus consultées peut contenir des correspondances pour www.example.co.uk/index.html et help.example.co.uk/more.html, mais ce rapport affichera les informations suivantes : 

   /index.html
   /more.html

Si vous souhaitez différencier vos sous-domaines, vous pouvez appliquer un filtre "Avancé" à votre profil avec les paramètres suivants : 

   Filter Type: Custom filter > Advanced
   Field A: cs_host (Raw)
   Extract A: (.*)
   Field B: Request_Stem (Auto)
   Extract B: (.*)
   Output To: Request_Stem (Auto)
   Constructor: /$A1$B1

Une fois ce filtre appliqué, les exemples précédents apparaîtront avec le sous-domaine associé : 

   www.example.co.uk/index.html
   help.example.co.uk/more.html

REMARQUE : Ce filtre s'appuie sur le nom d'hôte de votre site Web avec sous-domaine inscrit dans votre journal d'accès Web. Si votre journal d'accès Web ne comprend pas de nom d'hôte, les rapports Urchin 6 ne seront pas en mesure de distinguer les fichiers index.html situés, par exemple, sur les deux domaines : 

 www.example.co.uk/index.html
 help.example.co.uk/more.html
