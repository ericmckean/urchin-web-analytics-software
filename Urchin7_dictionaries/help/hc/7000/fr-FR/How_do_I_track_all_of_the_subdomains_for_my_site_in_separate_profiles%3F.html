Comment puis-je effectuer le suivi des sous-domaines de mon site dans des profils séparés ?

  • Cet article a été rédigé par ivantage Limited, un consultant agréé du logiciel Urchin.

La procédure à suivre dans Urchin pour effectuer le suivi des sous-domaines dans des profils séparés dépend de la configuration de votre serveur Web. Vous pouvez configurer votre serveur Web de sorte qu'il génère un fichier journal par sous-domaine. Urchin peut alors traiter séparément le fichier journal de chaque sous-domaine. Si vous ne produisez qu'un seul fichier journal et que le journal d'accès Web enregistre le nom d'hôte avec sous-domaine, vous pouvez également créer un filtre Inclure pour chaque profil de sous-domaine. Par exemple, le filtre d'un profil inclura seulement le trafic de help.example.co.uk, tandis que l'autre inclura uniquement le trafic associé à www.example.co.uk.

Ainsi, pour effectuer le suivi de vos sous-domaines dans des profils séparés, vous devez appliquer un filtre Inclure à chaque profil de sorte qu'il traite un seul nom d'hôte. La première étape consiste à créer un profil pour chaque sous-domaine. Vous pouvez ensuite créer les filtres décrits ci-dessous.

Par exemple, si vous souhaitez effectuer le suivi de sub.example.fr et de www.example.fr dans deux profils séparés, appliquez les filtres suivants :

  • Pour le profil sub.example.fr 

    Type de filtre : Inclure UNIQUEMENT la règle
    Champ de filtrage : utm_request_hostname (AUTO)
    Règle de filtrage : ^sub\.example\.fr
  • Pour le profil www.example.fr 

    Type de filtre : Inclure UNIQUEMENT la règle
    Champ de filtrage : utm_request_hostname (AUTO)
    Règle de filtrage : ^www\.example\.fr
