Gestion des sources du journal

Présentation

En général, vous devez indiquer la Source du journal lors de la création d'un Profil. La Source du journal permet à Urchin d'identifier les caractéristiques d'un journal d'accès (parfois appelé journal de transfert) associé à un site Web. Les journaux d'accès recensent tous les appels de fichier, ou demandes de document Internet, transmis à votre site Web. Les caractéristiques d'une source de journal incluent notamment le chemin d'accès au fichier journal, le format du journal (W3C ou NCSA par exemple), le fait que ce journal soit stocké sur un système local ou distant et les éventuels filtres à appliquer au cours du traitement.

Vous devez être conscient que les sources du journal sont indépendantes des profils. Chaque profil doit être associé à au moins une source de journal pour générer des rapports. Cependant, il est tout à fait concevable d'utiliser une source de journal dans plusieurs profils. Par exemple, vous pouvez attribuer une source à plusieurs profils et appliquer un filtre distinct à chaque profil afin de créer des rapports de nature différente. Ainsi, les associations ne se limitent pas à un profil pour une source.

Configuration des sources de journal

Pour ajouter une source de journal à votre système, connectez-vous tout d'abord à l'interface d'administration Urchin en tant qu'administrateur et cliquez sur le bouton Configuration à gauche. Cliquez ensuite sur le bouton Gestionnaire de journaux. Pour créer une Source du journal, cliquez sur le bouton Ajouter en haut à droite de l'écran. L'assistant d'ajout de source de journal s'affiche. Il s'agit d'une procédure en quelques étapes qui vous permettra de configurer votre source de journal en toute simplicité. À chaque étape, l'assistant fournit des instructions claires expliquant les données de configuration affichées à l'écran.

Dans l'écran Paramètres du journal, vous devez sélectionner le Format du journal. Ce paramètre indique à Urchin la manière dont votre fichier journal est organisé. Veillez à sélectionner le format de journal approprié. Dans le cas contraire, Urchin produira des rapports incohérents. Urchin prend en charge plusieurs formats de journaux que vous pouvez sélectionner via un menu déroulant. En voici la liste :

  • Auto : avec ce format, Urchin détecte automatiquement les formats de journaux NCSA, W3C, Netscape et ELF2. Si le format de vos journaux fait partie de cette liste, vous pouvez opter pour Auto afin qu'Urchin détermine la méthode de lecture appropriée, plutôt que de choisir un format précis.
  • NCSA : format étendu/combiné Apache modifié (voir la rubrique Journalisation - Apache et IIS pour obtenir une description de ce format).
  • W3C : les serveurs Microsoft IIS utilisent généralement ce format, bien que certains serveurs Web soient configurés pour produire des journaux de type W3C.
  • Netscape : format utilisé par défaut par les serveurs Netscape et iPlanet.
  • ELF2 : format de journaux E-Commerce.
  • Google : utilisez ce format pour les journaux contenant des données de coût par clic Google. Notez que le format de journaux Google ne peut pas être détecté automatiquement.
  • Overture : utilisez ce format pour les journaux contenant des données de coût par clic Overture. Notez que le format de journaux Overture ne peut pas être détecté automatiquement.
  • Personnalisé : ce format n'est pas répertorié dans le menu déroulant, mais vous pouvez créer votre format de journal personnalisé. Une fois configuré, celui-ci apparaît automatiquement dans le menu déroulant.

Si le format des journaux produits par votre serveur Web ne correspond à aucun des formats pris en charge par défaut, deux possibilités s'offrent à vous : modifier la configuration de votre serveur Web de manière à produire un format de journaux pris en charge ou créer un format de journal personnalisé adapté au mode de journalisation de votre serveur. Si vous choisissez de modifier la journalisation de votre serveur Web, optez pour le format W3C ou NCSA.

Équilibrage des charges et traitement des journaux en parallèle

Les sites Web dont la fréquentation est importante nécessitent parfois l'utilisation de plusieurs serveurs Web qui font appel à l'équilibrage des charges. Cette technique répartit le trafic sur plusieurs serveurs Web qui travaillent en parallèle. L'ensemble de ces serveurs forme une grappe de serveurs (un cluster). De cette manière, chaque activité enregistrée sur le site est consignée dans l'un des journaux parallèles. L'assistant de configuration du profil propose une option de traitement des journaux en parallèle. Lorsque l'option Traitement du journal parallèle est activée, Urchin ouvre tous les fichiers journaux simultanément et les lit selon un système de rotation, une section après l'autre, chaque section correspondant à 15 minutes d'activité des journaux. L'option Traitement du journal parallèle améliore considérablement les performances des sites à équilibrage de charge.
