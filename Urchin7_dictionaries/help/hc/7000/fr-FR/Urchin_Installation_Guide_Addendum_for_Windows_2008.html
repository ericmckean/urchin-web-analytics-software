Les informations présentées ici concernent l'installation d'Urchin sous Windows Server 2008 et Windows Vista. Même si la procédure indiquée dans Urchin - Guide d'installation (Windows) fonctionne sous Windows Server 2008 et Windows Vista, les renseignements suivants doivent être pris en compte.

Windows 2008 est doté d'une nouvelle fonctionnalité de sécurité appelée "Contrôle de compte d'utilisateur". Pour plus d'informations sur cette fonctionnalité, consultez la page http://technet.microsoft.com/en-us/library/cc709691.aspx.

L'activation de cette fonctionnalité entraîne deux problèmes, dont voici la description :

  1. Si le contrôle de compte d'utilisateur est activé, l'application ne peut pas écrire correctement dans le dossier "Program files". Pour plus d'informations, consultez la page suivante : http://msdn.microsoft.com/en-us/library/aa905330.aspx.

  2. Les exécutables d'Urchin ne peuvent pas être lancés à partir d'une invite de commande par défaut. Vous devez utiliser l'invite de commande en tant qu'administrateur ou définir les propriétés de l'exécutable comme suit : cliquez avec le bouton droit de la souris, puis sélectionnez Propriétés -> Compatibilité-> "Exécuter ce programme en tant qu'administrateur" -> OK. Notez que les services Urchin Windows fonctionnent comme prévu, sans action supplémentaire de votre part.

Solution :

  1. Désactivez le contrôle de compte d'utilisateur et configurez Urchin où vous le souhaitez. Exécutez Urchin à l'aide des fichiers binaires ou des services Windows.
  2. Si le contrôle de compte d'utilisateur est activé, N'INSTALLEZ PAS Urchin dans le dossier "Program files". Installez-le, par exemple, dans votre dossier d'utilisateur. Modifiez les propriétés afin d'exécuter les fichiers binaires en mode administrateur (comme indiqué au point 2 ci-dessus) ou utilisez les services Windows sans apporter aucune modification.
