Comment puis-je supprimer nos propres visites dans les rapports relatifs à notre site Web ?

Présentation

Les profils sont fréquemment associés à des filtres d'adresse IP, notamment pour inclure uniquement les visites provenant d'adresses IP spécifiques ou exclure les adresses IP des systèmes de contrôle internes. Cela est très facile à réaliser avec Urchin. Veillez toutefois à définir correctement le 'Champ de filtrage' lorsque vous configurez un filtre d'adresse IP.

Procédure

Pour configurer le filtrage du trafic interne de manière appropriée, la méthode la plus simple est la suivante :

  • Accédez à l'écran Configuration -> Profils Urchin -> Gestionnaire de filtres dans l'interface d'administration d'Urchin.

  • Cliquez sur le bouton "Ajouter" afin de créer un filtre.

  • Choisissez "Filtrer le trafic interne" dans le menu déroulant "=> Sélectionnez un filtre exemple".

  • Remplacez l'exemple d'adresse IP inscrit dans le champ "Règle de filtrage" par l'adresse IP à exclure. Veillez à insérer une barre oblique inverse ("\") avant chaque point dans l'adresse IP !
  • Cliquez sur "Terminer", puis sur "Autres options" dans la fenêtre suivante.
  • Dans la zone "Profils disponibles", sélectionnez les profils auxquels vous souhaitez appliquer ce filtre et cliquez sur la flèche vers la droite pour les copier dans la zone "Appliqué à".
  • Cliquez sur "Mettre à jour" pour appliquer le filtre aux profils sélectionnés.

Autres considérations

Gardez à l'esprit que les règles de filtrage Urchin sont des expressions rationnelles POSIX. C'est pourquoi vous devez insérer un caractère d'échappement avant les points (.) dans l'adresse IP spécifiée, de sorte qu'Urchin ne les interprète pas comme des métacaractères appartenant à l'expression rationnelle. Par exemple, pour filtrer l'adresse IP '172.16.1.1', vous devez saisir la règle de filtrage suivante : '172\.16\.1\.1'. Ou encore, pour filtrer la plage d'adresses IP '192.168.xxx.xxx, vous devez saisir la règle de filtrage suivante : '192\.168\.'. Voici quelques exemples supplémentaires :

Si vous entrez plusieurs adresses IP, séparez-les à l'aide d'une barre verticale 192\.168\.12\.32$|10\.10\.1\.23$|63\.212\.171\.5$. Notez l'utilisation du signe $ précisant qu'aucun numéro ne doit suivre le dernier numéro spécifié. Ainsi, pour inclure à la fois 63.212.171.5 et 63.212.171.53, vous pouvez recourir au filtre suivant : 63\.212\.171\.5. En revanche, si votre filtre ne doit concerner que 63\.212\.171\.5, rédigez votre règle de filtrage comme suit : 63\.212\.171\.5$.

Vous pouvez définir une plage d'adresses IP en une seul chaîne de caractères. Les adresses IP allant de 192.168.10.10 à 192.168.10.50 peuvent être désignées par l'expression 192.168.10.[1-5][0-9]$.

Les correspondances en début de ligne fonctionnent également. Ainsi, la règle de filtrage 63\.212\.171\.5 inclut aussi l'adresse IP 163\.212\.171\.5. Pour inclure uniquement une adresse IP commençant par un certain numéro, utilisez une expression excluant tout autre début de chaîne à l'aide du caractère '' comme suit : 63\.212\.171\.5$.

Liste exhaustive des expressions rationnelles 

 *'''Caractère générique'''     '''Signification'''
 *'''.'''       un seul caractère quel qu'il soit.
 *'''*'''       zéro ou plusieurs occurrences de l'élément précédent.
 *'''+'''       correspond à une ou plusieurs occurrences des éléments précédents.
 *'''?'''       correspond à zéro ou à une occurrence de l'élément précédent.
 *'''()'''      retient le contenu de la parenthèse comme élément.
 *'''[]'''      correspond à un élément de la liste.
 *'''-'''       crée une plage dans une liste.
 *'''|'''       ou.
 *'''^'''       correspond au début du champ.
 *'''$'''       correspond à la fin du champ.
 *'''\'''       insère un caractère d'échappement avant l'un des caractères ci-dessus.
