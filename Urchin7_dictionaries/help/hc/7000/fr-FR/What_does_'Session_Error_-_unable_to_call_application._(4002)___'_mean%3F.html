Que signifie le message "Erreur de session - Appel de l'application impossible. (4002)..." ?

Lorsque vous affichez des rapports Urchin, le navigateur demande l'extraction des données correspondant à la période et au rapport que vous avez demandés dans les fichiers de la base de données Urchin. Si aucune réponse à cette requête de données n'est reçue dans un délai de 45 secondes, une expiration du délai se produit et un message d'erreur s'affiche : "Erreur de session - Appel de l'application impossible. (4002) Veuillez vous reconnecter." Dans ce cas, vous devez augmenter le délai d'expiration en conséquence. Pour y parvenir, vous devez disposer d'un accès en ligne de commande aux fichiers de la distribution Urchin. Si ce n'est pas votre cas, veuillez contacter la personne chargée de fournir vos rapports afin de lui indiquer le présent article.

Pour augmenter le délai d'expiration de l'application :

  • Ouvrez un interpréteur de commande vers le serveur Urchin.
  • Ouvrez le fichier /path/to/urchin/etc/session.conf.

  • Recherchez la directive intitulée APPLICATION_TIMEOUT:45 dont la valeur est exprimée en secondes.

  • Augmentez cette valeur à votre convenance, puis enregistrez le fichier. Vos modifications prennent effet immédiatement.
