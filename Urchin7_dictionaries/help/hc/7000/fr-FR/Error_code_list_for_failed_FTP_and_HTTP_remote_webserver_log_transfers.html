Liste des codes d'erreur associés aux échecs de transfert des journaux de serveur Web distant via FTP et HTTP

Présentation

Dans Urchin, vous pouvez configurer une Source du journal de manière à collecter un journal situé sur un serveur Web distant à l'aide du protocole FTP ou HTTP. Dans des circonstances normales, les transferts réussissent et aucune erreur n'apparaît dans le journal d'exécution. Toutefois, si une erreur se produit durant le transfert (nom d'utilisateur ou mot de passe incorrect, serveur ou journal distant inaccessible, etc.), Urchin enregistre un code d'erreur dans le journal d'exécution présenté dans l'Historique des tâches associé au profil en question. Ce code d'erreur apparaît entre parenthèses avec la mention "échec" après la tentative de transfert du journal par le serveur Web, (-9) par exemple.

Vous trouverez ci-dessous la liste des codes d'erreur accompagnés d'un message décrivant le problème rencontré.

Liste des codes d'erreur

CURLE_UNSUPPORTED_PROTOCOL (1) - L'URL transmise à libcurl utilisait un protocole non pris en charge par cette bibliothèque libcurl. Cela peut provenir d'une option compile-time que vous n'avez pas utilisée, d'un protocole mal orthographié ou simplement d'un protocole dont libcurl ne possède pas le code.

CURLE_FAILED_INIT (2) - Le code d'initialisation a échoué très tôt. Il s'agit certainement d'une erreur ou d'un problème interne.

CURLE_URL_MALFORMAT (3) - Le format de l'URL était incorrect.

CURLE_COULDNT_RESOLVE_PROXY (5) - Impossible de résoudre le proxy. L'hôte proxy indiqué n'a pas pu être résolu.

CURLE_COULDNT_RESOLVE_HOST (6) - Impossible de résoudre l'hôte. L'hôte distant indiqué n'a pas pu être résolu.

CURLE_COULDNT_CONNECT (7) - Connexion à l'hôte ou au serveur proxy impossible.

CURLE_FTP_WEIRD_SERVER_REPLY (8) - Une fois connectée à un serveur FTP, libcurl attend une certaine réponse. Ce code d'erreur indique la réception d'une réponse anormale ou incorrecte. Il est probable que le serveur distant indiqué ne soit pas un serveur FTP opérationnel.

CURLE_REMOTE_ACCESS_DENIED (9) - L'accès à la ressource indiquée dans l'URL a été refusé. Dans le cadre du protocole FTP, cela se produit lors d'une tentative d'accès au répertoire distant.

CURLE_FTP_WEIRD_PASS_REPLY (11) - Après l'envoi du mot de passe FTP au serveur, libcurl attend une réponse appropriée. Cette erreur indique la réception d'un code inattendu.

CURLE_FTP_WEIRD_PASV_REPLY (13) - libcurl n'a pas obtenu un résultat adéquat du serveur en réponse à une commande PASV ou EPSV. Le serveur est défaillant.

CURLE_FTP_WEIRD_227_FORMAT (14) - Les serveurs FTP renvoient une ligne 227 en réponse aux commandes PASV. Ce code est renvoyé lorsque libcurl ne parvient pas à analyser cette ligne.

CURLE_FTP_CANT_GET_HOST (15) - Défaillance interne lors de la recherche de l'hôte utilisé pour la nouvelle connexion.

CURLE_FTP_COULDNT_SET_TYPE (17) - Réception d'une erreur lors de la tentative de définition du mode de transfert sur binaire ou ascii.

CURLE_PARTIAL_FILE (18) - Un transfert de fichier était plus léger ou plus lourd que prévu. Cela se produit lorsque le serveur indique la taille prévue pour un transfert et fournit des données d'une taille différente.

CURLE_FTP_COULDNT_RETR_FILE (19) - Ce code indique une réponse anormale à une commande "RETR" ou la réalisation d'un transfert ne comprenant aucun octet.

CURLE_QUOTE_ERROR (21) - Lors de l'envoi de commandes "QUOTE" personnalisées au serveur distant, l'une des commandes a renvoyé un code d'erreur de type 400 ou supérieur (pour FTP), ou bien l'exécution de la commande a échoué.

CURLE_HTTP_RETURNED_ERROR (22) - Ce code est renvoyé lorsque la valeur de CURLOPT_FAILONERROR est TRUE et que le serveur HTTP renvoie un code d'erreur de type 400 ou supérieur (auparavant, l'intitulé de ce code d'erreur était CURLE_HTTP_NOT_FOUND).

CURLE_WRITE_ERROR (23) - Une erreur s'est produite à l'écriture des données reçues dans un fichier local ou libcurl a reçu une erreur en réponse à la une fonction WriteCallback.

CURLE_UPLOAD_FAILED (25) - Échec au démarrage du transfert. Dans le cadre du protocole FTP, le serveur a refusé la commande STOR. Le tampon de l'erreur comprend généralement l'explication du serveur (auparavant, l'intitulé de ce code d'erreur était CURLE_FTP_COULDNT_STOR_FILE).

CURLE_READ_ERROR (26) - Un problème est survenu lors de la lecture d'un fichier local ou la fonction ReadCallback a renvoyé une erreur.

CURLE_OUT_OF_MEMORY (27) - Une demande d'allocation de mémoire a échoué. Il s'agit d'un problème très sérieux que vous n'êtes pas près de résoudre.

CURLE_OPERATION_TIMEDOUT (28) - Opération arrivée à expiration. Le délai d'expiration spécifié a été atteint conformément aux conditions.

CURLE_FTP_PORT_FAILED (30) - La commande FTP PORT a renvoyé une erreur. Le plus souvent, cela se produit lorsque l'adresse que vous avez fournie à libcurl n'est pas suffisamment précise. Voir également CURLOPT_FTPPORT.

CURLE_FTP_COULDNT_USE_REST (31) - La commande FTP REST a renvoyé une erreur. Cette erreur ne devrait jamais se produire avec un serveur en bon état de marche.

CURLE_RANGE_ERROR (33) - Le serveur ne prend pas en charge ou n'accepte pas les demandes de plage.

CURLE_HTTP_POST_ERROR (34) - Il s'agit d'une erreur exceptionnelle généralement due à un dysfonctionnement interne.

CURLE_SSL_CONNECT_ERROR (35) - Un problème s'est produit lors de la poignée de main entre SSL et TLS. Pensez à lire le message du tampon d'erreur, vous y trouverez quelques précisions sur le problème rencontré. Cela peut venir des certificats (formats de fichier, chemins, autorisations), des mots de passe, etc.

CURLE_FTP_BAD_DOWNLOAD_RESUME (36) - Tentative de reprise FTP au-delà de la taille du fichier.

CURLE_FILE_COULDNT_READ_FILE (37) - Ouverture impossible d'un fichier indiqué sous la forme FILE://. Cela vient certainement du chemin d'accès qui ne correspond à aucun fichier existant. Avez-vous vérifié les autorisations d'accès au fichier ?

CURLE_LDAP_CANNOT_BIND (38) - LDAP ne parvient pas à établir la liaison. L'établissement de la liaison LDAP a échoué.

CURLE_LDAP_SEARCH_FAILED (39) - La recherche LDAP a échoué.

CURLE_FUNCTION_NOT_FOUND (41) - Fonction introuvable. Une fonction zlib obligatoire n'a pas été trouvée.

CURLE_ABORTED_BY_CALLBACK (42) - Annulé par un rappel. Une fonction de rappel a renvoyé "abort" à libcurl.

CURLE_BAD_FUNCTION_ARGUMENT (43) - Erreur interne. Une fonction a été appelée avec un paramètre erroné.

CURLE_INTERFACE_FAILED (45) - Erreur d'interface. Une interface de sortie spécifiée n'a pas pu être utilisée. Définissez l'interface à utiliser pour l'adresse IP source des connexions sortantes à l'aide de CURLOPT_INTERFACE (auparavant, l'intitulé de ce code d'erreur était CURLE_HTTP_PORT_FAILED).

CURLE_TOO_MANY_REDIRECTS (47) - Trop de redirections. libcurl a suivi un nombre de redirections maximal. Définissez votre limite avec CURLOPT_MAXREDIRS.

CURLE_UNKNOWN_TELNET_OPTION (48) - Une option définie à l'aide de CURLOPT_TELNETOPTIONS n'a pas été reconnue/est inconnue. Reportez-vous à la documentation correspondante.

CURLE_TELNET_OPTION_SYNTAX (49) - La chaîne d'une option telnet a été formatée de manière illégale.

CURLE_PEER_FAILED_VERIFICATION (51) - Le certificat SSL du serveur distant ou l'empreinte md5 SSH a été considéré(e) comme incorrecte.

CURLE_GOT_NOTHING (52) - Le serveur n'a renvoyé aucune réponse et cela constitue une erreur au vu des circonstances.

CURLE_SSL_ENGINE_NOTFOUND (53) - Le moteur de cryptographie spécifié est introuvable.

CURLE_SSL_ENGINE_SETFAILED (54) - Impossible de définir le moteur de cryptographie sélectionné comme moteur par défaut !

CURLE_SEND_ERROR (55) - L'envoi des données réseau a échoué.

CURLE_RECV_ERROR (56) - La réception des données réseau a échoué.

CURLE_SSL_CERTPROBLEM (58) - Problème lié au certificat du client local.

CURLE_SSL_CIPHER (59) - Utilisation du chiffre spécifié impossible.

CURLE_SSL_CACERT (60) - Impossible d'authentifier le certificat pair à l'aide des certificats connus de l'autorité de certification.

CURLE_BAD_CONTENT_ENCODING (61) - Codage de transfert non reconnu.

CURLE_LDAP_INVALID_URL (62) - URL LDAP incorrecte.

CURLE_FILESIZE_EXCEEDED (63) - Taille de fichier maximale dépassée.

CURLE_USE_SSL_FAILED (64) - Échec du niveau SSL FTP demandé.

CURLE_SEND_FAIL_REWIND (65) - Lors d'une opération d'envoi, curl a dû récupérer des données envoyées afin de les retransmettre, mais cette récupération a échoué.

CURLE_SSL_ENGINE_INITFAILED (66) - L'initialisation du moteur SSL a échoué.

CURLE_LOGIN_DENIED (67) - Le serveur distant a refusé la connexion de curl (ajouté le 13/07/2001)

CURLE_TFTP_NOTFOUND (68) - Fichier introuvable sur le serveur TFTP

CURLE_TFTP_PERM (69) - Problème d'autorisation sur le serveur TFTP

CURLE_REMOTE_DISK_FULL (70) - Espace disque insuffisant sur le serveur

CURLE_TFTP_ILLEGAL (71) - Opération TFTP non autorisée

CURLE_TFTP_UNKNOWNID (72) - ID de transfert TFTP inconnu

CURLE_REMOTE_FILE_EXISTS (73) - Ce fichier existe déjà et ne sera pas remplacé.

CURLE_TFTP_NOSUCHUSER (74) - Cette erreur ne devrait jamais être renvoyée par un serveur TFTP fonctionnant correctement.

CURLE_CONV_FAILED (75) - Échec de la conversion de caractère

CURLE_CONV_REQD (76) - L'appelant doit enregistrer les rappels de conversion.

CURLE_SSL_CACERT_BADFILE (77) - Problème de lecture lié au certificat SSL de l'autorité de certification (chemin ou droits d'accès)

CURLE_REMOTE_FILE_NOT_FOUND (78) - La ressource référencée dans l'URL n'existe pas.

CURLE_SSH (79) - Une erreur non spécifiée s'est produite durant la session SSH.

CURLE_SSL_SHUTDOWN_FAILED (80) - Échec de la déconnexion à SSL
