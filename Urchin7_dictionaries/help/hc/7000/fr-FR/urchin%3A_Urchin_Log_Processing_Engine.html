urchin : traitement des journaux Urchin

Présentation

Le moteur Urchin Log Processing Engine, urchin, est le composant central du système de traitement des journaux d'Urchin. D'ordinaire, le moteur de traitement des journaux est appelé à partir du planificateur Urchin Scheduler (urchind) lorsqu'une tâche est exécutée. Toutefois, il est possible d'exécuter urchin directement à partir d'un interpréteur de commande afin d'exécuter un profil spécifique. Cela est particulièrement utile sous les environnements comportant de nombreux scripts, lorsque vous exécutez des tâches à partir d'une source externe telle que le Planificateur de tâches Windows (ou cron sur les systèmes de type UNIX). Cela est également utile pour exécuter un profil dans des circonstances exceptionnelles, par exemple, si vous souhaitez traiter uniquement les visites effectuées à une date spécifique, ou pour effectuer une opération de débogage.

Urchin n'est pas véritablement un utilitaire. Il est présenté comme tel ici, car il est doté de capacités limitées d'utilitaire de ligne de commande qui peuvent se révéler utiles sous certains environnements.

Utilisation

urchin se trouve dans le répertoire bin de la distribution Urchin. Son utilisation est la suivante : 

urchin [options...] (-p <profile name> | -P <profile id>)

Options : 

-a <account name>       Indique le nom de compte du profil. Le nom de compte par défaut est "(NONE)".

-D                      Active le mode de débogage.

-d <YYYYMMDD>           Indique la date de retraitement.

-H                      Indique que les résultats de l'exécution doivent être consignés dans le fichier d'historique. Un nouvel enregistrement de l'historique des tâches est créé si -H est spécifié et que -T ne l'est pas.

-h                      Imprime les informations d'aide et quitte le processus.

-P <profile id>         Indique l'ID du profil à traiter.

-p <profile name>       Indique le nom du profil à traiter.

-T <task id>            Indique l'enregistrement de l'historique des tâches à mettre à jour. Le fait de spécifier -T implique nécessairement l'utilisation de -H.

-t                      Imprime les données de configuration et quitte le processus.

-v                      Imprime le numéro de version et quitte le processus.

Considérations

  • Sur les systèmes de type UNIX, vous devez exécuter urchin avec le nom d'utilisateur/l'ID utilisateur ayant servi à l'installation d'Urchin afin de vous assurer que les bases de données correspondant au profil sont la propriété de cet ID utilisateur, puisque le service urchin les créera si elles n'existent pas déjà.
