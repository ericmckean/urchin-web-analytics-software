Pourquoi mes tâches planifiées ne sont-elles pas effectuées alors que le Planificateur Urchin est en cours d'exécution ?

Description du problème

Dans certaines situations, le Planificateur Urchin (urchind) n'effectue pas les tâches planifiées alors même qu'il est en cours d'exécution. Cela peut se produire de deux manières :

  1. Les tâches en question apparaissent comme planifiées sur l'écran Configuration -> Planificateur -> Planificateur de tâches de l'outil d'administration Urchin et sont replanifiées comme prévu, mais elles ne sont jamais réellement exécutées.

  2. Lorsque vous cliquez sur le bouton "Exécuter" dans l'écran Exécuter/Planifier du profil en question, la fenêtre de progression s'affiche normalement, mais la barre de progression indique simplement "Erreur (0 %)" et la tâche ne parvient pas à se terminer.

Origine

En règle générale, ce dysfonctionnement vient du fait que le Planificateur Urchin a été lancé à partir d'un répertoire supprimé ou dont les autorisations ont été modifiées de telle manière qu'Urchin n'y a plus accès.

Par exemple, il est fréquent de décompresser le logiciel d'installation d'Urchin dans un répertoire temporaire pour réaliser une installation ou une mise à niveau. Au cours de la procédure d'installation ou de mise à niveau, le Planificateur Urchin est lancé à partir du répertoire de travail courant, c'est-à-dire le répertoire temporaire. Si ce répertoire est ensuite supprimé, le Planificateur Urchin n'est plus en mesure de fonctionner correctement.

Solution

Pour garantir que les applications Urchin puissent démarrer et s'exécuter correctement, veillez à lancer le Planificateur Urchin à partir d'un emplacement permanent pour lequel Urchin dispose de droits d'accès suffisants. Si vous rencontrez des problèmes de Planificateur, il est préférable de redémarrer les services Urchin à l'aide de la procédure suivante :

Systèmes Windows

  1. Démarrer -> Programmes -> Urchin -> Désactiver les services Urchin.

  2. Démarrer -> Programmes -> Urchin -> Activer les services Urchin.

Systèmes de type UNIX

  1. Lancez un interpréteur de commandes en tant qu'utilisateur "root" ou avec l'utilisateur avec lequel Urchin s'exécute.
  2. Ouvrez le répertoire dans lequel Urchin est installé avec la commande : "cd /usr/local/urchin".
  3. Redémarrez les services Urchin via la commande "bin/urchinctl restart".
