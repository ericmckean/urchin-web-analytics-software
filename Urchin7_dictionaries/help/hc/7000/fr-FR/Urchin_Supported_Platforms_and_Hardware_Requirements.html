Plates-formes Urchin prises en charge et configuration matérielle requise

Urchin fonctionne sur de nombreux systèmes d'exploitation et architectures. Urchin ne doit être installé que sur un système destiné à traiter des journaux. Un navigateur Web suffit pour consulter les rapports.

Plates-formes prises en charge

Windows

  • Windows 2003 Server
  • Windows XP
  • Windows 2000
  • Windows Vista
  • Windows 2008 Server

UNIX-type Systems

  • Linux (kernel 2.6) - générique pour la plupart des systèmes d'exploitation utilisés
  • Linux (kernel 2.4) - générique pour la plupart des systèmes d'exploitation utilisés
  • FreeBSD 4 (non pris en charge pour 6.5+)
  • FreeBSD 5.3+ (non pris en charge pour Urchin 6.602+)
  • FreeBSD 6.2+

Supported databases

  • MySQL 4.1+ (non pris en charge pour Urchin 6.6+)
  • MySQL 5.03+
  • PostgreSQL 8.2.5

Navigateurs pris en charge

  • Internet Explorer 6.x, 7.x
  • Firefox 2.x, 3.x
  • Chrome 1.x, 2.x

Remarque sur les plates-formes non prises en charge

S'il n'existe pas de version d'Urchin compatible avec votre plate-forme spécifique, vous pouvez peut-être utiliser une version d'Urchin compatible avec votre système d'exploitation.

  • Windows 98, Windows 3.x:

Urchin ne peut pas être installé sous Windows 98 or 3.x. Cependant, ces plates-formes permettent de consulter les rapports à l'aide d'Internet Explorer (5.x ou version ultérieure).

  • Linux:

Linux propose de nombreux systèmes d'exploitation. Tous ne sont pas explicitement pris en charge par une version spécifique d'Urchin. Cependant, il existe généralement une forte compatibilité entre ces différentes versions de Linux. C'est pourquoi il est fort probable qu'une des versions d'Urchin fonctionne sur votre machine. Voici quelques-unes des associations les plus courantes :

  • Ubuntu (kernel 2.6)

  • RedHat Enterprise Linux 2 & higher

  • SuSE Linux 8 & higher

  • Trustix 2 & higher

  • Solaris:

Solaris n'est actuellement pas pris en charge.

Configuration système requise pour Urchin 6

Les performances d'Urchin sont telles qu'elles vous permettent d'obtenir des résultats satisfaisants avec un investissement matériel minimum. Par exemple, un vieux Pentium II serait bien trop lent pour des applications bureautiques, mais il ferait un excellent serveur Urchin. Pour des performances optimales, nous recommandons le matériel suivant :

Analyse d'un site Web de taille petite à moyenne

  • Processeur 2 Ghz ou supérieur
  • 2 Go de RAM
  • Disque dur IDE 250 Go minimum
  • Interface Ethernet 100 Mbs

Fournisseur de services / installations pour entreprise

  • Pentium dual-core 2,5 Ghz/UltraSPARC 2 Ghz/même vitesse de CPU PPC/MIPS/etc.
  • 3GB RAM
  • Disque dur Ultra2/Wide SCSI (tel qu'un Seagate Cheetah)
  • Ethernet 100base-T
  • Système de sauvegarde

Utilisation mémoire/système/disque

  • La part de mémoire RAM allouée à Urchin peut être configurée entre 20 et 500 Mo (entre 20 et 1024 Mo pour Urchin 6.5).
  • Urchin peut fonctionner avec une priorité basse, normale ou élevée.
  • Les données stockées par Urchin utilisent environ 10 % de l'espace des journaux bruts.

Remarque importante à l'attention des utilisateurs de FreeBSD

(1) FreeBSD doit respecter une limite figée dans le code concernant la taille des données de processus. En raison de la taille des données géographiques d'Urchin devant être conservées dans la mémoire lors du traitement du journal, cette limite, fixée à 500 Mo, n'est pas suffisante. Vous verrez, par conséquent, apparaître l'erreur d'exécution suivante : 

ERROR: (8011-323-1057) Failed to allocate memory

Pour résoudre ce problème, vous devez augmenter la taille des données de processus par défaut du système FreeBSD. Pour cela, modifiez le fichier /boot/loader.conf, puis ajoutez les lignes suivantes : 

# Increase max process data segment size to 1GB
kern.maxdsiz="1073741824"

Un redémarrage est nécessaire à l'issue de cette opération.

AVERTISSEMENT!!! Soyez très vigilant lorsque vous modifiez ces limites système. Si vous définissez une taille trop importante pour kern.maxdsiz, votre système risque de ne pas pouvoir redémarrer. Nous vous recommandons vivement de consulter la documentation FreeBSD avant d'apporter des modifications et d'évaluer les risques potentiels s'appliquant à la configuration spécifique de votre site.

(2) FreeBSD utilise un pilote spécial nommé accf_http. Il se charge de mettre en mémoire tampon les connexions entrantes jusqu'à ce qu'un certain nombre de demandes HTTP arrivent pour accélérer et optimiser les performances. Par défaut, ce pilote n'est pas chargé dans le noyau et l'avertissement suivant s'affichera : 

[warn] (2)No such file or directory: Failed to enable the
‘httpready’ Accept Filter

Pour éviter d'obtenir un tel avertissement, vous devez activer le module du noyau FreeBSD du filtre d'acceptation HTTP (accf_http). Pour cela, modifiez le fichier /boot/loader.conf, puis ajoutez les lignes suivantes : 

# Wait for full HTTP request accept filter
accf_http_load="YES"

Un redémarrage est nécessaire à l'issue de cette opération.
