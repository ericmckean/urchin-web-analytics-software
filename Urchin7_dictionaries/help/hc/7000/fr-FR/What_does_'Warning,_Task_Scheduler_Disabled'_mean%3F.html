Que signifie le message "Avertissement ! Planificateur de tâches désactivé" ?

Le planificateur de tâches Urchin est un service chargé de lancer l'exécutable de traitement Urchin. Si le Planificateur de tâches est désactivé, Urchin ne peut pas traiter les tâches. Les raisons courantes pour lesquelles le message d'erreur : "Avertissement ! Planificateur de tâches désactivé" est susceptible de s'afficher sont les suivantes :

  • Le service urchind n'est pas en cours d'exécution. Cela peut se produire lorsque le serveur est redémarré et qu'Urchin n'est pas inclus dans les scripts de démarrage du système.
  • (Unix uniquement) Le service urchind est en cours d'exécution, mais il appartient à un utilisateur autre que le propriétaire de la distribution Urchin.

Pour vérifier que le planificateur est bien exécuté, appliquez la procédure suivante :

  • Ouvrez un interpréteur de commandes et un cd dans le répertoire /path/to/urchin/bin/.
  • Tapez : urchinctl status (sous Unix, tapez ./urchinctl status).

Si le planificateur n'est pas en exécution, vous pouvez le démarrer à partir de la ligne de commande en appliquant la procédure suivante :

  • Ouvrez une interface de commande (shell) et modifiez le répertoire (commande cd) pour le définir sur /path/to/urchin/bin/.
  • Tapez : urchinctl -s start (sous Unix, tapez ./urchinctl -s start).
  • Si aucun des deux services n'est exécuté, tapez : urchinctl start.

Si le service est exécuté, vérifiez les traitements en cours d'exécution sur le serveur pour identifier le propriétaire du traitement Urchin de la façon suivante : ps -ef | grep urchind

Notez le "Nom d'utilisateur" du propriétaire du traitement dans la sortie générée : 

Nom d'utilisateur 33597 0.0 0.2 352 204 ?? Ss 27Feb08 17:42.18 /usr/local/urchin6/bin/urchind

Si cet utilisateur n'est pas le propriétaire de la distribution Urchin, vous devez arrêter le service et le redémarrer en ouvrant une session sous le nom d'utilisateur approprié. Pour cela, vous pouvez appliquer une opération "changer d'utilisateur" à l'utilisateur actuellement propriétaire du traitement urchind et arrêter l'exécution du service urchinctl. Utilisez la bascule "-s" comme cela est décrit dans la procédure ci-dessus. Revenez ensuite à l'utilisateur Urchin et démarrez le service urchinctl, également via la bascule "-s".
