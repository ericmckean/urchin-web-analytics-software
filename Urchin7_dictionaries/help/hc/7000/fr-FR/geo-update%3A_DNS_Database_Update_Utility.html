geo-update : utilitaire de mise à jour de la base de données DNS

Présentation

L'utilitaire de mise à jour des données géographiques vérifie l'existence de mises à jour pour la base de données DNS interne d'Urchin et les télécharge, le cas échéant. Il vous permet de télécharger les mises à jour depuis les serveurs de téléchargement Google, ainsi que depuis un répertoire local (nouveauté Urchin 6).

Utilisation

L'utilitaire de mise à jour des données géographiques peut être lancé automatiquement tous les mois grâce au planificateur Urchin (urchind) ou manuellement via une ligne de commande. Lorsqu'il est lancé via le planificateur, il télécharge systématiquement les mises à jour depuis le serveur de téléchargement Google. Lorsqu'il est lancé via une ligne de commande, il est également possible de mettre à jour la base de données DNS à partir d'un chemin local. Cette tâche est définie pour vérifier l'existence de nouvelles mises à jour une fois par mois et peut être configurée par l'utilisateur à l'aide de l'interface d'administration pour qu'elle s'exécute à un moment précis.

Si une tâche a été créée pour la mise à jour des données géographiques, elle apparaît dans l'écran de l'historique des tâches sous le nom "mise à jour des données géographiques".

Le comportement par défaut est le suivant : recherche des mises à jour, téléchargement des nouvelles bases de données et remplacement des anciennes. Les comportements par défaut peuvent être remplacés en exécutant la mise à jour des données géographiques depuis la ligne de commande en utilisant les options appropriées. Son utilisation est la suivante : 

geo-update [-Hhv] [-D | -F] [-i file]

où 

  -F            Force le téléchargement des bases de données géographiques
  -f            Nom du chemin du fichier geo.zip sur le système de fichiers local [facultatif]
  -H            Indique que les résultats de l'exécution doivent être consignés dans le fichier d'historique
  -h            Imprime les informations d'aide et quitte le processus
  -T <task id>  Indique l'enregistrement de l'historique des tâches à mettre à jour
                Le fait de spécifier -T implique nécessairement l'utilisation de -H
  -V            Imprime le numéro de version de la base de données géographiques et quitte le processus
  -v            Imprime le numéro de version et quitte le processus

L'utilisation de l'option -F force le téléchargement de nouvelles bases de données même si les bases de données existantes sont à jour. Cette fonction s'avère utile si vous avez importé des informations de domaine personnalisé incorrectes ou que vous avez endommagé de quelque manière que ce soit votre base de données de domaine et que vous voulez recommencer en utilisant une version neuve.

La consignation des résultats dans un fichier d'historique avec l'option -H permet d'accéder aux informations d'exécution via l'écran Historique des tâches dans la zone Configuration->Planificateur de l'interface d'administration Urchin.

L'utilisation de l'option -T vous permet d'appliquer la mise à jour d'une tâche particulière à partir de l'interpréteur de ligne de commande indiqué par <Task_ID>. La valeur ID_Tâche se trouve dans la base de données Urchin 6 et peut être obtenue à l'aide de la requête suivante : 

mysql> select * from ugeo_tasks\G

Exemples

Pour forcer le téléchargement de la base de données des domaines la plus récente : 

geo-update -F

Pour mettre à jour manuellement la tâche à partir de l'interpréteur de ligne de commande (lorsque l'ID de la tâche a été récupéré depuis la base de données) : 

geo-update -T <Task_ID> -H

L'utilitaire de mise à jour des données géographiques requiert une connexion Internet pour pouvoir rechercher et télécharger les nouvelles mises à jour. L'utilitaire utilise le port 80 pour communiquer avec le serveur Web qui fournit les mises à jour. Il est possible que vous rencontriez des problèmes si vous passez à travers des pare-feu et des serveurs proxy lors des mises à jour. Si c'est le cas, consultez votre administrateur réseau.
