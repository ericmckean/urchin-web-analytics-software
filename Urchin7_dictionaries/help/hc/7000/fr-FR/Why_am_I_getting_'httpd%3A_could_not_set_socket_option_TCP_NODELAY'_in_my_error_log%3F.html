Pourquoi mon journal d'erreurs indique-t-il le message : "httpd: could not set socket option TCP_NODELAY" ?

Présentation

Ce message indique presque toujours que le client s'est déconnecté avant que le serveur Web Apache d'Urchin (urchinwebd) puisse définir certaines options de connexion réseau (socket) entre le navigateur et le serveur Apache.

Solution

Ce message vous est envoyé à titre indicatif uniquement. Toutefois, il serait judicieux d'en informer votre administrateur réseau, afin de déterminer l'origine de cette désactivation prématurée des connexions réseau. Ce problème peut être provoqué par :

  • un robot/explorateur Web ou un agent de surveillance mal configuré qui ne referme pas correctement la connexion réseau au serveur Apache d'Urchin ;
  • un pare-feu ou un réseau mal configuré qui supprime de façon inappropriée certains paquets transmis ;
  • l'analyse d'un port réseau pour rechercher des vulnérabilités ;
  • une attaque par saturation sur le serveur Web Apache d'Urchin.
