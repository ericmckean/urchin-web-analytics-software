Corrélation des visiteurs

Présentation

La corrélation des visiteurs est le processus qui consiste à identifier le comportement des visiteurs même si les sessions proviennent de différents fichiers journaux ou de serveurs Web ou E-commerce indépendants. Urchin utilise les données des diverses sources de journaux pour analyser les relations entre les sessions et les transactions, puis corrèlent ces informations pour fournir une image précise de la relation entre l'activité des visiteurs et les achats, fournissant ainsi un rapport utile sur le retour sur investissement. Par exemple, les visites générées par des moteurs de recherche et des mots clés spécifiques peuvent être corrélées avec le montant des achats provenant de votre site de E-commerce pour vous indiquer les visites générées et les termes de recherche qui génèrent le plus de revenus. En général, vous aurez un journal pour votre site Web en contact avec le public et un autre journal pour votre site Web de transactions sécurisées. Afin de corréler ensemble les sources de données disparates, Urchin doit utiliser la même méthode d'identification des visiteurs pour chaque site.

Types de corrélation des visiteurs

Urchin peut corréler les visiteurs selon diverses méthodes. Votre choix de la méthode de suivi des visiteurs affectera directement la précision des informations qu'Urchin a à sa disposition pour corréler les transactions de E-commerce avec d'autres activités de site Web. Choisissez la méthode qui est adaptée au niveau de détail que vous souhaitez.

Les données suivantes sont requises dans le fichier journal E-commerce pour chacune des méthodes de suivi des visiteurs :

  • UTM: cookies
  • IP+User-Agent: hôte distant ou adresse IP et agent utilisateur (c'est-à-dire type de navigateur).

Ces éléments doivent être pris en compte lorsque vous examinez votre format de journalisation du E-commerce. Pour plus d'informations sur la procédure à suivre pour vous assurer que vos journaux contiennent les données appropriées, consultez les sections Format de journal ELF2 et Journaux E-commerce personnalisés.

Configuration

Cette section présente un aperçu général des enjeux liés à la configuration de la corrélation des visiteurs de e-commerce.

Sites non UTM

Si le code de détection UTM n'est pas installé sur vos sites, votre configuration de la corrélation des visiteurs dépendra du format de e-commerce que vous utilisez.

  • ELF2: Utilisez IP+User-Agent comme méthode de suivi des visiteurs. Dans ce cas, Urchin corrélera automatiquement les diverses sessions uniquement si tous les journaux contiennent les champs IP. et User-Agent. Choisissez simplement IP+User-Agent comme Méthode de suivi des visiteurs dans l'écran Création de rapports du profil.

Sites UTM

Pour les sites UTM, la même version du code de détection UTM doit être installée sur toutes les pages dont vous souhaitez effectuer le suivi. Dans l'écran Paramètres de profil, définissez la Méthode de suivi des visiteurs sur Urchin Traffic Monitor. En fonctionnement normal, laissez le champ Domaine UTM vide. UTM détecte automatiquement le domaine du site. La fonction Domaine UTM est utilisée uniquement lorsque vous disposez de domaines qui se chevauchent et qui doivent faire l'objet d'un suivi séparé.
