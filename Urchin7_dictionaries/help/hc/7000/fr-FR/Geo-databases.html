Urchin 6.602 Geo-databases

Urchin 6.602 contient deux bases de données géographiques :

  • geo.db  : contient des informations détaillées sur les zones de diffusion (country/region/city), les réseaux (organisations) et les domaines. Le niveau d'information est le même que dans les versions précédentes d'urchin 6 (6.400 - 6.601), mais les informations sont à jour.

  • geo.light.db  : contient uniquement des informations sur les zones de diffusion jusqu'au niveau national. Elle ne contient pas d'informations sur les villes, les réseaux ou les organisations. L'empreinte mémoire et le temps de chargement de cette base de données sont nettement moins importants. Aussi, utilisez-la lorsqu'il est crucial que le moteur de traitement fournisse des performances optimales. L'utilisation de cette base de données géographiques allégée a un effet sur les rapports suivants :

    • o Les rapports "géographiques" suivants affichent les données concernant les zones de diffusion jusqu'au niveau national uniquement (pas d'informations sur les régions ni les villes) :

      • Affichage "Cadres" -> "Vue d'ensemble pour les cadres" (tableau de bord)

      • Affichage "Cadres" -> "Synthèse géographique"

      • Affichage "Responsable marketing" -> "Vue d'ensemble pour les responsables marketing" (tableau de bord)

      • Affichage "Webmaster" -> "Vue d'ensemble pour les webmasters" (tableau de bord)

      • "Optimisation du marketing" -> "Performances par segment" -> "Synthèse géographique"

      o Le rapport "Fournisseurs d'accès" n'affiche pas de données ("Optimisation du marketing" -> "Performances par segment" -> "Fournisseurs d'accès").

      o Le rapport "Domaines" n'affiche pas de données ("Optimisation du marketing" -> "Performances par segment" -> "Domaines").

La base geo.db est utilisée par défaut. Pour basculer vers la base de données geo.light.db, supprimez le caractère "#" (indiquant un commentaire) dans le fichier urchin.conf : 


# Paramètres de base de données géographiques

# Supprimez la marque de commentaire dans la ligne ci-dessous pour activer la base de données géographiques allégée.
# GeodbLight: ON
