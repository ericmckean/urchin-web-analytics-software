Comment puis-je activer le cryptage SSL sur le serveur Web Urchin ?

Le serveur Web Urchin inclus dans le logiciel Urchin permet le cryptage des communications via le protocole SSL. Pour activer SSL, vous devez disposer d'un certificat approprié, signé par une autorité de certification, ou d'un certificat auto-signé.

Pour activer le protocole SSL sur le serveur Web Urchin :

  1. Copiez votre fichier de certificat SSL dans le répertoire var d'Urchin et nommez-le server.crt.
  2. Copiez votre fichier de clé SSL dans le répertoire var d'Urchin et nommez-le server.key.
  3. Modifiez le fichier urchinwebd.conf.template qui se trouve dans le répertoire var d'Urchin.
    1. (Si vous utilisez Urchin 6.6+, passez à l'étape suivante.) Annulez le caractère "#" (indiquant un commentaire) de la ligne Port et supprimez ce caractère de la directive Listen. 
                #XXXPortXXX
          Listen 443

    2. Sur la ligne ServerName, remplacez localhost par le nom de votre serveur Web. Exemple : 

                ServerName:  www.urchin.com

    3. REMARQUE : Le paramètre ServerName du fichier urchinwebd.conf.template doit correspondre au nom de serveur indiqué dans le fichier de certificat.

  4. Lancez ou redémarrez le serveur Web à l'aide de la commande urchinctl associée à l'option -e. Urchinctl se trouve dans le répertoire bin d'Urchin. L'option -e indique à urchinctl d'activer SSL sur le serveur Web. Par exemple, utilisez la commande suivante pour redémarrer le serveur Web avec le cryptage SSL activé sur le port 443 : Pour les versions antérieures à Urchin 6.6 : 
       urchinctl -e -w restart
    Pour Urchin 6.6+: 
       urchinctl -e -p 443 -w restart

    Remarque : Pour lancer le serveur sans activer SSL, supprimez l'option -e.

Vous pouvez à présent accéder à votre serveur utilisant SSL à l'adresse https://servername.domain.com:port/.

Remarque : La personnalisation des paramètres SSL dans le fichier urchinwebd.conf.template peut entraîner des dysfonctionnements susceptibles d'empêcher le démarrage du serveur Web.

Instructions Windows

Pour activer le protocole SSL sur le serveur Web Urchin :

  1. Copiez votre fichier de certificat SSL dans le répertoire var d'Urchin et nommez-le server.crt.
  2. Copiez votre fichier de clé SSL dans le répertoire var d'Urchin et nommez-le server.key.
  3. Modifiez le fichier urchinwebd.conf.template qui se trouve dans le répertoire var d'Urchin.
    1. (Si vous utilisez Urchin 6.6+, passez à l'étape suivante.) Annulez le caractère "#" (indiquant un commentaire) de la ligne Port et supprimez ce caractère de la directive Listen. 
                #XXXPortXXX
          Listen 443
    2. Supprimez le caractère "#" (indiquant un commentaire) de la ligne suivante pour charger mod_ssl.so lors du démarrage du serveur Web : 
                LoadModule ssl_module etc/mod_ssl.so

    3. Sur la ligne ServerName, remplacez localhost par le nom de votre serveur Web. Exemple : 

                ServerName:  www.urchin.com

    4. REMARQUE : Le paramètre ServerName du fichier urchinwebd.conf.template doit correspondre au nom de serveur indiqué dans le fichier de certificat.

  4. Supprimez et installez le service du serveur Web à l'aide de la commande urchinctl associée à l'option -e. Urchinctl se trouve dans le répertoire bin d'Urchin. L'option -e indique à urchinctl d'activer SSL sur le serveur Web. Par exemple, pour supprimer et installer le serveur Web avec le cryptage SSL activé sur le port 443 : Pour les versions antérieures à Urchin 6.6 : 
       urchinctl.exe -w remove
   urchinctl.exe -w -e install
    Pour Urchin 6.6+: 
       urchinctl.exe -w remove
   urchinctl.exe -e -p 443 -w install

    Remarque : Pour lancer le serveur sans activer SSL, supprimez l'option -e.

Vous pouvez à présent accéder à votre serveur utilisant SSL à l'adresse https://servername.domain.com:port/.

Remarque : La personnalisation des paramètres SSL dans le fichier urchinwebd.conf.template peut entraîner des dysfonctionnements susceptibles d'empêcher le démarrage du serveur Web.
