모니터링 소프트웨어에서 조회수 전체 필터링

개요

대부분의 호스팅 환경에서는 서비스 수준 계약(SLA)을 유지하기 위한 일종의 고객 웹서버 모니터링 기능을 제공합니다.그러나 이에 따른 부작용으로 세션, 페이지뷰, 조회, 바이트 카운트 등을 인위적으로 증가시켜 이 모니터링을 통한 조회수가 모니터링된 웹사이트의 실제 Urchin 보고를 부풀릴 수 있습니다.

권장사항

이러한 모니터링 방식을 사용하는 호스팅 환경에서는 각 고객의 구성된 Urchin 프로필에 표준/전체 Urchin 필터를 적용하여 모니터링 소프트웨어에서 생성한 조회수를 검증하는 것이 좋습니다.이 작업은 중앙 집중식 Urchin 설치(호스팅 업체에서 관리)를 통해 각 고객 웹사이트에 대한 보고 기능을 사용할 수 있는 환경에서는 편리하게 수행할 수 있습니다.고객 자신이 서버에 Urchin 인스턴스를 유지하는 전용/공동 위치 환경에서는 호스팅 업체가 사용된 모니터링에 적합한 샘플 필터를 제공해야 합니다.

Urchin 필터링 구현을 위해 호스트 및 고객은 함께 협력하여 소프트웨어 활용만 모니터링하는 특정 페이지를 고객 웹사이트에 만들어야 합니다. 예를 들면 다음과 같습니다. 

     http://www.customerdomain.com/healthpage.html

예

예 1:모니터링 시스템에 대한 IP 주소를 필터링 

      Filter Type: Exclude
      Filter Field: IP
      Filter Spec: 172\.16\.1\.1
  • 이 예에서는 Urchin이 처리하는 웹서버 로그 중 IP 주소가 172.16.1.1인 조회를 모두 제거합니다.

예 2:모니터링 시스템에서 조회한 특정 페이지 필터링 

      Filter Type: Exclude
      Filter Field: REQUEST
      Filter Spec: ^/healthpage.html
  • 이 예에서는 Urchin이 처리하는 웹서버 로그 중 /healthpage.html에 대한 요청이 있는 조회를 모두 제거합니다.

주의사항

  1. 이 방법은 고객이 웹서버에 대한 실제 트래픽 로드(필터링 포함)를 확인할 수 있도록 고객에 대해 필터링되지 않는 추가 프로필을 만드는 경우 유용합니다.
  2. 호스팅 업체는 모니터링 소프트웨어에서 조회만 필터링하는 것처럼 조회 모니터링을 독점적으로 보고하는 프로필을 제공하고자 할 수 있습니다.이 프로필은 모니터링이 올바르게 수행되었으며 SLA를 충족하는지 확인하는 데 사용할 수 있습니다.
