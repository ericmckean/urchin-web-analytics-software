맞춤 보고서 만들기

맞춤 보고서를 만드는 방법은 다음과 같습니다.

  1. 데이터 지도 테이블을 선택하거나 새 테이블을 정의합니다(맞춤 데이터 지도 참조);

  2. <Urchin>/lib/custom/profiletypes/Default/default.rs2.sample 파일을 복사하여 ‘.rs2’ 확장자로 새 보고서 파일을 만듭니다.

  3. 보고서 파일에 행을 추가하여 새 보고서를 정의합니다.

보고서 파일의 형식은 다음과 같습니다.

  • ID – 보고서 ID:
    • C=카테고리
    • D=데이터
    • M=메뉴
    • R=보고서
    • S=하위 보고서
    • P=페이지 보고서
    • B=대시보드
  • RS – 대시보드/보고 섹션:
    • 1=실무진
    • 2=마케팅 담당자
    • 4=웹마스터
  • RR - 보고서 요구사항:
    • 1=소프트웨어
    • 2=고급
    • 4=전자상거래
    • 8=UTM
  • NTITLE – 탐색 제목(사전 항목 번호)
  • ONCOLOR – 탐색 설정 색상
  • OFFCOLOR – 탐색 해제 색상
  • HELP – 보고서 도움말(사전 항목 번호)
  • NDATA – 데이터 요소 개수(바로 다음에 항목이 나와야 함)
    • 이 입력란 다음에 D(데이터) 유형의 추가 하위 보고서가 나올 수 있습니다.
  • CTITLES – 열 제목(사전 항목 번호)
  • CTYPES – 형식 지정에 사용된 열 유형
    • 10=문자열
    • 11=날짜/연월
    • 12=날짜
    • 20=번호
    • 21=수익
    • 22=바이트
    • 23=시간비
    • 24=비율
    • 25=pct
    • 26=시간
    • 30=총 pct
    • 40=수익
    • 41=수익/수
    • 42=수/수익
  • TABLE – 데이터 테이블 번호:
    • T=합계
    • TR=거래
    • H=히스토그램
    • V=방문자
    • 1-n 테이블
  • DXFORMAT – 데이터 추출 형식
  • RF – 필수 입력란
  • RT – 보고서 유형:
    • 1=전체 그래프
    • 2=히스토그램
    • 3=스택
    • 4=인기
    • 5=트리
    • 6=디렉토리
    • 7=그래프(제한)
    • 8=점
    • 9=클릭률
    • 10=정의된 유입경로
    • 11=홈페이지 분포
    • 12=방문자 분포
    • 13=목표 확인
    • 14=목표 경로
    • 20=KPI 날짜 보고서
    • 21=KPI 상위 10개
    • 22=동향 요약
    • 23=전환 요약
    • 24=Mini 지도
  • GTYPES – 그래프 유형:
    • 1=막대형
    • 2=선형
    • 3=원형
    • 4=세로 막대형
    • 5=펀
    • 6=다중 선형
  • FSEP – 입력란 구분기호
  • SP – 특수 옵션(비트맵 번호 추가):
    • 1=합계
    • 2=평균
    • 4=날짜 비교
    • 8=소문자 변환
    • 16=시간별 데이터
    • 32=오버레이
    • 64=언어 검색
    • 128=세그먼트
    • 256=평생 가치
    • 512=거래 내역
    • 1024=방문자 드릴다운:
    • 2048=용어 검색
  • SR – 특수 보고서 플래그:
    • 1=키워드 제안
    • 2=클릭률
    • 3= 시간대별 분석
    • 4=방문자 목록
    • 5=방문자 분포
    • 6=맞춤 사용을 위한 맞춤 사전
    • 7=날짜 비교
  • FNT -필터 번호 필터 유형:
    • 1: =

    • 2: >=

    • 3: <=

    • 4: >

    • 5: <

  • FNF -필터 번호 입력란
  • FNV – 필터값
  • FST – 필터 문자열 유형:
    • 1=제외
    • 2=포함
  • FSF - 필터 문자열 입력란
  • FSX – 정규식 문자열
