맞춤 날짜/시간 형식

소개

Urchin은 로그 파일에 포함된 거의 모든 날짜 또는 시간 형식을 처리할 수 있습니다.유일한 요구사항은 로그 파일에 포함된 날짜 형식과 일치하도록 Urchin에 필요한 날짜 또는 시간 형식을 제공하는 것입니다.이 문서에서는 날짜 및 시간 형식을 지정하는 데 사용되는 변수에 대해 설명합니다.이러한 형식은 맞춤 로그 형식 파일 안에 지정됩니다.이 섹션 상단의 '맞춤 로그 형식'을 참조하십시오.

날짜/시간 구문 분석의 작동원리

Urchin은 지정된 형식을 로그 파일의 날짜/시간 입력란과 비교하여 날짜/시간을 확인합니다.

예를 들어 IIS 로그에 다음과 같은 형식의 날짜가 포함되어 있는 경우 2002-11-12 Urchin은 다음 형식을 사용하여 연/월/일을 확인할 수 있습니다. %Y-%m-%d

날짜/시간 형식 만들기

맞춤 날짜/시간 형식을 만들려면 먼저 로그 파일에 포함된 날짜/시간 데이터의 순서와 패턴을 살펴봅니다.그런 다음 아래 목록의 날짜/시간 변수 중에서 선택하여 형식을 구성합니다.전세계 시간대는 UTC(coordinated universal time)의 양수 및 음수 오프셋으로 표현됩니다.

예를 들어 로그 파일에 '07:01:47'라는 시간이 포함되어 있으면 이와 일치하는 패턴을 만들어야 합니다.이 패턴에서 먼저 파악해야 할 사항은 시간:분:초의 순서라는 점입니다.아래 변수를 살펴보면 %H가 시간(hours) 변수이고 %M은 분(minutes) 변수이며 %S는 초(seconds) 변수임을 알 수 있습니다.이러한 변수를 '%H:%M:%S' 형식으로 조합합니다.날짜 또는 시간 형식 입력란에 '%' 문자가 있는 경우 리터럴 %를 %%로 지정할 수 있습니다.

가장 많이 사용되는 변수는%Y, %m, %d, %H, %M 및 %S입니다.

날짜/시간 변수 정의

  • %A = 평일 전체 이름에 대한 국내 표기
  • %a = 평일 약어 이름에 대한 국내 표기
  • %B = 달 전체 이름에 대한 국내 표기
  • %b = 달 약어 이름에 대한 국내 표기
  • %d = 10진수 형식의 월의 날짜(01-31).
  • %e = 10진수 형식의 월의 날짜(1-31), 1자리인 경우 앞에 공백 추가
  • %H = 10진수 형식의 24시간제 시간(00-23)
  • %I = 10진수 형식의 12시간제 시간(01-12)
  • %j = 10진수 형식의 연도의 날짜(01-31).
  • %k = 10진수 형식의 24시간제 시간(0-23), 1자리인 경우 앞에 공백 추가
  • %l = 10진수 형식의 12시간제 시간(1-12), 1자리인 경우 앞에 공백 추가
  • %M = 10진수 형식의 분(00-59)
  • %m = 10진수 형식의 월(01-12)
  • %p = '오전' 또는 '오후'에 대한 국내 표기
  • %S = 10진수 형식의 초(00-60)
  • %s = 시간 기점, UTC(mktime(3) 참조) 이후 초 수
  • %w = 일요일을 주의 첫 날로 한 10진수 평일(0-6)
  • %Y = 10진수 형식의 세기가 있는 연도
  • %y = 10진수 형식의 세기가 없는 연도(00-99)
  • %z = UTC로부터의 시간대 오프셋. 앞에 나오는 더하기(+) 기호는 UTC 동부를 나타내고 빼기(-) 기호는 UTC 서부를 나타내며 각각 두 자리의 시간 및 분이 구분 기호 없이 나옴(RFC 822 날짜 헤더에 대한 일반 형식)
  • %% = `%'.
