실패한 FTP 및 HTTP 원격 웹서버 로그 전송에 대한 오류 코드 목록

개요

FTP 또는 HTTP를 통해 원격 서버에서 웹서버 로그를 수집하도록 Urchin 로그 소스를 구성할 수 있습니다.이러한 전송은 대개 성공적으로 수행되며 런타임 로그에 오류가 나타나지 않습니다.그러나 전송 중에 오류가 발생하는 경우(예: 잘못된 사용자 이름/비밀번호, 확인할 수 없는 원격 서버, 확인할 수 없는 원격 로그 등) Urchin은 프로필의 작업 기록에서 볼 수 있는 형태로 런타임 출력에 오류 코드를 기록합니다.이 오류 코드는 웹서버에서 로그 전송을 시도한 후 "실패한" 메시지 옆에 괄호로 묶여 (-9)와 같이 표시됩니다.

발생한 문제를 설명하는 텍스트 메시지와 함께 오류 코드가 아래 나열됩니다.

오류 코드 목록

CURLE_UNSUPPORTED_PROTOCOL (1) - libcurl에 전달한 URL에서 이 libcurl이 지원하지 않는 프로토콜을 사용했습니다.사용하지 않는 컴파일 타임 옵션일 수 있으며, 프로토콜 문자열 철자가 틀렸거나 프로토콜 libcurl에 해당 코드가 없을 수 있습니다.

CURLE_FAILED_INIT (2) - 매우 이른 코드 초기화에 실패했습니다.내부 오류/문제일 가능성이 높습니다.

CURLE_URL_MALFORMAT (3) - URL 형식이 올바르게 지정되지 않았습니다.

CURLE_COULDNT_RESOLVE_PROXY (5) - 프록시를 분석할 수 없습니다.지정된 프록시 호스트를 분석할 수 없습니다.

CURLE_COULDNT_RESOLVE_HOST (6) - 호스트를 분석할 수 없습니다.지정된 원격 호스트가 분석되지 않았습니다.

CURLE_COULDNT_CONNECT (7) - 호스트 또는 프록시로 connect()에 실패했습니다.

CURLE_FTP_WEIRD_SERVER_REPLY (8) - FTP 서버에 연결한 후 libcurl은 특정 응답을 기다립니다.이 오류 코드에는 불완전하거나 잘못된 응답도 포함됩니다.지정된 원격 서버가 정상적인 FTP 서버가 아닐 수 있습니다.

CURLE_REMOTE_ACCESS_DENIED (9) - URL에 지정된 리소스에 대한 액세스가 거부되었습니다.FTP의 경우 원격 디렉토리의 변경을 시도할 때 이 오류가 발생합니다.

CURLE_FTP_WEIRD_PASS_REPLY (11) - 서버에 FTP 비밀번호를 전송한 후 libcurl은 해당하는 응답을 기다립니다.이 오류 코드는 예상치 못한 코드가 반환되었음을 나타냅니다.

CURLE_FTP_WEIRD_PASV_REPLY (13) - libcurl이 PASV 또는 EPSV 명령에 대한 응답으로 서버로부터 합당한 결과를 얻는 데 실패했습니다.서버에 오류가 있습니다.

CURLE_FTP_WEIRD_227_FORMAT (14) - FTP 서버에서 PASV 명령에 대한 응답으로 227줄을 반환합니다.libcurl에서 이 줄을 구문 분석하지 못하면 이 반환 코드가 다시 전달됩니다.

CURLE_FTP_CANT_GET_HOST (15) - 새 연결에 사용된 호스트를 조회하는 데 내부 오류가 발생했습니다.

CURLE_FTP_COULDNT_SET_TYPE (17) - 전송 모드를 바이너리(binary) 또는 아스키(ascii)로 설정하려고 시도할 때 오류가 수신되었습니다.

CURLE_PARTIAL_FILE (18) - 전송된 파일이 예상 길이보다 짧거나 더 깁니다.서버가 먼저 예상 전송 크기를 보고한 다음 이전에 지정한 크기와 일치하지 않는 데이터를 전송한 경우 이 오류가 발생합니다.

CURLE_FTP_COULDNT_RETR_FILE (19) - 'RETR' 명령에 대해 잘못된 응답이 반환되었거나 0바이트가 전송되었습니다.

CURLE_QUOTE_ERROR (21) - 원격 서버로 맞춤 'QUOTE' 명령을 전송할 때 명령 중 하나에서 FTP에 대해 400 이상의 오류 코드를 반환했거나 명령이 성공적으로 완료되지 않았음을 나타냅니다.

CURLE_HTTP_RETURNED_ERROR (22) - CURLOPT_FAILONERROR가 TRUE로 설정되고 HTTP 서버가 >= 400인 오류 코드를 반환하면 이 오류 코드가 반환됩니다(이 오류 코드는 이전에는 CURLE_HTTP_NOT_FOUND로 알려져 있었습니다).

CURLE_WRITE_ERROR (23) - 수신된 데이터를 로컬 파일에 쓸 때 오류가 발생하거나 쓰기 콜백에서 libcurl로 오류가 반환됩니다.

CURLE_UPLOAD_FAILED (25) - 업로드를 시작하지 못했습니다.FTP에 대해 일반적으로 서버는STOR 명령을 거부합니다.오류 버퍼에는 이에 대한 서버의 설명이 포함됩니다(이 오류 코드는 이전에는 CURLE_FTP_COULDNT_STOR_FILE로 알려져 있었습니다).

CURLE_READ_ERROR (26) - 로컬 파일을 읽는 데 문제가 있거나 읽기 콜백에서 오류가 반환되었습니다.

CURLE_OUT_OF_MEMORY (27) - 메모리 할당 요청에 실패했습니다.심각한 오류이며 발생할 경우 치명적인 문제가 발생합니다.

CURLE_OPERATION_TIMEDOUT (28) - 작업이 시간초과되었습니다.이 조건에 따라 지정된 시간초과 기간에 도달했습니다.

CURLE_FTP_PORT_FAILED (30) - FTP PORT 명령에서 오류를 반환했습니다.이 오류는 대부분 libcurl에서 사용할 적당한 주소를 지정하지 않은 경우 발생합니다.CURLOPT_FTPPORT를 참조하십시오.

CURLE_FTP_COULDNT_USE_REST (31) - FTP REST 명령에서 오류를 반환했습니다.서버가 정상적인 상태인 경우에는 발생하지 않습니다.

CURLE_RANGE_ERROR (33) - 서버에서 해당 범위의 요청을 지원하거나 수락하지 않습니다.

CURLE_HTTP_POST_ERROR (34) - 주로 내부 혼동으로 인해 발생하는 특이한 유형의 오류입니다.

CURLE_SSL_CONNECT_ERROR (35) - SSL/TLS 핸드쉐이크 중 임의 위치에서 문제가 발생했습니다.문제가 더 자세히 설명되어 있는 오류 버퍼에서 실제로 메시지를 확인할 수 있습니다.인증서(파일 형식, 경로, 권한), 비밀번호 등일 수 있습니다.

CURLE_FTP_BAD_DOWNLOAD_RESUME (36) - 파일 크기를 초과하여 FTP 재개를 시도했습니다.

CURLE_FILE_COULDNT_READ_FILE (37) - FILE://로 지정된 파일을 열 수 없습니다.파일 경로에서 기존 파일을 식별하지 못했기 때문일 가능성이 높습니다.파일 권한을 확인하십시오.

CURLE_LDAP_CANNOT_BIND (38) - LDAP에서 바인딩할 수 없습니다.LDAP 바인딩 작업에 실패했습니다.

CURLE_LDAP_SEARCH_FAILED (39) - LDAP 검색에 실패했습니다.

CURLE_FUNCTION_NOT_FOUND (41) - 함수가 없습니다.필요한 zlib 함수가 없습니다.

CURLE_ABORTED_BY_CALLBACK (42) - 콜백에 의해 중단되었습니다.콜백에서 libcurl에 'abort'를 반환했습니다.

CURLE_BAD_FUNCTION_ARGUMENT (43) - 내부 오류.잘못된 매개변수를 사용하여 함수가 호출되었습니다.

CURLE_INTERFACE_FAILED (45) - 인터페이스 오류.지정된 발신 인터페이스를 사용할 수 없습니다.나가는 연결의 소스 IP 주소에 사용할 인터페이스를 CURLOPT_INTERFACE로 설정합니다.(이 오류 코드는 이전에는 CURLE_HTTP_PORT_FAILED로 알려져 있었습니다.)

CURLE_TOO_MANY_REDIRECTS (47) - 리디렉션이 너무 많습니다.리디렉션 결과, libcurl이 최대 양에 도달했습니다.CURLOPT_MAXREDIRS로 한도를 설정하십시오.

CURLE_UNKNOWN_TELNET_OPTION (48) - CURLOPT_TELNETOPTIONS를 이용한 옵션 설정이 인식되지 않거나 알려지지 않았습니다.해당 문서를 참조하십시오.

CURLE_TELNET_OPTION_SYNTAX (49) - Telnet 옵션 문자열의 형식이 잘못 지정되었습니다.

CURLE_PEER_FAILED_VERIFICATION (51) - 원격 서버의 SSL 인증서나 SSH md5 지문이 올바르지 않은 것으로 간주되었습니다.

CURLE_GOT_NOTHING (52) - 서버에서 반환된 항목이 없으며 이 경우 오류로 간주되는 항목이 없습니다.

CURLE_SSL_ENGINE_NOTFOUND (53) - 지정된 암호화 엔진이 없습니다.

CURLE_SSL_ENGINE_SETFAILED (54) - 선택한 SSL 암호화 엔진을 기본값으로 설정하지 못했습니다.

CURLE_SEND_ERROR (55) - 네트워크 데이터를 전송하지 못했습니다.

CURLE_RECV_ERROR (56) - 네트워크 데이터를 수신하지 못했습니다.

CURLE_SSL_CERTPROBLEM (58) - 로컬 클라이언트 인증서에 문제가 발생했습니다.

CURLE_SSL_CIPHER (59) - 지정된 암호화를 사용할 수 없습니다.

CURLE_SSL_CACERT (60) - 알 수 없는 CA 인증서로 피어 인증서를 인증할 수 없습니다.

CURLE_BAD_CONTENT_ENCODING (61) - 전송 인코딩을 인식할 수 없습니다

CURLE_LDAP_INVALID_URL (62) - LDAP URL이 잘못되었습니다.

CURLE_FILESIZE_EXCEEDED (63) - 최대 파일 크기를 초과했습니다.

CURLE_USE_SSL_FAILED (64) - 요청된 FTP SSL 수준에 실패했습니다.

CURLE_SEND_FAIL_REWIND (65) - 전송 작업을 수행할 때 curl에서 재전송을 위해 데이터를 되감기해야 하나 되감기 작업에 실패했습니다.

CURLE_SSL_ENGINE_INITFAILED (66) - SSL 엔진 초기화에 실패했습니다.

CURLE_LOGIN_DENIED (67) - 원격 서버에서 curl 로그인을 거부했습니다(7.13.1에 추가됨)

CURLE_TFTP_NOTFOUND (68) -TFTP 서버에 파일이 없습니다.

CURLE_TFTP_PERM (69) - TFTP 서버에 권한 문제가 있습니다.

CURLE_REMOTE_DISK_FULL (70) - 서버에 공간이 없습니다.

CURLE_TFTP_ILLEGAL (71) - 잘못된 TFTP 작업입니다.

CURLE_TFTP_UNKNOWNID (72) - 알 수 없는 TFTP 전송 ID입니다.

CURLE_REMOTE_FILE_EXISTS (73) - 파일이 이미 있고 덮어써지지 않습니다.

CURLE_TFTP_NOSUCHUSER (74) - 정상적으로 작동하는 TFTP 서버에서는 이 오류가 반환되지 않습니다.

CURLE_CONV_FAILED (75) - 문자 변환에 실패했습니다.

CURLE_CONV_REQD (76) - 호출자가 변환 콜백을 등록해야 합니다.

CURLE_SSL_CACERT_BADFILE (77) - SSL CA 인증서를 읽는 데 문제가 발생했습니다(경로 또는액세스 권한 문제).

CURLE_REMOTE_FILE_NOT_FOUND (78) - URL에서 참조되는 리소스가 없습니다.

CURLE_SSH (79) - SSH 세션 중에 지정되지 않은 오류가 발생했습니다.

CURLE_SSL_SHUTDOWN_FAILED (80) - SSL 연결을 종료하는 데 실패했습니다.
