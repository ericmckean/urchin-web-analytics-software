Urchin이 표준 입력(stdin)으로부터 웹서버 로그를 읽도록 하려면 어떻게 해야 합니까?

때로는 Urchin 로그 처리 엔진이 웹서버 로그 데이터를 실제 파일로부터 읽는 것보다 표준 입력(stdin)으로부터 직접 읽는 것이 나을 수도 있습니다. Urchin에서는 실제 파일 대신 표준 입력으로부터 읽도록 로그 소스를 구성할 수 있습니다.

Urchin을 이와 같이 구성하려면 먼저 다음 매개변수를 사용하여 로그 소스 관리자에서 'stdin'이라는 새 로그 소스를 구성하십시오.

로그 설정

  • 로그 파일 위치:로컬
  • 로그 파일 경로:-
  • 로그 형식:자동

고급 설정

  • 로그 저장 위치:방치
  • 검색어 토큰:?
  • URI 스템을 소문자로 변환:끄기

참고: '로그 파일 경로'가 반드시 단일 대시 '-' 문자로 설정되어 있어야 합니다(따옴표는 제외).

이제 'stdin'으로부터 로그를 읽도록 할 모든 프로필에 대해 '프로필' -> '로그 소스 구성' 화면에서 새 'stdin' 로그 소스를 선택하기만 하면 됩니다.

중요사항: 'stdin' 로그 소스를 사용하여 Urchin 스케줄러를 통해 실행되도록 구성된 Urchin 프로필은 실행할 수 없습니다. 스케줄러에는 로그 입력 스트림을 올바르게 연결할 수 있는 방법이 없기 때문입니다.'stdin' 로그 소스가 있는 로그를 처리할 수 있는 유일한 방법은 Urchin을 수동으로 실행하는 것입니다.다음은 Urchin을 호출하는 아주 간단한 예입니다.

  1. Urchin 시스템에서 명령 쉘 창을 엽니다.
  2. 디렉토리를 Urchin 폴더의 'bin' 디렉토리로 변경합니다.

  3. [UNIX 계열 시스템] './urchin -p myprofile < /path/to/webserver.log'를 사용하여 urchin을 실행합니다.

    • [Windows 시스템] 'urchin -p myprofile < C:\path\to\webserver.log'를 사용하여 urchin을 실행합니다.

'stdin'을 로그 소스로 사용하면 유연한 작업이 가능합니다. Urchin에서 읽기 전에 로그에 몇 가지 사전 처리 작업을 쉽게 수행할 수 있기 때문입니다.

추가 예제

다양한 가상 호스트의 조회로 구성된 로그에서 단일 가상 호스트에 대한 조회를 추출하고 그 결과를 Urchin에 제공합니다. 

grep " www.mydomain.com "/path/to/alldomains.log | urchin -p mydomain.com

일종의 포함된 SQL 스크립트를 사용하여 관계형 데이터베이스에서 원시 웹서버 조회를 추출하고, Urchin에서 처리하도록 제공합니다. 

sql-extract --date 20040302 --domain mydomain.com | urchin -p mydomain.com
