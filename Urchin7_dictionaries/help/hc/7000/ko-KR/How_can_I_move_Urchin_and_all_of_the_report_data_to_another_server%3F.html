Urchin 6.x 및 모든 보고서 데이터를 다른 서버로 이동하려면 어떻게 해야 합니까?

Urchin 6.x를 다른 서버로 이동하려면 다음과 같은 기본적인 절차가 필요합니다.

  1. Urchin DB 준비
  2. 새 서버에 Urchin 6.x 설치
  3. 해당 프로필에 대한 Urchin 보고 데이터베이스 이동
  4. 맞춤 파일 및 템플릿 이동
  5. Urchin 맞춤 설정 적용
  6. 새 파일 권한 확인
  7. Urchin 라이센스 재활성화

참고:다음 세부 설명은 플랫폼 간 이동에도 적용할 수 있습니다.Urchin 보고 데이터베이스 파일은 Urchin에서 지원하는 모든 운영체제로 이동할 수 있습니다.플랫폼 유형 간에 데이터 및 파일을 이동할 때는 명령행 구문을 고려해야 합니다(예를 들어 Unix 시스템의 경우 './' 추가).'Standard\Windows\folders\' 및 '/standard/unix/directories/'와 같은 디렉토리 구문 변경도 고려해야 합니다.

1. Urchin DB 준비

새 Urchin 설치에서 동일한 DB 서버를 사용할 경우 이 단계를 건너뛰어도 됩니다.그렇지 않은 경우 새 서버에 새로 설치하기 전에 Urchin DB를 준비해야 합니다.

a. Urchin 사용자에 대한 적절한 권한이 있는 비어있는 Urchin DB를 만듭니다.

MySQL 백엔드 DB 생성 시 SQL 명령 예: 

   mysql> create database urchin character set utf8;
   mysql> GRANT ALL ON urchin.* to 'urchin'@'localhost' IDENTIFIED BY 'urchinpassword';

PostreSQL 백엔드 DB 생성 시 PSQL 명령 예: 

   postgres=# CREATE DATABASE urchin WITH ENCODING 'UTF8';
   postgres=# CREATE USER urchin WITH PASSWORD 'urchin password';
   postgres=# GRANT ALL ON DATABASE urchin TO urchin;
   postgres=# \connect urchin;
   postgres=# CREATE LANGUAGE 'PLPGSQL';

b. 이전 DB의 SQL 덤프를 만들고 새 DB로 가져오기하여 이전 서버에서 Urchin DB 데이터를 복사합니다.이 작업을 통해 프로필, 로그 소스, 사용자, 계정, 그룹, 스케줄러 및 기타 설정을 포함한 필요한 대부분의 Urchin 구성이 자동 복사됩니다.

MySQL 백엔드 DB에 대한 SQL 명령 예:

  • 원본 DB: 

   mysqldump -h {oldDBhostname} -u {username} -p {urchinDBname} > urchin.sql
  • 대상 DB: 

   mysql -h {newDBhostname} -u {username} -p {urchinDBname} < urchin.sql

PostreSQL 백엔드 DB에 대한 PSQL 명령 예:

  • 원본 DB: 

   pg_dump -h {oldDBhostname} -p {oldDBport} -U {username} {urchinDBname} > urchin.sql
  • 대상 DB: 

   psql -h {newDBhostname} -p {newDBport} -U {username} {urchinDBname} < urchin.sql

2. 새 서버에 Urchin 6 설치

이전에 준비한 DB를 사용하여 새 서버에 Urchin 6을 새로 설치합니다.

Windows에 설치하는 경우 '기존의 Urchin 용 데이터베이스 테이블 삭제 및 재생성' 옵션을 선택하지 않아야 하며 Unix에 설치하는 경우 '설치 시 구성 데이터베이스를 초기화하시겠습니까?'에 대해 '아니요'를 선택해야 합니다.

Urchin 설치에 대한 자세한 내용은 다음을 참조하십시오(DB 구성 단계를 건너뛰고 'Urchin 설치' 장부터 시작하십시오).

  • Urchin 설치 가이드(FreeBSD 및 Linux)

  • Urchin 설치 가이드(Windows)

urchin 데몬 또는 서비스를 중지한 후 다음 단계를 진행합니다. 

~/urchin/bin/urchinctl -stop

3. 해당 프로필에 대한 Urchin 보고 데이터베이스 이동

기본적으로 모든 Urchin 보고 데이터는 기본 Urchin 디렉토리의 하위 디렉토리인~/urchin/data/에 저장됩니다. 이 디렉토리는 하위 디렉토리 구조를 유지하기 위해 보관된 후 Urchin이 해당 위치에 설치되면 새 서버로 이동되어야 합니다.가능하면 새 서버에서 ~/urchin/data와 동일한 디렉토리 위치를 복원해야 합니다.

참고:이전 Urchin 설치에서 맞춤 기록 및 보고서 디렉토리를 사용한 경우 ~/urchin/data/reports 및 ~/urchin/data/history 대신 맞춤 디렉토리가 복사됩니다.

4. 맞춤 파일 및 템플릿 이동

맞춤 로그 형식, 보고서 집합 등 Urchin의 맞춤 항목은 Urchin의 'data' 디렉토리에 보관되지 않습니다.대신, 이러한 맞춤 항목은 Urchin 폴더의 "lib" 디렉토리 아래 별도의 위치에 보관됩니다.새 서버에 Urchin이 설치되면 이전 Urchin 시스템에 대한 맞춤 항목을 새 Urchin 설치로 복사해야 합니다.이러한 파일은 보통 ~/urchin/lib/custom/에 보관됩니다.

이 작업을 수행하려면 이전 Urchin 서버로 이동하여 Urchin 폴더의 "lib" 디렉토리로 이동합니다.그런 다음 tar 또는 ZIP과 같은 압축 유틸리티를 사용하여 전체 'custom' 디렉토리(모든 하위 디렉토리 포함)에 대한 아카이브를 만듭니다. 이 아카이브를 새 Urchin 서버로 복사한 후 Urchin 폴더의 "lib" 디렉토리에 압축을 해제합니다.

5. Urchin 맞춤 설정 적용

이전 Urchin 6.x 설치에 대한 기본 설정이 변경된 경우: 

/etc/session.conf
/etc/urchin.conf
/var/urchinwebd.conf

해당 파일을 수정하여 같은 설정을 새 설치에 반드시 적용해야 합니다.

6. 새 파일 권한 확인

파일을 모두 새 서버로 이동한 후에는 모든 파일에 필요한 권한이 있고 모든 파일의 소유자가 동일한 사용자인지를(UNIX 시스템만 해당) 확인해야 합니다.

Urchin 권한을 수정하려면 ~/urchin/util/ 디렉토리에서 다음 명령을 실행합니다. 

        inspector -r

파일 소유권을 확인하려면(UNIX만) ~/urchin/ 디렉토리에서 다음 명령을 실행합니다. 

        chown -R urchin_user:urchin_group./

7. Urchin 라이센스 재활성화

Urchin 데몬 또는 서비스를 시작합니다. 

~/urchin/bin/urchinctl start

Urchin 설치가 완료되면 Urchin 웹 인터페이스를 엽니다.

시스템에 '라이센스 재활성화' 링크가 포함된 'Urchin 라이센스' 화면이 표시됩니다.

이전에 준비한 DB를 사용하여 Urchin을 설치했으므로 시스템 키가 변경되어 라이센스를 활성화해야 합니다.

참고:재활성화 전에 라이센스를 다시 설정하려면 Urchin 판매처에 문의하십시오.
