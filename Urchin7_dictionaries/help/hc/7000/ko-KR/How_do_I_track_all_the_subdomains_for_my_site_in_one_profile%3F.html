한 프로필에서 내 사이트의 하위 도메인을 모두 추적하려면 어떻게 해야 합니까?

  • 이 문서는 Urchin 소프트웨어 공인 컨설턴트인 ivantage Limited에서 작성했습니다.

같은 프로필 내의 하위 도메인을 하나의 도메인으로 추적할 수 있습니다.이렇게 하려면 하위 도메인의 각 페이지의 추적 코드에 한 행을 추가해야 합니다.다음 도움말에서는 프로필의 기본 도메인을 아래에 나와 있는 웹사이트(예: example.co.uk)로 설정했다고 가정합니다.

  1. 하위 도메인과 기본 도메인의 모든 페이지에서 다음 행(진하게 표시됨)을 추적코드에 추가하십시오. 

         <script src="/urchin.js" type="text/javascript">
         </script>
         <script type="text/javascript">
           _udn="example.co.uk";
          </script>
  • 이 행은 기본 도메인(example.co.uk)을 참조해야 합니다.

위와 같이 추적코드를 변경하면 한 프로필 내의 하위 도메인을 모두 추적할 수는 있지만 보고서에는 www.example.co.uk와 help.example.co.uk의 페이지가 구분되어 표시되지 않습니다.예를 들어 상위 콘텐츠 보고서의 경우 www.example.co.uk/index.html 및 help.example.co.uk/more.html에 대한 조회수가 있을 수 있으나 보고서에는 다음과 같이 표시됩니다. 

   /index.html
   /more.html

하위 도메인을 구분하려면 다음과 같이 설정하여 프로필에 대한 고급 필터를 만드십시오. 

   Filter Type: Custom filter > Advanced
   Field A: cs_host (Raw)
   Extract A: (.*)
   Field B: Request_Stem (Auto)
   Extract B: (.*)
   Output To: Request_Stem (Auto)
   Constructor: /$A1$B1

이 필터를 적용하면 앞의 예에 다음과 같은 하위 도메인이 추가되어 나타납니다. 

   www.example.co.uk/index.html
   help.example.co.uk/more.html

참고:이 필터는 웹 액세스 로그 파일에 기록되는 하위 도메인을 포함한 웹사이트의 호스트 이름에 따라 달라집니다.웹 액세스 로그 파일에 호스트 이름이 없는 경우 Urchin 6 보고서는 두 도메인의 index.html 간의 차이를 기록할 수 없게 됩니다. 

 www.example.co.uk/index.html
 help.example.co.uk/more.html
