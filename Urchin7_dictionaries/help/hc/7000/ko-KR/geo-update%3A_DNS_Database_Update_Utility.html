geo-update: DNS 데이터베이스 업데이트 유틸리티

개요

geo-update 유틸리티는 Urchin의 내부 DNS 데이터베이스에 대한 업데이트를 확인하여 업데이트가 있을 경우 이를 다운로드합니다. 이 유틸리티를 사용하면 Google 다운로드 서버뿐 아니라 로컬 디렉토리에서도 업데이트를 다운로드(Urchin 6의 새로운 기능)할 수 있습니다.

사용법

geo-update 유틸리티는 Urchin 스케줄러(urchind)에서 매월 호출할 수도 있고 명령행에서 수동으로 호출할 수도 있습니다. 스케줄러를 통해 호출하는 경우 이 유틸리티는 항상 Google 다운로드 서버를 통해 업데이트됩니다. 명령행에서 호출하는 경우에는 일부 로컬 경로에서 DNS 데이터베이스를 업데이트할 수 있는 옵션이 추가로 제공됩니다. 이 작업은 매월 한 번씩 새 업데이트를 확인하도록 설정되며 사용자가 관리 인터페이스를 사용하여 일정 시간에 업데이트가 수행되도록 구성할 수도 있습니다.

geo-update를 위해 만든 작업은 'geo-update'라는 이름으로 작업 기록 화면에 표시되며,

이 유틸리티에서 하는 기본 작업은 업데이트를 확인하고 새 데이터베이스를 다운로드하여 기존 데이터베이스를 덮어쓴 다음 domain.local에서 로컬 데이터를 가져오는 것입니다. 명령행에서 geo-update 유틸리티를 실행할 때 적절한 옵션을 적용하면 기본 작업을 무시할 수 있습니다. 사용법은 다음과 같습니다. 

geo-update [-Hhv] [-D | -F] [-i file]

여기서 각 옵션의 용도는 다음과 같습니다. 

  -F            지리 데이터베이스를 다운로드합니다.
  -f            로컬 파일 시스템에서의 geo.zip 파일 경로입니다(선택사항).
  -H            실행 결과를 기록 파일에 저장하도록 지정합니다.
  -h            도움말 정보를 인쇄하고 종료합니다.
  -T <task id>  업데이트할 작업 기록 레코드를 지정합니다.
                -T를 지정하는 경우 -H도 함께 지정해야 합니다.
  -V            지리 데이터베이스의 버전 번호를 인쇄하고 종료합니다.
  -v            버전 번호를 인쇄하고 종료합니다.

-F 옵션을 사용하면 데이터베이스가 이미 최신 상태인 경우에도 새 데이터베이스가 다운로드됩니다. 이 기능은 잘못된 사용자설정 도메인 정보를 가져왔거나 도메인 데이터베이스가 손상되어 새로운 사본으로 작업을 다시 시작하려는 경우에 유용합니다.

-H 옵션을 사용하여 실행 결과를 기록 파일에 저장하면, Urchin 관리 인터페이스의 '구성'->'스케줄러' 영역에 있는 '작업 기록' 화면에서 실행 정보를 사용할 수 있습니다.

-T 옵션을 사용하면 명령행에서 <Task_ID>로 지정된 특정 작업을 업데이트할 수 있습니다. Task_ID 값은 다음 명령어를 사용하여 Urchin 6 데이터베이스에서 찾을 수 있습니다. 

mysql> select * from ugeo_tasks\G

예제

최신 도메인 데이터베이스를 다운로드합니다. 

geo-update -F

DB에서 이미 작업 ID를 검색한 경우 명령행에서 수동으로 작업을 업데이트합니다. 

geo-update -T <Task_ID> -H

geo-update 유틸리티에서 새 업데이트를 확인 및 다운로드하려면 인터넷에 연결되어 있어야 합니다. 이 업데이트는 포트 80을 사용하여 업데이트를 제공하는 웹서버와 통신합니다. 업데이트를 수행할 때 방화벽 및 프록시 서버를 거쳐 통신하는 경우 문제가 발생할 수 있습니다. 이 경우에는 네트워크 관리자에게 문의하시기 바랍니다.
