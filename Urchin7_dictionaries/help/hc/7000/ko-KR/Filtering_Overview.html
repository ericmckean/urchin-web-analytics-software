필터링 개요

이 문서는 Urchin 소프트웨어 공인 컨설턴트인 ['Panalysis']에서 작성했습니다.

필터링은 Urchin 6에서 중요한 개념입니다.필터를 통해 Urchin의 보고서 데이터 표시 방법을 제어할 수 있습니다.필터링은 간단하게 데이터 또는 복잡한 수정을 포함시키거나 제외하는 것일 수 있습니다.

필터의 일반적인 용도는 다음과 같습니다.

  • 웹사이트에서 자신의 활동을 제외합니다.
  • 웹사이트에서 특정 영역 또는 디렉토리에 제한된 보고서를 생성합니다.
  • 인기 콘텐츠 보고서에 도메인 이름을 추가합니다.
  • 인기 콘텐츠 보고서에 URL 쿼리 매개변수를 추가합니다.

필터는 가능한 5가지 작업 중 하나를 로그 파일 또는 Urchin 보고서에 있는 특정 입력란에 적용하여 작동합니다.가능한 작업은 다음과 같습니다.

  • 제외 - 보고서에서 데이터를 삭제합니다.

  • 포함 - 특정 항목에 대한 데이터만 보고서에 표시되도록 제한합니다.

  • 찾기 및 바꾸기 - 데이터를 다른 데이터로 바꿔서 데이터를 수정합니다.

  • 조회 테이블 - 외부 데이터 테이블을 기준으로 보고서에 콘텐츠를 추가합니다.예를 들어, ?id=1을 ?id=My Home Page로 전환합니다.

  • 고급 - 추출 중인 데이터를 수정하고 데이터를 결합합니다.

필터를 사용하려면 관리자로 로그인해야 합니다.로그인한 후에 화면의 오른쪽 상단에서 '구성' 탭을 클릭한 다음 Urchin 프로필로 이동하여 필터를 만들 수 있습니다.여기에서 '필터 관리자'를 통해 필터를 추가할 수 있습니다. 또는 프로필을 수정하거나 생성한 후 '필터' 탭에서 필터를 추가하고 '플러스' 아이콘을 클릭할 수도 있습니다.

필터링 순서

Urchin 6에서는 프로필 또는 로그 소스에 필터를 지정할 수 있습니다.

Urchin은 보고서를 생성하기 직전에 필터를 처리합니다.Urchin에서 사용하는 처리 순서는 다음과 같습니다.

원시 로그 파일 읽기 -> '원시' 로그 파일 입력란 생성 -> 입력란을 최종 '자동' 입력란으로 처리 -> 프로필에서 필터 처리 -> 로그 소스에서 필터 처리 -> 최종 보고서 표시

이는 보고서에 표시된 데이터를 수정하는 데만 필터를 사용할 수 있다는 의미입니다.따라서 대부분의 필터는 '(AUTO)'라고 표시된 입력란을 사용하여 최종 보고서에 표시되는 데이터를 수정합니다.물론 '(RAW)'라고 표시된 입력란을 사용해야 하는 경우도 있지만 이러한 경우는 비교적 드뭅니다.

중요사항:서로 다른 유형의 필터가 사용되는 경우에는 다음 순서대로 적용됩니다.

  • 고급, 찾기 및 바꾸기
  • 조회 테이블
  • 포함 및 제외

같은 유형의 필터가 여러 개 지정된 경우에는 관리 인터페이스를 통해 또는 DB 내에서 직접 로그 소스에 연결된 순서 그대로 처리됩니다.각 필터는 이전 필터가 적용된 후에 사용 가능한 데이터만 수락합니다.

두 개의 포함 필터를 연속적으로 적용할 때에는 실수를 범하기 쉽습니다.예를 들어, 다음과 같습니다.

필터 1:/somedirectory/ 디렉토리 포함, 필터 2:또 다른 디렉토리 /someotherdirectory/ 포함

페이지가 두 디렉토리 모두에 있을 수 없으므로 최종 보고서에는 0개의 페이지가 표시됩니다.그 이유는 두 번째 디렉토리는 /somediretory 디렉토리에 있는 데이터만 수락하기 때문입니다.두 번째 필터는 패턴 /someotherdirectory만 포함시키므로 결과가 0 페이지가 됩니다. 필터 1 이후에 남아 있는 페이지 중에서 이 패턴과 일치하는 항목이 없기 때문입니다.

이 문제를 해결하는 방법은 두 디렉토리 모두를 검색하는 정규식을 한 필터 안에서 사용하는 것입니다.예: ^/(somedirectory|somotherdirectory)/.*

도움말: 보고서 상단의 필터 입력란을 사용하여 Urchin 프로필에 추가하기 전에 관련 보고서에서 필터 패턴을 테스트하십시오.예를 들어, '콘텐츠 최적화' -> '콘텐츠 실적' -> '상위 콘텐츠' 보고서에서 URL을 테스트하십시오.
