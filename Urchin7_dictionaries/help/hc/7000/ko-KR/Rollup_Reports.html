전체 보고서 (6.602)

전체 보고서를 통해 Urchin "홈" 페이지에 있는 여러 프로필의 실적을 검토할 수 있습니다. "홈" 페이지는 "기본", "관리", "비즈니스"의 세 가지 보기를 제공하며, "보기" 드롭다운 목록에서 그 중 하나를 선택할 수 있습니다.

"편집" 버튼을 클릭하면 프로필 설정/보고 탭으로 이동합니다. "상태" 항목에는 프로필의 현재 상태가 표시됩니다. 이 상태는 선택한 날짜 범위의 영향을 받지 않습니다. 필터링된 프로필 옆에는 "필터링됨" 아이콘 ( ) 이 나타납니다. CPC 소스가 연결된 프로필에는 "광고 계정" 아이콘 ( ) 이 표시됩니다. "광고 계정" 아이콘을 클릭하면 AdWords로 이동합니다. CPC 소스가 프로필에 연결되어 있는 경우 "광고 계정" 항목에 CPC 소스의 수가 표시됩니다. 보고서를 보려면 프로필 이름을 클릭합니다. 실제 사이트로 이동하려면 프로필 이름 옆의 링크를 클릭합니다.

기본, 관리 및 비즈니스 보기가 아래에 표시되어 있습니다.

날짜 범위 선택기를 사용하여 가장 최근의 일, 주, 월 및 연도를 선택할 수 있습니다.

선택한 날짜 범위에 대한 데이터가 "조회수", "평균 방문 시간", "바이트수", "방문수", "이탈률", "목표 완료율", "페이지뷰" 항목에 표시됩니다. 나머지 항목("프로필 이름", "상태", "로그 소스", "필터링됨" 및 "광고 계정")에는 프로필에 대한 현재 정보가 표시됩니다.

각 사용자는 자신의 권한 수준에 맞는 데이터를 보게 됩니다. 다음 표에서는 권한 설정을 비교합니다.

Urchin 6.602부터 각 사용자에 대한 기본 홈 페이지(RollUp 보고서) 보기를 Urchin 구성 데이터베이스에서 uglobals.uigl_rollup_view_type 및 uusers.uius_rollup_view_type 입력란을 통해 수동으로 설정할 수 있습니다.

  • uglobals.uigl_rollup_view_type 값:
    • 0: 자동
      • 데이터베이스에서 기본값입니다. 이 경우 보기 유형에 대한 전체 값은 사용자 유형에 따라 결정됩니다.

        • 수퍼 관리자, 계정 관리자 -> 관리 보기

          사용자, 사용자 - 업데이트 없음 -> 비즈니스 보기

    • 1: 기본 보기
    • 2: 관리 보기
    • 3: 비즈니스 보기
  • uusers.view_type:
    • -1: 전체 값 사용
      • 새 사용자가 만들어질 때 항상 기본값으로 사용됩니다. 글로벌 전체보기 유형(uglobals.uigl_rollup_view_type)을 이 사용자에 대한 보기 유형으로 사용하는 것에 해당합니다.
    • 0: 자동
      • 이 경우 사용자의 보기 유형은 사용자 유형에 따라 결정됩니다.

        • 수퍼 관리자, 계정 관리자 -> 관리 보기

          사용자, 사용자: 업데이트 없음 -> 비즈니스 보기

    • 1: 기본 보기
    • 2: 관리 보기
    • 3: 비즈니스 보기

사용법 관련 중요사항

  • 대량의 데이터를 포함하고 있는 계정에 로그인하려고 하면 다음과 같은 오류 메시지가 나타납니다. 

    Session Error - unable to call default routine. (2002) Please login again.
  • 이는 전체 보고서를 위한 데이터를 추출할 때 걸리는 시간이 기본 APPLICATION_TIMEOUT인 45초를 벗어나기 때문입니다. 다음 해결 방법이 권장됩니다.
  • 이 문제가 해결되도록 XYZ 파일에서 APPLICATION_TIMEOUT 값을 보다 큰 값으로 늘립니다. 이렇게 변경할 경우 그에 따르는 간접적인 영향에 주의하십시오.
  • uglobals.uigl_rollup_view_type 그리고 Urchin 구성 데이터베이스에 있는 모든 uusers.uius_rollup_view_type 입력란을 값 '1'로 업데이트하여 보기 유형을 "기본 보기"로 변경합니다.
