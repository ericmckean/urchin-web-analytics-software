내 사이트의 도메인 이름이 여러 개인 경우추적을 어떻게 설정해야 합니까?

귀하의 사이트에서 다른 루트 도메인(예: www.mystore.com 및 www.securecart.com)에 있는 보안 쇼핑 바구니를 사용하거나 여러 도메인 별칭을 사용하여 동일 콘텐츠를 가리키는 경우 사이트와 UTM이 호환되도록 하려면 적절히 변경해야 합니다.

  • 여러 하위 도메인과 여러 루트 도메인은 다릅니다. www.example.com 및 securecart.example.com은 하위 도메인의 예이고 www.example.com 및 www.sample.com은 여러 루트 도메인을 나타냅니다. 사이트에 하위 도메인이 여러 개 있는 경우 다음을 참조하십시오. 한 프로필에서 내 사이트의 하위 도메인을 모두 추적하려면 어떻게 해야 합니까? 다른 프로필에 내 사이트의 하위 도메인을 모두 추적하려면 어떻게 해야 합니까? 설정 정보:

다중 도메인 이름:보안 상점

귀하의 사이트에서 여러 루트 도메인 이름(예: www.mystore.co.kr, www.securecart.com)을 사용하는 경우 내 쇼핑 바구니의 도메인 이름이 상점 사이트와 다릅니다.전자상거래 추적을 어떻게 설정해야 합니까?를 참조하십시오.

여러 도메인 이름:도메인 별칭

귀하의 사이트가 여러 도메인 별칭을 사용하는 경우 별칭의 트래픽을 기본 사이트로 리디렉션할 것을 권장합니다.그러면 UTM 방문자 추적이 기본 도메인에 설정되어 모든 방문자를 일관적으로 추적할 수 있습니다.

다음 내용에서는 Apache 및 IIS 서버에서 별칭 도메인을 기본 도메인으로 리디렉션하는 방법의 예를 보여 줍니다.

Apache에서 별칭 리디렉션

  • Apache 웹서버를 사용 중인 경우 별칭 도메인에서 발생하는 모든 트래픽이 기본 사이트로 리디렉션되도록 편리하게 설정을 수정할 수 있습니다.이렇게 설정을 변경하기 위한 방법 중 하나는 VirtualHost 항목을 두 개 생성하는 것입니다.첫 번째 항목은 일반 설정을 포함하는 기본 도메인이 되고 두 번째 VirtualHost 항목은 모든 별칭용으로 기본 사이트로 리디렉션됩니다. 

        #---primary virtualhost
        Servername www.mysite.com
        Serveralias mysite.com
        ...
        #---second virtualhost
        Servername mysite.org
        Serveralias www.mysite.org mysite.net www.mysite.net
        RewriteEngine on
        RewriteRule ^(.*) http://www.mysite.co.kr/$1 [R=301]

  • 두 번째 VirtualHost는 301(영구 이동) 리디렉션 코드가 있는 재작성 규칙을 사용하여 모든 트래픽을 원래 사이트로 전달합니다.이 때 UTM은 이 사이트를 정상적으로 UTM을 설치할 수 있는 하나의 도메인 사이트로 인식합니다.

IIS에서 별칭 도메인 리디렉션

  • Microsoft IIS 웹서버를 사용 중인 경우 별칭 도메인에서 발생하는 모든 트래픽이 기본 사이트로 리디렉션되도록 편리하게 설정을 수정할 수 있습니다.이렇게 설정을 변경하기 위한 방법 중 하나는 IIS 구성에 웹사이트를 두 개 생성하는 것입니다.첫 번째 항목은 일반설정을 포함하는 기본 도메인(www.mysite.com)이 되고 두 번째 항목은 다른 모든 별칭(mysite.net, mysite.org 등)용으로 기본 사이트로 리디렉션됩니다. IIS 관리자에서 웹사이트 중 하나를 마우스 오른쪽 버튼으로 클릭하여 속성 대화상자를 여십시오.'웹사이트' 탭에서 '고급...' 버튼을 클릭하십시오.그러면 '호스트 헤더' 입력란을 통해 웹사이트에 추가 도메인을 지정할 수 있는 창이 나타납니다.기본 웹사이트에서 기본 도메인을 설정하고 두 번째 웹사이트에 모든 별칭을 저장하십시오. 모든 별칭을 포함하는 두 번째 웹사이트를 구성 및 설정한 후 다음과 같은 리디렉션 코드를 삽입하여 홈페이지를 만드십시오. 

        <META HTTP-EQUIV=Refresh CONTENT="0; URL=http://www.mysite.com/">
  • 그러면 방문자의 브라우저를 기본 URL로 리디렉션하도록 지시됩니다.이때 기본 웹사이트는 단일 도메인 구성으로 표시되며, 기본설정을 사용하여 정상적으로 UTM 설치를 진행할 수 있습니다.
