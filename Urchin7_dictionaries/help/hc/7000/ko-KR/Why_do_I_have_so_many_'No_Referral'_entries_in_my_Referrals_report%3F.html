참조 보고서에 여러 '참조 없음' 항목이 있는 이유는 무엇입니까?

일반적으로 Urchin 참조 보고서의 '참조 없음' 항목은 다음과 같은 조건에 부합하는 경우 표시됩니다.

  • 웹서버 로그 파일 형식이 올바르지 않은 경우
  • 방문자가 URL을 직접 입력하거나 사이트를 즐겨찾기에 추가하여 사이트에 직접 액세스한 경우
  • 방문자가 이메일에 포함된 링크를 클릭하여 사이트에 직접 액세스한 경우
  • 검색엔진 웹 스파이더가 웹페이지를 식별하는 경우
  • 캐싱된 페이지 요청으로 인해 내부 참조가 발생하는 경우(UTM(Urchin Traffic Monitor)이 올바르게 구성된 경우는 해당하지 않음)

로그 형식: Urchin에서 참조를 보고서를 작성하도록 하려면 '참조' 입력란을 포함하도록 로그 파일을 구성해야 합니다.Apache 또는 IIS 로그 파일 구성에 대한 자세한 내용을 보려면 로깅- Apache 및 IIS를 참조하십시오.

직접 주소: 방문자가 해당 웹사이트 주소를 Explorer 의 주소 표시줄에 입력하면 웹서버로 참조 정보가 전달되지 않습니다.웹서버에 대한 모든 파일 요청 또는 '조회'가 파일에 기록되며 각 항목에는 데이터가 있거나 없는 참조 입력란이 포함됩니다.위에 설명된 대로 주소에 수동으로 액세스하면 방문자는 다른 사이트의 링크를 통해 오지 않았으므로 참조 입력란에 정보가 기록되지 않습니다.

이메일 링크: 방문자가 이메일 메시지(예: 이메일 광고 캠페인)에 포함된 링크를 클릭하면 참조 정보가 전달되지 않을 수 있습니다.사이트가 상주하는 웹서버로 참조 정보를 전달하는 이메일 링크를 만들 수 있습니다.단, 이 링크는 이메일 콘텐츠를 작성한 사용자가 설정해야 합니다.

웹 스파이더: 일반적으로 로봇 크롤러 및 스파이더는 대부분의 '참조 없음' 조회를 유발하는 검색엔진에 의해 생성되며 검색엔진 색인 항목을 생성하기 위해 웹사이트를 방문하여 해당 페이지와 기타 정보를 읽는 모든 프로그램을 말합니다.

내부 참조: Urchin은 올바르게 구성된 경우 보고서에 어떤 내부 참조도 포함하지 않습니다.내부 참조는 최초 요청된 페이지가 속하는 동일한 웹사이트의 참조를 말합니다.내부 참조는 참조 입력란에 포함되며 보고서에는 '참조 없음' 조회로 포함됩니다.내부 참조가 웹페이지에 대한 초기 요청에 나타나는 일은 드뭅니다.그러나 이러한 참조가 일반적으로 발생하는 한 가지 특정 조건이 있습니다.

대부분의 대형 ISP 및 검색엔진 사이트에서는 캐시 서버를 사용하여 웹사이트 콘텐츠를 캡처합니다.이 방법으로 이러한 페이지를 서비스하는 데 필요한 시간 및 네트워크 트래픽을 줄여 방문자에게 보다 빠른 서비스를 제공합니다.그러나 방문자가 웹페이지를 요청하고 캐시 서버가 이 요청에 응답하는 경우 웹서버 로그에서 이러한 요청에 대해 알지 못합니다.그 결과, 웹서버 로그 파일에는 요청에 대한 어떠한 정보도 남아 있지 않게 됩니다.이러한 방식에서는 대개 두 번째 또는 세 번째 요청을 할 때까지 웹서버가 신규 방문자를 파악하지 못하게 됩니다.이때까지 방문자가 요청한 페이지는 웹사이트의 다른 페이지에 있는 링크를 통해 액세스됩니다.따라서 로그 파일에 있는 참조 입력란에 자체 페이지가 참조로 표시됩니다.그 결과 urchin에서 '참조 없음' 보고서에 추가하는 내부 참조 조회가 발생합니다. Urchin 6에서는 UTM(Urchin Traffic Monitor)를 구성한 경우 누락된 로그 파일 데이터를 반환하도록 요구할 수 있습니다.자세한 내용은 UTM(Urchin Traffic Monitor) 작동원리를 참조하십시오.
