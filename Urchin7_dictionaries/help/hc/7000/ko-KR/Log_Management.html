로그 관리

개요

로그 관리는 Urchin 같은 소프트웨어를 실행할 때 중요한 부분입니다.트래픽이 많은 사이트는 비교적 빠른 시간에 대량의 로그 파일을 작성하기 때문에(어떤 경우에는 한 달에 수 기가바이트까지 가능함) 로그 관리를 신중하게 수행해야 합니다.표준 로그 로테이션 방식을 구축하는 것이 좋습니다.오프라인에서 파일을 압축하거나 오프라인으로 보관하는 것이 표준 방식입니다.이러한 절차를 구축하는 방법에 대한 자세한 내용은 로그 로테이션 모범사례를 참조하십시오.로그 관리는 데이터를 다시 처리하는 것을 방지하는 것이 아니라 디스크 리소스 사용을 조절하는 용도로만 필요합니다. Urchin은 이전에 읽은 로그 데이터가 다시 처리되지 않도록 하는 로그 추적 기능을 갖추고 있기 때문에 데이터 중복을 방지하기 위한 로그 로테이션이 필요 없습니다.Urchin은 처리된 로그 파일을 다시 읽어야 할 필요가 전혀 없기 때문에 각각의 처리 실행 후에 로그를 자유재량에 따라 삭제할 수도 있습니다.그러나 기록 또는 감사 등의 이유로 이전 로그를 지정된 기간 동안 보관하는 것도 일반적입니다.

Urchin을 통해 로그 관리

각 로그 소스에는 '방치', '보관/압축' 및 '삭제' 옵션이 제공되는 로그 저장 위치 설정이 있습니다('구성' -> 'Urchin 프로필' -> '로그 관리자' -> '수정'(프로필 근처) -> '고급 설정' 탭).로그 소스를 사용하는 모든 프로필의 처리가 끝난 후, Urchin은 로그 저장 위치 설정을 사용하여 로그 소스의 처리 방법을 결정합니다.로그 저장 위치 설정은 해당 로그 소스의 고급 설정 탭에서 액세스할 수 있습니다.로그를 일정 기간 동안 보관하려는 경우에는 로그 저장 위치를 '보관/압축'으로 설정하여 디스크 공간을 절약하는 것이 좋습니다.로그를 처리한 후에 삭제하는 것이 편한 경우에는 로그 저장 위치를 삭제로 선택해도 됩니다.그러나 이와 같이 설정하는 경우에는 다른 곳에 백업이 있지 않는 한, 나중에 해당 로그에 대해 Urchin을 다시 실행할 수 없게 됩니다.

주의사항

몇 가지 특수한 상황에 주의해야 합니다.

  • 실행 중인 로그를 처리하는 경우에는 로그 소스에 '보관' 또는 '삭제' 옵션을 사용하지 마십시오.실행 중인 로그는 웹서버에서 현재 작성 중인 로그입니다.실행 중인 로그에 이 설정을 사용하면 데이터가 손실됩니다.
  • 원격에서 검색되는 로그 소스의 로그 저장 위치가 '방치'로 설정된 경우에는 로그가 생성되는 컴퓨터에서 로그 관리를 수행하는 Urchin 외부의 프로세스가 있지 않는 한, 해당 로그가 점점 커지게 됩니다.Urchin은 처리하기 이전에 원격 로그 파일의 복사본을 로컬 시스템에 전송해야 하므로, 로그 파일이 커질수록 Urchin이 파일을 전송하는 데 시간이 오래 걸립니다.이는 전체적인 Urchin 실행 시간을 늘리는 부작용이 있습니다.
