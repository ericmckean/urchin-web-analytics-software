urchin: Urchin 로그 처리

개요

urchin의 Urchin Log Processing Engine인 Urchin은 로그 처리를 위한 핵심 구성요소입니다. 통상 작업이 실행되면 Urchin Scheduler(urchind)에서 로그 처리 엔진을 호출하지만, 특정 프로필을 확인하기 위해 명령 쉘에서 직접 urchin을 실행할 수도 있습니다. 이 방법은 Windows 작업 스케줄러 또는 UNIX 계열 시스템의 cron과 같은 외부 소스를 통해 Urchin 작업이 실행되는 스크립트 비중이 높은 환경에서, 그리고 특정일에만 발생하는 작업이나 디버깅과 같은 유형의 작업 등 특수한 상황에서만 프로필을 실행하는 경우에 유용합니다.

urchin은 범용 유틸리티라기 보다는 특정 환경에서만 사용할 수 있는 일부 한정된 명령행 기능을 제공합니다.

사용법

urchin은 Urchin 폴더의 bin 디렉토리에 있습니다. 사용법은 다음과 같습니다. 

urchin [options...] (-p <profile name> | -P <profile id>)

여기에서 각 옵션의 의미는 다음과 같습니다. 

-a <account name>       프로필의 계정 이름을 지정하며, 기본 계정 이름은 '(NONE)'입니다.

-D                      디버그 모드를 활성화합니다.

-d <YYYYMMDD>           로그 재처리일을 지정합니다.

-H                      로그 처리 결과를 기록 파일에 저장하도록 지정합니다. -T 옵션 없이 -H 옵션만 지정하면 새 기록 파일이 생성됩니다.

-h                      도움말 정보를 출력하고 종료합니다.

-P <profile id>         처리할 프로필 ID를 지정합니다.

-p <profile name>       처리할 프로필 이름을 지정합니다.

-T <task id>            업데이트할 작업 기록 레코드를 지정합니다. -T 옵션을 지정할 경우 -H도 함께 지정해야 합니다.

-t                      구성 데이터를 출력하고 종료합니다.

-v                      버전 번호를 인쇄하고 종료합니다.

주의사항

  • UNIX 계열 시스템에서 Urchin은 Urchin을 설치한 사용자/UID로 실행되어야 합니다. 이는 기존에 프로필 데이터베이스가 존재하지 않을 경우 Urchin에서 만드는 데, 이때 Urchin을 설치한 UID로 생성해야 하기 때문입니다.
