맞춤 로그 형식

소개

Urchin에서는 거의 모든 로그 파일 형식을 처리할 수 있습니다.Urchin에서 로그 파일 정보를 선택하면 Urchin이 해당 설정에 따라 원시 데이터를 읽고 구문 분석합니다.이 문서에서는 맞춤 로그 형식을 만드는 방법을 단계별로 설명합니다.생성된 맞춤 형식은 관리 인터페이스에서 로그 파일의 형식으로 선택될 수 있습니다.

일부 보고서는 특정 로그 데이터가 있어야만 생성할 수 있습니다.Urchin 로그에 최소 필수 입력란이 포함되어 있는지 확인하십시오.자세한 내용은 '문서 센터'의 'Urchin 관리' 섹션에서 '로그 파일' - '로깅' - '기타 웹서버' 문서를 참조하십시오.

맞춤 형식 생성

맞춤 로그 형식은 형식 사양 파일을 

   urchin install dir]/lib/custom/logformats/

폴더에 생성함으로써 만들어집니다.'custom.lf'라는 샘플 파일이 제공됩니다.여러 개의 맞춤 파일을 생성할 수 있으며각각의 파일에는 Urchin이 인식할 수 있도록 '.lf' 확장자가 필요합니다.apache, w3c, netscape 등의 기본적으로 제공되는 로그 형식은 다음 위치에 있습니다. 

   urchin install dir]/lib/reporting/logformats/

위에 있는 각각의 디렉토리의 fieldlist.txt 파일 안에 사용 가능한 입력란 목록이 있습니다.맞춤 폴더에는 맞춤 입력란 목록이 있고 보고 폴더에는 표준 입력란 목록이 있습니다.맞춤 로그 형식은 입력란 ID 번호를 사용하여 두 목록 중 하나에 있는 입력란을 참조할 수 있습니다.생성된 맞춤 형식은 관리 인터페이스에서 선택할 수 있습니다.다음은 맞춤 형식을 만드는 기본적인 단계입니다.

1단계:예제를 복사합니다. Urchin 폴더의 'lib/custom/logformats' 폴더에서 'custom.lf' 파일을 복사합니다.새 파일 이름에는 공백이 포함될 수 없으며 '.lf' 확장자가 있어야 합니다.

2단계:기본 위치를 설정합니다. 위 단계에서 생성한 파일을 편집합니다.이 파일에는 각 변수에 대해 유용한 정보가 다수 포함되어 있습니다.첫 번째 단계는 'PrimaryPositions' 변수를 설정하는 것입니다.이 쉼표로 구분되는 목록은 로그 형식에 있는 각 입력란(ID 번호 기준)과 각각의 상대적 위치를 식별합니다.앞에서 언급한 두 디렉토리에 있는 fieldlist.txt 파일을 사용하여 사용할 입력란 ID를 결정합니다.Apache 및 IIS 형식과 다른 맞춤 날짜와 시간 형식이 있는 경우에는 각각 입력란 16과 17을 사용하십시오.날짜와 시간이 한 입력란 안에 있으면 입력란 16만 사용하십시오.

3단계:입력란 구분기호를 확인합니다. 'FieldSeparator1' 변수를 확인하고, 입력란 사이의 구분기호로 설정합니다.공백의 경우에는 \s를 사용하고, 탭의 경우에는 \t를 사용합니다.

4단계:HTTP 상태 입력란을 사용할 수 있습니까? HTTP 상태 입력란을 사용할 수 있는 경우에는 'StatusRequired' 변수를 YES로 그대로 둡니다.그러면 유효한 조회와 오류 조회가 적절하게 분리됩니다.로그에 상태가 없는 경우에는 이 변수를 NO로 설정하십시오. 그러면 모든 조회가 유효한 것으로 간주됩니다.

5단계:맞춤 날짜/시간 형식을 사용하고 있습니까? 그렇다면 입력란 16의 'CustomDateFormat'과 입력란 17의 'CustomTimeFormat'을 수정하십시오.맞춤 날짜/시간 형식에 정의된 % 변수를 사용하여 형식을 지정합니다.

6단계:다른 변수를 확인합니다. 대부분의 다른 변수를 대다수 맞춤 로그 형식에서 사용할 수 있습니다. 하지만 각 변수에 대한 파일에 제공되어 있는 의견을 확인하여 현재 상황에 맞는지 확인하십시오.

7단계:맞춤 계산 입력란이 있습니까? 사양 파일에서 형식 외에도 맞춤 계산 입력란을 지정할 수 있습니다.파일의 아래쪽에 예제가 설명과 함께 제공됩니다.맞춤 계산 입력란은 필터링 섹션에 있는 고급 필터와 동일한 방식으로 작동합니다. 단, 맞춤 계산 입력란이 먼저 처리됩니다.자세한 내용은 맞춤 계산 입력란 양식에 대한 문서를 참조하십시오.

파일을 저장하면 관리 인터페이스에서 해당 파일을 로그 파일에 대한 형식으로 지정할 수 있게 됩니다.로그 형식의 풀다운 메뉴에 형식 옵션 중 하나로 자동으로 표시됩니다.
