설정 권장 사항

개요

Urchin이 설치된 다음에는 초기 실행 매개변수를 구성해야 합니다. 이 작업은 처음으로 Urchin 관리 인터페이스에 연결할 때 실행되는 설정 마법사에서 프로필을 설정하는 과정의 첫 번째 단계에서 수행됩니다. 초기 구성 작업에는 다음과 같은 사항이 포함됩니다.

  • Urchin 라이센스 부여
  • 데이터센터 모드 실행 설정 및 원격 보고 및 관리 액세스를 위한 관리 설정 구성
  • Urchin 관리자 계정 비밀번호 설정
  • 각 프로필에 대해 데이터 처리 작업 예약
  • 로그 관리

처리 절차

Urchin 관리 인터페이스에 연결합니다. Windows 시스템에서는 시작->프로그램->Urchin->Urchin 관리를 클릭합니다. UNIX 계열 시스템에서는 http://hostname:9999 URL을 사용할 수 있으며, 여기에서 hostname은 Urchin 시스템의 등록된 호스트 이름입니다. Urchin을 새로 설치한 경우 다음 로그인 정보를 사용해야 합니다.

사용자 이름: admin, 비밀번호: urchin

로그인하면 Urchin 설정 마법사 초기 화면이 표시됩니다. 계속을 클릭하여 각 마법사 화면의 안내에 따라 설정을 진행합니다. 초기 구성에서 선택된 설정은 나중에 언제든지 변경할 수 있습니다.

Urchin 라이센스 얻기

Urchin 라이센스 얻기의 안내에 따라 라이센스를 발급받습니다.

관리자 설정

  • 원격 액세스 설정 - 원격 브라우저 연결을 허용할 경우별로 설정을 선택합니다. 원격 액세스를 설정하지 않는 경우에는 Urchin이 설치된 시스템의 콘솔에서만 액세스할 수 있습니다.
  • 데이터센터 모드 - 이 설정에서는 계정 작성을 허용하도록 Urchin을 구성할지 여부를 결정할 수 있습니다. 계정 작성이 허용되면 관리자가 프로필, 그룹 및 사용자를 특정 액세스 카테고리로 분류할 수 있습니다. 이 옵션을 설정해도 추가 부하가 전혀 발생하지 않으며 향후 Urchin을 보다 유연하게 구성하여 사용할 수 있습니다.

관리 사용자

  • 관리자 계정의 비밀번호를 재설정한 후 기록하여 보관합니다.

작업 예약

프로필을 만들 때 해당 프로필에 대한 데이터 처리 작업을 실행할 시간을 예약할 수 있습니다. 각 프로필에 대한 설정을 확인하여 로그 데이터 사용 가능 시간, 데이터 처리에 소요되는 시간, 업데이트된 보고서가 사용자에게 제공되는 시간 등을 고려했을 때 작업 예약 시간이 적절한지 여부를 확인합니다.

로그 관리

Urchin에는 지금까지 각 로그가 어느 정도 처리되었는지 확인할 수 있는 추적 모듈이 포함되어 있습니다. 따라서 Urchin 작업에서는 로그 파일 순환을 반드시 설정하지 않아도 됩니다. 그렇지만 Urchin은 각 로그 소스의 고급 설정에서 자동 로그 순환 또는 삭제 기능을 제공합니다.

자세한 내용은 로그 관리를 참조하십시오.
