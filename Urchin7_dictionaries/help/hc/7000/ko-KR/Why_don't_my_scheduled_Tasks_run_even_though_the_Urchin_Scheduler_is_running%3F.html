Urchin 스케줄러가 실행되는데도 예약된 작업을 실행할 수 없는 이유는 무엇입니까?

문제 설명

특정 환경에서 Urchin 스케줄러(urchind)가 실행되지만 예약된 작업을 실행할 수 없는 경우가 있습니다.다음과 같은 경우입니다.

  1. 첫째, 작업이 Urchin 관리 도구의 '구성' -> '스케줄러' -> '작업 스케줄러'에 예약된 것으로 표시되고 예정대로 다시 예약되었으나 작업이 실제로 실행되지 않습니다.

  2. 둘째,"지금 실행" - '프로필 실행/예약' 화면에서 이 버튼을 클릭하면 예상되는 팝업 진행 상태 창이 나타나지만 진행 표시줄에는"오류(0%)" 만 나타나고 완료되지 않습니다.

원인

일반적인 이유는 Urchin 스케줄러가 삭제되었거나 권한이 변경된 디렉토리에서 시작되어 Urchin에서 더 이상 액세스할 수 없기 때문입니다.

설치 또는 업그레이드를 위해 임시 디렉토리에 Urchin 설치 프로그램의 압축을 푼 경우가 가장 일반적인 예입니다.설치/업그레이드 과정 중에 Urchin 스케줄러는 임시 디렉토리인 현재 작업 디렉토리에서 시작됩니다.나중에 이 디렉토리가 삭제되면 Urchin 스케줄러가 작동하지 않습니다.

해결 방법

Urchin 응용 프로그램이 올바르게 시작되어 실행되도록 하려면 Urchin이 항상 적절한 액세스 권한을 가지는 변하지 않는 위치에서 Urchin 스케줄러가 시작되도록 해야 합니다.스케줄러에 문제가 있는 경우 다음 절차에 따라 Urchin 서비스를 다시 시작하는 것이 좋습니다.

Windows 시스템

  1. '시작' -> '프로그램' -> 'Urchin' -> 'Urchin 서비스 사용 안 함'

  2. '시작' -> '프로그램' -> 'Urchin' -> 'Urchin 서비스 사용'

UNIX 계열 시스템

  1. 명령 쉘을"루트" 또는 Urchin을 실행하는 사용자로 시작합니다.
  2. Urchin이 설치된 위치로 디렉토리를 변경합니다. 예:"cd /usr/local/urchin"
  3. 다음 명령으로 Urchin 서비스를 다시 시작합니다."bin/urchinctl restart"
