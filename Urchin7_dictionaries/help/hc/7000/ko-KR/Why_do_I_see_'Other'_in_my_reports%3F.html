보고서에 '기타' 항목이 나타나는 이유는 무엇입니까?

Urchin 보고서의 모든 레코드는 고정된 크기의 데이터베이스에 저장됩니다.데이터베이스를 구성하는 이 파일은 매달 로테이션되므로매달 데이터베이스에서 목록화할 수 있는 고유 레코드 수가 제한됩니다.

이 한도에 도달한 경우 Urchin은 '기타'라는 마지막 레코드를 하나 생성합니다.'기타' 레코드는 여러 보고서에 나타날 수 있습니다.한계에 도달한 후 보고서에 추가되는 고유 레코드는 모두 '기타'로 이동됩니다.따라서 데이터베이스가 가득 찬 상태에서 5개의 새 페이지를 보는 경우 5개의 추가 페이지뷰가 '기타'라는 레코드로 분류됩니다.

데이터베이스 레코드에 대한 한도는 늘릴 수 있습니다.기본적으로 100,000개의 레코드로 설정되어 있습니다.이는 Urchin에서 매달 모든 프로필에 대해 총 100,000개의 고유 레코드 항목을 기록한다는 것을 의미합니다.'수퍼 관리자' 권한이 있는 경우 이 값을 늘릴 수 있습니다.해당 권한이 없는 경우 Urchin 관리자에게 문의하여 변경하도록 요청해야 합니다.

전체적으로 DB 테이블 제한 상향조정:

  • 인터페이스의 오른쪽 상단의 '구성' 버튼을 클릭합니다.
  • 왼쪽 탐색 창에서 '설정'을 클릭합니다.
  • 'DB 설정' 탭을 클릭합니다.
  • 드롭다운 목록 화살표를 사용하여 'DB 테이블 제한'을 원하는 값으로 늘립니다.
  • '업데이트'를 클릭하여 변경사항을 저장합니다.

기존 프로필에 대한 DB 테이블 제한 상향조정:

  • 인터페이스의 오른쪽 상단의 '구성' 버튼을 클릭합니다.
  • 해당 프로필에서 수정 렌치를 클릭합니다.
  • '저장/DB' 탭을 클릭합니다.
  • DB 테이블 제한 값을 원하는 새 레코드 제한 값으로 변경합니다.'전체 기본값'을 사용하면 위에 설명한 바와 같이 프로필 제한 값과 현재 글로벌 설정 값을 일치시킬 수 있습니다.
  • '업데이트'를 클릭합니다.

참고:이렇게 하면 Urchin 보고서 인터페이스가 방대한 데이터베이스에서 데이터를 검색해야 하기 때문에 디스크 공간 사용량과 여러 보고서 보기를 여는 데 소요되는 시간이 늘어납니다.
