CPC 데이터 가져오기 유틸리티(Urchin 6.5 이상)

Urchin 6.5에 CPC 데이터 처리용 유틸리티가 추가되었습니다.

urchincpc - Urchin CPC 데이터 가져오기 도구로 추가 처리 및 보고에 사용할 CPC 데이터를 Google 애드워즈 및 Yahoo! Search Marketing API를 통해 추출합니다.

  • 참고: 애드워즈에서 다운로드하여 보관 중인 CPC 데이터는 250MB를 초과할 수 없습니다(Urchin 6.501 이상).

urchincpcd - Urchin CPC 소스 스케줄러 또는 CPC 스케줄러라고도 하며 사용자가 정의한 일정에 따라 CPC 다운로드 작업을 예약하는 데 사용됩니다(프로필 스케줄러와 유사).

CPC 스케줄러(urchincpcd)는 Urchin 마스터 스케줄러(urchind)와 함께 로컬 호스트에서 실행됩니다. CPC 소스 일정을 예약하려면 마스터 스케줄러가 실행 중이어야 합니다. CPC 스케줄러는 주기적으로 CPC 데이터 다운로드 작업 대기열을 확인하고 실행을 관리합니다.

예약된 작업이 있으면 CPC 가져오기 도구(urchincpc)가 자동으로 실행됩니다. CPC 스케줄러는 하위 작업 프로세스를 내부에 생성하지만, 명령행을 통해 이 프로세스를 직접 호출하여 대기열에 있는 작업을 실행할 수 있습니다.

다운로드된 CPC 데이터는 로컬 디렉토리인 /path/to/urchin/data/cpc/에 저장되며 연결된 프로필 처리 시 Urchin 엔진에 의해 자동 검색됩니다.

Urchin CPC 스케줄러 관리에는 urchinctl 유틸리티가 사용됩니다. 명령행에서 CPC 소스 스케줄러를 호출하려면 '-c' 옵션을 사용합니다. 예를 들면 다음과 같습니다. 

urchinctl -c start/stop/restart

이 명령은 CPC 스케줄러만 실행/중지/재실행합니다.

CPC 데이터를 즉시 다운로드하려면 특정 CPC 소스의 구성 -> CPC 소스 관리자-> 다운로드 또는 수정 -> 예약 탭에서 ‘지금 다운로드’ 버튼을 클릭합니다. 이 요청이 처리되려면 CPC 스케줄러가 실행 중이어야 합니다.

사용법: CPC 스케줄러 - urchincpcd

CPC 스케줄러는 Urchin(6.5 이상) 내 bin 폴더에 있습니다. 명령행에서 다음과 같은 옵션을 사용할 수 있습니다.

  • -d 디버그 모드를 활성화합니다. 자세한 디버그 정보가 출력 페이지에 기록되며, Windows에서 -console 옵션과만 사용할 수 있습니다. 예: 

    •            urchincpcd.exe -console -d

  • -h 도움말 정보를 출력하고 종료합니다.

  • -v 버전 정보를 출력하고 종료합니다.

Windows에서는 첫 번째 인수로 다음과 같은 명령을 사용할 수 있습니다.

  • -install CPC 스케줄러 Windows 서비스를 설치합니다.

  • -remove CPC 스케줄러 Windows 서비스를 제거합니다.

  • -start CPC 스케줄러 Windows 서비스를 시작합니다.

  • -stop CPC 스케줄러 Windows 서비스를 중지합니다.

  • -console 콘솔 모드로 CPC 스케줄러를 시작합니다.

사용법: CPC 가져오기 도구 - urchincpc

명령행에서 다음과 같은 옵션을 사용할 수 있습니다.

  • -c [id] 특정 비용 소스를 다운로드합니다. ID는 CPC 소스의 ID입니다.

  • -d 디버그 모드를 활성화합니다.

  • -h 도움말 정보를 출력하고 종료합니다.

  • -v 버전 정보를 출력하고 종료합니다.

CPC 스케줄러를 매개변수 없이 또는 -d 옵션만으로 실행하면 대기열에 있는 작업을 모두 처리한 후 대기 중인 작업이 없으면 종료합니다.

프록시를 통해 CPC 데이터 다운로드하기(6.501 이상)

프록시를 통한 CPC 데이터 다운로드를 설정하려면 urchin.conf 파일을 수정하고 WebServiceProxyPort, WebServiceProxyUsername 및 WebServiceProxyPassword 항목의 주석을 해제한 후 원하는 값을 입력합니다. 

 ######################################################
# CPC 소스 관리자 설정
#...
# WebServiceProxyHost, WebServiceProxyHost: HTTP/HTTPS 프록시의 호스트 및 포트 번호로
# CPC 소스 관리자에서 외부 CPC 소스에 액세스하는 데 필요합니다.
# WebServiceProxyUsername, WebServiceProxyPassword: 승인을 실행하는 프록시의 사용자 이름 및
# 비밀번호입니다.
######################################################
#WebServiceProxyHost: <프록시 호스트>
#WebServiceProxyPort: <프록시 포트>
#WebServiceProxyUsername: <사용자 이름>
#WebServiceProxyPassword: <비밀번호>

WebServiceProxyHost에는 IP 주소나 전체 도메인 이름을 입력할 수 있습니다.
