로그 재처리

개요

DNS 서버가 처리 중에 다운되었거나, 필터가 잘못 적용되거나 하는 등의 이유로 로그 데이터를 다시 처리해야 하는 경우가 있을 수 있습니다.다음 문서에서는 웹서버 로그 데이터를 취소하고 다시 처리하는 올바른 절차를 설명합니다.

로그를 다시 처리하려면 명령행 쉘 환경에서만 사용할 수 있는 Urchin 유틸리티를 사용해야 합니다.Urchin 웹 기반 관리 GUI에서 모든 절차를 단독으로 수행하는 것은 불가능합니다.

하루 재처리

  • Urchin 관리 GUI에서 프로필을 수정하고 '저장/DB' 탭에서 '로그 추적'을 끕니다.'업데이트'를 클릭하여 변경 내용을 저장해야 합니다.
  • '로그 소스' 탭 아래에서 다시 처리해야 하는 로그 파일이 올바르게 지정되었는지 확인합니다.로그 데이터에는 통계에서 제거할 날짜의 조회만 포함되어 있어야 합니다.
  • Urchin 시스템에서 명령 쉘을 호출합니다. 

cd /<URCHIN_DIR>/util
  • 다음 명령을 사용하여 Urchin 폴더의 'util' 디렉토리/폴더에서 udb-sanitizer 유틸리티를 실행합니다. 

udb-sanitizer -p profile-name -d YYYYMM
  • 여기서 YYYYMM은 다시 처리할 날이 포함되어 있는 연도와 달입니다.
  • 옵션 5(하루 이상 제거)를 선택합니다.유틸리티에서 날짜가 올바른지 묻는 메시지가 나타나고 해당 날짜에 대한 통계를 제거합니다.udb-sanitizer 사용 방법에 대한 더 많은 예를 보려면 ['udb-sanitizer:데이터베이스 유지관리']를 참조하십시오.
  • 프로필의 '실행/예약' 탭 아래에서 '지금 실행' 버튼을 클릭하여 로그 날짜를 다시 처리합니다.
  • 로그 파일 경로를 원래의 설정으로 다시 변경하여 로그 소스를 재설정합니다.
  • 프로필 수정 영역의 '저장/DB' 탭 아래에서 '로그 추적'을 다시 켭니다.

한달 전체 재처리:

한 달 전체에 해당하는 데이터를 다시 처리하는 절차는 위에서 설명한 하루 데이터 재처리 절차와 유사합니다. 대신 서로 다른 udb-sanitizer 옵션을 사용할 뿐입니다.예제는 ['udb-sanitizer:데이터베이스 유지관리']를 참조하십시오.
