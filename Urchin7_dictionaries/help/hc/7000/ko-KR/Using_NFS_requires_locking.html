NFS사용 시 잠금 필요

data/conf 폴더에 위치한 데이터 파일을 사용하는 Urchin 구성은 설정·관리 시 읽기 및 쓰기 잠금을 설정합니다.data/conf 폴더가 NFS에 설치된 경우에는 NFS 서버가 적절한 잠금 데몬을 실행하여 원격 잠금을 처리해야 합니다.

잠금이 제대로 작동하지 않으면 Urchin 설치 중에 시스템 정지 오류가 발생할 수 있습니다.

기술적 참고사항

Urchin은 UNIX에서 fcntl() 함수를 사용하여 전문적 및 배타적(읽기/쓰기) 잠금을 수행합니다.일부 이전 시스템은 여러 플랫폼의 모든 rpc 잠금 요청을 지원하지 못할 수도 있습니다.최신 패치가 설치된 최신 버전의 OS를 실행 중인지 확인하십시오.NFS 서버가 'statd' 및 'lockd' 또는 이와 동등한 항목을 실행 중인지 확인하십시오.
