본 페이지에는 Urchin을 Windows Server 2008 및 Windows Vista에 설치할 때 참고할 사항이 설명되어 있습니다. Windows Server 2008 및 Windows Vista에서의 설치 방법은 Urchin 설치 가이드(Windows)에 설명되어 있지만 다음 사항을 참고하는 것도 도움이 됩니다.

Windows 2008은User account control (사용자 계정 제어, UAC)이라고 하는 새로운 보안 기능을 제공합니다. 자세한 내용은 다음 페이지를 참조하시기 바랍니다. http://technet.microsoft.com/en-us/library/cc709691.aspx

UAC 사용 시 자주 발생하는 문제는 다음과 같습니다.

  1. UAC를 사용 설정하면 응용프로그램에서 ‘Program files’ 폴더에 파일을 저장하는 데 문제가 발생합니다. 자세한 내용은 다음 사이트를 참조하십시오. http://msdn.microsoft.com/en-us/library/aa905330.aspx

  2. 기본 사용자 권한으로 Urchin의 실행 파일을 명령 프롬프트로 실행할 수 없게 되므로, 관리자 권한으로 명령 프롬프트를 사용하거나 실행 파일의 속성을 다음과 같이 지정해야 합니다. 마우스 오른쪽 버튼으로 클릭한 후 속성 -> 호환성 -> ‘관리자 모드로 이 프로그램 실행’ -> 확인을 선택합니다. Urchin Windows 서비스는 이러한 변경 작업 없이도 원래대로 잘 작동됩니다.

해결 방법:

  1. UAC를 사용중지한 후 원하는 폴더에 Urchin을 설치하고 실행 파일이나 Windows 서비스를 사용하여 Urchin을 실행합니다.
  2. UAC가 사용 중이면 Program files 폴더에 Urchin을 설치하지 말고 다른 폴더(예: Users 디렉토리)에 설치하십시오. 속성을 수정하여 실행 파일을 관리자 모드로 실행(위 단락의 2번째 항목 설명 참조)하거나 수정 없이 Windows 서비스를 이용합니다.
