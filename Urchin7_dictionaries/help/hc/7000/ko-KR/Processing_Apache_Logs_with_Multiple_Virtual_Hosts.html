여러 개의 가상 호스트가 있는 Apache 로그 처리

소개

여러 개의 가상 호스트가 있으며 호스트된 모든 웹사이트에 대한 로깅이 한 로그 파일에서 수행되는 Apache 웹서버를 조직에서 보유하고 있을 수도 있습니다.Urchin에서는 각 가상 호스트에 대해 개별 로그를 처리하는 것이 보다 효율적이지만, 결합된 단일 로그를 읽는 각 가상 호스트에 대한 프로필을 구성할 수도 있습니다.

처리 절차

1단계: Apache 웹서버가 Urchin에 알맞은 형식으로 로그를 생성하도록 구성되어 있는지 확인합니다.특히 다음 입력란이 아래의 순서대로 로깅되는지를 확인해야 합니다.

  1. 원격 호스트 [%h]
  2. 요청된 호스트 [%v]
  3. 사용자 이름 [%u]
  4. 날짜 및 시간 (NCSA) [%t]
  5. 요청 (NCSA) [%r]

  6. 상태 [%>s]

  7. 바이트 [%b]
  8. 추천자 [%{Referer}i]
  9. 사용자 에이전트 [%{User-Agent}i]
  10. 쿠키(예를 들어, UTM을 위해 쿠키를 로깅하는 경우) [%{Cookie}i]

2단계: Urchin 관리 인터페이스 안에서 '구성' -> 'Urchin 프로필' -> '로그 관리자'로 이동하고 단일 로그 파일을 가리키는 로그 소스를 추가합니다.'로그 형식'으로 '자동'을 선택합니다.

3단계: '구성' -> 'Urchin 프로필' 아래에서 각 가상 호스트에 대한 새 프로필을 만듭니다.'프로필 추가 마법사'에서 각 프로필에 대한 로그 소스를 추가할 때 '기존 로그 소스 추가'를 선택하고 위에서 구성한 로그 소스를 선택합니다.

4단계: 이제 '필터 관리'로 이동하고 '추가' 버튼을 클릭하여 새 필터를 추가합니다.'필터 마법사:설정' 화면에서 '패턴만 포함'을 필터 종류로 선택하고, 'cs_host(RAW)'를 필터 입력란으로 선택하고, '필터 패턴' 입력란에 가상 호스트 이름을 입력합니다.그런 다음 이 필터를 해당하는 프로필에 지정합니다.

5단계: 각 가상 호스트에 대해 3-4단계를 반복합니다.

주의사항

이 절차는 여러 개의 가상 호스트를 제공하는 데 있어서는 편리하지만 Urchin에서 각 가상 호스트에 대해 전체 로그 파일을 다시 처리해야 합니다.큰 로그 파일에서는 이로 인해 상당한 부하가 발생하여 Urchin의 성능에 영향을 줄 수 있습니다.따라서 대신 각 가상 호스트에 대해 별도의 로그 파일을 만드는 것이 좋습니다.

이 절차는 Apache에 맞춰 특별히 작성되어 있지만 같은 NCSA 유형의 입력란에서 로깅하는 다른 웹서버 로그 형식에서도 유사합니다.

현재 Urchin에서 새로운 가상 호스트를 자동으로 감지하고 이들에 대한 새 프로필을 자동으로 구성하는 것은 불가능합니다.모든 프로필은 명시적으로 설정되어야 합니다.
