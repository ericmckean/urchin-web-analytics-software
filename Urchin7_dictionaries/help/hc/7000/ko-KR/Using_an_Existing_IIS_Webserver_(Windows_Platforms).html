기존 IIS 웹서버 사용하기(Windows 플랫폼)

기본적으로 Urchin 관리 및 보고 작업은 Urchin 제품과 함께 제공되는 독립형 Apache 서버를 사용합니다. 실제로 설치된 대다수 Urchin에서 Urchin 관리 및 보고 인터페이스로 이 방법을 사용합니다. 그러나 드물지만 기존 IIS 웹서버를 사용해야 할 경우도 있습니다. 서버에서 타사 웹서버 제품의 사용을 금지하거나 기존 IIS 서버의 가상 호스트로 Urchin 보고를 설정해야 하는 등의 사이트 요구사항에 따라 이러한 경우가 발생할 수 있습니다.

면책조항: Urchin 설치에서 이러한 수정은 지원되지 않습니다.

처리 절차

참고: 이 절차에서는 C:\Program Files\Urchin 기본 위치에 Urchin이 설치되어 있다고 가정합니다. 다른 위치에 Urchin을 설치한 경우 아래 예제의 위치를 적절하게 변경해야 합니다.

1단계: Urchin 웹 인터페이스를 위한 새로운 사용자 만들기

  • 관리 도구 -> 컴퓨터 관리로 이동합니다.

  • 컴퓨터 관리 화면 왼쪽에 있는 로컬 사용자 및 그룹을 클릭합니다.

  • 사용자 폴더를 마우스 오른쪽 버튼으로 클릭한 후 새 사용자...를 선택합니다.

  • 사용자 이름: 입력란에 IUSR_URCHIN을 입력합니다.

  • 다시 로그인할 때 반드시 비밀번호 변경 확인란을 선택 해제합니다.

  • 사용자가 비밀번호를 변경할 수 없음 확인란을 선택합니다.

  • 만들기 및 닫기를 차례로 클릭합니다.

  • 왼쪽의 사용자 폴더를 두 번 클릭합니다.

  • IUSR_URCHIN을 마우스 오른쪽 버튼으로 클릭한 후 속성...을 선택합니다.

  • 소속 그룹 탭에서 모든 기존 항목을 삭제하고 추가.. 버튼을 클릭합니다. 그런 다음 팝업 창에서 Guests를 선택하고 추가와 확인을 차례로 클릭합니다.

  • 적용 및 닫기를 차례로 클릭하여 변경사항을 저장합니다.

2단계: Urchin 설치(설치되지 않은 경우)

3단계: Urchin Apache 웹서버 비활성화

  1. 관리 도구 -> 서비스로 이동합니다.

  2. 서비스에서 Urchin 웹서버 레코드를 찾습니다.

  3. Urchin 웹서버를 마우스 오른쪽 버튼으로 클릭한 후 중지를 선택합니다.

  4. Urchin 웹서버를 마우스 오른쪽 버튼으로 클릭한 후 속성을 선택한 다음 시작 유형을 사용 안 함으로 변경합니다.

  5. 확인을 클릭합니다.

4단계: IIS에 새 웹사이트 추가 관리 도구 -> IIS(Internet Information Services) 관리자로 이동합니다.

  1. 웹사이트를 마우스 오른쪽 버튼으로 클릭합니다.

  2. 설명 입력란에서 Urchin을 입력하고 다음을 클릭합니다.

  3. IP 주소와 포트 번호(보통 9999)를 선택하고 다음을 클릭합니다.

  4. 경로: 입력란에서 Urchin이 설치된 위치(보통 C:\Program Files\Urchin\htdocs)를 찾고 다음을 클릭합니다.

  5. 실행: 확인란을 선택한 후 다음을 클릭한 다음 종료합니다.

  6. 새 Urchin 웹사이트를 마우스 오른쪽 버튼으로 클릭한 후 속성을 선택합니다.

  7. 웹사이트 탭에서 로깅 사용을 선택 해제합니다.

  8. 홈 디렉토리 탭을 클릭하고 스크립트 소스 액세스를 선택합니다.

  9. 문서 탭을 클릭하여 기본 항목을 모두 삭제한 다음 추가를 클릭하여 나타나는 팝업 창에 session.cgi를 입력하고 확인을 클릭합니다.

  10. 디렉토리 보안 탭을 클릭하고 인증 및 액세스 제어 영역에서 수정을 클릭합니다. 익명 액세스 확인란을 선택한 다음 '수정...'을 클릭하여 익명 액세스에 사용되는 계정을 변경합니다. 팝업 창에서 사용자 이름으로 IUSR_URCHIN을 선택합니다. 확인을 차례로 두 번 클릭하여 속성 창으로 돌아갑니다.

  11. 확인을 클릭하여 변경사항을 저장하고 속성 창을 종료합니다.

5단계: 디렉토리 권한 설정 1. 시작을 마우스 오른쪽 버튼으로 클릭한 후 탐색을 선택합니다.

  1. Urchin이 설치된 위치(보통 C:\Program Files\Urchin)를 찾습니다.

  2. Urchin 폴더를 마우스 오른쪽 버튼으로 클릭한 후 속성을 선택합니다.

  3. 보안 탭을 클릭합니다.

  4. 고급 버튼을 클릭합니다. 상위 개체에서 권한 상속 허용을 선택 해제한 다음 팝업 창에서 삭제를 클릭합니다.

  5. 추가를 클릭하고 관리 사용자를 선택한 다음 추가를 다시 클릭합니다.

  6. 추가를 클릭하고 IUSR_URCHIN을 선택한 다음 추가를 다시 클릭합니다.

  7. 이름: 입력란에 관리자 및 IUSR_URCHIN 항목만 표시되어야 합니다.

  8. <urchin_home>\var 폴더에 쓰기 권한을 부여합니다.

  9. 환경설정=>보안으로 돌아가 IUSR_URCHIN 사용자에게 다음 권한이 허용되었는지 확인합니다.

    • 읽기 및 실행
    • 폴더 내용 보기
    • 읽기
    • 참고: 시스템에 기본으로 설정된 모든 권한은 그대로 두십시오.
  10. 확인을 클릭하여 권한을 저장합니다.

  11. Windows 탐색기에서 Urchin 폴더를 클릭합니다.

  12. data 폴더를 마우스 오른쪽 버튼으로 클릭한 후 속성을 선택합니다.

  13. 보안 탭을 클릭합니다.

  14. '고급'을 클릭합니다. 상위 개체에서 권한 상속 허용을 선택 해제한 다음 팝업 창에서 삭제를 클릭합니다.

  15. 추가를 클릭하고 관리 사용자를 선택한 다음 추가를 다시 클릭합니다.

  16. 추가를 클릭하고 IUSR_URCHIN을 선택한 다음 추가를 다시 클릭합니다.

  17. 이름: 입력란에 관리자 및 IUSR_URCHIN 항목만 표시되어야 합니다.

  18. 환경설정=>보안으로 돌아가 관리자 및 IUSR_URCHIN 사용자에게 다음 권한이 허용되었는지 확인합니다.

  19. 전체 제어
    • 수정
    • 읽기 및 실행
    • 폴더 내용 보기
    • 읽기
    • 쓰기

  20. 확인을 클릭합니다.

6단계: IIS 6 전용 웹서비스 확장 관리자로 이동합니다.

  1. 새 웹서비스 확장자 추가..를 클릭합니다.

  2. 확장자 이름:에 Urchin CGI를 입력합니다.

  3. '추가'를 클릭합니다.

  4. report.cgi 및 session.cgi 파일을 찾아 선택합니다(기본적으로 *.dll 파일만 표시되므로 모든 파일이 표시되도록 필터를 변경해야 함).

  5. 기본 위치: C:\Program Files\Urchin\htdocs

  6. 확장자 상태를 허용으로 설정을 선택합니다.

  7. 확인을 클릭합니다.

  8. 호스트 이름 서버의 기본 IIS 항목으로 이동합니다.

  9. 마우스 오른쪽 버튼을 클릭한 다음 속성을 선택합니다.

  10. 마임 유형을 클릭합니다.

  11. 새 유형을 클릭합니다.

  12. 확장자: 입력란에 .cgi를 입력합니다.

  13. 마임 유형 입력란에 application/octet-stream을 입력합니다.

  14. 확인을 클릭합니다.

  15. 확인을 클릭합니다.

이제 http://my.server.com:port URL을 사용하여 IIS 웹서버에 연결할 경우 Urchin 웹 인터페이스를 호출할 수 있도록 설정되었습니다. 여기에서 port는 설치할 때 Urchin에서 사용하도록 설정한 포트입니다(기본적으로 9999).

Urchin 데이터 API 웹 서비스에 기존 IIS 웹서버 사용하기(Urchin 6.6 이상)

Urchin 데이터 API 웹 서비스(Urchin 6.6 이상)는 IIS 모듈과 함께 제공되며 Urchin 데이터 API의 요청을 처리하도록 배포 및 구성할 수 있습니다. IIS 웹서버에서 데이터 API를 구성하려면 IIS와 Urchin 데이터 API를 통합할 수 있습니까?를 참조하시기 바랍니다.
