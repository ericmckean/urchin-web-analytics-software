이전 로그 처리

개요

Urchin을 설치한 후에 이전 로그를 처리할 수도 있습니다.방법은 간단합니다.로그 관리자의 '로그 설정' 화면에서 디렉토리와 파일 이름의 일부 및/또는 와일드카드(정규식 포함)를 지정하기만 하면 됩니다.참고로,원격 HTTP 및 HTTPS 로그 소스에는 와일드카드를 사용할 수 없습니다.

이전 로그 처리 방법

먼저 시스템에 로그 소스를 추가합니다.상단 메뉴에서 '구성 ' 버튼을 클릭하고 'Urchin 프로필' -> '로그 관리자' 링크를 클릭합니다.기본 화면에서 오른쪽 상단에 있는 '추가' 버튼을 클릭합니다.첫 번째 화면에서 '로컬 로그 소스 추가'를 선택하고 계속 진행합니다.다음 화면에서 '찾기'를 클릭하면 파일 탐색기가 나타납니다.왼쪽 창에서 올바른 디렉토리를 찾습니다.오른쪽 창에는 디렉토리에 있는 파일이 표시되고, 왼쪽에는 현재 열려 있는 디렉토리의 다른 모든 하위 디렉토리가 표시됩니다.올바른 디렉토리 안에서 파일 이름의 일부와 별표(또는 기타 정규식)을 입력하고 '확인' 버튼을 클릭합니다.패턴과 일치하는 모든 항목을 보여주는 창이 열립니다.파일 이름을 클릭하면 해당 파일에 대한 정보(위치, 크기, 마지막 수정 날짜, 파일 권한)를 볼 수 있습니다.패턴 일치가 정확하면 '확인'을 클릭하고 파일 탐색기 창에서 '확인'을 다시 클릭합니다.그런 다음에 로그 소스 이름을 지정하고 '다음'을 클릭합니다.드롭다운 목록에서 형식을 지정하고 '마침' 버튼을 클릭합니다.유효한 로그 소스 이름(고유해야 함)의 경우 '완료' 버튼을 누르면 로그 관리자 목록에 로그 소스가 나타납니다.

그런 다음 아직 이 로그 파일을 프로필과 연결하지 않은 경우 상단 메뉴에서 '구성' 버튼을 클릭하고 'Urchin 프로필' -> '프로필' 버튼을 클릭하여 연결합니다.연결이 완료되면 기본 프로필 목록에서 프로필 옆에 있는 '실행/예약' 버튼을 클릭합니다.프로필의 실행을 예약하거나, '지금 실행' 버튼을 클릭하여 바로 처리할 수 있습니다.

Urchin에는 데이터 중복을 방지하기 위한 로그 로테이션이 필요하지 않습니다. 새로운 조회만 처리되도록 하는 로그 추적 기능이 Urchin에 구현되어 있기 때문입니다.그러나 앞에서 언급한 대로 로그는 많은 양의 디스크 공간을 빠르게 소비할 수 있습니다. 따라서 로그 파일을 주기적으로 압축하고 보관하는 것이 좋습니다.Urchin은 처리된 로그 파일을 다시 읽어야 할 필요가 전혀 없기 때문에 각각의 처리 실행 후에 로그를 삭제해도 아무런 문제가 발생하지 않습니다.하지만 많은 사람들은 몇 가지 이유로 인해 필요할 때를 대비하여 일정 기간 동안 로그를 보관합니다. 해당 사이트에 대해 새로운 프로필이 생성되는 경우나 이전 분석이 필요한 경우 등이 그러한 이유에 해당합니다.

권장사항

로그 관리가 처음부터 반드시 필요한 것은 아니지만 로그가 커질수록 중요해집니다.따라서 Urchin을 처음 배포할 때부터 로그 관리 계획을 세우는 것이 좋습니다.
