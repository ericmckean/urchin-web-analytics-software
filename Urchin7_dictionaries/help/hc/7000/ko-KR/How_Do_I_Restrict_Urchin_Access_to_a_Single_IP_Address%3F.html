단일 IP 주소로 Urchin 액세스를 제한하려면 어떻게 해야 합니까?

개요

여러 개의 네트워크 인터페이스가 있거나 여러 개의 IP 주소가 단일 네트워크 인터페이스에 연결된 시스템에서 Urchin 웹 인터페이스는 기본적으로 구성된 모든 IP 주소에 연결되어 있습니다.그러나 관리 및 보고를 단일 IP 주소로 제한하도록 Urchin을 구성하는 것이 바람직할 수도 있습니다.

처리 절차

Urchin 관리/보고를 제한하려면 Urchin Apache 웹서버에서 사용하는 구성 파일을 간단히 변경하기만 하면 됩니다.다음 단계를 따르시기 바랍니다.

UNIX 계열 시스템:

  • Urchin을 실행하는 사용자로서 명령 쉘을 엽니다.
  • 'cd' 명령을 통해 Urchin이 설치된 디렉토리로 이동합니다.
  • 'bin/urchinctl -w stop' 명령을 통해 Urchin 웹서버를 중지합니다.
  • 텍스트 편집기를 사용하여 Urchin 폴더의 'var' 디렉토리에서 'urchinwebd.conf.template' 파일을 엽니다.

  • <Directory /> 행 위에 다음 행을 삽입합니다. 

       Listen XXX.XXX.XXX.XXX:port
  • 여기에서 'XXX.XXX.XXX.XXX' 및 'port'는 Urchin을 제한할 IP 주소 및 포트 번호입니다.
  • 다음 명령을 통해 Urchin 웹서버를 다시 시작합니다. 

bin/urchinctl -w start

Windows 시스템:

  • Urchin 서비스를중지합니다('시작'->'프로그램'->'Urchin 6'->'Urchin 서비스 사용 안 함').

  • Windows Explorer를 열고 'C:\Program Files\Urchin6\var'로 이동합니다.
  • 텍스트 편집기를 사용하여 'urchinwebd.conf.template' 파일을 엽니다.

  • <Directory /> 행 위에 다음 행을 삽입합니다. 

       Listen XXX.XXX.XXX.XXX:port
  • 여기에서 'XXX.XXX.XXX.XXX' 및 'port'는 Urchin을 제한할 IP 주소 및 포트 번호입니다.

  • Urchin 서비스를 다시시작합니다('시작'->'프로그램'->'Urchin 6'->'Urchin 서비스 사용').

중요사항:IIS 5 이상에서는 소켓 풀링을 사용하기 때문에 실제로 어떤 IP 주소가 구성되었건 관계없이 특정 포트에 바인딩된 모든 웹사이트에 대해 소켓을 공유하는 IIS의 기본 동작을 비활성화해야 할 수도 있습니다.이는 가상 호스트가 사용되는 환경에서는 일반적이며, 특정 가상 호스트에 대해서는 Urchin이 포트 80에 보고하도록 설정할 수도 있습니다.

주의사항

  • 이 기술은 Urchin 관리/보고를 특정 IP 주소로 제한하는 데만 사용됩니다.이름 기반 가상 호스트에 대한 액세스는 제한할 수 없습니다.
  • Urchin으로 업그레이드할 경우에는 백업 복사본을 저장한 후에 'urchinwebd.conf.template' 파일을 덮어씁니다.업그레이드 후에 해당 파일에서 수정한 내용을 다시 적용해야 합니다.
