계정 작업

개요

계정은 관련 프로필, 로그 소스, 사용자 및 그룹을 하나의 식별 라벨로 그룹화하는 데 사용되는 높은 수준의 연관이며 일반적으로 기업 또는 클라이언트 이름입니다.여러 복잡한 클라이언트 조직을 지원해야 하는 Urchin 설치의 경우, 계정을 만들면 Urchin 관리자가 특정 클라이언트나 법인에 대해 모든 Urchin 보고 구성 요소를 간편하게 추적할 수 있습니다.Urchin 보고서에 대한 액세스 권한은 계정 연관을 통해 제어할 수 있으며 계정 내에서 특정 보고서에 대한 보다 세분화된 액세스 권한을 그룹이나 사용자에게 할당하여 데이터를 필요에 따라 다양한 수준으로 보호할 수 있습니다.마찬가지로 계정 수준 관리 권한을 사용자에게 할당하여 사용자가 해당 계정에서 로컬 Urchin 관리자로 역할을 수행하도록 할 수 있습니다.이를 통해 계정에서 Urchin 보고서의 관리, 구성 및 유지 책임을 분산시킬 수 있습니다.

계정은 Urchin 관리 인터페이스에서 가장 높은 수준의 구성 '요소'이므로 계정과 연관시킬 프로필, 로그 소스, 사용자 또는 그룹을 만들기 전에 계정을 먼저 만들어야 합니다.Urchin 요소가 만들어진 후에는 계정을 선택해야 하며 선택한 계정은 나중에 변경할 수 없습니다.생성 시 특정 계정을 선택하지 않으면 기본 계정(NONE)이 설정됩니다.이 값은 Urchin에 해당 요소에 계정이 없음을 알려줍니다.

계정 만들기

계정을 만들려면 '구성'->'사용자 및 그룹'->'계정' 화면으로 이동하고 '추가' 버튼을 클릭합니다.'계정 이름'만 필요하며 '연락처' 및 '이메일'은 Urchin에서 사용하지 않는 관리자 편의를 위한 정보 제공 입력란입니다.'보고서 데이터 위치'(선택사항)는 계정에 속하는 모든 프로필의 보고서 데이터 저장 위치를 지정합니다.기본 위치는 Urchin 폴더 내의 data/reports 디렉토리입니다.보고서 데이터 위치를 변경하면 서로 다른 조직에 대한 보고서 데이터를 파일 시스템 내에서 물리적으로 분리할 수 있습니다.'디렉토리 찾기 위치'(선택사항)는 프로필 생성을 위해 계정 관리자의 액세스를 허용하는 경우 보안 조치로서 제공됩니다.이 입력란에 입력된 디렉토리는 계정 관리자가 해당 디렉토리에서만 로그 파일을 찾도록 권한을 제한합니다.기본적으로 계정 관리자가 로그 파일을 찾을 수 있는 위치에는 제한이 없습니다.

계정 사용

계정에 프로필, 로그 소스, 사용자 또는 그룹을 할당하려면 지정된 요소를 만들 때 설치 마법사의 초기 화면에서 '옵션 계정' 드롭다운 메뉴를 사용합니다.프로필, 로그 소스, 사용자 등에 계정을 할당한 후에는 Urchin 관리 인터페이스에서 이 계정과 연관된 요소에 대한 수정을 제한합니다.이 방법으로 계정에서 높은 수준으로 액세스 권한을 제어할 수 있으므로 모든 보고서에서 특정 액세스 권한을 설정하지 않고도 조직을 서로 격리시킬 수 있습니다.

계정은 또한 Urchin 구성에 대한 관리 책임을 분산시키는 데도 도움이 됩니다.계정에서 기본 Urchin 관리자는 로컬 관리자 권한을 계정 사용자에게 할당할 수 있습니다.애드워즈에는 사용자에게 할당할 수 있는 4가지 제어 관리 수준이 제공됩니다.사용자 및 그룹 작업을 참조하십시오.

Urchin 구성 매개변수에 대한 관리 화면을 보는 경우 테이블 상단 표시줄에 있는 계정 드롭다운 메뉴를 사용하여 특정 계정 이름의 항목만 선택적으로 표시하도록 필터링할 수 있습니다.(NONE)은 계정 연관이 없음을 의미하지만 필터링을 위해 계정과 연관되지 않은 항목만 확인할 수 있도록 (NONE)이 계정 드롭다운 메뉴에 옵션으로 표시됩니다.
