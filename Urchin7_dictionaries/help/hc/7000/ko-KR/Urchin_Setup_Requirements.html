Urchin 6 설치 요구 사항

  • '이 문서는 Urchin 소프트웨어 공인 컨설턴트인 EpikOne이(가) 작성했습니다.

이 문서에서는 Urchin을 설치하고 실행하기 전에 해결되어야 할 운영상의 문제점을 설명하며, 올바른 Urchin 운영 및 설치 후 Urchin의 효과적인 사용을 위해 필요한 정보를 제공합니다.

Urchin 설치 시 기본 고려사항

  • Windows에서는 관리자로 로그인하여 설치해야 합니다.
  • UNIX 계열 시스템에서는 일반 사용자 권한으로 설치할 수 있지만, 수퍼 사용자 권한이 없으면 파일 시스템 내 일부 영역에서만 설치할 수 있습니다.
  • Urchin은 구성 및 보고서 전송에 사용할 수 있도록 Apache 웹서버 실행 프로그램이 번들로 제공됩니다. 시스템 관리자는 Urchin을 설치하면 새로운 웹 서비스가 실행된다는 것을 알고 있어야 합니다.
  • 최소 수백 메가바이트 이상의 여유 공간이 충분한 디스크에 Urchin 6을 설치해야 향후 Urchin 데이터베이스가 증가해도 문제가 없습니다. 자세한 내용은 성능 및 관리 문제 섹션을 참조하시기 바랍니다.

  • Urchin 5에서 업그레이드할 경우 uconf2sql 및 convert-u5data 유틸리티를 사용하여 기존 데이터베이스를 Urchin 6으로 가져와야 합니다. Urchin 6 설치 프로그램을 실행하여 Urchin 5에서 Urchin 6으로 바로 업그레이드할 수는 없습니다. 자세한 내용은 문서 센터의 시작하기 섹션에서 업그레이드를 참조하시기 바랍니다.

  • Urchin 5에서 Urchin 6으로 업그레이드하려면 제품 라이센스를 다시 발급받아야 합니다.

Urchin 실행 관련 기본 고려사항

  • 웹서버 로그에 대한 액세스 - 해당 사이트에 대한 로그 파일 경로를 알아야 하며 로그 파일에 액세스할 수 있는 권한이 있어야 합니다. 원격 시스템에 있는 로그(FTP/HTTP/HTTPS/UNC)를 조회할 때에는 사용할 계정 이름과 비밀번호가 있어야 합니다.
  • 적절한 로그 형식으로 구성 - Urchin은 사용자설정 로그 형식을 처리할 수 있지만 표준 형식의 로그에 맞게 웹서버를 구성하면 관리 요구사항을 간소화할 수 있습니다. 확장 결합 로그 형식(예: NCSA 또는 Apache 로그)이나 W3C 확장 로그 형식(예: IIS 로그)을 사용할 것을 권장합니다. IIS 사이트에서는 프로세스 통계에 대한 로깅을 중지해야 합니다. 자세한 내용은 고급 구성 섹션을 참조하시기 바랍니다.
  • Urchin 실행을 위한 전용 사용자 계정 – UNIX 계열 시스템에서는 보안을 위해 제한된 권한을 갖는 Urchin 전용 사용자 ID로 Urchin 프로그램을 실행하는 것이 좋습니다. 이러한 전용 사용자 계정을 설정하려면 해당 시스템에 대해 관리자 또는 수퍼 사용자 권한이 있어야 합니다.
  • 예약 – 데이터가 많아 처리 시간이 오래 걸리거나 Urchin에서 원하는 시간에 보고서가 제공되도록 하려면 예약을 실행하도록 선택해야 합니다.

Urchin 실행 관련 세부 고려사항

  • 순 방문자 추적을 설정하려면 다음 기본 단계를 수행해야 합니다.
    • 사이트 웹페이지에 UTM 센서 코드를 설치합니다.
    • 웹서버 로그 형식에서 쿠키 로깅을 활성화합니다.
    • 웹사이트의 Urchin 프로필에서 추적 방법을 UTM으로 설정합니다.
  • 순 방문자 추적을 사용하지 않으려면 원하는 방문자 또는 세션 보고 수준을 정하고 각 사이트에 대한 방문자 추적 방법을 적절하게 선택합니다. Urchin 6에서는 다른 추적 방법으로 IP/User-Agent를 제공합니다.

성능 및 관리 문제

  • 로그 순환 - Urchin에서 처리가 끝난 웹서버 로그를 보관 또는 삭제하는 데 외부 솔루션을 사용하지 않으려는 경우, 각 로그 소스의 고급 설정을 통해 이 작업을 수행하도록 Urchin을 구성할 수 있습니다.
  • 이전 Urchin 데이터베이스 보관 – 특정 월에 데이터베이스가 작성되면 그 이후에도 기록을 분석하는 데 사용할 수 있습니다. 사용자가 데이터 보관 기간을 선택하면 그 전의 불필요한 데이터는 삭제하여 디스크 공간을 절감할 수 있습니다. 특정 날짜 이전의 데이터베이스는 압축하도록 Urchin을 구성할 수 있습니다.
  • 메모리 요구사항 - Urchin에서는 로그를 처리할 때 사용할 수 있는 메모리 양을 정할 수 있습니다. 기본값은 128MB이지만 로그 파일 크기가 10MB 이상인 사이트에서는 값을 늘리는 것이 좋습니다. 시스템 메모리를 충분하게 보유해야 필요에 따라 Urchin 메모리 양을 늘릴 수 있으며, 소프트웨어의 메모리 설정을 조정하여 최적의 처리 성능을 제공할 수 있습니다.
  • Urchin 데이터 저장 위치 - etc/urchin.conf 파일을 사용하여 Urchin 폴더 외부의 파일 시스템 영역에 보고서 데이터베이스가 저장되도록 Urchin을 구성할 수 있습니다. 이렇게 하면 데이터베이스 증가에 대비한 전용 파일 시스템 공간을 편리한 위치에 넉넉하게 할당할 수 있습니다.

원격 액세스 및 통합 문제

  • Urchin 관리 및 보고를 위해 SSL 사용 – Urchin과 함께 제공되는 Apache 웹서버는 SSL을 지원하도록 컴파일됩니다. SSL이 기본으로 활성화되어 있지는 않지만 사용자가 원할 경우 설정하여 사용할 수 있습니다.
  • 방화벽 구성 – 네트워크 토폴로지에 방화벽이나 프록시 서버 등이 포함되어 Urchin 운영 서버와 검색해야 할 보고서 또는 로그가 보관된 서버를 조회하려는 사용자 사이를 차단할 경우, 해당 보안 장치가 Urchin의 원격 액세스를 방해하지 않도록 구성해야 합니다. 일반적으로 이와 같이 구성해도 해당 장치들이 제공하는 보안 기능은 계속 유지할 수 있습니다.
