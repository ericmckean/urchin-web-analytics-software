UNC 로그 소스 작업

Urchin에서 UNC 로그 소스가 생성되면 Server Name/IP, Remote Server Login, Local Server Login, Remote Log Source File 등의 매개변수가 지정됩니다.

Urchin이 UNC 소스에서 로그를 가져오는 흐름은 다음과 같습니다. - Urchin이 "Remote Log Source File"에 지정된 파일을 탐색합니다. - 탐색에 실패하면 Urchin이 "Server Name/IP" 입력란에 지정된 UNC 공유에 대한 연결을 만듭니다. - 연결에 실패하면 Urchin이 로컬 시스템 계정과 같은 네트워크 액세스 권한이 없는 사용자 계정으로 실행되고 있다고 가정합니다. 이 경우 "Local Server Login"에 지정된 사용자 계정에 대한 로그온 세션을 만들고 스스로를 이 계정으로 가장합니다. 즉, Urchin 프로세스가 이 계정으로 실행되기 시작합니다. "Same Local 및 Remote Login" 옵션이 "예"로 설정되어 있으면 가장 인증 정보를 "Remote Server Login"에서 가져옵니다. - 가장에 성공하면 Urchin이 네트워크 공유에 대한 UNC 연결을 만들려고 다시 시도하고 탐색을 다시 시도합니다. - Urchin이 로그 소스 작업을 마치면 UNC 연결을 닫고 가장을 종료합니다. 즉, Urchin 프로세스가 가장하기 이전에 실행되었던 계정으로 실행되기 시작합니다.

가장에는 다음과 같은 단점이 있습니다. - 가장하는 동안 Urchin 프로세스가 다른 계정으로 실행되기 때문에 Urchin 데이터 디렉토리에 액세스하지 못해 보고 데이터베이스를 만들지 못할 수 있습니다. - 가장하는 동안에 생성된 보고 데이터베이스가 가장이 종료된 후 Urchin 프로세스에 액세스하지 못할 수 있습니다. - 속도가 느려집니다. 특히 여러 개의 UNC 로그 파일이 프로필에 연결되어 있을 경우에 그렇습니다. Urchin이 각 파일에 대한 가장을 시작하고 종료해야 하기 때문입니다.

따라서 Urchin에서 UNC 공유를 사용하기에 가장 좋은 방법은 다음과 같습니다. - 네트워크에 액세스할 수 있는 권한을 가진 계정으로 Urchin을 실행합니다. 그러면 가장할 필요가 없습니다. - Urchin을 실행하고 있는 사용자 계정에 UNC 로그 공유에 대한 읽기 권한을 부여합니다. 그러면 UNC 연결을 만들고 가장하는 과정이 필요하지 않게 됩니다.
