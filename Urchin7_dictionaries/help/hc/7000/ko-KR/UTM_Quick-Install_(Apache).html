UTM 빠른 설치(아파치)

이 문서는 Urchin 소프트웨어 공인 컨설턴트인 Panalysis이(가) 작성했습니다.

다음은 모든 플랫폼에서 아파치를 기반으로 운영 중인 웹사이트에 UTM을 설치하는 단계를 알려주는 요약 설명입니다.

1단계: UTM 파일을 웹사이트 문서 루트에 복사합니다. urchin.js 및 __utm.gif 파일은 Urchin 폴더의 'util/utm' 디렉토리에 있습니다. 이 두 파일을 웹사이트 콘텐츠의 기본 디렉토리에 복사합니다.

2단계: HTML에서 UTM을 참조합니다. 모든 HTML 페이지에 다음 행을 입력합니다. 페이지의 어느 위치든 상관 없지만 <head> 섹션에 입력할 것을 권장합니다. 일반 포함(common include) 또는 템플릿을 사용하면 해당 섹션에 입력할 수 있습니다. 중요사항: GIF 파일 이름은 두 개의 밑줄로 시작합니다. 

<script src="/urchin.js" type="text/javascript"></script>
<script type="text/javascript">
        urchinTracker();
</script>

'HTML Tidy'와 같은 패키지를 사용한다면 다음과 같은 자바스크립트 행을 해당 페이지의 HEAD 섹션에 포함시킬 수도 있습니다. 

<html>
<head>
  <meta http-equiv="Content-Type" content="text/html; charset=ISO-8859-1">
  <script src="/urchin.js" type="text/javascript"></script>
  <script type="text/javascript">
        urchinTracker();
  </script>
  ...
</head>

3단계: 아파치 로깅에서 쿠키를 활성화합니다. 아직 활성되지 않았다면 다음 httpd.conf 예를 이용하여 쿠키 로깅을 활성화할 수 있습니다. 

LogFormat "%h %v %u %t \"%r\" %>s %b \"%{Referer}i\"\"%{User-Agent}i\"\"%{Cookie}i\"" special
...
<VirtualHost 63.212.171.6>
ServerName www.mysite.com
DocumentRoot /www/www.mysite.com
CustomLog /var/logs/mysite.log special
</VirtualHost>

4단계: UTM을 사용하도록 Urchin 프로필을 설정합니다. Urchin 관리 인터페이스에서 해당 프로필을 수정하고 보고 탭을 클릭합니다. 방문자 추적 방법을 UTM으로 설정합니다. UTM 도메인을 www를 제외한 웹사이트 주소로 설정합니다. 설정이 끝났으면 업데이트 버튼을 클릭합니다. 프로필 설정 탭을 클릭하고 ‘기본 보고서 설정에 대해 전체 UTM 활성화’를 선택한 다음, 업데이트를 다시 클릭하면 Urchin 프로필 설정 단계가 완료됩니다. 이제 웹사이트에서 일반 로그 파일에 UTM 데이터를 기록하기 시작하며, 이 데이터는 다음 번 Urchin을 실행할 때 인식됩니다.

기록되고 있습니까? UTM에서 로그 파일 기록이 이루어지고 있는지 확인하려면, UTM을 설치하고 사이트의 페이지 몇 개를 클릭한 다음 로그 파일을 검토해 보면 됩니다. 로그 파일 맨 밑에 다음과 유사한 항목이 있어야 합니다. 

... "GET /__utm.gif?..." 200 ..."__utma=..."

__utma 항목이 없으면 로깅에서 쿠키가 올바르게 활성화되어 있는지 확인해 보십시오. 상태 코드가 200이 아니면 문서 루트에 파일이 제대로 복사되었는지 확인해 보십시오.
