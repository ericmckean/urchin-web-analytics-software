로그 소스 작업

개요

일반적으로 프로필을 만드는 중에 로그 소스를 추가하게 됩니다.로그 소스는 웹사이트에 대한 액세스 로그(때로는 전송 로그라고도 함)의 특성을 식별하는 Urchin의 방법입니다.액세스 로그에는 웹사이트에서 수행되는 모든 조회 또는 웹 문서 요청이 포함됩니다.로그 소스와 연결된 로그 파일 특성으로는 로그 파일의 경로, 로그 파일의 형식(예: W3C 또는 NCSA), 로컬 시스템과 원격 시스템 중에서 어느 시스템에 로그가 있는지 여부, 처리 도중에 로그 파일에 필터를 적용해야 하는지 여부 등이 있습니다.

반드시 이해해야 할 중요한 개념은 로그 소스가 프로필과는 독립적으로 존재한다는 것입니다.모든 프로필은 보고를 얻기 위해 관련된 로그 소스를 하나 이상 포함해야 합니다.그러나 여러 개의 프로필이 같은 로그 소스를 사용하는 경우도 있을 수 있습니다.예를 들어 같은 로그 소스를 사용하는 프로필을 여러 개 만들지만 각 프로필에 서로 다른 필터를 제공하여 다양한 보고서 결과를 얻을 수도 있습니다.따라서 로그 소스와 프로필 간의 비율이 반드시 1:1인 것은 아닙니다.

로그 소스 구성

시스템에 로그 소스를 추가하려면 먼저 Urchin 관리 시스템에 관리자로 로그인하고 왼쪽에 있는 '구성' 버튼을 클릭합니다.그런 다음 '고객 관리자' 버튼을 클릭합니다.새 로그 소스를 만들려면 화면 오른쪽 상단에 있는 '추가' 버튼을 클릭합니다.로그 소스 추가 마법사가 나타납니다.마법사가 제공하는 일련의 간단한 절차를 통해 로그 소스를 쉽고 빠르게 설정할 수 있습니다.마법사의 각 화면에는 해당 화면에 표시되는 구성 정보를 설명해 주는 명확한 도움말 정보가 있습니다.

'로그 설정' 화면에서 로그 형식을 선택해야 합니다.이 설정은 로그 파일에 있는 데이터가 어떻게 정렬되는지를 Urchin에 알려줍니다.로그에 대한 올바른 형식을 선택해야 합니다. 그렇지 않으면 Urchin에서 의미 있는 보고서 데이터를 생성할 수 없습니다. Urchin은 드롭다운 메뉴를 통해 선택할 수 있는 로그 형식의 기본 집합을 이해합니다.선택 가능한 로그 형식은 다음과 같습니다.

  • 자동:Urchin은 이 형식을 사용하여 NCSA, W3C, Netscape 및 ELF2 로그 형식을 자동으로 감지합니다.이러한 형식 중에 하나를 명시적으로 선택하는 대신 '자동'을 선택할 수 있습니다. 그러면 이 목록에 있는 로그 형식일 경우 Urchin이 데이터를 읽는 방법을 정확히 추론할 수 있습니다.
  • NCSA: Apache에서 수정된 확장/결합 형식입니다(이 형식에 대한 설명은 '로깅 - Apache 및 IIS' 참조).
  • W3C: Microsoft IIS 서버가 주로 이 형식을 사용합니다. 하지만 다른 서버도 W3C 로그를 생성하도록 구성할 수 있습니다.
  • Netscape: Netscape 및 iPlanet 서버는 기본적으로 이 형식을 사용합니다.
  • ELF2: 전자상거래 로그 형식
  • Google:이 형식은 Google의 CPC 지출 데이터를 포함하는 로그에 사용하십시오.Google 로그 형식은 자동 감지될 수 없습니다.
  • Overture:이 형식은 Overture CPC 지출 데이터를 포함하는 로그에 사용하십시오.Overture 로그 형식은 자동 감지될 수 없습니다.
  • 맞춤:처음에는 드롭다운 메뉴에 나타나지 않지만, 자신만의 맞춤 로그 형식을 만들면 해당 형식이 올바르게 구성되었을 때 드롭다운 메뉴에 자동으로 나타납니다.

웹서버가 인식되는 기본 형식 중 하나로 로그를 생성하고 있지 않다고 판단되는 경우에는 기본 형식 중 하나로 로깅하도록 웹서버를 다시 구성하거나, 웹서버의 현재 로깅 방식을 따르는 맞춤 로그 형식을 만들 수 있습니다.웹서버 로깅을 다시 구성하고자 하는 경우에는 W3C 또는 NCSA 스타일의 로깅을 선택해야 합니다.

분산 처리 및 병렬 로그 처리

트래픽이 많은 웹사이트에서는 웹서버를 하나 사용하는 것만으로는 부족할 수도 있습니다.분산 처리를 사용함으로써 병렬로 작동하는 여러 웹서버에 트래픽을 분산시킬 수 있습니다.사용되는 병렬 웹서버 집합을 클러스터라고 부릅니다.이러한 방식으로 각 웹사이트 활동에 대한 로깅이 사용 가능한 병렬 로그에 로깅됩니다. 프로필 설정 마법사는 병렬 로그 처리 옵션을 제공합니다.병렬 로그 처리가 활성화되면 Urchin은 모든 로그 파일을 한 번에 열어 한 번에 한 섹션씩 순서대로 읽어들입니다. 여기서 각 섹션은 15분 동안의 로그 활동에 해당합니다.병렬 로그 처리를 활성화하면 분산 처리된 사이트의 성능이 크게 향상됩니다.
