Urchin 6과 Urchin 5의 차이점은 무엇입니까?

이 문서는 Urchin 5에서 Urchin 6로 넘어오면서 변경된 내용에 대해 설명합니다.

주요 특징

  • 제한 없는 프로필, 로그 소스 및 캠페인 추적이 모두 기본 라이센스에 포함되어 있음(부가기능 모듈이 없음).
  • 포괄적인 SEO/SEM 캠페인 추적 기능, 프로필당 4개의 목표
  • 대부분의 보고서에서 사용할 수 있는 다양한 교차 세그먼트
  • 완벽한 방문자 지리적 위치 보고서 모음(방문자 도메인만 있는 것이 아님)
  • Urchin 5와 처리 속도는 동일하지만 훨씬 더 많은 데이터를 처리함

플랫폼 지원

  • 단 2개의 빌드(Linux 2.4 및 2.6 kernels)로 광범위한 Linux 플랫폼 지원
  • FreeBSD 5 및 FreeBSD 6가 새로 지원됨
  • MacOS X 및 Solaris가 더 이상 지원되지 않음

설치 프로그램

  • 이제 Windows 설치 프로그램이 MSI 패키지로 배포되기 때문에 자동 설치 지원이 향상되고 SMS와 통합됨

구성

  • 관계형 데이터베이스(MySQL 또는 PostgreSQL) 백엔드
  • 원격 구성 서버에서 호스트되는 구성 데이터베이스 지원

웹서버

  • 최신 Apache 1.3.X 릴리즈로 업그레이드
  • OpenSSL 및 mod-ssl이 최신 버전으로 업그레이드됨
  • Urchin에서 사용하지 않는 기본 모듈 삭제
  • 올바른 캐시 제어 헤더를 위해 mod_expires 추가

작업 스케줄러

  • 스케줄러가 이제 2개의 프로세스로 실행됨(마스터 스케줄러 및 슬레이브 스케줄러)
  • 인터페이스를 백엔드 구성 DB로 스크립팅함으로써 작업을 쉽게 관리

방문자 추적

  • Urchin 5 __utm.js 추적 자바스크립트가 Google Analytics 호환 urchin.js 추적 javascript로 대체되었습니다.__utm.js를 사용하여 사이트를 추적하는 경우에는 urchin.js를 사용하도록 사이트 태그를 변경해야 합니다.

로그 처리

  • 지리 데이터가 메모리에 저장됨(보다 큰 런타임 메모리 공간)
  • 로그 소스에 있는 일수 기능을 통해 특정 날짜 패턴과 여러 날짜를 로그 파일에서 검색할 수 있음
  • 메모리에서 프로필을 전체적으로 실행할 수 있음

데이터 및 저장

  • 프로필 데이터베이스의 기본값이 매달 100,000개의 레코드(10,000개에서 늘어남)이며 매달 최대 500,000개의 레코드까지 증가할 수 있음

  • 확장된 Geodata:Quova에서 제공하는 전체 지리적 위치 데이터가 도메인 전용 MaxMind 데이터를 대체함

  • 월별 테이블 레코드 제한이 기본적으로 10,000개에서 100,000개로 늘어남
  • 하위 디렉토리로 구성되는 월별 파일이 프로필당 50개씩 제공됨

보고 UI

  • Adobe SVG를 대신하여 Flash로 그래프와 차트를 렌더링
  • CSV 및 XML로만 보고서 내보내기 가능(안정적이지 못한 MS Word/Excel 내보내기는 사라짐)
  • 모든 프로필 보고서를 없앰

전자상거래

  • 추적 자바스크립트(GA와 동일)에 있는 특수 기능을 통해 전자상거래를 웹서버 로그에 직접 작성할 수 있음
  • ELF2 형식의 외부 장바구니 로그도 지원됨

보안

  • 교차 사이트 스크립팅(XSS) 및 XSRF 취약점에 대한 철저한 테스트
