Urchin 지원 플랫폼 및 하드웨어 요구사양

Urchin은 다양한 아키텍처 및 운영체제에서 실행될 수 있고, Urchin 설치는 로그 처리 시스템에만 필요합니다. 보고서를 볼 때는 웹브라우저만 있으면 됩니다.

지원되는 플랫폼

Windows

  • Windows 2003 Server
  • Windows XP
  • Windows 2000
  • Windows Vista
  • Windows 2008 Server

UNIX-type Systems

  • Linux(kernel 2.6) - 대부분의 버전에서 사용
  • Linux(kernel 2.4) - 대부분의 버전에서 사용
  • FreeBSD 4(6.5+ 버전에서는 지원되지 않음)
  • FreeBSD 5.3+(Urchin 6.602+ 버전에서는 지원되지 않음)
  • FreeBSD 6.2+

Supported databases

  • MySQL 4.1+(Urchin 6.6+ 버전에서는 지원되지 않음)
  • MySQL 5.03+
  • PostgreSQL 8.2.5

지원되는 브라우저

  • Internet Explorer 6.x, 7.x
  • Firefox 2.x, 3.x
  • Chrome 1.x, 2.x

참고: 간접적으로 지원되는 플랫폼

사용 중인 플랫폼에 해당하는 Urchin 배포판이 없는 경우 해당 OS와 호환 가능한 배포판을 사용해 보시기 바랍니다.

  • Windows 98, Windows 3.x:

Windows 98 or 3.x에는 Urchin을 설치할 수 없지만 Internet Explorer(5.x 버전 이상)에서 보고서를 볼 때는 이 두 시스템을 사용할 수 있습니다.

  • Linux:

Linux의 버전은 매우 다양해서 특정 Urchin 배포판에서 지원되는 버전을 모두 명시할 수는 없습니다. 그러나 Linux는 버전 간 호환성이 매우 높기 때문에 Urchin의 배포판 중 하나를 무작위로 설치해도 작동할 가능성이 높습니다. 다음은 호환 가능한 것으로 알려진 배포판입니다.

  • Ubuntu (kernel 2.6)

  • RedHat Enterprise Linux 2 & higher

  • SuSE Linux 8 & higher

  • Trustix 2 & higher

  • Solaris:

Solaris는 현재 지원되지 않습니다.

Urchin 6 시스템 요구사양

Urchin의 뛰어난 성능으로 인해 적은 하드웨어 비용으로 큰 효과를 얻을 수 있습니다. 예를 들어 구형 Pentium II를 데스크톱 컴퓨터로 사용할 경우 속도는 느리지만 Urchin 서버로 사용하기에는 충분합니다. 최고의 성능을 얻으려면 다음 하드웨어 사양을 권장합니다.

소규모 및 중간 규모의 단일 웹사이트 분석

  • 2Ghz 프로세서 이상
  • 2GB RAM
  • 250GB 이상 IDE 하드 디스크
  • 100Mbs 이더넷 인터페이스

서비스 제공업체/엔터프라이즈 설치

  • 2.5Ghz Pentium dual-core/2Ghz UltraSPARC/동급 CPU 속도 PPC/MIPS/
  • 3GB RAM
  • Seagate Cheetah와 같은 Ultra2/Wide SCSI 하드 디스크
  • 100BASE-T 이더넷
  • 백업 시스템

메모리/시스템/디스크 사용량

  • Urchin은 20~500Mb 정도의 메모리(RAM)를 사용하도록 구성할 수 있습니다(Urchin 6.5의 경우 20~1024Mb).
  • Urchin은 낮음/중간/높음과 같은 우선순위 순으로 실행되도록 설정할 수 있습니다.
  • Urchin은 데이터 저장에 원시 로그 공간의 약 10%를 사용합니다.

FreeBSD 사용자를 위한 중요사항

(1) FreeBSD의 하드 코딩된 프로세스 데이터 크기는 최대 500MB로 정해져 있습니다. 로그를 처리하는 과정에서Urchin의 지리 데이터가 메모리에 상주해야 하기 때문에 이 한도가 초과될 수 있습니다. 그 결과 다음과 같은 런타임 오류가 발생합니다. 

ERROR: (8011-323-1057) Failed to allocate memory

이를 해결하려면 /boot/loader.conf 파일에 다음 행을 추가하여 FreeBSD 시스템의 기본 프로세스 데이터 크기를 늘려야 합니다. 

# Increase max process data segment size to 1GB
kern.maxdsiz="1073741824"

수정한 후에는 재부팅해야 합니다.

경고!!! 이와 같이 시스템의 한도를 변경할 때는 주의해야 합니다. kern.maxdsiz를 너무 크게 늘리면 시스템이 부팅되지 않을 수 있습니다. 수정하기 전에 FreeBSD 설명서를 읽고 사이트의 특정 구성에 영향을 미칠 수 있는 잠재적인 위험은 없는지 확인할 것을 적극 권장합니다.

(2) FreeBSD에는 accf_http라는 특수 드라이버가 있습니다. 이 드라이버는 특정 HTTP 요청 내용이 모두 도착할 때까지 수신되는 연결 단위를 버퍼링하여 속도를 높이고 성능을 최적화합니다. 기본적으로 이 드라이버는 커널에 로드되지 않으며 다음과 같은 경고가 표시됩니다. 

[warn] (2)No such file or directory: Failed to enable the
‘httpready’ Accept Filter

이를 방지하려면 HTTP 수락 필터 FreeBSD 커널 모듈(accf_http)을 설정해야 합니다. 그러려면 /boot/loader.conf 파일을 편집하고 다음 행을 추가하면 됩니다. 

# Wait for full HTTP request accept filter
accf_http_load="YES"

수정한 후에는 재부팅해야 합니다.

Linux glibc 라이브러리 버전

Urchin 6.602와 함께 제공되는 PostgreSQL 8.4.01을 Linux(커널 2.4 또는 2.6)에 설치하려면 glibc 2.3.3+가 필요합니다.
