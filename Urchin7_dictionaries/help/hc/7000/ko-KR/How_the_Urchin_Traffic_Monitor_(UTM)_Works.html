UTM(Urchin Traffic Monitor) 작동원리

개요

특허 출원 중인 UTM(Urchin Traffic Monitor)는 원래 Urchin 4에서 제공되던 것으로 웹사이트 순방문자수를 가장 정확하게 측정하기 위해 특별히 설계되었습니다.온라인 방문자 동작을 보다 심도 있게 이해하려는 기업에 있어 UTM은 데이터를 제어할 수 있으면서도 클라이언트 및 서버측의 최적의 정보를 결합하는 매우 유용한 기술입니다.간편하게 설치할 수 있는 이 기술을 통해 기업 소유주는 소유자는 순방문자, 클릭 경로 및 신규 방문자, 재방문자 및 사용 빈도를 포함하는 재방문 충성도 통계를 정확하게 파악할 수 있습니다.최신 버전의 UTM은 향상된 캠페인 추적 기능을 제공합니다.

UTM(Urchin Traffic Monitor) 시스템에는 웹사이트 콘텐츠에 설치된 코드인UTM 센서와 로그 처리 Urchin 엔진의 일부인 UTM 엔진의 두 구성 요소가 포함됩니다.UTM 센서를 사용하여 클라이언트측 데이터를 수집한 후 각 페이지뷰에 대해 Urchin으로 다시 퍼널링할 수 있습니다.클라이언트측 정보는 UTM 엔진에 의해 기존 서버측 데이터와 결합되어 정확하고 완벽한 웹사이트 활동 정보를 제공합니다.

UTM 센서는 작은 크기의 자바스크립트 코드이며 두 가지 중요 기능을 합니다.첫째, 이 센서는 각 페이지뷰에 대해 1회 이상의 조회수를 Google 웹서버로 강제 적용하여 캐싱 효과를 무효화합니다.둘째, UTM 센서는 클라이언트측 '당사' 쿠키를 사용하여 각 방문자를 고유하게 식별하여 신규 방문자와 재방문자를 추적합니다.이 쿠키 식별자는 세션 ID와 동일한 방식의 웹서버에만 표시되는 통신 태그입니다.정보를 시스템 외부로 전송하여 여러 개인정보 보호정책을 위반하는 타사 쿠키가 아닙니다.

UTM 시스템은 다음과 같이 작동합니다.웹서버는 콘텐츠 전달 및 로깅이라는두 가지 기본 기능을 제공합니다.웹사이트의 콘텐츠에는 사용자 브라우저에 전달되는UTM 센서가 포함됩니다.UTM 센서는 고유 식별자를 설정하고 동일한 웹서버에 추가 요청을 다시 전송합니다.Urchin 로그 처리 엔진 중 하나인 UTM 엔진은 이 요청으로부터 데이터를 사용하여 정확하고 완벽한 방문자 활동 정보를 제공합니다.

UTM 센서

UTM 센서는 캐싱 및 프록싱의 효과를 무효화하여 로그 파일 데이터의 정확도와 완전성을 높입니다.예를 들어 사용자가 이전에 방문한 페이지로 다시 이동하는 경우 페이지 콘텐츠는 브라우저의 캐싱된 메모리에서 전달됩니다.서비스 제공업체에 의해 캐싱이 제공되는 경우 동일한 방식이 적용됩니다.웹서버에는 이 페이지뷰에 대한 정보가 없으나UTM 센서는 하나 이상의 작은 레코드를 웹서버로 다시 강제 적용하는 추가 고유 조회수를 활성화합니다.이 정보는 로그 파일에 기록되어 페이지뷰의 발생과 관련한 정보를 확보할 수 있게 됩니다.

UTM 센서의 두 번째 중요한 기능은 세션과 순방문자를 모두 식별하는 것입니다.특허 출원 중인 브라우저 캐시 조합 기술을 통해 센서는 순방문자와 세션 식별자를 감지 및 초기화하여 서비스 제공업체 프록시의 동작에 관계없이 신규 및 재방문자를 정확하게 모니터링할 수 있습니다.대부분의 서비스 제공업체는 IP 주소를 재사용하고 사용자를 방화벽 뒤로 클러스터링하여 프록싱을 활용합니다..이 방식은 사용자 식별자로 IP 주소를 활용하며 일반적인 로그 파일 추적에 문제를 야기할 수 있습니다.

UTM 센서는 IP 주소가 아닌 쿠키 식별자를 활용하여 프록시를 통과할 수 있습니다.예를 들어 신규 순방문자가 단일 IP 주소를 사용하여 방화벽을 통해 웹사이트에 액세스하는 경우, 전달된 페이지뷰에는 방문자 브라우저에 식별자를 설정하는 UTM 센서가 포함됩니다.

동일한 사용자가 재방문하는 경우 요청 시마다 고유 ID가 웹서버로 전달됩니다.사용자에게 두 번째 IP 주소가 할당된 경우에도 UTM 기술은 원래 ID의 방문자를 적절하게 식별합니다.복잡한 프록싱 기법의 효과를 무효화하는 것 외에도 여러 장소를 이동하며 여러 공급업체를 통해 노트북으로 접속하는 방문자도 추적합니다.
