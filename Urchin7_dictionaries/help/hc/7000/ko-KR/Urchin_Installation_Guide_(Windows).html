Urchin 6 설치 가이드(Windows)

이 문서는 Urchin 소프트웨어 공인 컨설턴트인 EpikOne에서 작성했습니다.

시작하기 전에 – Urchin 6.500 사용자를 위한 중요사항

Urchin 6.501에서는 CPC 스케줄러와 관련된 여러 문제가 해결되었습니다. Urchin 6.500에서 Urchin 6.602로 업그레이드하는 경우, 처리된 CPC 데이터가 부정확하거나 불완전하여 Urchin 보고서에 잘못된 결과가 나타날 수 있습니다. Google Adwords에서 정확한 결과를 얻을 수 있도록 (a) 새로운 위치에 Urchin 6.602 버전을 설치하거나 (b) 기존 6.500 버전에 설치하려는 경우 이전 CPC 데이터를 삭제하십시오. 후자의 경우 다음 디렉토리의 모든 내용을 삭제합니다. 

<path_to_urchin_6500_installation>\data\cpc\...
<path_to_urchin_6500_installation>\data\history\cpc\...

Urchin 6.600, 6.501 또는 Urchin 6.402 이하의 버전에서 업그레이드하는 경우에는 별다른 조치를 취하지 않으셔도 됩니다.

설치 단계 개요

  • Urchin 시스템에 원하는 데이터베이스 서버를 설치하거나 기존 서버에 Urchin 데이터베이스를 추가합니다. 현재 지원되는 데이터베이스 엔진은 다음과 같습니다.
    • MySQL 5.0.3 이상(참고: Urchin 6.6부터 MySQL 4.x.x는 더 이상 지원되지 않음)
    • PostgreSQL 8.2.5 이상
  • 데이터베이스 서버에 Urchin 데이터베이스 및 사용자를 만들고 액세스 권한을 부여합니다.
  • 임시 폴더에 Urchin 설치 파일의 압축을 풉니다.
  • MSI 설치 프로그램을 실행합니다.
  • Urchin을 설치할 수 있는 라이센스를 발급받습니다.

옵션 1: Urchin에 기존 MySQL 데이터베이스 사용하기

  • 시스템에 MySQL 서버가 설치되어 있지 않은 경우 http://dev.mysql.com/downloads/ 페이지에서 설치 파일을 다운로드할 수 있습니다. Windows Essentials 패키지에는 Urchin 사용에 필요한 모든 구성요소가 포함되어 있습니다.

  • MySQL 사이트의 지침에 따라 MySQL 서버를 설치합니다.
  • MySQL 데이터베이스 관리자 로그인(예: root) 및 비밀번호를 알고 있어야 합니다.
  • Urchin에 사용할 데이터베이스 이름, 로컬 MySQL 사용자 및 비밀번호를 선택합니다. 이 정보는 Urchin을 설치할 때 필요하므로 저장해 두십시오. 아래의 예에서는 MySQL 사용자 이름과 비밀번호가 각각 urchin 및 urchinpassword인 urchin이라는 데이터베이스를 만듭니다(실제로 사용할 비밀번호는 직접 선택해야 함).

  • 시작 -> 프로그램 -> MySQL -> MySQL 서버 -> MySQL 명령행 클라이언트로 이동하여 MySQL 명령행 클라이언트를 실행합니다.

  • MySQL 명령행 클라이언트 창에서 다음 명령으로 Urchin 데이터베이스와 사용자를 만들고 액세스 권한을 부여합니다. 

mysql> create database urchin character set utf8;
mysql> GRANT ALL ON urchin.* to 'urchin'@'localhost' IDENTIFIED BY
'urchinpassword';

옵션 2: Urchin에 기존 PostgreSQL 데이터베이스 사용하기

  • 시스템에 PostgreSQL 서버가 설치되어 있지 않은 경우 http://www.postgresql.org 페이지에서 배포판을 다운로드할 수 있습니다.

  • PostgreSQL 사이트의 지침에 따라 PostgreSQL 서버를 설치합니다.
  • PostgreSQL 데이터베이스 관리자 로그인(예: postgres) 및 비밀번호를 알고 있어야 합니다.
  • Urchin에 사용할 데이터베이스 이름, 로컬 PostgreSQL 사용자 및 비밀번호를 선택합니다. 이 정보는 Urchin을 설치할 때 필요하므로 저장해 두십시오. 아래의 예에서는 PostgreSQL 사용자 이름과 비밀번호가 각각 urchin 및 urchinpass인 urchin이라는 데이터베이스를 만듭니다(실제로 사용할 비밀번호는 직접 선택해야 함).

  • 시작 -> 프로그램 -> PostgreSQL 8.2 -> psql to 'postgres'를 실행하여 PostgreSQL 명령행 클라이언트를 실행합니다.

  • PostgreSQL 명령행 클라이언트 창에서 다음 명령으로 Urchin 데이터베이스와 사용자를 만들고 액세스 권한을 부여합니다. 

postgres=# CREATE DATABASE urchin WITH ENCODING 'UTF8';
postgres=# CREATE USER urchin WITH PASSWORD 'urchinpass';
postgres=# GRANT ALL ON DATABASE urchin TO urchin;
postgres=# \connect urchin;
postgres=# CREATE LANGUAGE 'PLPGSQL';

옵션 3: 함께 제공되는 PostgreSQL 서버를 Urchin에서 사용하기

  • Urchin 6.602 설치 프로그램에서는 Urchin 설치 도중 PostgreSQL 서버가 옵션으로 함께 제공됩니다.

  • 이 옵션을 사용하려면 Urchin 설치 프로그램이 관리자 권한이 있는 사용자에 의해 호출되어야 합니다. 설치 프로그램에서 포트 번호와 사용자 "postgres"에 대한 비밀번호를 제공하라는 메시지가 나타납니다. 참고: 사용자 "postgres"가 시스템에 이미 있는 경우에는 기존 비밀번호를 제공해야 합니다. 사용자 "postgres"가 시스템에 없는 경우에는 제공된 비밀번호로 생성됩니다.

  • 함께 제공되는 PostgreSQL 서버에는 다음과 같은 제한사항이 있습니다.
    • 업그레이드 모드로 설치될 수 없습니다.
    • 설치하려면 관리자 권한이 필요합니다.

Urchin 설치하기

  • 임시 폴더에 Urchin 설치 파일의 ZIP 압축을 풉니다.

  • 함께 제공되는 PostgreSQL 서버를 설치하려면 시스템에 이미 PostgreSQL 서버가 있지 않은지 확인합니다. 설치하려면 "PostgreSQL 설치" 옵션을 선택하고 포트 번호, 사용자 "postgres" 비밀번호 등의 PostgreSQL 서버 세부정보를 제공합니다. 참고: 함께 제공되는 PostgreSQL 서버를 설치하는 데 문제가 있으면 http://www.postgresql.org를 참조하십시오.

  • 기존 데이터베이스를 사용하려면 위에서 선택한 MySQL 또는 PostgreSQL에 대한 Urchin 데이터베이스 이름, 사용자 및 비밀번호가 있는지 확인합니다.
  • Urchin 6에서는 시스템당 하나의 Urchin 6 인스턴스만 허용합니다(Windows만 해당). 시스템에 기존 버전의 Urchin이 설치되어 있는 경우 업그레이드만 허용됩니다(다운그레이드는 허용되지 않음).
  • '새로' 설치를 다시하려면 대상 시스템에서 기존 Urchin 6 버전을 제거합니다. Urchin을 삭제하려면 다음 단계를 따르십시오.

    • 시작 -> 프로그램 -> Urchin 6 -> Urchin 서비스 사용 안 함

    • 시작 -> 설정 -> 제어판 -> 프로그램 추가/제거

    • "Google의 Urchin 소프트웨어"를 클릭하고 삭제를 선택합니다.
    • Urchin이 삭제된 후 urchin 설치 폴더 이름을 다시 지정합니다(예: C:\Program Files\Urchin6 폴더를 Urchin6-OLD 폴더로).
  • Urchin 6.602은 특정 Urchin 6 초기 버전을 업그레이드하는 데만 사용할 수 있습니다. 업그레이드가 가능한 버전은 다음과 같습니다.
    • Urchin 6.6 - 서비스 팩 1 : 6.601 버전
    • Urchin 6.6 - 영어 전용 출시 버전 : 6.600 버전
    • Urchin 6.5 - 서비스 팩 2 : 6.501 버전
    • Urchin 6.5 : 6.500 버전(위의 중요사항 참조)
    • Urchin 6 - 서비스 팩 1 : 6.402 버전
    • Urchin 6 – 전 세계 출시 버전: 6.401 버전
    • Urchin 6 - 영어 전용 출시 버전 : 6.400 버전
  • 업그레이드 시 기존 Urchin 6 버전을 종료하거나 제거할 필요는 없습니다. Urchin 6.602 설치 프로그램이 먼저 Urchin 6을 확인하여 기존 설치를 제거합니다. 그런 다음 Urchin 6.602 파일 및 구성을 설치합니다.
  • Urchin 6.602 설치를 시작하려면 urchin6602_windows_installer.msi 파일을 두 번 클릭한 후 나타나는 안내에 따라 설치를 진행합니다.

  • 참고: Urchin 6 이전 버전을 사용한 경우 이전 구성을 유지하려면 "기존의 Urchin용 데이터베이스 테이블 재생성" 옵션을 선택하지 않습니다. 이 경우 Urchin은 기존 구성을 유지하지만 '추가 테이블'이나 '기존 테이블 항목'을 생성할 수 있습니다.

  • 이제 시스템에서 Urchin을 실행할 수 있습니다. 설정을 완료하려면 다음 URL에서 원하는 웹브라우저를 지정합니다. 

http://yourserver:<port>

여기에서 <port>는 설치 시 선택한 네트워크 포트입니다(기본값은 9999).

  • Urchin 6의 이전 버전으로부터 프로필 보고 데이터를 복원하려면 history 및 reports 폴더를 C:\Program Files\Urchin6-OLD\data에서 C:\Program Files\Urchin6으로 복사합니다.
  • 데모 라이센스 얻기 링크를 클릭하고 라이센스 발급 단계에 따라 Urchin을 활성화하고 초기 설정 단계를 수행합니다.

추적 구성

Urchin이 설치된 후 경우에 따라서는 웹사이트 추적 옵션 설정 단계를 추가로 수행해야 합니다.

1. Urchin을 IP+User-Agent 추적 옵션과 함께 사용하는 경우

추가 구성이 필요하지 않습니다. Urchin에 프로필을 추가할 때 방문자 추적 방법으로 IP+User-Agent를 지정해야 합니다.

2. Urchin을 UTM 추적 옵션과 함께 사용하는 경우

UTM 추적과 함께 Urchin을 사용하려는 경우 간단한 javascript 추적코드를 웹사이트 페이지에 설치해야 합니다. 방법은 다음과 같습니다.

  • Urchin 폴더 내 util/utm 디렉토리에 있는 urchin.js 및 __utm.gif 파일을 웹사이트 문서 루트(예: 콘텐츠 최상위 디렉토리)에 복사합니다.

  • 웹사이트의 각 페이지에서 HEAD 섹션의 META 태그 바로 뒤에 다음 추적코드를 추가합니다. 

<script src="/urchin.js" type="text/javascript">
</script>
<script type="text/javascript">
  _userv=0;
  urchinTracker();
</script>
  • Urchin에 프로필을 추가할 때 방문자 추적 방법으로 UTM(Urchin Traffic Monitor)을 지정해야 합니다.

3. Urchin을 Google Analytics와 함께 사용하는 경우

Google Analytics의 새로운 추적 기능인 "ga.js" 추적코드는 Urchin 6과 호환되지 않습니다. Urchin 소프트웨어와 Google Analytics를 동시에 사용하여 웹사이트를 추적하려면 urchin.js-based 추적코드를 사용하는 것이 가장 좋습니다. ga.js를 사용하기로 결정한 경우에는 새로운 ga.js 추적 기능을 사용할 수 없습니다.

이미 Google Analytics를 사용하여 추적하는 웹사이트에서 Urchin을 사용하려면 기존의 Google Analytics 추적코드를 약간 수정해야 합니다. 방법은 다음과 같습니다.

  • Urchin 폴더 내 util\utm 디렉토리에 있는 __utm.gif 파일만 웹사이트 문서 루트(예: 콘텐츠 최상위 디렉토리)에 복사합니다.

  • 웹사이트 페이지의 기존 추적코드에 _userv=2;를 다음과 같이 추가합니다. 추적코드의 다른 부분은 수정하지 마십시오. 

<script src="http://www.google-analytics.com/urchin.js"
type="text/javascript">
<script type="text/javascript">
  _userv=2;
  _uacct="UA-XXXXXXX-Y";
  urchinTracker();
</script>
  • Urchin에 프로필을 추가할 때 방문자 추적 방법으로 UTM(Urchin Traffic Monitor)을 지정해야 합니다.

Windows Vista 및 Server 2008을 위한 추가 설치 단계

Windows 버전(2000, XP 및 Server 2003) 외에도 Urchin 6.6+는 Windows Vista 및 Server 2008을 지원합니다. 이러한 최신 Windows 버전의 특정 보안 기능이 Urchin 실행에 영향을 줄 수 있습니다. 다음 설명을 참고하십시오.

UAC - 문제

Windows 2008은 User Access Control(UAC)이라고 하는 새로운 보안 기능을 제공합니다. 자세한 내용은 다음 페이지를 참조하시기 바랍니다. http://technet.microsoft.com/en-us/library/cc709691.aspx

UAC 사용 시 자주 발생하는 문제는 다음과 같습니다.

  1. . UAC를 사용 설정하면 응용프로그램에서 "Program files" 폴더에 파일을 저장하는 데 문제가 발생합니다. 자세한 내용은 다음 사이트를 참조하십시오. http://msdn.microsoft.com/en-us/library/aa905330.aspx

  2. . 기본 사용자 권한으로 Urchin의 실행 파일을 명령 프롬프트로 실행할 수 없게 되므로, 관리자 권한으로 명령 프롬프트를 사용하거나 실행 파일의 속성을 다음과 같이 지정해야 합니다. 마우스 오른쪽 버튼으로 속성을 클릭한 후 호환성 -> "관리자 모드로 이 프로그램 실행" -> 확인을 선택합니다. Urchin Windows 서비스는 이러한 변경 작업 없이 원래대로 잘 작동됩니다.

UAC - 해결 방법

  1. UAC를 사용중지한 후 원하는 폴더에 Urchin을 설치하고 실행 파일이나 Windows 서비스를 사용하여 Urchin을 실행합니다.
  2. UAC가 사용 중이면 Program files 폴더에 Urchin을 설치하지 말고 다른 폴더(예: Users 디렉토리)에 설치하십시오. 속성을 수정하여 실행 파일을 관리자 모드로 실행(위 단락의 2번째 항목 설명 참조)하거나 수정 없이 Windows 서비스를 이용합니다.
