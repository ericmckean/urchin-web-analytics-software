Urchin 프로필 월별 데이터베이스의 디스크 저장용량 줄이기

개요

Urchin 보고 데이터는 Urchin 내에 구성된 별도의 월별 프로필 데이터베이스에 저장됩니다.월별 프로필 데이터베이스는 일반적으로 Urchin 폴더의 data/reports 디렉토리에 위치하는데,기본적으로 Urchin에 저장할 수 있는 월별 프로필 데이터베이스의 수에는 제한이 없습니다.대부분의 중소 규모 사이트에서 나오는 데이터는 소용량 공간에 저장할 수 있습니다.Urchin 보고서를 작성할 때는 이미 처리된 원시 웹서버 로그를 사용하지 않기 때문에 웹서버 로그를 보관할 필요가 없습니다.처리된 Urchin 월별 데이터베이스의 크기는 Urchin 데이터베이스에 저장하기 위해 처리된 원시 웹서버 로그의 약 5-10%이며 이는 모든 Urchin 데이터베이스를 영구적으로 유지한다 해도 최소한의 공간만을 차지하는 수준입니다.

하지만 수천 또는 수백 메가바이트 규모의 웹서버 로그를 매일 생성하는 대규모 사이트나 구성된 프로필의 수가 많은 호스팅 서비스 제공업체에 대해서는 Urchin 내에서 지속 데이터 저장에 요구되는 공간을 줄이는 것이 좋습니다.다음 중 한 가지 방법을 이용하십시오.

  1. 로그 처리 후 원시 추적 데이터를 자동 삭제하도록 프로필을 설정합니다.
  2. 이전 데이터를 보관하도록 프로필을 설정합니다.
  3. 보관되는 이전 보고 데이터의 개월 수를 제한합니다.

각 방법에 대한 자세한 내용은 이 문서의 끝부분에 나와 있습니다.

Urchin 데이터베이스 저장에 대한 기술적 개요

Urchin은 각 Urchin 프로필에 대해 YYYYMM이라는 이름의 디렉토리에 매달 저장되는 데이터베이스 파일 집합을 보관합니다.각 디렉토리에는 보고 엔진에 데이터를 제공하는 파일이 최대 50개 들어 있습니다.데이터를 저장할 달 뒤에 디렉토리 및 데이터베이스 파일 이름이 지정됩니다.전체 데이터베이스 목록은 다음과 같습니다.

YYYYMM-uhed --> 데이터베이스 헤더

YYYYMM-usti --> 문자열 색인

YYYYMM-ustd --> 문자열 데이터

YYYYMM-udai --> 집계 테이블 색인

YYYYMM-udXX --> 집계 데이터 테이블(XX는 데이터 맵의 테이블 번호로 바뀜)

YYYYMM-uvii --> 방문자 색인

YYYYMM-uvid --> 방문자 데이터

YYYYMM-used --> 세션 데이터

YYYYMM-upad --> 경로 데이터

YYYYMM-utrd --> 거래 데이터(전자상거래)

YYYYMM-uitd --> 항목 데이터(전자상거래)

YYYYMM-ulti --> 로그 추적 색인

YYYYMM-ultd --> 로그 추적 데이터

YYYYMM-utod --> 데이터 합계

YYYYMM-uhid --> 히스토그램 데이터

YYYYMM-umad --> 방문자 매트릭스 데이터

각 데이터베이스 집합에는 해당 월의 데이터가 모두 포함됩니다.월별 데이터베이스 집합은 상호 의존적이지 않아 다른 달의 데이터베이스에 영향을 주지 않으면서 보관 및 잘라내기 작업을 독립적으로 수행할 수 있습니다.

일반적인 작업에서 전체 월별 데이터베이스 파일 집합은 월별로 보관됩니다.그러나 다음 4개의 데이터베이스 파일은 Urchin 로그 처리 엔진에서만 사용합니다.해당 데이터베이스 파일은 다음과 같습니다.

YYYYMM-usti

YYYYMM-udai

YYYYMM-ulti

YYYYMM-ultd

다음 데이터베이스 파일은 교차 세그먼트화 및 방문자 드릴다운 보고를 위해 Urchin 로그 처리 엔진에서 사용됩니다.콘텐츠를 제거할 경우 보고 기능에만 영향을 줍니다.

YYYYMM-uvii

YYYYMM-uvid

YYYYMM-used

YYYYMM-upad

YYYYMM-utrd

YYYYMM-uitd

이러한 데이터베이스에는 방문자, 세션, 경로, 거래 및 제품에 대한 정보가 들어 있습니다.이러한 파일은 해당 달에 필요한 총 저장용량의 상당한 부분, 즉 10-50% 정도를 차지할 수 있습니다.따라서 프로필 구성의 '저장/DB' 화면에서 원시 추적 데이터 보관 옵션을 해제하여 상당한 디스크 공간 이점을 얻을 수 있습니다.

원시 추적 데이터를 보관하는 경우 디스크 또는 CPU 리소스 소모 문제가 나타나는 트래픽이 매우 많은 사이트에서만 원시 추적 데이터 보관 기능을 해제하는 것이 좋습니다.

디스크 공간을 절약할 수 있는 다른 방법은 이전 Urchin 월별 데이터베이스를 ZIP 아카이브로 압축하는 방법입니다.이렇게 압축할 경우 생성되는 파일은 압축하지 않은 데이터베이스 집합 크기의 20-30% 정도에 지나지 않습니다.Urchin 보고 엔진에서는 ZIP 아카이브를 직접 읽을 수는 없지만 필요할 경우 ZIP 아카이브에서 데이터베이스를 즉시 추출할 수 있습니다.데이터베이스 압축을 해제하는 동안 약간의 지연이 발생하는 것을 제외하면 Urchin 보고서를 확인하는 데는 문제가 없습니다.보고 엔진은 압축이 해제된 데이터베이스를 제거하지 않으므로 Urchin 보고서를 보는 동안 데이터에 더 빨리 액세스할 수 있습니다.원래 ZIP 아카이브가 그대로 남아 있더라도 정기적인 정리 작업을 통해 압축이 해제된 데이터베이스를 제거하여 디스크 공간을 다시 확보할 수 있습니다.

Urchin의 필수 저장용량을 줄이기 위한 마지막 방법은 Urchin이 제공하는 이전 보고 기간에 대한 정책을 수립하는 것입니다.예를 들어 Urchin이 호스팅 패키지의 보고 서비스로 제공되는 환경에서는 1년 동안의 Urchin 기록을 제공하는 것이 매우 일반적입니다.Urchin 데이터베이스의 월 단위 구성 덕분에, 자동 스크립팅 메커니즘을 통해 특정 임계 기간을 넘어서는 과거 월별 데이터베이스를 매우 간편하게 자동 삭제할 수 있습니다.이전 보고 기간 정책이 구현된 경우 일반적으로 Urchin의 필수 데이터 저장용량은 일정한 수준으로 유지되거나 이전 데이터 보관 한도에 도달할 경우 약간 늘어날 뿐입니다.

데이터 저장용량을 줄이는 방법 - 방법

방법 1:로그 처리 후 원시 추적 데이터 삭제

처리 이후 방문자 및 세션 관련 원시 데이터를 삭제하도록 프로필을 구성할 수 있습니다.대형 사이트의 경우 이를 통해 성능이 크게 개선되어 저장된 데이터의 양이 줄어듭니다.참고로,이 구성을 선택하면 여러 날짜에 겹치는 세션이 한 개 세션이 아닌 두 개 세션(하루에 하나)으로 나타납니다.대부분의 사이트의 경우 결과물에는 그다지 큰 차이가 없습니다.

처리 이후 방문자 및 세션 관련 원시 데이터를 삭제하도록 프로필을 구성하려면 다음 단계를 따르십시오.

  1. 관리 인터페이스에서 '구성'을 클릭하고 'Urchin 프로필'-->'프로필'을 클릭합니다.

  2. 원하는 프로필을 수정합니다.
  3. '저장/DB' 탭에서 '원시 추적 데이터 보관' 입력란을 '해제'로 설정합니다.
  4. '업데이트'를 클릭합니다.

방법 2:이전 데이터 자동 보관

이전 월별 데이터를 아카이브로 압축하도록 프로필을 구성할 수 있습니다.보고서에서 보관된 데이터를 볼 수 있으나 보관된 달에 대한 추가 조회는 처리되지 않을 수 있습니다.

이전 데이터를 보관하도록 프로필을 구성하려면 다음 단계를 따르십시오.

  1. 관리 인터페이스에서 '구성'을 클릭하고 'Urchin 프로필'-->'프로필'을 클릭합니다.

  2. 원하는 프로필을 수정합니다.
  3. '저장/DB' 탭에서 'DB 보관' 입력란을 '설정'으로 설정합니다.
  4. 'DB 보관' 입력란에 달 수를 지정합니다.
  5. '업데이트'를 클릭합니다.

방법 3:이전 보고에 대한 데이터베이스 보관 제한

각 Urchin 프로필의 data/reports/profile-name 디렉토리에서 이전 보고에 대해 필요한 임계값을 초과했으며 YYYYMM 프리픽스로 시작하는 모든 데이터베이스를 제거하면 됩니다.예를 들어 1년 동안의 보고 기록을 보관하려 하고 현재 날짜가 2004년 2월인 경우 Urchin 프로필에 대한 2003년 1월 이후의 보고 데이터를 삭제하려면 이름이 200301-*data.un*인 모든 데이터베이스를 삭제합니다.2003년 1월 이전의 모든 데이터베이스에 대해 이 작업을 반복할 수 있습니다.
