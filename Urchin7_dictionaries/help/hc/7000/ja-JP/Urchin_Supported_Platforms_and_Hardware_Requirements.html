Urchin の対応プラットフォームとハードウェアの要件

Urchin は多くのアーキテクチャやオペレーティング システム上で実行することができます。 Urchin はログ処理を行うシステムにのみインストールします。 レポートの表示に必要なのはウェブブラウザだけです。

対応プラットフォーム

Windows

  • Windows 2003 Server
  • Windows XP
  • Windows 2000
  • Windows Vista
  • Windows 2008 Server

UNIX-type Systems

  • Linux（kernel 2.6） - 最も一般的に使用されています
  • Linux（kernel 2.4） - 最も一般的に使用されています
  • FreeBSD 4 （6.5+ では未対応）
  • FreeBSD 5.3+ （Urchin 6.602+ では未対応）
  • FreeBSD 6.2+

Supported databases

  • MySQL 4.1+ （Urchin 6.6+ では未対応）
  • MySQL 5.03+
  • PostgreSQL 8.2.5

対応ブラウザ

  • Internet Explorer 6.x, 7.x
  • Firefox 2.x, 3.x
  • Chrome 1.x, 2.x

注（未承認プラットフォームについて）

ご使用のプラットフォームに対応する Urchin がない場合でも、OS と互換性のある Urchin をご使用いただける場合があります。

  • Windows 98, Windows 3.x:

Windows 98 または Windows 3.x には Urchin をインストールできませんが、これらのプラットフォームでは、バージョン 5.x 以降の Internet Explorer でレポートを表示できます。

  • Linux:

Linux 系のオペレーティング システムは種類が多く、それぞれに対応する Urchin は提供していませんが、 一般に Linux 系のオペレーティング システムは互換性が高く、現在提供されている Urchin 製品のいずれかがお使いのコンピュータ上で動作する可能性があります。 互換性が確認されている製品は下記のとおりです。

  • Ubuntu (kernel 2.6)

  • RedHat Enterprise Linux 2 & higher

  • SuSE Linux 8 & higher

  • Trustix 2 & higher

  • Solaris:

Solaris は現在サポートされていません。

Urchin 6 のシステム要件

Urchin は低価格のハードウェアでも高いパフォーマンスを実現できます。 たとえば旧式の Pentium II の処理速度はデスクトップ用としては遅すぎるかもしれませんが、Urchin サーバーとしては十分に機能します。 高パフォーマンスを実現するための推奨ハードウェア構成は次のとおりです。

中小規模のウェブサイトを 1 つ解析する場合

  • 2 GHz 以上のプロセッサ
  • 2 GB の RAM
  • 250 GB 以上の IDE ハード ディスク
  • 100 Mbs のイーサネット インターフェース

サービス プロバイダまたは企業に Urchin をインストールする場合

  • 2.5GHz Pentium dual-core / 2GHz UltraSPARC / 同等の CPU 速度の PPC/MIPS/など
  • 3GB RAM
  • Ultra2/Wide SCSI ハード ディスク（Seagate Cheetah など）
  • 100 Base-T イーサネット
  • バックアップ システム

メモリ、システム、ディスクの使用量

  • Urchin のメモリ（RAM）使用量は 20 MB から 500 MB（Urchin 6.5 の場合は 20 MB から 1,024 MB）の間で設定できます
  • Urchin は優先度を低、中、高のいずれかに設定して実行できます
  • Urchin のデータ保存容量は元のログ サイズの約 10 % 程度になります

注（FreeBSD を使用している場合）

(1) FreeBSD では、プロセスのデータサイズを 500 MB に制限するようハードコードされていますが、 ログ処理中にメモリを使用する Urchin の地域データによってこの制限を超過してしまうため、 下記のようなランタイム エラーが表示されます。 

ERROR: (8011-323-1057) Failed to allocate memory

この問題を解決するには、FreeBSD システムのデフォルトのプロセス データサイズを増やす必要があります。これには、/boot/loader.conf ファイルを編集し、下記の行を追加します。 

# Increase max process data segment size to 1GB
kern.maxdsiz="1073741824"

その後、再起動を行ないます。

警告このような方法でシステムの制限値を変更する場合は、慎重に作業を行なってください。 kern.maxdsiz の設定値が大きすぎると、システムを起動できなくなる場合があります。 変更の前に FreeBSD のマニュアルを確認し、独自のサイト構成に適用した場合に予測されるリスクを検証することをおすすめします。

(2) FreeBSD には accf_http と呼ばれる特殊なドライバがあり、 処理速度とパフォーマンスを高めるため、完全な HTTP リクエストが届くまで外部からの接続をバッファします。 デフォルトではこのドライバはカーネルに読み込まれず、下記の警告が表示されます。 

[warn] (2)No such file or directory: Failed to enable the
‘httpready’ Accept Filter

この問題を解決するには、HTTP Accept Filter FreeBSD カーネル モジュール（accf_http）を有効にする必要があります。これには、/boot/loader.conf ファイルを編集し、下記の行を追加します。 

# Wait for full HTTP request accept filter
accf_http_load="YES"

その後、再起動を行ないます。

Linux glibc ライブラリのバージョン

Urchin 6.602 にバンドルされている PostgreSQL 8.4.01 を Linux（カーネル 2.4 または 2.6）にインストールするには、glibc 2.3.3+ が必要です。
