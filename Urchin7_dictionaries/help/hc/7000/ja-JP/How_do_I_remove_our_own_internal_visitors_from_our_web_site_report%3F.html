ウェブサイトのレポートから内部のユーザーを削除する

概要

特定の IP アドレスを経由した IP アドレスのみを取得して、内部の監視システムなどの IP アドレスを除外する場合、プロファイルで IP アドレス フィルタを使用する方法が非常に一般的です。 Urchin では簡単にこの設定を行えますが、 IP アドレス フィルタを設定する際は、[フィルタ フィールド] を正しく指定する必要があります。

手順

内部トラフィック フィルタを適切に設定する最も簡単な手順は次のとおりです。

  • Urchin 管理インターフェースで [構成]、[Urchin プロファイル]、[フィルタ マネージャ] の順にクリックします。
  • [追加] ボタンをクリックして新しいフィルタを作成します。

  • [=> サンプル フィルタの選択] プルダウン メニューから [内部トラフィックの除外] を選択します。

  • [フィルタ パターン] ボックスのサンプルの IP アドレスを、除外する IP アドレスに置き換えます。 IP アドレスの各ドット (.) の前に必ずバックスラッシュ (\) を入れてください。
  • [完了] をクリックし、次の画面で [詳細] をクリックします。
  • [使用可能なプロファイル] ボックスで、フィルタを適用するプロファイルをすべて選択し、右向きの矢印をクリックして [適用済み] ボックスに移動します。
  • [更新] をクリックして、選択したプロファイルにフィルタを適用します。

その他の考慮事項

Urchin のフィルタ パターンは、POSIX 正規表現です。 IP アドレスを指定する際は、Urchin で正規表現のメタ文字として解釈されないよう、「.」文字をエスケープ処理する必要があります。 たとえば、「172.16.1.1」という IP アドレスをフィルタリングする場合、フィルタ パターンは「172\.16\.1\.1」と指定します。 「192.168.xxx.xxx」という IP アドレスの範囲をフィルタリングする場合は、フィルタ フィールドに「192\.168\.」と指定します。他にも、次のような例があります。

複数の IP アドレスをフィルタリングする場合は、「192\.168\.12\.32$|10\.10\.1\.23$|63\.212\.171\.5$」のようにパイプ記号 (|) で区切ります。「$」は最後の数字の後に他の数字が続かないことを示すために使用します。 つまり、「63.212.171.5」と「63.212.171.53」の両方を含める場合は、フィルタに「63\.212\.171\.5」と指定し、 「63\.212\.171\.5」のみを含める場合は、フィルタ パターンを「63\.212\.171\.5$」と指定します。

IP アドレスの範囲は、1 つの文字列で表すことができます。たとえば、「192.168.10.10」から「192.168.10.50」の範囲は「192.168.10.[1-5][0-9]$」と指定できます。

フィルタ パターン「63\.212\.171\.5」には、行の先頭一致により「163\.212\.171\.5」も含まれます。 特定の番号で始まる IP のみを含めるには、「63\.212\.171\.5$」のように文字列の先頭一致を示す「」表現を使用します。

正規表現の全リスト 

 *'''ワイルドカード''' '''意味'''
 *'''.'''       任意の 1 文字を表します
 *'''*'''       直前のアイテムの 0 回以上の繰り返しを表します
 *'''+'''       直前のアイテムの 1 回以上の繰り返しを表します
 *'''?'''       直前のアイテムの 0 回または 1 回の繰り返しを表します
 *'''()'''      かっこの内容を 1 つのアイテムとして扱います
 *'''[]'''      かっこ内のアイテムの任意の 1 つを表します
 *'''-'''       範囲を指定します
 *'''|'''       複数のアイテムのいずれかを表します
 *'''^'''       フィールドの先頭のみを対象とします
 *'''$'''       フィールドの最後のみを対象とします
 *'''\'''       ワイルドカード文字をエスケープ処理します
