Urchin 6.6、6.601、6.602 の変更リスト

追加された機能

Urchin 6.6、Urchin 6.601、Urchin 6.602 リリースの主な機能の概要は次のとおりです。

  • Google AdWords との深い統合

    • 予算のアラート: AdWords のキャンペーンの予算を調整する必要がある場合に警告を表示します。 これは（参照されているプロファイルに関連する）いずれかの CPC ソースに関する予算のアラートが AdWords システムで発生した場合、自動的にレポート画面に表示されます。

    • キーワード生成ツール: 関連するキーワードを生成したり、そのキーワードの掲載結果情報を確認できます。 また、新しく生成されたキーワードを追加したり、AdWords キャンペーンから既存のキーワードを削除することもできます。

    • AdWords との直接リンク: Urchin から AdWords アカウントの適切な画面に直接移動することができます。

    • Urchin タグ マネージャ: 広告のリンク先 URL にキーワード挿入タグ {keyword} を直接挿入できます。 この機能により URL のタグ設定が簡単になり、AdWords の費用データを Urchin に途切れなくインポートできます。

      • （https://adwords.google.com/support/bin/answer.py?hl=ja&answer=74996 をご覧ください。）

    • AdWords オプティマイザー: AdWords で直接キャンペーンの最適化ができます。

  • 広告主向けの表示とセクション

    • 広告主向けダッシュボード テンプレート（表示）と広告の最適化セクション: 検索マーケティング/広告主向けに強化された新しい機能で、検索広告を中心にデータを表示したり、上述の AdWords の関連機能を利用することができます。

  • 新しいレポート

    • "サイト滞在時間" レポート: [コンテンツの最適化] で [コンテンツのパフォーマンス]、[滞在情報] の順にクリックすると確認できます。
    • "パフォーマンスの比較" レポート: ソース/メディア間（Google の CPC 広告と Yahoo のオーガニック検索など）のキャンペーン、キーワード、コンテンツのパフォーマンスを比較できます。 レポートは [広告の最適化] の [マーケティング キャンペーンの結果] にあります。
    • "CPC 構成": [広告の最適化] にあります。
      • [キャンペーン ビュー] には、有料 UTM キャンペーン情報（サポートされている Google と Yahoo のキャンペーン）が表示されます。
      • [キーワード ビュー] には、キーワード広告キーワードの情報が表示されます。

  • データ API

    • データ エクスポート API: これは主な新機能であり、SOAP 1.x と REST プロトコルを使って、Urchin のレポート データを取得できます。

  • 外部認証（LDAP）

    • Urchin 固有の認証（"ネイティブ" といいます）に加えて、Urchin 6.6 ではユーザーごとに設定可能な外部認証がサポートされています。
    • 外部認証モジュール: Urchin 6.6 では LDAP と MSAD がサポートされていますが、設定ファイルを変更するとカスタム モジュールを統合することもできます。
    • SSL/TLS サポート: これらのプロトコルも現在サポートされています。

  • 新しい Urchin "ホーム" ページ

    • Urchin "ホーム" （[ロール アップ] レポートともいいます）では、ログインしているユーザーに表示されているすべてのプロファイルの統計情報を提供できるようになりました。 表示は、 [基本]、[ビジネス]、[管理] のビューから選択できます。
    • プロファイルの統計情報には、直近の日（昨日）、週、月、年のデータが表示されます。

  • Yahoo! Search Marketing（YSM）からの CPC データ自動インポート

    • Urchin では Google AdWords だけでなく、Yahoo! Search Marketing の CPC データもインポートできるようになりました。

  • ワンクリックでインストール

    • FreeBSD 6 以外のすべての Urchin インストーラには、インストール中にインストール可能な Postgresql データベース サーバーが含まれています。 これによって、Urchin のインストール前に、データベース サーバーをインストールして設定する必要がなくなります。 また、インストール中にこのオプションを無効にして、既存のデータベース サーバーを使用することも可能です。 注: Linux Kernel 2.4 で問題が発生した場合は、「報告されている問題」ドキュメントをご覧ください。

  • 最適化された地域データベースの読み込み

    • 地域データベースは、完全版と軽量版を選択でき、ログ処理中のメモリ消費量を低く抑えることができます。 軽量版の地域データベースでは、国レベルまでの大都市圏情報のみを提供しています。

  • Zip\UnZip ユーティリティの更新

    • Zip ユーティリティが新しくなり、2 ギガバイトを超えるデータをアーカイブできるようになりました。

  • マーケティングの概要ダッシュボードの並び替え

    • マーケティングの概要レポートでは、統計情報やその誤差順にソース、キーワード、キャンペーンを並び替えることができます。

  • その他

    • プロセス コントロール: Urchin 6.6 では、ジョブを適切に強制終了できるようになりました（これはデータベースを破損しないことを意味します）。これは、ログを処理するジョブが何らかの理由で "スタック" した場合に便利です。

    • AdWords ツールへのユーザー アクセス: 新しい "CPC ソースへのアクセス" をユーザー単位に設定できるようになりました。 デフォルトで AdWords ツールが有効になっているのは管理者のみです。 他のすべてのユーザーに対しては、この新しい設定で明確にアクセスを許可する必要があります。

    • Bing.com: Microsoft の新しい bing.com 検索エンジンに対するサポートが追加されました。
    • プロファイル フィルタ: このフィルタは、現在のところ CPC ソースに適用されます。
    • プロファイル タブ: "プロファイル" という新しいタブが [ログ ソース管理] に追加され、ログ ソースを複数のプロファイルにリンクできるようになりました。
    • リモート ログ ファイルのダウンロード: リモート ログをダウンロードする際のタイムアウト値が urchin.conf ファイルに追加され、設定できるようになりました。

修正されたバグ

Urchin 6.6、Urchin 6.601、Urchin 6.602 リリースの主な修正されたバグは次のとおりです。

  • 管理

    • Urchin 6.5 とそれ以前のバージョンでは、管理画面でプロファイルの名前を変更しても、プロファイルに関連するフォルダ名は変更されませんでしたが、 これを修正しました。

  • ログの処理

    • 場合によっては Urchin で ELF2 ログの処理中に、セグメンテーション フォルトが発生する場合がありました。 これはログ処理が不十分で、レポート データと一致しなくなった場合に発生していました。 これを修正しました。
    • プロファイルのタイムゾーンの設定の処理が正しくなかったため、レポート データの時間がその期間の時間分ずれていました。 これは、指定されたプロファイルのタイムゾーンでの夏時間の開始月や終了月に発生していました。 これを修正しました。

    • 複数の UNC ログ ソースを処理する際の問題を修正しました。 UNC ログ ソースに関する他の問題とその対処方法については、次のオンライン ヘルプの記事をご覧ください。 https://secure.urchin.com/helpwiki/en/Working_with_UNC_log_sources

  • SSL サポート

    • Windows 2003 で https 経由で Urchin に接続する際の問題を修正しました。

  • CPC ソースのパスワードにおける特殊文字

    • CPC ソースのパスワードでは、特定の特殊文字を使用できませんでしたが、 !@#$%^~`&* _+»<>{}[]()¹?-=:;\|/,. の文字を使用できるようになりました。

  • ライセンス

    • Linux や FreeBSD で 6.600 にアップグレードした場合は、標準ライセンスのリセットや再アクティブ化が必要でしたが、 Urchin 6.601 では再アクティブ可が必要なくなりました。
    • Urchin 6.602 より前のバージョンでは、デモ ライセンスの使用中にアップグレードすると、ログインやライセンスに問題が発生していましたが、 これを修正しました。 注: デモ ライセンスを使用している Urchin をアップグレードした場合、デモ期間は延長されません。
    • 標準ライセンスでは、ログ ソースの最大数が 1,000 から 3,000 に増えました。これには urchin ライセンス サーバーで新しいライセンス キーを生成する必要があります。 urchin を '新しく' インストールした場合は、urchin のバージョンに関係なく、これらの変更が自動的に適用されています。 既存のインストールやアップグレードを行なった場合は、urchin 管理者が ugetlicense ユーティリティを手動で実行する必要があります。

  • 移行

    • レポート データ移行ユーティリティ（convert-u5data）が大幅に改善され、パフォーマンスがさらに最適化されました。 これにより、'サイズの大きな' レポート データをインポートする場合に、所要時間が短縮されることが多くなりました。
    • Urchin 5 から Urchin 6 にレポート データベースを移行する際にクラッシュしたり中断した場合、インポートをその後再開しようとしても次の警告メッセージがでて失敗することがありました。 "WARNING: (8010-577-1192) Database file is the wrong size - run sanitizer". これを修正しました。
    • u5 から u6 にレポート データを移行する際に、urchin 6 に正しくインポートされないデータがありました。 これを修正しました。

  • その他

    • Urchin の埋め込みヘルプとさまざまなオンラインの Urchin ヘルプ センターの記事を更新しました。

使用上の重要な注意

  • MySQL

    • Urchin 6.6xx では MySQL 4.x.x をサポートしていません。 既存の MySQL は、Urchin 6.6 が正しく機能するように MySQL 5.03+ にアップグレードする必要があります。

  • FreeBSD 5.x

    • FreeBSD 5.x プラットフォームは許可されていません。

  • クライアント側の Cookie

    • レポート インターフェースの操作性を高めるため、Urchin 6.6 ではクライアント サイドの Cookie を使用します。 レポートを適切に表示するには、クライアントのブラウザでクライアント サイドの Cookie が有効になっている必要があります。

  • 'デモ' ライセンスを使用している場合の使用制限

    • Urchin 6.6 ではデモ ライセンスに特定の制限を設けています。 プロファイルとログ ソースは、それぞれ 5 つまでに制限されています。 フィルタ数、CPC ソース数、ユーザー数、アカウント数、グループ数に制限はありません。

  • アップグレード

    • Urchin 6.6 には新しいレポート セクションと表示が導入されています。 以前のバージョンからアップグレードした場合は、このレポートと表示に何も表示されない可能性があります。 その場合は、既存のプロファイルを再処理して、新しいレポートを作成することをおすすめします。
