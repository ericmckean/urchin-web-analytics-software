サイトに複数のドメイン名がある場合に トラッキングを設定する

セキュリティで保護されたショッピング カートがサイトの異なるルート ドメインにある場合 (例: www.mystore.com と www.securecart.com) や、複数の別名ドメインで同じコンテンツを提供している場合は、UTM をサイトの構造に対応させるため、いくつかの変更を加える必要があります。

  • 複数のサブドメインをトラッキングする場合と複数のルート ドメインをトラッキングする場合とでは、設定方法が異なるので注意してください。 www.example.com と securecart.example.com はサブドメインが異なり、www.example.com と www.sample.com はルート ドメインが異なります。 サイトで複数のサブドメインを使用している場合は、 サイトのすべてのサブドメインを 1 つのプロファイルでトラッキングする サイトのすべてのサブドメインを個別のプロファイルでトラッキングする をご覧ください。

複数のドメイン名: セキュリティで保護されたショップ サイト

サイトで複数のルート ドメイン名を使用している場合 (例: www.mystore.com と www.securecart.com) は、ショッピング カートがショップ サイトとは異なるドメイン名にある場合に e コマース トラッキングを設定するをご覧ください。

複数のドメイン名: 別名ドメイン

サイトで複数の別名ドメインを使用している場合は、別名ドメインからメインのサイトにトラフィックをリダイレクトすることをおすすめします。 これにより、メインのドメインで UTM ユーザーがトラッキングされるようになり、すべてのユーザーを重複することなくトラッキングできます。

次に Apache サーバーや IIS サーバーで別名ドメインをメインのドメインにリダイレクトする方法を紹介します。

Apache で別名ドメインをリダイレクトする場合

  • Apache ウェブサーバーを使用している場合は、簡単な修正を加えることで、別名ドメインで発生したすべてのトラフィックをメインのサイトにリダイレクトできます。 方法としては、2 つの VirtualHost エントリを作成します。 一方の VirtualHost はメインのドメインとして通常の設定を行い、もう一方の VirtualHost は別名ドメイン用としてメインのドメインにリダイレクトするよう設定します。 

        #---primary virtualhost
        Servername www.mysite.com
        Serveralias mysite.com
        ...
        #---second virtualhost
        Servername mysite.org
        Serveralias www.mysite.org mysite.net www.mysite.net
        RewriteEngine on
        RewriteRule ^(.*) http://www.mysite.com$1 [R=301]

  • 2 つ目の VirtualHost では、301 (完全に移動) リダイレクト コードによる書き換えルールを使用して、すべてのトラフィックをメインのサイトに転送します。 これにより、UTM でサイトが 1 つのドメインのサイトと見なされるようになります。後は UTM の通常の設定でトラッキングできます。

IIS サーバーで別名ドメインをリダイレクトする場合

  • Microsoft IIS ウェブサーバーを使用している場合は、簡単な修正を加えることで、別名ドメインで発生したすべてのトラフィックをメインのサイトにリダイレクトできます。 方法としては、IIS サーバーの構成上に 2 つのウェブサイトを作成します。 一方のウェブサイトはメインのドメイン (www.mysite.com) として通常の設定を行い、もう一方のウェブサイトは別名ドメイン (mysite.net、mysite.org など) 用としてメインのドメインにリダイレクトするよう設定します。 IIS マネージャーでいずれかのウェブサイトを右クリックし、[プロパティ] ダイアログボックスを表示します。 [Web サイト] タブで [詳細設定] ボタンをクリックします。 表示されたウィンドウの [ホスト ヘッダー] フィールドを使用して、別名ドメインをウェブサイトに割り当てます。 メインのウェブサイトにメインのドメインを設定し、2 つ目のウェブサイトにすべての別名ドメインを設定します。 2 つ目のウェブサイトにすべての別名ドメインを設定して有効にした後、次のリダイレクト コードを設定した空のホームページを作成します。 

        <META HTTP-EQUIV=Refresh CONTENT="0; URL=http://www.mysite.com/">
  • これで、ユーザーのブラウザがすぐにメインの URL にリダイレクトされるようになり、 メインのウェブサイトが単一のドメイン構成と見なされるようになります。後は UTM の通常の設定でトラッキングできます。
