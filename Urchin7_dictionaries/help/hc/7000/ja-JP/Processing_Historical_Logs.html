過去のログの処理

概要

インストールした Urchin で過去のログを処理する場合、 手順は簡単です。 ログ マネージャの [ログ設定] 画面で、ディレクトリとファイル名の一部やワイルドカード (正規表現を含む) を指定します。 注: リモートの HTTP や HTTPS のログ ソースではワイルドカードを使用できません。

過去のログの処理方法

まず、システムにログ ソースを追加します。 トップ メニューの [構成] ボタンをクリックして、[Urchin プロファイル] -> [ログ マネージャ] の順にクリックし、 メイン画面の右上にある [追加] ボタンをクリックします。 最初の画面で [ローカル ログ ソースの追加] をクリックして続行し、 次の画面で [参照] をクリックすると [ファイルの参照] が表示されます。 左側のウィンドウで適切なディレクトリを選択すると、 右側のウィンドウにそのディレクトリのファイルが表示され、左側に現在開いているディレクトリのサブディレクトリがすべて表示されます。 適切なディレクトリでファイル名の一部とアスタリスク (またはその他の正規表現) を入力して [確認] ボタンをクリックすると、 入力したパターンに一致するすべてのファイルが新しいウィンドウに表示されます。 ファイル名をクリックすると、ファイルの場所、サイズ、最終更新日、権限に関する情報が表示されます。 パターン マッチが正しければ [OK] をクリックして、[ファイルの参照] ウィンドウでもう一度 [OK] をクリックします。 ログ ソース名を指定してから [次へ] をクリックします。 プルダウン リストからログ形式を選択して [完了] ボタンをクリックします。 ログ ソース名が有効な (固有のものである) 場合は、[完了] ボタンを押すと、ログ マネージャのリストに表示されます。

次に、このログ ファイルをプロファイルに関連付けるため (まだ関連付けられていない場合)、トップ メニューの [構成] ボタンをクリックして [Urchin プロファイル] -> [プロファイル] ボタンをクリックします。 関連付けが完了したら、メインのプロファイル リストのプロファイルの横にある [実行/スケジュール] ボタンをクリックし、 プロファイルの実行スケジュールを設定するか、[今すぐ実行] ボタンをクリックして処理を開始します。

Urchin には、新規のヒットのみを処理するログ トラッキング機能が内蔵されているため、データの重複を回避するログ ローテーションを行う必要はありません。 ただし前述のように、ログは大量のディスク容量を速いペースで消費するため、ログ ファイルは定期的に圧縮、アーカイブすることをおすすめします。 Urchin では、一度処理したログ ファイルを再読み込みする必要がないので、処理が完了するたびにログを削除することができます。 ただし、サイトの新しいプロファイルを作成する際に過去のログ分析が必要になる場合など、さまざまな必要性を考慮して、一定期間のログを保存するのが一般的です。

推奨

ログの管理は当初はそれほど重要ではありませんが、ログが増大するにつれて重要性が増すため、 Urchin の導入時にログの管理方法を決めておくことをおすすめします。
