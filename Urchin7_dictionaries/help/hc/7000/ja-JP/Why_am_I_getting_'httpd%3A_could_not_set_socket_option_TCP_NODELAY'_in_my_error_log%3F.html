エラー ログに「httpd: could not set socket option TCP_NODELAY」と表示される

概要

このメッセージは、ブラウザと Apache サーバー間のネットワーク接続 (ソケット) に設定された特定のオプションのポイントに Urchin Apache ウェブサーバー (urchinwebd) が達する前に、クライアントが接続を切断した場合に表示されます。

解決方法

ネットワーク管理者に問い合わせ、ネットワーク接続が早期に切断される原因を特定することをおすすめします。 考えられる原因は次のとおりです。

  • ロボットやスパイダーの不適切な設定やエージェントの監視により、Urchin Apache サーバーへのネットワーク接続を適切に切断できない。
  • ファイアウォールやネットワークの不適切な設定により、一部のパケット トラフィックが適切に破棄されない。
  • 脆弱性を探すネットワーク ポート スキャンの結果。
  • Urchin Apache ウェブサーバーに対するサービス拒否攻撃。
