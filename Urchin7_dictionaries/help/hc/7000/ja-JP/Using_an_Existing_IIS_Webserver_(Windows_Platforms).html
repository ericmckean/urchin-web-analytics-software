既存の IIS ウェブサーバーの使用（Windows プラットフォーム）

デフォルトでは、Urchin の管理とレポート作成は、Urchin にバンドルされている単体の Apache サーバーを使用して行うよう設定されています。Urchin の管理画面とレポート画面は、ほとんどの場合この方法を使用して表示されます。しかし、まれに既存の IIS ウェブサーバーの使用が必要となる場合がございます。これは、サーバー上でのサードパーティのウェブサーバー製品の使用がサイトで許可されていない場合や、Urchin レポート作成を既存の IIS サーバー上の仮想ホストとして設定する必要がある場合などです。

免責条項: Urchin へのこれらの修正はサポート対象外です。

手順

注: この手順は Urchin がデフォルトの場所（C:\Program Files\Urchin）にインストールされていることを前提としています。Urchin を別の場所にインストールした場合は、下記の例の場所を適切に置き換えてください。

ステップ 1: Urchin ウェブ インターフェースで新しいユーザーを作成する

  • [管理ツール]、[コンピュータ管理] の順にクリックします
  • コンピュータ管理画面の左側にある [ローカル ユーザーとグループ] をクリックします
  • Users フォルダを右クリックし、[新しいユーザー...] を選択します
  • [ユーザー名] フィールドに「IUSR_URCHIN」と入力します
  • [次回ログイン時にパスワードを変更する] チェックボックスをオフにします
  • [ユーザーにパスワードの変更を許可しない] チェックボックスをオンにします
  • [作成]、[閉じる] の順にクリックします
  • 左側にある Users フォルダをダブル クリックします
  • 右側にある IUSR_URCHIN を右クリックし、[プロパティ] を選択します
  • [メンバー] タブで既存エントリをすべて削除し、[追加] ボタンをクリックします。ポップアップ ウィンドウで [ゲスト] を選択し、[追加]、[OK] の順にクリックします
  • [適用]、[閉じる] の順にクリックし、変更を保存します

ステップ 2: Urchin をインストールする（インストールしていない場合）

ステップ 3: Urchin Apache ウェブサーバーを無効にする

  1. [管理ツール]、[サービス] の順にクリックします
  2. [サービス] で [Urchin ウェブサーバー] レコードを見つけます
  3. [Urchin ウェブサーバー] を右クリックして [停止] を選択します
  4. [Urchin ウェブサーバー] を右クリックして [プロパティ] を選択し、[起動タイプ] を [無効] に変更します
  5. [OK] をクリックします

ステップ 4: ISS に新しいウェブサイトを追加する [管理ツール]、[インターネット インフォメーション サービス（IIS）マネージャ] の順にクリックします

  1. [ウェブサイト] を右クリックします
  2. [説明] フィールドに「Urchin」と入力し、[次へ] をクリックします
  3. IP アドレスとポート番号（通常 9999）を選択し、[次へ] をクリックします
  4. [パス] フィールドで Urchin がインストールされている場所（通常は C:\Program Files\Urchin\htdocs）を選択して、[次へ] をクリックします
  5. [実行] チェックボックスをオンにして、[次へ]、[完了] をクリックします
  6. 新しい Urchin ウェブサイトで右クリックし、[プロパティ] を選択します
    1. [ウェブサイト] タブで [ログを有効にする] チェックボックスをオフにします
    2. [ホーム ディレクトリ] タブをクリックし、[スクリプト ソース アクセス] チェックボックスをオンにします
    3. [ドキュメント] タブをクリックし、デフォルトのエントリを両方とも削除します。[追加] をクリックして表示されるポップアップ ウィンドウに「session.cgi」と入力し、[OK] をクリックします。
    4. [ディレクトリ セキュリティ] タブの [認証とアクセス管理] で [編集] をクリックします。[匿名アクセス] チェックボックスがオンになっていることを確認し、[編集] をクリックして [匿名アクセスに使用するアカウント] を変更します。ポップアップ ウィンドウで、ユーザー名に [IUSR_URCHIN] を選択します。[OK] を 2 回クリックしてプロパティ ウィンドウに戻ります。
    5. [OK] をクリックして変更を保存し、プロパティ ウィンドウを閉じます

ステップ 5: ディレクトリの権限を設定する

  1. [スタート] を右クリックして [エクスプローラ] を選択します
  2. Urchin がインストールされている場所（通常は C:\Program Files\Urchin）へ移動します
  3. Urchin フォルダを右クリックし、[プロパティ] を選択します
  4. [セキュリティ] タブをクリックします
  5. [詳細設定] ボタンをクリックします。[継承可能な権限を親からこのオブジェクトに反映させる] チェックボックスをオフにし、ポップアップ ウィンドウで [削除] をクリックします。
  6. [追加] をクリックして管理者ユーザーを選択し、[追加] をクリックします
  7. [追加] をクリックして IUSR_URCHIN を選択し、[追加] をクリックします 1.[名前] フィールドのエントリが管理者と IUSR_URCHIN のみであることを確認します

  8. <urchin_home>\var フォルダに書き込み権限を与えます

  9. [設定] から [セキュリティ] に戻り、IUSR_URCHIN ユーザーに下記の権限が与えられていることを確認します
    • 読み取りと実行
    • フォルダ コンテンツの一覧表示
    • 読み取り
    • 注: デフォルトで設定されている SYSTEM の全権限はそのままにしてください
  10. [OK] をクリックして権限を保存します。
  11. Windows Explorer ウィンドウで Urchin フォルダをクリックします。
  12. data フォルダを右クリックして [プロパティ] を選択します
  13. [セキュリティ] タブをクリックします
  14. [詳細設定] をクリックします。[継承可能な権限を親からこのオブジェクトに反映させる] チェックボックスをオフにし、ポップアップ ウィンドウで [削除] をクリックします。
  15. [追加] をクリックして管理者ユーザーを選択し、[追加] をクリックします
  16. [追加] をクリックして IUSR_URCHIN を選択し、[追加] をクリックします
  17. [名前] フィールドのエントリが管理者と IUSR_URCHIN のみであることを確認します
  18. [設定] から [セキュリティ] に戻り、管理者と IUSR_URCHIN ユーザーの両方に下記の権限が与えられていることを確認します。
    • 自由設定
    • 変更
    • 読み取りと実行
    • フォルダ コンテンツの一覧表示
    • 読み取り
    • 書き込み
  19. [OK] をクリックします

ステップ 6: IIS 6 限定 [ウェブ サービス拡張マネージャ] に移動します

  1. [新しいウェブ サービスの拡張を追加] をクリックします
  2. [拡張名] フィールドに「Urchin CGI」と入力します
  3. [追加] をクリックします
  4. report.cgi と session.cgi を表示して選択します（デフォルトでは *.dll ファイルしか表示されないため、フィルタを変更してすべてのファイルを表示してください）
    • デフォルトの場所: C:\Program Files\Urchin\htdocs
  5. [拡張ステータスを許可に設定] をオンにします
  6. [OK] をクリックします
  7. サーバーのメインの IIS エントリ（ホスト名）に移動します
  8. 右クリックして [プロパティ] を選択します
  9. [MIME 形式] をクリックします
  10. [新規] をクリックします
  11. [拡張] フィールドに「.cgi」と入力します
  12. [MIME 形式] フィールドに「application/octet-stream」と入力します
  13. [OK] をクリックします
  14. [OK] をクリックします

これで IIS ウェブサーバーの設定が終了しました。http://my.server.com:port（port は Urchin のインストール時に使用するよう設定したポートで、デフォルトは 9999）に接続すると、Urchin ウェブ インターフェースを呼び出すことができます。

Urchin データ API Web サービス（Urchin 6.6+）での既存の IIS ウェブサーバーの使用

Urchin データ API Web サービス（Urchin 6.6+）は IIS モジュールにバンドルされており、実装して設定すると Urchin データ API へのリクエストを処理できます。IIS ウェブサーバーでデータ API を設定するには、[Urchin データ API と IIS を統合できますか]をご覧ください。
