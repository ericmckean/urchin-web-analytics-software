失敗した FTP と HTTP のリモート ウェブサーバー ログ転送のエラー コード リスト

概要

Urchin ログ ソースは、FTP または HTTP 経由でリモート サーバーのウェブサーバー ログを収集するよう設定することができます。 通常、転送が成功するとランタイム ログにエラーは表示されませんが、 転送中にエラーが発生した場合 (無効なユーザー名やパスワード、リモート サーバーにアクセスできない、リモート ログが解読不可能など)、Urchin はプロファイルのタスク履歴に表示されるランタイム出力にエラー コードを記録します。 エラー コードは、ウェブサーバー ログの転送が試行された後、エラー メッセージの横に (-9) のようにかっこ内に表示されます。

エラー コードと発生した問題について説明するテキスト メッセージを次に示します。

エラー コード リスト

CURLE_UNSUPPORTED_PROTOCOL (1) - libcurl に渡された URL は、libcurl がサポートしないプロトコルを使用しました。 使用しなかったコンパイル時のオプションが原因であるか、プロトコル文字列のつづりの間違いや、プロトコル libcurl にコードがないだけという可能性があります。

CURLE_FAILED_INIT (2) - 最初の初期化コードが失敗しました。 内部にエラーや問題がある可能性があります。

CURLE_URL_MALFORMAT (3) - URL は適切にフォーマットされませんでした。

CURLE_COULDNT_RESOLVE_PROXY (5) - プロキシを解決できませんでした。 指定のプロキシ ホストは解決されませんでした。

CURLE_COULDNT_RESOLVE_HOST (6) - ホストを解決できませんでした。 指定のリモート ホストは解決されませんでした。

CURLE_COULDNT_CONNECT (7) - ホストまたはプロキシに接続できませんでした。

CURLE_FTP_WEIRD_SERVER_REPLY (8) - FTP サーバーに接続した後、libcurl は特定の応答を待機します。 このエラー コードは予期しない応答を取得したことを示しています。 指定のリモート サーバーが適切な FTP サーバーでない可能性があります。

CURLE_REMOTE_ACCESS_DENIED (9) - URL で指定されたリソースへのアクセスが拒否されました。 FTP の場合、リモート ディレクトリに変更しようとしている場合に発生します。

CURLE_FTP_WEIRD_PASS_REPLY (11) - FTP のパスワードをサーバーに送信した後、libcurl は適切な応答を待機します。 このエラー コードは、予期しないコードが返されたことを示しています。

CURLE_FTP_WEIRD_PASV_REPLY (13) - libcurl は、PASV コマンドまたは EPSV コマンドへの応答として、サーバーから適切な結果を取得できませんでした。 サーバーに問題があります。

CURLE_FTP_WEIRD_227_FORMAT (14) - FTP サーバーは PASV コマンドへの応答として 227 行を返しました。 libcurl が該当の行の解析に失敗した場合、このリターン コードが返されます。

CURLE_FTP_CANT_GET_HOST (15) - 新しい接続に使用するホストのルックアップ中に内部エラーが発生しました。

CURLE_FTP_COULDNT_SET_TYPE (17) - 転送モードをバイナリまたは ASCII に設定しようとしているときにエラーが発生しました。

CURLE_PARTIAL_FILE (18) - ファイルの転送に予期するより短いまたは長い時間がかかりました。 これは、サーバーが予測される転送サイズを最初に報告し、予測されたサイズに一致しないデータが配信された場合に発生します。

CURLE_FTP_COULDNT_RETR_FILE (19) - RETR コマンドへの不適切な応答か、転送がまったく行われませんでした。

CURLE_QUOTE_ERROR (21) - カスタムの QUOTE コマンドをリモート サーバーに送信したときに、コマンドの 1 つが FTP について 400 以上のエラー コードを返したか、コマンドが成功しなかったことを示しています。

CURLE_HTTP_RETURNED_ERROR (22) - CURLOPT_FAILONERROR が TRUE に設定され、HTTP サーバーが 400 以下のエラー コードを返した場合に返されます (以前は CURLE_HTTP_NOT_FOUND と呼ばれていました)。

CURLE_WRITE_ERROR (23) - 取得したデータをローカル ファイルに書き込んでいるときにエラーが発生したか、書き込みコールバックから libcurl にエラーが返されました。

CURLE_UPLOAD_FAILED (25) - アップロードの開始に失敗しました。 FTP の場合、一般的にサーバーでは STOR コマンドを拒否します。 通常、エラー バッファにはこのコマンドに対するサーバーの説明が含まれています (以前は CURLE_FTP_COULDNT_STOR_FILE と呼ばれていました)。

CURLE_READ_ERROR (26) - ローカル ファイルの読み込みに問題があるか、読み込みコールバックによってエラーが返されました。

CURLE_OUT_OF_MEMORY (27) - メモリ割り当てのリクエストに失敗しました。 これは深刻な状態で、発生すると重大な失敗につながります。

CURLE_OPERATION_TIMEDOUT (28) - 操作がタイムアウトしました。 条件に従って、指定されたタイムアウト時間に達しました。

CURLE_FTP_PORT_FAILED (30) - FTP PORT コマンドがエラーを返しました。 これは、使用する libcurl に十分なアドレスを指定しなかった場合に頻繁に起こります。 CURLOPT_FTPPORT を参照してください。

CURLE_FTP_COULDNT_USE_REST (31) - FTP REST コマンドがエラーを返しました。 これは、サーバーが正常な場合には発生しません。

CURLE_RANGE_ERROR (33) - サーバーで範囲リクエストがサポートまたは承認されません。

CURLE_HTTP_POST_ERROR (34) - 主に内部の問題が原因で発生するエラーです。

CURLE_SSL_CONNECT_ERROR (35) - SSL/TLS Handshake プロトコルで発生する問題です。 エラー バッファが必要です。問題をより特定できるよう、エラー バッファでメッセージを読み込む必要があります。 証明書、(ファイル形式、パス、権限)、パスワードなどに原因がある可能性もあります。

CURLE_FTP_BAD_DOWNLOAD_RESUME (36) - ファイル サイズを超えて FTP を再開しようとしています。

CURLE_FILE_COULDNT_READ_FILE (37) - FILE:// で指定されたファイルを開くことができませんでした。 指定されたパスのファイルにアクセスできない可能性があります。 ファイルの権限を確認してください。

CURLE_LDAP_CANNOT_BIND (38) - LDAP を関連付けることができません。 LDAP のバインド操作に失敗しました。

CURLE_LDAP_SEARCH_FAILED (39) - LDAP の検索に失敗しました。

CURLE_FUNCTION_NOT_FOUND (41) - 関数が見つかりませんでした。 必要な zlib 関数が見つかりませんでした。

CURLE_ABORTED_BY_CALLBACK (42) - コールバックによって異常終了しました。 コールバックは libcurl に「abort」を返しました。

CURLE_BAD_FUNCTION_ARGUMENT (43) - 内部エラーです。 関数は不適切なパラメータを使用して呼び出されました。

CURLE_INTERFACE_FAILED (45) - インターフェース エラーです。 指定された発信インターフェースを使用できませんでした。 CURLOPT_INTERFACE を使用して、外部接続のソース IP アドレスに使用するインターフェースを設定してください (以前は CURLE_HTTP_PORT_FAILED と呼ばれていました)。

CURLE_TOO_MANY_REDIRECTS (47) - リダイレクトが多すぎます。 リダイレクトに従うと、libcurl は最大量をヒットします。 CURLOPT_MAXREDIRS を使用して上限を設定してください。

CURLE_UNKNOWN_TELNET_OPTION (48) - CURLOPT_TELNETOPTIONS のオプション セットが認識されないか不明です。 該当のドキュメントを参照してください。

CURLE_TELNET_OPTION_SYNTAX (49) - telnet オプションの文字列が不正にフォーマットされました。

CURLE_PEER_FAILED_VERIFICATION (51) - リモート サーバーの SSL 認証または SSH md5 指紋が承認されませんでした。

CURLE_GOT_NOTHING (52) - サーバーから何も返されませんでした。この場合、何も返されないのはエラーとみなされます。

CURLE_SSL_ENGINE_NOTFOUND (53) - 指定された crypto エンジンが見つかりませんでした。

CURLE_SSL_ENGINE_SETFAILED (54) - 選択された SSL crypto エンジンをデフォルトに設定できませんでした。

CURLE_SEND_ERROR (55) - ネットワーク データの送信に失敗しました。

CURLE_RECV_ERROR (56) - ネットワーク データの受信に失敗しました。

CURLE_SSL_CERTPROBLEM (58) - ローカル クライアントの認証に問題があります。

CURLE_SSL_CIPHER (59) - 指定された暗号を使用できませんでした。

CURLE_SSL_CACERT (60) - 既知の CA 証明書を使用してピア認証を承認できませんでした。

CURLE_BAD_CONTENT_ENCODING (61) - 転送エンコーディングを認識できません。

CURLE_LDAP_INVALID_URL (62) - 無効な LDAP の URL です。

CURLE_FILESIZE_EXCEEDED (63) - ファイル サイズの上限を超えました。

CURLE_USE_SSL_FAILED (64) - リクエストされた FTP SSL レベルが失敗しました。

CURLE_SEND_FAIL_REWIND (65) - 送信処理時に curl はデータを巻き戻して再送信する必要がありましたが、巻き戻しの処理に失敗しました。

CURLE_SSL_ENGINE_INITFAILED (66) - SSL エンジンの起動に失敗しました。

CURLE_LOGIN_DENIED (67) - リモート サーバーが curl のログインを拒否しました (7.13.1 に追加)。

CURLE_TFTP_NOTFOUND (68) - TFTP サーバー上にファイルが見つかりません。

CURLE_TFTP_PERM (69) - TFTP サーバーに権限に関する問題があります。

CURLE_REMOTE_DISK_FULL (70) - サーバー上にディスク容量がありません。

CURLE_TFTP_ILLEGAL (71) - 不正な TFTP 操作です。

CURLE_TFTP_UNKNOWNID (72) - 不明な TFTP 転送 ID です。

CURLE_REMOTE_FILE_EXISTS (73) - ファイルは既に存在し、上書きすることはできません。

CURLE_TFTP_NOSUCHUSER (74) - このエラーは、適切に機能している TFTP サーバーによって返されることはありません。

CURLE_CONV_FAILED (75) - 文字変換に失敗しました。

CURLE_CONV_REQD (76) - 発信者は変換のコールバックを登録する必要があります。

CURLE_SSL_CACERT_BADFILE (77) - SSL CA 認証の読み込みに問題があります (パスまたはアクセス権限)。

CURLE_REMOTE_FILE_NOT_FOUND (78) - URL で参照されているリソースが存在しません。

CURLE_SSH (79) - SSH セッション中に不明なエラーが発生しました。

CURLE_SSL_SHUTDOWN_FAILED (80) - SSL 接続の切断に失敗しました。
