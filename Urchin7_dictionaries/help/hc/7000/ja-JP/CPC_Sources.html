CPC（クリック単価）ソース（Urchin 6.5+）

Urchin 6.5 では新しいデータソースとして、 クリック単価（CPC）ソースを使用できます。 ログ ソースと同様に、CPC ソースにより外部ソースから CPC キャンペーン データを自動的に抽出することができます。 CPC ソースはログ ソースやフィルタなど既存のプロファイルにリンクされ、Urchin レポートに CPC データを自動的に含めることができます。 作成できる CPC ソースは Google AdWords（Urchin 6.5+）と Yahoo! Search Marketing（Urchin 6.6+）です。

注: AdWords 初心者モードには制限があるため、初心者モードの AdWords アカウントでは CPC ソースを使用できません。 CPC ソースを使用するには、AdWords 標準モードにアップグレードする必要があります。

6.500 から 6.501 へのアップグレードに関する注意事項: 6.500 から 6.501 にアップグレードする際は、古い CPC データを手動で削除する必要があります。 これには、下記のディレクトリのコンテンツをすべて削除します。 

<path_to_urchin_6500_installation>/data/cpc/...
<path_to_urchin_6500_installation>/data/history/cpc/...

CPC ソース マネージャ

CPC ソース マネージャ セクションは、[設定] の [Urchin プロファイル] セクションにあります（Urchin 6.5 +）。

CPC ソース マネージャには、現在の Urchin インスタンスで設定されているクリック単価（CPC）ソースのリストが表示され、管理者は UI を使用して CPC ソースを作成、編集することができます。

新しい CPC ソースの作成

新しい CPC ソースを追加、設定するには、CPC ソース マネージャの右上にある [追加] をクリックします。

"CPC ソースの追加ウィザード" が起動し、手順に従って CPC ソースを追加、設定できます。

ウィザードの最初の手順では、下記のデータを入力します。

- オプションのアカウント - 新しく追加した CPC ソースにリンクするアカウントを指定します（データ センター モードのみ）。

- CPC ソース名 – CPC ソースの名前を指定します。

- CPC ソースの種類:

  • Google AdWords

    • メール* - アクセスする AdWords アカウントかクライアント センター アカウントのログイン用メール アドレスを指定します。

    • パスワード* – ログイン アカウントのパスワードを指定します。

    • 開発者トークン – AdWords API の承認された開発者として識別するための固有の文字列です（オプション）。Urchin にはデフォルトの開発者トークンが設定されるため、独自の開発者トークンを使用しない場合は空白にします。

    • アプリケーション トークン - AdWords API へのアクセスに使用するアプリケーション トークンです。 開発者トークンと同様に、空白にすることもできます。

    • クライアントのメール - クライアント センターのログインを使用する場合（[メール] にクライアント センター アカウントのログインを指定した場合）は、アクセスするクライアント アカウントのログイン用メールを入力します。

    • お客様 ID – クライアント センターのログインを使用する場合は、アクセスする AdWords アカウントのお客様 ID（Google AdWords の画面右上に 123-456-7890 のように表示される数字）を入力します。 これはクライアントのメールの代わりに使用することができます。

"CPC ソースのテスト" ボタンを押して、CPC ソースの設定を検証できます。

  • Yahoo! Search Marketing

    • ユーザー名* - アクセスする YSM アカウントのユーザー名です。 広告主のメール アドレス、または広告主の代わりにレポートにアクセスする代理店のメール アドレスを使用できます。
    • パスワード* - アクセスする YSM アカウントのパスワードです。
    • マスター アカウント ID* - 広告主のマスター アカウント ID です。
    • アカウント ID* - 広告主のアカウント ID です。
    • ライセンス キー - ログインしているユーザー、広告主、または代理店のライセンス キーです。
    • 予備のユーザー名 - 広告主の予備のユーザー名です。 広告主のユーザー名ではなく代理店のユーザー名を設定した場合に必要です（オプション）。
    • 予備のパスワード - 広告主の予備のパスワードです（オプション）。

'完了' をクリックすると手順を完了します。

CPC ソースの編集

CPC ソースを追加した後、設定の変更、削除、ダウンロードのスケジュール設定を行うことができます。

既存の CPC ソースの設定を編集するには、該当の CPC ソースの横にある [編集] をクリックします。

AdWords アカウントや YSM アカウントのログインの設定を変更したり、CPC ソースの今後のダウンロードを有効または無効にできます。 注: レポートのダウンロードを無効にした場合でも、CPC ソースにリンクされたプロファイルではダウンロード済みのレポートが引き続き使用されます。

'プロファイル' タブでは、CPC ソースを 1 つまたは複数のプロファイルにリンクできます。 リンクできるプロファイルは、現在の Urchin アカウントのものに限定されます。 リンクされた CPC ソースの CPC データはこれらのプロファイル レポートに表示されます。

CPC ソースのスケジュール設定

CPC ソースの編集 画面の スケジュール タブで、CPC ソースのダウンロードを設定できます。

  • - スケジュール – ダウンロードの間隔 – CPC のダウンロードは 2 段階に分けられます。 最初に CPC 証明書の検証を行い、CPC サーバーにレポートをリクエストします。 次に、レポートが利用可能になるまで定期的に確認を行います。 レポートのダウンロードが完了すると、2 つ目の処理は終了します。 サーバーでレポートが利用可能になるまでには、アカウントのキャンペーン数、取得するデータ数、現在のサーバーの負荷などに応じて、ある程度の時間がかかります。ダウンロードの間隔では、Urchin がリクエストしたレポートが利用可能かどうかを確認する間隔を設定します。 基本的に、大量のクリック トラフィックが発生している大規模なアカウントでは、間隔を長く設定することをおすすめします。

    - ダウンロード - 時間帯 – CPC ソースで更新されたレポートをダウンロードする時間帯を設定できます。 重要: CPC データは 1 時間ごとではなく 1 日ごとに抽出されるため、AdWords または YSM アカウントのタイムゾーンで 1 日が終了してから 2 時間程度後にダウンロードすることを おすすめ します。 これにより、AdWords と YSM のサーバーで 1 日のすべての CPC データを利用できます。 CPC ソースを設定したプロファイルは、CPC データをダウンロードしてから処理するようスケジュールすることをおすすめします。 これにより、レポートに最新の CPC データを反映できます。

    - アカウント データ - ダウンロードの間隔（Urchin 6.6+ の AdWords のみ） - このオプションでは AdWords のアカウント構成をダウンロードする間隔を設定できます。 キャンペーンやキーワードのメタデータを表示したり、キーワード生成ツール、キャンペーン コピー ツール、AdWords タグ マネージャ ツールなどを使用する場合は、事前に AdWords のアカウント構成をダウンロードしておく必要があります。

[全般設定] 画面の [CPC ソースの更新] タブで、CPC ソースのグローバルなデフォルト ダウンロード設定を指定できます。
