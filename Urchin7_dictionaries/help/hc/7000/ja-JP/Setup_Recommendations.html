推奨設定

概要

Urchin をインストールしたら、初期オペレーション パラメータを設定する必要がございます。 これは、Urchin 管理インターフェースに初めて接続した時に実行されるセットアップ ウィザードで、はじめにプロファイルを設定する時に行なうことができます。 初期設定の内容は下記のとおりです。

  • Urchin のライセンス
  • リモート レポートと管理アクセスの管理者設定、およびデータセンター モードの設定
  • Urchin 管理者アカウントのパスワード設定
  • 各プロファイルのデータ処理タスクのスケジュール
  • ログの管理

手続き

Urchin 管理インターフェースに接続します。 Windows システムの場合、[スタート]、[プログラム]、[Urchin]、[Urchin 管理] の順にクリックします。 UNIX タイプのシステムの場合、URL http://hostname:9999 (登録した Urchin システムのホスト名を入力) からアクセスします。 新規にインストールする場合、下記を使用してログインします。

ユーザー名: admin パスワード: urchin

Urchin 設定ウィザードの画面が表示されます。 下記の画面に進むには、[次へ] をクリックします。 初期設定で選択した設定は、後でいつでも変更することができます。

Urchin のライセンス

Urchin のライセンス に記載された手順に従って操作します。

管理者設定

  • リモート アクセス設定 - リモート ブラウザ接続を許可する場合、それぞれ [オン] を選択します。 いずれかに [オフ] を選択した場合、Urchin がインストールされたシステムの画面からのみアクセスが許可されます。
  • データセンター モード - アクセス制限のカテゴリ別にプロファイル、グループ、ユーザーを効果的に構成するため、アカウントを作成できるようにするかどうかを設定します。 どうするか決めていない場合、将来必要になることを考えて [オン] に設定することをお勧めします。追加費用がかかることはございません。

管理ユーザー

  • 管理者アカウントのパスワードをリセットし、パスワードを安全に記録します。

タスクのスケジュール

プロファイル作成時に、プロファイルのデータ処理時間をスケジュールするオプションを設定できます。 ログ データはいつ利用可能になるか、データ処理にかかる時間はどれくらいか、また、レポートをユーザーにいつ提供するかなどを考慮し、効果的にタスクをスケジュールするため、各プロファイルの設定を確認する必要がございます。

ログの管理

Urchin には、各ログがどこまで処理されたかを測定するためのログ トラッキング モジュールが含まれています。 このため、ログ ファイルのローテーションと Urchin の処理を調整する必要はございません。 しかし、Urchin では、ログ ソースの [詳細設定] から、ログ ローテーション、または削除を自動化することができます。

詳細については、ログの管理 をご覧ください。
