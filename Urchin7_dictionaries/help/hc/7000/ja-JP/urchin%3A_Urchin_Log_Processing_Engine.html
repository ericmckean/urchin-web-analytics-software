urchin: Urchin ログ処理

概要

Urchin ログ処理エンジン、urchin は、Urchin のログ処理の中心となるコンポーネントです。 通常、ログ処理エンジンは、タスク実行時に Urchin Scheduler (urchind) から呼び出されます。 しかし、コマンド シェルから直接 urchin を実行し、特定のプロファイルを実行することもできます。 これは、Windows タスク スケジューラ、または UNIX タイプのシステムの cron など、外部ソースから Urchin タスクを実行するため高度にスクリプト化された環境下で有用な機能です。 また、特定の日のアクセス数のみを処理する場合や、特定のタイプのデバッグを行なう場合など、特別の状況下でプロファイルを実行する場合にも便利です。

Urchin は厳密にはユーティリティではございませんが、特別な場合に役に立つコマンド ライン機能についてここで説明します。

使用方法

urchin は Urchin ディストリビューションの bin ディレクトリにございます。 使用方法は下記のようになります。 

urchin [options...] (-p <profile name> | -P <profile id>)

オプション: 

-a <account name>       プロファイルのアカウント名を指定します。 デフォルトでは、アカウント名は '(なし)' に設定されています。

-D                      デバッグ モードを有効化します

-d <YYYYMMDD>           処理日を指定します

-H                      実行出力を履歴ファイルに記録するよう指定します。 -H が指定され、-T が指定されていない場合、新しいタスクの履歴が作成されます

-h                      ヘルプ情報を出力して終了します

-P <profile id>         処理するプロファイルの ID を指定します

-p <profile name>       処理するプロファイル名を指定します

-T <task id>            更新するタスク履歴レコードを指定します。 -T を指定した場合は -H も実行されます

-t                      設定データを出力して終了します

-v                      バージョン番号を出力して終了します

アドバイス

  • UNIX システムで urchin を実行する場合、Urchin がインストールされたユーザー、UID を使用し、プロファイルのデータベースと同じ UID で開始されるようにします。データベースが見つからない場合、urchin によりデータベースが作成されます。
