UTM Quick-Install (IIS) (英語)

  • この記事は、Urchin ソフトウェア認定コンサルタントの EpikOne:EpikOne によって作成されています

各種 Windows プラットフォームの Microsoft IIS サーバー上で実行するウェブサイトへの UTM のインストールについて、概要を説明します。 UTM に関する詳細情報については、このセクションの "Urchin トラフィック モニタ (UTM)" を参照してください。

ステップ 1: UTM ファイルをウェブサイトのドキュメント ルートにコピーします。 UTM ファイル (urchin.js、__utm.gif) は Urchin の "utils\utm" フォルダにございます。 この 2 つのファイルをウェブサイトのコンテンツのメイン フォルダにコピーします。 重要: GIF ファイル名は 2 文字のアンダースコアで始まります。

ステップ 2: ウェブサイトの HTML に UTM への参照を設定します。 すべての HTML ページに下記の行を挿入します。 ページ内のどの場所でも挿入できますが、</body> タグの直前をおすすめします。 インクルードまたはテンプレートを共有している場合は、これらのファイルのいずれかに挿入してください。 

<script src="/urchin.js" type="text/javascript"></script>

<script type="text/javascript">

        urchinTracker();

</script>

"HTML Tidy" などのパッケージを使用している場合は、ページの HEAD セクションに下記のような Javascript 行を含めることができます。 

<html>

<head>

  <meta http-equiv="Content-Type" content="text/html; charset=ISO-8859-1">

  <script src="/urchin.js" type="text/javascript"></script>

  <script type="text/javascript">

        urchinTracker();

  </script>

  ...

</head>

ステップ 3: IIS ログで cookies を有効にします。 IIS マネージャを開き、ウェブサイトの設定ウィンドウに移動します。 ログが有効になっていること、ログ形式が W3C Extended になっていることを確認してください。 [拡張プロパティ] タブの下の "設定" の隣りにある [プロパティ] ボタンをクリックし、"Cookie" の隣りのチェック ボックスをオンにします。

ステップ 4: Urchin トラフィック モニタ (UTM) で使用する Urchin プロファイルを設定します。 Urchin 管理インターフェースで、対象のプロファイルを編集して [設定] タブをクリックします。 [ユーザーのトラッキング方法] を「Urchin トラフィック モニタ (UTM)」に設定します。 [UTM ドメイン] にウェブサイトの URL (「www.」を除く) を入力します。 その後、[レポート セット] でデフォルトを選択し、[更新] ボタンをクリックします。

上記の手順を完了すると ウェブサイトの通常のログ ファイルに UTM データが記録されるようになります。このログ ファイルは、Urchin の下記実行時に識別されます。

動作の確認 UTM でログ ファイルにエントリが正しく作成されているかどうかを確認するには、UTM をインストールし、サイトのページをいくつかクリックしてからログを確認します。 ログ ファイルの末尾に下記のようなエントリが記述されます。 

... "GET /__utm.gif?..." 200 ..."__utma=..."

utma エントリがない場合は、ログで cookie が有効になっていることを確認してください。 ステータス コードが「200」でない場合は、ファイルがドキュメントのルート ディレクトリに正しくコピーされていることを確認してください。
