CPC データのインポート ユーティリティ（Urchin 6.5+）

Urchin 6.5 では CPC データを処理する新しいユーティリティが追加されました。

urchincpc - Urchin CPC データ インポート ツールでは、Google AdWords と Yahoo! Search Marketing の API により CPC データを抽出し、詳細な処理とレポート作成を行います。

  • 注: AdWords からダウンロードする CPC アーカイブ データは、250 MB 以下である必要がございます（Urchin 6.501+）。

urchincpcd - Urchin CPC ソース スケジューラ（または CPC スケジューラ）では、プロファイル スケジューラと同様にユーザー定義のスケジュールに基づいて CPC ダウンロード タスクのスケジュールを設定します。

CPC スケジューラ（urchincpcd）は Urchin マスター スケジューラ（urchind）とともにローカル ホストで実行します。CPC ソースをスケジュール設定するには、マスター スケジューラを起動して実行する必要がございます。CPC スケジューラは CPC データのダウンロード タスクのキューを定期的に確認し、実行を管理します。

CPC インポート ツール（urchincpc）はスケジュール設定されたタスクがあると自動的に開始します。CPC スケジューラによりワーカー プロセスが内部で起動されますが、コマンド ラインを使用して直接呼び出し、キューに追加されているタスクの実行を開始することができます。

ダウンロードした CPC データは /path/to/urchin/data/cpc/ でローカルに保存され、リンクしているプロファイルの処理中に Urchin エンジンにより自動的に抽出されます。

Urchin CPC スケジューラは urchinctl ユーティリティで管理します。CPC ソース スケジューラのみを呼び出すには、-c オプションを使用します。たとえば下記のように指定します。 

urchinctl -c start/stop/restart

CPC スケジューラのみを開始、停止、再起動します。

CPC データをすぐにダウンロードするようスケジュール設定するには、[設定]、[CPC ソース マネージャ]、[ダウンロード] または [編集] の順にクリックし、対象の CPC ソースの [スケジュール] タブで [今すぐダウンロード] ボタンをクリックします。リクエストを処理するには、CPC スケジューラを起動して実行する必要がございます。

使用法: CPC スケジューラ - urchincpcd

CPC スケジューラは Urchin 6.5+ の bin フォルダにございます。下記のオプションはコマンド ラインから使用できます。

  • -d デバッグ モードを有効化します。デバッグに関する詳細が出力に記録されます。たとえば Windows では -console のみ動作します。 

    •            urchincpcd.exe -console -d

  • -h ヘルプ情報を出力して終了します。

  • -v バージョン番号を出力して終了します。

Windows では、下記のいずれかのコマンドを最初の引数として指定します。

  • -install CPC スケジューラを Windows サービスとしてインストールする

  • -remove CPC スケジューラを Windows サービスから削除する

  • -start CPC スケジューラを Windows サービスとして開始する

  • -stop CPC スケジューラを Windows サービスとして停止する

  • -console CPC スケジューラをコンソール モードで開始する

使用法: CPC インポート ツール - urchincpc

下記のオプションはコマンド ラインから使用できます。

  • -c [id] 指定したコスト ソースをダウンロードします。「id」は CPC ソースの ID です。

  • -d デバッグ モードを有効化します。

  • -h ヘルプ情報を出力して終了します。

  • -v バージョン番号を出力して終了します。

パラメータを指定しないか、-d オプションのみを指定して CPC スケジューラを開始すると、キューに追加されているタスクの処理を開始し、処理するタスクがなくなると終了します。

CPC データのプロキシ経由でのダウンロード（6.501+）

CPC データをプロキシ経由でダウンロードするには、urchin.conf ファイルを編集して、WebServiceProxyHost、WebServiceProxyPort、WebServiceProxyUsername、WebServiceProxyPassword エントリのコメントを外し、必要な値を設定します。 

 ######################################################
# CPC ソース マネージャ設定
#...
# WebServiceProxyHost、WebServiceProxyHost: CPC ソース マネージャが外部 CPC ソースへの
# アクセスに使用する HTTP/HTTPS プロキシのホストおよびポートです。
# WebServiceProxyUsername、WebServiceProxyPassword: プロキシの認証に使用する
# ユーザー名とパスワードです。
######################################################
#WebServiceProxyHost: <プロキシ ホスト>
#WebServiceProxyPort: <プロキシ ポート>
#WebServiceProxyUsername: <ユーザー名>
#WebServiceProxyPassword: <パスワード>

WebServiceProxyHost には IP アドレスか完全なドメイン名を入力できます。
