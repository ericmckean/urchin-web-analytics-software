geo-update: DNS データベース更新ユーティリティ

概要

地域データ更新ユーティリティは、Urchin の内部 DNS データベースの更新を確認し、更新がある場合はダウンロードします。 このユーティリティでは、Google ダウンロード サーバーやローカル ディレクトリ (Urchin 6 の場合) から更新情報をダウンロードできます。

使用法

地域データ更新ユーティリティは、月 1 回 Urchin スケジューラ (urchind) から呼び出すか、コマンド ラインから手動で呼び出すことができます。 スケジューラから呼び出す場合は、Google ダウンロード サーバーの更新情報が使用されます。 コマンド ラインから呼び出す場合は、ローカル パスの DNS データベースの更新情報を使用することもできます。 このタスクは、月 1 回新しい更新情報を確認するよう設定されており、ユーザーが管理インターフェースで実行時間を指定することもできます。

作成された地域データ更新のタスクは、タスク履歴の画面に「地域データ更新」の名前で表示されます。

デフォルトの動作では、更新の有無を確認して新しいデータベースをダウンロードし、既存のデータベースを上書きした後。 デフォルトの動作を変更するには、コマンド ラインから適切なオプションを付けて geo-update を実行します。 使用方法は次のようになります。 

geo-update [-Hhv] [-D | -F] [-i ファイル]

各オプションの説明は以下のとおりです。 

  -F            地域データベースを強制的にダウンロードします
  -f            ローカル ファイルシステム上の geo.zip ファイルのパス名です [オプション]
  -H            実行出力を履歴ファイルに記録するよう指定します
  -h            ヘルプ情報を出力して終了します
  -T <task id>  更新するタスク履歴レコードを指定します
                -T を指定した場合は -H も実行されます
  -V            地域データベースのバージョン番号を出力して終了します
  -v            バージョン番号を出力して終了します

-F オプションを使用すると、データベースが既に最新版であっても、強制的に新しいデータベースをダウンロードします。 このオプションは、カスタムのドメイン情報を誤ってインポートした、またはドメイン データベースが破損した際に、最新のデータでやり直す場合に便利です。

-H オプションを使用して出力を履歴ファイルに書き込むと、Urchin 管理インターフェースの [構成] -> [スケジューラ] にある [タスクの履歴] 画面で実行情報を確認できるようになります。

-T オプションを使用すると、コマンド ライン インターフェースで <Task_ID> を指定して特定のタスクを強制的に更新することができます。 Task_ID の値は、Urchin 6 のデータベースで次のクエリを使用して確認できます。 

mysql> select * from ugeo_tasks\G

例

最新のドメイン データベースを強制的にダウンロードする場合: 

geo-update -F

データベースから取得したタスク ID を使用して、タスクを CLI から手動で更新する場合: 

geo-update -T <Task_ID> -H

地域データ更新ユーティリティでは、更新情報を確認、ダウンロードするためインターネットへの接続が必要となります。 このユーティリティでは、ポート 80 を使用して更新情報を提供するウェブサーバーと通信します。 ファイアウォールやプロキシ サーバーを経由して更新する場合、問題が発生する可能性があります。 問題が発生した場合は、ネットワーク管理者にお問い合わせください。
