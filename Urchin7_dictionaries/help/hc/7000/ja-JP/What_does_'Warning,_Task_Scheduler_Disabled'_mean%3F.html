「警告 タスク スケジューラー 無効」とはどういう意味ですか。

Urchin タスク スケジューラとは、Urchin 処理の実行ファイルを実行するためのサービスです。 タスク スケジューラが無効になっていると、Urchin はジョブを実行することができません。 エラー: 「警告 タスク スケジューラー 無効」" に共通の原因:

  • urchind サービスを実行していない場合。 これは、サーバーの再起動中に、Urchin がシステム起動時のスクリプトに含まれていない場合に起こります。
  • (Unix のみ) 実行中の Urchind サービスが、Urchin の所有者ではない別のユーザーにより所有されている場合。

スケジューラが実行中かどうかを確認するには、下記のタスクを行なって下さい:

  • コマンド シェルを起動し、/path/to/urchin/bin/ ディレクトリに移動します。
  • 下記のように入力します: urchinctl status (Unix の場合、./urchinctl status を入力)

スケジューラ サービスが実行中でない場合、コマンド ラインから下記のタスクを行って開始します:

  • コマンド シェルを起動し、/path/to/urchin/bin/ ディレクトリに移動します。
  • 下記のように入力します: urchinctl -s start (Unix の場合、./urchinctl -s start と入力)
  • サービスが両方とも実行中でない場合、下記のように入力します: urchinctl start

サービスが実行中の場合、Urchin 処理の所有者を特定するため、下記のようにサーバー上で実行プロセスを確認します: ps -ef | grep urchind

出力された結果で、処理を所有する "ユーザー名" を確認します: 

ユーザー名33597 0.0 0.2 352 204 ?? Ss 27Feb08 17:42.18 /usr/local/urchin6/bin/urchind

処理を所有するユーザーが Urchin の所有者と異なる場合、サービスて停止し、正しいユーザーで再開する必要がございます。 これは、urchind 処理を所有するユーザーに"ユーザーを切り替え"、urchinctl を停止することで行なえます。 上記と同様の手順で "-s" 切り替え機能を使用してください。 下記に、再び Urchin ユーザーに切り替え、再度 "-s" 切り替え機能を使用して urchinctl を開始してください。
