UNC ログ ソースの設定

Urchin で UNC ログ ソースを作成する場合は、次のパラメータを指定します。 Server Name/IP, Remote Server Login, Local Server Login, Remote Log Source File.

Urchin が UNC ソースからログを入手する手順は次のとおりです。 - Urchin は "Remote Log Source File" で指定されているファイルの参照を試みます。 - 参照に失敗した場合、Urchin は "Server Name/IP" フィールドに指定されているデータを使って、UNC 共有への接続を作成します。 - 接続に失敗した場合、Urchin はその原因をローカル システム アカウントなど、ネットワークにアクセスできないユーザー アカウントで実行しているためだとみなします。 そのため、Urchin は "Local Server Login" フィールドで指定されているユーザー アカウントでログイン セッションを作成し、このアカウントで Urchin プロセスを起動するなどして、このユーザー アカウントになりすまします。 "Same LocalとRemote Login" オプションを "はい" に設定している場合、なりすましの認証情報は "Remote Server Login" から取得されます。 - なりすましが成功すると Urchin は再びネットワーク共有への UNC 接続を作成し、参照を再度試みます。 - Urchin がログ ソースを入手したら、Urchin プロセスをなりすまし前に実行していたアカウントで実行するなどして、UNC 接続を閉じてなりすましを終了します。

なりすましには次の欠点があります。 - なりしましを継続すると、Urchin プロセスは異なるアカウントで実行され続けるため、Urchin データ ディレクトリにアクセスできず、レポート データベースの作成に失敗する可能性があります。 - レポート データベースは、なりすまし実行中に作成されるため、なりすまし終了時には Urchin プロセスにアクセスできなくなる可能性があります。 - Urchin ではファイルごとになりすましを開始/終了する必要があるため、特に複数の UNC ログ ファイルをプロファイルに追加する場合は、パフォーマンスが低下します。

このため、Urchin で UNC 共有を使用する場合の最もよい方法は、次のとおりです。 - ネットワークへアクセス可能なアカウントで Urchin を実行します。 この場合、なりすましは起こりません。 - UNC ログ共有への読み取り許可を Urchin の実行アカウントに与えます。 これによって、UNC 接続の作成となりすましが不要になります。
