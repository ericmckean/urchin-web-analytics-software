管理ユーティリティの概要

概要

Urchin には診断や設定に役立つユーティリティ プログラムが用意されています。 ユーティリティ プログラムは Urchin の util ディレクトリにございます。 このガイドでは、これらのユーティリティの概要について説明します。 使用方法をすべて説明しているわけではございませんのでご注意ください。

ユーティリティはすべて、コマンド ラインの引数「-h」と「-v」をサポートしています。 「-h」引数を使用してユーティリティを呼び出すと、使用方法の概要とユーティリティで使用できるオプションが表示されます。 「-v」引数を使用して呼び出すと、ユーティリティの Urchin バージョンが出力されます。

geo-update

Urchin の内部 DNS データベースの更新を確認し、更新がある場合はダウンロードします。 Google ダウンロード サーバーやローカル ディレクトリ (urchin 6 の場合) から更新情報をダウンロードできます。

inspector

インストールした Urchin の基本チェックを行い、Urchin の全体構造が破損していないこと、製品に同梱されているすべてのバイナリが適切なバージョンであること、UNIX プラットフォームの場合は基本のパーミッションが正しいことを確認します。 また、Urchin プロファイル スケジューラ (urchind)、CPC スケジューラ (urchincpcd, urchincpc)、バンドルされている Apache ウェブサーバー (urchinwebd) のオペレーション ステータス レポートを作成したり、 ライセンス情報の確認も行います。

uconf-manager (Urchin 6.402+)

Urchin 設定マネージャ ユーティリティ (uconf-manager) では、データベース レコードを追加、変更、削除します。

  • Urchin 6.402: このユーティリティは Urchin 6.402 で初めて導入され、設定データベースの 'uusers' テーブルのみを対象としていました。 このユーティリティを使用して、Urchin の管理者パスワードをリセットする、ユーザーを自動作成する、ユーザーを別のアカウントに移動することなどができます。
  • Urchin 6.500+: urchin 6.5 以降のバージョンでは、より多くのテーブル (uaccounts, ucosts, ufilters, ugoals, ugroups, ugroups_uusers, ulogs, ulogs_ufilters, uprofiles, uprofiles_ucosts, uprofiles_ufilters, uprofiles_ugroups, uprofiles_ulogs, uprofiles_uusers, uusers) に対してデータベース処理を実行できます。 このユーティリティは、Urchin 管理インターフェースでは不可能な処理に使用できます。

uconf2sql

Urchin の設定情報は、Urchin 5 以前のバージョンでは適切なファイルに保存されますが、Urchin 6 ではリレーショナル データベース (MySql または PostgreSql) に保存されます。 設定情報をスムーズに移行するには、Urchin 5 と Urchin 6 の提供する各ユーティリティを使用して設定情報のエクスポートとインポートを行います。Urchin 5 の uconf-export ユーティリティを使用して設定データベースを XML 形式のファイルにエクスポートしてから、Urchin 6 の uconf2sql ユーティリティを使用して XML ファイルの設定データを Urchin 6 のデータベースにインポートします。

convert-u5data

設定データベースと同様に、Urchin 6 ではレポート情報を保存しているレポート作成データベースの構造も変更されました。 convert-u5data ユーティリティには、レポート作成データベースを Urchin 5 から Urchin 6 に直接インポートするオプションがございます。

udb-sanitizer

udb-sanitizer ユーティリティでは、レポート作成データベースの管理に関する操作を実行します。 たとえばアーカイブ データベースを元に戻したり、再処理が必要な場合に特定の日付や期間の情報を削除することができます。

urchinctl

(注: このユーティリティは util ディレクトリではなく、bin ディレクトリにございます。) このユーティリティでは、Urchin スケジューラと Urchin ウェブサーバー サービスを開始または停止します。 UNIX システムでは、通常 urchinctl はシステム起動時のスクリプトから呼び出され、Urchin サービスを自動的に開始または停止します。 windows システムの場合は、urchin サービスのインストールとアンインストールのオプションも提供されます。

urchincpc (Urchin 6.5+)

(注: このユーティリティは util ディレクトリではなく、bin ディレクトリにございます。) Urchin CPC データ インポート ツールでは、Google AdWords API の CPC データを抽出し、詳細な処理とレポート作成を行います。 詳細については、クリック単価データのインポート ユーティリティをご覧ください。

urchincpcd (Urchin 6.5+)

(注: このユーティリティは util ディレクトリではなく、bin ディレクトリにございます。) Urchin CPC ソース スケジューラ (または CPC スケジューラ) では、プロファイル スケジューラと同様にユーザー定義のスケジュールに基づいて CPC ダウンロード タスクのスケジュールを設定します。 詳細については、クリック単価データのインポート ユーティリティをご覧ください。

urchin_daemons

Urchin は UNIX コンピュータ上でスケジューラ デーモン (urchind) と Apache ウェブサーバー (urchinwebd) を実行します。 これらのプログラムは起動時に開始する必要がございます。 urchin_daemons スクリプトは、システムの初期化スクリプト (保存場所はご使用の UNIX OS により異なります) に追加して、Urchin サービスのデーモンを正しく起動できます。 windows システムの場合、これらのバイナリはシステムのサービスとして実行され、windows サービス マネージャから開始または停止することができます。

ugetlicense

Urchin ライセンス操作ユーティリティ (ugetlicense) では、シリアル番号に対する操作を実行できます。 これらの操作は、現在の Urchin インストールが使用できなくなり、再インストールや別のサーバーへの移行が必要になった場合に必要となります。
