Urchin のアクセスを 1 つの IP アドレスに制限する

概要

複数のネットワーク インターフェースを使用しているシステムや、複数の IP アドレスを 1 つのネットワーク インターフェースに関連付けているシステムの場合、デフォルトで Urchin のウェブ インターフェースは設定されているすべての IP アドレスをバインドします。 ただし、Urchin の管理とレポート操作を 1 つの IP アドレスに制限した方がよい場合もあります。

手順

Urchin の管理とレポート操作を制限するには、Urchin Apache ウェブサーバーが使用する設定ファイルに対して、簡単な変更を行う必要があります。 方法は次のとおりです。

UNIX システムの場合:

  • Urchin を実行するユーザーでコマンド シェルを開きます。
  • 「cd」コマンドを使用して、Urchin がインストールされているディレクトリに移動します。
  • 「bin/urchinctl -w stop」コマンドを使用して、Urchin ウェブサーバーを停止します。
  • テキスト エディタを使用して、Urchin の「var」ディレクトリにある「urchinwebd.conf.template」ファイルを開きます。

  • <Directory /> 行の上に次の行を挿入します。 

       Listen XXX.XXX.XXX.XXX:port
  • 「XXX.XXX.XXX.XXX」と「port」には、それぞれ Urchin アクセスを許可する IP アドレスとポート番号を指定します。
  • 次のコマンドを実行して Urchin ウェブサーバーを再起動します。 

bin/urchinctl -w start

Windows システムの場合:

  • [スタート]、[プログラム]、[Urchin 6]、[Urchin サービスの無効化] の順にクリックして、Urchin サービスを停止します。
  • Windows エクスプローラで「C:\Program Files\Urchin6\var」に移動します。
  • テキスト エディタを使用して「urchinwebd.conf.template」ファイルを開きます。

  • <Directory /> 行の上に次の行を挿入します。 

       Listen XXX.XXX.XXX.XXX:port
  • 「XXX.XXX.XXX.XXX」と「port」には、それぞれ Urchin アクセスを許可する IP アドレスとポート番号を指定します。
  • [スタート]、[プログラム]、[Urchin 6]、[Urchin サービスの有効化] の順にクリックして、Urchin サービスを再起動します。

注: IIS 5 以降では、ソケット プールの使用により、場合によっては、実際の IP アドレスの設定状況に関係なく、IIS のデフォルトの動作を無効にして、特定のポートにバインドされているすべてのウェブサイトでソケットを共有できるようにする必要があります。 この設定は仮想ホストを使用している環境では一般的で、ポート 80 で個別の仮想ホストの Urchin レポートを設定できます。

アドバイス

  • この方法は、Urchin の管理とレポート操作を特定の IP アドレスに制限する場合にのみ有効です。 名前ベースの仮想ホストへのアクセスを制限することはできません。
  • Urchin をアップグレードすると、「urchinwebd.conf.template」ファイルはバックアップ コピーを保存してから上書きされます。 ファイルを変更した場合は、アップグレード後にもう一度その変更を行う必要があります。
