広告などのリンクに設定している独自のキャンペーン トラッキング変数を Urchin の変数の代わりに使用する

オンライン リンクに独自のトラッキング変数を設定している場合、この既存の変数を Urchin のキャンペーン トラッキング変数に置き換えて使用できます。 タグ付きリンクには 3 つ以上の変数を設定する必要があり、 次の変数に置き換えられます。

  1. utm_source: キャンペーンのソース (Google、Yahoo など)
  2. utm_medium: キャンペーンのメディア (クリック単価、バナー、メールなど)
  3. utm_campaign: キャンペーンの名前 (「春のセール」など)

たとえば、サイトに次のような URL があるとします。 

            www.example.com/index.html?camp=GADS&kwd=memory&version=47&type=ppc&adver=google

ステップ 1: 独自の変数と Urchin の変数を関連付けるため、 下記の例のようなフォーマットで表を作成します。 

               キャンペーン     広告ソース         広告の種類     キーワード   A/B コンテンツ  マスター
ユーザー コード=    camp           adver       type        kwd       version      
Urchin コード=  utm_campaign utm_source    utm_medium  utm_term  utm_content  utm_id

ステップ 2: UTM ファイルを編集します。

  • urchin.js ファイルをテキスト エディタで開き、 ファイルの上部にあるユーザー定義セクションで、次の utm 変数を探します。 

                //-- UTM Campaign Tracking Settings
                var _uctm=1;
                var _ucto="15768000";
                var _uccn="utm_campaign";
                var _ucmd="utm_medium";
                var _ucsr="utm_source";
                var _uctr="utm_term";
                var _ucct="utm_content";
                var _ucid="utm_id";
                var _ucno="utm_nooverride";
  • 上の例のタグ付きリンクを基に、このセクションを次のように編集します。 

                //-- UTM Campaign Tracking Settings
                var _uctm=1;
                var _ucto="15768000";
                var _uccn="camp";
                var _ucmd="type";
                var _ucsr="adver";
                var _uctr="kwd";
                var _ucct="version";
                var _ucid="utm_id";
                var _ucno="utm_nooverride";

ステップ 3: 新しいトラッキング変数のフィルタを作成します。

  • 各フィルタでは、次の基本的なフィルタ設定を使用します。 

                フィルタの種類:  アドバンス
                フィールド A:  utm_campaign_uri (raw)
                引用 A:  new_variable =([^&]*)
                フィールド B:  (なし)
                引用 B:  (空欄)
                出力先:  utm _( old_variable )
                構成:  $A1
                フィールド A は必須:  はい
                フィールド B は必須:  いいえ
                出力フィールドを上書き:  はい
                大文字と小文字の区別:  いいえ
  • old_variable はデフォルトの Urchin トラッキング変数、 new_variable はユーザー独自のトラッキング変数です。 修正した変数ごとにフィルタを作成する必要があります。
