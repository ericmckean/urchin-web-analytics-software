この記事の内容は Windows Server 2008 と Windows Vista に Urchin をインストールする際に適用されます。 Urchin インストール ガイド (Windows) の手順は Windows Server 2008 と Windows Vista で実行できますが、下記の点に留意する必要がございます。

Windows 2008 にはユーザー アカウント コントロール (UAC) と呼ばれる新しいセキュリティ機能がございます。詳細につきましては、下記の URL をご覧ください: http://technet.microsoft.com/en-us/library/cc709691.aspx

UAC を有効にした場合の既知の問題は下記のとおりです。

  1. UAC が有効な場合、アプリケーションで「Program files」フォルダに正しく書き込めません。 詳細につきましては、下記の URL をご覧ください: http://msdn.microsoft.com/en-us/library/aa905330.aspx

  2. Urchin の実行ファイルは、デフォルトではコマンド プロンプトから実行できません。 管理者としてコマンド プロンプトを使用するか、実行ファイルのプロパティを下記のように設定する必要がございます。 右クリック後、[プロパティ] -> [互換性] -> [このプログラムを管理者として実行] -> [OK] と選択します。 Urchin Windows サービスは追加の作業を必要とせずに機能します。

対処方法:

  1. UAC を無効にして、希望の場所に Urchin を設定します。 バイナリまたは Windows サービスを使用して Urchin を実行します。
  2. UAC が有効な場合、Urchin を [Program files] フォルダにインストールしないで、 「ユーザー」 ディレクトリなどにインストールしてください。 プロパティを変更して管理者モードのバイナリで実行するか （上記の 2 の説明を参照)、何も変更しないで Windows サービスを使用します。
