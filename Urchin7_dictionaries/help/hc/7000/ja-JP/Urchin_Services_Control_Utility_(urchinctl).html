Urchin サービス コントロール ユーティリティ (urchinctl)

概要

urchinctl は、Urchin スケジューラ と Urchin ウェブサーバー サービスのステータス確認、開始、停止を行なうための Urchin サービス コントロール ユーティリティ です。 UNIX システムでは、通常 urchinctl はシステム起動時のスクリプトから呼び出され、Urchin サービスを自動的に開始または停止します。

urchinctl は下記の処理を行います。

  • Urchin 6 マスターとスレーブ スケジューラのステータスの確認
  • スケジューラ、またはウェブサーバー (またはその両方) の開始、停止、および再起動
  • 異なるポートでのウェブサーバーの開始
  • SSL による暗号化を用いたウェブサーバーの開始

使用方法

urchinctl は Urchin ディストリビューションの bin ディレクトリにございます。

このユーティリティの使用方法は下記のようになります。 

  urchinctl -h (使用方法を出力して終了します)

  urchinctl -v (バージョンを出力して終了します)

  urchinctl [-e] [-p port] [[-s] | [-w]] action

各オプションの説明は以下のとおりです。 

  -e  ウェブサーバーでの暗号化 (SSL) を有効化します

  -p  ウェブサーバーのポートを指定します

  -s  Urchin スケジューラでのみ操作を実行します

  -w  Urchin ウェブサーバーでのみ操作を実行します

操作とは、下記のいずれかを指します。 

  start   (サービスを開始します)

  stop   (サービスを停止します)

  restart   (サービスを再起動します)

  status  (ウェブサーバー、スケジューラの実行時間のステータスを表示します)

デフォルトでは、ウェブサーバーとスケジューラの両方で操作が実行されるように設定されています。いずれか一方のみで実行する場合には、"-s" (スケジューラ) または "-w" (ウェブサーバー) オプションを指定してください。 この 2 つのオプションは併用できません。

アドバイス

UNIX システムで Urchinctl を実行する場合、Urchin がインストールされたユーザー、UID を使用し、urchinwebd と urchind Urchin のプロセスと同じ UID で開始されるようにします。
