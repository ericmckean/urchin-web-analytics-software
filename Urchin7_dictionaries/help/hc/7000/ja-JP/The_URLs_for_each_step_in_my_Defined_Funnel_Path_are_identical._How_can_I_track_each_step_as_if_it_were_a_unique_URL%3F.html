定義した目標到達プロセス パスの各ステップで同じ URL を使用する場合に 各ステップを固有の URL としてトラッキングする

目標到達プロセスの各ステップで同じ URL を使用する場合でも、ユーザーの行動をトラッキングすることができます。 たとえば、登録の目標到達プロセスが下記のような URL であるとします。

  • ステップ 1 (登録)
    • www.mysite.com/sign_up.cgi
  • ステップ 2 (契約に同意)
    • www.mysite.com/sign_up.cgi
  • ステップ 3 (終了)
    • www.mysite.com/sign_up.cgi

対処方法としては、次のように各ステップの onload イベントで urchinTracker Javascript 関数を呼び出します。

  • urchinTracker("/funnel_G1/step1.html")
  • urchinTracker("/funnel_G1/step2.html")
  • urchinTracker("/funnel_G1/step3.html")

次に、目標到達プロセスの設定で各ステップを次のように設定します。

  • http://www.mysite.com/funnel_G1/step1.html

  • http://www.mysite.com/funnel_G1/step2.html

  • http://www.mysite.com/funnel_G1/step3.html

urchinTracker() に渡す path/filename 引数には、必ずしも既存のパスまたはファイル名を使用する必要はありません。 urchinTracker に渡す引数は、Urchin でページビュー数を集計するための架空のページ名となります。
