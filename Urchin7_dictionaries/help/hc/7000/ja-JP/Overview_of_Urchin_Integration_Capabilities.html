Urchin の統合機能の概要

はじめに

Urchin のフロント エンドは、モジュラー コンポーネントと業界標準技術で構築されており、管理者は既存または導入予定のインフラストラクチャに Urchin レポートへのアクセスを統合できます。 ホスティング プロバイダやデータ センターをサポートするため、できるだけ多くのアーキテクチャで Urchin の統合が可能となるよう、3 つの統合のポイントについてテストを行い、対応に取り組んでいます。

次の図は、Urchin のフロント エンドと 3 つの統合のポイントの主な要素を示しています。 管理とレポート作成の機能は、このシステムからウェブ ベースで提供されます。

Urchin システムのコンテンツはすべて、urchinwebd としてインストールされる Apache サーバー (図の左側) を介して配信されます。 リクエストは Apache サーバーによって処理され、CGI インターフェースを使用してセッション制御アプリケーション (session.cgi) に引き渡されます。 セッション制御では、認証を実行し、操作内容に応じてリクエストを管理エンジン (管理者) または Urchin のレポート エンジン (urchin.cgi) に引き渡します。 セッション ID が埋め込まれているコンテンツはユーザーに返されます。

A、B、C のポイントが、上述した 3 つの統合のポイントになります。 A のポイントでは、管理者は Urchin ウェブサーバーを変更または迂回することができます。 B のポイントでは、外部認証または認証なしを選択できます。 C のポイントでは、ラッパーまたはポータルを介してレポートに直接アクセスすることができます。

ポイント A: ウェブサーバーの統合

多くのホスティング会社では、実行中のウェブサーバーを使用しているか、自社システム内で実行する特別にコンパイルしたウェブサーバーを使用しています。 ウェブサーバーで基本的な CGI を処理できる場合は、Apache バイナリを変更したり、既存のウェブサーバーを使用して Urchin にアクセスすることができます。

この統合のポイントによって、Urchin システムの他の部分が影響を受けることはありません。 管理、認証、レポート配信には Urchin インターフェースが引き続き使用され、 ユーザーのシステム アクセス時に、Urchin がアクセスを許可するレポートを特定できるよう、Urchin システムでユーザーとプロファイルを設定する必要があります。

ポイント B: 外部認証または認証バイパスの使用

複雑なホスティング環境では、多くの場合 LDAP などのユーザー認証を行う一元管理機能を備えています。 設定を若干変更するだけで、セッション制御から指定した認証ルーチンを呼び出すことができます。 ログインの成功または失敗を返すためのシンプルなインターフェースが提供され、 既存の認証ルーチンを簡単にラップして、両システムに応じたフレームワークを提供できます。 B の統合のポイントでは、ダミーのルーチンを使用して認証を迂回し、リンク先ページに直接リンクすることもできます。

B の統合のポイントを使用すると、認証が外部アプリケーションによって実行される以外に Urchin システム全体への影響はありません。 ユーザーのシステム アクセス時に、Urchin がアクセスを許可するレポートを特定できるよう、Urchin システムでユーザーとプロファイルを設定する必要があります。 外部認証を使用するためのインターフェースの詳細と認証の回避例については、外部認証または認証バイパスの使用をご覧ください。

ポイント C: ラッパーまたはポータルからレポートへの直接リンク

ほとんどの場合、トラッキング対象のウェブサイトは １ つだけで、ウェブサイトの管理セクションからそのウェブサイトに関するレポートに直接リンクすることが多くあります。 通常、管理セクションは認証済みの領域にあります。 C の統合ポイントでは、ラッパー スクリプトまたは既存のポータルを使用して、レポートへの直接リンクを簡単に設定することができます。 この場合、Urchin 認証と最初のレポート選択画面はスキップされ、Urchin レポートがユーザーに直接表示されます。

C の統合ポイントを使用することで、サービス プロバイダは顧客とウェブサイトの関係または既存の設定データベースに基づき、ユーザーとその表示内容を管理しているとみなされます。Urchin では引き続き、プロファイル レポートを生成するための設定が必要となりますが、これは管理インターフェースまたは他の方法で自動的に行うことができます。Urchin では管理インターフェースで [レポートの直接リンク] が有効になっている場合、特定の URL で Urchin レポートへの直接リンクを設定できます。 Urchin レポートへのアクセスは、この URL により指定されるディレクトリの .report.conf ファイルによって制御されます。

複雑なポータル統合環境では、Urchin のレポート エンジンは、セッションと他のポータル変数を転送して、セッション処理を継続します。
