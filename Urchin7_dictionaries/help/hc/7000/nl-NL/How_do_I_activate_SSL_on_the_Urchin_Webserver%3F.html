Hoe activeer ik SSL op de Urchin-webserver?

De Urchin-webserver die wordt verzonden met Urchin, kan communicatie coderen via SSL. Om SSL in te schakelen, heeft u een geldig, door een certificeringsinstantie ondertekend certificaat of een zelfondertekend certificaat nodig.

SSL inschakelen in de Urchin-webserver:

  1. Kopieer uw SSL-certificaatbestand naar de directory 'var' van Urchin en geef het bestand de naam server.crt.
  2. Kopieer uw SSL-sleutelbestand naar de directory 'var' van Urchin en geef het bestand de naam server.key.
  3. Bewerk het bestand urchinwebd.conf.template dat staat in de directory 'var' van Urchin.
    1. (Sla deze stap over als u Urchin 6.6+ gebruikt.) Uitcommentarieer de regel Port en ontcommentarieer de opdracht Listen. 
                #XXXPortXXX
          Listen 443

    2. Wijzig de ServerName-opdracht van localhost in de naam van uw webserver. Bijvoorbeeld: 

                ServerName: www.urchin.com

    3. OPMERKING: de ServerName in het bestand urchinwebd.conf.template moet overeenkomen met de naam van de server die in het certificaatbestand wordt vermeld.

  4. Start de webserver (opnieuw) met urchinctl met de optie '-e'. Urchinctl bevindt zich in de directory 'bin' van Urchin. De optie '-e' geeft urchinctl de opdracht om SSL in de webserver in te schakelen. Als u de webserver bijvoorbeeld opnieuw wilt starten met SSL ingeschakeld via poort 443, gebruikt u: Voor versies lager dan Urchin 6.6: 
       urchinctl -e -w restart
    Voor Urchin 6.6+: 
       urchinctl -e -p 443 -w restart

    Opmerking: om de server te starten zonder SSL ingeschakeld, verwijdert u de optie '-e' uit de opdracht.

U heeft nu toegang tot de server waarop SSL is ingeschakeld met https://servername.domain.com:port/

Opmerking: als u de SSL-instellingen in urchinwebd.conf.template aanpast, kan dit problemen veroorzaken waardoor de webserver niet kan worden gestart.

Instructies voor Windows

SSL inschakelen in de Urchin-webserver:

  1. Kopieer uw SSL-certificaatbestand naar de directory 'var' van Urchin en geef het bestand de naam server.crt.
  2. Kopieer uw SSL-sleutelbestand naar de directory 'var' van Urchin en geef het bestand de naam server.key.
  3. Bewerk het bestand urchinwebd.conf.template dat staat in de directory 'var' van Urchin.
    1. (Sla deze stap over als u Urchin 6.6+ gebruikt.) Uitcommentarieer de regel Port en ontcommentarieer de opdracht Listen. 
                #XXXPortXXX
          Listen 443
    2. Ontcommentarieer de volgende regel om mod_ssl.so te laden wanneer de webserver wordt gestart: 
                LoadModule ssl_module etc/mod_ssl.so

    3. Wijzig de ServerName-opdracht van localhost in de naam van uw webserver. Bijvoorbeeld: 

                ServerName: www.urchin.com

    4. OPMERKING: de ServerName in het bestand urchinwebd.conf.template moet overeenkomen met de naam van de server die in het certificaatbestand wordt vermeld.

  4. Verwijder en installeer de webserverservice die urchinctl gebruikt met de optie '-e'. Urchinctl bevindt zich in de directory 'bin' van Urchin. De optie '-e' geeft urchinctl de opdracht om SSL in de webserver in te schakelen. Als u de webserver bijvoorbeeld opnieuw wilt verwijderen en installeren met SSL ingeschakeld via poort 443, gebruikt u: Voor versies lager dan Urchin 6.6: 
       urchinctl.exe -w remove
   urchinctl.exe -w -e install
    Voor Urchin 6.6+: 
       urchinctl.exe -w remove
   urchinctl.exe -e -p 443 -w install

    Opmerking: om de server te starten zonder SSL ingeschakeld, verwijdert u de optie '-e' uit de opdracht.

U heeft nu toegang tot de server waarop SSL is ingeschakeld met https://servername.domain.com:port/

Opmerking: als u de SSL-instellingen in urchinwebd.conf.template aanpast, kan dit problemen veroorzaken waardoor de webserver niet kan worden gestart.
