Lijst met foutcodes voor mislukte logbestandsoverdrachten van externe webservers via FTP of HTTP

Overzicht

U kunt de bron voor een logbestand configureren in Urchin zodat er een webserverlogbestand wordt opgehaald van een externe server via FTP of HTTP. Onder normale omstandigheden slaagt deze overdracht en worden er geen fouten vermeld in het logbestand van de uitvoer. Als er echter een fout optreedt tijdens de overdracht (bijvoorbeeld: de combinatie gebruikersnaam/wachtwoord is ongeldig, de externe server is niet bereikbaar, het externe logbestand kan niet worden gelezen, enzovoort), registreert Urchin een foutcode in de runtime-uitvoer, die u kunt bekijken in het taakoverzicht van het profiel. Deze foutcode wordt tussen haakjes weergegeven bij het foutbericht na de poging tot logbestandsoverdracht met de webserver, zoals (9).

De foutcodes worden hieronder vermeld, samen met een beschrijving van het opgetreden probleem.

Lijst met foutcodes

CURLE_UNSUPPORTED_PROTOCOL (1) - De URL die u heeft doorgegeven aan 'libcurl', gebruikt een protocol dat deze 'libcurl' niet ondersteunt. De ondersteuning betreft mogelijk een optie voor het compileren die u niet heeft opgegeven, een verkeerd gespelde protocoltekenreeks of een protocol waarvoor 'libcurl' niet over code beschikt.

CURLE_FAILED_INIT (2) - De initialisatiecode is vrijwel meteen mislukt. Dit betreft waarschijnlijk een interne fout of probleem.

CURLE_URL_MALFORMAT (3) - De URL is niet correct opgemaakt.

CURLE_COULDNT_RESOLVE_PROXY (5) - Kan de proxy niet vaststellen. De opgegeven proxyhost kan niet worden vastgesteld.

CURLE_COULDNT_RESOLVE_HOST (6) - Kan de host niet vaststellen. De opgegeven externe host kan niet worden vastgesteld.

CURLE_COULDNT_CONNECT (7) - Kan geen verbinding maken() met de host of proxy.

CURLE_FTP_WEIRD_SERVER_REPLY (8) - Na het maken van verbinding met een FTP-server, verwacht 'libcurl' een bepaald antwoord. Deze foutcode duidt erop dat het antwoord onverwacht of onjuist was. De opgegeven externe server is waarschijnlijk geen goede FTP-server.

CURLE_REMOTE_ACCESS_DENIED (9) - Er was geen toegang mogelijk tot de bron die is opgegeven in de URL. Dit kan bij FTP voorkomen bij het overschakelen naar de externe directory.

CURLE_FTP_WEIRD_PASS_REPLY (11) - Na het verzenden van het FTP-wachtwoord naar de server verwacht 'libcurl' een correct antwoord. Deze foutcode geeft aan dat een onverwachte code is geretourneerd.

CURLE_FTP_WEIRD_PASV_REPLY (13) - 'libcurl' krijgt geen zinvol resultaat van de server in reactie op een PASV- of EPSV-opdracht. Er is een probleem met de server.

CURLE_FTP_WEIRD_227_FORMAT (14) - FTP-servers retourneren een 227-regel als reactie op een PASV-opdracht. Als 'libcurl' deze regel niet kan verwerken, wordt deze reactiecode geretourneerd.

CURLE_FTP_CANT_GET_HOST (15) - Er is een interne fout opgetreden bij het opzoeken van de host die wordt gebruikt voor de nieuwe verbinding.

CURLE_FTP_COULDNT_SET_TYPE (17) - Er is een fout opgetreden bij het instellen van de overdrachtsmodus als binair of ascii.

CURLE_PARTIAL_FILE (18) - De overdracht van een bestand duurde korter of langer dan verwacht. Dit gebeurt wanneer de server eerst een verwachte grootte van de bestandsoverdracht opgeeft en vervolgens gegevens levert die niet overeenkomen met de eerder opgegeven grootte.

CURLE_FTP_COULDNT_RETR_FILE (19) - Dit was een onverwachte reactie op een 'RETR'-opdracht of een bestandsoverdracht van nul bytes.

CURLE_QUOTE_ERROR (21) - Bij het verzenden van aangepaste "QUOTE"-opdrachten naar de externe server retourneerde een van de opdrachten een foutcode van 400 of hoger (voor FTP) of gaf deze op een andere manier het mislukken van de opdracht aan.

CURLE_HTTP_RETURNED_ERROR (22) - Deze foutcode wordt geretourneerd als CURLOPT_FAILONERROR is ingesteld op 'TRUE' en de HTTP-server een foutcode retourneert van 400 of hoger. (Deze foutcode was vroeger bekend als CURLE_HTTP_NOT_FOUND.)

CURLE_WRITE_ERROR (23) - Er is een fout opgetreden bij het schrijven van ontvangen gegevens naar een lokaal bestand, of er is een fout opgetreden in 'libcurl' bij het aanroepen van een schrijfaanroep.

CURLE_UPLOAD_FAILED (25) - Het uploaden kan niet worden gestart. Bij FTP is de oorzaak meestal dat de server de STOR-opdracht heeft geweigerd. De foutbuffer bevat vaak de uitleg die de server hierover kan bieden. (Deze foutcode was voorheen bekend als CURLE_FTP_COULDNT_STOR_FILE.)

CURLE_READ_ERROR (26) - Er is een fout opgetreden bij het lezen van een lokaal bestand of bij het aanroepen van de leesopdracht.

CURLE_OUT_OF_MEMORY (27) - Een verzoek tot het toewijzen van geheugen is mislukt. Dit is een ernstig probleem en er is waarschijnlijk heel wat aan de hand als deze fout optreedt.

CURLE_OPERATION_TIMEDOUT (28) - Time-out tijdens de bewerking. De opgegeven time-outperiode is bereikt volgens de voorwaarden.

CURLE_FTP_PORT_FAILED (30) - De FTP-opdracht PORT heeft een fout geretourneerd. Dit gebeurt meestal wanneer u het adres dat 'libcurl' moet gebruiken, niet duidelijk genoeg heeft opgegeven. Zie CURLOPT_FTPPORT.

CURLE_FTP_COULDNT_USE_REST (31) - De FTP-opdracht REST heeft een fout geretourneerd. Dit zou niet moeten gebeuren als de server goed is ingesteld.

CURLE_RANGE_ERROR (33) - De server ondersteunt of accepteert geen bereikverzoeken.

CURLE_HTTP_POST_ERROR (34) - Dit is een vreemde fout, die meestal optreedt vanwege een interne communicatiefout.

CURLE_SSL_CONNECT_ERROR (35) - Er is een probleem opgetreden in de communicatie tussen SSL en TLS. U kunt het beste de foutmelding in de foutbuffer bekijken, omdat de fout hierin nauwkeuriger wordt aangegeven. Deze kan worden veroorzaakt door certificaten (bestandsextensies, paden, rechten), wachtwoorden, enzovoort.

CURLE_FTP_BAD_DOWNLOAD_RESUME (36) - Poging tot hervatten van FTP-overdracht vanaf een byte-aantal groter dan het bestandsformaat.

CURLE_FILE_COULDNT_READ_FILE (37) - Een bestand dat is opgegeven met 'FILE://' kan niet worden geopend. Dit komt waarschijnlijk omdat het bestandspad niet naar een bestaand bestand verwijst. Heeft u de rechten voor het bestand gecontroleerd?

CURLE_LDAP_CANNOT_BIND (38) - LDAP kan niet binden. De bindingsbewerking voor LDAP is mislukt.

CURLE_LDAP_SEARCH_FAILED (39) - Het zoeken met LDAP is mislukt.

CURLE_FUNCTION_NOT_FOUND (41) - Functie niet gevonden. Een vereiste 'zlib'-functie is niet gevonden.

CURLE_ABORTED_BY_CALLBACK (42) - Afgebroken door aanroep. Een aanroep heeft "abort" geretourneerd naar 'libcurl'.

CURLE_BAD_FUNCTION_ARGUMENT (43) - Interne fout. Er is een functie aangeroepen met een onjuiste parameter.

CURLE_INTERFACE_FAILED (45) - Interfacefout. Een opgegeven interface voor uitvoer kan niet worden gebruikt. Stel de interface die moet worden gebruikt voor het bron-IP-adres voor uitgaande verbindingen in met CURLOPT_INTERFACE. (Deze foutcode was vroeger bekend als CURLE_HTTP_PORT_FAILED.)

CURLE_TOO_MANY_REDIRECTS (47) - Te veel omleidingen. 'libcurl' heeft het maximale aantal omleidingen bereikt. U kunt het maximum instellen met CURLOPT_MAXREDIRS.

CURLE_UNKNOWN_TELNET_OPTION (48) - Een optie die is ingesteld met CURLOPT_TELNETOPTIONS is niet herkend/bekend. Raadpleeg de betreffende documentatie.

CURLE_TELNET_OPTION_SYNTAX (49) - Een tekenreeks voor de telnet-optie is onjuist opgemaakt.

CURLE_PEER_FAILED_VERIFICATION (51) - Het SSL-certificaat of de SSH md5-code van de externe server is onjuist.

CURLE_GOT_NOTHING (52) - De server heeft niets geretourneerd en dat wordt in de betreffende omstandigheden als een fout beschouwd.

CURLE_SSL_ENGINE_NOTFOUND (53) - De opgegeven codeermodule kan niet worden gevonden.

CURLE_SSL_ENGINE_SETFAILED (54) - De geselecteerde codeermodule voor SSL kan niet als standaard worden ingesteld.

CURLE_SEND_ERROR (55) - Het verzenden van netwerkgegevens is mislukt.

CURLE_RECV_ERROR (56) - Het ontvangen van netwerkgegevens is mislukt.

CURLE_SSL_CERTPROBLEM (58) - Er is een probleem opgetreden met het certificaat van de lokale client.

CURLE_SSL_CIPHER (59) - De opgegeven codering kan niet worden gebruikt.

CURLE_SSL_CACERT (60) - Het certificaat van de peer kan niet worden geverifieerd met bekende CA-certificaten.

CURLE_BAD_CONTENT_ENCODING (61) - De overdrachtscodering wordt niet herkend.

CURLE_LDAP_INVALID_URL (62) - Ongeldige LDAP-URL.

CURLE_FILESIZE_EXCEEDED (63) - De maximale bestandsgrootte is overschreden.

CURLE_USE_SSL_FAILED (64) - Het opgevraagde SSL-niveau van FTP is mislukt

CURLE_SEND_FAIL_REWIND (65) - Tijdens een verzendbewerking moet 'curl' de gegevens compatibel maken om ze opnieuw te verzenden, maar dit is mislukt.

CURLE_SSL_ENGINE_INITFAILED (66) - Het starten van de SSL-engine is mislukt.

CURLE_LOGIN_DENIED (67) - 'curl' krijgt geen toestemming van de externe server om zich aan te melden (toegevoegd in 7.13.1).

CURLE_TFTP_NOTFOUND (68) - Bestand niet gevonden op de TFTP-server.

CURLE_TFTP_PERM (69) - Er is een probleem opgetreden met de rechten voor de TFTP-server.

CURLE_REMOTE_DISK_FULL (70) - Onvoldoende schijfruimte op de server.

CURLE_TFTP_ILLEGAL (71) - Onjuiste TFTP-bewerking.

CURLE_TFTP_UNKNOWNID (72) - Onbekende id voor TFTP-overdracht.

CURLE_REMOTE_FILE_EXISTS (73) - Het bestand bestaat al en wordt niet overschreven.

CURLE_TFTP_NOSUCHUSER (74) - Deze foutcode zou nooit moeten worden geretourneerd door een correct functionerende TFTP-server.

CURLE_CONV_FAILED (75) - Het omzetten van tekens is mislukt.

CURLE_CONV_REQD (76) - De aanroepende instantie moet het aanroepen van omzettingen registreren.

CURLE_SSL_CACERT_BADFILE (77) - Er is een probleem opgetreden bij het lezen van het CA-certificaat voor SSL (pad? toegangsrechten?).

CURLE_REMOTE_FILE_NOT_FOUND (78) - De bron waarnaar in de URL wordt verwezen, bestaat niet.

CURLE_SSH (79) - Er is een niet nader omschreven fout opgetreden tijdens de SSH-sessie.

CURLE_SSL_SHUTDOWN_FAILED (80) - De SSL-verbinding kan niet worden verbroken.
