De opmerkingen in dit artikel zijn van toepassing op de installatie van Urchin in Windows Server 2008 en Windows Vista. Hoewel de stappen in Installatiehandleiding voor Urchin (Windows) werken voor Windows Server 2008 en Windows Vista, dient u op de volgende punten te letten.

Windows 2008 bevat een nieuwe beveiligingsfunctie met de naam ' Gebruikersaccountbeheer '. Ga naar http://technet.microsoft.com/en-us/library/cc709691.aspx voor meer informatie.

De volgende bekende problemen doen zich voor wanneer Gebruikersaccountbeheer is ingeschakeld:

  1. Als Gebruikerstoegangsbeheer is ingeschakeld, kan de toepassing niet correct schrijven naar de map 'Program Files'. Meer informatie hierover vindt u op: http://msdn.microsoft.com/en-us/library/aa905330.aspx

  2. De uitvoerbare bestanden van Urchin kunnen niet standaard worden uitgevoerd vanaf een opdrachtprompt. U moet de opdrachtprompt als beheerder uitvoeren of de eigenschappen van de uitvoerbare bestanden als volgt instellen: klik met de rechtermuisknop en selecteer 'Eigenschappen' -> 'Compatibiliteit' -> 'Dit programma als Administrator uitvoeren' -> 'OK'. De Windows-services van Urchin functioneren nu zoals verwacht zonder dat u hiervoor verder iets hoeft te doen.

Oplossing:

  1. Schakel Gebruikersaccountbeheer uit en installeer Urchin op een willekeurige locatie. Voer Urchin uit met de binaire bestanden of Windows-services.
  2. Als Gebruikersaccountbeheer is ingeschakeld, moet u Urchin NIET installeren in de map 'Program Files'. Installeer het programma bijvoorbeeld in de directory 'Users'. Wijzig de eigenschappen voor het uitvoeren van binaire bestanden in de beheermodus (zoals beschreven onder 2 hierboven) of gebruik de Windows-services zonder wijzigingen.
