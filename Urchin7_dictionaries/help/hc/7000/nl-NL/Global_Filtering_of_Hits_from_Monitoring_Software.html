Algemeen filteren van treffers afkomstig van software voor het controleren van gegevens

Overzicht

In de meeste hostingomgevingen is een bepaalde controlemethode beschikbaar voor webservers van klanten om SLA's (service level agreements) te kunnen controleren. Dit heeft echter een neveneffect: de treffers van deze controlesystemen kunnen gevolgen hebben voor de Urchin-rapportage voor de gecontroleerde websites, waardoor het aantal sessies, paginaweergaven, treffers en de byte-tellingen kunstmatig worden verhoogd.

Aanbeveling

In hostingomgevingen met dergelijke controlesystemen raden we u aan een algemeen Urchin-filter toe te passen op de geconfigureerde Urchin-profielen van elke klant om de treffers weg te filteren die zijn gegenereerd door de controlesoftware. U kunt dit gemakkelijk doen in een omgeving waarin een Urchin-installatie (beheerd door het hostingbedrijf) rapportagegegevens levert voor de website(s) van elke klant. In aangepaste omgevingen waarin de klanten zelf een instantie van Urchin op hun server(s) beheren, moet het hostingbedrijf een voorbeeldfilter ter beschikking stellen dat geschikt is voor het gebruikte type controlesoftware.

Voor hulp bij het implementeren van Urchin-filtering moeten de host en de klant samenwerken om een specifieke pagina te maken op de website van de klant die alleen wordt gebruikt door de controlesoftware, zoals: 

     http://www.customerdomain.com/healthpage.html

Voorbeelden

Voorbeeld 1: Filter het IP-adres voor het controlesysteem weg 

      Filter Type: Exclude
      Filter Field: IP
      Filter Spec: 172\.16\.1\.1
  • Hiermee worden alle treffers met het IP-adres 172.16.1.1 weggefilterd uit het logbestand van de webserver wanneer dit wordt verwerkt door Urchin.

Voorbeeld 2: Filter de specifieke pagina weg die het controlesysteem treft 

      Filter Type: Exclude
      Filter Field: REQUEST
      Filter Spec: ^/healthpage.html
  • Hiermee worden alle treffers met een verzoek voor /healthpage.html weggefilterd uit het logbestand van de webserver wanneer dit wordt verwerkt door Urchin.

Aandachtspunten

  1. U kunt eventueel aanvullende, ongefilterde profielen voor de klanten maken zodat ze de daadwerkelijke verkeersstroom op hun webserver(s) kunnen bekijken (inclusief de gefilterde gegevens).
  2. Het hostingbedrijf kan een profiel maken dat exclusief rapportagegegevens levert voor de gecontroleerde treffers. Met andere woorden: alles behalve de treffers van de controlesoftware wordt weggefilterd. Dit profiel kan worden gebruikt om aan te tonen dat de juiste gegevens worden gecontroleerd en dat wordt voldaan aan de SLA's.
