Urchin 6.602 Geo-databases

Urchin 6.602 wordt geleverd met twee geo-databases:

  • geo.db: bevat gedetailleerde gegevens over grootstedelijke gebieden (country/region/city), netwerken (organisaties) en domeinen. Deze geografische database bevat hetzelfde gegevensniveau (hoewel bijgewerkt) als in de eerdere versies van Urchin 6 (6.400 - 6.601).

  • geo.light.db: bevat alleen gegevens over grootstedelijke gebieden tot op landniveau. Deze geografische database bevat geen gegevens over regio's, steden, netwerken en organisaties. Deze database verbruikt minder geheugen en laadt sneller. U kunt deze database dus gebruiken wanneer de prestaties van de verwerkingsengine voor logbestanden belangrijk zijn. Als u deze eenvoudige geo-database gebruikt, heeft dit effect op de volgende rapporten.

    • o The following "Geo Map" reports only show metro data up to country level (no region and city info):

      • 'Weergave voor leidinggevenden' -> 'Overzicht voor leidinggevenden' (dashboard)

      • 'Weergave voor leidinggevenden' -> 'Geografische kaart-overlay'

      • 'Weergave voor marketeers' -> 'Overzicht voor marketeers' (dashboard)

      • 'Weergave voor webmasters' -> 'Overzicht voor webmasters' (dashboard)

      • 'Marketingoptimalisatie' -> 'Prestaties van bezoekerssegmenten' -> 'Geografische kaart-overlay'

      o Er worden geen gegevens weergegeven voor het rapport "Netwerklocaties" ('Marketingoptimalisatie' -> 'Prestaties van bezoekerssegmenten' -> 'Netwerklocatie')

      o Er worden geen gegevens weergegeven voor het rapport "Domeinen" ('Marketingoptimalisatie' -> 'Prestaties van bezoekerssegmenten' -> 'Domeinen')

De database geo.db wordt standaard gebruikt. Als u wilt overschakelen naar de database geo.light.db, zet u de commentaarregel in het bestand urchin.conf om in code door het #-teken te verwijderen: 


# Geo Database Settings (Instellingen geo-database)

# Uncomment the line below to enable the lightweight geo-database (Zet de onderstaande commentaarregel om in code om de eenvoudige geo-database te gebruiken)
# GeodbLight: ON (AAN)
