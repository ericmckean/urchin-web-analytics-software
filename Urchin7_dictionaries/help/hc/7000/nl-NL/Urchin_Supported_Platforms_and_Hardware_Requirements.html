Ondersteunde platforms en hardwarevereisten voor Urchin

Urchin kan op diverse architecturen en besturingssystemen worden uitgevoerd. Urchin hoeft alleen te worden geïnstalleerd op computers waarop logbestanden worden verwerkt. Voor het bekijken van rapporten is alleen een webbrowser nodig.

Ondersteunde platforms

Windows

  • Windows 2003 Server
  • Windows XP
  • Windows 2000
  • Windows Vista
  • Windows 2008 Server

UNIX-type Systems

  • Linux (kernel 2.6): algemeen voor de meeste Linux-versies
  • Linux (kernel 2.4): algemeen voor de meeste Linux-versies
  • FreeBSD 4 (niet ondersteund voor 6.5+)
  • FreeBSD 5.3+ (niet ondersteund voor Urchin 6.602+)
  • FreeBSD 6.2+

Supported databases

  • MySQL 4.1+ (niet ondersteund voor Urchin 6.6+)
  • MySQL 5.03+
  • PostgreSQL 8.2.5

Ondersteunde browsers

  • Internet Explorer 6.x, 7.x
  • Firefox 2.x, 3.x
  • Chrome 1.x, 2.x

Opmerking over niet-expliciet ondersteunde platforms

Als er geen Urchin- distributie voor uw specifieke platform bestaat, kunt u misschien een Urchin-distributie gebruiken die compatibel is met uw besturingssysteem.

  • Windows 98, Windows 3.x:

Urchin kan niet op Windows 98 of 3.x worden geïnstalleerd, maar deze platforms kunnen wel worden gebruikt voor het bekijken van rapporten met Internet Explorer (vanaf versie 5.x).

  • Linux:

Er zijn veel verschillende varianten van Linux, die niet allemaal expliciet worden ondersteund door een specifieke Urchin- distributie. De Linux-versies zijn doorgaans echter grotendeels compatibel, dus er is waarschijnlijk wel een Urchin- distributie die op uw computer werkt. Hier volgen enkele compatibele uitgaven:

  • Ubuntu (kernel 2.6)

  • RedHat Enterprise Linux 2 & higher

  • SuSE Linux 8 & higher

  • Trustix 2 & higher

  • Solaris:

Solaris wordt momenteel niet ondersteund.

Systeemvereisten voor Urchin 6

Dankzij de prestaties van Urchin haalt u meer uit een kleinere investering in hardware. Een oudere Pentium II is bijvoorbeeld mogelijk te langzaam als desktopcomputer, maar u kunt deze prima gebruiken als Urchin-server. Voor optimale prestaties wordt de volgende hardware aanbevolen:

Analyse van een kleine tot middelgrote website

  • 2 GHz processor of beter
  • 2 GB RAM
  • 250 GB+ IDE vaste schijf
  • 100 Mbs Ethernet-interface

Installaties van serviceproviders/bedrijven

  • 2,5 GHz Pentium dual-core/2 GHz UltraSPARC/soortgelijk CPU-snelheidsbereik PPC/MIPS/enzovoort.
  • 3GB RAM
  • Ultra2/Wide SCSI vaste schijf (bijvoorbeeld Seagate Cheetah)
  • 100base-T ethernet
  • Back-upsysteem

Geheugen-/systeem-/schijfgebruik

  • Het geheugengebruik (RAM) van Urchin kan worden geconfigureerd van 20 tot 500 MB (20 tot 1024 MB voor Urchin 6.5).
  • Urchin kan worden uitgevoerd op lage, normale of hoge prioriteit.
  • De gegevensopslag van Urchin gebruikt ongeveer 10% van het formaat van raw-logbestanden.

Belangrijke opmerking voor gebruikers van FreeBSD

(1) FreeBSD heeft een beperkte gegevenscapaciteit voor hardgecodeerde processen van 500 MB. Deze capaciteit wordt overschreden omdat de geografische gegevens van Urchin in het geheugen moeten blijven staan tijdens de logbestandsverwerking. Hierdoor wordt de volgende runtime-foutmelding weergegeven: 

ERROR: (8011-323-1057) Failed to allocate memory

Als u dit wilt oplossen, moet u de standaard procescapaciteit voor FreeBSD uitbreiden. Dit kunt u doen door het bestand /boot/loader.conf te bewerken en de volgende regels toe te voegen: 

# Increase max process data segment size to 1GB
kern.maxdsiz="1073741824"

Hierna moet u het systeem opnieuw opstarten.

WAARSCHUWING!!! Wees bijzonder voorzichtig wanneer u dergelijke wijzigingen aanbrengt in de systeemcapaciteit. Als u kern.maxdsiz instelt op een te grote waarde, kan het systeem mogelijk niet meer worden opgestart. We raden u met klem aan de documentatie van FreeBSD te lezen voordat u deze aanpassing aanbrengt en de potentiële risico's in te schatten voor uw specifieke configuratie.

(2) FreeBSD beschikt over een speciaal stuurprogramma dat accf_http heet. Hiermee worden inkomende verbindingen gebufferd tot een bepaald aantal volledige HTTP-verzoeken zijn ontvangen om zo snellere en optimale prestaties te leveren. Dit stuurprogramma is standaard niet geladen in de kernel en u ziet de volgende waarschuwing: 

[warn] (2)No such file or directory: Failed to enable the
‘httpready’ Accept Filter

U kunt dit voorkomen door de HTTP Accept Filter FreeBSD-kernelmodule (accf_http) in te schakelen. U kunt dit doen door het bestand /boot/loader.conf te bewerken en de volgende regels toe te voegen: 

# Wait for full HTTP request accept filter
accf_http_load="YES"

Hierna moet u het systeem opnieuw opstarten.

Bibliotheekversie van Linux glibc

Als u PostgreSQL 8.4.01 (meegeleverd met Urchin 6.602) wilt installeren op Linux(kernel 2.4 of 2.6), is hiervoor glibc 2.3.3+ vereist.
