Wie kann ich alle Sub-Domains für meine Website in einem Profil erfassen?

  • Dieser Artikel wurde von ivantage Limited erstellt, einem autorisierten Berater für Urchin-Software.

Sie können Sub-Domains in demselben Profil wie die Domain erfassen. Dafür müssen Sie dem Tracking-Code auf jeder Seite der Sub-Domain eine Zeile hinzufügen. Bei der folgenden Anleitung wird vorausgesetzt, dass dieses Profil mit der primären Domain als aufgelistete Website (z. B. "example.co.uk") eingerichtet wurde.

  1. Fügen Sie Ihrem Tracking-Code auf allen Seiten Ihrer Sub-Domain und Ihrer primären Domain folgende Zeile (in Fettschrift) hinzu: 

         <script src="/urchin.js" type="text/javascript">
         </script>
         <script type="text/javascript">
           _udn="example.co.uk";
          </script>
  • Diese Zeile muss sich auf die primäre Domain beziehen (example.co.uk).

Obwohl Sie mit dieser Änderung Sub-Domains innerhalb eines Profils erfassen können, wird in Ihren Berichten vermutlich nicht zwischen Seiten aus "www.example.co.uk" und "help.example.co.uk" unterschieden. Beispielsweise können in Ihrem Bericht "Beliebtester Content" Treffer für "www.example.co.uk/index.html" und "help.example.co.uk/more.html" vorliegen, im Bericht wird jedoch Folgendes angezeigt: 

   /index.html
   /more.html

Wenn Sie zwischen Ihren Sub-Domains unterscheiden möchten, können Sie für Ihr Profil einen erweiterten Filter mit den folgenden Einstellungen anlegen: 

   Filter Type: Custom filter > Advanced
   Field A: cs_host (Raw)
   Extract A: (.*)
   Field B: Request_Stem (Auto)
   Extract B: (.*)
   Output To: Request_Stem (Auto)
   Constructor: /$A1$B1

Mit diesem Filter werden die zuvor genannten Beispiele mit der angefügten Sub-Domain angezeigt: 

   www.example.co.uk/index.html
   help.example.co.uk/more.html

HINWEIS: Dieser Filter basiert darauf, dass der Hostname Ihrer Website, einschließlich der Sub-Domain in Ihre Zugriffsprotokolldatei geschrieben wird. Wenn die Zugriffprotokolldatei den Hostnamen nicht enthält, können Urchin 6-Berichte keinen Unterschied aufzeichnen, z. B. zwischen den Dateien "index.html" der beiden Domains: 

 www.example.co.uk/index.html
 help.example.co.uk/more.html
