Arbeiten mit UNC-Protokollquellen

Bei der Erstellung einer UNC-Protokolldatei in Urchin werden folgende Parameter angegeben: Server Name/IP, Remote Server Login, Local Server Login, Remote Log Source File.

Der Ablauf, nach dem Urchin das Protokoll aus einer UNC-Quelle abruft, ist wie folgt: – Urchin versucht, nach den in der "Remote Log Source File" angegebenen Dateien zu suchen. – Falls die Suche fehlschlägt, versucht Urchin, eine Verbindung zur UNC-Freigabe, die im Feld "Server Name/IP" angegeben ist, herzustellen. – Schlägt die Verbindung fehl, setzt Urchin eine Ausführung unter einem Nutzerkonto ohne Netzwerkzugriff, wie dem LocalSystem-Konto, als gegeben voraus. In diesem Fall erstellt Urchin eine Anmeldesitzung für ein Nutzerkonto, die unter "Local Server Login" festgelegt wird, und gibt sich selbst als dieses Konto aus. Das heißt, der Urchin-Prozess wird unter diesem Konto ausgeführt. Wenn die Optionen "Same Local und Remote Login" auf "Ja" gesetzt sind, werden die Anmeldedaten für den vorgetäuschten Auftritt aus "Remote Server Login" übernommen. – Ist der vorgetäuschte Auftritt erfolgreich, versucht Urchin erneut, eine UNC-Verbindung zur Netzwerkfreigabe herzustellen und wiederholt die Suche. – Sobald Urchin die Suche nach der Protokollquelle abgeschlossen hat, wird die UNC-Verbindung getrennt und der vorgetäuschte Auftritt beendet. Das heißt, der Urchin-Prozess wird wieder unter dem Konto ausgeführt, unter dem er vor dem vorgetäuschten Auftritt lief.

Ein vorgetäuschter Auftritt hat folgende Nachteile: – Während dessen wird der Urchin-Prozess unter einem anderen Konto ausgeführt, das eventuell keinen Zugriff auf das Urchin-Datenverzeichnis hat, sodass keine Berichtdatenbanken erstellt werden können. – Die Berichtdatenbanken, die während des Auftritts erstellt werden, sind nach Beendigung des vorgetäuschten Auftritts eventuell nicht mehr für den Urchin-Prozess zugänglich. – Die Leistung wird verringert, vor allem, wenn dem Profil mehrere UNC-Protokolldateien zugeordnet werden, da Urchin den vorgetäuschten Auftritt für jede einzelne Datei starten und beenden muss.

Deshalb haben sich für den Umgang mit UNC-Freigaben in Urchin folgende Vorgehensweisen bewährt: – Führen Sie Urchin unter dem Konto aus, das Zugriff auf das Netzwerk hat. In diesem Fall findet kein vorgetäuschter Auftritt statt. – Gewähren Sie Lesezugriff auf UNC-Protokollfreigaben für das Nutzerkonto, auf dem Urchin ausgeführt wird. In diesem Fall erübrigen sich die Herstellung einer UNC-Verbindung und ein vorgetäuschter Auftritt.
