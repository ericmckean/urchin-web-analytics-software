Cobranding von Urchin

Übersicht

Urchin ermöglicht ein Cobranding in der Verwaltungs- und der Berichterstellungsoberfläche sowie auf der Anmeldeseite. Drei Dateien müssen bearbeitet werden, um HTML in die Benutzeroberfläche aufzunehmen. Bei einer vollständigen Portalintegration kann die Urchin-Berichterstellung in einem Frameset oder einer Tabelle von Ihrem Anwendungsserver bereitgestellt werden.

Anleitungen zum Cobranding

Wenn Sie Ihre Benutzeroberfläche mit Cobranding versehen möchten, ist es notwendig, dass Sie die folgenden Dateien in der Urchin-Installation bearbeiten: 

     [urchin install dir]/lib/custom/cobrands/cobrand_admin.tpl.sample
     [urchin install dir]/lib/custom/cobrands/cobrand_report.tpl.sample
     [urchin install dir]/lib/custom/cobrands/cobrand_session.tpl.sample
  • Benennen Sie die Dateien um, indem Sie ".sample" entfernen (der Dateiname sollte z. B. "cobrand_admin.tpl" lauten).
  • Fügen Sie diesen Dateien nach Bedarf HTML-Inhalt hinzu, um Ihr Branding einzuschließen.

Mit der ersten Datei wird das Cobranding der Verwaltungsoberfläche gesteuert und mit der zweiten Datei das Cobranding der Berichterstellungsoberfläche. Die dritte Datei steuert das Cobranding der Anmeldeseite. Der HTML-Code in diesen Dateien wird über der Urchin-Benutzeroberfläche platziert.
