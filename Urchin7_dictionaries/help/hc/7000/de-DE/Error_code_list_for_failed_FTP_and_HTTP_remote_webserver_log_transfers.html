Liste der Fehlercodes zu FTP- und HTTP-Übertragungsfehlern des Remote-Webserver-Protokolls

Übersicht

Eine Urchin-Protokollquelle kann so konfiguriert werden, dass sie ein Webserver-Protokoll über FTP oder HTTP von einem Remote-Server abruft. Diese Übertragung erfolgt in der Regel problemlos und im Laufzeitprotokoll werden keine Fehler angezeigt. Es können jedoch Fehler bei der Übertragung auftreten, z. B. ein ungültiger Nutzername/Passwort, der Remote-Server reagiert nicht oder das Remote-Protokoll kann nicht gelesen werden. In diesem Fall wird von Urchin ein Fehlercode in der Laufzeitausgabe protokolliert und im Aufgabenverlauf für das Profil angezeigt. Dieser Fehlercode wird nach einem Übertragungsversuch des Webserver-Protokolls in Klammern neben der Meldung "fehlgeschlagen" angezeigt, z. B. (-9).

Die Fehlercodes und eine Übersetzung der Textmeldung zum Problem werden in der folgenden Liste aufgeführt.

Liste der Fehlercodes

CURLE_UNSUPPORTED_PROTOCOL (1) - Die an libcurl übergebene URL verwendete ein Format, das von dieser libcurl nicht unterstützt wird. Die Unterstützung besteht möglicherweise in einer Kompilierzeitoption, die nicht verwendet wurde. Vielleicht wurde auch eine Protokollzeichenfolge falsch geschrieben oder es handelt sich um ein Protokoll, für das libcurl keinen Code besitzt.

CURLE_FAILED_INIT (2) - Ein sehr früher Initialisierungscode schlug fehl. Es handelt sich möglicherweise um einen internen Fehler.

CURLE_URL_MALFORMAT (3) - Die URL war nicht ordnungsgemäß formatiert.

CURLE_COULDNT_RESOLVE_PROXY (5) - Proxy konnte nicht aufgelöst werden. Eine Auflösung des angegebenen Proxy-Hosts war nicht möglich.

CURLE_COULDNT_RESOLVE_HOST (6) - Host konnte nicht aufgelöst werden. Eine Auflösung des angegebenen Remote-Hosts war nicht möglich.

CURLE_COULDNT_CONNECT (7) - Mit dem Host oder Proxy konnte keine Verbindung() hergestellt werden.

CURLE_FTP_WEIRD_SERVER_REPLY (8) - Nach dem Herstellen einer Verbindung mit einem FTP-Server erwartet libcurl eine bestimmte Antwort. Dieser Fehlercode bedeutet, dass die Antwort ungewöhnlich oder falsch war. Der angegebene Remote-Server ist wahrscheinlich kein einwandfreier FTP-Server.

CURLE_REMOTE_ACCESS_DENIED (9) - Der Zugriff auf die in der URL angegebene Ressource wurde verweigert. Bei FTP tritt dieses Problem bei einem Wechsel des Remote-Verzeichnisses auf.

CURLE_FTP_WEIRD_PASS_REPLY (11) - Nachdem das FTP-Passwort an den Server gesendet wurde, erwartet libcurl eine ordnungsgemäße Antwort. Dieser Fehlercode gibt an, dass ein unerwarteter Code zurückgegeben wurde.

CURLE_FTP_WEIRD_PASV_REPLY (13) - libcurl erhielt auf einen PASV- oder EPSV-Befehl keine sinnvolle Antwort vom Server. Der Server ist fehlerhaft.

CURLE_FTP_WEIRD_227_FORMAT (14) - FTP-Server geben eine 227-Zeile als Antwort auf einen PASV-Befehl zurück. Wenn libcurl diese Zeile nicht analysieren kann, wird dieser Code zurückgegeben.

CURLE_FTP_CANT_GET_HOST (15) - Interner Fehler bei der Suche nach dem für die neue Verbindung verwendeten Host.

CURLE_FTP_COULDNT_SET_TYPE (17) - Fehler beim Festlegen des Übertragungsmodus auf binär oder ASCII.

CURLE_PARTIAL_FILE (18) - Eine Dateiübertragung war kürzer oder länger als erwartet. Das geschieht, wenn der Server zunächst eine erwartete Übertragungsgröße berichtet und dann Daten überträgt, die mit der zuvor angegebenen Größe nicht übereinstimmen.

CURLE_FTP_COULDNT_RETR_FILE (19) - Dabei handelte es sich entweder um eine unverständliche Antwort auf einen RETR-Befehl oder um eine abgeschlossene Übertragung von null Bytes.

CURLE_QUOTE_ERROR (21) - Beim Senden von QUOTE-Befehlen an den Remote-Server gab einer der Befehle einen Fehlercode 400 oder höher (für FTP) zurück oder wies in anderer Weise auf eine fehlgeschlagene Ausführung des Befehls hin.

CURLE_HTTP_RETURNED_ERROR (22) - Wird zurückgegeben, wenn CURLOPT_FAILONERROR auf TRUE festgelegt ist und der HTTP-Server einen Fehlercode 400 oder höher zurückgibt. (Dieser Fehlercode lautete früher CURLE_HTTP_NOT_FOUND.)

CURLE_WRITE_ERROR (23) - Beim Schreiben empfangener Daten in eine lokale Datei trat ein Fehler auf. Oder von einem Schreib-Callback wurde ein Fehler an libcurl zurückgegeben.

CURLE_UPLOAD_FAILED (25) - Fehler beim Starten des Uploads. Der Server verweigerte bei FTP typischerweise den STOR-Befehl. Der Fehlerpuffer enthält hierfür in der Regel die Erklärung des Servers. (Dieser Fehlercode lautete früher CURLE_FTP_COULDNT_STOR_FILE.)

CURLE_READ_ERROR (26) - Problem beim Lesen einer lokalen Datei oder vom Lese-Callback wurde ein Fehler zurückgegeben.

CURLE_OUT_OF_MEMORY (27) - Fehler bei einer Speicherzuordnungsanforderung. Das ist ein schwerwiegender Fehler. Tritt er auf, ist die Situation äußerst ernst.

CURLE_OPERATION_TIMEDOUT (28) - Zeitüberschreitung des Vorgangs. Die angegebene Zeitüberschreitung wurde den Bedingungen entsprechend erreicht.

CURLE_FTP_PORT_FAILED (30) - Der FTP-Befehl PORT gab einen Fehler zurück. Das ist meistens der Fall, wenn für libcurl keine ordnungsgemäße Adresse angegeben wurde. Siehe CURLOPT_FTPPORT.

CURLE_FTP_COULDNT_USE_REST (31) - Der FTP-Befehl REST gab einen Fehler zurück. Dieser Fall sollte bei einem fehlerfreien Server nie eintreten.

CURLE_RANGE_ERROR (33) - Der Server unterstützt oder akzeptiert keine Bereichsanforderungen.

CURLE_HTTP_POST_ERROR (34) - Das ist ein ungewöhnlicher Fehler, der vorwiegend aufgrund interner Probleme auftritt.

CURLE_SSL_CONNECT_ERROR (35) - Problem beim SSL/TLS-Handshake. Lesen Sie die Meldung im Fehlerpuffer, um genauere Hinweise zum Problem zu erhalten. Es könnte Zertifikate (Dateiformate, Pfade, Berechtigungen), Passwörter und Ähnliches betreffen.

CURLE_FTP_BAD_DOWNLOAD_RESUME (36) - FTP-Resume wird über die Dateigröße hinaus versucht.

CURLE_FILE_COULDNT_READ_FILE (37) - Eine mit "FILE://" angegebene Datei konnte nicht geöffnet werden. Wahrscheinlich gibt der Dateipfad keine vorhandene Datei an. Haben Sie die Dateiberechtigungen überprüft?

CURLE_LDAP_CANNOT_BIND (38) - Bindung durch LDAP ist nicht möglich. Fehler bei einem LDAP-Bindungsvorgang.

CURLE_LDAP_SEARCH_FAILED (39) - Fehler bei der LDAP-Suche.

CURLE_FUNCTION_NOT_FOUND (41) - Funktion wurde nicht gefunden. Eine erforderliche zlib-Funktion wurde nicht gefunden.

CURLE_ABORTED_BY_CALLBACK (42) - Von Callback verworfen. Ein Callback gab "abort" an libcurl zurück.

CURLE_BAD_FUNCTION_ARGUMENT (43) - Interner Fehler. Eine Funktion wurde mit einem falschen Parameter aufgerufen.

CURLE_INTERFACE_FAILED (45) - Interner Fehler. Eine angegebene ausgehende Schnittstelle konnte nicht verwendet werden. Legen Sie mit CURLOPT_INTERFACE die zu verwendende Schnittstelle für die Quell-IP-Adresse ausgehender Verbindungen fest. (Dieser Fehlercode lautete früher CURLE_HTTP_PORT_FAILED.)

CURLE_TOO_MANY_REDIRECTS (47) - Zu viele Weiterleitungen. Beim Folgen von Weiterleitungen hat libcurl das Maximum erreicht. Legen Sie das Limit mit CURLOPT_MAXREDIRS fest.

CURLE_UNKNOWN_TELNET_OPTION (48) - Eine mit CURLOPT_TELNETOPTIONS festgelegte Option wurde nicht erkannt. Weitere Informationen erhalten Sie in der entsprechenden Dokumentation.

CURLE_TELNET_OPTION_SYNTAX (49) - Die Formatierung einer Telnet-Optionszeichenfolge war ungültig.

CURLE_PEER_FAILED_VERIFICATION (51) - Das SSL-Zertifikat oder der SSH md5-Fingerprint des Remote-Servers wurde nicht akzeptiert.

CURLE_GOT_NOTHING (52) - Vom Server wurde nichts zurückgegeben. Unter den gegebenen Umständen gilt die Rückgabe von nichts als Fehler.

CURLE_SSL_ENGINE_NOTFOUND (53) - Das angegebene Verschlüsselungsmodul wurde nicht gefunden.

CURLE_SSL_ENGINE_SETFAILED (54) - Das ausgewählte SSL-Verschlüsselungsmodul konnte nicht als Standardmodul festgelegt werden.

CURLE_SEND_ERROR (55) - Netzwerkdaten konnten nicht gesendet werden.

CURLE_RECV_ERROR (56) - Fehler beim Empfangen von Netzwerkdaten.

CURLE_SSL_CERTPROBLEM (58) - Probleme mit dem lokalen Clientzertifikat.

CURLE_SSL_CIPHER (59) - Die angegebene Verschlüsselung konnte nicht verwendet werden.

CURLE_SSL_CACERT (60) - Peer-Zertifikat kann mit bekannten Zertifikaten nicht authentifiziert werden.

CURLE_BAD_CONTENT_ENCODING (61) - Unbekannte Übertragungskodierung

CURLE_LDAP_INVALID_URL (62) - Ungültige LDAP-URL

CURLE_FILESIZE_EXCEEDED (63) - Maximale Dateigröße wurde überschritten.

CURLE_USE_SSL_FAILED (64) - Fehler bei der angeforderten FTP SSL-Ebene

CURLE_SEND_FAIL_REWIND (65) - Bei einem Sendevorgang musste curl die Daten für eine erneute Übertragung zurücksetzen, doch der Rewind-Vorgang schlug fehl.

CURLE_SSL_ENGINE_INITFAILED (66) - Fehler beim Initiieren des SSL-Moduls

CURLE_LOGIN_DENIED (67) - Der Remote-Server verweigerte curl die Anmeldung (in 7.13.1 hinzugefügt).

CURLE_TFTP_NOTFOUND (68) - Datei wurde auf dem TFTP-Server nicht gefunden.

CURLE_TFTP_PERM (69) - Berechtigungsproblem auf dem TFTP-Server

CURLE_REMOTE_DISK_FULL (70) - Fehlender Speicherplatz auf dem Server

CURLE_TFTP_ILLEGAL (71) - Ungültiger TFTP-Vorgang

CURLE_TFTP_UNKNOWNID (72) - Unbekannte TFTP-Übertragungs-ID

CURLE_REMOTE_FILE_EXISTS (73) - Datei ist bereits vorhanden und wird nicht überschrieben.

CURLE_TFTP_NOSUCHUSER (74) - Dieser Fehler sollte bei einem ordnungsgemäß funktionierenden TFTP-Server niemals zurückgegeben werden.

CURLE_CONV_FAILED (75) - Fehler bei der Zeichenkonvertierung

CURLE_CONV_REQD (76) - Der aufrufende Prozess muss Konvertierungs-Callbacks registrieren.

CURLE_SSL_CACERT_BADFILE (77) - Problem beim Lesen des SSL-Zertifikats (Pfad? Zugriffsrechte?)

CURLE_REMOTE_FILE_NOT_FOUND (78) - Die Ressource, auf die in der URL verwiesen wird, ist nicht vorhanden.

CURLE_SSH (79) - Während der SSH-Sitzung trat ein unbekannter Fehler auf.

CURLE_SSL_SHUTDOWN_FAILED (80) - Fehler beim Beenden der SSL-Verbindung
