Urchin-Berichtdatenbankeinstellungen

(Überarbeitung: 16. Dezember 2008)

Übersicht

Urchin bietet zwei Optionen für die Verwaltung der Skalierbarkeit und Genauigkeit der Berichtdatenbank:

Datenbanktabellenlimit

Diese Option ermöglicht Ihnen die Verwaltung von Daten großer Websites innerhalb sinnvoller Beschränkungen. Diese Option dient zur Beschränkung der Anzahl der Datensätze in einer einzelnen Urchin-Berichtdatenbank-Tabelle. Der Standardwert ist für die meisten Installationen geeignet. Wenn für einen Monat die Anzahl der Datensätze in einer beliebigen Tabelle dieses Limit überschreitet, werden die Datensätze, die über dieses Limit hinausgehen, im letzten Datensatz ’Sonstiges’ gruppiert. Je größer also die Tabelle, desto genauer sind die Berichtdaten.

Zeigt ein Urchin-Bericht einen hohen Prozentsatz an Einträgen mit der Bezeichnung Sonstiges, sollten Sie das Limit der Datenbanktabelle erhöhen, um die Genauigkeit zu verbessern.

Speicherauslastungsziel

Um die Größe des Arbeitsspeichers zu beschränken, der während der Protokollverarbeitung zugewiesen werden kann, bietet Urchin diese Option, mit der Sie die Obergrenze für die Größe der Berichtdatenbank im Arbeitsspeicher festlegen können. Bei großen Websites kann durch einen größeren Speicher-Footprint die Leistung verbessert werden, da weniger Seitenwechsel stattfinden. Die Standardeinstellung lässt noch Spielraum zu. Es wird empfohlen, einen größeren Wert festzulegen, sofern das System über ausreichend Arbeitsspeicher verfügt.

Testeinstellungen

Derzeit unterstützt Urchin die folgenden Höchstwerte für Berichtdatenbankeinstellungen:

Datenbanktabellenlimit : <= 500.000 Datensätze

Speicherauslastungsziel : <= 1024 MB

Ab Version 6.5 kann Urchin einen höheren Wert für diese Einstellungen unterstützen, was sich nur für Erfahrene Urchin-Nutzer empfiehlt. Diese höheren Werte können nicht in der Verwaltungsoberfläche angezeigt werden und müssen über die Konfigurationsdatenbank konfiguriert werden. Für Urchin 6.5 gelten die folgenden Höchstwerte für erweiterte Berichtdatenbankeinstellungen:

Datenbanktabellenlimit : <= 2.000.000 Datensätze

Speicherauslastungsziel : <= 2 GB

Richtlinien für Testeinstellungen

  • Da diese erweiterten Einstellungen zum Experimentieren vorgesehen sind und in einigen Systemen möglicherweise Fehler aufgrund mangelnden Arbeitsspeichers verursachen, sollten die folgenden Informationen und Richtlinien bei der Verwendung der Einstellungen berücksichtigt werden.
  • In der Regel verfügt jede 32-Bit-Anwendung über einen virtuellen Adressspeicher von 4 Gigabyte (GB), der in den Speicherplatz für die Nutzeradresse und den Speicherplatz für die Kernel-Adresse aufgeteilt ist. Der Nutzerprozess kann nur im Nutzeradressenspeicher Arbeitsspeicher zuweisen.

  • Windows-Plattform: 2 GB stehen für die Nutzeranwendung und 2 GB für den Kernel zur Verfügung. Auch wenn das windows OS es ermöglicht, über "4-Gigabyte Tuning" den Nutzerspeicher auf 3 GB zu erweitern, berücksichtigen Urchin–Prozesse dieses äußerst unwahrscheinliche Szenario nicht und sind immer auf 2-GB-Limits ausgerichtet. Aus diesem Grund sollten Nutzer mit dem windows OS das Datenbanktabellenlimit nicht auf mehr als 500.000 Datensätze und das Speicherauslastungsziel nicht auf mehr als 1 GB festlegen.

    • Legen Sie für eine maximale Leistung Folgendes fest:

      • Speicherauslastungsziel : 1024 MB

      • Datenbanktabellenlimits : 500.000 Datenbanktabellenzeilen

    • Legen Sie für eine maximale Genauigkeit Folgendes fest:

      • Speicherauslastungsziel : 256 MB

      • Datenbanktabellenlimits : 1.000.000 Datenbanktabellenzeilen

  • Unix-Plattformen: 3 GB stehen für die Nutzeranwendung und 1 GB für den Kernel zur Verfügung. Aus diesem Grund sollten Nutzer der UNIX-Plattform das Datenbanktabellenlimit nicht auf mehr als 500.000 Datensätze und das Speicherauslastungsziel nicht auf mehr als 2 GB festlegen.

    • Legen Sie für eine maximale Leistung Folgendes fest:

      • Speicherauslastungsziel : 2048 MB

      • Datenbanktabellenlimits : 500.000 Datenbanktabellenzeilen

    • Legen Sie für eine maximale Genauigkeit Folgendes fest:

      • Speicherauslastungsziel : 1024 MB

      • Datenbanktabellenlimits : 2.000.000 Datenbanktabellenzeilen

Konfigurieren des erweiterten 'Datenbanktabellenlimits'

Diese höheren Werte können wie reguläre Werte für das 'Datenbanktabellenlimit' für individuelle Profile oder global konfiguriert werden. Mit der folgende Abfrage wird das Datenbanktabellenlimit auf 1.000.000 Datensätze für ein Profil mit der Profil-ID '1' festgelegt. 

UPDATE uprofiles SET uipr_db_table_limit=1000000 WHERE uspr_id=1;

Mit der folgenden Abfrage wird das globale Datenbanktabellenlimit auf 1.000.000 Datensätze festgelegt. Dies wird auf alle Profile angewendet, deren Wert für das Feld uipr_db_table_limit "0" ist. 

UPDATE uglobals SET uigl_db_table_limit=1000000 WHERE ucgl_name='Global Settings';

Konfigurieren des erweiterten 'Speicherauslastungsziels'

Das Speicherauslastungsziel kann nur global festgelegt werden.

Es wird automatisch auf Profile angewendet, wenn die Datenbankspeicherauslastung auf 'Mit Cachesystem begrenzen' oder das Datenbankfeld 'ubpr_limit_db_cache_size' auf "1" festgelegt ist.

Mit der folgenden Abfrage wird das globale Speicherauslastungsziel auf 2 GB festgelegt. Beachten Sie, dass für das Datenbankfeld die angegebene Zahl in 'Megabyte' berücksichtigt wird ("512" bedeutet also 512 MB). 

UPDATE uglobals SET uigl_db_cache_size=2048 WHERE ucgl_name='Global Settings';
