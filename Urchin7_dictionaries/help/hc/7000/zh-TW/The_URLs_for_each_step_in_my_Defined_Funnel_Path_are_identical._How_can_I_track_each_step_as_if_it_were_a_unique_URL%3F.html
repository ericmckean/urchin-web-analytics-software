在「定義程序路徑」中，每個步驟的網址都相同。如何將每個步驟視為個別網址來進行追蹤？

您不妨透過每個步驟都採用相同網址的程序來追蹤訪客的瀏覽過程。例如，您的申請程序可能如下：

  • 步驟 1 (申請)
    • www.mysite.com.tw/sign_up.cgi
  • 步驟 2 (接受合約)
    • www.mysite.com.tw/sign_up.cgi
  • 步驟 3 (完成)
    • www.mysite.com.tw/sign_up.cgi

您可以在每個步驟 (可能在 onload 事件內) 中呼叫 urchinTracker Javascript 函數來完成此作業，如下所示：

  • urchinTracker("/funnel_G1/step1.html")
  • urchinTracker("/funnel_G1/step2.html")
  • urchinTracker("/funnel_G1/step3.html")

接著將程序的每個步驟設定為：

  • http://www.mysite.com.tw/funnel_G1/step1.html

  • http://www.mysite.com.tw/funnel_G1/step2.html

  • http://www.mysite.com.tw/funnel_G1/step3.html

請注意，urchinTracker() 路徑 / 檔名的引數不須代表現有的路徑或檔名。urchinTracker 的引數只是提供虛設的網頁名稱，以便 Urchin 附加網頁檢視。
