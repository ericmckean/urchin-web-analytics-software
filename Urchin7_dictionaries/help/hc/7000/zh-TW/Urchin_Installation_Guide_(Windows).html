Urchin 6 安裝指南 (適用於 Windows)

本文由 Urchin 軟體授權顧問 EpikOne 提供

開始安裝之前：Urchin 6.500 使用者的重要注意事項

Urchin 6.501 版已修正單次點擊出價排程器內出現的多個問題。 如果您是從 Urchin 6.500 升級至 Urchin 6.602，可能會因為處理後的單次點擊出價資料不正確或不完整，導致 Urchin 報表顯示出不正確的結果。 為了從 Google AdWords 取得正確的資料，建議您 (a) 將Urchin 6.602 安裝在新的位置，或是 (b) 如果希望在現有 6.500 版本中進行安裝，請刪除舊有單次點擊出價資料。 如果選擇後者，請刪除下列目錄中的全部內容： 

<path_to_urchin_6500_installation>\data\cpc\...
<path_to_urchin_6500_installation>\data\history\cpc\...

如果您是從 Urchin 6.600、6.501 版或是 Urchin 6.402 及先前版本進行升級，則不需進行任何動作。

安裝步驟總覽

  • 您可以將所需的資料庫伺服器安裝在 Urchin 系統上，或是在現有的伺服器上新增 Urchin 資料庫。 目前支援的資料庫引擎如下：
    • MySQL 5.0.3 或更新版本 (請注意： 自Urchin 6.6 版起不再支援 MySQL 4.x.x )
    • PostgreSQL 8.2.5 或更新版本
  • 在資料庫伺服器中建立 Urchin 資料庫和使用者，並授予存取權限
  • 將 Urchin 發行版解壓縮到暫存資料夾中
  • 執行 MSI 安裝程式
  • 取得 Urchin 安裝授權

方法 1： 使用現有 MySQL 資料庫搭配 Urchin

  • 如果您的系統尚未設定 MySQL 伺服器，可以從 http://dev.mysql.com/downloads/ 取得發行版。 Windows Essentials 套裝軟體包含所有使用 Urchin 的必要元件。

  • 按照 MySQL 網站上的指示安裝 MySQL 伺服器。
  • 請務必取得 MySQL 資料庫管理員的登入資訊 (例如 root) 和密碼。
  • 選擇要用於 Urchin 中的資料庫名稱、MySQL 本機使用者和密碼。 請將這些資訊儲存起來，日後安裝 Urchin 時會用到。 以下範例將建立一個名為 urchin 的資料庫以及一個名為 urchin 的 MySQL 使用者名稱，密碼為 urchinpassword (這只是範例，您當然得選擇自己的密碼！)

  • 透過 [開始] > [程式集] > [MySQL] > [MySQL 伺服器] > [MySQL 指令行用戶端]，啟動「MySQL 指令行用戶端」。

  • 在「MySQL 指令行用戶端」視窗中，使用下列指令建立 Urchin 資料庫、使用者和權限： 

mysql> create database urchin character set utf8;
mysql> GRANT ALL ON urchin.* to 'urchin'@'localhost' IDENTIFIED BY
'urchinpassword';

方法 2： 使用現有 PostgreSQL 資料庫搭配 Urchin

  • 如果您的系統尚未安裝 PostgreSQL 伺服器，可以從 http://www.postgresql.org 取得發行版。

  • 按照 PostgreSQL 網站上的指示安裝 PostgreSQL 伺服器。
  • 請務必取得 PostgreSQL 資料庫管理員的登入資訊 (例如 postgres) 和密碼。
  • 選擇要用於 Urchin 中的資料庫名稱、PostgreSQL 本機使用者和密碼。 請將這些資訊儲存起來，日後安裝 Urchin 時會用到。 以下範例將建立一個名為 urchin 的資料庫以及一個名為 urchin 的 PostgreSQL 使用者名稱，密碼為 urchinpass (這只是範例，您當然得選擇自己的密碼！)

  • 透過 [開始] > [程式集] > [PostgreSQL 8.2] > [psql to 'postgres']，啟動「PostgreSQL 指令行用戶端」。

  • 在「PostgreSQL 指令行用戶端」視窗中，使用下列指令建立 Urchin 資料庫、使用者和權限： 

postgres=# CREATE DATABASE urchin WITH ENCODING 'UTF8';
postgres=# CREATE USER urchin WITH PASSWORD 'urchinpass';
postgres=# GRANT ALL ON DATABASE urchin TO urchin;
postgres=# \connect urchin;
postgres=# CREATE LANGUAGE 'PLPGSQL';

方法 3： 使用隨附的 PostgreSQL 伺服器搭配 Urchin

  • Urchin 6.602 安裝程式隨附一套 PostgreSQL 伺服器，讓您在 Urchin 安裝程序中選用。

  • 您必須以管理員權限啟動 Urchin 安裝程式，才能使用此方法。 安裝程式將提示您提供 "postgres" 這名使用者的埠號和密碼。 請注意： 如果系統中已經有名為 "postgres" 的使用者，您必須提供現有的密碼。 如果系統中沒有名為 "postgres" 的使用者，安裝程式就會以提供的密碼建立使用者。

  • 隨附的 PostgreSQL 伺服器有下列限制：
    • 不可在升級模式中進行安裝。
    • 需要管理員權限才能進行安裝。

安裝 Urchin

  • 將 Urchin 發行版 ZIP 封存解壓縮到暫存資料夾中。

  • 若要安裝隨附的 PostgreSQL 伺服器，請確認您的系統上沒有任何 PostgreSQL 伺服器。 若要進行安裝，請選取 "[安裝 PostgreSQL]" 選項，並提供 PostgreSQL 伺服器相關詳情，例如埠號和使用者 "postgres" 的密碼等等。 請注意： 如果在安裝隨附的 PostgreSQL 伺服器時遇到任何問題，請參閱：http://www.postgresql.org。

  • 若要使用現有的資料庫，請準備好您在上述步驟中針對 MySQL 或 PostgreSQL 選定的 Urchin 資料庫名稱、使用者和密碼。
  • Urchin 6 只允許在一台電腦上安裝一個 Urchin 6 程式 (僅適用於 Windows)。 如果電腦上已安裝了某個版本的 Urchin，則只能進行升級 (不允許降級)。
  • 如要進行'全新'安裝，請移除目標電腦上所有現有的 Urchin 6 版本。 若要移除 Urchin，請按照下列步驟進行：

    • [開始] > [程式集] > [Urchin 6] > [停用 Urchin 服務]

    • [開始] > [設定] > [控制台] > [新增或移除程式]

    • 按一下 "[Google 提供的 Urchin 軟體]"，然後選擇 [移除]
    • 移除 Urchin 後，請重新命名 urchin 資料夾 (例如將 C:\Program Files\Urchin6 資料夾重新命名為 Urchin6-OLD)。
  • 只有某些舊版 Urchin 6 可升級為 Urchin 6.602。 下列版本均可進行升級：
    • Urchin 6.6 - Service Pack 1 ： 6.601 版
    • Urchin 6.6 - 英文版 ： 6.600 版
    • Urchin 6.5 - Service Pack 2 ： 6.501 版
    • Urchin 6.5 : 6.500 版 (請參閱上述重要注意事項)
    • Urchin 6 - Service Pack 1 ： 6.402 版
    • Urchin 6 - 國際版： 6.401 版
    • Urchin 6 - 英文版 ： 6.400 版
  • 如果選擇升級，您不需要停用或解除安裝現有的 Urchin 6 版本。 Urchin 6.602 安裝程式會先檢查並解除安裝目前已安裝的所有 Urchin 6 版本， 再安裝 Urchin 6.602 的檔案和組態。
  • 若要開始安裝 Urchin 6.602，請按兩下 urchin6602_windows_installer.msi 檔案，然後按照互動式提示進行。

  • 請注意： 如果您想要保留先前安裝之 Urchin 6 版本的組態，請務必取消勾選 "[如果 Urchin 特定資料庫表格已存在，則重新建立現有表格]" 選項。 在此情況下，Urchin 會保留現有組態，但可能會建立'「其他表格」'，或在'「現有表格中建立其他欄」'。

  • 現在，Urchin 便可以在您的系統中執行了。 若要結束安裝程序，請將您常用的網路瀏覽器指向下列網址： 

http://yourserver:<port>

其中 <port> 是指您在安裝期間選擇的網路通訊埠 (預設為 9999)。

  • 如果您想要從舊版 Urchin 6中還原設定檔報表資料，請將記錄和報表資料夾從 C:\Program Files\Urchin6-OLD\data 複製到 C:\Program Files\Urchin6
  • 按一下 [取得試用授權] 連結，按照授權步驟啟用 Urchin 並執行初始設定步驟。

設定追蹤

Urchin 安裝完成後，根據使用個案的不同，您可能需要進行其他步驟來設定網站追蹤。

個案 1： 使用 Urchin 搭配 IP+User-Agent 追蹤

無需額外設定。 將設定檔新增至 Urchin 時，請務必指定 IP+User-Agent 做為「訪客追蹤方式」。

個案 2： 使用 Urchin 搭配 UTM 追蹤

若要使用 Urchin 搭配 UTM 進行追蹤，您必須在網站的每個網頁上安裝一小段 javascript 追蹤程式碼， 步驟如下：

  • 從 Urchin 發行版的 util\utm 目錄中，將 urchin.js 和 __utm.gif 檔案複製到網站的文件根目錄 (例如網站內容的最上層目錄)

  • 在網站的每個網頁上，將下列追蹤程式碼加到 HEAD 區段中，緊接在所有 META 標記之後： 

<script src="/urchin.js" type="text/javascript">
</script>
<script type="text/javascript">
  _userv=0;
  urchinTracker();
</script>
  • 將設定檔新增至 Urchin 時，請務必指定 Urchin 流量監視器 (UTM) 做為「訪客追蹤方式」

個案 3： 使用 Urchin 搭配 Google Analytics (分析)

請注意，Google Analytics (分析) "ga.js" 追蹤程式碼中的新追蹤功能與 Urchin 6 不相容。 如果您想要同時使用 Google Analytics (分析) 和 Urchin 軟體來追蹤網站，建議您使用以 urchin.js 為基礎的追蹤程式碼，以便獲得最佳成效。 如果您選擇使用 ga.js，將無法使用新的 ga.js 追蹤功能。

若要在已透過 Google Analytics (分析) 進行追蹤的網站上使用 Urchin，只要稍微修改 Google Analytics (分析) 現有的追蹤程式碼即可， 步驟如下：

  • 從 Urchin 發行版的 util/utm 目錄中，只將 __utm.gif 檔案複製到網站的文件根目錄 (例如網站內容的最上層目錄)。

  • 在網頁上現有的追蹤程式碼中，加入以下的 _userv=2; 程式行。 千萬不要修改現有追蹤程式碼的其他部分！ 

<script src="http://www.google-analytics.com/urchin.js"
type="text/javascript">
<script type="text/javascript">
  _userv=2;
  _uacct="UA-XXXXXXX-Y";
  urchinTracker();
</script>
  • 將設定檔新增至 Urchin 時，請務必指定 Urchin 流量監視器 (UTM) 做為「訪客追蹤方式」

Windows Vista 和 Windows Server 2008 適用的額外安裝步驟

除了原本支援的 Windows 版本 (2000、XP 和 Server 2003) 之外，Urchin 6.6+ 還支援 Windows Vista 和 Windows Server 2008。 不過，這些新版 Windows 系統中的某些安全性功能可能會影響 Urchin 的執行作業。 以下是對此問題的詳細說明。

使用者帳戶控制 (UAC) – 相關問題

Windows 2008 新增了一個叫做 User Access Control (使用者帳戶控制，UAC) 的安全性功能，詳情請參閱 http://technet.microsoft.com/en-us/library/cc709691.aspx。

啟用 UAC 時的已知問題：

  1. . 啟用 UAC 時，應用程式無法正確寫入至 "Program files" 資料夾。 詳情請參閱： http://msdn.microsoft.com/en-us/library/aa905330.aspx

  2. . 根據預設，Urchin 的執行檔無法從命令提示字元執行。 您必須以管理員身份使用命令提示字元，或是按照下列方式設定執行檔的屬性： 按一下滑鼠右鍵並選取 [內容] > [相容性] > [以管理員的身份執行此程式] > [確定]。 請注意，Urchin Windows 服務會自行運作，您不需進行其他動作。

UAC - 解決方案

  1. 停用 UAC 並將 Urchin 安裝到您想要的位置， 然後透過二進位編碼檔案或 Windows 服務執行 Urchin。
  2. 如果啟用了 UAC，千萬不要將 Urchin 安裝在 "Program files" 資料夾中， 最好安裝到 Users 目錄等其他位置。 您可以修改屬性以便在管理員模式中執行二進位編碼檔案 (請參閱第 2 個已知問題的說明)，或者不進行任何變更而直接使用 Windows 服務。
