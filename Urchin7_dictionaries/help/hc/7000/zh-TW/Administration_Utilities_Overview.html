管理公用程式總覽

總覽

Urchin 提供了多種可用於診斷和設定的公用程式。 這些公用程式可以在 Urchin 發行版的 util 目錄中找到。 本文件只是介紹這些公用程式的概貌， 並不提供全面的說明。

所有公用程式皆支援 "-h" 和 "-v" 指令行引數。 呼叫公用程式時只要加上 "-h" 引數，即可取得該工具的使用摘要以及可用選項。 呼叫公用程式時加上 "-v" 引數即可顯示公用程式的 Urchin 版本。

geo-update

此公用程式可檢查是否有 Urchin 內部 DNS 資料庫的更新並進行下載。 您可以透過它向 Google 的下載伺服器或某些本機目錄 (urchin 6 的新功能) 下載更新。

inspector

此公用程式可以對已安裝的 Urchin 分佈執行基本檢查，進而確保此發行版本的整體架構完好、確認此產品本身的二進位資料版本無誤，且 UNIX 類型平台上執行的所有基本權限皆正確無誤。 此公用程式也可以報告 Urchin 設定檔排程器 (urchind)、單次點擊費用排程器 (urchincpcd 和 urchincpc) 以及隨附的 Apache 網頁伺服器 (urchinwebd) 的作業狀態， 並能驗證授權資訊。

uconf-manager (Urchin 6.402+)

您可透過 Urchin Configuration Manager 公用程式 (uconf-manager) 來新增、編輯和删除使用者記錄。

  • Urchin 6.402：此公用程式最初是在 Urchin 6.402 中添加，僅可用於定資料庫的 'uusers' 表格。 您可以透過此公用程式重設 Urchin 管理員密碼、將使用者建立程序自動化，或是將使用者移至其他帳戶。
  • Urchin 6.500+：在 urchin 6.5 及之後的更新版本中，此公用程式可支援更多種表格 (如 uaccounts、ucosts、ufilters、ugoals、ugroups、ugroups_uusers、ulogs、ulogs_ufilters、uprofiles、uprofiles_ucosts、uprofiles_ufilters、uprofiles_ugroups、uprofiles_ulogs、uprofiles_uusers 和 uusers) 的資料庫作業， 還可執行透過 Urchin 管理介面無法進行的動作。

uconf2sql

Urchin 5 和其他舊版本程式將組態資訊儲存在適當的檔案中，Urchin 6 則會將此資訊儲存在關聯式資料庫 (MySql 或 PostgresSql) 中。 為了順利移轉組態資訊，Urchin 5 和 Urchin 6 分別提供了用於匯出和匯入設定資訊公用程式。Urchin 5 的公用程式為 uconf-export，可將組態資料庫匯出至檔案 (匯出資料為 XML 格式)，而 Urchin 6 的 uconf2sql 公用程式則可將此 XML 格式檔案中的組態資料匯入 Urchin 6 資料庫。

convert-u5data

與組態資料庫類似，Urchin 6 中的報表資料庫 (用來儲存報表資訊) 的架構也發生了變化。 透過convert-u5data 公用程式，您可以將報表資料庫直接從已安裝的 Urchin 5 匯入 urchin 6。

udb-sanitizer

udb-sanitizer 公用程式可對報表資料庫執行相關的管理作業， 包括還原封存的資料庫，以及在需要重新處理時刪除特定日期或日期範圍的資訊。

urchinctl

(請注意： 此公用程式位於 bin 目錄，而非 util 目錄)。 此公用程式可用於開始或停止 Urchin 排程器和 Urchin 網頁伺服器的各項服務。 對於 UNIX 系統，urchinctl 通常會由系統的某個開機時期指令碼叫用 ，以自動啟動或關閉 Urchin 服務。 對於 windows 平台，這個公用程式還可用於安裝和解除安裝 urchin 服務。

urchincpc (Urchin 6.5+)

(請注意： 此公用程式位於 bin 目錄，而非 util 目錄)。 「Urchin 單次點擊費用資料匯入工具」可透過 Google AdWords API 擷取單次點擊費用資料，以便進一步處理並顯示在報表中。 如需相關詳情，請參閱「單次點擊費用資料匯入公用程式」。

urchincpcd (Urchin 6.5+)

(請注意： 此公用程式位於 bin 目錄，而非 util 目錄)。 「Urchin 單次點擊費用來源排程器」 (簡稱為「單次點擊費用來源排程器」) 可根據使用者定義的時程安排單次點擊費用下載工作 (運作上類似「設定檔排程器」)。 如需相關詳情，請參閱「單次點擊費用資料匯入公用程式」。

urchin_daemons

對於 UNIX 類型的機器，Urchin 會執行排程常駐程式 (urchind) 和 Apache 網頁伺服器 (urchinwebd)。 這些程式會在開機時期啟動。 urchin_daemons 指令碼可加入相應 UNIX 類型 OS 適合位置的系統初始化指令碼中，這樣即可正確啟動 Urchin 服務常駐程式。 對於 Windows 平台，這些二進位資料會作為系統服務執行，並可透過 Windows 服務管理員啟動和中止。

ugetlicense

「Urchin 授權作業公用程式」 (又稱為 ugetlicense) 可讓您按序號執行作業。 若是發生目前安裝的 Urchin 無法繼續使用，需要重新安裝或移轉至其他伺服器等情況，這個公用程式就可以派上用場。
