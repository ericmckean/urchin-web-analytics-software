失敗的 FTP 和 HTTP 遠端網站伺服器記錄移轉之錯誤代碼清單

總覽

您可以設定 Urchin 記錄來源，透過 FTP 或 HTTP 收集來自遠端伺服器的網站伺服器記錄。在一般情況下，移轉應該成功進行，執行時期記錄中也不會出現錯誤。但是，移轉中如果發生錯誤 (例如，出現無效的使用者名稱 / 密碼、無法連接遠端伺服器、無法讀取遠端記錄等等)，Urchin 就會將錯誤代碼記錄在執行時期輸出中，以可檢視檔案的方式儲存在設定檔的工作記錄中。嘗試進行網站伺服器記錄移轉後，此錯誤代碼會以括號括起，出現在 "failed" 訊息後，例如 (-9)

下列的錯誤代碼和文字訊息會解釋所發生的問題。

錯誤代碼清單

CURLE_UNSUPPORTED_PROTOCOL (1) - 您傳送給 Libcurl 的網址使用了 Libcurl 不支援的通訊協定。支援項目可能是您未選擇使用的編譯時間選項，這個錯誤可能是由於拼字不正確的通訊協定字串，或是 Libcurl 中沒有通訊協定的對應程式碼而產生。

CURLE_FAILED_INIT (2) - 早期初始化程式碼失敗。這可能是內部錯誤或問題。

CURLE_URL_MALFORMAT (3) - 網址並未正確格式化。

CURLE_COULDNT_RESOLVE_PROXY (5) - 無法解析 Proxy。無法解析指定的 Proxy 主機。

CURLE_COULDNT_RESOLVE_HOST (6) - 無法解析主機。無法解析指定的遠端主機。

CURLE_COULDNT_CONNECT (7) - 無法連接() 至主機或 Proxy。

CURLE_FTP_WEIRD_SERVER_REPLY (8) - 連接至 FTP 伺服器後，Libcfurl 預期接收特定回覆。此錯誤代碼表示收到的回覆不對或不正確。指定的遠端伺服器可能不是適當的 FTP 伺服器。

CURLE_REMOTE_ACCESS_DENIED (9) - 存取網址提供的資訊時被拒在 FTP 上，這個問題可能在您嘗試變更至遠端目錄時發生。

CURLE_FTP_WEIRD_PASS_REPLY (11) - 傳送 FTP 密碼至伺服器後，Libcurl 預期接收正確的回覆。這個錯誤代碼表示傳回了預期外的代碼。

CURLE_FTP_WEIRD_PASV_REPLY (13) - Libcurl 無法從伺服器端接收有意義的結果，以做為 PASV 或 EPSV 指令的回應。伺服器產生問題。

CURLE_FTP_WEIRD_227_FORMAT (14) - FTP 伺服器傳回 227 行做為 PASV 指令的回應。Libcurl 若無法剖析此行，就會傳回此傳回碼。

CURLE_FTP_CANT_GET_HOST (15) - 內部出現錯誤，找不到新的連線使用的主機。

CURLE_FTP_COULDNT_SET_TYPE (17) - 嘗試將移轉模式設定為二進位或 ascii 時發生錯誤。

CURLE_PARTIAL_FILE (18) - 檔案移轉的大小小於或大於預期。當伺服器回報預期移轉的大小，而傳送的資料卻不符合之前指定的大小時，就會發生此錯誤。

CURLE_FTP_COULDNT_RETR_FILE (19) -"RETR" 指令收到不正確的回覆，或是完成了大小為 0 位元的移轉。

CURLE_QUOTE_ERROR (21) - 傳送自訂「QUOTE」指令至遠端伺服器時，其中一個指令傳回的錯誤代碼大於或等於 400 (在 FTP 上)，或是指出指令無法成功完成。

CURLE_HTTP_RETURNED_ERROR (22) - CURLOPT_FAILONERROR 若是設定為「TRUE」，且 HTTP 伺服器傳回 >= 400 的錯誤代碼，就會傳回此錯誤 (此錯誤代碼原本是 CURLE_HTTP_NOT_FOUND)。

CURLE_WRITE_ERROR (23) - 將已收到的資料寫入本機檔案時發生錯誤，或是寫入回呼將錯誤傳回 libcurl。

CURLE_UPLOAD_FAILED (25) - 無法開始上載。在 FTP 上，伺服器一般會拒絕「STOR」指令。錯誤緩衝區中一般會包含伺服器對此的解釋 (此錯誤代碼原本是 CURLE_FTP_COULDNT_STOR_FILE)。

CURLE_READ_ERROR (26) - 讀取本機檔案時發生問題，或是讀取回呼傳回錯誤。

CURLE_OUT_OF_MEMORY (27) - 記憶體配置要求失敗。這個問題十分重大，若發生此問題，代表內部發生嚴重問題。

CURLE_OPERATION_TIMEDOUT (28) - 作業逾時。按照條件而言，已達到指定的逾期時間。

CURLE_FTP_PORT_FAILED (30) - FTP 通訊埠指令傳回錯誤。這個問題通常在您尚未為 Libcurl 指定正確地址時發生。請參閱 CURLOPT_FTPPORT。

CURLE_FTP_COULDNT_USE_REST (31) - FTP 的 REST 指令傳回錯誤。如果伺服器正常的話，這種情況不應該發生。

CURLE_RANGE_ERROR (33) - 伺服器不支援或接受範圍要求。

CURLE_HTTP_POST_ERROR (34) - 這個錯誤較為特別，一般發生在發生內部混淆的情況下。

CURLE_SSL_CONNECT_ERROR (35) - 此錯誤發生在 SSL/TLS 交集處。由於錯誤緩衝區較能夠指出問題所在，因此您希望使用錯誤緩衝區並在此讀取訊息。問題的可能原因是憑證 (檔案格式、路徑或權限) 或是密碼等等。

CURLE_FTP_BAD_DOWNLOAD_RESUME (36) - 嘗試進行超過檔案大小的 FTP 恢復。

CURLE_FILE_COULDNT_READ_FILE (37) - 無法開啟 FILE:// 路徑下的檔案。原因可能是檔案路徑無法辨識已存在的檔案。您已經確認過權限了嗎？

CURLE_LDAP_CANNOT_BIND (38) - LDAP 無法連接。LDAP 連接作業失敗。

CURLE_LDAP_SEARCH_FAILED (39) - LDAP 搜尋失敗。

CURLE_FUNCTION_NOT_FOUND (41) - 找不到此函數。找不到所需的 zlib 函數。

CURLE_ABORTED_BY_CALLBACK (42) - 由回呼中止。回呼將「中止」傳回給 Libcurl。

CURLE_BAD_FUNCTION_ARGUMENT (43) - 內部錯誤。使用不正確的參數呼叫了函數。

CURLE_INTERFACE_FAILED (45) - 介面錯誤。無法使用指定的對外介面。使用 CURLOPT_INTERFACE 設定用來處理對外連線之來源 IP 位址的介面 (此錯誤代碼原本是 CURLE_HTTP_PORT_FAILED)。

CURLE_TOO_MANY_REDIRECTS (47) - 發生太多重新導向。進行下列重新導向時，Libcurl 達到最大數量。請使用 CURLOPT_MAXREDIRS 設定上限。

CURLE_UNKNOWN_TELNET_OPTION (48) - 無法辨識 / 未知的某個以 CURLOPT_TELNETOPTIONS 設定的選項。請參閱相關的文件。

CURLE_TELNET_OPTION_SYNTAX (49) - 以不當方式格式化的 Telnet 選項字串。

CURLE_PEER_FAILED_VERIFICATION (51) - 遠端伺服器的 SSL 憑證檔案或 SSH md5 特徵不正確。

CURLE_GOT_NOTHING (52) - 伺服器未傳回任何事項，在這種情況下，未取得任何事項可視為錯誤。

CURLE_SSL_ENGINE_NOTFOUND (53) - 找不到指定的密碼引擎。

CURLE_SSL_ENGINE_SETFAILED (54) - 無法將已選取的 SSL 密碼引擎設為預設選項。

CURLE_SEND_ERROR (55) - 無法傳送網路資料。

CURLE_RECV_ERROR (56) - 接收網路資料時出現錯誤。

CURLE_SSL_CERTPROBLEM (58) - 本機端憑證檔案出現問題。

CURLE_SSL_CIPHER (59) - 無法使用指定的密碼。

CURLE_SSL_CACERT (60) - 對等的憑證檔案無法驗證為已知的 CA 憑證檔案。

CURLE_BAD_CONTENT_ENCODING (61) - 無法辨認的移轉編碼

CURLE_LDAP_INVALID_URL (62) - 無效的 LDAP 網址

CURLE_FILESIZE_EXCEEDED (63) - 超出最大檔案大小

CURLE_USE_SSL_FAILED (64) - 要求的 FTP SSL 層級失敗。

CURLE_SEND_FAIL_REWIND (65) - 進行傳送作業時，curl 必須倒轉資料以進行重新轉送，但倒轉作業失敗

CURLE_SSL_ENGINE_INITFAILED (66) - 初始化 SSL 引擎失敗

CURLE_LOGIN_DENIED (67) - 遠端伺服器拒絕讓 curl 登入 (新增於 7.13.1)

CURLE_TFTP_NOTFOUND (68) - TFTP 伺服器上找不到檔案

CURLE_TFTP_PERM (69) - TFTP 伺服器上發生權限問題

CURLE_REMOTE_DISK_FULL (70) - 伺服器上磁碟空間不足

CURLE_TFTP_ILLEGAL (71) - 不合法的 TFTP 作業

CURLE_TFTP_UNKNOWNID (72) - 未知的 TFTP 移轉 ID

CURLE_REMOTE_FILE_EXISTS (73) - 檔案已經存在，將不會進行覆寫

CURLE_TFTP_NOSUCHUSER (74) - 只有運作不正常的 TFTP 伺服器才會傳回此種錯誤

CURLE_CONV_FAILED (75) - 字元轉換失敗

CURLE_CONV_REQD (76) - 呼叫端必須註冊轉換回呼

CURLE_SSL_CACERT_BADFILE (77) - 讀取 SSL CA 憑證檔案時發生問題 (路徑錯誤？您具有存取權限嗎？)

CURLE_REMOTE_FILE_NOT_FOUND (78) - 在網址中參考的資源不存在

CURLE_SSH (79) - 無法辨識的錯誤在 SSH 工作階段中發生

CURLE_SSL_SHUTDOWN_FAILED (80) - 無法中止 SSL 連線
