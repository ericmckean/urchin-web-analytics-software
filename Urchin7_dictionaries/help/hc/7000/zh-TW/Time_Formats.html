自訂日期 / 時間格式

簡介

Urchin 可以虛擬方式處理記錄檔中任何的日期或時間格式。您只需提供 Urchin 必要的日期或時間格式 (此格式必須符合記錄檔中的日期格式)。本文將說明用來指定日期或時間格式的變數。這些格式可在自訂記錄格式檔中指定。請參閱本節上面的「自訂記錄格式」一文。

日期 / 時間剖析的運作方式

Urchin 會將記錄檔中的日期 / 時間欄位與指定格式相互比較，藉以判斷日期 / 時間。

例如，IIS 記錄中包含日期，格式如下： 2002-11-12 Urchin 就可以使用下列格式，判斷其年、月及日。 %Y-%m-%d

建立日期 / 時間格式

若要建立自訂日期 / 時間格式，首先您必須檢視記錄檔中所包含之日期 / 時間資料的順序和模式。然後，選取下列日期 / 時間變數，並建立格式。時區會以 UTC (國際標準時間) 的正負位移方式表示。

例如，如果您的記錄檔中包含的時間為 "07:01:47"，您就需要建立與它相符的模式。首先，您會注意到此格式是以「小時:分鐘:秒鐘」的方式排列。請檢視下列變數，您會發現 %H 代表小時變數，%M 是分鐘變數，而 %S 則是秒的變數。將它們以「%H:%M:%S」的方式組合。若是日期 / 時間欄位中包含 '%' 字元，您可以將此 % 字元指定為 %%。

最常用的變數有：%Y、%m、%d、%H、%M 和 %S。

日期 / 時間變數定義

  • %A = 完整工作日名稱的全國表示方法。
  • %a = 縮寫營業日名稱的全國表示方式。
  • %B = 完整月份名稱的全國表示方式。
  • %b = 縮寫月份名稱的全國表示方式。
  • %d = 月份日期以小數點表示 (01-31)。
  • %e = 月份日期以小數點表示 (1-31)，單一數字前請保留空白。
  • %H = 小時 (24 小時制) 以小數點表示 (00-23)。
  • %I = 小時 (12 小時制) 以小數點表示 (01-12)。
  • %j = 年份日期以小數點表示 (001-366)。
  • %k = 小時 (24 小時制) 以小數點表示 (0-23)，單一數字前請保留空白。
  • %l = 小時 (12 小時制) 以小數點表示 (1-12)，單一數字前請保留空白。
  • %M = 分鐘以小數點表示 (00-59)。
  • %m = 月份以小數點表示 (01-12)。
  • %p = 依時段代表「上午」或「下午」的全國表示方式。
  • %S = 秒數以小數表示 (00-60)。
  • %s = 從世界標準時間 (UTC) Epoch 以來的秒數 (請參閱 mktime(3))。
  • %w = 星期幾 (星期日是一週的第一天) 以小數表示 (0-6)。
  • %Y = 年份 (含世紀) 以小數表示。
  • %y = 年份 (不含世紀) 以小數表示 (00-99)。
  • %z = 世界標準時間 (UTC) 時區設定，開頭的加號代表 UTC 東邊時間，減號代表 UTC 西邊時間，接下來的數字代表小時和分鐘 (分別以兩個數字表示)，且不需以分隔字元區隔 (RFC 822 日期標題的常見形式)。
  • %% = `%'。
