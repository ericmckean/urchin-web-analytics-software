單次點擊出價資料匯入公用程式 (Urchin 6.5+)

Urchin 6.5 中添加了用於處理單次點擊出價資料的公用程式。

urchincpc -「Urchin 單次點擊出價資料匯入工具」可透過 Google AdWords 和 Yahoo! 奇摩搜尋行銷 API 擷取單次點擊出價資料，以便進一步處理並產生報表。

  • 請注意：下載自 AdWords 的封存單次點擊出價資料大小不可超過 250 MB (Urchin 6.501+)。

urchincpcd -「Urchin 單次點擊出價來源排程器」(簡稱為「單次點擊出價排程器」) 會依使用者定義的時間來安排單次點擊出價下載工作，這一點與「單次點擊出價排程器」類似。

「單次點擊出價排程器」(urchincpcd) 可在本機電腦上與「Urchin 主排程器」(urchind) 一併執行。只有在「主排程器」已啟動並正在執行的情況下，才能安排下載單次點擊出價來源。「單次點擊出價排程器」會定期檢視佇列中是否有單次點擊出價資料下載工作，並控制這些工作的執行狀況。

發現有工作安排後，「單次點擊出價匯入工具」(urchincpc) 就會自動啟動。「單次點擊出價排程器」可以啟動內部工作程序，不過，您也可以透過指令行直接呼叫這些工作程序，並開始執行佇列中的任何工作。

下載完成的單次點擊出價資料會儲存在本機的 /path/to/urchin/data/cpc/ 路徑中，並會在系統處理連結的設定檔時由 Urchin 引擎自動擷取。

「Urchin 單次點擊出價排程器」是由 urchinctl 公用程式管理。若要單獨使用「單次點擊出價來源排程器」，請使用 "-c" 選項。例如： 

urchinctl -c start/stop/restart

這樣可以啟動/中止/重新啟動「單次點擊出價排程器」。

若要立即安排下載單次點擊出價資料，請依序選擇 [組態] > [單次點擊出價來源管理工具] > [下載] 或 [編輯] > [時段設定] 分頁以選擇特定單次點擊出價來源，然後按一下 [立即下載] 按鈕。請注意，只有在已經啟動並執行「單次點擊出價排程器」的情況下才能處理此要求。

使用方式：單次點擊出價排程器 - urchincpcd

「單次點擊出價排程器」位在 Urchin 6.5+ 安裝的 [bin] 資料夾中。您可以透過指令行使用下列選項：

  • -d 啟動偵錯模式。詳細偵錯資訊均記錄在輸出結果中。對於 Windows 系統，此選項必須與 –console 一起使用，例如： 

    •            urchincpcd.exe -console -d

  • -h 列印說明資訊然後離開

* -v 列印版本編號然後離開 在 Windows 系統下，請使用下列其中一項指令做為第一個引數：

  • -install 在 Windows 系統下安裝「單次點擊出價排程器」

  • -remove 在 Windows 系統下移除「單次點擊出價排程器」

  • -start 在 Windows 系統下啟動「單次點擊出價排程器」

  • -stop 在 Windows 系統下中止「單次點擊出價排程器」

  • -console 以主控台模式啟動「單次點擊出價排程器」

使用方式：單次點擊出價匯入工具 - urchincpc

您可以透過指令行使用下列選項：

  • -c [id] 下載特定費用來源資料，"id" 是單次點擊出價來源編號

  • -d 啟用偵錯模式

  • -h 列印說明資訊然後離開

  • -v 列印版本編號然後離開

請注意，如果您並未使用參數啟動「單次點擊出價排程器」，或是只使用了 -d 選項，排程器就會在佇列中任意挑選現有工作開始處理，並在所有工作處理完成後自行停止。

透過 Proxy (6.501+) 下載單次點擊出價資料

若要透過 Proxy 啟用單次點擊出價資料下載，請編輯 urchin.conf 檔案，移除 WebServiceProxyHost、WebServiceProxyPort、WebServiceProxyUsername 和 WebServiceProxyPassword 項目的註解，並提供您想要使用的值。 

 ######################################################
# CPC Source Manager Settings
#...
# WebServiceProxyHost, WebServiceProxyHost: Host and port of the HTTP/HTTPS
# proxy CPC Source Manager must use to access external CPC sources.
# WebServiceProxyUsername, WebServiceProxyPassword: username and password for
# proxy that enforces authorization.
######################################################
#WebServiceProxyHost: <proxy host>
#WebServiceProxyPort: <proxy port>
#WebServiceProxyUsername: <username>
#WebServiceProxyPassword: <password>

針對 WebServiceProxyHost，您可以直接輸入 IP 位址或完整網域名稱。
