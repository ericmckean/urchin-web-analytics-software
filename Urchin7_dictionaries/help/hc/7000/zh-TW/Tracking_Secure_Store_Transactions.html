追蹤安全商店交易

本文由 Urchin 軟體授權顧問 ivantage Limited 撰寫

只要加上一些程式碼，Urchin 即可追蹤來自您所屬網域或是其他網域上的購物車資料。若要設定讓您的網頁向 Urchin 回報交易資料，請參閱如何追蹤電子商務交易？

此外，如果您的商店網站位於與您的主要網站不同的子網域或網域中，請按照下列指示設定您的網頁。

如果您的網站在其他子網域上起始購買結帳程序 (例如將客戶從 www.mystore.com 導向到 shoppingcart.mystore.com)：

  • 請將下列程式碼 (粗體顯示) 添加到您商店網站和購物車網頁的追蹤程式碼中： 

   <script src="/urchin.js" type="text/javascript">
   </script>
   <script type="text/javascript">
    _udn="mystore.com";
   </script>

如果您的網站在其他商店網站上起始購買結帳程序 (例如將客戶從 www.mystore.com 導向到 www.securecart.com)：

  • 請將下列程式碼 (粗體顯示) 添加到您商店網站和購物車網頁的追蹤程式碼中： 

     <script src="/urchin.js" type="text/javascript">
     </script>
     <script type="text/javascript">
       _udn="none";
       _ulink=1;
     </script>

  • 如下所示，使用 utmLinker 將主要網站改為連結至安全網站。例如，如果您目前的連結為： 

     <a href="https://www.securecart.com/?store=parameters">立即購買</a>
  • 改成： 

   <script type="text/javascript">
   document.write('<a href="javascript:__utmLinker(\'https://www.securecart.com/?store=parameters\');">立即購買</a>');
   </script>
   <noscript>
   <a href="https://www.securecart.com/?store=parameters">立即購買</a>
   </noscript>

  • 無論是否啟用了 JavaScript，上述程式碼都可以為使用者提供連結。請務必注意，當簡縮號 (') 出現在連結或連結文字裡中時，應該使用反斜線 (\) 來逸出。 重要：如果網頁中包含對 urchinTracker()、utmLinker()、utmSetTrans() 或 utmLinkPost() 的呼叫，那麼您必須將 Analytics (分析) 追蹤程式碼放在 HTML 程式碼中這些呼叫的前面。在這些狀況下，追蹤程式碼可以放在開始標記 <body> 和 JavaScript 呼叫之間的任何位置。

如果您利用表單將資訊傳送到購物車，請使用 utmLinkPost() 函數，以便將 Cookie 資料附加到所提供表單的網址中。 

   <form action="http://newdomain.com/form.cgi" onSubmit="javascript:__utmLinkPost(this)">
  • 這個方法也適用於 method="GET" 的表單
