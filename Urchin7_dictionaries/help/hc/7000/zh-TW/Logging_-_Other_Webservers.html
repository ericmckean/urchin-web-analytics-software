記錄 - 其他網頁伺服器

總覽

本文提供如何設定 Apache 與 IIS 以外網頁伺服器的簡短總覽，以促進 Urchin 的正常運作與報告。只要可以瞭解各記錄檔項目中的資料組織方式，Urchin 就能處理任何網頁伺服器記錄。本文中的資訊僅適用於存取記錄。如果您對電子商務記錄詳細資料感興趣，請參閱說明文件中心的「電子商務模組」部分。

無論您的網頁伺服器類型或記錄格式為何，Urchin 都需要一些必填欄位才能產生報告。包括：

  • 日期
  • 時間
  • 要求系統的主機名稱或 IP 位址
  • 要求 (亦即要求系統向網頁伺服器要求的是哪一份文件)
  • 要求產出的狀態代碼 (數字)
  • 位元組 (從伺服器轉移至用戶端的位元組)

此外，針對多數完整報告，您需提供以下欄位：

  • 推薦連結
  • 使用者代理程式
  • Cookie (如果網站上裝有 Urchin 流量監視器)

組態

如何針對各網頁伺服器進行記錄特徵變更的確切方式，實在一言難盡。一般來說，最簡單的方式就是按照 Urchin 的 NCSA 或 W3C 表單設定記錄，然後從「記錄來源」的「記錄格式」下拉式選單中選擇恰當的預設格式。如果您的網頁伺服器支援此方式，接著請參閱「記錄 - Apache 與 IIS」文件。即使變更方式的詳細資料可能不適用於您的網頁伺服器，文件資訊中關於如何設定必要資料欄位的說明，對您來說可能仍相當實用。
