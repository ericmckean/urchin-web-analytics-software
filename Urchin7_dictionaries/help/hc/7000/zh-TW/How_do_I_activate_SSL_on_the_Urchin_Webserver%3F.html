如何在 Urchin 網頁伺服器上啟用安全資料傳輸層 (SSL)？

Urchin 隨附的 Urchin 網頁伺服器能夠透過安全資料傳輸層 (SSL) 加密通訊內容。若要啟用 SSL，您需要一份由認證機構簽署的有效憑證或是自我簽署憑證。

在 Urchin 網頁伺服器上啟用 SSL 的步驟如下：

  1. 將 SSL 憑證檔案複製到 Urchin var 目錄，並將其命名為 server.crt
  2. 將 SSL 金鑰檔案複製到 Urchin var 目錄，並將其命名為 server.key
  3. 編輯位於 Urchin var 目錄中的 urchinwebd.conf.template 檔案。
    1. (如果使用的是 Urchin 6.6+，請略過此步驟) 把通訊埠行標示成註解，並移除 Listen 指令的註解標示。 
                #XXXPortXXX
          Listen 443

    2. 將 ServerName 指令從 localhost 變更為您網頁伺服器的名稱。例如： 

                ServerName: www.urchin.com

    3. 請注意：urchinwebd.conf.template 檔案中的 ServerName 必須與憑證檔案中的伺服器名稱一致。4. 使用 urchinctl 以及 -e 選項啟動或重新啟動網頁伺服器。Urchinctl 位於 Urchin 的 bin 目錄中。-e 選項會指示 urchinctl 啟用網頁伺服器中的 SSL。例如，若要在通訊埠 443 將啟用 SSL 的網頁伺服器重新啟動：

    Urchin 6.6 之前的版本： 
       urchinctl -e -w restart
    Urchin 6.6+： 
       urchinctl -e -p 443 -w restart

    請注意：若要啟動伺服器但不啟用 SSL，請省略 -e 選項。

這樣，您應該就可以透過 https://servername.domain.com:port/ 存取啟用 SSL 的伺服器了

請注意：在 urchinwebd.conf.template 中自訂 SSL 設定可能會導致網頁伺服器無法啟動的問題。

Windows 指示

在 Urchin 網頁伺服器上啟用 SSL 的步驟如下：

  1. 將 SSL 憑證檔案複製到 Urchin var 目錄，並將其命名為 server.crt
  2. 將 SSL 金鑰檔案複製到 Urchin var 目錄，並將其命名為 server.key
  3. 編輯位於 Urchin var 目錄中的 urchinwebd.conf.template 檔案。
    1. (如果使用的是 Urchin 6.6+，請略過此步驟) 把通訊埠行標示成註解，並移除 Listen 指令的註解標示。 
                #XXXPortXXX
          Listen 443
    2. 當網頁伺服器啟動時，移除下面這行程式碼的註解以載入 mod_ssl.so： 
                LoadModule ssl_module etc/mod_ssl.so

    3. 將 ServerName 指令從 localhost 變更為您網頁伺服器的名稱。例如： 

                ServerName: www.urchin.com

    4. 請注意：urchinwebd.conf.template 檔案中的 ServerName 必須與憑證檔案中的伺服器名稱一致。4. 使用 urchinctl 以及 -e 選項移除和安裝網頁伺服器服務。Urchinctl 位於 Urchin 的 bin 目錄中。-e 選項會指示 urchinctl 啟用網頁伺服器中的 SSL。例如，若要在通訊埠 443 移除和安裝已啟用 SSL 的網頁伺服器：

    Urchin 6.6 之前的版本： 
       urchinctl.exe -w remove
   urchinctl.exe -w -e install
    Urchin 6.6+: 
       urchinctl.exe -w remove
   urchinctl.exe -e -p 443 -w install

    請注意：若要啟動伺服器但不啟用 SSL，請省略 -e 選項。

這樣，您應該就可以透過 https://servername.domain.com:port/ 存取啟用 SSL 的伺服器了

請注意：在 urchinwebd.conf.template 中自訂 SSL 設定可能會導致網頁伺服器無法啟動的問題。
