Why are "/" and "/index.htm" tracked separately in my Urchin reports, even though they're the same page?

On the Reporting tab under the Configuration settings for each profile, there is a setting called "Default Page". This should be set to whatever the default (or index) page is for your site. This information allows Urchin to reconcile log entries such as http://www.urchin.com/ and http://www.urchin.com/index.html, which are in fact the same page. Without the Default Page information entered correctly, these would be reported as two distinct pages. Only a single default page should be specified.
