處理歷史記錄

總覽

您安裝 Urchin 之後，最好處理歷史記錄。方法非常簡單，只要在「記錄管理員」的「記錄設定」畫面中指定目錄與部分檔名和 / 或萬用字元 (包括正規表示法) 即可。請注意：不可在遠端 HTTP 與 HTTPS 記錄來源使用萬用字元。

如何處理歷史記錄

首先，請新增「記錄來源」至系統。按一下上方選單的 [組態] 按鈕，然後按一下 [Urchin 設定檔] > [記錄管理員] 連結。在主畫面上，按一下右上角的 [新增] 按鈕。在第一個畫面上，選取 [新增本機記錄來源]，然後繼續。在接下來的畫面上按一下 [瀏覽]，就會帶出「檔案瀏覽器」。在左側視窗中找出正確的目錄。右側視窗會顯示該目錄中的檔案，左側會顯示目前開啟目錄的其他所有子目錄。您在正確的目錄中時，請輸入部分檔名和星號 (或其他正規表示法)，然後按一下 [確認] 按鈕。接著會開啟一個視窗，顯示與您的模式相符的所有資料。按一下任一檔名取得檔案資訊，包括位置、大小、上次修改日期以及檔案權限。如果模式比對正確，請按一下 [確定]，然後於「檔案瀏覽器」視窗內再按一次 [確定]。之後，請指定「記錄來源名稱」並按一下 [下一步]。從下拉式清單中指定記錄格式，然後按一下 [完成] 按鈕。如果「記錄來源名稱」有效 (應為專屬名稱)，請按下 [完成] 按鈕，然後您會看見自己的記錄來源「記錄管理員」清單。

接著，如果尚未完成，請建立此記錄檔與「設定檔」的關聯性，只要按一下上方選單的 [組態] 按鈕，然後按 [Urchin 設定檔] > [設定檔] 按鈕即可。關聯性建立之後，請在主要「設定檔」列表中的「設定檔」旁，按一下 [執行 / 排程] 按鈕，您就可以排定執行 [設定檔] 的時程，或按一下 [立即執行] 按鈕馬上開始處理。

Urchin 不需排定任何記錄輪替順序，來避免資料重複。Urchin 擁有記錄追蹤能力，可以確保僅處理新的點擊。不過，如前所述，記錄可能在短時間內快速消耗大量硬碟空間，所以您不妨定期壓縮並封存記錄檔。因為 Urchin 在處理完畢之後從來不需重新讀取記錄檔，所以您可以在每次執行處理之後，放心刪除記錄。但是，許多人習慣保有記錄一段特定時間，以防因為某些原因需要回頭使用這些檔案：例如為該網站建立新的「設定檔」，或是要進行歷史分析。

建議

記錄管理最初並非必要工作，但隨著記錄資料的增長，妥善管理已成為關鍵要務。建議您從一開始部署 Urchin 時，就先決定好記錄管理計劃。
