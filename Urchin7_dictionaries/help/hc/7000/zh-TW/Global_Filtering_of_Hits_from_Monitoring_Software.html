為監控軟體的點擊進行全域篩選

總覽

大多數的代管環境都提供某類型的客戶網站伺服器監控，以便維護服務等級協定 (SLA)。但是它有其缺點，來自此監控服務的點擊數會造成針對這些受監控的網站之 Urchin 報告的誤差，造成因人為因素而偏高不實的工作階段、網頁檢視、點擊和位元計算結果。

建議

在使用這類監控服務的代管環境中，我們強烈建議您為客戶已進行設定的 Urchin 設定檔套用標準或全域性 Urchin 篩選器，以篩去由監控軟體產生的點擊。對於使用集中式 Urchin 安裝 (由代管公司管理) 的環境，此安裝提供了各個客戶網站的報告服務，您在此可輕鬆套用這個功能。在專用 / 主機代管環境中，客戶可自行維護其伺服器上的 Urchin 例項，代管公司則需提供適合使用之監控服務的範例篩選器。

為了幫助您執行 Urchin 篩選，代管公司和客戶本身應共同合作，在客戶的網站上建立一個只有監控軟體使用的特定網頁，舉例如下： 

     http://www.customerdomain.com/healthpage.html

範例

範例 1：為監視系統篩選掉 IP 位址 

      Filter Type: Exclude
      Filter Field: IP
      Filter Spec: 172\.16\.1\.1
  • 這個方法會在 Urchin 進行處理時，同時剔除網站伺服器記錄中 IP 位址為 172.16.1.1 的點擊。

範例 2：篩選掉監視系統點擊的特定網頁 

      Filter Type: Exclude
      Filter Field: REQUEST
      Filter Spec: ^/healthpage.html
  • 這個方法會在 Urchin 進行處理時，同時剔除網站伺服器記錄中要求使用 /healthpage.html 的點擊。

注意事項

  1. 建議您在網站伺服器上為客戶建立不具備篩選功能的額外設定檔，以便客戶檢視實際的流量負載 (包含篩選)。
  2. 代管公司可能想要供應只為監控點擊提供報告功能的設定檔，意即此設定檔將只篩選來自監控軟體的點擊。此設定檔可用來顯示監控過程是否適當，以及是否達到了 SLA 要求。
