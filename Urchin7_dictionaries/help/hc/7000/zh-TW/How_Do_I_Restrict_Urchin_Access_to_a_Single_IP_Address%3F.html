如何限制 Urchin 只存取某一個 IP 位址？

總覽

在使用多個網路介面或是在一個網路介面套用多個 IP 位址的系統上，Urchin 網頁介面預設會套用所有已設定的 IP 位址。但是，建議您設定 Urchin，限制其透過單一 IP 位址進行管理和報告。

程序

若要限制 Urchin 的管理 / 報告功能，只需對 Urchin Apache 網頁伺服器所用的組態檔進行一項簡單的變更。請按照下列步驟進行：

適用於 UNIX 類型系統：

  • 以 Urchin 執行使用者的身分開啟指令殼層
  • 使用 "cd" 指令移至 Urchin 的安裝目錄
  • 使用下列指令停止 Urchin 網頁伺服器："bin/urchinctl -w stop"
  • 在 Urchin 分佈的 var 目錄中，使用文字編輯器開啟 "urchinwebd.conf.template" 檔案

  • 在 <Directory /> 這一行的上方插入下列文字行 

       Listen XXX.XXX.XXX.XXX:port
  • 其中 "XXX.XXX.XXX.XXX" 和 "port" 是只允許 Urchin 存取的 IP 位址和通訊埠號。
  • 使用下列指令重新啟動 Urchin 網頁伺服器： 

bin/urchinctl -w start

適用於 Windows 系統：

  • 停用 Urchin 服務：[開始] -> [程式集] -> [Urchin 6] -> [停用 Urchin 服務]

  • 開啟 Windows Explorer 並瀏覽至 "C:\Program Files\Urchin6\var"
  • 使用文字編輯器開啟 "urchinwebd.conf.template" 檔案

  • 在 <Directory /> 這一行的上方插入下列文字行 

       Listen XXX.XXX.XXX.XXX:port
  • 其中 "XXX.XXX.XXX.XXX" 和 "port" 是只允許 Urchin 存取的 IP 位址和通訊埠號。

  • 重新啟動 Urchin 服務：[開始] -> [程式集] -> [Urchin 6] -> [啟用 Urchin 服務]

重要注意事項：由於 IIS 5 和更新版本使用通訊端集區，無論實際設定的 IP 位址為何，您可能需要停用 IIS 的預設行為，以便在所有採用某特定通訊埠的網站上共用通訊端。此方法在使用虛擬主機的環境中特別常見，建議您使用通訊埠 80 來設定特定虛擬主機的 Urchin 報告。

注意事項

  • 此方法只能將 Urchin 管理 / 報告限制在特定的 IP 位址。您不能限制系統存取以名稱為主的虛擬主機。
  • Urchin 升級將在儲存備份副本後覆寫 "urchinwebd.conf.template" 檔案。您必須在升級後重新套用先前對該檔案所做的任何編輯。
