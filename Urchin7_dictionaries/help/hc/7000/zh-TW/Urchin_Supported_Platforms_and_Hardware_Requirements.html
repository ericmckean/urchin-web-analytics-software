Urchin 支援的平台及硬體需求

Urchin 可以在多種架構和作業系統上執行。 除非您要在系統上處理記錄，否則不需要安裝 Urchin， 只要透過網路瀏覽器即可查看報表。

支援的平台

Windows

  • Windows 2003 Server
  • Windows XP
  • Windows 2000
  • Windows Vista
  • Windows 2008 Server

UNIX-type Systems

  • Linux (kernel 2.6) - 大多數現有版本都支援
  • Linux (kernel 2.4) - 大多數現有版本都支援
  • FreeBSD 4 (不支援 6.5 及更新版本)
  • FreeBSD 5.3+ (不支援 Urchin 6.602 及更新版本)
  • FreeBSD 6.2 及更新版本

Supported databases

  • MySQL 4.1+ (不支援 Urchin 6.6 及更新版本)
  • MySQL 5.03 及更新版本
  • PostgreSQL 8.2.5

支援的瀏覽器

  • Internet Explorer 6.x, 7.x
  • Firefox 2.x, 3.x
  • Chrome 1.x, 2.x

未明確支援平台的注意事項

如果找不到特定平台適用的 Urchin 發行版，建議使用與您 OS 相容的 Urchin 發行版。

  • Windows 98, Windows 3.x:

Urchin 無法安裝在 Windows 98 或 3.x 上，但您可以在這兩個平台上使用 Internet Explorer 5.x 或更新版本來查看報表。

  • Linux：

Linux 的版本有很多種，但並非每個版本都有對應的 Urchin 發行版。 不過，Linux 版本通常都具有非常高的相容性，因此您應該可以找到與您電腦相容的 Urchin 發行版。 以下列出幾個已知相容的發行版：

  • Ubuntu (kernel 2.6)

  • RedHat Enterprise Linux 2 & higher

  • SuSE Linux 8 & higher

  • Trustix 2 & higher

  • Solaris:

我們目前不支援 Solaris。

Urchin 6 系統需求

由於 Urchin 的性能優越，只需很少的硬體投資即可獲得不錯的成效。 舉例來說，就連 Pentium II 這種因為速度太慢瀕臨淘汰的桌上型系統，也可以充當 Urchin 伺服器。 不過，為了發揮最大的成效，建議使用下列硬體：

小型至中型單一網站分析

  • 2 Ghz 或等級更高的處理器
  • 2 GB 的記憶體
  • 至少 250 GB IDE 的硬碟
  • 100 Mbs 乙太網路介面

服務供應商/企業安裝

  • 2.5 Ghz Pentium 雙核心/2 Ghz UltraSPARC/相近的 CPU 速度範圍 PPC/MIPS 等
  • 3 GB RAM
  • Ultra2/Wide SCSI 硬碟 (如 Seagate Cheetah)
  • 100 base-T 乙太網路
  • 備份系統

記憶體/系統/磁碟的使用率

  • Urchin 記憶體 (RAM) 使用率可設定在 20 至 500 MB 之間 (Urchin 6.5 則為 20 至 1024 MB 之間)
  • Urchin 的執行優先順序可設定為低、標準或高
  • Urchin 的資料儲存作業會佔用約 10% 的原始記錄空間

FreeBSD 使用者的重要注意事項

(1) FreeBSD 的固定式編碼程序有 500 MB 的資料大小上限。 不過，Urchin 的地理資料在記錄處理期間必須佔用記憶體，所以會超出這個上限， 因此，您會在執行時期看見下列錯誤： 

ERROR: (8011-323-1057) 無法配置記憶體

若要解決此問題，您必須編輯 /boot/loader.conf 檔案並加入下列這幾行程式碼，以提高 FreeBSD 系統預設的程序資料大小： 

# 將程序資料區段的上限提高為 1 GB
kern.maxdsiz="1073741824"

您必須重新開機，此設定才會生效。

警告!!! 進行這類變更系統上限的作業時請務必謹慎。 如果 kern.maxdsiz 設定太大，可能會導致系統無法開機。 我們極力建議您在進行這項修改前先參閱 FreeBSD 說明文件，並針對網站組態評估可能發生的潛在風險。

(2) FreeBSD 內含一個叫做 accf_http 的特殊驅動程式。 此驅動程式會緩衝傳入連線，直到某個完整 HTTP 要求加速並最佳化成效為止。 按照預設，此驅動程式不會載入核心，並且會出現下列警告： 

[warn] (2)No such file or directory: Failed to enable the ‘httpready’ Accept Filter

若要避免發生這種情況，您必須啟用 HTTP 接受篩選器 FreeBSD 核心模組 (accf_http)，方法是編輯 /boot/loader.conf 檔案並加入下列程式行： 

# 等待完整 HTTP 要求接受篩選器
accf_http_load="YES"

您必須重新開機，此設定才會生效。

Linux glibc 程式庫版本

若要在 Linux (kernel 2.4 or 2.6) 上安裝 Urchin 6.602 隨附的 PostgreSQL 8.4.01，必須使用 glibc 2.3.3+。
