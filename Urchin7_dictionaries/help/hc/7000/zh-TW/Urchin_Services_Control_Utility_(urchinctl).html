Urchin 服務控制公用程式 (urchinctl)

總覽

Urchin Services Control utility又名「urchinctl」，可驗證 Urchin Scheduler 和 Urchin Webserver 服務的狀態，並允許您將其啟動和關閉。 在 UNIX 系統中，urchinctl 通常由其中一個系統的開機時期指令碼叫用，藉以自動啟動或關閉 Urchin 的服務。

urchinctl 可執行的作業類型為：

  • 驗證 Urchin6 Master 和 Slave Scheduler 的狀態
  • 啟動、關閉或重新啟動排程器或網頁伺服器 (或兩者皆是)
  • 在替代埠中啟動網頁伺服器
  • 使用 SSL 加密啟動網頁伺服器

使用方式

urchinctl 位於 Urchin 分佈的 bin 目錄中。

此公用程式的使用方法如下： 

  urchinctl -h (列印用法訊息，然後離開)

  urchinctl -v (列印版本，然後離開)

  urchinctl [-e] [-p 通訊埠] [[-s] | [-w]] 動作

其中： 

  -e  啟動網頁伺服器的加密 (SSL)

  -p  指定網頁伺服器要接收的通訊埠

  -s  僅在 Urchin 排程器中執行動作

  -w  僅在 Urchin 網頁伺服器中執行動作

動作為以下之一： 

  start   (啟動服務)

  stop    (關閉服務)

  restart (關閉然後啟動服務)

  status  (顯示網頁伺服器 / 排程器執行時期狀態)

按照預設，動作會在網頁伺服器和排程器中執行，除非您指定 "-s" (排程器) 或 "-w" (網頁伺服器) 選項。 請注意，這些為互斥選項。

考量

在 UNIX 系統中，您應以安裝 Urchin 的使用者 / UID 身份來執行 urchinctl，以確保 urchinwebd 和 urchind 處理程序是以該 UID 啟動。
