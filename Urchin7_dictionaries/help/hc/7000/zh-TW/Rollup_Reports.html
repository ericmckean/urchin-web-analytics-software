彙總報表 (6.602)

透過「彙總報表」，您可以在 Urchin 的「首頁」上查看多個設定檔的成效。 「首頁」提供三種檢視： "「基本」"、「管理員」以及「公司」檢視；您可以在 [檢視] 下拉式選單中進行選取。

按一下 [編輯] 按鈕即可瀏覽 [設定檔設定/報表] 分頁。 「狀態」欄會顯示設定檔目前的狀態。 此狀態與選定的日期範圍無關。 已篩選的設定檔旁邊會顯示「篩選後」圖示 ( ) 。 不含單次點擊出價來源的設定檔會顯示「廣告帳戶」圖示 ( ) 。 按一下 [廣告帳戶] 圖示即可前往 AdWords。 如果單次點擊出價來源已經與設定檔建立關聯，「廣告帳戶」欄會顯示單次點擊出價來源的數量。 若要查看報表，請按一下設定檔名稱。 若要瀏覽實際網站，只要按一下設定檔名稱旁邊的連結即可。

自 Urchin 6.602 起，您可以在 Urchin 組態資料庫中透過 uglobals.uigl_rollup_view_type 和 uusers.uius_rollup_view_type 欄位手動設定每位使用者的預設「首頁」檢視 (RollUp 報表)：

  • uglobals.uigl_rollup_view_type 的值如下：
    • 0： 自動
      • 此為資料庫中的預設值。 在此情況下，檢視類型的全域值將取決於使用者類型：

        • 超級管理員、帳戶管理員 > 管理員檢視

          使用者、使用者 - 無更新 > 公司檢視

    • 1： 基本檢視
    • 2： 管理員檢視
    • 3： 公司檢視
  • uusers.view_type：
    • -1： 使用全域值
      • 新建立的使用者都會使用此預設值。 此設定會使用全域彙總檢視類型 (uglobals.uigl_rollup_view_type) 做為該使用者的檢視類型。
    • 0： 自動
      • 在此情況下，使用者的檢視類型將取決於使用者類型：

        • 超級管理員、帳戶管理員 > 管理員檢視

          使用者、使用者 - 無更新 > 公司檢視

    • 1： 基本檢視
    • 2： 管理員檢視
    • 3： 公司檢視

重要使用須知

  • 嘗試登入包含大量資料的帳戶時，可能會發生下列錯誤： 

    工作階段錯誤 - 無法呼叫預設常式。 (2002) 請再次登入。
  • 這是由於彙總報表的資料擷取作業比預設 APPLICATION_TIMEOUT 超出 45 秒所致。 建議的解決方式如下：
  • 將 XYZ 檔案中的 APPLICATION_TIMEOUT 提高為適當的值，即可解決此問題。 不過請注意，這項變更可能會對其他方面造成影響。
  • 將 uglobals.uigl_rollup_view_type 和 Urchin 組態資料庫中的所有 uusers.uius_rollup_view_type 欄位值變更為 '1'，即可將檢視類型變更為"「基本檢視」"。
