Working with Profiles

Overview

A profile is a set of reports for one website. It is necessary to configure at least one profile per site. In many circumstances, you'll want to set up more than one profile per site, but generally speaking, there is a 1:1 ratio of profiles to sites.

More technically speaking, a profile is a set of rules governing the processing of log files and the creation of reports for one website or parts thereof.

Profiles are the primary organizational tools of the Urchin system, so it's important to understand how they work.

Licensing Information

The 'regular' license for Urchin Software includes 1000 reporting profiles and 3000 log sources per instance. Profiles can share a Log Source (useful when one Profile is the "master", and one or more others have Filters applied).

The 'demo' license for Urchin Software includes 5 reporting profiles, 5 log sources and only allows 1 log source per profile.

Every website must have at least one Profile associated with it to obtain reporting. Usually one is sufficient, but there is no limit to how many can be created for specialized analysis of a single web property.

How to Use Profiles

To create a Profile, login to the Urchin administration interface as an Urchin admin and click on the Configuration link on the top right area of the screen. You will be taken to the Add Profile Wizard. This is a simple series of steps is designed to help you get the Profile set up in basic form quickly and easily. Each screen in the Wizard has explicit help information.

Recommendations

  • In order to obtain the optimum reporting quality from Urchin, it is important to enable the Urchin Traffic Monitor (UTM), particularly if unique visitor information is desired. There are three basic steps required to use the UTM:
    1. Under the Settings tab of any Profile, select "Urchin Traffic Monitor (UTM)" as the Visitor Tracking Method.
    2. Install the UTM sensor on each page that you want tracked with the highest accuracy. (If you do not use the UTM, pages will still be tracked, however, some actions such as reloads and use of the Back button will not be reported.)
    3. Modify your webserver's logging behavior to activate cookies.
  • If you intend to set up one or more Filters in conjunction with your profile, it is advisable to have more than one profile for that website or part thereof. We recommend having one profile that is the "master" -- it contains everything. If you wish, for example, to filter out spiders or robots, it's a good idea to put these filters in a second profile. This will allow you to compare the results of the filtered data profile to the master profile.
