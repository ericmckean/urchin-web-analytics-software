How To Create Custom Entries for a Language

  1. Copy the language file xx-XX.txt from folder <Urchin>/lib/admin/languages/ to <Urchin>/lib/custom/languages/.

  2. Append new entries to the end of the file and assign each entry a new number.

  3. The separator between the number and value must be a <Tab> or Urchin will not parse it.

  4. Ensure that the entry numbers are consecutive (incremented by one).
  5. Reload Urchin.

  6. After reload, Urchin should create a language DB file named xx-XX.db located in folder <Urchin>/lib/custom/languages/.

Once defined, custom entries can be used in the reporting section as column labels, titles or help descriptions.

Example of a language file

Entries 738, 739, and 740 are added custom entries. 

  ...
  731     Executive
  732     Marketer
  733     Webmaster
  734     Dashboards
  735     All Reports
  736     ja-jp-mac (Japanese)
  737     ja-jp-osaka (Japanese)
  738     Modules
  739     Actions
  740     Modules / Actions

Important Note: Upgrading Urchin may replace the original language files and add default entries. Please be sure to recreate your custom language files and update custom reports.
