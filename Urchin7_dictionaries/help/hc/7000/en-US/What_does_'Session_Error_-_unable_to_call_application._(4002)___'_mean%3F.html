What does 'Session Error - unable to call application. (4002)...' mean?

When you view Urchin reports, the browser queries the Urchin database files for the data that matches your requested date range and report view. If the reponse to the data query is not recevied within the default 45 seconds, a time out will occur and an error will be displayed; ""Session Error - unable to call application. (4002) Please log-in again." When this happens, it will be necessary to increase the time out value accordingly. In order to increase that value, you must have command line access to the Urchin distribution files. If you do not, please contact the person who provides your reports and refer them to this article.

To increase the Application Timeout:

  • Open a command shell to the Urchin server
  • Open the file: /path/to/urchin/etc/session.conf

  • Find the directive named: APPLICATION_TIMEOUT:45 where the value is represented in seconds

  • Increase the value accordingly and save the file. The changes will take effect immediately
